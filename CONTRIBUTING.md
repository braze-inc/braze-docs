# Contributing to Braze Docs

This repository is **private**. You need access to the `braze-inc/braze-docs` repository on GitHub to clone it, open pull requests, and use the links below.

## Handbook

Authoring workflows, local setup, style guides, and tooling live in the in-repo handbook:

**[docs/contributing/README.md](docs/contributing/README.md)**

## Expectations

- Follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
- Use branches and pull requests as your team’s process defines (for example, feature branches off the default integration branch).
- For customer-facing documentation content, follow the style guides linked from the handbook.

## Issues

- [Report an issue](https://github.com/braze-inc/braze-docs/issues/new?assignees=&labels=issue&projects=&template=report_an_issue.md&title=)
- [Request a feature](https://github.com/braze-inc/braze-docs/issues/new?assignees=&labels=enhancement&projects=&template=request_a_feature.md&title=)
