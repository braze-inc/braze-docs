# リッチ・コミュニケーション・サービス（RCS）について

> リッチ・コミュニケーション・サービス（RCS）は、従来のSMSを強化するもので、ブランドは情報提供だけでなく、はるかにエンゲージメントの高いメッセージを配信することができる。現在、Android と iOS の両方でサポートされている RCS では、高品質のメディア、インタラクティブなボタン、ブランド化された送信者プロファイルなどの機能がユーザーのプリインストール済みメッセージングアプリで直接使用可能になるため、別のアプリをダウンロードする必要がなくなります。

サードパーティのメッセージング・アプリケーションとは異なり、RCSはネイティブのメッセージング環境（AppleメッセージやGoogleメッセージ）を活用するため、カスタマーエクスペリエンスの枠を超え、リッチでインタラクティブなコミュニケーションを可能にする。 

## RCS を使用するメリット

- **リッチなカスタマーエクスペリエンス:**テキスト、ビジュアル、インタラクティブ要素をシームレスに統合し、エンゲージメントを高め、パーソナライズされたデータドリブン型のキャンペーンを実現できるように準備することで、より充実したユーザーエクスペリエンスを提供します。
- **信頼性が高くブランド化されたインタラクション:**ブランド資産をアピールするだけでなく、業界トップのプライバシー基準に準拠し、顧客の信頼とロイヤルティを高める認証済み送信者IDを通じて、信頼されたブランド交流を実現する。
- **柔軟なメッセージング配信:**一貫したユーザーエクスペリエンスを維持しながら、デバイスの機能に関係なく、すべてのオーディエンスセグメントにリーチするシームレスな SMS フォールバックにより、柔軟で信頼性の高いメッセージング配信が促進されます。
- **実用的なインサイト:**重要な KPI を追跡する高度なレポートにより、実用的なインサイトを引き出し、リアルタイムでキャンペーンを最適化し、測定可能な成功を促進します。
- **オムニチャネルの相乗効果:**包括的なマーケティング戦略に RCS をシームレスに統合し、一貫性のあるクロスチャネルのカスタマーエクスペリエンスを提供することで、キャンペーンの効果と全体的な ROI を高めます。

## ユースケース

| ユースケース | 説明 |
| --- | --- |
| インタラクティブな製品プロモーション | 魅力的な画像, 写真や短い動画と詳細な製品ドキュメントを組み合わせることで、製品プロモーションに命を吹き込む。おすすめの返信 (「カートに追加」や「詳細」など) と openURL アクションを活用して、リッチなメッセージ内エクスペリエンスでの迅速な製品探索とコンバージョンを促進します。 |
| パーソナライズされたロイヤルティとリワードの最新情報 | 高品質なビジュアルとリワードの詳細情報を使ってパーソナライズされたロイヤルティメッセージを配信します。提案された返信先（「今すぐ利用する」や「キャンペーンを見る」など）やopenURLアクションを使用して、インタラクティブなカスタマージャーニーを作成し、各更新を視覚的に魅力的なものにし、即座にエンゲージメントを促し、カスタマーリテンションを向上させるように調整する。 |
| 安全な取引とアカウントアラート | PDF の領収やドキュメントの画像を組み込み、セキュアアカウントアラートと取引通知を配信します。おすすめのアクション (「今すぐレビューする」や「サポートに連絡する」など) と openURL リンクにより、顧客はより詳細な情報に迅速にアクセスすることや、セキュリティステップを開始することができ、すべてのインタラクションにおいて信頼性と信用の両方を強化できまする。 |
| 旅程表と予約の強化 | 視覚的に豊かな旅程表、送信先ガイド、搭乗券を送信することで、旅行体験を向上させる。openURLアクションにより、顧客はメッセージングウィンドウを離れることなく、予約の変更やリアルタイムの更新（スケジュール変更など）に素早くアクセスすることができ、最初から最後までスムーズで魅力的なトラベルジャーニーを促進することができる。 |
| 顧客フィードバックとインタラクティブなアンケート | リッチメディアコンテンツとテキストを組み合わせたインタラクティブなアンケートを導入し、実用的なフィードバックを収集します。迅速に回答できるようにするためおすすめの返信と、より包括的な調査フォームにアクセスするための openURL アクションを統合することで、顧客が簡単に意見を共有できるようになり、マーケターがあらゆる業種からのリアルタイムのフィードバックに基づいて戦略を改善できるようにります。 |

## 要件

| 要件 | 説明 |
| --- | --- |
| メッセージクレジット | Braze アカウントマネージャーに連絡し、契約時にメッセージクレジットを購入していることを確認します。メッセージクレジットは、SMS、MMS、RCS、WhatsAppなど様々なチャネルでメッセージングボリュームを購入し、割り当てることができる柔軟な契約項目である。 |
| 対象国 | Braze が対応している国のユーザーに RCS を送信することを確認してください。対応している国は次のとおりです:アメリカ、イギリス、ドイツ、メキシコ、スウェーデン、スペイン、シンガポール、ブラジル、フランス、イタリア、コロンビア |
| RCS認証送信者 | 受信者が自分のデバイスでメッセージの送信元を特定するために確認する送信者。RCS 検証済み送信者は、会社名、ビジュアルブランディング、認証バッジで構成されています。<br><br> Braze は、対象地域における RCS 検証済み送信者の申請と登録を支援します。Braze の担当者にいくつかの基本情報を提出する必要があります。 |
| 電話番号付きユーザー一覧 | メールの送信を始めるには、あらかじめユーザーsを登録しておく必要があります。さらに、オーディエンスのおおよその規模を把握しておく必要があります。ユーザーや電話番号は、いくつかの異なる方法でBrazeに追加することができる。電話番号は 10 桁の数字で記述し、国コードも指定します。詳細については、「[ユーザーの電話番号]({{site.baseurl}}/user_phone_numbers/)」を参照してください。 |
| キーワードと応答 | メッセージングを開始する前に、すべての基本キーワードに応答が属性付けられている必要があります。Braze はオプトインキーワード、オプトアウトキーワード、ヘルプキーワードを自動的に処理します。カスタマイズオプションや追加のキーワード応答設定も利用できます。詳細については、「[オプトインおよびオプトアウトキーワード]({{site.baseurl}}/optin_optout/)」を参照してください。 |

## 知っておくべき用語

| 用語 | 定義 |
|----|----|
| 購読グループ | 特定のメッセージングユースケースにサブスクライバーしているユーザーのグループ。各サブスクリプショングループは、1つまたは複数のブランドの「送信者」に関連付けられています。送信者とは、RCS 検証済み送信者または SMS コード、あるいはこの両方です。例えば、トランザクションとプロモーションの両方の RCS メッセージを送信する予定がある場合、Braze ダッシュボードで別々の RCS 検証済み送信者からなる2つのサブスクリプショングループを設定できます。 |
| RCS認証済み送信者 | RCSメッセージの送信側エンティティ、またはRCSメッセージの受信者が、そのメッセージがどこから来たのかを識別するためにデバイス上で見るもの。RCSで認証された送信者には、会社名、キャプション、ビジュアルブランディング、認証バッジが含まれる。必要な RCS 送信者登録情報を Braze に提供した後で、Braze が登録とサブスクリプショングループの設定を行います。 |
| SMS フォールバック | RCS でメッセージを配信できない場合（例えば当該地域で通信事業者がサポートしていない場合など)、サブスクリプショングループ内に SMS コードが存在していれば、Braze は SMS によるメッセージ配信を試みます。 |
| 基本的なRCS | 160文字までのテキストのみのメッセージング。シングルメッセージとして請求される。<br><br> このカテゴリーはグローバルモデルでのみ使用される。 |
| シングルRCS | 160文字を超えるテキストのみの**メッセージや、**ボタンやメディアなどのリッチ要素を含むメッセージ。<br><br>このカテゴリーはグローバルモデルでのみ使用される。 |
| リッチ | テキストのみのメッセージで、限定的な提案やボタンの有無は問わない。セグメンテーション（160 UTF-8バイト）ごとに課金される。たとえば、161文字のプレーンテキストのメッセージは、2つのセグメンテーションとなる。<br><br> このカテゴリーは米国モデルでのみ使用される。 |
| リッチメディア | メディアファイル（画像、写真）またはリッチカードを含むメッセージング。メッセージの長さに関係なく、1つのメッセージとして課金される。<br><br> このカテゴリーは米国モデルでのみ使用される。 |
