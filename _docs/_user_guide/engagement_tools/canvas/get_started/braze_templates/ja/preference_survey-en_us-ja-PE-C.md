# オンボーディングと好みの調査

> 「オンボーディングと好みの調査」テンプレートを使用して、新しいユーザーをターゲットとするガイド付きオンボーディングワークフローを作成します。あなたのブランドを紹介し、彼らが使い始めるのを手助けし、彼らの好みを収集して、彼らの長期的なエンゲージメントを維持します。

この記事では、**環境設定サーベイ**テンプレートを使用したオンボーディングのユースケースについて説明します。これは、ユーザライフサイクルの検討段階用に設計されています。完了すると、セッションを開始したとき、およびオンボーディングが完了していないときに、ユーザーにメールとアプリ内メッセージを送信するキャンバスが作成されます。

## 前提条件

このテンプレートを正常に使用するには、次のものが必要です。

- オンボーディングを開始するようにユーザーに促すウェルカムメール。
- オンボーディングしたユーザー向けにアプリの使用を開始するためのヒントを含むフォローアップメール。
- ユーザーにオンボーディングを完了するように促すフォローアップ電子メール。
- ユーザ設定を決定するための複数の質問を含む[survey]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/traditional/templates/simple_survey)。

## テンプレートをニーズに合わせる

私たちは、行く必要のある場所に人をもらうオンデマンドのライドシェアアプリ、StyleRydeで働いているとしよう。キャンバスを作成する前に、ユーザーがアプリを初めて使用したときのエクスペリエンスと印象を判断するための、一連の魅力的な質問を含む[簡単な調査]({{site.baseurl}}/user_guide/data/activation/catalogs/catalog)を設定しました。

テンプレートにアクセスするには、新しいキャンバスを作成するときに、**キャンバステンプレートを使用**> **ブレーズテンプレート**を選択します。次に、**オンボーディングと好みの調査**の横にある**テンプレートを適用**を選択します。あとは、テンプレートを見ながら、自分たちのニーズに合わせていくだけだ。

### ステップ1:詳細を設定する

キャンバスの詳細を調整して、目標を反映しましょう。

1. テンプレート名の横にある**編集**を選択します。

]({% image_buster /assets/img/canvas_templates/preference_survey1.png %}){: style="max-width:50%;"}() キャンバスの現在のタイトルと説明。

{:start="2"}
2\.キャンバス名を更新し、アプリを初めて使うユーザーをターゲットにしたキャンバスであることを指定します。
3\.このキャンバスにパーソナライズされたメッセージングが含まれていることを説明する説明を更新します。
4\.**Onboarding** というタグを追加し、キャンバスのホームページでフィルターできるようにします。

]({% image_buster /assets/img/canvas_templates/preference_survey2.png %})キャンバスの新しい名前、説明、タグ。{: style="max-width:90%;"}()

### ステップ 2: 変換イベントの割り当て

**1次コンバージョンイベント - A** を**カスタムイベントを実行**に更新します。次に、カスタムイベントに**Last Used App** を選択します。

]({% image_buster /assets/img/canvas_templates/preference_survey3.png %})コンバージョンイベント用にカスタムイベント名として選択した、[最後に使用したアプリケーション]。{: style="max-width:90%;"}()

### ステップ 3: エントリスケジュールを調整する

ユーザーがアプリでセッションを開始したときにキャンバスに入るように、エントリスケジュールを**アクションベース**にしておいてください。こうすることで、タイムリーなエンゲージメントで関係を築き始めることができます。

ここでは、**入力ウィンドウ**を希望の日時に調整することで、このセクションの更新を行います。

]({% image_buster /assets/img/canvas_templates/preference_survey4.png %})開始時刻が2025年1月30日午後12時に設定されている「エントリ期間」セクション。{: style="max-width:90%;"}

### ステップ 4: ターゲットオーディエンスを選択する

1日前に初めてStyleRydeアプリを使用したユーザーをターゲットにするために、ターゲット視聴者をそのまま維持します。

]({% image_buster /assets/img/canvas_templates/preference_survey5.png %})エントリのオーディエンスをターゲットにするために選択された 「最初にこれらのアプリを使用したのは1日前未満」フィルター。{: style="max-width:90%;"}()

### ステップ 5: 送信設定を選択する

デフォルトのサブスクリプション設定を維持します。これにより、サイレント時間をオンにした状態でサブスクリプション登録したユーザーおよびメッセージまたは通知の受信を選択したユーザーのみに送信されるようになります。その他の設定 (フリークエンシーキャップ、シードグループ) は省略します。

]({% image_buster /assets/img/canvas_templates/preference_survey6.png %})「送信設定」セクションに、サイレント時間が午前12時から午後8時までの間にオンになっている、購読またはオプトインしたユーザーの購読設定が記載されています。{: style="max-width:90%;"}()

### ステップ6: キャンバスをカスタマイズする

ここでは、ユーザーに送信するコンテンツをカスタマイズして、Canvas を構築します。 

1. 最初のメッセージステップ**Welcome Email** では、StyleRyde ウェルカムメールを含めるようにこのステップを更新します。
2. 次に、アクションパスステップをそのまま維持します。このステップでは、3 日間のウィンドウでユーザーを2 つのグループに分割します。

- セッションを開始したか、オンボーディングメールをクリックしたユーザー
- セッションを開始していないユーザ、またはオンボーディングメールをクリックしていないユーザ

]({% image_buster /assets/img/canvas_templates/preference_survey8.png %})アクションパスステップは2つのパスに分割され、1つはセッションを開始したユーザー用、もう1つは他のすべてのユーザー用です。{: style="max-width:50%;"}()

ここから、前述のグループに基づいてユーザーとメッセージングをターゲットにします。

#### エンゲージメントの高いユーザーをターゲットにする

セッションを開始した、または最初のメッセージステップからオンボーディングメールに参加したユーザーのために、** Getting Started Tips** メッセージステップを更新し、新しいStyleRyde ユーザーのための重要な旅行と安全に関するヒントを含めます。

ユーザーがオンボーディングを完了すると、キャンバスを終了します。

次に、**コンテンツの好みについての調査**メッセージステップを更新して、ユーザーが今後情報を受け取ることに関心のあるトピックを選択するように求める好みに関する調査を含めます。

]({% image_buster /assets/img/canvas_templates/preference_survey7.png %})適用されるすべての関心を選択するようユーザーに求める好みに関する調査のプレビュー。{: style="max-width:90%;"}()

#### オンボーディングを開始していないユーザーを誘導する 

他のユーザーの場合は、**Winback Nudge** メッセージステップをフォローアップメールで更新し、ユーザーにオンボーディングを完了するように促します。

再エンゲージメントの最後のステップとして、**ステップ2**の名前を **Final Winback Nudge** に変更し、新しいユーザーにオンボーディングの完了を促すアプリ内メッセージでステップを更新します。

### ステップ 7:キャンバスをテストして起動する

キャンバスをテストし、期待通りに動作することを確認したら、[**キャンバスを開始**] を選択してキャンバスを開始します。
