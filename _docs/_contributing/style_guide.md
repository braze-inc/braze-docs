# Braze Docs style guide

Refer to the following style guides when contributing to Braze Docs.

- [Braze Docs Style Guide]({{site.baseurl}}/contributing/style_guide/writing_style_guide/) — Voice and tone, accessibility, inclusive language, language and grammar, punctuation, technical documentation, formatting and organizing, linking, and glossary
- [Image copy style guide]({{site.baseurl}}/contributing/style_guide/image_style_guide/) — Image styling tools, cropping, censorship, and emphasizing components
- [Alerts (best practices)]({{site.baseurl}}/contributing/style_guide/alerts/) — When and how to use Important, Note, Tip, and Warning alerts
- [API endpoint documentation guidelines]({{site.baseurl}}/contributing/style_guide/api_endpoint_guidelines/) — How to create and format API endpoint articles
