# 차단 목록과 삭제의 차이점

Braze에서 속성을 차단 목록에 추가하는 것과 삭제하는 것의 차이를 이해하기 위해 각 작업의 결과를 검토해 보겠습니다:

- **차단 목록:** 커스텀 속성, 이벤트 또는 구매가 차단 목록에 있는 경우 해당 속성은 고객 프로필에 남아 있지만 해당 속성에 대한 새로운 요청은 처리되지 않습니다.
- **삭제:** 커스텀 속성, 이벤트 또는 구매가 삭제되면 데이터가 제거됩니다. 그러나 Braze는 해당 속성이 여전히 SDK를 통해 추적되거나 API 또는 CSV를 통해 업로드되는 경우 해당 속성에 대한 새로운 수신 요청을 계속 수락합니다.

## 어떻게 해야 하나요?

차단 목록을 작성하려면 Braze는 차단 목록 정보를 각 사용자의 기기로 전송해야 하며, 이는 데이터 집약적인 작업으로, Braze는 이를 피하고자 합니다. 또한 목록이 너무 많으면(속성, 이벤트 또는 구매가 100개 이상) 앱 속도가 느려질 수 있습니다. 

더 이상 Braze에 속성을 전송할 계획이 없다면 삭제 경로를 사용하는 것이 좋습니다.

경로에 관계없이 제거하려는 사용자 지정 속성, 이벤트 및 구매는 더 이상 **워크스페이스 관리** 페이지에 표시되지 않으며, 세그먼트 필터로 제거됩니다. 사용자 수준 데이터는 프로필에 그대로 유지됩니다.
