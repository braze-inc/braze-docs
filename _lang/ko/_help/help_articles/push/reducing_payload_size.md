# 푸시 알림 페이로드 크기 줄이기

푸시 페이로드가 너무 커서 푸시 캠페인이나 캔버스 단계를 시작할 수 없는 경우 푸시 알림 페이로드 크기를 줄이기 위한 다음 팁을 확인하세요.

{% alert note %}
최대 페이로드 크기는 **3,807바이트입니다**. 푸시 크기가 이 크기를 초과하면 메시지가 전송되지 않을 수 있습니다. 페이로드를 수백 바이트로 유지하는 것이 가장 좋습니다.
{% endalert %}

## 푸시 페이로드란 무엇인가요?

푸시 서비스 제공업체는 전체 푸시 페이로드의 바이트 크기를 보고 사용자에게 푸시 알림을 표시할 수 있는지 여부를 계산합니다. 페이로드는 다음을 포함한 대부분의 푸시 서비스에서 **4KB(4,096바이트)**로 제한됩니다.

- Apple 푸시 알림 서비스(APN)
- Android의 Firebase Cloud Messaging(FCM)
- 웹 푸시
- Huawei 푸시

이러한 푸시 서비스는 이 한도를 초과하는 모든 알림을 거부합니다.

Braze는 통합 및 분석 목적으로 푸시 페이로드의 일부를 보유합니다. 이를 감안할 때 최대 페이로드 크기는 **3,807바이트**입니다. 푸시 크기가 이 크기를 초과하면 메시지가 전송되지 않을 수 있습니다. 페이로드를 수백 바이트로 유지하는 것이 가장 좋습니다.

푸시의 다음 요소는 푸시 페이로드를 구성합니다:

- 제목 및 메시지 본문과 같은 복사본
- Liquid 개인화의 최종 렌더링
- 이미지의 URL(이미지 자체의 크기는 제외)
- 클릭 타겟의 URL
- 버튼 이름
- 키-값 쌍

## 페이로드 크기를 줄이기 위한 팁

페이로드 크기를 줄이려면:

- 메시지를 간결하게 작성하세요. 일반적인 가이드라인은 40자 이내로 실행 가능하고 유익한 내용을 작성하는 것입니다.
- 사본에서 공백과 줄 바꿈은 생략하세요.
- Liquid가 전송 시 렌더링하는 방식을 고려하세요. Liquid 개인화의 최종 렌더링은 사용자마다 다르므로, Braze는 Liquid가 포함된 경우 푸시 페이로드가 크기 제한을 초과할지 여부를 판단할 수 없습니다. Liquid에서 더 짧은 메시지를 렌더링해도 괜찮을 수 있습니다. 그러나 Liquid로 인해 메시지가 길어지면 푸시가 페이로드 크기 제한을 초과할 수 있습니다. 사용자에게 푸시 메시지를 보내기 전에 항상 실제 디바이스에서 테스트하세요.
- URL 단축기를 사용하여 URL을 단축하는 것을 고려하세요.
