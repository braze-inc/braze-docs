# 2025년 11월 11일 출시

## 데이터 유연성

### `Live Activities Push to Start Registered for App` 세분화 필터

이[`Live Activities Push to Start Registered for App`]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters#live-activites-push-to-start-registered-for-app)[필터는]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters#live-activites-push-to-start-registered-for-app) 사용자가 특정 앱에 대해 iOS 푸시 알림을 통해 라이브 활동을 시작하도록 등록되었는지 여부에 따라 사용자를 세그먼트로 분류합니다.

### RFM SQL 세그먼트 확장

구매 습관을 측정하여 최고의 고객을 대상으로 타겟팅하기 위해 [RFM(최근성, 빈도, 금액) 세그먼트 확장을]({{site.baseurl}}/rfm_segments/) 생성할 수 있습니다.

RFM 분석은 마케팅 기법으로, 각 범주(최근성, 빈도, 구매 금액)별로 사용자에게 0에서 3점까지 점수를 부여하여 최상의 고객을 식별합니다. 여기서 3점이 최고 점수이며 0점이 최악의 점수입니다. 최근성, 빈도성 및 금전적 가치는 모두 귀하가 선택한 특정 시간 범위의 데이터를 기반으로 합니다.

### 커스텀 속성 — 값 

사용 현황 보고서를 볼 때 [**값** 탭을]({{site.baseurl}}/user_guide/data/activation/custom_data/custom_attributes/#values-tab) 선택하면 약 250,000명의 사용자 샘플을 기준으로 선택한 커스텀 속성의 상위 값을 확인할 수 있습니다.

### 클라우드 데이터 수집을 위한 로그 및 가시성 동기화

{% multi_lang_include release_type.md release="General availability" %}

클라우드 데이터 인제스트(CDI) [동기화 로그 대시보드를]({{site.baseurl}}/user_guide/data/unification/cloud_ingestion/sync_logs/) 통해 CDI에서 처리된 모든 데이터를 모니터링하고, 데이터가 성공적으로 동기화되었는지 확인하며, "잘못된" 데이터나 누락된 데이터와 관련된 문제를 진단할 수 있습니다.

### 다중 규칙 기능 플래그 롤아웃

[다중 규칙 기능 플래그 롤아웃을]({{site.baseurl}}/developer_guide/feature_flags/create/#multi-rule-feature-flag-rollouts) 사용하여 사용자 평가를 위한 규칙 시퀀스를 정의함으로써, 정밀한 세분화와 제어된 기능 출시를 가능하게 합니다. 이 방법은 동일한 기능을 다양한 오디언스에게 배포하는 데 이상적입니다.

### 드래그 앤 드롭 제품 블록을 위한 카탈로그 필드 매핑

카탈로그 설정에서 **제품** 토글을 선택하여 카탈로그의 [특정 필드]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/product_blocks/#catalog-setup) 및 정보[와 매핑할]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/product_blocks/#catalog-setup) 수 있습니다. 이를 통해 제품 제목, 제품 URL 및 이미지 URL로 사용할 필드를 선택할 수 있습니다.

### 커런츠에서 최대 게재빈도 설정 중단 이벤트

커런츠를 사용할 때 채널 중단 이벤트에서  를 `abort_type`참조할 수 있습니다. 이것은 메시지가 최대 게재빈도 설정으로 인해 중단되었음을 식별하며, 중단을 유발한 최대 게재빈도 설정 규칙을 포함합니다. 이는 최대 게재빈도 설정 규칙을 설정하는 방법에 대한 정보를 제공하는 데 도움이 됩니다. [메시지 참여 이벤트를]({{site.baseurl}}/user_guide/data/distribution/braze_currents/event_glossary/message_engagement_events) 참조하여 특정 커런츠 이벤트 세부 정보를 확인하십시오.

### 최적화된 전달 기능이 적용된 WhatsApp 메시지

메타의 고급 AI 시스템을 활용하여 마케팅 메시지를 가장 높은 참여 가능성을 가진 더 많은 사용자에게 전달함으로써, 전달 가능성 및 메시지 참여도를 크게 향상시키세요.

[최적화된 전달 기능을 갖춘 WhatsApp 메시지는]({{site.baseurl}}/user_guide/message_building_by_channel/whatsapp/whatsapp_campaign/optimized_delivery/) 기존 클라우드 API 대비 우수한 성능을 제공하는 메타의 새로운 [마케팅 메시지 라이트 API를](https://developers.facebook.com/docs/whatsapp/marketing-messages-lite-api/) 통해 전송됩니다. 이 새로운 발송 파이프라인은 귀하의 메시지를 소중히 여기고 수신하기를 원하는 사용자에게 더 효과적으로 도달할 수 있도록 지원합니다.

### WhatsApp Flows

WhatsApp Flow 메시지를 Braze 캔버스 또는 캠페인에 통합할 때, 사용자가 Flow를 통해 제출하는 특정 정보를 수집하고 활용하고자 할 수 있습니다. Braze는 필요한 중첩 고객 속성(NCA) 스키마를 생성하기 위해 사용자 응답 구조, 특히 예상되는 JSON 응답 형식에 관한 추가 정보를 수신해야 합니다.

이제 [플로우 응답을 커스텀 속성으로 저장하고]({{site.baseurl}}/user_guide/message_building_by_channel/whatsapp/whatsapp_campaign/whatsapp_flows/?tab=recommended%20method#step-1-generate-the-flow-custom-attribute) 테스트 발송을 완료하여 응답 구조에 대한 정보를 Braze에 제공할 수 있습니다.

## SDK 업데이트

다음 SDK 업데이트가 릴리스되었습니다. 주요 업데이트는 아래에 나열되어 있으며, 그 외의 모든 업데이트는 해당 SDK 체인지로그를 확인하면 확인할 수 있습니다.

- [React Native 소프트웨어 개발 키트 18.0.0](https://github.com/braze-inc/braze-react-native-sdk/blob/16.1.0/CHANGELOG.md)
    - 의 `subscribeToInAppMessage``addListener`콜백에 대한 Typescript`Braze.Events.IN_APP_MESSAGE_RECEIVED` 타입을 수정합니다.
        - 이제 이 리스너들은 새로운`InAppMessageEvent`유형으로 콜백을 올바르게 반환합니다. 이전에는 메서드들이  `BrazeInAppMessage`타입을 반환하도록 주석 처리되었지만, 실제로는  타입을 반환하고 `String`있었습니다.
         - 구독 API 중 하나를 사용 중인 경우, 이 버전으로 업데이트한 후에도 인앱 메시지의 동작이 변경되지 않도록 확인하십시오. .`BrazeProject.tsx`에서 우리의 샘플 코드를 참조하십시오.
    - API `logInAppMessageClicked`, `logInAppMessageImpression`, 및`logInAppMessageButtonClicked`  는 기존 공개 인터페이스와 일치하도록  `BrazeInAppMessage`객체만 허용합니다.
        - 이전에는 `BrazeInAppMessage`객체와 모두 `String`허용했습니다.
    - `BrazeInAppMessage.toString()` 이제 JSON 문자열 표현 대신 사람이 읽을 수 있는 문자열을 반환합니다.
        - 인앱 메시지의 JSON 문자열 표현을 얻으려면 다음을 사용하십시오`BrazeInAppMessage.inAppMessageJsonString`.

        - 이 새로운 메서드는 이제 인스턴스 메서드가 아닌 클래스 메서드입니다.
    - 메서드에`BrazeReactUtils` null 가능성 주석을 추가합니다.
