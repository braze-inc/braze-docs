# Peak

> 의사 결정 인텔리전스 플랫폼인 [Peak](https://platform.peak.ai/)는 성과가 입증된 시스템으로, 의사 결정 인텔리전스는 상용 인공지능 애플리케이션에 기반하여 의사 결정을 강화해 매출과 이익을 증대합니다.

Braze와 Peak 간의 파트너십을 통해 예측된 고객이탈 확률과 고객 행동 및 상호작용을 기반으로 한 속성을 가져와 Braze에 가져와 고객 세분화 및 타겟팅에 사용할 수 있습니다. 

## 전제 조건

시작점으로서 Peak 테넌트는 Peak와 Braze 간의 통합을 호스팅해야 합니다. 전통적으로 Peak 고객의 온보딩 중에 생성됩니다. 또한 이후 Braze에 통합되는 인공지능 기반 출력을 생성하므로 초반에 의사 결정 인텔리전스 솔루션이 필요합니다.

| 요구 사항 | 설명 |
| ----------- | ----------- |
| 피크 테넌트 | 통합을 호스팅하고 조율하려면 테넌트라고 하는 Peak 플랫폼의 인스턴스가 필요합니다. |
| 의사 결정 인텔리전스 솔루션 | Peak와 Braze 간 통합은 인공지능 기반 출력에 기반하므로, Peak 또는 고객이 배포한 솔루션이 테넌트에 있어야 합니다. |
| Braze REST API 키 | `users.track` 권한이 있는 Braze REST API 키. <br><br>Braze 대시보드의 **설정** > **API 키**에서 생성할 수 있습니다. |

## 통합

피크 솔루션 고객 인텔리전스는 고객 행동 및 상호작용을 기반으로 미래 지향적인 속성 범위를 예측하는 모델을 활용합니다. 이러한 속성은 Peak에 저장되며 고객이탈 확률을 포함하여 예측 세분화를 생성하는 데 사용할 수 있습니다. 이러한 예측 속성의 업데이트는 구성 가능한 주기(일별 또는 주별)를 기반으로 합니다.

### 1단계: 모델을 실행하고 고객을 추출합니다

통합은 AI 모델 실행 및 예측 고객 속성 재계산의 결과에 따라 역으로 트리거됩니다. 이 AI 출력은 속성이 새 상태 또는 값으로 업데이트될 때를 포함하여 Peak에 저장됩니다.

속성이 업데이트된 시점을 기준으로 Peak과 Braze 간의 마지막 동기화 이후 업데이트된 예측 속성을 가진 모든 고객을 수집하기 위해 선택이 수행됩니다.

### 2단계: 업데이트 Braze

업데이트된 고객 및 관련 속성을 사용하여 Peak은 [`/user/track` 엔드포인트][1]를 사용하여 Braze에 이를 POST하고 [bulk]({{site.baseurl}}/api/endpoints/user_data/post_user_track/#making-bulk-updates) 헤더를 사용합니다.

API로부터 성공 상태 코드를 수신하면 Peak는 Peak와 Braze 간 성공적인 동기화를 기록합니다.

### 3단계: 이 통합 사용

Peak과 Braze 간 동기화가 성공하면 업데이트된 사용자에 새 속성이 포함됩니다. 캠페인 및 캔버스에서 이러한 속성을 사용하여 사용자를 타겟팅하고 메시지를 개인화하세요.
