# 소프트웨어 개발 키트 개요 

> Braze SDK를 사용하면 웹사이트나 앱을 통해 세션 데이터를 수집하고, 사용자를 식별하고, 구매 및 커스텀 이벤트를 쉽게 기록할 수 있습니다. 또한 소프트웨어 개발 키트를 사용하여 Braze 대시보드에서 직접 인앱 메시지와 푸시 알림을 전송하여 사용자의 참여를 유도할 수 있습니다.

간단히 말해, Braze 소프트웨어 개발 키트입니다:
* 사용자 데이터를 수집하여 통합된 고객 프로필로 동기화합니다.
* 고객 마케팅 참여 데이터와 비즈니스에 특화된 커스텀 데이터를 캡처합니다.
* 푸시 알림, 인앱 메시지, 콘텐츠 카드 메시징 채널 지원

## 소프트웨어 개발 키트란 무엇인가요?
소프트웨어 개발 키트(SDK)는 디지털 애플리케이션에 추가하여 새로운 기능을 지원할 수 있는 미리 만들어진 도구 세트(작은 코드 블록)입니다. Braze 소프트웨어 개발 키트는 앱 또는 사이트와 정보를 주고받는 데 사용됩니다. 고객 프로필 생성, 커스텀 이벤트 로깅, 푸시 알림 트리거 등 필수 기능을 처음부터 바로 제공하도록 설계되었습니다. 

이 기능은 Braze에서 기본값으로 제공되므로 개발자는 핵심 비즈니스에 집중할 수 있습니다. 소프트웨어 개발 키트가 없다면 모든 Braze 클라이언트는 데이터 처리, 세분화 로직, 전달 옵션, 익명 사용자 처리, 캠페인 분석 등을 위한 모든 인프라와 툴을 처음부터 완전히 새로 만들어야 할 것입니다. 소프트웨어 개발 키트를 통합하는 데 걸리는 한 시간 정도보다 훨씬 더 오래 걸리고 훨씬 더 고통스러울 것입니다.

## 구현

앱이나 사이트에 소프트웨어 개발 키트를 통합하려면 누군가가 해당 애플리케이션을 구동하는 더 큰 전체 코드 베이스에 SDK의 코드를 추가해야 합니다. 즉, 엔지니어링 팀이 참여하여 기본적으로 앱 간에 정보와 작업이 흐르도록 앱을 서로 연결합니다. 하지만 개발자가 참여하더라도 SDK는 가볍고 사용자 친화적으로 통합할 수 있도록 설계되었습니다. 

시간을 저장하고 원활한 통합을 위해 고객과 엔지니어링 팀이 커스텀 이벤트, 커스텀 속성 및 SDK를 동시에 설정하는 것이 좋습니다. 마케팅 팀과 엔지니어링 팀이 함께 고려해야 할 단계에 대해 자세히 알아보려면 [구현 문서를]({{site.baseurl}}/user_guide/getting_started/integration/) 읽어보세요. 

## 데이터 집계

Braze 소프트웨어 개발 키트는 사용자 수준에서 방대한 양의 데이터를 자동으로 캡처하여 앱과 사용자 기반에 대한 주요 측정기준을 쉽게 확인할 수 있습니다. 비슷한 앱을 대시보드의 단일 작업 공간으로 그룹화합니다. 예를 들어 앱의 iOS 및 Android 버전을 동일한 작업 공간으로 그룹화하여 두 플랫폼에서 사용자로부터 수집한 데이터를 모두 볼 수 있습니다. 이를 통해 웹과 모바일 채널 전반에서 사용자를 보다 완벽하게 파악할 수 있습니다. 자세한 내용은 [홈 페이지의]({{site.baseurl}}/user_guide/analytics/dashboard/home_dashboard/) 도움말 문서를 참조하세요.

## 인앱 메시징

소프트웨어 개발 키트를 사용하면 인앱 메시지를 쉽게 작성하고 전송하여 사용자와 직접 소통할 수 있습니다. 캠페인 전략에 따라 슬라이드업, 모달 또는 전체 화면 메시지를 보낼 수 있습니다. 인앱 메시지 [작성에]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/traditional/create/) 대한 자세한 내용은 [인앱 메시지 작성하기]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/traditional/create/) 페이지를 참조하세요.

## 푸시 알림

푸시 알림은 사용자의 참여를 유도하는 또 다른 훌륭한 옵션으로, 특히 시간에 민감한 클릭 유도 문안을 처리하는 데 유용합니다. 모바일 푸시 알림은 사용자의 기기에 표시되며, 웹 푸시 알림은 사이트가 열려 있지 않을 때도 표시됩니다. 푸시 알림 사용에 대한 자세한 내용은 [푸시 알림 문서를]({{site.baseurl}}/user_guide/message_building_by_channel/push/about/) 참조하세요.

웹사이트 또는 앱 사용자가 푸시 알림을 받으려면 옵트인해야 합니다. 자세한 내용은 [푸시 준비를]({{site.baseurl}}/user_guide/message_building_by_channel/push/best_practices/push_primer_messages/) 참조하세요. 

## 세그먼트 세분화 및 전달 규칙

기본값으로 인앱 메시지가 포함된 캠페인은 해당 워크스페이스의 모든 버전의 앱에 전송됩니다. 예를 들어 이 메시지는 웹과 모바일 사용자 모두에게 전송됩니다. 인앱 메시지를 웹 또는 모바일에만 보내려면 그에 따라 캠페인을 세분화해야 하며, 이는 Braze SDK를 통해 기본값으로 지원됩니다. 

**특정 앱의 사용자를** **타겟팅하는 앱과 웹사이트를** 설정하여 웹 사용자 세그먼트를 만든 다음 **특정 앱에** 대해 웹사이트만 선택할 수 있습니다.

웹 앱에 초점을 맞춘 세그먼트 세부 정보 페이지]({% image_buster /assets/img_archive/web-users-segment.png %}){:style="max-width:60%"}

이를 통해 지능적인 방식으로 사용자의 행동을 기반으로 사용자를 타겟팅할 수 있습니다. 웹 사용자를 타겟팅하여 모바일 앱을 다운로드하도록 유도하려는 경우 이 세그먼트를 타겟 오디언스로 만들 수 있습니다. 웹 메시징이 아닌 모바일 인앱 메시지가 포함된 메시징 캠페인을 보내려면 세그먼트에서 웹사이트 아이콘을 선택 해제하면 됩니다.
