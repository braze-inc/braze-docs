# [![Braze 학습 과정]](https://learning.braze.com/most-engaged-channel) ( [{% image_buster /assets/img/bl_icon3.png %})](https://learning.braze.com/most-engaged-channel){: style="float:right;width:120px;border:0;" class="noimgborder"}지능형 채널 필터

> 지능형 또는 `Most Engaged` 채널 필터는 선택한 메시징 채널이 '최적' 채널인 오디언스 중 일부를 선택합니다. 

이 경우 최적이란 사용자의 기록을 고려할 때 참여 가능성이 가장 높은 채널을 의미합니다. 이메일, SMS, 웹 푸시 또는 모바일 푸시(사용 가능한 모든 모바일 OS 또는 디바이스 포함)를 채널로 선택할 수 있습니다.

![][1]{: style="float:right;max-width:40%;margin-left:10px;margin-top:10px;border:0"}

지능형 채널은 지난 6개월 동안 받은 메시지 수에 대한 메시지 상호 작용(열기 또는 클릭)의 비율을 취하여 세 가지 채널 각각에 대한 각 사용자의 참여율을 계산합니다. 사용 가능한 채널은 각각의 참여율에 따라 순위가 매겨지며, 참여율이 가장 높은 채널이 해당 사용자의 '가장 많이 참여한 채널'이 됩니다. 

사용자에게 메시지가 전송되거나 사용자가 메시지와 상호작용할 때마다 참여율은 몇 초 내에 다시 계산됩니다. 사용자는 한 번만 메시지와 상호작용한 것으로 계산됩니다(예를 들어, 같은 이메일을 열고 클릭하면 해당 메시지는 두 번이 아닌 한 번만 상호작용한 것으로 표시됨). 

지능형 채널 필터를 사용 설정하려면 이메일, 웹 푸시 또는 모바일 푸시 캠페인을 만들 때 **대상 사용자** 페이지에서 **지능형 채널** 필터를 선택합니다.

{% alert important %}
SMS 채널의 참여율을 계산하려면 고급 추적 및 클릭 추적을 통해 [SMS 링크 단축을]({{site.baseurl}}/user_guide/message_building_by_channel/sms/campaign/link_shortening/#overview/) 사용 설정하세요.
{% endalert %}

## '데이터 부족' 옵션

Braze가 어떤 채널이 "최적"인지 판단하려면 충분한 데이터가 있어야 합니다. 즉, 사용자는 사용 가능한 3개 채널 중 2개 이상의 채널에서 최소 3개 이상의 메시지를 수신해야 합니다. 메시지를 반드시 열어볼 필요는 없습니다. 

사용자가 여러 채널에서 충분한 메시지를 받지 못한 경우 해당 사용자는 이 필터의 '데이터 부족' 옵션에 속하게 됩니다. 이렇게 하면 사용 가능한 세 가지 메시징 채널 중 하나를 사용하여 이러한 사용자를 타겟팅할 수 있습니다.

예를 들어 푸시 메시지를 선호하는 사용자와 데이터가 충분하지 않은 사용자가 동일한 푸시 메시지를 수신하도록 하려고 한다고 가정해 보겠습니다. 이 경우 지능형 채널 필터를 **모바일로** 설정하고 **OR을** 사용하여 **데이터 부족으로** 설정된 두 번째 지능형 채널 필터를 추가할 수 있습니다. 지능형 채널 필터를 이메일로 설정한 별도의 캠페인은 이메일을 선호하는 사용자를 대상으로 할 수 있습니다.

![][2]

{% alert note %}
[빈도 제한을]({{site.baseurl}}/user_guide/engagement_tools/campaigns/testing_and_more/rate-limiting/#delivery-rules) 무시하는 캠페인 및 캔버스 단계는 인텔리전트 채널에 의해 계산되지 않으며 데이터 요구 사항에 기여할 수 없습니다.
{% endalert %}

## '모바일 푸시' 옵션

모바일 푸시에는 Android, iOS, Kindle 및 Braze에서 사용할 수 있는 기타 모바일 디바이스 채널이 통합되어 있습니다. 지능형 채널을 계산할 때 Braze는 각 종류의 모바일 디바이스를 개별적으로 살펴본 다음, 이메일 및 웹 푸시와 비교할 때 '모바일 푸시' 카테고리를 대표할 참여율이 가장 높은 디바이스를 선택합니다. 

예를 들어, 사용자가 여러 대의 모바일 기기를 사용하는 경우 모바일 참여율은 여러 기기에서 가장 높은 참여율로 표시됩니다. 그러나 이렇게 해도 사용자가 해당 디바이스에서만 푸시 알림을 수신하도록 강제하지는 않습니다. 이 요금은 이메일 및 웹 푸시와 요금을 비교할 때만 사용됩니다.

## 개별 채널

Braze가 사용자에게 가장 적합한 단일 채널을 선택하도록 하는 대신, 사용자가 선택한 특정 채널에서 메시지를 열 가능성이 있는지 여부에 따라 사용자를 필터링할 수도 있습니다. 이를 위해 [세분화 필터에서]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters) 메시지 공개 가능성 필터를 사용할 수 있습니다.

## 모범 사례 및 효과적인 사용 전략

### 타이브레이킹

일부 사용자는 수신한 메시지 수가 적기 때문에 특정 사용자의 사용 가능한 채널에서 참여율이 동률을 이루는 경우가 종종 있습니다(예: 한 사용자가 이메일과 모바일 푸시 **모두에서** 0.2의 참여율을 기록하는 경우). 이러한 경우 가장 최근에 열린 이벤트가 있는 채널에 우선순위를 부여(더 높은 순위 부여)하여 동점자가 나오면 동점이 깨집니다.

### 연결할 수 없는 채널

사용자가 순위를 결정하기에 충분한 데이터를 가지고 있지만 가장 참여도가 높은 채널에서 연락이 닿지 않으면 해당 사용자는 '탈락'하고 메시지를 수신하지 않게 됩니다. 특정 채널에서 연락이 닿지 않는 사용자는 별도로 타겟팅해야 합니다.

### 오디언스 크기 조정

인텔리전트 채널을 사용하면 나머지 잠재고객보다 메시지에 참여할 가능성이 훨씬 높은 일부 사용자를 미리 선택적으로 타겟팅할 수 있습니다. 이는 일반적인 오디언스의 사용자 대다수를 대표하지 않을 가능성이 높습니다. 오히려 이 필터는 특정 채널에 참여한 기록이 있는 일반 오디언스 중 5~20%를 찾아낼 수 있을 것으로 기대할 수 있습니다.
