# 자주 묻는 질문

> 이 문서에서는 캠페인에 대해 자주 묻는 질문에 대한 답변을 제공합니다.

### 멀티채널 캠페인을 만들려면 어떻게 해야 하나요?

멀티 채널 캠페인을 만들려면 **메시징** > 캠페인을 선택합니다. 그런 다음 **캠페인 만들기** > **멀티채널을** 선택합니다. 여기에서 다음 메시징 채널 중에서 선택할 수 있습니다: 콘텐츠 카드, 이메일, LINE, 푸시 알림, SMS/MMS/RCS, 웹훅 또는 WhatsApp.

### 멀티채널 캠페인에 대조군을 추가할 수 있나요?

아니요, 캠페인의 대조군은 이메일 A와 이메일 B와 같은 단일 채널 메시징을 위한 것입니다. 대신 [캔버스를]({{site.baseurl}}/user_guide/engagement_tools/canvas) 사용하여 다양한 채널, 메시징 콘텐츠 및 전달 타이밍을 테스트해 보세요. 

### 캠페인 테스트 및 최적화를 시작하려면 어떤 방법이 있나요?

다변량 캠페인과 여러 배리언트가 있는 캔버스를 실행하는 것은 시작하기에 좋은 방법입니다! 예를 들어, [다변량 캠페인을]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/) 실행하여 사본이나 제목란이 다른 하나의 메시지를 테스트할 수 있습니다. 여러 배리언트가 있는 캔버스는 전체 워크플로를 테스트하는 데 도움이 될 수 있습니다.

### 내 캠페인 열람율이 감소한 이유는 무엇인가요?

낮은 열람율이 항상 기술적 문제와 관련이 있는 것은 아닙니다. 이메일 클리핑에 문제가 있어 추적 픽셀이 누락될 수 있습니다. 그러나 콘텐츠나 오디언스 규모의 변화로 인해 이메일을 열어보는 사용자가 줄어들 수도 있습니다. 

### 캠페인 오디언스는 어떻게 평가되나요?

기본적으로 캠페인은 진입 시 오디언스 필터를 확인합니다. 지연이 있는 액션 기반 캠페인의 경우, 메시지 전송 시 사용자가 여전히 타겟 오디언스에 속해 있는지 확인하기 위해 전송 시점에 세그먼트 기준을 재평가하는 옵션이 있습니다. 

### 특정 캠페인 또는 캔버스의 고유 수신자 수와 전송 횟수 사이에 차이가 있는 이유는 무엇인가요?

한 가지 가능한 설명은 캠페인 또는 캔버스에 재자격이 설정되어 있어 해당 세그먼트 및 전달 설정에 적합한 사용자가 메시지를 두 번 이상 받을 수 있다는 것입니다. 재인증이 켜져 있지 않은 경우 발신자와 고유 수신자 간의 차이에 대한 가능한 설명은 사용자가 프로필과 연결된 여러 플랫폼에 걸쳐 여러 대의 기기를 가지고 있기 때문일 수 있습니다. 

예를 들어 iOS와 웹 푸시 알림이 모두 있는 캔버스가 있는 경우 모바일과 데스크톱 기기를 모두 사용하는 특정 사용자는 두 개 이상의 메시지를 받을 수 있습니다.

### 내 캠페인의 도달 가능한 사용자 기반이 캠페인에 사용하는 세그먼트보다 작은 이유는 무엇인가요?

[글로벌 컨트롤 그룹이]({{site.baseurl}}/user_guide/engagement_tools/testing/global_control_group/) 설정되어 있으면 도달 가능한 오디언스 중 일부가 캠페인을 수신하지 못하도록 차단할 수 있습니다. 즉, 캠페인에서 동일한 세그먼트를 사용하더라도 세그먼트의 도달 가능한 사용자 수가 캠페인의 도달 가능한 사용자 수보다 많을 수 있습니다.

### 현지 시간대 전달은 무엇을 제공하나요?

현지 시간대 전달을 사용하면 사용자의 개별 시간대를 기반으로 한 세그먼트에 메시징 캠페인을 전달할 수 있습니다. 현지 시간 전달 기능이 없는 경우, 캠페인은 Braze에서 회사의 시간대 설정에 따라 예약됩니다. 

예를 들어 런던에 본사를 둔 회사가 오후 12시에 캠페인을 보내면 미국 서부 해안의 사용자에게 새벽 4시에 도달하게 됩니다. 특정 국가에서만 앱을 사용할 수 있는 경우에는 이러한 위험이 발생하지 않을 수 있습니다. 그렇지 않으면 사용자 기반에 이른 아침 푸시 알림을 보내지 않는 것이 좋습니다.

### Braze는 사용자의 시간대를 어떻게 인식하나요?

Braze는 사용자의 기기에서 사용자의 시간대를 자동으로 확인합니다. 이를 통해 시간대의 정확성을 보장하고 사용자를 완벽하게 커버할 수 있습니다. 사용자 API를 통해 생성되거나 표준 시간대가 없는 사용자는 소프트웨어 개발 키트에서 앱에서 인식될 때까지 회사의 표준 시간대가 기본값으로 사용됩니다. 

대시보드의 [회사 설정에서]({{site.baseurl}}/user_guide/administrative/app_settings/company_settings/) 회사의 표준 시간대를 확인할 수 있습니다.

### Braze는 현지 시간대 전달을 위해 언제 사용자를 평가하나요?

현지 시간대 전달의 경우, Braze는 이 두 인스턴스 동안 사용자의 참가 자격을 평가합니다:

- 예정일의 사모아 시간(UTC+13) 기준
- 예정일 현지 시간 기준

사용자가 참가 자격을 얻으려면 두 가지 검사를 모두 통과해야 합니다. 예를 들어 캔버스가 2021년 8월 7일 오후 2시(현지 시간 기준)에 출시될 예정인 경우 뉴욕에 위치한 사용자를 타겟팅하려면 다음과 같은 자격 확인을 거쳐야 합니다:

- 2021년 8월 6일 오후 9시에 뉴욕
- 2021년 8월 7일 오후 2시 뉴욕

사용자는 출시 전 24시간 동안 해당 세그먼트에 있어야 한다는 점에 유의하세요. 사용자가 첫 번째 확인에서 자격이 없는 경우 Braze는 두 번째 확인을 시도하지 않습니다.

예를 들어, 오후 7시(UTC)에 캠페인이 전송될 예정인 경우 해당 시간대(예: 사모아)가 식별되는 즉시 캠페인 전송을 대기열에 추가하기 시작합니다. 이는 캠페인을 보내는 것이 아니라 메시지를 보낼 준비를 하고 있다는 의미입니다. 자격을 확인할 때 필터와 일치하지 않는 사용자는 타겟팅 대상에 포함되지 않습니다.

다른 예로, 같은 날 오전과 저녁에 각각 하나씩 두 개의 캠페인을 보내도록 예약하고 사용자가 이미 첫 번째 캠페인을 받은 경우에만 두 번째 캠페인을 받을 수 있도록 필터를 추가한다고 가정해 보겠습니다. 현지 시간 전달로 인해 일부 사용자는 두 번째 캠페인을 받지 못할 수 있습니다. 이는 사용자의 시간대가 식별될 때 자격을 확인하기 때문에 해당 시간대에 예약된 시간이 아직 발생하지 않았다면 첫 번째 캠페인을 받지 못한 것이므로 두 번째 캠페인을 받을 자격이 없습니다.

### 현지 시간대 캠페인은 어떻게 예약하나요?

캠페인을 예약할 때 지정된 시간에 캠페인을 보내도록 선택한 다음 **현지 시간대의 사용자에게 캠페인 보내기를** 선택합니다.

Braze는 모든 현지 시간대 캠페인을 24시간 전에 예약할 것을 적극 권장합니다. 이러한 캠페인은 하루 종일 보내야 하므로 24시간 전에 미리 예약하면 전체 메시지 세그먼트에 메시지가 도달할 수 있습니다. 그러나 필요한 경우 24시간 전에 캠페인을 예약할 수 있습니다. Braze는 전송 시간을 1시간 이상 놓친 사용자에게는 메시지를 보내지 않는다는 점에 유의하세요. 

예를 들어 오후 1시에 현지 시간대 캠페인을 오후 3시로 예약하면 현지 시간이 오후 3시에서 4시 사이인 모든 사용자에게 캠페인이 즉시 전송되지만 현지 시간이 오후 5시인 사용자에게는 캠페인이 전송되지 않습니다. 또한 캠페인에 대해 선택한 전송 시간은 회사 시간대에 아직 발생하지 않은 시간이어야 합니다.

24시간 전에 예약된 현지 시간대 캠페인을 편집해도 메시징 캠페인의 일정은 변경되지 않습니다. 현지 시간대 캠페인을 편집하여 이후 시간(예: 오후 6시 대신 오후 7시)에 전송하기로 결정한 경우, 원래 전송 시간을 선택할 때 타겟팅된 세그먼트에 속했던 사용자는 여전히 원래 시간(오후 6시)에 메시지를 받게 됩니다. 현지 시간대를 편집하여 더 이른 시간(예: 오후 5시 대신 오후 4시)에 전송하는 경우에도 캠페인은 원래 시간(오후 5시)에 모든 세그먼트 멤버에게 전송됩니다. 

{% alert note %}
캔버스 컴포넌트의 경우 현지 시간 전달을 위해 사용자 여정에서 다음 컴포넌트를 받기 위해 사용자가 24시간 동안 컴포넌트에 있을 필요는 없습니다.
{% endalert %}

사용자가 캠페인에 다시 참여할 수 있도록 허용한 경우 원래 시간(오후 5시)에 다시 캠페인을 받게 됩니다. 그러나 이후 발생하는 모든 캠페인의 경우 메시징은 업데이트된 시간에만 전송됩니다.

### 현지 시간대 캠페인 변경은 언제 적용되나요?

현지 시간대 캠페인의 타겟팅 세그먼트에는 시간 기반 필터의 경우 전체 세그먼트에 전달을 보장하기 위해 최소 48시간의 기간이 포함되어야 합니다. 예를 들어 다음 필터를 사용하여 둘째 날 사용자를 타겟팅하는 세그먼트를 생각해 보세요:

- 처음 사용한 지 1일 이상 지난 앱
- 앱을 처음 사용한지 2일 미만입니다.

현지 시간대 전달은 전달 시간과 사용자의 현지 시간대에 따라 이 세그먼트의 사용자를 놓칠 수 있습니다. 이는 사용자가 자신의 시간대가 전달을 트리거하는 시간까지 세그먼트를 떠날 수 있기 때문입니다.

### 출시 전에 예약된 캠페인을 변경하려면 어떻게 해야 하나요?

캠페인이 예약되면 보낼 메시지를 대기열에 추가하기 전에 메시지 구성 이외의 내용을 편집해야 합니다. 모든 캠페인과 마찬가지로 캠페인이 시작된 후에는 전환 이벤트를 편집할 수 없습니다.

### 예약된 캠페인을 업데이트했습니다. 왜 출시되지 않았나요?

이는 캠페인이 업데이트된 정확한 시간에 시작되도록 예약된 경우 발생할 수 있습니다. 예를 들어 현재 오후 3시 10분인데 캠페인을 오후 3시 10분에 시작하도록 변경하고 **캠페인 업데이트를** 선택했다면 현재 오후 3시 10분이 지났으므로 시작 예정 시간이 지났음을 의미합니다. 캠페인을 같은 시간에 예약하는 대신 **캠페인이 시작되는 즉시 보내기를** 선택합니다.

### 예약된 캠페인의 메시징 캠페인이 대기열에 추가되기 전의 '안전 지대'는 무엇인가요?

다음 시간 내에 메시지를 변경하는 것이 좋습니다:

- **일회성 예약 캠페인:** 예약된 전송 시간까지 편집합니다.
- **반복 예약 캠페인:** 예약된 전송 시간까지 편집합니다.
- **현지 전송 시간 캠페인:** 예정된 전송 시간 24시간 전까지 편집할 수 있습니다.
- **최적의 전송 시간 캠페인:** 캠페인 전송 예정일 24시간 전까지 편집할 수 있습니다.

이러한 권장 사항을 벗어나 메시지를 변경하면 전송된 메시징에 업데이트가 반영되지 않을 수 있습니다. 예를 들어 현지 시간으로 오후 12시에 캠페인을 전송할 예정인 경우 전송 시간을 3시간 전에 편집하면 다음과 같은 상황이 발생할 수 있습니다:

- Braze는 전송 시간을 1시간 이상 놓친 사용자에게는 메시지를 보내지 않습니다.
- 미리 대기열에 추가된 메시징은 조정된 시간이 아닌 원래 대기열에 추가된 시간에 전송될 수 있습니다.

변경이 필요한 경우 현재 캠페인을 중지하는 것이 좋습니다(이렇게 하면 대기 중인 메시징이 모두 취소됩니다). 그런 다음 캠페인을 복제하고 필요에 따라 변경한 다음 새 캠페인을 시작할 수 있습니다. 이미 첫 번째 캠페인을 받은 사용자를 이 캠페인에서 제외해야 할 수도 있습니다. 시간대 전송이 가능하도록 캠페인 일정 시간을 다시 조정해야 합니다.

### 캠페인에 참여하는 사용자 수가 예상한 수와 일치하지 않는 이유는 무엇인가요?

오디언스 및 트리거가 평가되는 방식에 따라 캠페인에 참여하는 사용자 수는 예상한 수와 다를 수 있습니다. Braze에서는 트리거 전에 오디언스가 평가됩니다( [속성 변경]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/triggered_delivery/attribute_triggers/#change-custom-attribute-value) 트리거를 사용하지 않는 한). 이렇게 하면 트리거 동작이 평가되기 전에 사용자가 처음에 선택한 오디언스에 속하지 않은 경우 캠페인에서 이탈하게 됩니다.

캠페인 문제 해결에 대한 추가 지원이 필요한 경우, 최근 30일간의 진단 로그만 보유하고 있으므로 문제 발생일로부터 30일 이내에 Braze 지원팀에 문의하세요.
{% endalert %}

### 내 캠페인 분석 페이지의 CSV 사용자 데이터 내보내기 옵션과 CSV 이메일 주소 내보내기 옵션의 차이점은 무엇인가요?

**CSV 이메일 주소 내보내기** 옵션을 선택하면 이메일 주소가 있는 사용자의 데이터만 다운로드됩니다. 예를 들어 100,000명의 사용자로 구성된 세그먼트가 있는데 그 중 50,000명의 사용자만 이메일 주소를 가지고 있고 **CSV 이메일 주소 내보내기를** 클릭하면 CSV 파일에 50,000개의 데이터 행만 표시될 것으로 예상할 수 있습니다. 이에 비해 **CSV 사용자 데이터 내보내기를** 선택하면 모든 사용자 데이터가 내보내집니다.

### API 식별자로 캠페인을 검색할 수 있나요?

예, **캠페인** 페이지에서 `api_id:YOUR_API_ID` 필터를 사용하여 API 식별자로 캠페인을 검색할 수 있습니다. 자세한 내용은 [캠페인 검색을]({{site.baseurl}}/user_guide/engagement_tools/campaigns/managing_campaigns/search_campaigns/) 참조하세요.

### API 캠페인과 API 트리거 캠페인의 차이점은 무엇인가요?

API 트리거 캠페인을 사용하면 자체 서버 및 시스템에서 해당 콘텐츠의 전달을 트리거하면서 Braze 대시보드 내에서 캠페인 카피, 다변량 테스트 및 재자격 규칙을 관리할 수 있습니다. 이러한 메시지에는 메시징에 실시간으로 템플릿화할 추가 데이터를 포함할 수도 있습니다.

API 캠페인은 API를 사용하여 전송된 메시지를 추적하는 데 사용됩니다. 대부분의 캠페인과 달리 메시지, 수신자 또는 일정을 지정하지 않고 API 호출에 식별자를 전달합니다.

#### 액션 기반

실행 기반 전달 캠페인 또는 이벤트 트리거 캠페인은 트랜잭션 또는 업적 기반 메시지에 매우 효과적이며, 사용자가 특정 이벤트를 완료한 후에 메시지를 보내도록 트리거할 수 있습니다. 

| 장점 | 단점 |

#### API 트리거형

API 트리거 및 서버 트리거 캠페인은 자체 서버와 시스템에서 캠페인 콘텐츠 전달을 트리거할 수 있어 보다 고급 트랜잭션을 처리하는 데 이상적입니다. 메시지를 트리거하는 API 요청에는 메시지에 실시간으로 템플릿화할 추가 데이터가 포함될 수도 있습니다.

| 혜택 | 고려 사항 | 
| ---- | ---- |
| \- 데이터 포인트를 기록하지 않음<br><br>\- 개인화 요소는 JSON 페이로드 속성에 포함됩니다. | \- JSON 페이로드 속성에서 메시지를 받을 수 있는 사용자 세그먼트를 만들 수 없습니다.<br><br>- **메시지 활동 로그를** 통해 들어오는 JSON 페이로드를 볼 수 없음|
