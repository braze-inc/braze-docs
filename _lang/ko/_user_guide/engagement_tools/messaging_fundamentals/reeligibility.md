# 캠페인 및 캔버스 재자격 부여

> 반복 또는 트리거되는 캠페인이나 캔버스를 예약할 때 사용자가 해당 캠페인에 다시 참여할 수 있도록 허용할 수 있는 옵션이 있습니다. 재자격은 사용자가 트리거에 따라 캠페인 또는 캔버스에 여러 번 참여할 수 있음을 의미합니다.

## 작동 방식

재인증은 별도로 설정해야 하므로 사용자가 여러 번 재인증하더라도 기본값으로 Braze는 사용자에게 메시지를 한 번만 보냅니다. 이 기능을 켜면 자격을 갖춘 회원은 캠페인 또는 캔버스의 첫 번째 인스턴스를 받은 후 다시 메시지를 받을 수 있습니다. 사용자가 궁극적으로 다시 자격을 얻을 수 있는 타임라인을 명시할 수 있습니다.

캠페인 **재수신** 자격을 사용 설정하려면 **전달 관리** 섹션에서 **사용자가 캠페인 재수신 자격을 갖도록 허용** 확인란을 선택합니다. 캠페인에 다시 참여할 수 있는 최대 기간은 720일입니다.

재자격이 켜져 있는 트리거된 캠페인의 경우, 트리거 이벤트를 완료했음에도 불구하고 [실제로 캠페인 메시지를 받지 못한]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/triggered_delivery/#why-did-a-user-not-receive-my-triggered-campaign) 사용자는 다음에 트리거 이벤트를 완료할 때 자동으로 메시지를 받을 자격을 얻게 됩니다. 재자격은 캠페인 참여가 아닌 메시지 수신을 기준으로 하기 때문입니다. 사용자가 트리거된 캠페인에 다시 참여할 수 있도록 하면 단순히 트리거되는 것이 아니라 실제로 메시지를 두 번 이상 수신할 수 있게 됩니다.

또한 0분의 재자격으로 즉시 메시지를 보내려는 경우, 사용자가 이전 버전의 캠페인이나 캔버스를 어떻게 수신했는지에 관계없이 항상 바로 예약을 시도합니다.

#### API 트리거 캠페인을 통한 재자격 부여

재자격 설정을 사용하여 사용자가 API 트리거 캠페인을 수신하는 횟수를 제한할 수 있습니다. 즉, 사용자는 API 트리거가 몇 번 실행되든 상관없이 지정된 기간에 한 번 또는 한 번만 캠페인을 수신하게 됩니다.

예를 들어 API 트리거 캠페인을 사용하여 사용자가 최근에 본 항목에 대한 캠페인을 보낸다고 가정해 보겠습니다. 이 경우 각 항목에 대해 API 트리거를 실행하는 동안 조회한 항목 수에 관계없이 하루에 최대 한 개의 메시지만 보내도록 캠페인을 제한할 수 있습니다. 반면 API 트리거 캠페인이 트랜잭션인 경우 지연 시간을 0분으로 설정하여 사용자가 트랜잭션을 수행할 때마다 캠페인을 수신할 수 있도록 해야 합니다.

캔버스에 대한 **재입장 허용을** 켜려면 **항목 제어** 섹션에서 **사용자가 이 캔버스에 다시 입장할 수 있도록 허용을** 선택합니다. 사용자가 캔버스의 최대 기간 이후 또는 지정된 기간이 지난 후 재입장할 수 있도록 허용할지 여부를 선택할 수 있습니다.

캔버스 배리언트에 대한 재자격은 메시지 수신이 아닌 캔버스 입력과 관련이 있습니다. 캔버스에 입장한 후 메시지를 받지 못한 사용자는 다시 입장할 수 있는 권한을 켜지 않는 한 캔버스에 다시 입장할 수 없습니다.

재참가 자격이 0초로 설정된 경우 사용자가 캔버스를 먼저 종료한 후 재참가할 필요가 없으므로 같은 캔버스에 다시 들어갈 수 있다는 점에 유의하세요. 필터를 추가하여 사용자가 동일한 단계나 메시지를 여러 번 수신하지 못하도록 할 수 있습니다. 그러나 사용자가 캔버스에 두 번째로 다시 들어가면 캔버스에 처음 들어갔을 때 이전에 받은 단계는 사용자에게 표시되지 않습니다. 즉, 사용자는 여전히 동일한 메시지를 다시 받을 수 있습니다. 이를 방지하기 위해 재입장을 방지하도록 캔버스를 구성하거나 캔버스의 최대 기간 동안 재입장할 수 있는 자격을 설정할 수 있습니다.

단계를 받은 사용자에 대한 [사용자 업데이트 컴포넌트를]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/user_update/) 사용하여 이를 커스텀 속성으로 기록하면 캔버스 여정 중에 단계를 받은 사용자를 필터링하는 데 사용할 수 있습니다.

### 예

예를 들어 이메일 주소가 없는 사용자가 사용자 여정의 한 단계가 포함된 매일 반복되는 캔버스에 들어왔다고 가정해 보겠습니다. 이 단계에는 이메일 메시지만 포함되어 있으므로 사용자는 참여를 얻지 못합니다. 이 사용자는 캔버스의 재인증이 켜져 있지 않으면 캔버스에 다시 들어갈 수 없습니다. 

재참가 자격이 없는 활성 반복 또는 트리거된 캔버스가 있고 사용자가 캔버스에서 메시지를 받을 때까지 캔버스에 재참가하도록 하려는 경우, 캔버스에서 메시지를 받은 고객을 제외하는 필터를 입력 기준에 추가하여 사용자가 재참가할 수 있도록 허용하는 것을 고려할 수 있습니다.

캔버스에 대한 재접속 자격을 캔버스 기간보다 짧게 설정하면 사용자가 캔버스에 두 번 이상 입장할 수 있으며, 이로 인해 지연 시간이 특히 긴 인앱 메시지를 사용하는 캔버스의 경우 잘못된 동작이 발생할 수 있습니다. 동일한 세션 시작 시 여러 개의 캔버스 인앱 메시지가 트리거될 수 있으므로 특정 컴포넌트가 다른 컴포넌트보다 빠르게 렌더링되는 경우 사용자는 동일한 메시지를 반복적으로 수신하는 경험을 할 수 있습니다.

## 자격 재취득 지연 계산

캠페인과 캔버스 모두 재적격 여부는 캘린더 날짜가 아닌 초 단위로 계산됩니다. 즉, 하루는 사용자가 메시지를 수신한 시점부터 24시간(또는 86,400초)으로 계산되며, 다음 날 자정이 아닌 캘린더 기준으로 계산됩니다. 마찬가지로 한 달은 정확히 2,592,000초로 계산되며, 이는 약 30일에 해당합니다.

### 예

다음 시나리오를 생각해 보세요:

* 캠페인은 매월 15일에 발송되도록 설정되어 있으며, 재신청 자격은 30일로 설정되어 있습니다.
* 2월 15일부터 3월 15일까지는 30일이 채 되지 않습니다. 

즉, 2월 15일에 캠페인을 수신한 사용자는 3월 15일에 발송되는 캠페인에 참여할 수 없습니다. 캠페인이 매일 오전 8시에 전송되도록 설정되어 있고 재자격이 1일로 설정되어 있으며 메시지 전송에 지연이 있는 경우 오전 8시 30분에 캠페인을 수신한 사용자는 다음 날 오전 8시에는 아직 재자격이 부여되지 않습니다.

## 다변량 테스트

다변량 테스트의 경우, Braze는 다음 규칙을 사용하여 모든 캠페인, 트리거된 인앱 메시지 및 캔버스에 대한 배리언트 재적격성을 결정합니다:

- 배리언트 비율을 변경하지 않으면 각 사용자는 자격이 다시 부여될 때마다 항상 동일한 캠페인 배리언트, 트리거된 인앱 메시지 또는 캔버스 항목을 입력하게 됩니다.
- 배리언트 비율이 변경되면 사용자는 다른 배리언트로 재배포될 수 있습니다.
- 배리언트 비율이 변경되지 않으면 대조군은 일관성을 유지하며, 이전에 메시지를 받은 사용자는 나중에 메시지를 보낼 때 대조군에 들어가지 않으며 대조군에 있는 사용자도 메시지를 받지 않습니다.
