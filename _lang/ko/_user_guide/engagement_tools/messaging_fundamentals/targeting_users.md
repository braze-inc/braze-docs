# 사용자 타겟팅하기

> 사용자를 타겟팅하는 방법을 결정하는 것은 캠페인이나 캔버스를 만들 때 가장 중요한 단계 중 하나입니다. 오디언스의 행동, 선호도 및 인구통계를 기반으로 오디언스를 세분화하는 방법을 이해하면 메시징을 맞춤화하고 개인화할 수 있습니다.

## 타겟팅 오디언스 만들기

### 1단계: 사용자 선택

**타겟팅 옵션에서** 다음 옵션을 사용하여 캠페인 또는 캔버스에 대해 타겟팅할 사용자를 선택할 수 있습니다. 정의한 기준과 일치하는 사용자만 메시지를 받게 됩니다. 정확한 세그먼트 멤버십은 항상 메시지 전송 직전에 계산된다는 점에 유의하세요.

{% tabs local %}
{% tab single segment %}
이전에 만든 세그먼트의 멤버를 타겟팅하려면 **세그먼트별 사용자 타겟팅** 아래의 드롭다운에서 하나의 세그먼트를 선택합니다.
{% endtab %}

{% tab multiple segments %}
이전에 만든 여러 세그먼트에 속하는 사용자를 타겟팅하려면 **세그먼트별 사용자 타겟팅** 아래의 드롭다운에서 여러 세그먼트를 추가합니다. 그 결과 타겟 오디언스는 첫 번째 세그먼트와 두 번째 세그먼트, 세 번째 세그먼트 등의 사용자가 될 것입니다.
{% endtab %}

{% tab multiple filters %}
세그먼트를 추가하지 않고 사용자를 타겟팅하려면 일련의 필터를 사용할 수 있습니다. 이는 메시지 작성 중 즉석 오디언스이며 일회성 오디언스에게 보낼 때 세그먼트 생성을 건너뛸 수 있습니다.

하루 이내에 마지막으로 앱을 열었고, 캠페인이나 캔버스 단계를 받은 적이 없으며, 구매한 지 30일이 지나지 않은 사용자를 타겟팅하는 메시지에 대한 추가 필터를 추가할 수 있습니다.]({% image_buster /assets/img_archive/additional_filters.png %}){: style="max-width:90%;"}
{% endtab %}

{% tab segments & filters %}
또한 추가 필터에 해당하는 하나 이상의 이전에 생성한 세그먼트의 사용자를 타겟팅할 수도 있습니다. 먼저 세그먼트를 선택한 후 **추가 필터** 섹션에서 오디언스를 더욱 세분화할 수 있습니다. 이는 '일일 활성 사용자' 세그먼트, '이메일을 열어보지 않음' 세그먼트에 속하며 30일 이상 전에 구매한 사용자를 타겟팅하는 다음 스크린샷에서 확인할 수 있습니다.

\![두 개의 세그먼트가 포함된 메시지의 타겟팅 옵션에 30일 전 마지막 구매에 대한 추가 필터가 있습니다.]({% image_buster /assets/img_archive/target_segmenter.png %}){: style="max-width:90%;"}
{% endtab %}

{% tab Specific apps %}

특정 앱에만 인앱 메시지 또는 푸시 알림을 보내는 등 특정 앱에 캠페인 메시지 또는 캔버스 단계를 전달할 수 있습니다(예: Android 또는 iOS 앱에만 메시지 또는 푸시 알림 전송).

하지만 한 사용자가 여러 앱을 사용할 수 있다는 점을 기억하세요. '앱 있음' 필터는 선택한 앱을 가지고 있는 모든 사용자를 식별하지만 메시지를 수신하는 앱은 제어하지 않습니다. 예를 들어 '앱 있음'이 Android로 설정된 세그먼트 필터를 적용하면 iOS 앱도 가지고 있는 모든 사용자가 iOS 앱에서도 해당 메시지를 받게 됩니다.

"Hello, World (Android)" 앱을 보유한 사용자를 위한 필터입니다.]({% image_buster /assets/img_archive/has_app_hello_world.png %}){: style="max-width:60%;"}

Android 앱에만 인앱 메시지를 보내고 싶다고 가정해 보겠습니다.

1. 세그먼트를 생성하고 **특정 앱의 사용자를** **타겟팅하는 앱과 웹사이트를** 설정한 다음, Android 앱을 선택합니다.

\![특정 앱의 사용자를 타겟팅하는 세그먼트, "Test_Android".]({% image_buster /assets/img_archive/app_test_android.png %}){: style="max-width:60%;"}

{: start="2"}
2\. 캠페인 또는 캔버스에서 **타겟 오디언스** 단계로 이동하여 **세그먼트별 타겟팅 사용자** 섹션에서 세그먼트가 추가되었는지 확인합니다. 

예시 세그먼트가 선택된 '타겟 오디언스' 단계입니다.]({% image_buster /assets/img_archive/target_users_by_segment_example.png %})

{% alert note %}
**추가 필터** 섹션에서 세그먼트 멤버십 필터를 통해 세그먼트를 추가하는 경우에는 이 기능이 작동하지 않습니다. 해당 앱에만 메시지를 전달하려면 **세그먼트별 사용자 타겟팅에서** 세그먼트를 직접 참조해야 합니다.
{% endalert %}

{% alert tip %}
이메일 캠페인의 경우 **시드 그룹** 섹션에서 시드 그룹을 타겟팅할 수 있습니다. 캠페인에서 API 트리거 항목을 통해 시드 그룹을 포함할 수는 있지만, API 캠페인에는 시드 그룹을 사용할 수 없다는 점에 유의하세요. 자세한 내용은 [시드 그룹을]({{site.baseurl}}/user_guide/administrative/app_settings/internal_groups_tab/#seed-groups) 참조하세요.
{% endalert %}

### 2단계: 오디언스 테스트하기

오디언스에 세그먼트와 필터를 추가한 후에는 [사용자를 조회하여]({{site.baseurl}}/user_guide/engagement_tools/segments/creating_a_segment/) 오디언스 기준과 일치하는지 확인하여 오디언스가 예상대로 설정되었는지 테스트할 수 있습니다.

'사용자 조회' 섹션에 '사용자 조회' 버튼이 있습니다.]({% image_buster /assets/img_archive/user_lookup.png %}){: style="max-width:70%"}

#### 오디언스 요약

**오디언스 요약에는** 타겟 오디언스에 누가 있는지 개요가 표시됩니다. 여기에서 최대 사용자 수 제한 또는 전송 속도 [제한을]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/) 설정하여 오디언스를 더욱 제한할 수 있습니다.

최대 사용자 수 제한 또는 전송 속도 제한을 설정할 수 있는 옵션이 있는 '오디언스 요약' 섹션입니다.]({% image_buster /assets/img_archive/audience_summary.png %})

#### A/B 테스트

**A/B 테스트** 섹션에서는 동일한 마케팅 캠페인의 여러 버전에 대한 사용자의 반응을 비교하는 테스트를 설정할 수 있습니다. 이 버전은 비슷한 마케팅 목표를 공유하지만 문구와 스타일이 다릅니다. 마케팅 목표를 가장 잘 달성할 수 있는 캠페인 버전을 식별하는 것이 목표입니다. 

자세한 내용과 모범 사례는 [다변량 & A/B 테스트를]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/) 참조하세요.

#### 오디언스 통계

Braze는 바닥글에 타겟팅한 채널의 상세한 오디언스 통계를 제공합니다. 사용자 기반이 클수록 **도달 가능한 사용자** 수는 대략적인 추정치일 가능성이 높습니다. [글로벌 컨트롤 그룹을]({{site.baseurl}}/user_guide/engagement_tools/testing/global_control_group/) 사용하거나 메시지 자격을 설정하면 도달 가능한 사용자 수가 감소할 수 있습니다. 

- 도달 가능한 사용자의 정확한 숫자를 확인하려면 [정확한 통계 계산을]({{site.baseurl}}/user_guide/engagement_tools/segments/creating_a_segment#calculating-exact-statistics) 선택하면 사용자 기반의 모든 사용자를 검색할 수 있습니다.
- 타겟팅 중인 사용자 기반 비율 또는 이 세그먼트의 생애주기 가치(LTV)를 확인하려면 **추가 통계 표시를** 선택합니다.

##### 타겟 오디언스 수가 도달 가능한 사용자 수와 다를 수 있는 이유

각 대상 채널에서 도달 가능한 사용자의 예상 수가 표시된 '총 인구수' 섹션입니다.]({% image_buster /assets/img_archive/multi_channel_footer.png %})

{% alert note %}
정확한 통계를 계산하는 데는 몇 분 정도 걸릴 수 있습니다. 이 기능은 필터 또는 필터 그룹 수준이 아닌 세그먼트 수준에서만 정확한 통계를 계산합니다.<br><br>
세그먼트가 큰 경우 정확한 통계를 계산할 때에도 약간의 편차가 발생하는 것이 정상입니다. 이 기능의 정확도는 99.999% 이상일 것으로 예상됩니다.
{% endalert %}

## 타겟팅 오디언스와 진입 기준이 함께 작동하는 방식

Braze에서 캠페인이나 캔버스를 구축할 때 타겟팅은 두 부분으로 이루어집니다:

1. **타겟 오디언스:** 자격 대상
2. **참가 기준:** 전달을 트리거하는 요소

순서가 중요합니다: Braze는 참가 기준을 평가하기 전에 타겟 오디언스에 해당하는지 여부를 확인합니다. 사용자가 해당 시점에 이미 오디언스 자격을 갖추지 않은 경우, 나중에 참가 이벤트를 트리거하더라도 캠페인이나 캔버스에 입장할 수 없습니다. 타겟 오디언스를 대기실이라고 생각하면 트리거가 발생했을 때 이미 내부에 있는 사용자만 앞으로 나아갈 수 있습니다.

### 예제 1

사용자의 첫 세션 중에 푸시 메시지를 보내려고 합니다.

설정했습니다:

- **타겟 오디언스:** 세션 수가 0인 사용자
- **응모 이벤트:** 세션 시작

사용자가 앱을 열면 Braze는 세션 수가 1로 표시되며 더 이상 오디언스 자격을 얻지 못합니다. 응모 이벤트는 자격을 갖춘 후에 발생하므로 메시징이 전송되지 않습니다.

이 기능을 사용하려면 세션이 시작되기 전에 사용자가 오디언스에 대한 자격을 부여해야 합니다(타겟 오디언스 및 진입 트리거를 뒤집음).

### 예 2

지난 7일 동안 10달러 이상을 지출한 사용자에게 이메일을 보내려고 합니다.

설정했습니다:

- **타겟 오디언스:** 지난 7일 동안 10달러 이상을 지출한 사용자
- **응모 이벤트:** 모든 구매

이제 사용자가 오늘 12달러를 지출한다고 가정해 보겠습니다. 이는 메시지를 트리거하는 것이 아니라 오디언스에 입장할 수 있는 자격을 부여할 뿐입니다. 나중에 다른 구매를 하지 않으면 이메일을 받지 못합니다.

더 나은 접근 방식은 더 넓은 오디언스를 사용하고 필터를 진입 기준으로 전환하는 것입니다:

- **오디언스:** 모든 사용자(또는 기본 오디언스)
- **응모 이벤트:** 구매하기
- **항목 필터:** 지난 7일간 총 지출액 $10 이상

이렇게 하면 적격 구매가 필터를 충족하고 메시징을 트리거할 때 별도의 추가 작업이 필요하지 않습니다.

## 모범 사례

- 진입 기준이 발생하기 전에 오디언스 세그먼트에 사용자가 포함되어 있는지 확인하세요.
- 이벤트 이후에만 적용되는 오디언스 필터는 사용하지 마세요. 필터가 트리거 시점에 발생하는 상황에 따라 달라지는 경우(예: "세션 수 = 0"), Braze가 확인할 때 해당 사용자는 더 이상 자격을 갖추지 못할 수 있습니다.
- 시간 기반 논리를 신중하게 사용하세요. 예를 들어 신규 사용자를 타겟팅하려는 경우입니다:
    - 타겟 오디언스를 "최근 7일 이내에 처음 사용한 앱"으로 설정합니다.
    - 참가 이벤트를 '세션 시작'으로 설정합니다.
    - 이렇게 하면 아직 첫 주 내에 있는 사용자만 세션을 시작할 때 자격이 주어지고 입장할 수 있습니다.
