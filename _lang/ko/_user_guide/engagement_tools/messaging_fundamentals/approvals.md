# 캠페인 및 캔버스 승인

> 승인 기능을 사용하여 캠페인 및 캔버스를 출시하기 전에 최종 점검 단계를 추가하세요. 이 워크플로를 통해 메시지의 모든 필수 섹션에 포함된 내용을 확인하고 승인할 수 있습니다.

## 작동 방식

캠페인 또는 캔버스의 세부 사항은 편집의 마지막 단계에서 검토할 수 있습니다. 

캔버스와 캠페인 모두에서, 승인하기 전에 모든 변경 사항을 저장해야 합니다. 본인 변경 사항이라도 예외는 없습니다. 메시지가 발송되기 전에 요약문의 각 섹션은 적절한 권한을 가진 사용자의 승인을 받아야 합니다. 각 섹션의 기본 상태는 **승인 보류 중입니다**.

{% tabs %}
{% tab campaign %}
캠페인을 시작하려면 다음 구성 요소를 승인해야 합니다:

- **메시지:** 이것이 캠페인 메시지입니다.
- **전달:** 이것은 전달 유형이며 사용자가 캠페인을 수신하는 시점을 결정합니다.
- **대상 고객:** 이것은 캠페인을 누가 받을지 결정합니다.
- **전환 이벤트:** 참여도 및 보고 목적으로 추적하는 지표입니다.
{% endtab %}

{% tab canvas %}
캔버스를 시작하려면 이러한 주요 구성 요소를 승인해야 합니다:

- **전환 이벤트:** 참여도 및 보고 목적으로 추적하는 지표입니다.
- **참가 일정:** 여기에는 입력 일정 유형과 사용자가 캔버스에 입력하는 시점이 포함됩니다.
- **대상 고객:** 이에 따라 이 캔버스에 들어갈 사람이 결정됩니다.
- **설정 보내기:** 다음은 캔버스의 모든 단계에 대한 전송 옵션입니다. 
- **캔버스 만들기:** 이것이 바로 캔버스 사용자 여정입니다.

## 승인 워크플로 켜기

기본값으로 캠페인 및 캔버스에 대한 승인 워크플로 설정은 꺼져 있습니다. 이 기능을 사용 **설정하려면 설정** > **승인 워크플로로** 이동하여 해당 토글을 선택하세요:

- **[사용자 작업 공간]의 모든 캠페인에 승인 워크플로를 사용하십시오.**
- **[사용자 작업 공간]의 모든 캔버스에 승인 워크플로를 사용하세요**

{% alert important %}
캠페인 승인은 [API 캠페인]({{site.baseurl}}/api/api_campaigns) 및 [트랜잭션 이메일 캠페인]({{site.baseurl}}/user_guide/message_building_by_channel/email/transactional_message_api_campaign)에서는 지원되지 않습니다.
{% endalert %}

## 사용자 권한 설정

승인 워크플로를 활성화한 후에는 회사 사용자가 캠페인과 캔버스를 승인하거나 거부할 수 있도록 사용자 권한을 설정해야 합니다. 두 권한 모두 작업 영역이나 [Teams에]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/teams/) 적용하거나 [권한 ]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/#permission-sets)세트에 추가할 수 있습니다.

{% tabs %}
{% tab campaign %}
['캠페인 승인 및 거부' 권한이]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/#managing-limited-and-team-role-permissions) 있어야 합니다. 이 권한은 캠페인의 승인 상태를 업데이트할 수 있는 사용자를 제어합니다. 이 권한으로 다음을 수행할 수 있습니다:

- 캠페인 자체 승인
- 캠페인 승인 및 실행
- 캠페인을 승인하지만 실행하지 않음 (다른 사용자가 "캠페인, 캔버스 발송" 권한을 가지고 캠페인을 실행할 수 있음)
- 캠페인을 승인하거나 시작하지 마십시오

**요약** 단계에서 승인 상태가 설정된 후, 캠페인에 대한 이후 변경 사항은 저장 시 모든 승인 상태를 재설정합니다. 이것은 초안 캠페인 또는 출시 후 캠페인에서 이루어진 모든 변경 사항에 적용됩니다. 예를 들어, 오디언스만 변경하는 경우 **요약** 단계에서 모든 섹션의 승인 상태가 기본값인 **'승인 대기** 중'으로 되돌려집니다.

{% endtab %}

{% tab canvas %}
["캔버스 승인 및 거부" 권한이]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/#managing-limited-and-team-role-permissions) 있어야 합니다. 이 권한은 캔버스의 승인 상태를 업데이트할 수 있는 사용자를 제어합니다. 이 권한으로 다음을 수행할 수 있습니다:

- 캔버스를 직접 승인하세요
- 승인하고 캔버스를 시작하세요
- 캔버스를 승인하지만 실행하지 않음(다른 사용자가 "캠페인, 캔버스 발송" 권한을 가지고 있을 경우 캔버스를 실행할 수 있음)
- 캔버스를 승인하거나 실행하지 마십시오

**요약** 단계에서 승인 상태가 설정된 후, 캔버스에 대한 이후 변경 사항은 저장 시 모든 승인 상태를 초기화합니다. 이것은 초안 캔버스 또는 출시 후 캔버스에서 이루어진 모든 변경 사항에 적용됩니다. 예를 들어, 오디언스만 변경하는 경우 **요약** 단계에서 모든 섹션의 승인 상태가 기본값인 **'승인 대기** 중'으로 되돌려집니다.

{% alert note %}
**승인 상태 및 저장**

- **요약** 단계에서 섹션에 대해 **'승인'을** 클릭하면 해당 승인이 즉시 저장됩니다.
- **저장** 버튼은 캔버스 콘텐츠와 설정에 대한 변경 사항을 저장하며, 승인 상태는 저장하지 않습니다.

승인 취소 방지를 위해:

1. 캔버스에서 필요한 편집을 한 후 **저장을** 클릭하세요.
2. 캔버스가 저장을 완료한 후, **요약** 단계에서 관련 섹션을 승인하십시오.
3. 승인 후 캔버스에 추가 변경 사항이 있을 경우에만 다시 **'저장'을** 클릭하십시오. 캔버스를 변경하고 저장하면 모든 승인 상태가 **'승인 대기** 중'으로 재설정됩니다.
{% endalert %}

{% alert important %}
실행 중인 캠페인을 편집하려면 "캠페인 승인 및 거부" 권한이 필요합니다. 사용자는 캠페인의 초안 버전이 아직 제공되지 않으므로 변경 사항을 반드시 승인해야 합니다. 이것은 캔버스의 경우가 아닙니다. 사용자가 변경을 하고 초안으로 저장할 수 있으며, 다른 사용자가 캔버스를 승인하고 시작할 수 있습니다.
{% endalert %}
