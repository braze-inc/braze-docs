# 캠페인 및 캔버스 승인

> 승인을 사용하여 캠페인과 캔버스를 출시하기 전에 최종 점검 사항을 추가하세요. 이 워크플로우를 사용하면 메시징의 모든 필수 섹션에서 콘텐츠를 확인하고 승인할 수 있습니다.

## 작동 방식

편집의 마지막 단계에서 캠페인 또는 캔버스의 세부 정보를 검토할 수 있습니다. 

캔버스와 캠페인 모두 본인이 직접 변경한 내용이라도 승인하기 전에 모든 변경 내용을 저장해야 합니다. 적절한 권한을 가진 사용자가 요약의 각 섹션을 승인해야 메시징을 시작할 수 있습니다. 각 섹션의 기본 상태는 **승인 보류 중입니다**.

{% tabs %}
{% tab campaign %}
캠페인을 시작하려면 이러한 구성 요소를 승인해야 합니다:

- **메시지:** 이것이 캠페인 메시지입니다.
- **전달:** 전달 유형이며 사용자가 캠페인을 받는 시기를 결정합니다.
- **대상 고객:** 이에 따라 캠페인을 수신할 대상자가 결정됩니다.
- **전환 이벤트:** 참여도 및 보고 목적으로 추적하는 지표입니다.
{% endtab %}

{% tab canvas %}
캔버스를 시작하려면 이러한 주요 구성 요소를 승인해야 합니다:

- **전환 이벤트:** 참여도 및 보고 목적으로 추적하는 지표입니다.
- **참가 일정:** 여기에는 입력 일정의 유형과 사용자가 캔버스에 입력하는 시기가 포함됩니다.
- **대상 고객:** 이에 따라 이 캔버스에 들어갈 사람이 결정됩니다.
- **설정 보내기:** 다음은 캔버스의 모든 단계에 대한 전송 옵션입니다. 
- **캔버스 만들기:** 이것이 바로 캔버스 사용자 여정입니다.
{% endtab %}

## 승인 워크플로 켜기

기본적으로 캠페인 및 캔버스에 대한 승인 워크플로 설정은 꺼져 있습니다. 이 기능을 사용 **설정하려면 설정** > **승인 워크플로로** 이동하여 해당 토글을 선택하세요:

- **내 워크스페이스]의 모든 캠페인에 승인 워크플로우 사용**
- **내 작업 공간]의 모든 캔버스에 승인 워크플로우 사용**

{% alert important %}
[API 캠페인]({{site.baseurl}}/api/api_campaigns) 및 [트랜잭션 이메일 캠페인에는]({{site.baseurl}}/user_guide/message_building_by_channel/email/transactional_message_api_campaign) 캠페인 승인이 지원되지 않습니다.
{% endalert %}

## 사용자 권한 설정

승인 워크플로를 켠 후에는 회사 사용자가 캠페인과 캔버스를 승인하거나 거부할 수 있도록 사용자 권한을 설정해야 합니다. 두 권한 모두 워크스페이스 또는 [Teams에]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/teams/) 적용하거나 [권한 집합에]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/#permission-sets) 추가할 수도 있습니다.

{% tabs %}
{% tab campaign %}
['캠페인 승인 및 거부' 권한이]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/#managing-limited-and-team-role-permissions) 있어야 합니다. 이 권한은 캠페인의 승인 상태를 업데이트할 수 있는 사용자를 제어합니다. 이 권한이 있으면 다음을 수행할 수 있습니다:

- 캠페인 자체 승인하기
- 캠페인 승인 및 시작하기
- 승인하지만 캠페인을 시작하지는 않습니다('캠페인, 캔버스 보내기' 권한이 있는 다른 사용자가 캠페인을 시작할 수 있음).
- 캠페인을 승인하거나 시작하지 않습니다.

**요약** 단계에서 승인 상태를 설정한 후 캠페인을 나중에 변경하면 저장 시 모든 승인 상태가 초기화됩니다. 이는 캠페인 초안 또는 출시 후 캠페인에서 변경한 모든 사항에 적용됩니다. 예를 들어 타겟 오디언스만 변경하는 경우 **요약** 단계에서는 모든 섹션의 승인 상태가 기본값인 **승인 보류 중** 상태로 되돌아갑니다.

{% endtab %}

{% tab canvas %}
["캔버스 승인 및 거부" 권한이]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/#managing-limited-and-team-role-permissions) 있어야 합니다. 이 권한은 캔버스의 승인 상태를 업데이트할 수 있는 사용자를 제어합니다. 이 권한이 있으면 다음을 수행할 수 있습니다:

- 캔버스 자체 승인하기
- 승인하고 캔버스를 시작하세요
- 캔버스를 승인하지만 실행하지는 않습니다('캠페인, 캔버스 보내기' 권한이 있는 다른 사용자가 캔버스를 실행할 수 있음).
- 캔버스를 승인하거나 실행하지 않습니다.

**요약** 단계에서 승인 상태를 설정한 후 이후에 캔버스를 변경하면 저장된 모든 승인 상태가 초기화됩니다. 이것은 초안 캔버스 또는 출시 후 캔버스에서 이루어진 모든 변경 사항에 적용됩니다. 예를 들어 타겟팅 오디언스만 변경하는 경우 **요약** 단계에서는 모든 섹션의 승인 상태가 기본값인 **승인 보류 중** 상태로 되돌아갑니다.

{% alert note %}
**승인 상태 및 저장**

- **요약** 단계에서 섹션에 대한 **승인을** 클릭하면 해당 승인은 즉시 저장됩니다.
- **저장** 버튼은 승인 상태가 아닌 캔버스 콘텐츠 및 설정에 대한 변경 사항을 저장합니다.

승인을 잃지 않으려면:

1. 필요한 캔버스를 편집한 다음 **저장을** 클릭합니다.
2. 캔버스 저장이 완료되면 **요약** 단계에서 관련 섹션을 승인합니다.
3. 승인 후 캔버스를 추가로 변경하는 경우에만 **저장을** 다시 클릭합니다. 캔버스를 변경하고 저장하면 모든 승인 상태가 **승인 보류 중으로** 재설정됩니다.
{% endalert %}
{% endtab %}
{% endtabs %}

{% alert important %}
라이브 캠페인을 편집하려면 '캠페인 승인 및 거부' 권한이 필요합니다. 아직 캠페인 초안 버전을 사용할 수 없으므로 사용자가 변경 사항을 승인해야 합니다. 이것은 캔버스의 경우가 아닙니다. 사용자가 변경을 하고 초안으로 저장할 수 있으며, 다른 사용자가 캔버스를 승인하고 시작할 수 있습니다.
{% endalert %}
