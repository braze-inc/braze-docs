# 메시지 예약하기

> Braze에서는 예약, 액션 기반, API 트리거 등 세 가지 방법으로 메시지를 예약할 수 있습니다. 메시지를 전달하는 방법과 시기를 선택하는 것은 효과적인 메시지를 개발하는 데 매우 중요합니다. 

## 배송 및 항목 유형

캠페인의 경우 전달 유형에 따라 사용자가 캠페인에 입장하는 시기와 캠페인이 전송되는 시기가 결정됩니다. 캔버스는 진행 중인 사용자 여정으로 구축되므로 스케줄링의 메시징 개념을 항목 유형이라고 합니다.

| 배달<nobr> 및 참가 유형 | 설명                                                                                                                                                                                                                                                                                                                                      |
|---------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **스케줄**       | 이 유형의 일정은 시사 이벤트에 대한 캠페인과 같이 즉시 전송하려는 일회성 메시지를 위해 고안되었습니다. <br><br>본인 또는 팀만을 대상으로 테스트 메시지를 보낼 때 이 옵션을 사용하면 즉시 메시지를 전달할 수 있습니다.                                                                                   |
| **실행 기반**    | 액션 기반 전달 메시지 또는 이벤트 트리거 캠페인과 캔버스는 트랜잭션 또는 업적 기반 메시지에 매우 효과적입니다. 특정 날짜에 메시지를 보내는 대신 사용자가 특정 이벤트를 완료한 후에 메시지를 보내도록 트리거할 수 있습니다.                                                                                           |
| **API 트리거됨**   | API 트리거 메시지를 사용하면 자체 서버 및 시스템에서 해당 콘텐츠의 전송을 트리거하면서 Braze 대시보드에서 메시지 복사, 다변량 테스트 및 재적격성 규칙을 관리할 수 있습니다. <br><br>API 요청으로 메시지를 트리거할 때 실시간으로 메시지에 템플릿화할 추가 데이터를 포함할 수도 있습니다. |

### 시간 기반 옵션

{% tabs %}
{% tab 캠페인 %}
예약 배송을 사용할 때는 다음 옵션 중에서 선택할 수 있습니다:

- 캠페인이 시작되는 즉시 보내기
- 지정한 시간에 발송
- [Intelligent Timing]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_timing/)
{% endtab %}

{% tab 캔버스 %}
예약된 전달을 통해 사용자는 캠페인을 예약하는 방식과 유사하게 시간 스케줄에 따라 입력하게 됩니다. 캔버스가 시작되는 즉시 또는 지정된 시간에 사용자를 등록할 수 있습니다.

#### 지정 시간

한 번만, 매일, 매주 또는 매월 등 특정 입력 주기로 캔버스를 전송하도록 선택할 수 있습니다. 예약 배송이 반복되는 캔버스의 경우, 사용자가 지정된 30회까지 캔버스에 입장할 수 있도록 반복을 설정할 수 있습니다.
{% endtab %}
{% endtabs %}

### 작업 기반 옵션

{% tabs %}
{% tab 캠페인 %}
액션 기반 전달은 특정 액션을 수행하는 사용자에게 캠페인을 전송합니다. 이 작업이 수행된 후 즉시, 특정 시간 이후, 특정 시간 또는 미래의 특정 시점에 캠페인을 보낼 시기를 결정할 수 있습니다.
{% endtab %}

{% tab 캔버스 %}
작업 기반 옵션은 사용자가 캔버스에 들어가기 위해 수행해야 하는 작업(또는 트리거)과 입력 시작이 허용되는 특정 시간을 결정합니다. 예를 들어 다음과 같은 작업으로 사용자를 평가할 수 있습니다:

- 앱 열기
- 이메일 주소 추가하기
- 위치 입력

#### 입력 창

캔버스의 입력 창은 지정된 시작 시간(및 선택적 종료 시간)에 캔버스에 입장할 수 있는 사용자를 결정합니다. 액션 기반 캠페인과 마찬가지로 현지 시간대의 사용자를 입력하도록 선택할 수 있습니다.
{% endtab %}
{% endtabs %}

### API 트리거 옵션

{% tabs %}
{% tab 캠페인 %}
전달 옵션으로 API 트리거를 선택하면 [`/campaigns/trigger/send` 엔드포인트로]({{site.baseurl}}/api/endpoints/messaging/send_messages/post_send_triggered_campaigns/#prerequisites) 전송할 캠페인을 식별할 수 있는 캠페인 ID를 받게 됩니다.
{% endtab %}

{% tab 캔버스 %}
입력 유형으로 API 트리거를 선택하면 [`/canvas/trigger/send` 엔드포인트로]({{site.baseurl}}/api/endpoints/messaging/send_messages/post_send_triggered_canvases) 보낼 캠페인을 식별할 수 있는 캔버스 ID를 받게 됩니다.
{% endtab %}
{% endtabs %}
