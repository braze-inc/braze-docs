# 전환 이벤트

> 전환 이벤트는 메시지 수신자가 메시지 참여 후 설정된 시간 내에 높은 가치의 액션을 수행하는지 여부를 추적하는 일종의 성공 측정기준입니다. 이러한 이벤트를 사용하여 나중에 캠페인이나 캔버스에 대한 인사이트를 얻는 데 사용할 수 있는 관련성 있고 유용한 정보를 수집할 수 있습니다.

## 작동 방식

활성 사용자를 위한 개인화된 휴가 캠페인을 만든다고 가정할 때, 메시지 수신으로 사용자 참여도를 파악할 수 있으므로 2~3일 이내에 **세션 시작이라는** 전환 이벤트가 적절할 수 있습니다. **구매하기**, **앱 업그레이드** 또는 커스텀 이벤트와 같은 추가 이벤트를 전환 이벤트로 선택할 수 있습니다.

{% alert tip %}
전환에 대한 자세한 내용은 캠페인 설정에 대한 [Braze 학습 과정을](https://learning.braze.com/campaign-setup-delivery-targeting-conversions) 확인하세요.
{% endalert %}

### 전환 추적 규칙

전환 이벤트를 사용하면 사용자 행동을 참여 시점으로 다시 속성화할 수 있습니다. 하지만 Braze가 다중 전환을 처리하는 방식과 관련하여 몇 가지 주의해야 할 사항이 있습니다. 다음 시나리오를 통해 Braze가 이러한 전환을 추적하는 방법을 확인하세요:

- 전환은 기기 단위가 아닌 사용자 단위로 이루어집니다. 즉, 사용자는 메시징을 여러 기기로 전송하더라도 한 번만 변환할 수 있습니다. 다른 예로, 캠페인에 '구매하기'라는 전환 이벤트가 하나만 있다고 가정해 보겠습니다. 이 캠페인을 수신한 사용자가 전환 기한 내에 두 번의 별도 구매를 하는 경우 전환은 한 번만 계산됩니다.
- 사용자가 수신한 두 개의 개별 캠페인 또는 캔버스의 전환 기한 내에 하나의 전환 이벤트를 수행하면 전환이 두 캠페인 모두에 등록됩니다.
- 사용자가 메시지를 열거나 클릭하지 않았더라도 창에서 특정 전환 이벤트를 수행한 경우 전환된 것으로 계산됩니다.
- 캔버스의 경우 전환 추적은 개별 메시지 타이밍이 아닌 사용자가 캔버스에 들어올 때 시작되는 최종 전환 기한을 기준으로 작동합니다. 즉, 캔버스에서 메시지 간의 지연 기간 동안에도 전환을 계산할 수 있습니다.

### 주요 전환 이벤트

주요 전환 이벤트는 캠페인 또는 캔버스 생성 중에 추가되는 첫 번째 이벤트입니다. 이 이벤트는 참여 및 보고와 가장 밀접한 관련이 있습니다. 주요 전환 이벤트에 익숙합니다:

- [다변량]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/#multivariate-and-ab-testing) 캠페인 또는 캔버스에서 승리한 메시지 변형을 계산합니다.
- 캠페인 또는 캔버스에 대한 매출이 계산되는 기간을 결정합니다.
- [지능형 선택을]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_selection/) 사용하여 캠페인 및 캔버스에 대한 메시지 배포를 조정합니다.

{% alert note %}
Liquid `abort` 태그를 사용하여 메시징이 중단되면 배리언트를 통과하는 사용자만 잠재적으로 중단될 수 있습니다. 즉, 대조군을 통과한 사용자에게 보내는 메시징이 중단되지 않으므로 배리언트와 대조군 간에 전환율이 왜곡될 수 있습니다. 해결 방법으로 [세그먼트를 세분화하여]({{site.baseurl}}/user_guide/engagement_tools/segments/creating_a_segment) 캠페인 및 캔버스 항목에서 사용자를 타겟팅하세요.
{% endalert %}

## 전환 추적 기능이 있는 캠페인 만들기

### 1단계: 캠페인 설정하기

원하는 메시징 채널에 대한 [캠페인을 만듭니다]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/creating_campaign). 캠페인 메시지와 일정을 설정한 후에는 추적을 위해 최대 4개의 전환 이벤트를 추가할 수 있는 옵션이 제공됩니다.

두 번째(또는 세 번째) 전환 이벤트를 추가하면 보고가 훨씬 더 풍부해질 수 있으므로 필요하다고 생각하는 만큼 전환 이벤트를 사용하는 것이 좋습니다. 예를 들어 휴면 사용자를 타겟팅하는 캠페인이 있다고 가정해 보겠습니다. 이 경우 보조 전환 이벤트와 주요 **시작 세션** 전환 이벤트를 추가하면 캠페인이 사용자를 애플리케이션으로 다시 유도하는 데 얼마나 효과적인지 더 잘 이해할 수 있습니다. 

### 2단계: 전환 이벤트 추가하기

먼저 사용하려는 일반 이벤트 유형을 선택합니다:

| 전환 이벤트 유형         | 설명                                                                                                                                                                                                                                                                                                                                 |
|-------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **세션 시작**      | 사용자가 지정한 앱 중 하나라도 열면 전환한 것으로 계산됩니다(기본값은 작업 영역의 모든 앱).                                                                                                                                                                                                         |
| **구매하기**      | 사용자가 지정한 제품을 구매하면 전환한 것으로 계산됩니다(기본값은 모든 제품).                                                                                                                                                                                                                                 |
| **커스텀 이벤트 수행** | 사용자가 기존 커스텀 이벤트 중 하나를 수행하면 전환한 것으로 계산됩니다(기본값은 없으며, 이벤트를 지정해야 함).                                                                                                                                                                                                        |
| **앱 업그레이드**         | 사용자가 지정한 앱 중 하나에서 앱 버전을 업그레이드하면 전환한 것으로 계산됩니다(기본값은 워크스페이스의 모든 앱으로 설정됨). Braze는 변경 사항이 업그레이드인지 확인하기 위해 최선의 수치 비교를 수행합니다. 숫자가 아닌 버전은 버전이 변경되는 경우 전환으로 계산됩니다.              |
| **이메일 열기**         | 사용자가 이메일을 열면 전환한 것으로 계산됩니다(이메일 캠페인의 경우에만 해당).                                                                                                                                                                                                                                                 |
| **이메일 클릭**        | 사용자가 이메일 내의 링크를 클릭하면 전환한 것으로 계산됩니다(이메일 캠페인의 경우에만 해당).                                                                                                                                                                                                                                  |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

전환 기한을 설정하세요. 전환을 고려할 수 있는 최대 시간입니다. 사용자가 지정된 작업을 수행하면 최대 30일 동안 전환이 카운트되도록 허용하는 옵션이 있습니다.

구매를 한 사용자의 전환을 기록하는 "구매" 전환 이벤트 유형을 예로 들어 보겠습니다. 전환 기한은 12시간입니다.]({% image_buster /assets/img_archive/conversion_event_selection.png %})

전환 이벤트를 선택한 후 캠페인 생성 프로세스를 계속 진행하고 캠페인 전송을 시작하세요.

### 3단계: 결과 보기

**세부 정보** 페이지로 이동하여 방금 생성한 캠페인과 관련된 각 전환 이벤트에 대한 세부 정보를 확인합니다. 선택한 전환 이벤트에 관계없이 주요 전환 이벤트 기간 동안 이 특정 캠페인에 기여할 수 있는 총 매출과 특정 배리언트를 확인할 수도 있습니다.

{% alert note %}
캠페인 생성 시 전환 이벤트를 선택하지 않은 경우 기본값은 3일입니다.
{% endalert %}

또한 다변량 메시지의 경우 대조군 및 각 배리언트에 대한 전환 수와 전환 비율을 확인할 수 있습니다.

3시간 이내 구매, 2시간 이내 구매, 30분 이내 세션 시작, 25분 이내 세션 시작 시점을 기준으로 전환을 추적하는 4가지 전환 이벤트가 있습니다.]({% image_buster /assets/img_archive/conversion_event_details.png %})
