# 전환 이벤트

> 전환 이벤트는 메시징 수신자가 참여를 받은 후 설정된 시간 내에 높은 가치의 액션을 수행하는지 여부를 추적하는 일종의 성공 지표입니다. 이러한 이벤트를 사용하여 나중에 캠페인이나 캔버스에 대한 인사이트를 얻는 데 사용할 수 있는 관련성 있고 유용한 정보를 수집할 수 있습니다.

## 작동 방식

활성 사용자를 타겟팅하는 개인화된 홀리데이 캠페인의 경우, 메시지 수신으로부터 사용자 참여도를 파악할 수 있는 '2~3일 내 **세션 시작** '이라는 전환 이벤트가 적절할 수 있습니다. **주문하기**, **앱 업그레이드** 또는 커스텀 이벤트와 같은 추가 이벤트를 전환 이벤트로 선택할 수도 있습니다.

{% alert tip %}
전환에 대한 자세한 내용은 캠페인 설정에 대한 [Braze 학습 과정을](https://learning.braze.com/campaign-setup-delivery-targeting-conversions) 확인하세요.
{% endalert %}

### 전환 추적 규칙

전환 이벤트는 사용자 행동을 참여 시점으로 되돌리는 속성입니다. Braze에서 다중 전환을 처리하는 방법에 대해서는 다음 사항을 참고하세요:

- **단일 채널 캠페인**: 전환은 사용자별로 발생하며, 기기별로 발생하지 않습니다. 단일 채널 내에서 사용자는 여러 기기에 메시지를 보내더라도 전환 이벤트당 한 번만 전환할 수 있습니다. 예를 들어, 캠페인에 '모든 구매'로 설정된 전환 이벤트가 하나만 있고 사용자가 전환 기한 내에 두 번의 별도 구매를 하는 경우, Braze는 전환을 한 번만 계산합니다.
- **멀티채널 캠페인**: 멀티채널 캠페인의 경우 각 채널마다 고유한 전환 기회가 있습니다. 사용자는 해당 채널에서 메시지를 받은 후 채널당 한 번만 전환할 수 있습니다. 즉, 사용자가 여러 채널(예: 이메일과 푸시 모두)에서 메시지를 수신하고 전환 액션을 수행하면 Braze는 각 채널당 하나의 전환을 계산하므로 전환율이 100%를 초과할 수 있습니다.
- 사용자가 수신한 두 개의 개별 캠페인 또는 캔버스의 전환 기한 내에 하나의 전환 이벤트를 수행하면 전환이 두 캠페인 모두에 등록됩니다.
- 사용자가 메시지를 열거나 클릭하지 않았더라도 창에서 특정 전환 이벤트를 수행한 경우 전환된 것으로 간주합니다.
- 캔버스의 경우 전환 추적은 개별 메시지 타이밍이 아닌 사용자가 캔버스에 들어올 때 시작되는 최종 전환 기한을 기준으로 작동합니다. Braze는 캔버스에서 메시지 사이의 지연 기간 동안에도 전환을 계산합니다.

### 주요 전환 이벤트

주요 전환 이벤트는 캠페인 또는 캔버스를 만드는 동안 가장 먼저 추가하는 이벤트입니다. 이 이벤트는 인게이지먼트 및 보고에 가장 큰 영향을 미칩니다. Braze는 주요 전환 이벤트를 사용합니다:

- [다변량]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/#multivariate-and-ab-testing) 캠페인 또는 캔버스에서 우승한 메시지 변형을 계산합니다.
- 캠페인 또는 캔버스의 매출이 계산되는 창을 결정합니다.
- [지능형 선택을]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_selection/) 사용하여 캠페인 및 캔버스에 대한 메시지 배포를 조정합니다.

주요 전환 이벤트 수는 발생한 전환 이벤트의 수입니다. 멀티채널 캠페인의 경우 Braze는 채널당 전환 수를 계산하므로( [전환 추적 규칙에](#conversion-tracking-rules) 설명된 대로) 전환 수가 순 사용자 수를 초과하여 전환율이 100%를 초과할 수 있습니다. Braze는 이 수를 고유 수신자 수로 나누어 주요 전환 이벤트 비율을 계산합니다. Braze는 메시징 채널에 따라 메시지가 전송되거나 표시될 때 사용자를 수신자로 간주합니다. 예를 들어 푸시나 이메일에서 사용자는 Braze가 메시지를 보낸 후 수신자가 됩니다. 인앱 메시지 또는 콘텐츠 카드의 경우 사용자가 메시지를 확인해야 수신자로 간주됩니다.

{% alert note %}
Liquid `abort` 태그를 사용하여 메시지를 중단하는 경우, Braze는 배리언트를 통과한 사용자에 대해서만 메시지를 중단합니다. 대조군의 사용자에게 보내는 메시지는 중단되지 않으므로 배리언트 및 대조군 간에 전환율이 왜곡될 수 있습니다. 해결 방법으로 [세분화]({{site.baseurl}}/user_guide/engagement_tools/segments/creating_a_segment)를 사용하여 캠페인 및 캔버스 항목에서 사용자를 타겟팅하십시오.
{% endalert %}

## 전환 추적 기능이 있는 캠페인 만들기

### 1단계: 캠페인 설정

원하는 메시징 채널에 대해 [캠페인을 생성하세요]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/creating_campaign). 캠페인의 메시지와 일정을 설정한 후 추적을 위해 최대 4개의 전환 이벤트를 추가할 수 있습니다.

필요한 만큼 전환 이벤트를 사용하세요. 두 번째 또는 세 번째 전환 이벤트를 추가하면 보고가 훨씬 더 풍부해집니다. 예를 들어, 휴면 사용자를 타겟팅하는 캠페인의 경우 주요 **세션 시작** 전환 이벤트와 함께 보조 전환 이벤트를 추가하면 캠페인이 사용자를 애플리케이션으로 다시 유도하는 데 얼마나 효과적인지 파악하는 데 도움이 됩니다. 

### 2단계: 전환 이벤트 추가

먼저 사용하려는 일반 이벤트 유형을 선택합니다:

| 전환 이벤트 유형   | 설명                |
|-------------------------|----------------------------|
| **세션 시작**      | 사용자가 변환된 것으로 간주되는 경우는 사용자가 지정한 앱 중 하나를 열 때입니다(기본값은 워크스페이스의 모든 앱입니다).|
| **구매**      | 사용자가 [구매 이벤트를]({{site.baseurl}}/api/objects_filters/purchase_object/) 기록하면 전환한 것으로 계산됩니다. 기본값으로 모든 구매를 추적하거나 특정 제품을 지정할 수 있습니다.|
| **주문하기**        | 사용자가 [이커머스 주문 완료 추천 이벤트를]({{site.baseurl}}/user_guide/data/activation/custom_data/recommended_events/ecommerce_events?tab=ecommerce.order_placed#ecommerce-recommended-events) 트리거하면 전환한 것으로 계산됩니다. 기본값으로 모든 주문을 추적하거나 특정 제품을 기준으로 필터링할 수 있습니다.|
| **커스텀 이벤트 수행**| 사용자가 기존 커스텀 이벤트 중 하나를 수행하면 전환된 것으로 간주됩니다 (기본값 없음, 이벤트를 지정해야 합니다).|
| **앱 업그레이드**         | 사용자가 변환된 것으로 간주되는 경우는 지정한 앱 중 하나에서 앱 버전을 업그레이드할 때입니다(기본값은 워크스페이스의 모든 앱입니다). Braze는 변경 사항이 업그레이드인지 확인하기 위해 최선의 수치 비교를 수행합니다. 숫자가 아닌 버전은 버전이 변경되는 경우 전환으로 계산됩니다.|
| **이메일 열람**         | 사용자가 이메일을 열면 (이메일 캠페인에 한함) 전환된 것으로 간주됩니다.|
| **이메일 클릭**        | 사용자가 이메일(이메일 캠페인에 한함) 내의 링크를 클릭할 때 전환된 것으로 간주됩니다.|
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

{% alert important %}
**중첩된 이벤트 속성정보는 전환 이벤트에서 지원되지 않습니다**. 전환 이벤트에는 중첩된 이벤트 속성정보를 사용할 수 없습니다. 예를 들어 `product_code` 또는 `product_name` 이 `products` 배열(예: `products[].product_code`) 내에 중첩된 이벤트 속성정보인 경우 전환 이벤트에서 특정 제품 구매가 이루어졌는지 확인하는 데 사용할 수 없습니다.
{% endalert %}

전환 기한을 설정하세요. 이 시간은 Braze가 전환을 고려하기 전까지 경과할 수 있는 최대 시간입니다. 사용자가 지정된 작업을 수행하면 Braze가 전환을 계산하는 기간을 최대 30일로 설정할 수 있습니다.

!["구매 완료" 전환 이벤트 유형은 사용자가 구매를 완료할 때 전환을 기록하는 예입니다. 이것은 12시간의 전환 마감 시간을 가지고 있습니다.]({% image_buster /assets/img_archive/conversion_event_selection.png %})

전환 이벤트를 선택한 후 캠페인 생성 프로세스를 계속하고 캠페인 전송을 시작하세요.

### 3단계: 결과 보기

**세부 정보** 페이지로 이동하여 생성한 캠페인과 관련된 각 전환 이벤트에 대한 세부 정보를 확인합니다. 선택한 전환 이벤트에 관계없이 주요 전환 이벤트 기간 동안 이 특정 캠페인에 기여한 총 매출과 특정 배리언트를 확인할 수도 있습니다.

{% alert note %}
캠페인을 만드는 동안 전환 이벤트를 선택하지 않으면 기본값은 3일입니다.
{% endalert %}

또한 다변량 메시지의 경우 대조군 및 각 배리언트에 대한 전환 수 및 전환 비율을 확인할 수 있습니다.
