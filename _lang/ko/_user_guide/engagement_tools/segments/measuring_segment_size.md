# 세그먼트 크기 측정

> 이 페이지에서는 세그먼트의 멤버십 및 규모를 모니터링하는 방법에 대해 설명합니다.

## 세그먼트 멤버십 계산

Braze는 데이터가 서버로 전송되어 처리되는 즉시 사용자의 세그먼트 멤버십을 업데이트하며, 이는 일반적으로 즉시 이루어집니다. 사용자의 세그먼트 멤버십은 해당 세션이 처리될 때까지 변경되지 않습니다. 예를 들어 세션이 처음 시작될 때 경과된 사용자 세그먼트에 속하는 사용자는 세션이 처리되면 즉시 경과된 사용자 세그먼트에서 이동합니다.

### 총 도달 가능한 사용자 계산

각 세그먼트는 해당 세그먼트의 구성원인 사용자 수를 표시합니다. **모든 앱의 사용자를** 필터링할 때 가장 자주 사용되는 메시징 채널(예: 웹 푸시 또는 이메일)과 해당 특정 채널에 도달 가능한 사용자 수도 표시합니다. 

총 사용자 수가 각 채널에서 도달할 수 있는 사용자 수와 다를 수 있습니다. 또한 도달 가능한 사용자 표에 모든 채널이 나열되는 것은 아닙니다. 예를 들어 콘텐츠 카드, 웹훅 및 WhatsApp은 분석에 표시되지 않습니다. 즉, 도달 가능한 총 사용자 수가 표시된 각 채널의 사용자 수를 합한 것보다 클 수 있습니다.

![이메일, iOS 푸시, Android 푸시, 웹 푸시, Kindle 푸시 및 Android China 푸시로 도달할 수 있는 사용자를 세분화하여 총 도달 가능한 사용자를 표시하는 표입니다.][3]

특정 채널을 통해 도달 가능한 사용자로 표시되려면 사용자가 두 가지 채널을 모두 보유해야 합니다.
* 유효한 이메일 주소 또는 프로필과 연결된 푸시 토큰; 및
* 앱에 가입하거나 구독했습니다.

단일 사용자는 서로 다른 접근 가능한 사용자 그룹에 속할 수 있습니다. 예를 들어, 사용자가 유효한 이메일 주소와 유효한 Android 푸시 토큰을 모두 가지고 있고 두 가지 모두에 옵트인되어 있지만 연결된 iOS 푸시 토큰이 없을 수 있습니다. 도달 가능한 총 사용자 수와 다른 채널의 합계 사이의 격차는 세그먼트에 해당하지만 해당 커뮤니케이션 채널을 통해 도달할 수 없는 사용자 수입니다.

## 세그먼트 크기 통계

예상 통계는 세그먼트의 일부만 샘플링하여 근사치를 구한 것이므로 실제 값보다 크거나 작은 예상 크기가 표시될 수 있으며, 작업 영역이 클수록 오차 범위가 더 클 수 있습니다. 세그먼트의 정확한 사용자 수를 확인하려면 **정확한 통계 계산을** 선택합니다. 정확한 세그먼트 멤버십은 항상 세그먼트가 캠페인이나 캔버스에서 보낸 메시지의 영향을 받기 전에 계산됩니다.

Braze는 세그먼트 크기에 대한 다음 통계를 제공합니다. 

### 필터 통계

각 필터 그룹에 대해 예상 도달 사용자 수를 볼 수 있습니다. **추가 퍼널 통계를 확장**을 선택하여 채널별 분석을 확인하세요.

![성별이 알려지지 않은 필터가 있는 필터 그룹입니다.][2]{: style="max-width:80%;"}

### 세그먼트 통계

전체 세그먼트에 대해 페이지 하단에서 예상 도달 사용자 수와 각 채널의 예상 사용자 수를 확인할 수 있습니다. 또한 **정확한 통계 계산**을 선택하여 도달 가능한 사용자 수(세그먼트 전체 및 채널별 기준)를 정확하게 확인할 수 있습니다.

참고:
- 정확한 통계를 계산하는 데 몇 분이 걸릴 수 있습니다. 이 함수는 필터 또는 필터 그룹 수준이 아닌 세그먼트 수준에서만 정확한 통계를 계산합니다.
- 큰 세그먼트의 경우, 정확한 통계를 계산할 때에도 약간의 변동이 있는 것이 정상입니다. 이 기능의 정확도는 99.999% 이상일 것으로 예상됩니다.

## 과거 세그먼트 멤버십 규모 보기

모든 세그먼트에 대해 각 날짜의 예상 세그먼트 멤버십을 보여주는 과거 멤버십 차트를 볼 수 있습니다. 이 차트는 시간 경과에 따라 세그먼트의 크기가 어떻게 변화했는지 보여줍니다. 드롭다운을 사용하여 날짜 범위별로 멤버십을 필터링할 수 있습니다.

![과거 멤버십 드롭다운을 사용하여 날짜 범위별로 멤버십을 세분화할 수 있습니다.][1]

이 차트의 목표는 전체 세그먼트 멤버십 추세를 파악하는 것이므로 **정확한 통계 계산을** 선택하기 전에 세그먼트 크기가 추정치인 것과 마찬가지로 일일 카운트는 추정치입니다. 그리고 이 그래프는 추정치를 보여주기 때문에 실제 크기( **정확한 통계 계산을** 선택한 후 확인할 수 있음)가 "0"이 아닌데도 이 차트에서 세그먼트의 크기가 "0"으로 표시될 수 있습니다. 특히 작업 공간 인구 규모에 비해 세그먼트가 매우 작은 경우 차트에 "0"이라는 추정치가 표시될 가능성이 높습니다.

Braze는 사용자의 하위 집합을 쿼리하여 세그먼트 회원 수를 추정한 다음, 그 결과를 전체 잠재고객으로 추정합니다. 즉, 차트의 결과는 해당 날짜의 세그먼트 회원 수에 대한 추정치일 뿐이며, 매일 다른 사용자 샘플이 이 추정치에 대해 쿼리될 수 있으므로 매일 변동될 것으로 예상됩니다.

{% alert note %}
모든 추정치는 표시된 값보다 워크스페이스 전체 인구 규모의 약 1%만큼 더 높거나 낮을 수 있습니다. 사용자가 많은 대규모 워크스페이스일수록 그 차이가 워크스페이스 사용자 인구의 1%에 불과하더라도 정확한 계산과 더 큰 수치로 차이가 날 가능성이 높습니다. 즉, 대규모 작업 공간에서는 추정치와 정확한 수치의 차이가 더 클 것으로 예상됩니다.
{% endalert %}

### 중요한 변경 사항의 이유

회원 수는 이 표에 나와 있는 것과 같은 여러 가지 이유로 인해 크게 변경될 수 있습니다.

| 이유 | 예시 |
| --- | --- |
| 정상적인 사용자 행동 | 사용자는 특히 성공적인 캠페인 이후 구독을 신청합니다. |
| CSV로 사용자 가져오기 | 세그먼트 멤버십을 크게 늘린 사용자의 CSV 파일을 가져왔습니다. |
| 세그먼트 대상 기준이 수정되었습니다. | 기존 세그먼트의 오디언스 규칙(예: 필터)이 변경되어 세그먼트 멤버십에 중대한 변경이 발생했습니다. |
| 사용자가 삭제됩니다. | 상당수의 사용자가 삭제되었습니다. |
| Braze와 동기화된 파트너 통합 | 제3자가 세그먼트 멤버십에 큰 영향을 미치는 데이터를 Braze에 보냈습니다. |
| 휴면 사용자가 보관됩니다. | 상당수의 비활성 프로필이 보관되었습니다. 예를 들어, 많은 수의 CSV 가져오기 사용자는 활동을 기록하지 않고 동시에 아카이브됩니다. |
