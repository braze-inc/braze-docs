# 억제 목록

> 차단 목록은 메시지를 받지 않을 사용자 그룹을 지정합니다. 관리자는 세그먼트 필터를 사용하여 동적 금지 목록을 만들어 세분화할 때와 같은 방식으로 사용자 그룹을 좁힐 수 있습니다.

{% 경고 중요 %} 억제 목록은 현재 베타 버전입니다. 이 베타 버전에 참여하고 싶다면 고객 성공 관리자에게 문의하세요. 베타 기간 동안 기능이 변경될 수 있으며 한 번에 최대 5개의 활성 수신 거부 목록을 보유할 수 있지만 더 필요한 경우 고객 성공 관리자에게 알려주세요. {% endalert %}

## 작동 방식

금지 목록은 동적이며 특정 형태의 메시지에 자동으로 적용되지만 선택한 태그에 대해 예외를 설정할 수 있습니다. 선택한 예외 태그가 캠페인이나 캔버스에서 사용되는 경우 해당 금지 목록은 해당 캠페인이나 캔버스에 적용되지 않습니다. 예외 태그가 있는 캠페인 또는 캔버스의 메시지는 여전히 타겟 세그먼트에 속하는 모든 금지 목록 사용자에게 전달됩니다.

### 차단 목록의 영향을 받지 않는 메시지

베타 버전에서는 다음 메시지 유형에는 뮤트 리스트가 적용되지 않습니다(즉, 뮤트 리스트 사용자는 다음에 속하는 메시지를 **계속** 받게 됩니다): - 기능 플래그 - 트랜잭션 사용 사례 - API 캠페인 - API 트리거 캠페인 - API 트리거 캔버스 - Braze API로 트리거된 캠페인`(/메시지` 및 `/보내기`)

이러한 사용 사례에는 금지 목록이 자동으로 적용되지 않으므로 예외 태그를 추가할 필요가 없습니다. 이러한 사용 사례 내에서 메시지에서 사용자 그룹을 제외하려면 이러한 사용자를 제외하는 대상 세그먼트를 만들어야 합니다.

{% 경고 중요 %} 베타 버전에서는 제품 개선에 도움이 되는 고객 피드백을 수집합니다. 트랜잭션 사용 사례에 금지 목록을 적용할 계획이라면 고객 성공 관리자에게 알려 주세요. {% endalert %}

### 차단 목록의 영향을 받는 채널

금지 목록은 동적이며 다음 모든 채널에 자동으로 적용됩니다(캠페인 또는 캔버스에 예외 태그가 포함되지 않은 경우): - SMS - 이메일 - 푸시 - 인앱 메시지 - 콘텐츠 카드 - 배너 - SMS/MMS - 웹훅 - WhatsApp - LINE

## 금지 목록 설정하기

차단 목록은 보내는 메시지에 상당한 영향을 미칠 수 있으므로 관리자만 차단 목록을 편집, 저장, 활성화 및 비활성화할 수 있습니다(모든 사용자는 차단 목록을 볼 수 있음).

1. **대상** > **금지 목록으로** 이동합니다.<br><br>세 가지 금지 목록 목록이 있는 '금지 목록' 페이지]\[1]<br><br>
2. **금지 목록 만들기를** 선택하고 이름을 추가합니다.<br><br>이름을 입력하는 필드가 있는 '금지 목록 만들기' 창이 표시됩니다.]\[2]{:: style="max-width:80%;"}<br><br>
3. 세그먼트 필터를 사용하여 차단 목록에서 사용자를 식별하세요. 하나 이상을 선택해야 합니다.

{% 경고 중요 %} 설정 프로세스는 [세그먼트 만들기와]({{site.baseurl}}/user_guide/engagement_tools/segments/creating_a_segment/) 비슷해 보이지만 금지 목록은 세그먼트 멤버십에 관계없이 메시지를 보내지 **않으려는** 사용자 그룹입니다. {% endalert %}

90일 이상 전에 마지막으로 이메일을 열어본 사용자를 위한 필터가 있는 수신 거부 목록 작성기]\[3]

{: start="4"} 4. 세그먼트 이름 아래의 확인란을 선택하여 태그에 따라 예외를 적용할지 여부를 결정한 다음(자세한 내용은 [작동 방식](#how-it-works) 참조), 이 차단 목록에 있는 사용자가 계속 수신해야 하는 캠페인 또는 캔버스의 태그를 추가합니다. <br><br>즉, "배송 확인" 예외 태그를 추가하면 차단 목록에 있는 사용자는 "배송 확인" 태그를 사용하는 메시지를 제외한 모든 메시징에서 제외됩니다.<br><br>'배송 확인'이라는 예외 태그가 적용된 '배송 목록 세부 정보' 섹션.]\[4]<br><br>
5\. 차단 목록을 저장하거나 활성화합니다. - 저장하면 차단 목록은 저장되지만 활성화되지 않으므로 차단 목록이 적용되지 않습니다. 차단 목록은 활성화할 때까지 비활성 상태로 유지되며, 비활성 차단 목록은 메시지에 영향을 미치지 않습니다(사용자가 메시지에서 제외되지 않음). - 활성화하면 차단 목록이 저장되고 즉시 적용되므로 차단 목록에 있는 사용자는 캠페인이나 캔버스에서 즉시 제외됩니다(예외 태그가 포함된 경우 제외).

{경고 노트 %} 관리자만 차단 목록을 저장하거나 활성화할 수 있습니다. 베타 버전에서는 한 번에 최대 5개의 활성 차단 목록을 만들 수 있습니다. {% endalert %}

더 이상 필요하지 않은 경우 차단 목록을 비활성화하거나 보관할 수 있습니다. - 비활성화하려면 활성 차단 목록을 선택하고 **비활성화를** 선택합니다. 비활성화된 차단 목록은 나중에 다시 활성화할 수 있습니다. - 보관하려면 **차단 목록** 페이지에서 보관하세요.

## 억제 목록 사용

### 캠페인의 경우

'마케팅 상태 점수가 낮음'이라는 하나의 활성 억제 목록이 있는 '억제 목록' 섹션.]\[5]

사용자가 차단 목록에 있는 경우 해당 차단 목록이 적용되는 캠페인은 수신되지 않습니다. 쪽지 차단 목록이 적용되지 않는 경우는 쪽지 차단 목록의 [영향을 받지 않는 메시지를](#messages-not-affected-by-suppression-lists) 참조하세요.

#### 어떤 억제 목록이 적용되는지 확인

캠페인 내에서 금지 목록 사용을 확인하려면 **타겟 오디언스** 페이지의 금지 **목록** 섹션으로 이동하여 해당 캠페인에 적용 중인 금지 목록을 확인합니다.

### 캔버스의 경우

사용자가 금지 목록에 있는 경우 캔버스에 계속 들어갈 수 있지만 캔버스 내에서 메시지 단계를 받을 수 없습니다. 메시지 단계로 진행하면 캔버스에서 종료됩니다. 그러나 수신 차단 목록에 있는 사용자는 메시지 단계 이전에 메시지 단계가 아닌 단계를 계속 받을 수 있습니다. 

#### 세그먼트가 캔버스에 들어가지 못하도록 하기

세그먼트를 캔버스에 **전혀** 입력하지 않으려면 다음 단계에 따라 해당 세그먼트를 제외하도록 해당 캔버스의 대상 설정을 구성할 수 있습니다:

1. 억제 목록과 동일한 필터 및 기준을 사용하여 세그먼트를 작성합니다.
2. **타겟** 단계에서 세그먼트 **멤버십** 필터를 사용하여 세그먼트에 포함되지 않은 사용자를 타겟팅합니다.

예를 들어 억제 목록이 적용된 캔버스가 있다고 가정해 보겠습니다. 캔버스에는 사용자 업데이트 단계와 메시지 단계가 있습니다. 이 시나리오에서 억제 목록 사용자는 캔버스에 들어가서 사용자 업데이트 단계(해당 단계의 구성 방식에 따라 사용자가 업데이트될 수 있음)를 진행한 다음 메시지 단계(이 시점에서 사용자가 '종료됨' 지표에 포함됨)에서 종료됩니다. 

#### 어떤 억제 목록이 적용되는지 확인

캔버스 내에서 억제 목록 사용을 확인하려면 **타겟 대상** 페이지의 **억제 목록** 섹션으로 이동하여 해당 캔버스에 어떤 억제 목록이 적용되고 있는지 확인합니다. **요약** 단계에서 적용된 차단 목록을 볼 수도 있습니다.
