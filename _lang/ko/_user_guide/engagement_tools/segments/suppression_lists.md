# 억제 목록

> 수신 금지 목록은 캠페인이나 캔버스를 자동으로 받지 않는 사용자 그룹입니다. 금지 목록은 세그먼트 필터에 의해 정의되며, 사용자는 필터 기준을 충족할 때 금지 목록에 들어가고 나갈 수 있습니다. 또한 예외 태그를 설정하여 해당 태그가 있는 캠페인이나 캔버스에는 금지 목록이 적용되지 않도록 할 수도 있습니다. 예외 태그가 있는 캠페인 또는 캔버스의 메시지는 여전히 대상 세그먼트에 있는 뮤트 리스트 사용자에게 도달합니다.

## 금지 목록을 사용하는 이유는 무엇인가요?

금지 목록은 동적이며 모든 형태의 메시징에 자동으로 적용되지만 선택한 태그에 대해서는 예외를 설정할 수 있습니다. 선택한 예외 태그가 캠페인 또는 캔버스에서 사용되는 경우 해당 억제 목록은 해당 캠페인 또는 캔버스에 적용되지 않습니다. 예외 태그가 있는 캠페인 또는 캔버스의 메시지는 타겟 세그먼트에 속하는 모든 차단 목록 사용자에게 계속 도달합니다.

### 차단 목록의 영향을 받는 메시지 유형 및 채널

억제 목록은 [기능 플래그를]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/feature_flags/) 제외한 모든 메시지 유형 및 채널에 적용됩니다. 즉, 기본값으로 억제 목록이 모든 채널, 캠페인 및 캔버스에 적용됩니다:
- [API 캠페인]({{site.baseurl}}/api/api_campaigns/)
- API 트리거 캠페인 및 캔버스
- [트랜잭션 이메일]({{site.baseurl}}/user_guide/message_building_by_channel/email/transactional_message_api_campaign/)

억제 목록이 적용되지 않는 유일한 메시지 유형은 기능 플래그입니다. 비공개 목록에 있는 사용자는 기능 플래그에서 비공개되지 않지만 다른 모든 채널에서는 비공개됩니다. 

예외 태그를 사용하여 금지 목록 사용자가 특정 캠페인 및 캔버스에서 계속 타겟팅되도록 할 수 있습니다. 자세한 내용은 [차단 목록 설정하기의](#setup) 4단계를 참조하세요. 예외 태그를 금지 목록에 추가하지 않으면 해당 금지 목록에 있는 사용자는 기능 플래그 외에 어떤 메시징도 타겟팅되지 않습니다. 

{% alert note %}
억제 목록은 Braze 대시보드에서 생성된 API 캠페인에 `campaign_id`. 억제 목록은 관련 `campaign_id` 이 없는 [Braze 메시징 엔드포인트를]({{site.baseurl}}/api/endpoints/messaging/) 통해 전송된 메시지에는 적용되지 않습니다.
{% endalert %}

'예외 설정' 섹션에서 API 트리거 캠페인 및 캔버스에 금지 목록을 적용하지 않으려면 체크박스를 선택합니다.]({% image_buster /assets/img/suppression_list_checkbox.png %}){: style="max-width:70%;"}

## 금지 목록 설정하기 {#setup}

{% alert note %}
모든 사용자가 차단 목록을 볼 수 있지만 [관리자 권한이]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/?tab=admin#list-of-permissions) 있는 사용자만 차단 목록을 만들고 관리할 수 있습니다.
{% endalert %}

1. **오디언스** > **억제 목록으로** 이동합니다.<br><br>세 가지 차단 목록이 있는 '차단 목록' 페이지로 이동합니다.]({% image_buster /assets/img/suppression_lists_home.png %})<br><br>
2. **금지 목록 만들기를** 선택하고 이름을 추가합니다.<br><br>이름을 입력할 수 있는 필드가 있는 '차단 목록 만들기'라는 창이 나타납니다.]({% image_buster /assets/img/create_suppression_list.png %}){: style="max-width:80%;"}<br><br>
3. 세그먼트 필터를 사용하여 차단 목록에서 사용자를 식별할 수 있습니다. 하나 이상을 선택해야 합니다.

{% alert important %}
설정 과정은 [세그먼트 생성과]({{site.baseurl}}/user_guide/engagement_tools/segments/creating_a_segment/) 비슷해 보이지만, 금지 목록은 세그먼트 멤버십에 관계없이 메시지를 보내지 **않으려는** 사용자 그룹입니다.
{% endalert %}

90일 이상 전에 마지막으로 이메일을 열어본 사용자를 위한 필터가 있는 수신 거부 목록 작성기입니다.]({% image_buster /assets/img/suppression_list_filters.png %})

{: start="4"}
4\. 세그먼트 이름 아래의 확인란을 선택하여 태그를 기준으로 예외를 적용할지 여부를 결정한 다음(자세한 내용은 [금지 목록을 사용하는 이유](#why-use-suppression-lists) 참조), 이 금지 목록에 있는 사용자가 계속 수신해야 하는 캠페인 또는 캔버스의 태그를 추가합니다. <br><br>즉, "배송 확인" 예외 태그를 추가하면 차단 목록에 있는 사용자는 "배송 확인" 태그를 사용하는 메시지를 제외한 모든 메시징에서 제외됩니다.<br><br>'배송 확인'이라는 예외 태그가 적용된 '배송 목록 세부 정보' 섹션입니다.]({% image_buster /assets/img/exception_tags.png %})<br><br>
5\. 차단 목록을 저장하거나 활성화합니다.
- 저장하면 차단 목록은 저장되지만 활성화되지 않으므로 차단 목록이 적용되지 않습니다. 차단 목록은 활성화할 때까지 비활성 상태로 유지되며, 비활성 차단 목록은 메시징에 영향을 미치지 않습니다(사용자가 메시지에서 제외되지 않음).
- 활성화하면 차단 목록이 저장되고 즉시 적용되므로 차단 목록에 있는 사용자는 캠페인이나 캔버스에서 즉시 제외됩니다(예외 태그가 있는 경우 제외).

{% alert note %}
관리자만 차단 목록을 저장하거나 활성화할 수 있습니다. 베타 버전에서는 한 번에 최대 5개의 활성 차단 목록을 만들 수 있습니다.
{% endalert %}

차단 목록이 더 이상 필요하지 않은 경우 비활성화하거나 보관할 수 있습니다. 
- 비활성화하려면 활성 억제 목록을 선택하고 **비활성화를** 선택합니다. 비활성화된 차단 목록은 나중에 다시 활성화할 수 있습니다.
- 보관하려면 **차단 목록** 페이지에서 보관하세요.

## 억제 목록 사용

수신 금지 목록으로 인해 사용자가 메시지를 받지 못했는지 확인하려면 캠페인 또는 캔버스 내 **타겟 오디언스** 단계에서 **사용자 조회를** 사용하세요. 여기에서 해당 항목이 어떤 억제 목록에 속하는지 확인할 수 있습니다.

사용자가 차단 목록에 있음을 보여주는 '사용자 조회' 창이 표시됩니다.]({% image_buster /assets/img/suppression_list_user_lookup.png %}){: style="max-width:70%;"}

{% alert tip %}
**요약** 단계에서도 적용된 차단 목록을 찾을 수 있습니다.
{% endalert %}

캠페인 또는 캔버스를 만드는 동안 **타겟 오디언스** 단계에서 사용자 **조회를** 사용하여 사용자를 검색하면 해당 사용자가 타겟 오디언스에 포함되지 않은 경우 해당 사용자가 속한 금지 목록을 볼 수 있습니다. 

사용자가 차단 목록에 있음을 보여주는 '사용자 조회' 창이 표시됩니다.]({% image_buster /assets/img/suppression_list_user_lookup.png %}){: style="max-width:70%;"}

### 캠페인 

사용자가 차단 목록에 있는 경우 해당 차단 목록이 적용되는 캠페인은 수신되지 않습니다. 차단 목록이 적용되지 않는 경우는 차단 목록의 [영향을 받는 메시지 유형 및 채널을](#message-types-and-channels-affected-by-suppression-lists) 참조하세요.

"마케팅 상태 점수가 낮음"이라는 활성 억제 목록이 하나 있는 "억제 목록" 섹션입니다.]({% image_buster /assets/img/active_suppression_list.png %})

### 캔버스 

사용자가 차단 목록에 추가되는 순간부터 해당 사용자는 캔버스에 들어갈 수 없습니다. 이미 캔버스에 들어간 경우에는 메시지 단계를 받지 못합니다. 즉, 사용자가 금지 목록에 추가될 때 이미 캔버스 안에 있는 경우 다음 메시지 단계까지 캔버스를 진행하게 되며, 이 시점에서는 메시지 단계를 받지 않고 종료됩니다. 

예를 들어 캔버스에 사용자 업데이트 단계와 메시지 단계가 있다고 가정해 보겠습니다. 사용자가 캔버스에 들어온 후 억제 목록에 추가된 경우에도 해당 사용자는 사용자 업데이트 단계(업데이트될 수 있는 곳)를 거쳐 메시지 단계에서 종료되며, 이 시점에서 해당 사용자는 종료된 측정기준에 포함됩니다.
