# 세그먼트 확장

> 세그먼트 확장을 사용하면 사용자 기록의 긴 기간에 걸쳐 매우 정밀한 세그먼트를 구축할 수 있습니다. 예를 들어 세그먼트 확장을 사용하면 지난 16개월 동안 특정 제품을 구매했거나 서비스에 일정 금액을 지출한 사용자를 타겟팅할 수 있습니다. 이벤트 속성정보를 사용하여 오디언스를 세분화하여 타겟팅을 더욱 세분화할 수 있습니다.

Braze 세그먼트를 사용하면 커스텀 이벤트 또는 구매 행동을 기반으로 사용자를 타겟팅할 수 있습니다. 세그먼트 확장은 이 용량을 향상시켜 사용자 프로필에 저장된 과거 데이터를 활용할 수 있도록 합니다. 세그먼트 확장을 사용하면 지난 2년(730일) 동안 커스텀 이벤트 또는 구매 이벤트를 몇 번이나 완료한 사용자를 식별하고 도달할 수 있습니다. 

## 세그먼트 확장을 사용하는 이유는 무엇인가요?

Braze 세그먼트는 동적인 사용자 그룹을 생성할 수 있는 강력한 타겟팅 툴을 제공합니다. 대부분의 사용 사례에서 이 정도면 오디언스에게 효과적으로 도달할 수 있습니다. 세그먼트 확장은 데이터 보존이나 시스템 성능/성과에 영향을 주지 않으면서 최대 2년 전의 행동 데이터를 분석하거나 복잡한 로직을 적용해야 하는 고급 사용 사례를 위해 설계되었습니다. [SQL]({{site.baseurl}}/user_guide/engagement_tools/segments/sql_segments) 쿼리(SQL 세그먼트 확장) 또는 자체 [데이터 웨어하우스의]({{site.baseurl}}/user_guide/engagement_tools/segments/segment_extension/cdi_segments) 데이터를 사용하여 오디언스를 더욱 세분화할 수 있습니다.

예를 들어, Braze 기본값 세분화는 최근에 제품 중 하나를 구매한 사용자를 식별하는 등 정의한 특정 기준에 맞는 사용자를 찾습니다. 세그먼트 확장을 사용하면 18개월에서 24개월 사이에 특정 제품의 특정 색상을 두 번 이상 구매한 사용자를 식별하는 등 더 심층적으로 분석할 수 있습니다. 세그먼트 확장은 필수 사항이 아닌 개선 사항입니다. 고급 필터나 더 긴 룩백 기간이 필요한 경우, 데이터 사용량을 최적화하면서 도움을 받을 수 있는 훌륭한 도구입니다.

{% alert note %}
특정 시점에 워크스페이스당 25개의 활성 세그먼트 확장이 기본값으로 할당되어 있습니다. 이 한도를 늘려야 하는 경우 Braze 고객 성공 매니저에게 문의하여 사용 사례에 대해 논의하세요.
{% endalert %}

## 세그먼트 확장 만들기

세그먼트 확장을 만들려면 커스텀 이벤트 속성정보를 기반으로 사용자 세그먼트를 구체화하는 필터를 만듭니다. 세그먼트 확장을 만들 때 세그먼트를 정적으로 만들지 아니면 설정된 간격으로 동적으로 새로고침할지 선택합니다.

### 1단계: 세그먼트 확장으로 이동하기

**오디언스** > 세그먼트 **확장으로** 이동합니다.

세그먼트 확장 표에서 **새 확장 프로그램 만들기를** 선택한 다음 세그먼트 확장 프로그램 만들기 환경을 선택합니다:

- **간단한 확장:** 안내 양식을 사용하여 단일 이벤트에 초점을 맞춘 세그먼트 확장을 만듭니다.
SQL을 사용하고 싶지 않을 때 가장 적합합니다.
- **템플릿으로 시작하세요:** Snowflake 데이터를 사용하여 사용자 지정 가능한 템플릿으로 SQL 세그먼트를 만듭니다.
- **증분 새로고침:** 지난 2일간의 데이터를 자동으로 새로고침하거나 필요에 따라 수동으로 새로고침하는 Snowflake SQL 세그먼트를 작성합니다. 정확성과 비용 효율성의 균형을 맞추는 데 가장 적합합니다.
- **새로고침:** 수동으로 새로고침할 때 전체 오디언스를 다시 계산하는 Snowflake 데이터 또는 [CDI 연결 소스가]({{site.baseurl}}/cdi_segment_extensions/) 있는 SQL 세그먼트를 작성합니다. 오디언스에 대한 완전한 최신 정보가 필요할 때 가장 적합합니다.

다양한 세그먼트 확장 생성 환경을 선택할 수 있는 표입니다.]({% image_buster /assets/img/segment/segment_extension_modal.png %}){: style="max-width:50%"}

SQL을 사용하는 경험을 선택하는 경우 자세한 내용은 [SQL 세그먼트 확장을]({{site.baseurl}}/user_guide/engagement_tools/segments/sql_segments/) 참조하십시오. **단순 확장을** 선택한 경우 2단계를 계속 진행합니다.

#### SQL 크레딧 사용량

다음 세그먼트 확장 유형은 SQL 크레딧을 소비합니다:

- SQL 세그먼트 확장(증분 및 전체 새로고침 모두)
- 카탈로그 세그먼트 세분화
- CDI 세그먼트 
    - 크레딧은 자체 데이터 웨어하우스 내에서 소비됩니다.

### 2단계: 세그먼트 확장 이름 지정하기

필터링하려는 사용자 유형을 설명하여 세그먼트 확장에 이름을 지정합니다. 이렇게 하면 세그먼트에 필터로 적용할 때 이 확장을 쉽고 정확하게 검색할 수 있습니다.

세그먼트 확장 이름은 "온라인 쇼핑객 확장 - 90일"입니다.]({% image_buster /assets/img/segment/segment_extension2.png %})

### 3단계: 기준 선택

타겟팅할 구매, 메시지 참여 또는 커스텀 이벤트 기준 중에서 선택합니다. 원하는 이벤트 유형 기준을 선택한 후 사용자 목록에 타겟팅할 구매 아이템, 메시지 상호 작용 또는 특정 커스텀 이벤트를 선택합니다. 그런 다음 사용자가 이벤트를 완료해야 하는 횟수(이상, 미만 또는 같음)와 기간(특히 세그먼트 확장의 경우 과거 730일(2년)까지 거슬러 올라갈 수 있음)을 선택합니다.

730일 이상의 이벤트 데이터를 기반으로 한 세분화는 **세그먼트에** 있는 다른 필터를 사용하여 수행할 수 있습니다. 기간을 선택할 때 상대적인 날짜 범위를 지정하여 지난 X일, 시작일, 종료일 또는 정확한 날짜 범위(날짜 A~날짜 B)를 선택할 수 있습니다.

2025년 3월 1일부터 2025년 3월 31일까지의 날짜 범위에서 커스텀 이벤트를 2회 이상 수행한 사용자에 대한 세분화 기준입니다.]({% image_buster /assets/img/segment/segment_extension1.png %})

#### 이벤트 속성정보 세분화

타겟팅 정밀도를 높이려면 **속성 필터 추가** 확인란을 선택합니다. 이렇게 하면 구매 또는 커스텀 이벤트의 특정 속성정보를 기준으로 드릴다운할 수 있습니다. 문자열, 숫자, 부울, 시간 객체를 기반으로 이벤트 속성정보 세분화를 지원합니다.

문자열 속성의 경우 한 번에 여러 값을 입력할 수 있습니다. 아래 예에서 이 필터는 골드, 실버 또는 브론즈 중 하나에 해당하는 상태를 가진 사용자를 찾습니다.

!!! 부울 속성을 기반으로 세그먼트화하기.]({% image_buster /assets/img/segment/property3.png %})

날짜/시간 개체를 기반으로 세그먼트화하기.]({% image_buster /assets/img/segment/property4.png %})

[중첩된 이벤트 속성정보를]({{site.baseurl}}/user_guide/data/custom_data/custom_events/nested_objects/) 기반으로 한 세그먼트 세분화도 지원합니다.

\![중첩된 이벤트 속성정보를 기반으로 세분화.]({% image_buster /assets/img/segment/nested_segment_extensions.png %})

세그먼트 확장은 이벤트 속성정보의 장기 저장에 의존하며 타임스탬프가 찍힌 속성 저장 제한이 없습니다. 지난 2년 동안 추적한 이벤트 속성정보를 다시 볼 수 있습니다. 세그먼트 확장 내에서 이벤트 속성정보를 사용해도 데이터 포인트 사용량에는 영향을 미치지 않습니다.

{% alert note %}
세그먼트에서 이벤트 속성정보나 중첩된 고객 속성을 사용하기 위해 세그먼트 확장이 필요하지 않습니다. 세그먼트 확장은 기본값 세그먼트를 만드는 데 사용되는 기록 창을 확장하는 것입니다. 지난 30일 동안의 이벤트 속성정보를 사용하거나 중첩된 고객 속성을 사용하는 실시간 기본 [세그먼트를]({{site.baseurl}}/user_guide/engagement_tools/segments/) 만들 수 있습니다. 마찬가지로, 세그먼트 확장 없이도 이벤트 속성정보를 기반으로 [메시지가]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/triggered_delivery/) 실시간으로 트리거되도록 [예약할]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/triggered_delivery/) 수 있습니다.
{% endalert %}

### 5단계: 세그먼트 확장 저장하기

**저장을** 선택하면 세그먼트 확장이 처리가 시작됩니다. 세그먼트 확장을 생성하는 데 걸리는 시간은 사용자 수, 캡처하는 커스텀 이벤트 또는 구매 이벤트의 수, 기록에서 되돌아보는 일 수에 따라 달라집니다.

세그먼트 확장이 처리되는 동안에는 세그먼트 확장 이름 옆에 작은 애니메이션이 표시되고 세그먼트 확장 목록의 **마지막 처리** 열에 "처리 중"이라는 단어가 표시됩니다. 세그먼트 확장이 처리되는 동안에는 편집할 수 없다는 점에 유의하세요.

두 개의 활성 확장이 있는 "세그먼트 확장" 페이지로 이동합니다.]({% image_buster /assets/img/segment/segment_extension5.png %})

세그먼트 확장이 처리되는 동안 Braze는 오디언스 세분화를 위해 처리 시작 전의 기본값 세그먼트의 버전 기록을 계속 사용합니다. 처리는 저장 또는 새로고침이 발생할 때마다 이루어지며, 고객 프로필을 쿼리하고 업데이트하는 작업이 포함되므로 기본값인 세그먼트의 멤버십이 즉시 업데이트되지는 않습니다. 즉, 새로고침 처리가 시작되기 전에 사용자의 작업이 수행되지 않으면 특정 새로고침이 완료된 후 해당 사용자가 세그먼트 확장에 포함될지 보장할 수 없습니다. 반대로 새로고침 전에 세그먼트 확장에 속했지만 더 이상 기준을 충족하지 않는 사용자는 새로고침 프로세스가 완료되고 업데이트가 적용될 때까지 청각 장애 세그먼트와 계속 일치하게 됩니다.

### 6단계: 세그먼트에서 확장 기능 사용하기

세그먼트 확장을 만든 후에는 세그먼트를 만들거나 캠페인 또는 캔버스에 대한 오디언스를 정의할 때 필터로 사용할 수 있습니다. **사용자 속성** 섹션의 필터 목록에서 **Braze 세그먼트 확장을** 선택하여 시작하세요.

"필터" 섹션에 필터 드롭다운이 표시되고 "Braze 세그먼트 확장"이 표시됩니다.]({% image_buster /assets/img/segment/segment_extension7.png %})

Braze 세그먼트 확장 필터 목록에서 이 세그먼트에 포함하거나 제외할 세그먼트 확장을 선택합니다.

'지난 56일 동안 이메일 클릭 1회'라는 세그먼트가 포함된 'Braze 세그먼트 확장' 필터입니다.]({% image_buster /assets/img/segment/segment_extension6.png %})

세그먼트 확장 기준을 보려면 **확장 세부정보 보기를** 선택하여 새 창에 세부 정보를 표시합니다.

!"[지난 56일 동안 이메일 클릭 횟수 1회"에 대한 확장.]({% image_buster /assets/img/segment/segment_extension8.png %}){: style="max-width:70%;"}

이제 평소처럼 [세그먼트 생성을]({{site.baseurl}}/user_guide/engagement_tools/segments/creating_a_segment/) 계속 진행하면 됩니다.

## 자주 묻는 질문

### 여러 커스텀 이벤트를 사용하는 세그먼트 확장을 만들 수 있나요?

예. [SQL 세그먼트 확장을]({{site.baseurl}}/user_guide/engagement_tools/segments/sql_segments/) 사용할 때 여러 이벤트를 추가하거나 여러 Snowflake 테이블을 참조할 수 있습니다. 

**단순 확장** 세그먼트 확장을 사용하는 경우 하나의 커스텀 이벤트, 하나의 구매 이벤트 또는 하나의 채널 상호 작용을 선택할 수 있습니다. 그러나 기본값 세그먼트를 만들 때 여러 세그먼트 확장을 AND 또는 OR로 결합할 수 있습니다.

### 세그먼트 확장이 활성 캠페인에 존재하는 경우 아카이브할 수 있나요?

아니요. 세그먼트 확장을 보관하려면 먼저 모든 활성 메시징에서 해당 확장을 제거해야 합니다.

### 세그먼트 확장에 배열을 사용할 수 있나요?

예. 배열을 사용하려면 속성 이름에 대괄호(`[]`)를 추가합니다. 숙소가 `location_code` 인 경우 `location_code[]` 을 입력합니다. 

Braze는 `[]` 를 사용하여 배열을 트래버스하고 트래버스된 배열의 항목이 이벤트 속성정보와 일치하는지 확인합니다. 예를 들어 배열 속성의 값과 하나 이상 일치하는 사용자의 세그먼트 확장을 만들 수 있습니다.

### Braze는 "지난 __ 일"이라는 상대적인 기간의 기간을 어떻게 계산하나요?

세그먼트 확장이 상대적인 기간("지난 X일")을 계산할 때 시작 시간은 UTC 자정으로 설정됩니다. 예를 들어 2024-09-16 21:00 UTC에 새로고침하고 10일을 지정하는 세그먼트 확장의 경우 시작 시간은 2024-09-06 21:00 UTC가 아니라 2024-09-06 00:00 UTC로 설정됩니다. 

그러나 SQL 세그먼트를 사용하여 시간대를 지정하여 회사 시간 자정을 기준으로 10일 전에 커스텀 이벤트를 수행한 사용자를 식별하거나 현재 시간을 기준으로 10일 전에 이벤트를 수행한 사용자를 식별할 수 있습니다.
