# 사용자 프로필

> 사용자 프로필은 특정 사용자에 대한 정보를 찾는 훌륭한 방법입니다. 사용자와 관련된 모든 지속적인 데이터는 해당 사용자 프로필에 저장됩니다.

## 프로필 액세스

사용자의 프로필에 액세스하려면 **사용자 검색** 페이지로 이동하여 다음 중 하나로 사용자를 검색하세요:

- 외부 사용자 ID
- Braze ID
- 이메일
- 전화번호
- 푸시 토큰
- 형식 "[user_alias]:[alias_name]",의 사용자 별칭, 예: "amplitude_id:user_123"

일치하는 항목이 발견되면 Braze SDK를 사용하여 이 사용자에 대해 기록한 정보를 볼 수 있습니다. 그렇지 않으면 검색 결과가 여러 사용자 프로필을 반환하는 경우 각 프로필을 개별적으로 병합하거나 대량 사용자 병합을 수행할 수 있습니다. 전체 절차는 [중복 사용자]({{site.baseurl}}/user_guide/engagement_tools/segments/user_profiles/duplicate_users/)를 참조하세요.

\!["여러 사용자가 검색 기준에 일치합니다"라는 배너와 이전 및 다음이라는 레이블이 붙은 두 개의 버튼이 있는 검색 결과.]({% image_buster /assets/img_archive/User_Search_Nonunique.png %}){: style="max-width:60%;"}

## 사용 사례

사용자 프로필은 사용자의 참여 기록, 세그먼트 구성원, 장치 및 운영 체제에 대한 정보를 쉽게 액세스할 수 있기 때문에 문제 해결 및 테스트에 유용한 자원입니다.

예를 들어, 사용자가 문제를 보고하고 어떤 장치와 운영 체제를 사용하고 있는지 확실하지 않은 경우 [개요 탭](#overview-tab)을 사용하여 이 정보를 찾을 수 있습니다(이메일이나 사용자 ID가 있는 경우). 사용자의 언어를 볼 수 있으며, 이는 예상대로 작동하지 않는 [다국어 캠페인]({{site.baseurl}}/user_guide/engagement_tools/campaigns/ideas_and_strategies/campaigns_in_multiple_languages/#campaigns-in-multiple-languages) 문제를 해결하는 데 도움이 될 수 있습니다.

특정 사용자가 캠페인을 받았는지 확인하려면 [참여 탭](#engagement-tab)을 사용할 수 있습니다. 또한, 이 특정 사용자가 캠페인을 받았다면, 그들이 언제 받았는지 확인할 수 있습니다. 사용자가 특정 세그먼트에 있는지, 푸시, 이메일 또는 둘 다에 가입했는지 확인할 수 있습니다. 이 정보는 문제 해결 목적으로 유용합니다. 예를 들어, 사용자가 예상했던 캠페인을 받지 않거나 예상하지 못한 캠페인을 받는 경우 이 정보를 확인해야 합니다.

## 사용자 프로필의 요소

사용자 프로필에는 네 가지 주요 섹션이 있습니다.

- **개요:** 사용자에 대한 기본 정보, 세션 데이터, 사용자 정의 속성, 사용자 정의 이벤트, 구매 및 사용자가 마지막으로 로그인한 장치에 대한 정보입니다.
- **참여:** 사용자의 연락처 설정, 수신한 캠페인, 세그먼트, 커뮤니케이션 통계, 설치 귀속 및 무작위 버킷 번호에 대한 정보입니다.
- **메시징 기록:** 지난 30일 동안 이 사용자에 대한 최근 메시징 관련 이벤트입니다.
- **기능 플래그 적격성:** 사용자가 현재 롤아웃, 캔버스 단계 및 실험에서 적격한 기능 플래그를 확인합니다. 

### 개요 탭 {#overview-tab}

**개요** 탭에는 사용자 및 그들의 앱 또는 웹사이트와의 상호작용에 대한 기본 정보가 포함되어 있습니다.

| 개요 카테고리 | 포함 |
| --- | --- |
| 프로필 | 성별, 연령대, 위치, 언어, 로케일, 시간대 및 생일. |
| 세션 개요 | 그들이 가진 세션 수, 첫 번째 및 마지막 세션이 언제였는지, 어떤 앱에서였는지. |
| 사용자 정의 속성 | 이 사용자에게 할당된 사용자 정의 속성과 그에 따른 값, 중첩된 사용자 정의 속성 포함. |
| 최근 장치 | 그들이 로그인한 장치 수, 각 장치에 대한 세부 정보 및 관련 광고 ID(있는 경우). |
| 사용자 정의 이벤트 | 이 사용자가 수행한 사용자 정의 이벤트, 수행 횟수 및 각 이벤트를 마지막으로 수행한 시간. |
| 구매 | 이 사용자에게 귀속된 평생 수익, 마지막 구매, 총 구매 수 및 각 구매 목록. |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

이 데이터에 대한 자세한 내용은 [사용자 데이터 수집]({{site.baseurl}}/user_guide/data/user_data_collection/)을 참조하십시오.

### 참여 탭 {#engagement-tab}

**참여** 탭은 Braze를 사용하여 그들에게 보낸 메시지와의 사용자 상호작용에 대한 정보를 포함합니다.

| 참여 카테고리 | 포함 |
| --- | --- |
| 연락처 설정 | 이 사용자가 이러한 세 가지 채널에 대해 연결된 구독 그룹과 함께 이메일, SMS 및 푸시에 대한 구독 상태. 이 섹션에는 푸시 토큰에 대한 변경 로그 정보도 포함됩니다. 구독 및 옵트인 설정 방법에 대한 정보는 [이메일]({{site.baseurl}}/user_guide/message_building_by_channel/email/managing_user_subscriptions/), [SMS]({{site.baseurl}}/sms_rcs_subscription_groups/), 및 [푸시]({{site.baseurl}}/user_guide/message_building_by_channel/push/users_and_subscriptions/)를 참조하십시오. |
| 수신된 캠페인 | 사용자가 캠페인을 수신할 때 또는 사용자의 상호작용 데이터가 처음 감지될 때 수신된 캠페인이 표시됩니다. 목록에서 캠페인을 선택하여 보기. |
| 세그먼트 | 이 사용자가 포함된 세그먼트입니다. 목록에서 세그먼트를 선택하여 보기. |
| 커뮤니케이션 통계 | 이 사용자가 각 채널에서 마지막으로 메시지를 수신한 시점입니다. |
| 설치 귀속 | 사용자가 귀하의 앱을 어떻게 그리고 언제 설치했는지에 대한 정보입니다. [사용자 설치 이해하기]({{site.baseurl}}/user_guide/engagement_tools/campaigns/ideas_and_strategies/install_attribution/)에 대해 자세히 알아보세요. |
| 기타 | 사용자의 [무작위 버킷 번호]({{site.baseurl}}/user_guide/engagement_tools/testing/random_bucket_numbers/)입니다. |
| 수신된 캔버스 메시지 | 이 사용자가 수신한 캔버스 메시지와 그 시점입니다. 목록에서 메시지를 선택하여 보기. |
| 예측 | [이탈 예측]({{site.baseurl}}/user_guide/brazeai/predictive_churn/) 및 [이벤트 예측]({{site.baseurl}}/user_guide/brazeai/predictive_events/) 점수입니다. |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

### 메시징 기록 탭

사용자 프로필의 **메시지 기록** 탭은 지난 30일 동안 개별 사용자의 최근 메시징 관련 이벤트(약 40개)를 보여줍니다. 이 이벤트에는 사용자가 보낸 메시지, 받은 메시지, 상호작용한 메시지 등이 포함됩니다. 이 탭의 데이터는 사용자가 병합된 후에는 업데이트되지 않습니다.

{% alert note %}
이 표에 대한 피드백이 있거나 특정 이벤트를 보고 싶다면, 제목을 "메시징 기록 탭 피드백"으로 하여 [user-targeting@braze.com](mailto:user-targeting@braze.com?subject=Messaging%20History%20Tab%20Feedback)로 이메일을 보내주세요.
{% endalert %}

\![사용자가 받은 캠페인 및 캔버스를 보여주는 메시징 기록 탭.]({% image_buster /assets/img_archive/profiles_messaging_history_tab.png %})

#### 이벤트 보기 및 이해하기

**메시징 기록** 테이블의 각 이벤트에 대해 메시징 채널, 이벤트 유형, 이벤트 발생 시간, 관련 캠페인 또는 캔버스 메시지, 사용자의 장치 데이터를 볼 수 있습니다. 특정 이벤트를 필터링하려면 **필터**를 클릭하고 목록에서 이벤트를 선택하세요.

##### 메시지 중단 이벤트

메시지 중단 이벤트는 사용자가 수신한 메시지가 [Liquid]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/aborting_messages/) 또는 [Connected Content]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/connected_content/aborting_connected_content#aborting-messages)의 조건 논리로 인해 중단되었거나 Liquid 렌더링 시간 초과로 인해 중단되었을 때 발생합니다.

중단 이벤트는 다음 채널에서 사용할 수 있습니다:

- 이메일
- SMS
- 푸시
- 웹후크

중단 이벤트는 현재 인앱 메시지 및 콘텐츠 카드에서 사용할 수 없습니다.

##### 주파수 제한 이벤트

주파수 제한 이벤트는 사용자가 메시지를 받을 자격이 있지만 [주파수 제한]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/#frequency-capping) 설정으로 인해 실제로 수신하지 못할 때 발생합니다. **설정** > **주파수 제한 규칙**에서 주파수 제한 설정을 사용자 정의할 수 있습니다.

##### 빈 대상

일부 메시지 전송은 빈 대상("—"로 표시됨)으로 메시징 기록에 나타날 수 있습니다. 이는 콘텐츠 카드 및 웹훅과 같은 일부 채널이 메시지 전송 시 장치 데이터를 수집하지 않기 때문입니다.

콘텐츠 카드 전송은 카드가 볼 수 있을 때 기록됩니다. 콘텐츠 카드는 여러 장치에서 볼 수 있으므로 전송에 대한 장치 데이터는 기록되지 않습니다. 대신 이 정보는 인상 시(카드가 실제로 볼 때) 기록됩니다. 웹훅은 시스템 엔드포인트(장치가 아님)로 전송되므로 장치 데이터는 적용되지 않습니다.

#### 이메일 열기 이벤트에 대한 참고 {#note-on-email-open-event}

이메일 열기 추적은 Braze를 포함한 모든 도구에서 오류가 발생하기 쉽습니다. 다양한 이메일 클라이언트에서 제공하는 다양한 개인 정보 보호 기능이 이미지의 자동 로딩을 차단하거나 서버에서 사전 로드하는 경우, 이메일 열기 이벤트는 잘못된 긍정 및 잘못된 부정에 취약합니다.

이메일 열기 통계는 집계에서 유용할 수 있지만, 예를 들어 서로 다른 제목의 효과를 비교하기 위해, 개별 사용자의 개별 열기 이벤트가 의미가 있다고 가정해서는 안 됩니다.

#### 메시지 기록 탭에서 특정 필드가 비어 있는 이유는 무엇인가요?

다음 시나리오에서 사용자의 **메시지 기록** 탭에 필드가 없을 수 있습니다:

- 이벤트에 **메시지 전송**에 대한 데이터가 누락된 경우, 이는 캠페인에 메시지 변형이 없음을 나타냅니다.
- 이벤트에 **캠페인/캔버스** 및 **메시지 전송**에 대한 데이터가 누락된 경우, 이는 이 메시지가 `campaign_id` 및 `message_variation_id`를 지정하지 않은 API 캠페인(API 트리거 캠페인이 아님)에서 전송되었음을 나타냅니다. 이 필드는 선택 사항이며 요청 본문에서 생략할 수 있습니다. 이 필드가 지정되면 해당 정보가 메시지 기록 로그에 채워집니다.
   - 특정 메시지가 메시징 기록에서 완전히 누락되었지만 **수신한 캠페인** 로그에 나타나는 경우, 사용자가 현재 사용자로 식별되기 전에 캠페인을 수신했을 가능성이 높습니다. 기존 프로필이 고아 상태인 경우, **수신한 캠페인** 로그가 전송되지만 메시징 기록은 전송되지 않습니다. 
- **캠페인/캔버스**에 대한 데이터가 누락된 경우, 수동 테스트가 전송되었을 수 있습니다. 수동 테스트는 **메시징 기록** 탭에 기록되지만, 전송된 캠페인이나 캔버스는 기록되지 않습니다.
