# [![Braze 학습 과정]({% image_buster /assets/img/bl_icon3.png %})](https://learning.braze.com/segmentation-course){: style="float:right;width:120px;border:0;" class="noimgborder"} 세그먼트 만들기

> 세그먼트를 세분화하면 인구통계학적, 행동적, 기술적 특성 및 행동에 따라 사용자를 타겟팅할 수 있습니다. 세분화와 메시징 자동화를 창의적이고 지능적으로 활용하면 사용자를 첫 접촉에서 장기 고객으로 원활하게 전환할 수 있습니다. 세그먼트는 데이터가 변경되면 실시간으로 업데이트되며, 타겟팅 및 메시징 목적에 따라 필요한 만큼 세그먼트를 생성할 수 있습니다.

## 1단계: 세그먼트 섹션으로 이동합니다.

**오디언스** > 세그먼트로 이동합니다.

## 2단계: 세그먼트 이름 지정하기

세그먼트 **생성을** 선택하여 세그먼트 구축을 시작합니다. 필터링하려는 사용자 유형을 설명하여 세그먼트의 이름을 지정합니다. 이렇게 하면 캠페인이나 캔버스에서 세그먼트를 타겟팅할 때 해당 세그먼트를 식별하는 데 도움이 됩니다. 모호한 세그먼트 제목은 혼란을 줄 수 있습니다.

선택적으로 다음을 수행할 수 있습니다:
- 세그먼트에 설명을 추가하여 이 오디언스의 의도에 대한 자세한 내용을 제공하고 다른 팀원들이 다시 참조할 수 있도록 메모를 남기세요.
- 세그먼트에 [팀을]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/teams/) 추가합니다.
- 세그먼트에 [태그를]({{site.baseurl}}/user_guide/administrative/app_settings/tags/) 추가하여 더욱 세분화하여 정리하세요.

세그먼트 이름이 "휴면 사용자"이고 세그먼트 설명이 "지난 14일 이내에 활동하지 않은 사용자를 타겟팅하기 위한 주요 휴면 사용자 세그먼트입니다."인 세그먼트 만들기 모달을 두 개의 버튼과 함께 생성합니다: 세그먼트 취소 및 생성.]({% image_buster /assets/img_archive/segment_app_selection.png %}){: style="max-width:80%;"}

## 3단계: 앱 또는 플랫폼 선택하기

**모든 앱의 사용자** (기본값) 또는 **특정 앱의 사용자를** 선택하여 타겟팅할 앱 또는 플랫폼을 선택합니다. **특정 앱의 사용자는** 지정된 앱에 세션이 하나 이상 있는 사용자를 타겟팅합니다.

예를 들어 iOS 기기에만 인앱 메시지를 보내려면 iOS 앱을 선택하세요. 이렇게 하면 iOS와 Android 기기를 모두 사용하는 사용자는 iOS 기기에서만 메시지를 수신할 수 있습니다. 특정 앱 목록에서 앱이 **없는 사용자** 옵션을 사용하면 세션이 없고 앱 데이터가 없는 사용자(일반적으로 사용자 가져오기 또는 REST API를 통해 생성됨)를 포함할 수 있습니다.

사용 앱 섹션에서 '모든 앱의 사용자' 옵션을 선택한 상태에서 세그먼트 세부정보 패널을 클릭합니다.]({% image_buster /assets/img_archive/Segment2.png %}){: style="max-width:80%;"}

## 4단계: 세그먼트에 필터 추가하기

세그먼트에 필터를 하나 이상 추가합니다. 원하는 만큼 필터를 조합하여 세그먼트를 보다 구체적으로 세분화할 수 있습니다. 

{% alert note %}
Braze는 앱을 처음 사용할 때까지 사용자 프로필을 생성하지 않으므로 아직 앱을 열지 않은 사용자를 타겟팅할 수 없습니다.
{% endalert %}

#### 필터 그룹

필터는 필터 그룹으로 구성됩니다. 모든 필터는 최소 하나의 필터가 있는 필터 그룹에 속해야 합니다. 세그먼트에는 여러 개의 필터 그룹이 있을 수 있습니다. **필터 그룹을** 추가하려면 **필터 그룹 추가를** 선택합니다. 필터 그룹 이름 옆에 마우스를 가져가면 표시되는 아이콘을 선택하여 필터 그룹 이름을 편집합니다.

그룹 이름 옆에 편집 아이콘이 있는 그룹을 필터링합니다.]({% image_buster /assets/img_archive/edit_filter_group_name.png %})

각 필터 옆에 있는 아이콘을 선택하여 필터 에디터를 축소하거나 개별 필터를 복제합니다. 필터를 복제한 후 각 드롭다운에서 해당 필터의 값을 조정할 수 있습니다.

#### AND 및 OR을 사용한 세그먼트 세분화 논리

필터 그룹 내에서 필터는 'AND' 또는 'OR'로 연결할 수 있습니다. 필터 그룹 간에는 'AND' 또는 'OR'로 그룹을 연결할 수 있습니다. 필터 그룹을 사용할 때 다음과 같은 세그먼트 로직을 만들 수 있습니다:
- (가, 나, 다) 또는 (다, 라, 마)
- (가 또는 나 또는 다) 및 (다 또는 라 또는 프)

필터에 'OR'을 선택하면 세그먼트에 해당 필터 중 하나, 일부 또는 전부를 충족하는 사용자가 포함된다는 의미입니다. "AND"를 선택하면 해당 필터를 통과하지 못한 사용자는 세그먼트에 포함되지 않습니다.

{% alert tip %}
네거티브 필터가 포함된 필터(예: 구독 그룹에서 "is not")에 대해 "OR"을 선택하는 경우 사용자는 "OR" 필터 중 하나만 충족하면 세그먼트에 포함될 수 있다는 점을 기억하세요. 다른 필터와 관계없이 [제외](#exclusion) 필터를 적용하려면 [제외 그룹을](#exclusion) 사용합니다.
{% endalert %}

{% details When to avoid the OR operator %}

`OR` 연산자 사용을 피해야 하는 사용자 타겟팅 상황이 있을 수 있습니다. `OR` 연산자는 사용자가 문에 있는 하나 이상의 필터 기준을 충족하면 참으로 평가되는 문을 만듭니다. 예를 들어 '식도락가'에 속하지만 '비식도락가'나 '사탕 애호가'에 속하지 않는 사용자 세그먼트를 만들려는 경우 `OR` 연산자를 사용하면 됩니다.

'식도락가' 또는 '사탕을 좋아하지 않는' 세그먼트가 아닌 '식도락가' 세그먼트에 있는 사용자를 필터링합니다.]({% image_buster /assets/img_archive/or_operator_segment.png %})

그러나 '식도락가' 세그먼트에 속하고 '비식도락가' 및 '사탕 애호가' 세그먼트에 속하지 않는 사용자를 세분화하는 것이 목표인 경우 `AND` 연산자를 사용하세요. 이렇게 하면 캠페인이나 캔버스를 수신하는 사용자가 의도한 세그먼트('미식가')에 속하고 다른 세그먼트('비미식가' 및 '캔디 애호가')에 동시에 속하지 않게 됩니다. 

두 개 이상의 필터가 동일한 속성을 참조하는 경우 다음과 같은 네거티브 타겟팅 기준을 `OR` 연산자와 함께 사용해서는 안 됩니다:

- `not included`
- `is not`
- `does not equal`
- `does not match regex`

`not included`, `is not`, `does not equal`, `does not match regex` 이 문장에서 `OR` 연산자와 함께 두 번 이상 사용되는 경우 해당 속성에 대한 모든 값을 가진 사용자를 타겟팅합니다.

{% enddetails %}

#### 필터 연산자

선택한 특정 필터에 따라 필터 값을 식별하는 연산자가 달라집니다. 다양한 유형의 커스텀 속성에 사용할 수 있는 연산자에 대해 자세히 알아보려면 [커스텀 속성 저장소를]({{site.baseurl}}/user_guide/data_and_analytics/custom_data/custom_attributes/#setting-custom-attributes) 참조하세요. "is any of" 연산자를 사용할 때 해당 필드에 포함할 수 있는 항목의 최대 개수는 256개입니다.

{% alert note %}
Braze는 앱을 처음 사용할 때까지 사용자 프로필을 생성하지 않으므로 아직 앱을 열지 않은 사용자를 타겟팅할 수 없습니다.
{% endalert %}

AND 연산자로 그룹을 세분화하여 필터링합니다.]({% image_buster /assets/img_archive/segmenter_filter_groups.png %})

{% alert important %}
세그먼트 **멤버십** 필터를 이미 사용하고 있는 세그먼트는 더 이상 포함하거나 다른 세그먼트에 중첩할 수 없습니다. 이렇게 하면 세그먼트 A가 세그먼트 B를 포함하고 다시 세그먼트 A를 포함하려고 하는 사이클을 방지할 수 있습니다. 그렇게 되면 세그먼트가 계속 자신을 참조하게 되어 실제로 누가 그 세그먼트에 속해 있는지 계산할 수 없게 됩니다.

또한 이와 같이 세그먼트를 중첩하면 복잡성이 증가하여 속도가 느려질 수 있습니다. 대신 동일한 필터를 사용하여 포함하려는 세그먼트를 다시 만듭니다.
{% endalert %}

#### 제외 그룹(선택 사항) {#exclusion}

세그먼트를 구축할 때 하나 또는 여러 개의 제외 그룹을 적용할 수 있습니다. 제외 그룹에는 세그먼트에서 제외할 사용자를 식별하는 기준이 포함되며 항상 "AND NOT" 연산자로 필터 그룹에 연결됩니다.

제외 그룹은 세그먼트 기준을 재정의합니다. 사용자가 제외 그룹 기준에 해당하는 경우 필터 그룹 내 기준을 충족하더라도 해당 사용자는 세그먼트에 포함되지 않습니다.

필터 그룹을 만들 때와 마찬가지로 필터를 추가하여 제외 그룹을 만듭니다. 제외 그룹의 _예상 도달 가능한 사용자_ 통계에는 제외 기준이 적용된 후 세그먼트에 남아있는 예상 사용자 수가 표시됩니다.

제외된 사용자는 세그먼트의 _총 도달 가능한 사용자_ 통계에 포함되지 않습니다.

두 개의 필터가 있는 제외 그룹입니다.]({% image_buster /assets/img_archive/segmenter_exclusion_groups.png %})

#### 퍼널 통계 보기

**퍼널 통계 보기를** 선택하여 해당 필터 그룹에 대한 통계를 표시하고 추가된 각 필터가 세그먼트 통계에 어떤 영향을 미치는지 확인합니다. 해당 시점까지 모든 필터에 의해 타겟팅된 사용자의 예상 수와 백분율을 확인할 수 있습니다. 필터 그룹에 대한 통계가 표시되면 필터를 변경할 때마다 통계가 자동으로 업데이트됩니다. 이러한 통계는 추정치이며 생성하는 데 시간이 걸릴 수 있습니다.

필터 사이에 AND를 사용하면 퍼널 통계가 감소하고, 필터 사이에 OR을 사용하면 퍼널 통계가 증가한다는 점에 유의하세요.

세그먼트 퍼널 통계가 있는 두 개의 필터.]({% image_buster /assets/img_archive/segment_funnel_statistics.png %})

사용자 흐름을 설명하는 필터를 추가하면 사용자가 이탈하는 지점을 확인할 수 있습니다. 예를 들어, 소셜 네트워킹 앱의 경우 온보딩 과정에서 사용자를 잃을 수 있는 위치를 확인하려면 가입, 친구 추가, 첫 메시지 전송에 대한 커스텀 데이터 필터를 추가할 수 있습니다. 85%의 사용자가 가입하고 친구를 추가했지만 45%만이 첫 번째 메시지를 보냈다면 온보딩 및 마케팅 캠페인에서 더 많은 메시지를 보내도록 유도하는 데 집중해야 한다는 것을 알 수 있습니다.

#### 테스트 세그먼트

세그먼트에 앱과 필터를 추가한 후 사용자를 조회하여 세그먼트 기준과 일치하는지 확인하여 세그먼트가 예상대로 설정되었는지 테스트할 수 있습니다. 이렇게 하려면 **사용자 검색** 섹션에서 사용자의 `external_id` 또는 `braze_id` 을 검색합니다. **사용자 조회에서는** 이메일 주소로 검색할 수 없습니다.

검색 필드가 있는 사용자 조회 섹션.]({% image_buster /assets/img_archive/user_lookup.png %}){: style="max-width:70%;"}

사용자 조회는 언제 사용할 수 있습니다:
- 세그먼트 만들기
- 캠페인 또는 캔버스 오디언스 설정하기
- 오디언스 경로 단계 설정하기

사용자가 세그먼트, 필터 및 앱 기준과 일치하면 알림에 해당 내용이 표시됩니다.

'testuser'의 사용자 조회는 "testuser가 모든 세그먼트, 필터 및 앱과 일치합니다."라는 알림을 트리거합니다.]({% image_buster /assets/img_archive/user_lookup_match.png %})

사용자가 세그먼트, 필터 또는 앱 기준의 일부 또는 전체와 일치하지 않는 경우 문제 해결을 위해 누락된 기준이 나열됩니다.

"test1이 다음 타겟팅 기준과 일치하지 않습니다:"라는 경고와 함께 누락된 기준을 표시하는 사용자 조회.]({% image_buster /assets/img_archive/user_lookup_nomatch.png %})

#### 단일 사용자 세그먼트

사용자 이름이나 사용자 ID와 같이 사용자를 식별하는 고유 속성을 사용하여 단일 사용자 세그먼트(또는 소수의 사용자로 구성된 세그먼트)를 만들 수 있습니다.

그러나 세그먼트 통계는 95% 신뢰 구간에서 결과가 +/- 1% 이내인 무작위 샘플을 기반으로 계산되므로 세분화 통계 또는 미리 보기에 이 개별 사용자가 표시되지 않을 수 있습니다. 사용자 기반이 클수록 세그먼트의 크기가 대략적인 추정치일 가능성이 높습니다. 세그먼트에 타겟팅하려는 단일 사용자가 포함되어 있는지 확인하려면 **정확한 통계 계산을** 선택합니다. 이렇게 하면 세그먼트의 정확한 사용자 수를 99.999% 이상의 정확도로 계산할 수 있습니다.

Braze에는 사용자 ID 또는 이메일 주소로 특정 사용자를 타겟팅하는 테스트 필터가 있습니다.

## 5단계: 세그먼트 저장하기

**저장을** 선택합니다. 이제 사용자에게 메시지를 보낼 준비가 되었습니다!

## 세그먼트 크기 측정

세그먼트의 멤버십 및 규모 모니터링에 대해 자세히 알아보려면 [세그먼트 규모 측정하기를]({{site.baseurl}}/user_guide/engagement_tools/segments/measuring_segment_size/) 참조하세요.

## 세그먼트 보관하기

특정 세그먼트가 더 이상 필요하지 않거나 폐기하려는 경우 **세그먼트** 페이지로 이동하여 해당 세그먼트 행의 메뉴에서 **보관을** 선택하여 해당 세그먼트를 보관할 수 있습니다.

{% alert warning %}
세그먼트를 보관하면 해당 세그먼트가 단일 캔버스 컴포넌트에서만 사용되는 경우에도 해당 세그먼트를 사용하는 모든 캠페인이나 캔버스도 보관됩니다. 여기에는 중첩된 세그먼트도 포함되며, 세그먼트와 이를 사용하는 캠페인 또는 캔버스도 모두 아카이브됩니다.
<br><br>
관련 세그먼트를 아카이브하면 어떤 캠페인과 캔버스가 아카이브될 것인지에 대한 경고 메시지가 표시됩니다.
{% endalert %}

**세그먼트** 페이지에서 해당 세그먼트로 이동한 다음 **보관 해제를** 선택하여 세그먼트의 보관을 해제할 수 있습니다.

## 사용자가 여러 기기를 보유한 경우의 타겟팅 동작

사용자가 여러 기기에서 동일한 계정으로 로그인하는 경우 두 개 이상의 기기를 보유한 것으로 간주합니다. [고객 프로필의]({{site.baseurl}}/user_guide/engagement_tools/segments/user_profiles/) **최근 기기** 섹션에서 여러 기기를 확인할 수 있습니다.

기기 종속 필터(기기 모델, 기기 OS, 앱 버전)를 사용하여 세그먼트를 세분화하는 경우 세그먼트에는 필터 기준에 일치하는 모든 사용자가 포함됩니다. 이러한 사용자에게는 필터 기준에 맞지 않을 수 있는 기기를 포함하여 모든 기기에 메시지가 전송됩니다. 예를 들어 사용자 A가 두 대의 기기를 가지고 있다고 가정해 보겠습니다: 기기 1은 OS 13.0, 기기 2는 OS 10.0입니다. 세그먼트가 OS 10.0 사용자를 타겟팅하는 경우 이 사용자는 해당 세그먼트의 일부가 되어 두 기기 모두에서 메시지를 수신하게 됩니다.

### 푸시 알림

각 사용자에게 푸시 알림을 하나만 보내도록 지정할 수 있습니다. [메시지를 작성할]({{ssite.baseurl}}/user_guide/message_building_by_channel/push/creating_a_push_message#step-4-compose-your-push-message) 때 **추가 설정에서** **사용자가 마지막으로 사용한 기기로만 보내기를** 선택합니다.

'추가 설정'에서 사용자가 마지막으로 사용한 기기로만 전송하도록 체크박스를 선택합니다.]({% image_buster /assets/img_archive/send_to_last_device.png %}){: style="max-width:60%;"}

### 고려 사항

- **전송된 메시징은 오디언스 규모를 초과할 수 있습니다.** 일부 사용자에게 두 대 이상의 기기가 있는 경우 각 기기가 메시지를 수신할 수 있습니다. 이로 인해 세그먼트의 사용자보다 더 많은 수의 메시지를 보낼 수 있습니다.
- **사용자의 세그먼트 멤버십이 예상과 다르게 표시될 수 있습니다.**
    - 사용자는 다른 기기와 관련된 속성을 기반으로 현재 기기에서 타겟팅될 수 있습니다. 사용자가 메시지를 받을 것으로 예상하지 못한 경우 여러 기기에 대한 고객 프로필을 확인하세요.
    - 사용자가 전송 시점에 타겟팅 세그먼트에 포함되었을 수 있지만, 해당 기기와 관련된 행동으로 인해 이후에는 해당 세그먼트에 포함되지 않을 수 있습니다. 이로 인해 사용자가 현재 필터 기준과 일치하지 않는데도 캠페인이나 캔버스를 받게 될 수 있습니다. <br><br>예를 들어, 사용자가 현재 OS 13.0을 사용하고 있어도 가장 최신 앱 버전인 OS 10.0을 사용하는 사용자를 타겟팅하는 메시지를 수신할 수 있습니다. 이 경우 메시징이 전송될 당시 사용자는 OS 10.0을 사용 중이었으며 이후 OS 13.0으로 업그레이드했습니다.<br><br> 마찬가지로 사용자가 나중에 앱 버전이 다른 기기를 사용하는 경우 고객 프로필이 최신 앱 버전으로 업데이트됩니다. 이로 인해 사용자가 메시징을 보낼 때 자격을 갖추었음에도 불구하고 메시징을 받을 자격이 없는 것처럼 보일 수 있습니다.
