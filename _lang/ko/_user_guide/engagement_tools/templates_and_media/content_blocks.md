# 콘텐츠 블록 라이브러리

> 콘텐츠 블록 라이브러리를 사용하면 재사용 가능한 크로스채널 콘텐츠를 중앙 집중식 단일 위치에서 관리할 수 있습니다.

콘텐츠 블록을 사용하면 가능합니다:

- 헤더와 푸터를 사용하여 이메일 캠페인의 일관된 모양과 느낌을 만드세요.
- 동일한 오퍼 코드를 여러 채널에 배포합니다.
- 일관된 정보와 자산으로 메시지를 구축하는 데 사용할 수 있는 사전 정의된 자산을 생성하세요.
- 전체 메시지 본문을 다른 메시지로 복사합니다.

## 콘텐츠 블록 만들기

콘텐츠 블록을 생성하는 데 사용되는 편집기에는 클래식과 드래그 앤 드롭의 두 가지 유형이 있습니다. 이 두 가지 유형의 편집기는 콘텐츠 블록 유형에 해당합니다: HTML 및 드래그 앤 드롭. [API를 사용하여]({{site.baseurl}}/api/endpoints/templates/) 콘텐츠 블록을 생성하고 매니저를 관리할 수도 있습니다.

### 콘텐츠 블록 사양

| 콘텐츠 블록 속성 | 사양 |
|---|---|
| 이름 | 최대 100자까지 입력할 수 있는 필수 입력란입니다. 콘텐츠 블록이 저장된 후에는 이름을 변경할 수 없습니다. 또한 이전 콘텐츠 블록이 보관되어 있더라도 새 콘텐츠 블록의 이름을 이전 콘텐츠 블록과 같은 이름으로 지정할 수 없습니다. |
| 설명 | (선택 사항) 최대 250자까지 입력할 수 있습니다. 콘텐츠 블록의 용도와 사용처를 다른 Braze 사용자가 알 수 있도록 콘텐츠 블록에 대해 설명하세요. |
| 콘텐츠 크기 | 최대 50KB. |
| 배치 | 콘텐츠 블록은 이메일 바닥글 내에서 사용할 수 없습니다. |
| 생성 | HTML 편집기 또는 드래그 앤 드롭 편집기. |

## 콘텐츠 블록 사용

콘텐츠 블록을 만든 후에는 다음 단계에 따라 메시징에 콘텐츠 블록을 삽입할 수 있습니다: 

1. **콘텐츠 블록 세부 정보** 섹션에서 **콘텐츠 블록 Liquid 태그를** 복사합니다.
2. 메시징에 콘텐츠 블록 Liquid 태그를 삽입합니다. Liquid를 입력하기 시작하면 태그가 자동으로 채워지도록 할 수도 있습니다.

### 알아두어야 할 사항

- 드래그 앤 드롭 이메일에 HTML 콘텐츠 블록을 **사용하거나** HTML 이메일에 콘텐츠 블록을 드래그 앤 드롭하면 예기치 않은 렌더링 문제가 발생할 수 있습니다. 드래그 앤 드롭 편집기는 콘텐츠를 동적으로 렌더링하는 HTML과 CSS를 생성하는 반면 HTML 편집기는 정적이기 때문입니다.
- 캔버스 이벤트 속성정보는 캔버스에서만 지원됩니다. 캠페인에서 캔버스 항목 속성이 있는 콘텐츠 블록을 참조하는 경우 콘텐츠 블록이 채워지지 않습니다.

### 콘텐츠 블록 업데이트 및 복사하기

콘텐츠 블록을 업데이트하도록 선택하면 Liquid를 통해 콘텐츠 블록이 삽입된 경우 해당 콘텐츠 블록이 사용되는 모든 메시징에서 업데이트됩니다. 드래그 앤 드롭 편집기의 **행** 아래 **콘텐츠** 블록 드롭다운을 사용하여 콘텐츠 블록을 가져오면 모든 메시징에서 콘텐츠 블록이 업데이트되지 않습니다.

하나의 메시징에 대한 콘텐츠 블록을 업데이트하거나 다른 메시징에 사용할 사본을 만들려면 원래 메시지의 HTML을 새 메시지에 복사하거나 원래 콘텐츠 블록(이미 메시징에 사용되었어야 함)을 편집하여 저장하면 됩니다. 새 콘텐츠 블록으로 저장할 수 있는 메시지가 표시됩니다.

콘텐츠 블록을 편집한 후 **콘텐츠 블록 시작을** 선택하여 업데이트된 콘텐츠 블록을 저장하고 시작할 수 있습니다. 또는 **더보기** > **복제를** 선택하여 콘텐츠 블록의 복제본을 만들 수 있습니다.

"뉴스레터에 오신 것을 환영합니다"라고 표시되는 콘텐츠 블록입니다.]({% image_buster /assets/img/copy-content-block.png %})

콘텐츠 블록을 [복제할]({{site.baseurl}}/user_guide/engagement_tools/templates_and_media/managing_templates/) 수도 있습니다. 이렇게 하면 콘텐츠 블록의 초안 사본이 생성됩니다.

### 콘텐츠 블록 미리 보기

활성 캠페인 또는 캔버스에 콘텐츠 블록을 추가한 후 콘텐츠 블록 위에 마우스를 올리고 <i class="fa fa-eye preview-icon"></i> **미리보기** 아이콘을 선택하면 콘텐츠 블록 라이브러리에서 이 콘텐츠 블록을 미리 볼 수 있습니다. 

이 미리보기에는 콘텐츠 블록을 만든 사람, 태그, 만든 날짜, 마지막 편집 날짜, 설명, 편집기 유형, 세부 정보가 포함된 포함 수, 콘텐츠 블록의 실제 미리보기와 같은 콘텐츠 블록에 대한 정보가 포함되어 있습니다.

자전거와 춤을 위한 콘텐츠 블록( "Workout_Promo" )의 미리보기로, 6개의 콘텐츠가 포함되어 있습니다.]({% image_buster /assets/img/preview_tab_content_block.png %}){: style="max-width:60%;"} 

### 콘텐츠 블록 중첩하기

콘텐츠 블록은 중첩할 수 있지만 한 번만 중첩할 수 있습니다. 콘텐츠 블록 A를 콘텐츠 블록 B에 중첩할 수는 있지만 콘텐츠 블록 B를 콘텐츠 블록 C에 중첩할 수는 없습니다.

{% alert warning %}
콘텐츠 블록의 세 번째 레벨을 중첩하는 것을 막는 것은 없지만 콘텐츠가 두 번째 레벨 이상으로 중첩되어 확장되는 것은 볼 수 없습니다. 콘텐츠와 Liquid 스니펫이 메시지에서 제거됩니다.
{% endalert %}

또한 이메일 바닥글은 콘텐츠 블록 내에서 사용할 수 있지만 콘텐츠 블록은 이메일 바닥글 내에서 사용할 수 없습니다.

### 콘텐츠 블록 아카이빙

세 가지 옵션이 표시되는 설정 드롭다운 메뉴가 확장되었습니다: 보관, 복제 및 작업 공간에 복사.]({% image_buster /assets/img/template_archive_cog.png %}){: style="max-width:20%;float:right;margin-left:15px;" }

콘텐츠 블록 사용을 마치면 [템플릿 & 미디어]({{site.baseurl}}/user_guide/engagement_tools/templates_and_media/managing_templates/) 페이지에서 콘텐츠 블록을 보관할 수 있습니다. 아카이브된 콘텐츠 블록은 읽기 전용이므로 편집하기 전에 콘텐츠 블록의 아카이브를 해제하세요. 콘텐츠 블록이 메시징에 사용된 경우에는 보관할 수 없습니다.

#### 모범 사례

- 블록이 몇 개의 이메일에만 사용되는 경우 오래된 블록을 보관하고 보관되지 않은 최신 블록으로 실시간 메시지를 업데이트하는 것이 좋습니다.
- 블록에 오타만 있거나 사소한 변경이 필요한 경우에는 블록을 보관하지 않는 것이 좋습니다. 대신 블록을 업데이트하고 전송을 시작하세요!
- 이 목록의 첫 번째 제안으로 합리적으로 관리할 수 있는 것보다 많은 메시징에 블록이 사용되는 경우 해당 블록에서 모든 콘텐츠를 삭제하고 보관하는 것이 좋습니다. 이렇게 하면 새로 전송되는 이메일에 오래된 정보가 포함되지 않습니다.
- 실수로 콘텐츠 블록을 아카이브한 경우 아카이브를 취소할 수 있습니다.

저장된 콘텐츠 블록 패널에서 "Test_32" 에 대한 설정 드롭다운 메뉴가 확장되어 세 가지 옵션이 표시됩니다: 보관 해제, 복제 및 작업 공간으로 복사]({% image_buster /assets/img/unarchive-content-block.png %})
