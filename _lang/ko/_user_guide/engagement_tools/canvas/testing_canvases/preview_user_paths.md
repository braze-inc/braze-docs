# 캔버스에서 사용자 경로 미리보기

> 사용자를 위해 만든 캔버스 여정을 직접 경험해 보세요. 사용자가 받는 타이밍과 메시지를 미리 볼 수 있습니다. 이러한 테스트 실행은 캔버스를 발송하기 전에 메시지가 올바른 오디언스에게 전송되는지 확인하는 품질 보증 역할을 합니다.

## 테스트 실행 생성

사용자 여정을 미리 보려면 다음 단계를 따르세요:

1. 캔버스 빌더로 이동합니다. 저장하지 않은 변경 사항을 저장하고 오류를 해결합니다.
2. 바닥글에서 **테스트 캔버스**를 선택합니다.
3. 테스트 사용자를 선택합니다.
4. (선택 사항) 테스트 수신자를 선택합니다.
5. **테스트 실행**을 선택합니다.

캔버스를 편집할 권한이 없어도 미리보기를 실행할 수 있지만, 저장되지 않은 변경 사항이 있는 경우 해당 변경 사항이 포함된 상태로 미리보기가 실행됩니다.

### 지원되는 단계

다음 단계가 지원됩니다:
- 메시지 
- 오디언스 경로
- 결정 분할
- 지연
- 행동 경로
- 실험 경로
- 사용자 업데이트(UI 편집기에서만 지원되며, JSON 편집기를 사용하는 단계는 건너뜁니다)

테스트가 위에 나열되지 않은 단계 유형과 겹치는 경우, 지원되지 않는 단계는 건너뛰고 테스트 사용자는 다음 지원되는 단계로 계속 진행합니다.

### 캔버스 단계 세부 정보

진입 기준에 대한 자세한 내용을 보려면 **자세히 보기**를 선택합니다. 세분화가 있는 단계는 충족된 기준 또는 충족되지 않은 기준을 보여줍니다. 메시지는 전달 검증 및 채널 적격성에 대해서도 이를 보여줍니다. 메시지 단계는 어떤 채널이 전송되었고 어떤 채널이 전송되지 않았는지를 보여줍니다.

### Liquid

Braze는 실제 테스트 메시지를 전송하지 않더라도 테스트 실행 중에 Liquid 로직을 처리합니다. 즉, [메시지 중단 로직]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/aborting_messages/#aborting-messages)과 기타 Liquid 로직이 반영되어 캔버스 사용자 여정에 영향을 미칠 수 있습니다.

미리보기가 중단하지 않고 사용자 여정의 마지막 단계를 전송하는 경우, 미리보기는 캔버스 진입 시간을 기준으로 사용자가 실제로 해당 단계에 있을 시간이 아닌 현재 시간을 Liquid 평가를 위한 테스트 시간으로 사용할 수 있습니다.

## 타이밍 미리보기

예약된 캔버스의 경우, 테스트 사용자는 다음 예약된 진입 시간에 들어갑니다. 시작 날짜가 있는 액션 기반 캔버스의 경우, 테스트 사용자는 시작 날짜와 시간에 들어갑니다. 

기본 시작 시간이 여전히 적용되지만, 모든 경우에 진입 시간은 구성 가능하므로 과거 또는 미래의 날짜를 시뮬레이션할 수 있습니다. 그러나 캔버스의 시작 날짜 이전이나 종료 날짜 이후에는 테스트할 수 없습니다.

메시지 및 지연 단계는 지연을 재구성할 필요 없이 사용자가 진행하거나 메시지를 수신하는 시간을 표시합니다. 단계에서 Intelligent Timing 사용 여부를 나타내지만, 이 사용자 경로 미리보기는 테스트 사용자에 대한 추정치를 계산하지 않는다는 점에 유의하세요.

"커스텀 속성 값의 변경"과 같은 액션 트리거가 있는 캔버스의 경우, Braze는 트리거에서 사용자의 속성을 일시적으로 비워서 변경을 시뮬레이션하려고 시도합니다. 이는 **캔버스의 테스트 실행에만 해당**되며 고객 프로필에는 영향을 미치지 않습니다. 이는 속성이 현재 값에서 변경되는지를 테스트하기 위한 것입니다.

## 사용자의 진입과 이탈

테스트 사용자는 실제로 적격하지 않더라도 미리보기에 진입합니다. 적격하지 않은 경우, 기준을 충족하지 못한 이유를 확인할 수 있습니다. 테스트 사용자가 미리보기에 진입하면, 테스트 사용자가 타겟 오디언스 기준을 충족하고 액션 트리거 기준을 수행한 것으로 가정합니다. 예를 들어, 진입 기준에 커스텀 이벤트를 사용하는 캔버스의 경우, 테스트 사용자는 진입 기준에서 예상대로 커스텀 이벤트를 수행한 것으로 가정됩니다. 그러나 동일한 커스텀 이벤트가 캔버스의 다른 곳(예: 종료 기준)에서 사용되는 경우, 이것이 사용자 경로에 어떤 영향을 미칠 수 있는지 고려하세요.

테스트 사용자가 캔버스에 진입할 수 있도록 가정된 이벤트, API 트리거, 커스텀 속성 및 캔버스 진입 등록정보는 실제 고객 프로필에서 업데이트되지 않으며 테스트 실행을 넘어 지속되지 않습니다. 예를 들어, 테스트 중에 커스텀 속성을 캔버스 트리거로 사용하면 커스텀 속성 변경을 트리거한 **것처럼** 트리거 기준이 사용자의 미리보기에 적용됩니다.

### 고려 사항

종료 기준(이벤트 등록정보 포함)에 해당하는 액션이 있는 행동 경로를 테스트하면 종료 기준이 트리거되고 테스트 실행이 종료됩니다. 종료 기준에 해당하는 메시지 단계를 테스트하면 종료 기준이 트리거되고 테스트 실행이 종료됩니다. 

현재로서는 행동 경로 내에서 특정 이벤트나 등록정보를 선택하여 종료 기준을 트리거할 수 없습니다(경로 전체만 트리거할 수 있습니다). 사용자가 잠재적으로 여러 종료 기준을 충족할 수 있는 경우, 가장 먼저 처리되어 충족하는 기준이 결과로 표시됩니다.

## 실험 경로 및 캔버스 배리언트

- 최상위 배리언트가 있는 캔버스의 경우 테스트를 시작할 때 배리언트를 선택합니다.
- 실험 경로의 경우 테스트 사용자가 해당 단계에 도달했을 때 진행할 배리언트를 선택합니다.
- 개인화된 경로 또는 위닝 배리언트를 사용하는 실험 경로의 경우, 테스트 사용자가 메시지 단계에서 대기하는 지연 기간이 있지만, Braze는 사용자가 선택한 배리언트를 즉시 진행했다고 가정하므로 이 지연은 고려되지 않습니다.

## 테스트 전송

테스트 실행이 진행될 때 내부 테스트 그룹 또는 개별 사용자에게 테스트 메시지를 보내도록 선택할 수 있습니다. 이는 사용자가 테스트 경로에서 만나는 메시지만 전송된다는 것을 의미합니다. 수신자는 기본적으로 자신의 속성이 포함된 메시지를 받지만, 테스트 사용자의 속성으로 이를 재정의할 수 있습니다.

경로를 미리 보지 않고 경로에 관계없이 캔버스의 모든 테스트 메시지를 한 번에 보내려면 **테스트 전송** 탭에서 **모든 테스트 메시지 보내기**를 선택하면 됩니다.

## 반응형 동작

캔버스 단계는 사용자 경로를 미리 볼 때 타이밍에 반응합니다. 사용자 업데이트 단계를 통해 수행한 업데이트는 흐름의 후속 단계에 반영되지만 실제 고객 프로필에는 적용되지 않습니다. 사용자가 배리언트에 진입한 효과는 미리보기의 이후 단계에 반영됩니다.

마찬가지로, 필터는 테스트 사용자가 캔버스의 다른 단계와 상호작용한 결과로 발생한 액션을 인식합니다. 예를 들어, 이 미리보기 모드는 사용자가 캔버스에서 이전에 "전송됨"으로 표시된 메시지 단계를 만났음을 인식하고, 테스트 사용자가 행동 경로를 진행하기 위해 "액션을 취했다"고 인식합니다.

반응형 동작에 대한 자세한 내용은 [종료 기준]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/exit_criteria)을 참조하세요.

## 연결된 콘텐츠

연결된 콘텐츠는 캔버스에 포함되어 있으면 실행됩니다. 즉, 연결된 콘텐츠 호출이 있는 캔버스 또는 연결된 콘텐츠가 포함된 콘텐츠 블록을 테스트하는 경우 캔버스가 연결된 콘텐츠 호출을 전송하여 다른 캠페인이나 캔버스에서 참조된 데이터를 수정할 수 있습니다.

사용자 경로를 미리 볼 때 다른 캔버스 또는 캠페인에서 참조된 고객 프로필이나 데이터를 변경하는 연결된 콘텐츠를 제거하는 것을 고려하세요.

## 웹훅

웹훅은 테스트 메시지가 전송될 때 실행되지만, 테스트 실행 중에는 실행되지 않습니다. 연결된 콘텐츠와 마찬가지로, 다른 캔버스나 캠페인에서 참조된 고객 프로필이나 데이터를 변경하는 웹훅을 제거하는 것을 고려하세요.

## 컨텍스트 변수 및 시드 그룹

이메일을 메시징 채널로 사용하는 메시지 단계의 경우, 시드 그룹은 사용자가 캔버스에서 이 단계에 도달하면 이메일의 시드 사본을 전송합니다. 이러한 시드 사본은 시드 그룹 수신자 자체의 캔버스 여정의 일부로 전송되지 않으므로, Braze는 해당 수신자에 대해 [컨텍스트 단계]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/context)를 실행하거나 컨텍스트 변수를 평가하지 않습니다. 이메일 콘텐츠가 컨텍스트 변수를 참조하는 경우, 시드 그룹 수신자는 해당 데이터가 채워지지 않은 시드 사본을 받습니다. 컨텍스트 변수 데이터에 의존하는 메시지를 테스트하려면 시드 그룹 대신 **테스트 캔버스** 미리보기에서 테스트 전송을 사용하세요.

## 사용 사례

이 시나리오에서 캔버스는 앱에서 세션이 없는 사용자를 타겟팅하도록 설정되어 있습니다. 이 여정에는 환영 이메일이 포함된 메시지 단계, 하루로 설정된 지연 단계, 그리고 세션이 하나 이상 있는 사용자와 그 외 모든 사용자로 분할되는 오디언스 경로 단계가 포함됩니다. 사용자가 어떤 오디언스 경로에 속하는지에 따라 후속 메시지 단계가 전송됩니다.
