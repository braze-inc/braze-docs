# 캔버스 분석

> 지금 구축하는 것이 바늘을 움직이고 있는지 알아야 합니다. 캔버스 분석을 통해 구축 중인 경험이 목표에 어떤 영향을 미치는지 이해하기 위한 전체적인 그림을 구축할 수 있습니다. 

캔버스를 구축하고 라이브로 설정한 후에는 **캔버스** 페이지로 이동하여 캔버스를 선택하여 세부 정보 페이지를 엽니다. 여기에서 캔버스의 성능/성과를 측정하고 테스트할 수 있습니다.

## 캔버스 개요

**캔버스 세부 정보** 페이지 상단에는 최상위 캔버스 통계가 포함되어 있습니다. 여기에는 캔버스 내에서 전송된 메시지 수, 고객이 캔버스에 들어온 총 횟수, 전환한 고객 수 및 총 전환율, 캔버스에서 발생한 매출, 예상 총 오디언스 수 등이 포함됩니다. 

캔버스의 목표 대비 성능/성과를 확인할 수 있는 개략적인 개요를 확인할 수 있는 좋은 곳입니다.

\![]({% image_buster /assets/img_archive/Journey_5.png %})

### 마지막으로 본 이후 변경 사항

다른 팀원이 캔버스를 업데이트한 횟수는 캔버스 개요 페이지의 *마지막으로 본 이후 변경사항* 측정기준을 통해 추적할 수 있습니다. **마지막으로 본 이후 변경사항을** 선택하면 캔버스 이름, 일정, 태그, 메시지, 오디언스, 승인 상태 또는 팀 액세스 구성에 대한 업데이트 체인지로그를 볼 수 있습니다. 각 업데이트에 대해 누가 언제 업데이트를 수행했는지 확인할 수 있습니다. 이 체인지로그를 사용하여 캔버스에 대한 변경 사항을 감사할 수 있습니다.

## 성능/성과 시각화

캔버스 **세부 정보** 페이지로 이동하면 캔버스에 들어온 사용자 수, 다음 단계로 진행한 사용자 수, 캔버스에서 나간 사용자 수 등 각 구성 요소의 성능/성과를 확인할 수 있습니다. 

{% alert note %}
캔버스 플로우의 경우, 사용자는 사용자 여정의 마지막 단계에서 메시지 페이로드를 입력하고 수신한 후 캔버스를 종료합니다.
{% endalert %}

측정기준에는 노출 횟수, 고유 수신자 수, 전환 수, 발생한 매출도 포함됩니다. 컴포넌트를 클릭하면 데이터를 더 자세히 분석하고 채널별 성능/성과를 확인할 수 있습니다.

캔버스 컴포넌트의 성능/성과에 대한 두 가지 예시입니다. 왼쪽에는 하나의 캔버스 컴포넌트가 있는 사용자 경로에 대한 성능/성과가 자세히 나와 있습니다. 오른쪽에는 확장된 캔버스 컴포넌트에 대한 성능 세부 정보와 인앱 메시지 노출 횟수를 보여주는 중첩 단계가 표시됩니다.]({% image_buster /assets/img_archive/Journey_6.png %})

## 배리언트별 성능/성과 분석

**캔버스 세부 정보** 페이지 하단에서 **배리언트 분석을** 클릭하여 **캔버스 분석** 모달을 엽니다. 이 모달에는 세 개의 탭이 있습니다: 

- 배리언트 분석
- 캔버스 퍼널 보고서
- 캔버스 리텐션 보고서

### 배리언트 분석

**배리언트 분석** 탭에서 배리언트가 두 개 이상인 경우 배리언트 및 대조군별 성능/성과를 분석할 수 있습니다. 캔버스 API 식별자를 복사하고 측정기준의 CSV 파일을 다운로드한 후 셀을 복사할 수도 있습니다. **배리언트 분석** 탭에는 여러 수준에서 각 배리언트의 분석을 보여주는 표가 포함되어 있습니다. 

효과적인 배리언트를 빠르게 추론하고 적절한 케이던스, 콘텐츠, 트리거, 타이밍 등을 식별할 수 있습니다.

\![]({% image_buster /assets/img_archive/analyze_variants.png %})

기본 측정기준은 다음과 같습니다:  

- **배리언트 API 식별자:** API 호출에 사용할 수 있는 배리언트의 API 식별자입니다.
- **총 항목 수:** 캔버스 배리언트를 입력한 총 사용자 수입니다.
- **총 전송 수:** 캔버스 배리언트에서 보낸 총 메시지 수입니다.
- **총 걸음 수:** 캔버스 배리언트의 총 단계 수입니다.
- **총 매출:** 설정된 기본 전환 기간 내에 캔버스 수신자로부터 발생한 달러 단위의 총 매출입니다.

{% alert note %}
전환과 마찬가지로 매출은 기술적으로 캔버스 수준에서 추적되지만, 사용자가 메시지를 수신한(또는 아직 메시지를 수신하지 않은 경우 입력한) 가장 최근의 구성 요소와 가장 최근의 배리언트에 속성이 부여됩니다.<br><br>
예를 들어, 사용자가 두 단계를 완료한 후 구매를 하면 해당 매출은 두 번째 구성 요소와 사용자가 입력한 배리언트에 기여도가 부여됩니다. 고객이 캔버스에 들어왔지만 첫 번째 캔버스 컴포넌트를 받기 전에 구매를 한 경우, 해당 매출은 어떤 컴포넌트에도 속성이 부여되지 않고 입력한 배리언트에만 속성이 부여됩니다.
{% endalert %}

그 외에도 다음과 같은 [전환 이벤트에]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/conversion_events/) 대한 보다 명확한 분석을 확인할 수 있습니다:

- 각 전환 이벤트의 전환 총계 및 전환율
- 제어 배리언트에 대한 업리프트
- 각 전환 이벤트에 대한 통계적 신뢰도

### 전환 추적 방법 

사용자는 캔버스 항목당 전환 이벤트당 한 번만 전환할 수 있습니다. 전환은 해당 항목에 대해 사용자가 가장 최근에 수신한 메시징에 할당됩니다. 캔버스 요약에는 해당 경로에서 사용자가 수행한 모든 전환 성능/성과와 메시지 수신 여부가 반영됩니다. 이후의 각 단계는 사용자가 가장 최근에 받은 단계인 동안에 발생한 전환만 표시합니다. 

캔버스에 푸시 알림이 10개 있고 전환 이벤트가 "앱 열기"(또는 "세션 시작")인 경우를 예로 들어 보겠습니다.
- 사용자 A는 입장 후 첫 번째 메시지를 받기 전에 앱을 엽니다.
- 사용자 B는 푸시 알림을 받을 때마다 앱을 엽니다.

캔버스 요약에는 두 개의 전환이 표시되고 개별 단계에는 첫 번째 단계에서는 하나의 전환이 표시되고 이후 모든 단계에서는 전환이 표시되지 않습니다. 전환 이벤트가 발생할 때 방해금지 시간이 활성화되어 있는 경우에도 동일한 규칙이 적용됩니다. 

이제 방해금지 시간이 있는 캔버스가 있고 다음과 같은 이벤트가 발생한다고 가정해 보겠습니다:

1. 사용자 A가 캔버스에 들어갑니다.
2. 첫 번째 단계는 설정된 방해금지 시간 내에서 지연 단계이므로 메시징이 억제됩니다.
3. 사용자 A가 전환 이벤트를 수행합니다.

사용자 A는 전체 캔버스 배리언트에서는 전환된 것으로 계산되지만 단계를 받지 않았으므로 단계는 계산되지 않습니다.

마지막 예로, 재적격성이 켜져 있는 캔버스가 있다고 가정해 보겠습니다. 재적격 사용자가 첫 번째 항목과 두 번째 항목에서 전환 이벤트를 수행하면 두 번의 전환이 계산됩니다.

### 퍼널 보고서

퍼널 보고서는 캔버스를 받은 후 고객이 어떤 여정을 거치는지 분석할 수 있는 시각적 보고서를 제공합니다. 캔버스에서 대조군 또는 여러 배리언트를 사용하는 경우, 다양한 배리언트가 전환 퍼널에 미치는 영향을 보다 세분화된 수준에서 파악하고 이 데이터를 기반으로 최적화할 수 있습니다. 퍼널 보고서에 대한 자세한 내용은 [퍼널 보고서를]({{site.baseurl}}/user_guide/analytics/reporting/funnel_reports/) 참조하세요.

### 리텐션 보고서

사용자 리텐션은 모든 마케터에게 가장 중요한 측정기준 중 하나입니다. 참여도가 높은 사용자가 계속 재방문한다는 것은 비즈니스가 건강하다는 것을 의미합니다. 이제 Braze를 사용하면 **캔버스 분석** 페이지에서 바로 사용자 리텐션을 측정할 수 있습니다. 리텐션 보고서를 읽고 해석하는 방법에 대한 자세한 내용은 [리텐션 보고서에서]({{site.baseurl}}/user_guide/analytics/reporting/retention_reports/) 확인하세요.
