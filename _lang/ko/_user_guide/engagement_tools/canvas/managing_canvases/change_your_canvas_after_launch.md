# 런칭 후 캔버스를 편집하세요

> 이 참조 기사에서는 초기 출시 후 캔버스에서 변경할 수 있는 내용을 다룹니다.

런칭 후 캔버스를 편집할 수 있습니다:

* 사용자 여정에 새 캔버스 단계를 삽입
* 새 변형 및 연결 추가
* 배리언트 분포 조정
* 모든 캔버스 단계를 중지하거나 재개

{% alert note %}
배리언트 분포는 출시 후에만 감소할 수 있습니다.
{% endalert %}

사용자 여정에서 다음 중 어느 것이든 삭제할 수 있습니다.

- [Canvas steps]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/about/)
- 캔버스 변형 
- 캔버스 단계 간의 연결

캔버스 사용자 여정에 더 많은 단계를 편집하거나 추가하려면 다음 세부정보가 적용됩니다:

- 캔버스에 아직 들어가지 않은 사용자는 새로 생성된 모든 단계에 참여할 수 있습니다. 
- 캔버스 항목 설정이 사용자가 단계를 다시 입력할 수 있도록 허용하는 경우, 새로 생성된 단계를 이미 통과한 사용자는 다시 입력할 수 있습니다.
- 현재 시작된 캔버스에 있는 사용자 중에서 사용자 여정의 새로운 단계가 추가되는 지점에 도달하지 않은 사용자는 새로 추가된 단계를 받을 수 있습니다. 

[지연]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/delay_step/) 또는 [작업 경로]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/action_paths/) 단계를 삭제하면 현재 단계에서 대기 중인 사용자를 다른 캔버스 단계로 선택적으로 리디렉션할 수 있습니다. 지연의 경우, 사용자는 지연 기간이 끝날 때까지 해당 단계에 남아 있습니다. 액션 경로의 경우, 사용자는 평가 기간이 끝날 때까지 해당 단계에 남아 있습니다.

캔버스를 처음 시작할 때 Braze는 사용자를 캔버스의 모든 후속 메시지가 아닌 현재 메시지 단계에 대기시킵니다. 런칭 후 캔버스를 편집하면 일부 사용자는 이미 대기열에 있을 수 있으며 변경 사항을 반영하지 않을 수 있습니다. 캔버스를 중지하고 복제한 다음 변경하여 이 새로운 버전을 런칭하면, 캔버스는 이미 대기열에 있지 않은 사용자뿐만 아니라 모든 사용자를 다시 평가합니다.

특정 편집 사용 사례에 대해서는 [모범 사례](#best-practices) 섹션을 참조하세요. 일반적으로, 예기치 않은 동작이 발생할 수 있으므로 라이브 캔버스를 편집하지 않는 것이 가장 좋습니다.

{% details Expand for original Canvas editor details %}

Keep in mind the following permissible post-launch Canvas edits, depending on which workflow your Canvas was created with. 캔버스가 원래 캔버스 워크플로를 사용하는 경우, 런칭 후 편집을 수행하려면 먼저 캔버스 흐름으로 복제해야 합니다.

You can't edit or delete existing connections, and you can't insert a step between existing connected steps. 캔버스 사용자 여정에 더 많은 단계를 편집하거나 추가하려면 다음 세부정보가 적용됩니다:

- 캔버스에 아직 들어가지 않은 사용자는 새로 생성된 모든 단계에 참여할 수 있습니다. 
- 캔버스 항목 설정이 사용자가 단계를 다시 입력할 수 있도록 허용하는 경우, 새로 생성된 단계를 이미 통과한 사용자는 다시 입력할 수 있습니다.
- 현재 시작된 캔버스에 있는 사용자 중 사용자 여정의 새로 추가된 단계에 도달하지 않은 사용자는 새로 추가된 단계를 받을 수 있습니다.
- 캔버스에서 지연 단계가 마지막 단계인 경우, 해당 단계에 도달한 사용자는 자동으로 캔버스를 벗어나게 되며 새로 생성된 단계를 받지 않습니다.

{% alert important %}
캔버스 단계의 **지연** 또는 **창** 설정을 업데이트하면, 업데이트 시점에 해당 단계에 있는 사용자는 원래 들어갔을 때 할당된 지연 시간에 따릅니다. 캔버스에 들어오는 새로운 사용자와 아직 해당 단계에 대기열에 있지 않은 사용자만 업데이트된 시간에 메시지를 받습니다.
{% endalert %}

캔버스를 중지해도 메시지를 받기 위해 대기 중인 사용자는 종료되지 않습니다. 캔버스를 다시 활성화하고 사용자가 여전히 메시지를 기다리고 있다면, 그들은 메시지를 받습니다(단, 메시지를 받아야 할 시간이 지나면 받지 않습니다).

{% enddetails %}

## 캔버스 세부사항

캔버스를 런칭한 후 다음 설정 및 세부정보를 편집할 수 있습니다:

* 캔버스 이름 및 설명
* Teams 및 태그
* 입력 유형, 일정 및 제어
* 구독 상태
* 사용량 제한
* 최대 게재빈도 설정
* 방해금지 시간
* 타겟 오디언스

캔버스가 런칭된 후:

- 전환 이벤트는 편집할 수 없습니다. 
- 다음 단계는 추가하거나 제거할 수 없으며, 순위를 조정하기 위해 재정렬할 수 없습니다. [오디언스 경로]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/audience_paths/), [행동 경로]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/action_paths/), and [실험 경로]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/experiment_step/).
  - **우회 방법 1:** 새로운 오디언스 경로, 행동 경로 또는 실험 경로를 생성하고 해당 새로운 단계로 경로를 재구성하십시오.
  - **우회 2:** 캔버스를 복제하여 수정하세요.

### 개별 단계

개별 캔버스 단계의 경우 시작 후 다음 세부 정보를 편집할 수 있습니다.

* 이름
* 메시지 내용
* 트리거
* 오디언스
* 예외 이벤트
* 지연(지연 단계에만 해당)

그러나 단계의 일정 유형 및 제어 비율은 시작 후 편집할 수 없습니다. 액션 경로 및 오디언스 경로 단계의 경우, 순위 및 평가 창은 런칭 후 편집할 수 없습니다.

### 캔버스 배리언트 백분율

캔버스를 시작한 후에는 제어 배리언트 비율만 줄일 수 있습니다. 캔버스에서 배리언트 비율이 수정되면 사용자가 다른 배리언트로 재분배될 수 있습니다.

처음에, 이 사용자들은 특정 배리언트에 무작위로 할당된 후 처음으로 캠페인을 받습니다. 그 이후로 캠페인이 수신될 때마다(또는 사용자가 캔버스 변형에 다시 들어갈 때마다), 변형 비율이 수정되지 않는 한 동일한 변형을 받습니다.

배리언트 비율이 변경되면 사용자가 다른 배리언트로 재분배될 수 있습니다. 사용자는 백분율이 다시 수정될 때까지 이러한 변형을 유지합니다. 캔버스를 사용하는 경우 `NOT` 필터와 무작위 버킷 번호가 있는 분기를 사용하면 사용자가 캔버스를 다시 입력할 때마다 사용자 여정에서 동일한 브랜치를 받지 못할 수 있습니다.

#### 대조군

대조군은 배리언트 비율이 변하지 않으면 일관성을 유지합니다. 대조군의 비율이 감소하거나 증가하면 이전에 메시지를 받은 사용자는 나중에 대조군에 들어갈 수 없으며, 대조군의 어떤 사용자도 메시지를 받을 수 없습니다.

### 로컬 전송 시간

지역 발송 시간에 런칭될 예정인 캔버스는 예정된 발송 시간 24시간 전까지 편집할 수 있습니다. 이 창은 "안전 구역"이라고 합니다. 

{% alert tip %}
새로운 캔버스를 완전히 만들기 위해 더 큰 편집을 할 의도가 있다면, 첫 번째 캔버스를 받은 사용자를 제외하고 시간대 전송을 허용하기 위해 캔버스 일정 시간을 다시 조정해야 합니다.
{% endalert %}

사용자에게 즉시 진입하도록 설정된 항목 일정이 있을 때, 캔버스는 5분 단위로 가장 가까운 시간에 시작됩니다. 예를 들어, 캔버스를 8:31 AM PST에 사용자에게 즉시 진입하도록 업데이트하면, 시작 시간은 8:30 AM PST로 설정되며 회사의 시간대에 맞춰집니다.

### 변형 삭제

캔서스에서 변형이 삭제되면 다음과 같은 일이 발생합니다:

- 변형 내의 단계(다른 변형과 공유되는 단계 포함)가 삭제됩니다. 
- 캔버스의 단계 분석 및 최상위 분석, 예를 들어 _총 진입_, _총 퇴장_, _전환율_이 삭제됩니다.
- 삭제된 변형의 사용자들은 단계에서 퇴장하며, 이후의 메시지는 전송되지 않습니다.

### 캔버스 항목 속성

캔버스 항목 속성은 전송될 때 단계로 템플릿화되지 않습니다. 이는 캔버스가 시작된 후 캔버스 진입 속성이 편집되면 이러한 변경 사항이 캔버스에 진입하는 새로운 사용자에게만 적용됨을 의미합니다. 캔버스가 사용자에게 재진입을 허용하는 경우, 재진입하는 사용자는 업데이트된 캔버스 진입 속성에 의해 결정됩니다.

## Best practices

출시된 후 캔버스를 편집하거나 추가할 때 염두에 두어야 할 모범 사례를 확인하세요.

{% alert important %}
일반적으로 캔버스가 활성화되고 사용자를 대기열에 추가하는 동안 변경을 피하는 것이 좋습니다.
{% endalert %}

### 연결이 끊긴 단계

연결이 끊긴 단계로 캔버스를 시작할 수 있으며 시작 후 이러한 캔버스를 저장할 수도 있습니다. 단계를 워크플로에서 분리하기 전에 사용자가 보류 중인 단계를 분석 보기에서 확인하는 것이 좋습니다.

사용자가 캔버스 워크플로의 연결이 끊긴 단계에 있다고 가정해 봅시다. 이 사용자는 다음 단계로 진행합니다(있는 경우). 단계의 설정은 사용자가 어떻게 진행해야 하는지를 결정합니다. 

연결이 끊긴 단계를 생성하거나 편집하여 이러한 독립 단계를 직접 캔버스의 나머지 부분에 연결하지 않고도 변경할 수 있습니다. 이는 캔버스를 다시 시작하기 전에 단계를 테스트하는 데 도움이 됩니다. 

### 실험 경로 단계

캔버스에 활성 또는 진행 중인 승리 경로 또는 개인화된 경로 실험이 있고 활성 캔버스를 업데이트하면(실험 경로 단계를 업데이트하든 상관없이), 진행 중인 실험이 종료되고 실험 경로 단계는 승리 경로 또는 개인화된 경로를 결정하지 않습니다. To restart the experiment, you can disconnect the existing Experiment Path and launch a new one, or duplicate the Canvas and launch a new Canvas. 그렇지 않으면 사용자는 최적화 방법이 선택되지 않은 것처럼 실험 경로를 통해 흐릅니다.

### 시간 지연

시간 지연이 있는 캔버스를 편집하는 것은 다소 까다로울 수 있으므로, 캔버스를 편집할 때 다음 세부 사항을 염두에 두십시오:

- 지연 단계에서 지연을 업데이트하면, 캔버스에 진입하는 새로운 사용자와 해당 단계에 대기열에 추가되지 않은 사용자만 업데이트된 시간 지연에 따라 메시지를 받습니다.
- 시간 지연이 있는 단계를 삭제하고(예: 지연 또는 작업 경로) 해당 사용자를 다른 캔버스 단계로 리디렉션하기로 결정하면, 사용자는 단계의 시간 지연이 완료된 후에만 리디렉션됩니다. 예를 들어, 하루 지연이 있는 지연 단계를 삭제하고 해당 사용자를 메시지 단계로 리디렉션한다고 가정해 보겠습니다. 이 경우, 사용자는 하루 지연이 완료된 후에만 리디렉션됩니다.
- 캔버스에 하나 이상의 실험 경로 단계가 있는 경우, 단계를 삭제하면 이 단계의 결과가 무효화될 수 있습니다.

### 캔버스 중지

캔버스를 중지해도 단계에서 대기 중인 사용자는 종료되지 않습니다. 캔버스를 다시 활성화하고 사용자가 여전히 대기 중인 경우, 그들은 단계를 완료하고 다음 단계로 이동합니다. 그러나 사용자가 다음 단계로 진행해야 할 시간이 지나면, 그들은 대신 캔버스를 종료합니다. 

예를 들어, 오후 2시에 시작하도록 설정된 캔버스 흐름을 사용하여 생성된 캔버스가 있고, 두 단계가 있는 하나의 변형이 있다고 가정해 보겠습니다: 메시지 단계로 들어가는 1시간 지연이 있는 지연 단계입니다. 

사용자가 오후 2시 1분에 이 캔버스에 들어가고 동시에 지연 단계에 들어갑니다. 이는 사용자가 오후 3시 1분에 사용자 여정의 다음 단계(메시지 단계)로 이동하도록 예정되어 있음을 의미합니다. 오후 2시 30분에 캔버스를 중지하고 오후 3시 30분에 캔버스를 다시 활성화하면, 사용자는 오후 3시 1분 이후이므로 캔버스를 종료합니다. 그러나 오후 2시 40분에 캔버스를 다시 활성화하면, 사용자는 오후 3시 1분에 예상대로 메시지 단계로 이동합니다.

## 알아두어야 할 사항

다음의 일반적인 문제는 캔버스를 시작한 후 다른 구성 요소에 더 많은 구성 요소를 편집하거나 추가함으로써 발생할 수 있습니다. 

{% alert important %}
다음 문제는 피할 수 있습니다. 캔버스를 시작한 후 편집이 필요하다면, 이미 캔버스에 들어간 모든 사용자가 사용자 여정을 완료했는지 먼저 확인하는 것이 좋습니다. 또한, 최소한 한 명의 사용자가 이미 처리한 단계를 삭제하지 않는 것이 좋습니다.
{% endalert %}

- 보고 데이터 누락(메시지 변형이 삭제되고 다시 추가될 때)
- 사용자가 예상 경로를 따르지 않음
- 메시지가 예상치 못한 시간에 전송됨
- 편집이 커런츠 데이터를 덮어쓰지 않으므로, 삭제로 인해 캔버스에 존재하지 않는 단계(예: `canvas_step_ids`) 간의 불일치를 발견할 수 있습니다.
- 사용자가 동일한 메시지를 두 번 받을 수 있음
- 기존 속도 제한으로 인해 사용자가 메시지를 받지 못함
  - 활성 캔버스에서 속도 제한을 업데이트하면, 새로운 속도 제한은 캔버스에 이미 있는 사용자 포함하여 모든 향후 메시지 전송에 적용됩니다. 그러나 내부 캐싱(최대 30초)으로 인해 새로운 속도 제한이 완전히 적용되기 전에 잠시 지연이 있을 수 있습니다. 브레이즈는 사용자가 현재 있는 메시지 단계에 대해 큐에 추가하므로, 각 단계의 메시지가 실제로 전송될 때 적용되는 속도 제한이 적용됩니다.
- 캔버스가 [자동으로 중지]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/about_statuses/#available-statuses)되면, 캔버스의 출시 후 초안도 삭제됩니다.
