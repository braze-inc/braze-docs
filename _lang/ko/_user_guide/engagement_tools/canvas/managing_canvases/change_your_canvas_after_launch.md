# 출시 후 캔버스 편집하기

> 이 참조 문서에서는 초기 실행 후 캔버스에서 변경할 수 있는 사항에 대해 설명합니다.

실행 후 캔버스를 편집할 수 있습니다:

* 사용자 여정에 새로운 캔버스 단계 삽입하기
* 새로운 배리언트 및 연결 추가하기
* 배리언트 분포 조정하기
* 모든 캔버스 단계 중지 또는 재개하기

{% alert note %}
컨트롤 배리언트 배포는 출시 이후에만 감소할 수 있습니다.
{% endalert %}

캔버스를 만든 워크플로에 따라 다음과 같이 출시 후 허용되는 캔버스 편집을 염두에 두세요. 캔버스가 원래 캔버스 워크플로를 사용하는 경우, 실행 후 편집을 수행하려면 먼저 캔버스 플로우에 복제해야 합니다.

사용자 여정에서 다음 중 하나를 삭제할 수 있습니다:

- [캔버스 단계]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/about/)
- 캔버스 배리언트 
- 캔버스 단계 간 연결

캔버스 사용자 여정에 단계를 편집하거나 추가하려는 경우 다음 세부 사항이 적용됩니다:

- 아직 캔버스에 들어가지 않은 사용자는 새로 생성된 단계를 이용할 수 있습니다. 
- 캔버스 항목 설정에서 사용자가 단계를 다시 입력할 수 있도록 허용하는 경우 이미 새로 만든 단계를 통과한 사용자도 다시 입력할 수 있습니다.
- 현재 시작된 캔버스에 있지만 사용자 여정에서 새로운 단계가 추가되는 지점에 도달하지 않은 사용자는 새로 추가된 단계를 받을 수 있습니다. 

[지연]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/delay_step/) 또는 [행동 경로]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/action_paths/) 단계를 삭제하면 선택적으로 해당 단계에서 현재 대기 중인 사용자를 다른 캔버스 단계로 리디렉션할 수 있습니다. 지연의 경우 사용자는 지연 기간이 끝날 때까지 해당 단계에 머물게 됩니다. 행동 경로의 경우 사용자는 평가 기간이 끝날 때까지 해당 단계에 머물게 됩니다.

처음에 캔버스를 실행하면 Braze는 사용자가 현재 있는 메시지 단계의 메시지를 대기열에 넣지, 캔버스의 모든 후속 메시지를 대기열에 넣지 않는다는 점에 유의하세요. 실행 후 캔버스를 편집하면 일부 사용자는 이미 대기열에 대기 중이므로 변경 내용이 적용되지 않습니다. 캔버스를 중지하고 복제한 다음 변경하고 이 새 버전을 실행하면 캔버스는 아직 대기열에 추가되지 않은 사용자뿐만 아니라 모든 사용자를 다시 평가합니다.

구체적인 편집 사용 사례는 [모범 사례](#best-practices) 섹션을 참조하세요. 일반적으로 라이브 캔버스는 예기치 않은 동작이 발생할 수 있으므로 편집하지 않는 것이 좋습니다.

{% details Expand for original Canvas editor details %}

기존 연결은 편집하거나 삭제할 수 없으며, 기존 연결된 단계 사이에 단계를 삽입할 수 없습니다. 캔버스 사용자 여정에 단계를 편집하거나 추가하려는 경우 다음 세부 사항이 적용됩니다:

- 아직 캔버스에 들어가지 않은 사용자는 새로 생성된 단계를 이용할 수 있습니다. 
- 캔버스 항목 설정에서 사용자가 단계를 다시 입력할 수 있도록 허용하는 경우 이미 새로 만든 단계를 통과한 사용자도 다시 입력할 수 있습니다.
- 현재 출시된 캔버스에 있지만 사용자 여정에서 새로 추가된 단계에 도달하지 않은 사용자는 새로 추가된 단계를 받을 수 있습니다.
- 지연 단계가 캔버스의 마지막 단계인 경우 해당 단계에 도달한 사용자는 자동으로 캔버스 밖으로 진행되며 새로 생성된 단계가 표시되지 않습니다.

{% alert important %}
캔버스 단계의 **지연** 또는 **윈도우** 설정을 업데이트하면 업데이트 시점에 현재 해당 단계에 있는 사용자는 처음 입력할 때 할당된 지연 시간을 준수합니다. 캔버스에 새로 들어오는 사용자와 아직 해당 단계의 대기줄에 서지 않은 사용자만 업데이트된 시간에 메시지를 받게 됩니다.
{% endalert %}

캔버스를 중지해도 메시지 수신을 대기 중인 사용자는 종료되지 않습니다. 캔버스를 다시 활성화했는데 사용자가 여전히 메시지를 기다리는 경우 메시지를 받게 됩니다(메시지를 보내야 하는 시간이 지나지 않았다면 메시지를 받지 못합니다).

{% enddetails %}

## 캔버스 세부 정보

캔버스가 실행된 후 다음과 같은 캔버스 설정 및 정보를 편집할 수 있습니다:

* 캔버스 이름 및 설명
* Teams 및 태그
* 항목 유형, 일정 및 제어
* 구독 상태
* 요금 제한
* 최대 게재빈도 설정
* 방해금지 시간
* 타겟 오디언스

캔버스가 시작된 후:

- 전환 이벤트는 편집할 수 없습니다. 
- 다음 단계는 추가하거나 제거할 수 없으며 순위를 조정하기 위해 순서를 변경할 수 없습니다: [오디언스 경로]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/audience_paths/), [행동 경로]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/action_paths/) 및 [실험 경로]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/experiment_step/).
  - **해결 방법 1:** 새 오디언스 경로, 행동 경로 또는 실험 경로를 만들고 해당 새 단계의 경로를 재구성합니다.
  - **해결 방법 2:** 캔버스를 복제하여 편집합니다.

### 개별 단계

개별 캔버스 단계의 경우 실행 후 다음 세부 정보를 편집할 수 있습니다:

* 이름
* 메시지 내용
* 트리거
* 오디언스
* 예외 이벤트
* 지연(지연 단계만 해당)

그러나 단계의 스케줄 유형 및 제어 비율은 실행 후에는 편집할 수 없습니다. 행동 경로 및 오디언스 경로 단계의 경우 실행 후에는 순위와 평가 창을 편집할 수 없습니다.

### 캔버스 배리언트 비율

캔버스를 시작한 후에는 제어 배리언트 비율을 줄일 수만 있습니다. 캔버스에서 배리언트 비율을 수정하면 사용자가 다른 배리언트로 재배포될 수 있습니다.

처음에 이러한 사용자는 캠페인을 처음 받기 전에 특정 배리언트를 무작위로 할당받게 됩니다. 이후부터는 캠페인을 수신할 때마다(또는 사용자가 캔버스 배리언트를 다시 입력할 때마다) 배리언트 퍼센티지를 수정하지 않는 한 동일한 배리언트를 수신하게 됩니다.

배리언트 비율이 변경되면 사용자는 다른 배리언트로 재배포될 수 있습니다. 사용자는 백분율이 다시 수정될 때까지 이러한 배리언트를 유지하게 됩니다. 무작위 버킷 번호가 있는 `NOT` 필터로 브랜칭을 사용하는 캔버스의 경우, 사용자가 캔버스에 다시 들어갈 때 사용자 여정에서 매번 동일한 브랜칭을 받지 못할 수 있습니다.

#### 대조군

배리언트 비율이 변경되지 않은 경우 대조군은 일관성을 유지합니다. 대조군의 비율을 줄이거나 늘리면 이전에 메시지를 받은 사용자는 나중에 메시지를 보낼 때 대조군에 들어갈 수 없으며 대조군에 속한 사용자는 메시지를 받지 못합니다.

### 현지 전송 시간

현지 전송 시간에 시작하도록 예약된 캔버스는 예정된 전송 시간 24시간 전까지 편집할 수 있습니다. 이 창을 '안전 영역'이라고 합니다. 

{% alert tip %}
완전히 새로운 캔버스 사본을 만드는 대규모 편집을 하려면 첫 번째 캔버스를 받은 사용자를 제외하고 시간대 전송을 허용하도록 캔버스 예약 시간을 다시 조정해야 한다는 점을 잊지 마세요.
{% endalert %}

### 배리언트 삭제하기

캔버스에서 배리언트를 삭제하면 다음과 같은 일이 발생합니다:

- 배리언트 내의 단계(다른 배리언트에서 공유하는 단계 포함)는 삭제됩니다. 
- 단계 분석과 캔버스의 최상위 분석(예: _총 진입_, _총 이탈_, _전환율_)이 삭제됩니다.
- 삭제된 배리언트의 사용자는 단계에서 종료되며 다음 메시징은 전송되지 않습니다.

### 캔버스 항목 속성

캔버스 항목 속성은 전송 시 단계로 템플릿화되지 않습니다. 즉, 캔버스가 시작된 후 캔버스 항목 속성을 편집하면 이러한 변경 사항은 캔버스에 새로 들어온 사용자에게만 적용됩니다. 캔버스에서 사용자가 캔버스에 다시 입장할 수 있도록 허용하는 경우, 다시 입장하는 모든 사용자는 업데이트된 캔버스 항목 속성에 따라 결정됩니다.

## 모범 사례

캔버스를 시작한 후 편집하거나 캔버스에 추가할 때 염두에 두어야 할 모범 사례를 확인하세요.

{% alert important %}
일반적으로 캔버스가 활성화되어 사용자를 대기열에 추가하는 동안에는 변경하지 마세요.
{% endalert %}

### 연결이 끊어진 단계

연결이 끊어진 단계로 캔버스를 실행하고 실행 후 이러한 캔버스를 저장할 수도 있습니다. 워크플로에서 단계의 연결을 끊기 전에 보류 중인 사용자의 단계에 대한 분석 보기를 확인하는 것이 좋습니다.

사용자가 캔버스 워크플로우의 연결이 끊어진 단계에 있다고 가정해 보겠습니다. 이 사용자는 다음 단계가 있는 경우 다음 단계로 진행됩니다. 단계의 설정에 따라 사용자가 진행해야 하는 방식이 결정됩니다. 

연결이 끊어진 단계를 만들거나 편집하면 나머지 캔버스에 직접 연결하지 않고도 이러한 독립 단계를 변경할 수 있습니다. 이렇게 하면 캔버스를 다시 시작하기 전에 단계를 테스트하는 데 도움이 됩니다. 

### 실험 경로 단계

캔버스에 활성 상태이거나 진행 중인 우승 경로 또는 개인화된 경로 실험이 있는 경우 활성 캔버스를 업데이트하면 실험 경로 단계 자체를 업데이트하더라도 진행 중인 실험이 종료되고 실험 단계에서는 우승 경로 또는 개인화된 경로가 결정되지 않습니다. 실험을 다시 시작하려면 기존 실험 경로를 연결 해제하고 새 실험 경로를 시작하거나 캔버스를 복제하고 새 캔버스를 시작하면 됩니다. 그렇지 않으면 사용자는 최적화 방법을 선택하지 않은 것처럼 실험 경로를 통과하게 됩니다.

### 시간 지연

시간 지연이 있는 캔버스를 편집하는 것은 약간 까다로울 수 있습니다! 따라서 캔버스를 편집할 때 다음 세부 사항을 염두에 두세요.

지연 단계에서 지연을 업데이트하면 캔버스에 새로 들어오는 사용자와 해당 단계에 대기줄이 없는 사용자만 업데이트된 시간 지연으로 메시지를 받게 됩니다.

시간 지연이 있는 단계(예: 지연 또는 행동 경로)를 삭제하고 해당 사용자를 다른 캔버스 단계로 리디렉션하기로 결정하면 해당 단계의 시간 지연이 완료된 후에만 사용자가 리디렉션됩니다. 예를 들어 하루 지연이 있는 지연 단계를 삭제하고 해당 사용자를 메시지 단계로 리디렉션한다고 가정해 보겠습니다. 이 경우 사용자는 하루 지연이 완료된 후에만 리디렉션됩니다.

캔버스에 하나 이상의 실험 경로 단계가 있는 경우 단계를 삭제하면 이 단계의 결과가 무효화될 수 있습니다.

### 캔버스 중지하기

캔버스를 중지해도 단계에서 대기 중인 사용자는 종료되지 않습니다. 캔버스를 다시 활성화했는데 사용자가 여전히 대기 중인 경우 해당 단계를 완료하고 다음 단계로 이동합니다. 그러나 사용자가 다음 단계로 진행했어야 하는 시간이 지났다면 대신 캔버스를 종료합니다. 

예를 들어 캔버스 플로우 워크플로를 사용하여 만든 캔버스가 오후 2시에 시작되도록 설정되어 있고 두 단계로 구성된 하나의 배리언트, 즉 1시간 지연이 있는 지연 단계가 메시지 단계로 들어간다고 가정해 보겠습니다. 

사용자가 오후 2시 1분에 이 캔버스에 들어가면서 동시에 지연 단계로 들어갑니다. 즉, 사용자는 오후 3시 1분에 사용자 여정의 다음 단계(메시지 단계)로 이동하도록 예약됩니다. 오후 2시 30분에 캔버스를 중지했다가 오후 3시 30분에 캔버스를 다시 인에이블하면 사용자는 오후 3시 1분 이후이므로 캔버스를 종료하게 됩니다. 그러나 오후 2시 40분에 캔버스를 다시 활성화하면 사용자는 오후 3시 1분에 예상대로 메시지 단계로 이동합니다.

## 알아두어야 할 사항

다음과 같은 일반적인 문제는 실행 후 캔버스에 있는 다른 컴포넌트를 편집하거나 추가하면 트리거될 수 있습니다. 

{% alert important %}
다음과 같은 문제는 피할 수 있습니다. 캔버스를 시작한 후 캔버스를 편집해야 하는 경우 먼저 캔버스에 이미 입장한 모든 사용자가 사용자 여정을 완료했는지 확인하는 것이 좋습니다. 또한 이미 한 명 이상의 사용자를 처리한 단계는 삭제하지 않는 것이 좋습니다.
{% endalert %}

- 누락된 보고 데이터(메시지 배리언트가 삭제되었다가 다시 추가된 경우)
- 사용자가 예상 경로를 따르지 않음
- 예상치 못한 시간에 메시징이 전송되는 경우
- 편집 내용은 커런츠 데이터를 덮어쓰지 않으므로 캔버스 단계 간에 불일치가 있을 수 있습니다(예: `canvas_step_ids` 삭제로 인해 캔버스에 존재하지 않는 경우).
- 사용자는 동일한 메시지를 두 번 수신할 수 있습니다.
- 기존 속도 제한으로 인해 메시지 수신이 중단된 사용자
  - 사용자가 캔버스에 파견되면 사용자가 파견될 때 해당 캔버스에 적용된 요금 제한이 사용자에게 적용됩니다. 캔버스가 전송된 후에는 해당 사용자에 대한 요금 제한을 편집할 수 없으므로 전송 후 요금 제한을 늘리거나 줄여도 이미 전송된 사용자에게는 영향을 미치지 않습니다.
