# 캔버스를 캔버스 흐름에 복제하기

{% alert important %}
더 이상 기존 캔버스 환경을 사용하여 캔버스를 만들거나 복제할 수 없습니다. 기존 캔버스 경험을 사용 중인 고객은 현재 캔버스 경험인 캔버스 플로우로 전환할 것을 권장합니다.
{% endalert %}

> 원본 편집기에서 기존 캔버스가 있는 경우 이 캔버스를 복제하여 캔버스 플로우에서 사본을 만들 수 있습니다. 현재 캔버스 워크플로로 전환하면 가벼운 [캔버스 구성 요소]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/about/), [지속성 항목 속성정보]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/canvas_entry_properties_event_properties/canvas_persistent_entry_properties/), [실행 후 편집]({{site.baseurl}}/post-launch_edits) 기능에 액세스할 수 있습니다. 원본 캔버스는 변경되거나 삭제되지 않습니다.

캔버스를 복제하려면 다음과 같이 하세요:

1. 캔버스 대시보드로 이동합니다. 
2. 캔버스 흐름 워크플로에서 사본을 만들려는 캔버스를 식별합니다. **초안**, **활성** 또는 **중지** 상태의 캔버스를 복제할 수 있습니다. 
3. <i class="fas fa-ellipsis-vertical"></i> **추가 작업을** 클릭하고 **캔버스 흐름에 복제를** 선택합니다.

\![]({% image_buster /assets/img_archive/clone_to_v2_workflow.png %}){: style="max-width:25%;"}

{: start="4"}
4\. 새 캔버스의 이름을 입력하고 **캔버스 흐름에 복제를** 클릭합니다. 

\![]({% image_buster /assets/img_archive/clone_to_v2_modal.png %}){: style="max-width:70%;"}

이제 캔버스에는 원래 캔버스와 캔버스 플로우 버전의 두 가지 버전이 있습니다. 원본 캔버스는 여전히 원본 상태를 유지하고 복제된 캔버스는 **초안** 상태를 유지합니다. 원래 캔버스에 계속 액세스할 수 있지만, Braze는 캔버스 플로우 워크플로우를 사용하여 캔버스를 계속 구축할 것을 권장합니다.

이전에는 분기가 있는 일부 캔버스는 복제할 수 없었습니다. 이제 분기로 캔버스를 복제할 수 있습니다. 분기가 있는 캔버스를 복제하면 단계가 연결이 끊어질 수 있다는 점에 유의하세요. 이러한 단절된 단계(이전 단계가 연결되어 있지 않은 단계)를 해결하여 캔버스 여정이 제대로 매핑되었는지 확인하세요.

{% alert note %}
활성 캔버스를 복제하면 Braze는 원본 캔버스를 통해 사용자를 계속 전송합니다. 두 캔버스에서 사용자에게 중복 메시지를 보내지 않도록 복제하기 전에 캔버스를 중지하는 것이 좋습니다.
{% endalert %}

두 개의 캔버스가 나열된 캔버스 대시보드: 캔버스 V1 및 캔버스 V1의 V2 사본. 캔버스 V1의 V2 사본에는 캔버스 플로 워크플로우를 사용하고 있음을 나타내는 아이콘이 있습니다.]({% image_buster /assets/img_archive/clone_to_v2_dashboard.png %})

캔버스 플로우 워크플로에 캔버스 복제를 완료했습니다. 이제 업데이트된 환경에서 캔버스를 계속 구축할 수 있습니다!

## 권장 사항

원본 캔버스를 캔버스 플로우에 복제한 후 기존 사용자가 사용자 여정을 계속할 수 있도록 하려면 기존 캔버스에 필터를 추가하여 새 사용자가 새 캔버스에 들어가지 못하도록 할 수 있습니다.

재자격이 꺼져 있으면 "입력된 캔버스 변형" 필터를 추가합니다. 재인증이 켜져 있는 경우 사용자가 같은 캔버스에 두 번 입력하지 않도록 하기 위해 고려할 수 있는 방법은 다음과 같습니다:
- 고유 태그를 포함하도록 기존 캔버스를 업데이트합니다. 새 캔버스의 경우 "태그가 있는 캠페인 또는 캔버스에서 마지막으로 받은 메시지" 필터를 추가합니다. 이렇게 하면 특정 입력 날짜(원래 캔버스에서 마지막으로 메시지를 보낸 날짜와 전환 기간을 더한 총 일수) 이후에는 사용자가 캔버스에 두 번 입력할 수 없게 됩니다. 
- **다음 방법은 데이터 포인트를 기록합니다.** 입력 시 커스텀 속성 날짜 타임스탬프를 트리거하는 Braze 간 웹훅을 포함하도록 원본 캔버스를 업데이트합니다. 이 속성을 사용하여 지정된 날짜(원래 캔버스에서 마지막 메시지를 보낸 후 전환 기간까지 더한 총 일수) 이후에는 사용자가 새 캔버스에 들어가지 못하도록 할 수 있습니다.

API 트리거 캔버스의 경우 엔지니어링 팀과 협력하여 새 캔버스가 출시될 준비가 되면 해당 캔버스가 새 캔버스 ID를 사용하도록 하세요.

기존 캔버스 편집기와 캔버스 플로우 환경의 차이점에 대한 자세한 내용은 [캔버스 FAQ를]({{site.baseurl}}/user_guide/engagement_tools/canvas/faqs/#what-are-the-main-differences-between-canvas-flow-and-the-original-canvas-editor) 참조하세요.
