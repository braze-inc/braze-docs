# 캔버스 버전 기록

> 버전 기록을 통해 캔버스의 이전 버전에 대한 캔버스 분석 및 사용자 여정을 확인하고 액세스할 수 있습니다. 

캔버스 버전 기록을 참조하면 캔버스의 진화 과정을 기록하는 데 특히 유용할 수 있습니다. 예를 들어, 대규모 변경을 하는 경우 이전 캔버스 버전을 참조하여 워크플로우가 어떻게 진행되었는지 더 잘 이해할 수 있습니다.

## 버전 관리

\![]({% image_buster /assets/img_archive/canvas_version_history.png %}){: style="float:right;max-width:35%;margin-left:15px;"}

새 버전을 만들려면 **캔버스 업데이트를** 클릭합니다. 이렇게 하면 캔버스의 이전 설정을 덮어쓰지 않고 변경할 수 있습니다. 새 캔버스 버전이 생성되면 이미 캔버스에 있는 사용자는 새 버전의 워크플로를 진행하게 됩니다. 캔버스에 들어가는 사용자도 새 버전으로 들어갑니다. 

버전 기록에 액세스하려면 캔버스 상단의 캔버스 세부 정보로 이동하여 **\#버전을** 선택합니다. 여기에서 **버전 기록** 사이드바에 액세스할 수 있습니다. 사이드바에서 캔버스 버전을 선택하여 캔버스 세부 정보를 보고 비교합니다. 캔버스 분석과 캔버스 설정 사이를 토글하려면 하단 툴바에서 **분석 보기** 또는 **캔버스 보기를** 클릭합니다.

{% alert note %}
**버전 기록** 아래에 나열된 **캔버스는** 보기 전용입니다.
{% endalert %}

버전이 활성화되어 있는 동안 변경된 사항 목록을 보려면 버전 **변경내용** 사이드바에서 **변경내용 보기를** 선택합니다. 캔버스 체인지로그에서 버전과 관련된 모든 변경 사항을 볼 수도 있습니다. 

캔버스를 실행하고 두 번째 버전을 만드는 사이에 편집을 하지 않은 경우 캔버스의 첫 번째 버전에 대한 **변경 사항 보기에** 변경 사항이 표시되지 않습니다.

버전 기록 수가 증가함에 따라 사이드바에서 각 버전의 이름을 변경하여 정리할 수도 있습니다. 기본값으로 버전 이름은 이전에 생성된 버전 수에 따라 숫자로 생성됩니다. 더 이상 활성화되지 않은 상태에서 버전 이름을 변경하면 캔버스 체인지로그에는 표시되지만 버전 변경내용 기록 보기 내에서는 해당 버전의 변경내용 로그가 표시되지 않습니다.

두 개의 새로운 캔버스 버전이 생성되었음을 보여주는 캔버스 체인지로그 예제입니다.]({% image_buster /assets/img_archive/canvas_version_history_changelog.png %}){: style="max-width:85%" }

### 버전 삭제

캔버스당 최대 10개의 버전을 만들 수 있습니다. 이 제한에 도달하면 버전을 삭제하여 새 버전을 위한 공간을 확보할 수 있습니다. 버전은 캔버스를 업데이트할 때가 아니라 **폐기** 버튼을 클릭할 때 삭제됩니다. 버전을 삭제하면 특정 버전의 체인지로그가 아니라 전체 캔버스 체인지로그에 반영됩니다.

버전을 삭제하면 캔버스 설정은 즉시 손실되지만 삭제된 버전과 관련된 분석은 유지됩니다. 

## 분석 보기

버전 기록 내에서 캔버스 수준 및 단계 수준 분석을 볼 수 있습니다. 캔버스 버전 보기에서 데이터는 해당 버전의 날짜 범위뿐만 아니라 전체 날짜 범위에 대해 채워집니다. 그러나 단계 수준에서는 해당 버전이 활성화되어 있는 동안에 존재했던 단계에 대해서만 분석 결과가 표시됩니다. 이러한 분석은 회사의 표준 시간대에 해당하는 캘린더 날짜를 사용하여 채워지므로 버전이 생성된 정확한 시간대에 대한 분석은 제공되지 않습니다.
