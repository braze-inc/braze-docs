# 캔버스의 콘텐츠 카드

> 콘텐츠 카드는 캔버스 여정의 일부로 고객에게 보낼 수 있습니다. 이 문서에서는 캔버스 내에서 콘텐츠 카드를 메시징 채널로 사용하는 것과 관련된 기능과 뉘앙스에 대해 설명합니다.

다른 캔버스 메시징 채널과 마찬가지로, 콘텐츠 카드는 해당 단계에 지정된 오디언스 및 타겟팅 기준을 충족하면 사용자의 기기로 전송됩니다. 콘텐츠 카드가 전송된 후에는 다음에 카드 피드를 새로고침할 때 해당 사용자의 피드에서 해당 카드를 볼 수 있습니다.

\![메시지 단계의 메시징 채널로 콘텐츠 카드가 선택되었습니다.]({% image_buster /assets/img_archive/content-cards-in-canvas.png %})

콘텐츠 카드 단계가 캔버스와 상호 작용하는 방식을 변경하는 두 가지 옵션은 [만료와](#content-card-expiration) [제거입니다](#removal).

## 콘텐츠 카드 만료 {#content-card-expiration}

새 콘텐츠 카드를 작성할 때 전송 시간을 기준으로 사용자 피드에서 콘텐츠 카드가 만료되는 시기를 선택할 수 있습니다. 콘텐츠 카드의 만료 카운트다운은 사용자가 카드가 전송된 캔버스 단계의 메시지 단계에 도달하면 시작됩니다. 이 시점부터 카드가 만료될 때까지 사용자의 피드에서 카드가 활성화됩니다. 카드는 최대 30일 동안 사용자의 피드에 존재할 수 있습니다. 

!!! 사용자 피드에서 3시간 후에 제거되는 메시지 단계의 콘텐츠 카드에 대한 만료 설정입니다.]({% image_buster /assets/img_archive/content-cards-in-canvas-expiration.png %})

### 만료 유형

카드가 사용자의 피드에서 사라지는 시기를 설정하는 방법은 상대 날짜 또는 절대 날짜 두 가지가 있습니다.

#### 상대 날짜

'사용자 피드에서 5일 후에 보낸 카드 제거'와 같이 상대 날짜를 선택하면 최대 30일의 만료일을 설정할 수 있습니다.

#### 절대 날짜

"2023년 12월 1일 오후 4시에 보낸 카드 제거"와 같이 절대 날짜를 선택하면 약간의 뉘앙스가 달라질 수 있습니다.

만료 기간을 30일보다 길게 지정할 수 있지만 콘텐츠 카드는 최대 30일 동안 사용자의 피드에 존재합니다. 기간을 30일보다 길게 지정하면 메시지 단계를 트리거하기 전에 지연 시간을 고려할 수 있지만, 사용자 피드에서 카드의 최대 수명이 연장되지는 않습니다.

캔버스를 시작한 후 30일보다 더 일찍 만료일을 설정할 때는 주의하세요. 사용자가 지정된 만료일 30일 전에 메시지 단계에 도달하면 카드가 전송되지 않습니다.

### 만료 동작

콘텐츠 카드는 사용자가 캔버스 여정의 다음 단계로 진행하더라도 만료 날짜에 도달할 때까지 사용자의 피드에서 계속 사용할 수 있습니다. 캔버스의 다음 단계가 전달될 때 콘텐츠 카드가 라이브 상태가 되지 않도록 하려면 만료 기간을 후속 단계의 지연 시간보다 짧게 설정하세요.

콘텐츠 카드가 만료되면 사용자가 아직 보지 않았더라도 다음 새로고침 시 사용자의 피드에서 자동으로 제거됩니다.

## 콘텐츠 카드 제거 {#removal}

콘텐츠 카드는 사용자가 구매를 완료하거나 커스텀 이벤트를 수행하면 제거할 수 있습니다. 다음 중 하나를 제거 이벤트로 선택할 수 있습니다: **커스텀 이벤트 진행** 및 **구매하기**. 그런 다음 **이벤트 추가를** 선택합니다.

"사용자가 구매를 완료하거나 커스텀 이벤트를 수행하면 카드 제거" 트리거로 선택하면 '팔찌'에 대해 특정 구매를 한 사용자의 카드를 제거할 수 있습니다.]({% image_buster /assets/img_archive/content-cards-in-canvas-removal-event.png %})

## 보고 및 분석

캔버스에서 콘텐츠 카드 단계를 시작하면 이 단계에 대한 여러 가지 측정기준을 분석할 수 있습니다. 이러한 측정기준에는 전송된 메시지 수, 고유 수신자 수, 전환율, 총 매출 등이 포함됩니다.

!!\![콘텐츠 카드 메시지 성능/성과를 사용한 메시지 분석 단계.]({% image_buster /assets/img_archive/content-cards-in-canvas-analytics.png %})

사용 가능한 측정기준과 그 정의에 대한 자세한 내용은 [보고서 측정기준 용어집을]({{site.baseurl}}/user_guide/data/report_metrics/) 참조하세요.

## 사용 사례

#### 프로모션 혜택

특정 프로모션 및 광고에 대한 자격이 있는 사용자의 피드에 카드를 추가하세요. 예를 들어, 사용자가 어떤 작업을 수행하거나 구매를 한 후 새로운 오퍼를 받을 자격이 되는 경우, 캔버스를 사용하면 다른 메시징 채널 외에도 콘텐츠 카드를 보내 다음 번에 앱을 열 때 해당 오퍼를 사용할 수 있도록 할 수 있습니다.

#### 푸시 알림 받은편지함

사용자가 푸시 알림을 수신 거부하거나 이메일을 삭제할 수 있지만, 마음이 바뀔 경우를 대비하여 사용자에게 다시 알림을 보내거나 오퍼를 홍보하고 싶은 경우가 있습니다.

캔버스를 사용하면 콘텐츠 카드와 푸시 알림을 모두 보내는 컴포넌트를 추가하여 푸시를 통해 전송된 프로모션 메시지와 일치하는 카드의 지속적인 '받은편지함'을 사용자에게 제공할 수 있습니다. 

#### 카테고리에 따른 여러 피드

사용자가 탐색할 수 있는 다양한 주제나 트랜잭션 및 마케팅 피드 등의 카테고리에 따라 콘텐츠 카드를 여러 개의 피드로 분리할 수 있습니다. 키-값 페어를 사용하여 여러 개의 피드를 만드는 방법에 대한 자세한 내용은 [콘텐츠 카드 피드 커스텀하기]({{site.baseurl}}/developer_guide/customization_guides/content_cards/customizing_feed/#multiple-feeds) 가이드를 참조하세요.
