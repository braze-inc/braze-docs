# 종료 기준

> 캔버스 입력 규칙에 [예외 이벤트를]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/exception_events) 직접 추가하면 사용자가 단계가 끝날 때 이벤트가 발생하는 즉시 캔버스를 종료할 수 있습니다. 이를 통해 오디언스에게 캔버스 메시지를 보다 타겟팅된 방식으로 전달할 수 있습니다.

캔버스 빌더의 **대상 고객** 단계에서 종료 기준을 설정하여 캔버스를 종료하려는 사용자를 식별할 수 있습니다. 예외 이벤트를 추가한 다음 **트리거 추가를** 선택합니다. 

세그먼트와 필터를 종료 기준에 포함할 수도 있는데, 이는 세그먼트나 필터와 일치하는 사용자가 캔버스를 종료하고 더 이상 메시지를 받지 않도록 하는 것입니다. 캔버스의 첫 단계가 5일 지연이 있는 지연 단계인 경우 이 단계가 끝날 때 종료 기준이 적용됩니다. 따라서 사용자가 퇴출 기준을 충족하면 5일이 지나면 퇴출됩니다.

{% alert note %}
배열 속성은 현재 예외 이벤트의 종료 조건으로 지원되지 않습니다.
{% endalert %}

## 예외 이벤트

추가 예외 이벤트는 다음과 같습니다:
- 구매하기
- 세션 시작하기
- 사용자 지정 이벤트 수행
- 전환 이벤트 수행
- 이메일 주소 추가하기
- 사용자 지정 속성 값 변경하기
- 구독 상태 업데이트하기
- 구독 그룹 상태 업데이트하기
- 캠페인과 상호 작용하기
- 위치 입력
- 지오펜스 트리거
- SMS 인바운드 메시지 보내기
- WhatsApp 인바운드 메시지 보내기
- LINE 인바운드 메시지 보내기
- 카트 업데이트 이벤트 수행
- 결제 완료 이벤트 수행
- 결제 시작 이벤트 수행

## 사용 사례

회사에서 아직 구매를 하지 않은 사용자를 타겟팅하고 싶다고 가정해 보겠습니다. 먼저 예외 이벤트로 **구매하기를** 선택합니다. 다음으로 **트리거 추가를** 선택합니다. 캔버스가 시작되면 다음 **종료 기준** 설정으로 구매한 적이 있는 사용자는 대상에서 제외됩니다. 

다음 예제에서는 이 종료 기준이 정확히 한 번만 구매한 사용자의 "마지막 날에 사용됨" 세그먼트에도 적용됩니다.
