# 종료 기준

> 캔버스 입력 규칙에 예외 이벤트를 직접 추가하면 단계가 끝날 때 이벤트가 발생하는 즉시 사용자가 캔버스를 종료할 수 있습니다. 이를 통해 오디언스에게 캔버스 메시징을 보다 타겟팅된 방식으로 전달할 수 있습니다.

### 사용자 이탈 방법

종료 이벤트를 수행하면 사용자는 현재 있는 단계가 종료되는 즉시 캔버스에서 종료됩니다. 예를 들어 사용자가 30일 동안 지연 단계에 있는 경우 지연 단계 첫날에 종료 이벤트를 수행하면 사용자는 앞으로 29일 동안 캔버스를 종료하지 않습니다.

시간 기반 종료 기준을 사용할 때의 다른 예를 살펴보겠습니다. 사용자가 7월 1일 오전 12시에 24시간으로 설정된 지연 단계를 입력합니다. 이 지연 기간에는 새벽 3시에 '마지막 구매 시간 1시간 미만' 종료 이벤트를 진행합니다. 이 사용자는 지연 단계의 기간이 끝나는 7월 2일 오전 12시에 종료 기준에 대한 평가를 받게 됩니다. 7월 1일 새벽 3시에 구매한 후 21시간이 지났으므로 7월 2일 지연 단계를 종료한 후 1시간 이내에 구매를 하지 않았으므로 캔버스에서 종료되지 않습니다. 이는 사용자가 캔버스를 완전히 종료한 후에만 업데이트되는 캔버스 분석의 '종료 기준별 총 종료 수'에 영향을 미칩니다.

## 종료 기준 설정

캔버스 빌더의 **타겟 오디언스** 단계에서 종료 기준을 설정하여 캔버스를 종료할 사용자를 식별할 수 있습니다. 

종료 기준에는 사용자가 캔버스를 종료할 수 있는 특정 동작인 예외 이벤트가 포함됩니다.

제품을 둘러보았지만 아직 장바구니에 추가하거나 주문하지 않은 사용자의 재참여를 유도하기 위해 설정한 이탈 기준입니다.]({% image_buster /assets/img/exit_criteria.png %}){: style="max-width:90%;"}

### 예외 이벤트 선택하기 {#exception-events}

사용자가 예외 이벤트를 수행하면 캔버스를 종료합니다. 예외 이벤트는 사용자가 캔버스 내에 있고 사용자 여정을 진행할 때만 이탈을 트리거한다는 점에 유의하세요.

신제품을 홍보하기 위해 캔버스를 설정했다고 가정해 보겠습니다. 이 경우 제품 구매가 예외 이벤트에 해당합니다. 이렇게 하면 사용자가 구매를 한 후 이미 구매한 제품에 대한 메시징을 더 이상 받지 않게 됩니다. 예외 이벤트는 메시징을 관련성 있고 개인화된 상태로 유지합니다.

추가 예외 이벤트는 다음과 같습니다:

- 구매하기
- 세션 시작하기
- 커스텀 이벤트 수행하기
- 전환 이벤트 수행하기
- 이메일 주소 추가하기
- 커스텀 속성 값 변경하기
- 구독 상태 업데이트하기
- 구독 그룹 상태 업데이트하기
- 캠페인과 상호 작용하기
- 위치 입력
- 지오펜스 트리거하기
- SMS 인바운드 메시지 보내기
- WhatsApp 인바운드 메시지 보내기
- LINE 인바운드 메시지 보내기
- 카트 업데이트 이벤트 수행하기
- 결제 완료 이벤트 수행하기
- 결제 시작 이벤트 수행하기

#### 예약된 단계

캔버스 단계가 예약된 경우 예외 이벤트가 발생하면 사용자는 캔버스에서 즉시 드롭아웃됩니다. 사용자가 첫 번째 단계가 1주일 지연되고 예외 이벤트가 있는 캔버스에 들어왔다고 가정해 보겠습니다. 사용자가 5일째에 예외 이벤트를 수행하면 예외 이벤트를 수행한 후(5일째) 바로 종료됩니다. 
 
#### 트리거된 단계

이벤트에 의해 캔버스 단계가 트리거되면 해당 트리거에서 마지막으로 대기열에 추가된 예약된 전송은 취소되지만 사용자는 해당 기간 동안 캔버스 안에 남아있게 됩니다. 즉, 사용자가 창 내에서 트리거 이벤트를 다시 수행하면 여전히 단계를 전송할 수 있습니다. 창이 지나면 사용자는 캔버스를 종료합니다.

### 세그먼트 및 필터 사용

종료 기준에 세그먼트와 필터를 추가할 수도 있습니다. 즉, 해당 세그먼트 및 필터와 일치하는 사용자는 캔버스를 종료하고 더 이상 메시징을 수신하지 않습니다. 

예를 들어 캔버스의 첫 번째 단계가 5일 지연이 있는 지연 단계인 경우 이 단계가 끝날 때 종료 기준이 적용됩니다. 따라서 최종 사용자가 퇴출 기준을 충족하면 5일이 지나면 퇴출됩니다.

{% alert note %}
배열 속성은 현재 예외 이벤트의 종료 기준으로 지원되지 않습니다.
{% endalert %}

### 동일한 종료 이벤트와 전환 이벤트가 있는 경우

종료 이벤트와 전환 이벤트가 동일한 경우 전환 이벤트와 종료 이벤트가 모두 계산됩니다. 예를 들어 캔버스에 지연 단계가 있고 사용자가 해당 지연 단계에 있는 동안 종료 기준을 수행한 경우, 사용자가 지연 단계를 종료하자마자 종료 이벤트가 증가합니다. 이벤트가 사용자 프로필에 기록되는 즉시 전환도 증가합니다.

전환은 캔버스가 종료된 후에도 추적되지만, 사용자가 캔버스에서 나가면 종료는 추적되지 않습니다. 전환 기간은 캔버스의 최대 기간을 넘어 3일까지 연장됩니다. 즉, 이탈 추적이 중지된 후에도 전환은 계속 추적됩니다. 

전환 기간의 최소 시간은 5분입니다. 전환 이벤트가 종료 이벤트와 최대한 비슷해지도록 전환 기간을 5분으로 설정하세요. 또한 전환 창을 최소한 캔버스에서 가장 긴 경로와 일치하도록 설정하는 것이 좋습니다.

분석이 계산되는 방법에 대한 다음 예시를 살펴보세요:

1. 10명의 사용자가 캔버스를 통과합니다.
2. 세 명의 사용자가 5분 이내에 전환 이벤트를 수행합니다(종료 이벤트 수는 3개, 전환 이벤트 수는 3개).
3. 또 다른 5명의 사용자는 5분 후에 캔버스를 종료하지만 이틀 후에 전환 이벤트를 수행합니다(종료 이벤트 수는 동일하게 유지되지만 전환 이벤트는 8개로 증가).
4. 마지막 두 사용자는 5분 후에 캔버스를 종료하지만 전환 이벤트를 수행하지 않거나, 3일과 5분 후에 전환 이벤트를 수행합니다(종료 이벤트 또는 전환 이벤트 측정기준에 포함되지 않음).

## 예

배낭 공급 회사에서 아직 구매를 하지 않은 사용자를 타겟팅하고 싶다고 가정해 보겠습니다. 종료 기준을 설정하려면 다음과 같이 하세요:

1. 예외 이벤트로 **구매하기를** 선택합니다.
2. **트리거 추가를** 선택합니다. 
3. **세그먼트에서** **지난 날에 사용을** 선택하면 캔버스가 시작될 때 오디언스에서 구매한 적이 있는 사용자가 제외됩니다.
4. **필터의** 경우 **구매 행동** > **구매 횟수** > **구매한 제품을** 선택합니다.
5. 필터 그룹을 `backpack-example exactly 1` 으로 설정합니다. 즉, 백팩 제품을 구매한 사용자는 캔버스에서 나가게 됩니다.

종료 기준 설정을 "모든 구매"를 예외 이벤트로 설정하여 사용자가 모든 구매를 하면 이 캔버스를 종료합니다.]({% image_buster /assets/img_archive/exit_criteria_example.png %}){: style="max-width:80%;"}
