# 종료 기준

> 캔버스 입력 규칙에 예외 이벤트를 직접 추가하면 사용자가 단계가 끝날 때 이벤트가 발생하는 즉시 캔버스를 종료할 수 있습니다. 이를 통해 오디언스에게 캔버스 메시지를 보다 타겟팅된 방식으로 전달할 수 있습니다.

## 종료 기준 설정

캔버스 빌더의 **대상 고객** 단계에서 종료 기준을 설정하여 캔버스를 종료하려는 사용자를 식별할 수 있습니다. 

종료 기준에는 사용자가 캔버스를 종료할 수 있는 특정 동작인 예외 이벤트가 포함됩니다.

![The exit criteria set up to re-engage users who have browsed products but haven't added them to their cart or placed an order yet.]({% image_buster /assets/img/exit_criteria.png %}){: style="max-width:90%;"}

### 예외 이벤트 선택 {#exception-events}

사용자가 예외 이벤트를 수행하면 캔버스를 종료합니다. 예외 이벤트는 사용자가 캔버스에 있고 사용자 여정을 진행 중인 경우에만 종료가 트리거된다는 점에 유의하세요.

신제품을 홍보하기 위해 캔버스를 설정했다고 가정해 보겠습니다. 이 경우 제품 구매는 예외 이벤트가 됩니다. 이렇게 하면 사용자가 구매를 한 후에는 이미 구매한 제품에 대한 메시지를 더 이상 받지 않게 됩니다. 예외 이벤트는 관련성 있고 개인화된 메시지를 유지합니다.

추가 예외 이벤트는 다음과 같습니다:

- 구매하기
- 세션 시작하기
- 사용자 지정 이벤트 수행
- 전환 이벤트 수행
- 이메일 주소 추가하기
- 사용자 지정 속성 값 변경하기
- 구독 상태 업데이트하기
- 구독 그룹 상태 업데이트하기
- 캠페인과 상호 작용하기
- 위치 입력
- 지오펜스 트리거
- SMS 인바운드 메시지 보내기
- WhatsApp 인바운드 메시지 보내기
- LINE 인바운드 메시지 보내기
- 카트 업데이트 이벤트 수행
- 결제 완료 이벤트 수행
- 결제 시작 이벤트 수행

### 세그먼트 및 필터 사용

종료 기준에 세그먼트와 필터를 추가할 수도 있습니다. 즉, 세그먼트 또는 필터와 일치하는 사용자는 캔버스를 종료하고 더 이상 메시지를 수신하지 않습니다. 

예를 들어 캔버스의 첫 번째 단계가 5일 지연이 있는 지연 단계인 경우 이 단계가 끝날 때 종료 기준이 적용됩니다. 따라서 사용자가 퇴출 기준을 충족하면 5일이 지나면 퇴출됩니다.

{% alert note %}
배열 속성은 현재 예외 이벤트의 종료 조건으로 지원되지 않습니다.
{% endalert %}

## Example

배낭 공급 회사에서 아직 구매를 하지 않은 사용자를 타겟팅하고 싶다고 가정해 보겠습니다. 종료 기준을 설정하려면 다음과 같이 하세요:

1. 예외 이벤트로 **구매하기를** 선택합니다.
2. **트리거 추가를** 선택합니다. 
3. **세그먼트에서** **지난 날 사용을** 선택하면 캔버스가 시작될 때 대상에서 구매한 적이 있는 사용자가 제외됩니다.
4. **필터에서** **구매 행동** > **구매 횟수** > **구매한 제품을** 선택합니다.
5. 필터 그룹을 `backpack-example exactly 1` 으로 설정합니다. 즉, 백팩 제품을 구매한 사용자는 캔버스에서 나가게 됩니다.
