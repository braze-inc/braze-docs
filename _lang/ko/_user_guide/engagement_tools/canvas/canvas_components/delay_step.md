# 지연

> 지연 구성 요소를 사용하면 Canvas에 독립적인 지연을 추가할 수 있습니다. 관련 메시지를 추가할 필요 없이 Canvas에 지연을 추가할 수 있습니다. 

지연은 Canvas를 더 깔끔하게 보이게 할 수 있습니다. 이 구성 요소를 사용하여 특정 날짜, 특정 요일 또는 특정 주의 요일까지 다른 단계를 지연시킬 수도 있습니다. <br> \![Canvas의 첫 번째 단계로 1일 지연이 있는 지연 단계.]({% image_buster /assets/img/canvas_delay.png %}){: style="float:right;max-width:35%;margin-left:15px;"}

## 지연 만들기

지연을 만들려면 Canvas에 단계를 추가하세요. 사이드바에서 지연 구성 요소를 드래그 앤 드롭하거나 단계 하단의 <i class="fas fa-plus-circle"></i> 더하기 버튼을 클릭하고 **Delay**를 선택하세요.

Canvas 여정에서 지연을 만들 때 고려해야 할 여러 세부 사항이 있습니다.

- 지연 한도는 30일입니다.
- 지연 구성 요소는 다음 단계와만 연결할 수 있습니다.

#### 연장된 지연

이제 지연 단계를 최대 2년까지 연장할 수 있습니다. 예를 들어, 앱의 신규 사용자를 온보딩하는 경우, 세션을 시작하지 않은 사용자에게 알림 메시지를 보내기 전에 2개월 동안 연장된 지연을 추가할 수 있습니다.

## 시간 지연 유형

Canvas에서 다음 메시지 전에 지연 유형을 선택할 수 있습니다. 사용자에게 지정된 기간이 지난 후까지 지속되는 지연을 설정하거나 특정 날짜와 시간까지 사용자를 지연시킬 수 있습니다.

{% tabs %}
{% tab Duration %}

**지속 시간**을 선택하면 사용자를 설정된 초, 분, 시간, 일 또는 주 수만큼 지연시킬 수 있으며, 특정 시간에 지연됩니다. 예를 들어, 사용자를 4시간 또는 1일 동안 지연시킬 수 있습니다.
  
"일"과 "달력 일"이 계산되는 방식의 차이를 주목하세요.
  
- "일"은 24시간이며 사용자가 지연 단계에 들어간 시간부터 계산됩니다. 
- "달력 일"은 다음 지정된 시간까지 기다리는 시간을 정의하며, 이는 24시간 미만일 수 있습니다. 회사의 시간 또는 사용자의 지역 시간에 따라 지연할 수 있습니다. 시간이 지정되지 않으면 사용자는 회사 시간으로 다음 날 자정까지 지연됩니다.

또한 **특정 시간에**을 선택하여 사용자가 캔버스에서 진행할 시간을 지정할 수 있습니다. 이 옵션은 사용자가 지연 단계에 들어간 시간을 고려합니다. 이 시간이 설정에서 구성된 시간을 초과하면 지연에 더 많은 시간을 추가합니다. 

예를 들어, 오늘이 12월 11일이고 우리의 지연 단계가 UTC 오전 8시에 1주일의 **지속 시간**으로 설정되어 있다고 가정해 보겠습니다. 사용자가 12월 4일에 지연 단계에 들어갔다면, 원래 지연 단계에 오전 8시 UTC 이전에 들어갔다면 오늘 지연 단계에서 해제됩니다. 이 시간 이후에 지연 단계에 들어갔다면 사용자는 다음 날(이 시간의 다음 발생 시점)까지 지연됩니다. 

{% endtab %}
{% tab Calendar date %}

**달력 날짜**을 선택하면 특정 날짜와 시간까지 사용자를 단계에 유지할 수 있습니다.

#### 고려 사항

##### 사용자는 과거 날짜의 단계나 메시지를 받지 않습니다.

선택한 날짜와 시간이 사용자가 지연 단계로 진행할 때 이미 지나갔다면 사용자는 캔버스를 종료합니다. 캔버스 시작과 "정확한 날까지 기다리기" 단계에 선택된 날짜 사이에는 최대 31일이 있을 수 있습니다.

{% alert important %}
[캔버스 컨텍스트 조기 액세스]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/context)에 참여하고 있다면 최대 2년까지 지연을 설정할 수 있습니다.
{% endalert %}

예를 들어, 사용자는 이러한 시나리오에서 단계나 메시지를 받지 않습니다:

- 메시지가 5월 3일 오후 9시에 전송되도록 예약되어 있지만, 지연 단계는 5월 3일 오전 9시에 만료됩니다. 
- 캔버스 단계는 사용자의 로컬 시간대의 특정 시간까지 지연되지만, 사용자는 사용자 프로필에 시간대가 설정되어 있지 않습니다. 그런 다음 이 사용자에 대한 지연은 회사 시간대로 기본 설정되며, 이는 이미 지정된 시간을 지나갔습니다. 
  
##### 후속 지연 단계가 이전 지연 단계의 타임라인 내에 있으면 사용자는 종료됩니다.

캔버스에 두 개의 지연 단계가 있지만 첫 번째 지연 단계가 두 번째 지연 단계보다 길면 사용자는 캔버스를 종료합니다. 

예를 들어, 캔버스에 이러한 단계가 있다고 가정해 보겠습니다:
- 단계 1: 메시지 단계
- 단계 2: 12월 13일 오후 10시까지의 지연 단계
- 단계 3: 메시지 단계
- 단계 4: 12월 13일 오후 7시까지의 지연 단계
- 단계 5: 메시지 단계
  
단계 4에 들어가는 사용자는 단계 5를 받기 전에 캔버스를 종료합니다. 왜냐하면 단계 4의 지연이 단계 2의 시간 프레임의 일부이기 때문입니다.

{% endtab %}
{% tab Day of the week %}

**요일**을 선택하면 특정 요일의 특정 시간까지 사용자를 단계에 유지할 수 있습니다. 예를 들어, 회사 시간대에서 목요일 오후 4시에 다음 목요일이 도래할 때까지 사용자를 지연시킬 수 있습니다. 

이를 성공적으로 구성하려면 사용자가 선택한 요일(예: 목요일)에 캔버스에 들어올 때 지정된 시간 이후에 어떤 일이 발생하는지도 선택해야 합니다. 사용자가 같은 날에 진행되거나 다음 주까지 보류할 수 있습니다.
{% endtab %}
{% endtabs %}

## 지연 단계를 사용하여

6월 10일이라고 가정해 보겠습니다. 6월 11일에 사용자가 캔버스에 들어가고 다가오는 프로모션에 대한 메시지를 받기를 원합니다. 그런 다음, 6월 17일 오후 3시까지 사용자를 캔버스에 보류하고 싶습니다. 6월 17일 오후 3시에 프로모션에 대한 알림 메시지를 사용자에게 보내고 싶습니다.

캔버스 단계의 순서는 다음과 같을 수 있습니다:

1. 6월 11일에 사용자가 캔버스에 들어간 후 즉시 전송되는 메시지 단계를 추가하는 것으로 시작합니다.
2. 6월 17일 오후 1시까지 사용자를 보류하는 지연 단계를 만듭니다.
3. 지연 단계를 다른 메시지 단계에 연결하여 즉시 메시지를 전송합니다.

### 캔버스 끝에 지연 구성 요소 {#delay-as-last-step}

캔버스에 지연 구성 요소를 추가하고 후속 단계가 없으면 마지막 단계에 도달한 모든 사용자는 자동으로 캔버스에서 진행됩니다. 지연 단계의 시간이 아직 도달하지 않았더라도 이 사실은 변하지 않습니다. 이는 지연 단계에 이미 도달한 사용자가 이 단계 이후에 추가한 메시지를 받지 않음을 의미합니다. 그러나 사용자가 지연 단계에 도달하지 않았고 메시지가 추가되면 해당 메시지를 받을 것입니다.

### 개인화된 지연

{% alert important %}
개인화된 지연 및 연장된 지연은 초기 액세스 중입니다. 이 초기 액세스에 참여하고 싶다면 Braze 계정 관리자에게 문의하십시오.
{% endalert %}

사용자를 위한 개인화된 지연을 설정하려면 **개인화 지연** 토글을 선택하십시오. 지연할 컨텍스트 변수를 선택하기 위해 [컨텍스트 단계]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/context)와 함께 사용할 수 있습니다. 이것은 선택한 속성이나 속성에 설정된 시간을 덮어씁니다. 이는 날짜나 주에 오프셋을 적용할 때 유용하며, 사용자가 특정 시간에 앞으로 나아가기를 원할 때 사용됩니다. 시간대는 속성이나 속성에서 가져오거나, 사용 가능한 것이 없으면 대체를 사용합니다. 

#### "특정 시간에" 대한 시간대 동작

**특정 시간에** 옵션으로 개인화된 지연을 구성할 때, 시간대 동작은 속성 또는 컨텍스트 변수의 데이터 유형에 따라 다릅니다:

- **시간대가 있는 문자열 데이터 유형:** 속성이나 컨텍스트 변수가 시간대 정보를 포함하는 문자열 데이터 유형인 경우, 문자열에 지정된 시간대에 따릅니다. 예를 들어, `2025-06-10T10:00:00-08:00`는 UTC-8을 사용합니다.
- **시간대가 없는 문자열 데이터 유형:** 속성이나 컨텍스트 변수가 시간대 정보가 없는 문자열 데이터 유형인 경우, 대체 시간대에 따릅니다. 예를 들어, `2025-06-10`는 대체 시간대를 사용합니다.
- **시간 데이터 유형:** 속성이나 컨텍스트 변수가 시간 데이터 유형인 경우, UTC에 따릅니다. 이는 시간 데이터 유형이 데이터베이스에 저장될 때 항상 UTC로 변환되기 때문에, "특정 시간에"는 변수의 데이터 유형이 시간 데이터 유형으로 설정될 때 항상 UTC를 참조합니다. 예를 들어, `2025-06-10T10:00:00-08:00`는 UTC+0을 사용합니다.

{% alert note %}
사용자 정의 속성이나 컨텍스트 변수가 문자열 데이터 유형인 경우, 특정 시간이나 시간대가 없을 수 있습니다. 시간 데이터 유형인 경우, 시간과 시간대를 지정해야 합니다. 그러나 사용자 정의 속성이나 컨텍스트 변수가 "무관한" 문자열인 경우(예: "product_name"), 사용자는 캔버스를 종료합니다.
{% endalert %}

#### 사용 사례

고객에게 30일 후에 치약을 구매하라고 알리려는 경우를 가정해 보겠습니다. Context 단계와 Delay 단계를 조합하여 이 컨텍스트 변수를 지연할 수 있습니다. 이 경우, 귀하의 Context 단계는 다음 필드를 가집니다:

- **컨텍스트 변수 이름:** product_reminder_interval
- **데이터 유형:** 시간
- **값:** {% raw %}`{{custom_attribute.${Order_filled_time}}}`{% endraw %}

\!["product_reminder_interval" 및 그 값.]({% image_buster /assets/img/context_step1.png %})

다음으로, 고객에게 30일 후에 알림을 주고 싶으므로, 지연 옵션으로 **특정 날짜까지**를 선택하고 **지연 개인화**를 선택하여 Context 단계의 정보를 사용합니다. 이는 사용자가 선택한 Context 변수까지 지연됨을 의미합니다.

## 지연 분석

지연 구성 요소는 활성 또는 이전에 활성화된 Canvas의 분석 보기에서 다음 메트릭을 제공합니다.

| 메트릭 | 설명 |
|---|---|
| _입장한_ | 단계에 들어간 횟수를 반영합니다. Canvas에 재적격성이 있고 사용자가 Delay 단계에 두 번 들어가면 두 개의 항목이 기록됩니다. |
| _다음 단계로 진행_ | Canvas에서 다음 단계로 진행한 항목 수를 반영합니다. |
| _Canvas 종료_ | Canvas를 종료하고 다음 단계로 진행하지 않은 항목 수를 반영합니다. |
| _개인화 실패_ | 사용자를 위해 의도된 개인화된 메시지나 콘텐츠가 다음과 같은 이유로 전달되지 못한 횟수를 반영합니다:<br> {::nomarkdown}<ul><li>지연 값이 과거에 있습니다</li><li>지연 값이 2년 이상 미래에 있습니다</li><li><b>지속 시간</b> 값이 숫자가 아닙니다</li><li><b>특정 날짜까지</b> 값이 날짜 또는 날짜 형식의 문자열이 아닙니다</li></ul>{:/} <br>자세한 내용은 [개인화 실패 오류](#personaliztion-failed-errors)를 참조하십시오. |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

이 분석을 위한 시계열은 확장된 구성 요소 보기에서 사용할 수 있습니다.

## 문제 해결

### 개인화 실패 오류

사용자가 개인화된 지연을 트리거하지 않는 경우, 지연 단계에 대해 자격을 부여하기 위해 설정한 컨텍스트 단계가 예상대로 작동하지 않을 수 있습니다. [컨텍스트 변수가 유효하지 않을 때]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/context/#troubleshooting), 사용자는 컨텍스트 단계에 의해 컨텍스트가 설정되지 않고 캔버스를 계속 진행합니다. 이로 인해 사용자가 캔버스의 이후 단계, 예를 들어 개인화된 지연에 대해 자격을 갖추지 못할 수 있습니다.
