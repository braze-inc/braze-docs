# 행동 경로 

> 캔버스의 작업 경로를 사용하면 사용자의 작업에 따라 사용자를 정렬할 수 있습니다. 

![캔버스 사용자 여정의 액션 경로 단계입니다.]({% image_buster /assets/img/canvas_actionpath.png %}){: style="float:right;max-width:40%;margin-left:15px;"}

작업 경로를 사용하면 가능합니다.

* 사용자 참여 이벤트 및 사용자 지정 이벤트 등 특정 행동을 기반으로 사용자 경로를 사용자 지정합니다.
* 이 평가 기간 동안 사용자의 행동에 따라 다음 경로의 우선순위를 정하기 위해 주어진 기간 동안 사용자를 보류합니다.

## 작업 경로 만들기

액션 경로를 만들려면 캔버스에 컴포넌트를 추가합니다. 사이드바에서 구성요소를 끌어서 놓거나 단계 하단의 <i class="fas fa-plus-circle"></i> 더하기 버튼을 선택하고 **작업 경로**를 선택합니다.

### 작업 설정

**작업 설정**에서 **평가 기간**을 설정하여 사용자가 단계에 머무르는 시간을 결정합니다. 기본적으로 사용자는 하루 이내에 평가되지만, 캔버스에 따라 이 기간을 초, 분, 시간, 일, 주 단위로 조정할 수 있습니다. 작업 경로의 최대 평가 기간은 31일입니다.

**작업 설정에서** **순위 순서에 따른 고급 사용자** 토글을 켜서 구성 요소의 순위 순서를 설정할 수도 있습니다.

![1일 평가 기간을 가진 액션 설정입니다.]({% image_buster /assets/img/actionpath_settings.png %})

기본적으로 **순위**는 꺼져 있습니다. 사용자가 액션 경로에 들어가고 어떤 액션 그룹에 연결된 트리거 이벤트를 수행하면, 그들은 단계에 들어간 후 수행한 **첫 번째 자격을 갖춘 액션**에 따라 관련 액션 그룹을 즉시 진행합니다. 사용자가 다른 액션 그룹과 일치하는 두 번째 액션을 수행하면, 경로를 전환하지 않습니다. 첫 번째 액션이 그들의 경로를 결정합니다. 사용자가 트리거 이벤트를 수행하지 않으면, 평가 기간이 끝날 때 기본 **모든 다른 사람** 그룹을 통해 진행합니다.

**순위에 따라 사용자 진행**이 켜져 있으면, 이는 **순위**가 켜져 있음을 의미합니다. 따라서 모든 사용자는 평가 기간이 끝날 때까지 대기합니다. 평가 기간이 끝날 때, 사용자는 평가 기간이 끝날 때 자격이 있는 가장 높은 우선 순위의 액션 그룹을 통해 진행합니다. 평가 기간 동안 어떤 액션도 수행하지 않은 사용자는 기본 **모든 다른 사람** 그룹을 통해 진행합니다.

{% alert tip %}
사용자가 수행하는 액션이 아닌 현재 속성이나 세그먼트 멤버십에 따라 사용자를 라우팅하려면 대신 [오디언스 경로]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/audience_paths/)를 사용하십시오.
{% endalert %}

중첩 커스텀 속성 객체가 변경될 때 액션 경로를 트리거할 수 있지만, 중첩 커스텀 속성 배열이나 오브젝트 배열 데이터 유형의 변경에 대해서는 트리거할 수 없습니다.

#### 인앱 메시지

액션 그룹 트리거가 세션을 시작하고 다음 단계가 인앱 메시지일 때, 사용자는 인앱 메시지를 받기 위해 두 번의 세션 시작을 수행해야 합니다. 첫 번째 세션은 사용자를 행동 경로 내의 동작 그룹에 할당하고, 두 번째 세션은 인앱 메시지를 트리거합니다.

#### 랭킹 상태 예시

두 개의 작업 그룹이 있는 평가 기간이 하루인 행동 경로가 있다고 가정해 보겠습니다: 그룹 1 및 그룹 2. 그룹 1에는 "세션 시작" 트리거 이벤트가 있고 그룹 2에는 "구매하기" 트리거 이벤트가 있습니다. **순위**가 켜져 있으면 행동 경로에 있는 모든 사용자가 하루 동안 '보류'됩니다. 하루가 끝나면 사용자가 세션을 시작하고 구매를 한 경우 가장 높은 등급 경로로 올라갑니다. 이 경우 사용자는 그룹 1로 진행됩니다. 

앞의 예에서 **순위**가 꺼져 있고 사용자가 트리거 이벤트 중 하나("세션 시작" 또는 "구매하기")를 수행하면, 해당 사용자는 트리거 액션에 따라 관련 액션 그룹에서 진행됩니다.

캔버스 항목 속성은 이벤트 속성과 다릅니다. 캔버스 항목 속성은 캔버스를 트리거한 이벤트의 속성입니다. 이러한 속성은 원래 캔버스 워크플로우를 사용할 때 캔버스의 첫 번째 전체 단계에서만 사용할 수 있습니다. 캔버스를 사용할 때, 지속적인 진입 속성이 활성화되어 캔버스 전반에 걸쳐 진입 속성을 재사용할 수 있습니다. 반대로 이벤트 속성정보는 사용자가 워크플로우를 진행하면서 발생하는 이벤트 또는 작업에서 비롯됩니다.

### 액션 그룹

트리거 하나 또는 여러 개의 트리거를 추가하여 작업 그룹을 정의하세요. 여기에서 사용자가 다음과 같은 경우의 트리거 범위를 선택할 수 있습니다:

- 구매하기
- 세션 시작
- [사용자 지정 이벤트]({{site.baseurl}}/user_guide/data/custom_data/custom_events/) 수행
- 전환 이벤트 수행
- 이메일 주소 추가
- 커스텀 속성 값을 변경합니다.
  - 이는 속성이 이전에 존재하지 않았을 때 사용자 프로필에 값을 가진 새로운 속성을 추가하는 것을 포함합니다.
  - 속성 트리거는 배열 속성에 대해 사용할 수 없습니다.
- 구독 상태 또는 구독 그룹 상태 업데이트하기
- 캠페인 또는 콘텐츠 카드와 상호 작용
- 위치 입력
- 지오펜스 트리거
- SMS 또는 WhatsApp 인바운드 메시지 보내기

![구매를 하는 사용자들을 위한 "그룹 1"이라는 이름의 액션 그룹입니다.]({% image_buster /assets/img/actionpath_group.png %})

각 작업 그룹 설정에서, 사용자는 **이 그룹이 캔버스를 종료하도록 선택할 수 있는 체크박스**를 선택할 수 있는 옵션이 있습니다. 이는 이 그룹의 사용자들이 평가 기간이 끝날 때 캔버스를 종료함을 의미합니다.

### 재자격이 있는 캔버스

사용자가 액션 경로를 여러 번 입력하고 동시에 액션 경로에 여러 항목이 있는 경우, 예상되는 동작은 **순위** 상태에 따라 달라집니다.

| 랭킹 상태 | 동작 경로 동작 |
|---|--------------|
| **끄기** | 관련 작업이 수행되면, Braze는 항목을 중복 제거하고 즉시 가장 이른 항목을 관련 작업 그룹으로 진행시킵니다. <br><br> 관련 작업이 수행되지 않으면, 모든 항목은 관련 평가 창의 끝에서 진행됩니다. 중복 제거가 발생하지 않습니다. |
| **켜기** | 모든 항목은 관련 평가 창의 끝에서 진행됩니다. 중복 제거가 발생하지 않습니다. |
