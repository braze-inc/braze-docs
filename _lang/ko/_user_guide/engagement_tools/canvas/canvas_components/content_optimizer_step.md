# 콘텐츠 최적화 에이전트 단계

> 콘텐츠 최적화 에이전트 단계를 사용하면 단일 단계 내에서 여러 버전의 콘텐츠 컴포넌트를 구성하고 테스트할 수 있습니다. 콘텐츠 변형을 실험하고 시간이 지남에 따라 성능/성과가 가장 좋은 조합으로 자동 최적화하는 데 도움이 됩니다. 소개는 [콘텐츠 옵티마이저를]({{site.baseurl}}/user_guide/brazeai/content_optimizer/) 참조하세요.

{% alert important %}
콘텐츠 최적화 도구는 현재 베타 버전입니다. 시작하는 데 도움이 필요하면 고객 성공 매니저에게 문의하세요.
{% endalert %}

## 콘텐츠 최적화 도구 만들기 단계

최상의 결과를 얻으려면 사용자가 시간이 지남에 따라 단계를 점진적으로 입력하는 캔버스에서 콘텐츠 최적화 에이전트를 사용하세요. 모든 사용자가 한꺼번에 단계를 입력하면 상담원이 초기 결과에서 학습할 시간이 없습니다. 

### 1단계: 단계 추가

사이드바에서 **콘텐츠 최적화** 도구 구성 요소를 드래그 앤 드롭하거나 단계 하단에 있는 <i class="fas fa-plus-circle"></i> 더하기 버튼을 선택하고 **콘텐츠 최적화** 도구를 선택합니다.

### 2단계: 기본 메시지 만들기

기본 메시지는 단계의 시작점입니다. 각 콘텐츠 컴포넌트의 배리언트는 **콘텐츠 최적화 도구 설정** 탭에 정의된 조합에 따라 동적으로 삽입됩니다. 

{% alert note %}
베타 기간 동안에는 이메일만 지원되는 채널입니다.
{% endalert %}

**메시징 채널** 탭에서 **이메일을** 선택하고 기본 이메일 메시지를 작성합니다. 도움이 필요하면 전용 [이메일]({{site.baseurl}}/user_guide/message_building_by_channel/email) 섹션을 참조하세요. 

콘텐츠 최적화 에이전트는 이 배리언트에 지정된 전송 설정(예: 이메일 도메인 및 회신 주소)을 사용하여 모든 메시지를 전송합니다. 새 디자인으로 시작하거나 이 메시징에 대한 기존 템플릿을 선택할 수 있습니다. 이 단계에서는 메시징의 어떤 구성 요소를 최적화할지 고려하세요. [4단계에서](#step-4) 이를 정의합니다.

최적화할 수 있는 지원되는 컴포넌트는 다음과 같습니다:

- Subject
- 본문 헤더
- 본문 콘텐츠
- 기본 CTA

### 3단계: 전달 설정 지정

**전달 설정** 탭에서 단계가 Intelligent Timing 또는 전달 유효성 검사를 사용할지 여부를 지정할 수 있습니다. 자세한 내용은 메시지 단계에서 [전달 설정 편집하기를]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/message_step/#step-2-edit-delivery-settings) 참조하세요.

### 4단계: 콘텐츠 컴포넌트 및 배리언트 추가하기 {#step-4}

콘텐츠 구성 요소는 다양한 제목란, 헤드라인, 본문 텍스트 또는 주요 클릭 유도 문안 등 테스트하려는 메시지의 개별 요소입니다. 이러한 구성 요소를 사용하면 메시징의 여러 버전을 생성하고 시간 경과에 따른 성능/성과에 따라 자동으로 최적화할 수 있습니다.

한 단계당 최대 3개의 콘텐츠 구성 요소와 구성 요소당 최대 5개의 배리언트를 추가하여 총 125개의 고유한 콘텐츠 조합을 만들 수 있습니다.

![콘텐츠 최적화 도구 인터페이스에서 콘텐츠 구성 요소를 추가하고 구성하기 위한 옵션입니다. 인터페이스에는 제목, 본문 헤더, 본문 콘텐츠 및 기본 CTA와 같은 선택 가능한 구성 요소가 표시되며, 각각 다른 배리언트를 입력할 수 있는 필드가 있습니다.]({% image_buster /assets/img/content_optimizer/add_content_components.png %})

#### Step 4.1: 콘텐츠 구성 요소 구성

구성 요소를 구성하려면

1. **콘텐츠 최적화 도구 설정** 탭으로 이동합니다.
2. 최적화할 컴포넌트를 선택합니다. 지원되는 옵션:
  - Subject
  - 본문 헤더
  - 본문 콘텐츠
  - 기본 CTA
3. 선택한 각 컴포넌트에 대해 해당 콘텐츠의 대체 버전(배리언트) 집합을 정의합니다. 어조, 구조 또는 내용이 다른 명확하고 뚜렷한 배리언트를 사용하세요. 이를 통해 콘텐츠 최적화 도구는 성능/성과가 우수한 콘텐츠를 보다 효과적으로 식별할 수 있습니다. 할 수 있습니다:
  - 배리언트를 직접 작성하세요.
  - AI가 생성한 제안을 사용하여 새로운 옵션을 빠르게 탐색하세요.

![이메일 최적화를 위한 콘텐츠 구성 요소를 추가하고 구성하는 옵션을 보여주는 콘텐츠 최적화 도구 설정 인터페이스입니다. 각 컴포넌트에는 다양한 배리언트를 입력할 수 있는 입력 필드가 있습니다. 표시되는 텍스트에는 컴포넌트 이름과 배리언트 텍스트를 입력할 수 있는 필드가 포함됩니다.]({% image_buster /assets/img/content_optimizer/content_optimizer_settings.png %})

#### Step 4.2: 메시징에 Liquid 추가하기

각 컴포넌트에 대해 최소 두 가지 이상의 배리언트를 정의한 후 각 배리언트에 연결된 Liquid 태그를 복사하여 기본 메시징의 해당 위치에 붙여넣으세요.

- 예를 들어 제목란을 최적화하는 경우 이메일 작성기의 제목 필드에 {% raw %}`{% message_component "Subject" %}`{% endraw %} 태그를 붙여넣습니다.
- 컴포넌트의 일부만 테스트하기 위해 긴 텍스트 안에 컴포넌트 태그를 포함할 수도 있습니다. 예: {% raw %}`Hey there, {% message_component "Subject" %}`{% endraw %}.

![제목, 본문 헤더, 본문 콘텐츠 및 기본 CTA와 같은 콘텐츠 구성 요소를 추가하고 구성하는 옵션입니다. 각 컴포넌트에는 다양한 배리언트를 입력할 수 있는 필드가 있습니다.]({% image_buster /assets/img/content_optimizer/optimization_liquid_in_use.png %})

선택한 콘텐츠 컴포넌트에 대해 Liquid 태그를 추가하지 않으면 **콘텐츠 최적화 도구 설정** 탭에 경고가 표시되고 **메시징 채널** 탭에 오류가 표시됩니다. 선택한 모든 컴포넌트가 기본 메시징에 제대로 추가될 때까지 캔버스를 시작할 수 없습니다.

캔버스가 실행되면 에이전트는 여러 컴포넌트에서 배리언트를 믹스 앤 매치하여 다양한 콘텐츠 조합을 생성합니다. 시간이 지남에 따라 더 높은 성능의 조합이 우선적으로 전달되므로 수동 개입 없이도 성능/성과를 개선할 수 있습니다.

| Component | Liquid 스니펫 |
| --- | --- |

| 본문 헤더 | {% raw %}`{% message_component "Body Header" %}`{% endraw %} |

| 기본 CTA | {% raw %}`{% message_component "Primary CTA" %}`{% endraw %} |

### 5단계: 최적화 이벤트 선택

최적화 이벤트는 콘텐츠 최적화 에이전트가 성능/성과를 평가하고 시간에 따라 콘텐츠 조합에 트래픽을 할당하는 방법을 결정합니다. 

이메일의 경우 다음 이벤트 중 하나에 최적화할 수 있습니다. 에이전트는 메시지를 보낸 후 7일 이내에 등록된 열기 및 클릭을 사용하여 성능/성과가 더 높은 콘텐츠 조합으로 전달을 전환합니다.

| Event | 설명 | 사용 사례 |
| --- | --- | --- |
| 열람 수 | 수신자가 이메일을 열도록 유도하는 조합을 최적화합니다. | 제목란을 테스트하거나 가시성을 높이려는 경우 |
| 클릭 수 | 링크 참여를 유도하는 조합을 최적화합니다. 봇 클릭 또는 Braze에서 인식한 탈퇴 클릭은 포함되지 않습니다. | 링크에서 트래픽, 참여 또는 전환 유도하기 |
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3 role="presentation" }

선택한 최적화 이벤트는 이 단계의 모든 콘텐츠 구성 요소에 적용됩니다. 

## 분석

성능을 검토하려면 단계 수준 분석 패널을 열어 콘텐츠 배리언트별 측정기준과 전체 조합 성과를 확인하세요. 콘텐츠 최적화 단계는 메시지 단계와 동일한 분석을 사용합니다. 자세한 내용은 메시지 단계의 [분석을]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/message_step/#analytics) 참조하세요.  

## 문제 해결

| Issue | 설명 | 고치다 |
| --- | --- | --- |
| 누락된 Liquid 태그 | 콘텐츠 컴포넌트(예: 제목 또는 CTA)를 추가했지만 해당 Liquid 태그를 기본 메시지에 삽입하지 않은 경우 표시됩니다: <br>- **콘텐츠 최적화 도구 설정** 탭의 경고 표시 <br>- **메시징 채널** 탭에 오류가 발생했습니다. | **콘텐츠 최적화 도구 설정** 탭의 각 컴포넌트 아래에 표시된 Liquid 스니펫을 복사하여 메시징의 적절한 부분에 붙여넣습니다. |
| 고아 Liquid 태그 | 콘텐츠 컴포넌트를 삭제했지만 기본 메시징에 해당 Liquid 태그를 남겨두면 메시징이 전송될 때 예상대로 렌더링되지 않을 수 있습니다. | 시작하기 전에 기본 메시지에서 사용하지 않는 `message_component` 태그를 모두 제거하세요. |
