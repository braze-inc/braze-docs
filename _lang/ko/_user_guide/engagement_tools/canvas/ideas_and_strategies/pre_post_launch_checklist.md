# 출시 전 및 출시 후 체크리스트

> 이 문서에서는 캔버스를 시작하기 전과 후에 확인해야 할 사항에 대한 가이드라인을 제공합니다.

## 출시 전 고려해야 할 사항

캔버스를 시작하기 전에 메시징 및 전송 시간을 오디언스의 선호도에 맞게 조정하기 위해 확인할 수 있는 몇 가지 세부 정보가 있습니다. 고려해야 할 사항에는 시간대, 항목 설정 등의 변동 사항이 포함됩니다. 이 체크리스트를 가이드로 삼아 사용 사례에 따라 이러한 영역을 세밀하게 조정하여 캔버스의 성공에 기여할 수 있도록 하세요. 

### 시간대 설정 검토

예약된 입력 일정을 사용하여 현지 시간대에 따라 사용자를 입력하는 경우 사용자가 캔버스에 입력할 시간보다 최소 24시간 전에 캔버스를 시작해야 합니다. 예를 들어, 다음은 시작과 예정된 입장 시간 사이에 충분한 시간이 남지 않은 캔버스입니다. 이 시나리오에서는 특정 시간대의 경우 예정된 입장 시간이 이미 지났기 때문에 캔버스에 입장하지 않는 사용자가 있을 수 있습니다. 

{% alert tip %}
버퍼를 충분히 예약하지 않은 경우 알림이 표시됩니다. 빠른 해결책은 사용자가 24시간 동안 타겟팅된 세그먼트에 머물 수 있도록 전송 시간을 조정하는 것입니다.
{% endalert %}

### 오디언스 필터에 정규표현식 사용 고려하기

사용자가 언제 캔버스에 입장해야 하는지에 대한 사전 세부 정보를 설정했다면, 이제 캔버스 구축의 **타겟 오디언스** 단계에서 세그먼트 또는 필터를 확인하는 것이 좋습니다. 이 단계에서는 **대상 집단** 요약을 검토하여 대상 집단이 어떻게 설정되었는지 확인할 수도 있습니다. 

여기서 오디언스 경로 단계의 세그먼트 또는 필터, 메시지 및 결정 분할 단계의 전달 유효성 검사 설정에도 정규표현식을 사용하는 것을 고려하세요. [정규표현식(정규식이라고도]({{site.baseurl}}/user_guide/engagement_tools/segments/regex/) 함)은 문자열로, 대문자 대신 패턴을 인식하고 문자를 고려한다는 의미입니다. 즉, '같음/같지 않음'을 사용하는 경우 단순한 구문 오류로 인해 오디언스 규모가 제한될 수 있습니다.

타겟팅 오디언스가 예상보다 적은 경우 '같음' 또는 '같지 않음' 대신 '정규식 일치' 또는 '정규식 일치하지 않음'을 사용해 보세요. 이렇게 하면 누락된 사용자를 설명하고 더 많은 오디언스를 타겟팅할 수 있습니다. 

### 엔트리 설정 및 경합 조건 식별자 확인

경합 조건은 **참가 일정과** **타겟 오디언스** 설정 모두에서 동일한 참가 기준을 사용한 경우 발생할 수 있습니다. 

액션 기반 항목을 사용하는 경우 타겟 오디언스에서와 동일한 트리거 동작을 여기에 사용하지 않았는지 확인하세요. 트리거 이벤트를 수행할 때 사용자가 오디언스에 없는 경합 조건이 발생할 수 있으며, 이는 사용자가 캔버스에 입장하지 않음을 의미합니다.

{% alert tip %}
오디언스 필터와 동일한 트리거를 사용하여 액션 기반 캔버스를 설정할 때 이러한 경합 조건을 피하기 위한 [모범 사례를]({{site.baseurl}}/user_guide/engagement_tools/testing/race_conditions/#scenario-3-matching-action-based-triggers-and-audience-filters) 확인하세요.
{% endalert %}

### 캔버스 항목 속성 및 이벤트 속성정보 확인

이름은 비슷하지만 [캔버스 항목 속성정보와 이벤트 속성정보는]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/canvas_entry_properties_event_properties) 캔버스 워크플로 내에서 서로 다른 방식으로 작동합니다. 캔버스 항목 속성은 항목 설정에 연결되며 캔버스 전체에 걸쳐 모든 메시지 구성 요소에서 참조할 수 있습니다. 캔버스 입력 속성정보는 액션 기반 또는 API 트리거 입력 설정을 사용하여 사용자의 캔버스 입력을 트리거하는 이벤트 또는 API 호출의 속성입니다.

반면 이벤트 속성정보는 행동 경로 단계 다음에 나오는 첫 번째 메시지 단계에서만 참조할 수 있습니다. 이벤트 속성은 사용자가 행동 경로 단계의 평가 기간 동안 수행한 커스텀 이벤트 또는 구매 이벤트의 속성이며, 정의된 행동 경로 중 하나로 진행을 트리거합니다.

메시지 미리보기에서 캔버스 항목 속성 또는 이벤트 속성정보를 참조하는 메시지 단계가 있는지 확인하세요.

### 사용자 진행을 위한 메시지 단계 검토하기

기본값으로 사용자는 메시지 수신 여부에 관계없이 모든 메시지 단계를 진행합니다. 특정 메시지를 수신하는 사용자를 진행하려면 메시지 컴포넌트 바로 뒤에 결정 분할 단계를 추가하면 됩니다. 추가 필터로 '캔버스 단계에서 메시지 수신' 필터를 추가한 다음 캔버스 및 메시지 단계를 선택합니다.

인앱 메시징이 포함된 메시지 단계의 경우 결정 분할 컴포넌트 대신 행동 경로 컴포넌트를 사용할 수 있습니다. 이렇게 하면 사용자가 인앱 메시지를 보았는지 여부에 따라 진행을 진행할 수 있습니다. "단계와 상호 작용" 필터를 추가하여 작업 그룹을 정의하고 **인앱 메시지 보기를** 선택합니다. 그런 다음 단계의 평가 기간을 인앱 메시지의 만료 기간으로 설정합니다.

멀티채널 메시징의 메시지 컴포넌트의 경우 다음을 권장합니다:
* 메시지 및 결정 분할 단계 사이에 지연 단계를 포함하고 지연을 5초 이상으로 설정합니다.
* 컴포넌트에 Intelligent Timing이 포함된 경우 지연을 24시간으로 설정합니다.
* 컴포넌트에 속도 제한이 포함된 경우 메시지를 여러 개의 단일 채널 메시지 단계로 분할하여 서로 연결하세요. 그런 다음 마지막 메시지 단계 바로 뒤에 결정 분할 단계를 연결하여 사용자가 메시지를 받았는지 확인합니다. 이 방법을 Intelligent Timing을 사용하는 멀티채널 메시지 단계의 대안으로 사용할 수도 있습니다.

## 출시 후 고려해야 할 사항

캔버스를 시작하셨습니다! 이제 어떻게 할까요? 이 체크리스트를 사용하여 이러한 시나리오에 따라 출시 후 불일치하는 경우 캔버스를 검토하고 조정하는 방법을 알아보세요.

### 항목은 많은데 전송은 적은 경우

예를 들어, 보낸 메시지 수와 총 항목 수 사이에 불균형이 발생했다고 가정해 보겠습니다. 이러한 주요 영역을 확인하여 캔버스를 조정할 영역을 식별하고 발견할 수 있습니다.

#### 엔트리 오디언스

예약 전송 캠페인을 사용하는 경우 대상 집단을 검토하여 타겟 오디언스를 다시 확인하세요. 채널 전체에서 숫자는 어떻게 보이며, 캔버스에서 사용한 채널과 어떤 관련이 있나요? 가장 낮은 숫자가 캔버스에서 사용한 채널과 일치한다면 문제를 발견한 것일 수 있습니다.

#### 캔버스의 첫 번째 구성 요소

캔버스의 시작 구성 요소에 사용된 오디언스 필터, 동작 트리거 또는 세그먼트를 검토하세요. 캔버스를 제대로 시작하지 못하게 하는 철자가 틀렸거나 너무 엄격한 조건이 있나요? '정규식 일치'를 사용해야 할 때 '같음'을 사용하고 있나요?

#### 캔버스 대조군 

배리언트와 대조군 간의 사용자 분포를 검토합니다. 대조군이 의도한 것보다 더 큰가요? 그렇다면 이 설정을 편집할 수 있습니다. **지능형 선택이** 켜져 있고 대조군이 이기고 있다면 캔버스를 중지하고 새로운 접근 방식을 시도해 보세요.

### 비어 있는 총 오디언스 수

캔버스에 대한 입력 데이터가 표시되지 않는 경우 사용자가 캔버스에 입력하지 않는 이유는 경합 조건 및 제한적인 오디언스 세분화 필터 때문일 수 있습니다.

입력 일정에서 액션 기반 입력을 사용하는 경우, **타겟 오디언스에서와** 동일한 트리거 동작을 여기에 사용하지 않았는지 확인하세요. 트리거 이벤트를 수행할 때 사용자가 오디언스에 없는 경합 조건이 발생할 수 있으며, 이는 사용자가 캔버스에 입장하지 않음을 의미합니다.

또한 **대상 오디언스** 설정에서 **대상 집단** 표를 검토하여 선택한 세그먼트에 사용자가 있는지 확인합니다. 이 숫자가 낮으면 입력 설정을 조정하거나 선택한 세그먼트 또는 필터에 오류가 있는지 검토하는 방법을 참조하세요.

### 단계 간 예기치 않은 드롭오프

한 캔버스 단계에서 다음 캔버스 단계로 넘어갈 때 큰 차이가 있을 때 캔버스 조정 영역을 식별할 수 있는 또 다른 명백한 방법이 있습니다. 이 경우 오디언스 필터와 예외 이벤트에 철자 오류나 대/소문자 오류가 없는지 확인하세요. 그리고 항상 그렇듯이 오디언스 필터가 너무 엄격해서 대다수의 사용자가 캔버스에 들어가지 못하는 것은 아닌지 확인하세요. 

다음으로, 사용자에게 메시징을 보내는 시기와 방법에 영향을 줄 수 있는 이러한 설정을 식별하는 것이 중요합니다:
- [Intelligent Timing]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_timing/)
- 방해금지 시간
- 전달 유효성 검사

일반적으로 캔버스에는 Intelligent Timing 또는 조용한 시간 중 하나를 선택하되, 둘 다는 선택하지 마세요. Intelligent Timing 또는 [속도 제한]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/) 중 하나를 사용하는 경우에도 동일한 제안이 적용되며, 둘 다 사용하지 않는 경우에도 마찬가지입니다. Intelligence Suite를 가장 효과적으로 사용하는 방법에 대한 자세한 내용은 [Intelligence Suite 사용 사례를]({{site.baseurl}}/user_guide/brazeai/intelligence/#use-cases) 참조하세요.

### 경로 간 의심스러운 전송 볼륨

두 개 이상의 경로(오디언스 경로 또는 행동 경로) 간의 전송량이 예상과 다른 경우 세그먼트, 필터 또는 트리거 동작을 확인할 수 있는 기회가 될 수 있습니다. 또한 겹치는 필터가 있으면 식별자를 찾아서 제거하세요.
