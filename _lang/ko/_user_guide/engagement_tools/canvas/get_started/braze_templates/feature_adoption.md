# 기능 채택

> 이 템플릿은 새로운 기능, 기존 제품, 추가 오퍼링 또는 고객이 경험하기를 원하는 기타 영역의 사용을 유도하기 위해 고안되었습니다. 개인화된 커뮤니케이션과 구조화된 메시지 세트를 활용하면 사용자에게 새로운 기능을 원활하게 소개하고 사용자로부터 귀중한 피드백을 얻을 수 있습니다. 

이 문서에서는 사용자 라이프사이클의 리텐션 및 충성도 단계를 위한 **기능 채택** 템플릿의 사용 사례를 안내합니다. 이 글을 읽고 나면 사용자가 새로운 기능을 사용하도록 장려하고 사용자 감정을 수집하는 사용자 여정을 맞춤 설정하게 됩니다.

## 필수 조건

이 템플릿을 성공적으로 사용하려면 사용자가 기능을 사용한 시점을 참조하는 [사용자 지정 이벤트가]({{site.baseurl}}/user_guide/data_and_analytics/custom_data/custom_events) 필요합니다.

## 필요에 맞게 템플릿 조정하기

최근 반복적인 음식 배달을 예약하는 기능인 크루즈 컨트롤을 출시한 음식 배달 앱인 칼로리 로켓에서 일하면서 더 많은 사용자가 이 새로운 기능을 채택하도록 장려하고 싶다고 가정해 보겠습니다. 이 예에서는 사용자 지정 이벤트 `scheduled_delivery` 를 사용하여 사용자가 크루즈 컨트롤 기능을 시도한 시기를 추적합니다.

재고가 있는 템플릿에 액세스하려면 새 캔버스를 만들 때 **캔버스 템플릿 사용** > **브레이즈 템플릿을** 선택합니다. 그런 다음 **기능 채택** 옆의 **템플릿 적용을** 선택합니다. 이제 템플릿을 검토하여 필요에 맞게 조정할 수 있습니다.

### 1단계: 세부 정보 설정

목표를 반영하도록 캔버스 세부 사항을 조정해 보겠습니다.

1. 템플릿 이름 옆의 **편집을** 선택합니다.

![캔버스의 현재 제목과 설명입니다.]({% image_buster /assets/img/canvas_templates/feature_adoption/select_edit_details.png %}){: style="max-width:60%;"}

{:start="2"}
2\. 캔버스 이름을 업데이트하여 캔버스가 사용자 피드백을 수집하기 위한 타겟팅 사용자용임을 지정합니다.
3\. 설명을 업데이트하여 캔버스가 새로운 크루즈 컨트롤 기능에 대한 사용자의 피드백을 제출하고 사용자 의견을 추적하기 위한 것임을 명시합니다.
4\. 캔버스 홈페이지에서 필터링할 수 있도록 **기능 채택이라는** 태그를 추가합니다.

![캔버스의 새 이름과 설명입니다. 새 설명에는 다음과 같이 명시되어 있습니다: '반복되는 음식 배달을 예약하는 기능인 크루즈 컨트롤의 채택과 사용자 정서를 추적하는 기능 채택 캔버스']({% image_buster /assets/img/canvas_templates/feature_adoption/enter_new_canvas_name.png %}){: style="max-width:60%;"}

### 2단계: 전환 이벤트 할당

다음으로 캔버스에 전환 이벤트를 추가하여 기능 채택을 알리겠습니다. 이를 통해 나중에 사용자 여정에서 실험 경로를 맞춤화할 수 있습니다.

1. **전환 이벤트 할당** 아래에서 **전환 이벤트 추가를** 선택합니다.
2. **기본 전환 이벤트 - A에서** **전환 이벤트 유형으로** **사용자 지정 이벤트 수행을** 선택합니다.
3. 사용자 지정 이벤트 선택 `scheduled_delivery`.
4. 전환 기한은 3일로 유지됩니다.

![캔버스의 전환 이벤트 창.]({% image_buster /assets/img/canvas_templates/feature_adoption/assign_conversion_event_cruise_control.png %}){: style="max-width:90%;"}

### 3단계: 참가 일정 맞춤 설정

저희의 목표는 사용자가 크루즈 컨트롤을 사용하도록 장려하는 것이지만, 메시지가 너무 자주 전달되는 것은 원하지 않습니다. 따라서 이 캔버스를 예약된 배송으로 유지하고 **시간 기반 옵션** 섹션을 다음과 같이 조정하겠습니다.

1. **입력 주기를** **매주로** 업데이트합니다.
2. 리커버리를 그대로 유지합니다.
3. 주 초에 사용자를 타겟팅하려면 **월을** 선택합니다.
4. 캔버스의 시작 시간을 선택합니다.
5. **종료 매개변수를** 업데이트하여 한 해의 마지막 날에 캔버스를 종료합니다.

사용자가 현지 시간대로 캔버스에 입장할 수 있도록 하는 옵션은 계속 유지됩니다.

### 4단계: 타겟 오디언스 선택

이제 템플릿에서 다음 세부 정보를 업데이트하여 타겟 오디언스를 설정해 보겠습니다:

1. **모든 사용자** 세그먼트를 선택합니다.
2. 템플릿의 추가 필터를 제거합니다. 
3. 사용자 지정 이벤트를 사용하여 이 필터를 만듭니다: `Has scheduled_delivery for exactly 0 times`. 이를 통해 이미 해당 기능을 사용한 사용자는 캔버스에 입장하지 못하도록 제외할 수 있습니다.

![크루즈 컨트롤을 사용하지 않은 모든 사용자를 위한 세그먼트입니다.]({% image_buster /assets/img/canvas_templates/feature_adoption/cruise_control_segment.png %}){: style="max-width:90%;"}

{: start="4"}
4\. 이전에 칼로리 로켓에서 일부 사용자에게 새로운 기능인 크루즈 컨트롤의 베타 테스트를 허용했던 것을 염두에 두고, 이러한 사용자는 캔버스 진입에서 제외되도록 종료 기준을 업데이트할 예정입니다.

### 5단계: 전송 설정을 선택하세요

기본 구독 설정을 유지하여 구독했거나 메시지 또는 알림 수신을 선택한 사용자에게만 보내고 다른 설정(빈도 제한, 조용한 시간, 시드 그룹)은 건너뜁니다.

### 6단계: 캔버스 사용자 지정

#### 액션 경로 구축

다음으로, 사용자가 새 기능에 관심이 있는지 여부를 나타내는 첫 번째 액션 경로 단계를 구축해 보겠습니다. 템플릿을 다음과 같이 조정하겠습니다:

1. 크루즈 컨트롤 기능은 주문이 카트에 **추가된** 후에만 사용할 수 있으므로 첫 번째 작업 그룹의 이름을 **카트에 추가됨으로** 지정하고 사용자 지정 이벤트에 `added_to_cart` 을 선택합니다.

![작업 그룹 이름을 "장바구니에 추가됨"으로 설정하고 "사용자 지정 이벤트 수행"을 "장바구니에 추가됨"으로 설정합니다.]({% image_buster /assets/img/canvas_templates/feature_adoption/action_path_added_to_cart.png %}){: style="max-width:60%;"}

{: start="2"}
2\. 사용자가 앱을 둘러보았는지 평가하고, 둘러본 경우 두 번째 경로로 진행하기 때문에 두 번째 작업 그룹인 **둘러보기는** 그대로 유지합니다.
3\. **사용량 평가라는** 이름의 후속 작업 경로에서 **사용 기능 >3배를** **크루즈 컨트롤 보기 설정으로** 바꿉니다.
4\. 사용자 지정 이벤트 **수행** 드롭다운을 선택한 다음 사용자 지정 이벤트에 대해 `scheduled_delivery` 을 선택합니다.

![액션 그룹 이름은 '사용된 기능 >3배'로 설정하고 '사용자 지정 이벤트 수행'을 '예정된_배달'로 설정합니다.]({% image_buster /assets/img/canvas_templates/feature_adoption/action_path_assess_usage.png %}){: style="max-width:60%;"}

#### 피드백 설문조사 설정

다음으로 **피드백 설문조사라는** 메시지 단계로 이동하여 사용자가 크루즈 컨트롤을 처음 사용한 후 작성할 수 있는 피드백 설문조사를 포함하겠습니다. 사용자를 위한 설문조사 응답 옵션은 다음과 같습니다:

- **마음에 들었습니다!**
- **저에게는 해당되지 않습니다.**

1. 두 가지 설문조사 선택 항목에 대해 사용자 지정 속성으로 **경험 피드백을** 선택하여 크루즈 컨트롤에 대한 피드백을 캡처하고 추적합니다. 이 사용자 지정 속성에는 설문조사 응답을 나타내는 두 가지 값(`good` 및 `bad`)이 있습니다.
2. 설문조사 옵션과 일치하도록 속성 값을 업데이트합니다. 이를 통해 사용자의 응답을 추적할 수 있습니다.

### 7단계: 캔버스 테스트 및 실행

캔버스를 테스트하고 검토하여 예상대로 작동하는지 확인한 후 캔버스 **시작을** 선택하여 캔버스를 실행합니다. 이제 개인화된 사용자 여정을 통해 사용자를 타겟팅하여 새로운 기능인 크루즈 컨트롤을 채택하도록 유도할 수 있습니다.
