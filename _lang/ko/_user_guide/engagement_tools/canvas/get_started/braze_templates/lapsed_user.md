# 휴면 사용자

> 휴면 사용자 템플릿을 사용하여 브랜드가 사용자에게 제공하는 가치를 상기시키고, 과거 참여를 기반으로 한 흥미로운 제안과 인센티브로 재참여를 유도하세요.

이 문서에서는 사용자 라이프사이클의 리텐션 및 충성도 단계를 위해 설계된 휴면 **사용자** 템플릿의 사용 사례를 안내합니다. 완료되면 프로모션 메시지를 받은 후 앱에서 세션을 시작했는지 여부 등 사용자의 행동에 따라 달라지는 프로모션을 통해 사용자가 앱으로 돌아오도록 유도하는 캔버스를 만들 수 있습니다.

## 필수 조건

만료된 사용자 템플릿을 성공적으로 사용하려면 사용하는 파트너 및 오디언스와 [Braze]({{site.baseurl}}/partners/canvas_steps) 오디언스 [동기화를]({{site.baseurl}}/partners/canvas_steps) 구성해야 합니다.

## 필요에 맞게 템플릿 조정하기

영화 및 쇼 전용 콘텐츠를 제공하는 스트리밍 서비스인 MovieCanon에서 일한다고 가정해 보겠습니다. 휴면 사용자 템플릿을 사용하여 30일 동안 앱을 방문하지 않은 사용자를 대상으로 특전 및 프리미엄 콘텐츠를 홍보할 수 있습니다.

캔버스를 만들기 전에, 행동 트리거, 세분화 등을 기반으로 광고를 전송하기 위해 Braze의 사용자 데이터를 Google 오디언스에 추가할 수 있도록 Braze 오디언스 [동기화를 Google에]({{site.baseurl}}/partners/canvas_steps/google_audience_sync/) 통합했습니다.

경과 중인 사용자 템플릿에 액세스하려면 새 캔버스를 만들 때 **캔버스 템플릿 사용** > **브레이즈 템플릿을** 선택합니다. 그런 다음 **휴면 사용자** 옆의 **템플릿 적용을** 선택합니다. 이제 템플릿을 검토하여 필요에 맞게 조정할 수 있습니다.

### 1단계: 세부 정보 설정 

목표를 반영하도록 캔버스 세부 사항을 조정해 보겠습니다.

1. 템플릿 이름 옆의 **편집을** 선택합니다.

![캔버스의 현재 제목과 설명입니다.]({% image_buster /assets/img/canvas_templates/lapsed_user_old_name_description.png %}){: style="max-width:45%;"}

{:start="2"}
2\. 캔버스 이름을 업데이트하여 이 캔버스가 사용자에게 프로모션 메시지를 보내고 세션을 시작하는 사용자를 대상으로 대상 동기화를 수행하도록 지정합니다.
3\. 설명을 업데이트하여 이 캔버스에 특전 및 프로모션이 포함되어 있음을 설명합니다.
4\. 캔버스 홈 페이지에서 이 캔버스를 필터링할 수 있도록 **경과/유지** 태그를 추가합니다.

!["캔버스 세부 정보 설정" 단계에서 캔버스 이름 "경과된 사용자 - 앱 방문"과 간단한 캔버스 설명이 포함됩니다.][1]

### 2단계: 전환 이벤트 할당

**기본 전환 이벤트 - A를** 업데이트하여 앱(MovieCanon)의 사용자를 타겟팅하고 **기본 전환 이벤트 - B를** 모든 구매의 기본값으로 남겨둡니다.

!['전환 이벤트 할당' 섹션에서 특정 앱에서 세션을 시작하는 사용자의 기본 전환도 포함할 수 있습니다.][2]

### 3단계: 참가 일정 맞춤 설정

입력 일정을 **예약됨과** 기본 시간 기반 옵션으로 유지하여 캔버스가 매일 경과된 사용자를 확인하도록 합니다.

이 단계에서 두 가지를 조정하겠습니다: 

1. 시작 날짜와 시간을 선택합니다.
2. **특정 날짜에** 및 2개월 후 날짜의 종료 매개변수를 선택합니다. 이 캔버스 이후에 시작하려는 다른 휴면 사용자 캔버스가 있다고 가정해 보겠습니다.

!["지정된 시간에 사용자를 입장시키는 예약된 캔버스의 '입장 일정' 단계입니다.][3]

### 4단계: 타겟 고객 선택

30일 이상 앱을 사용하지 않은 사용자로 설정된 응모 대상의 기본 설정은 그대로 유지됩니다. 또한 사용자가 4주 후에 캔버스에 다시 들어갈 수 있도록 기본 입력 컨트롤을 유지합니다. 즉, 사용자가 30일 이상 연속으로 앱을 방문하지 않을 때마다 캔버스에 입력됩니다.

!['타겟 오디언스' 단계에서는 30일 동안 앱을 마지막으로 사용한 사용자를 타겟팅합니다.][4]

### 5단계: 전송 설정을 선택하세요

대부분의 기본 구독 설정은 그대로 유지됩니다:

- 메시지 또는 알림을 구독하거나 수신에 동의한 사용자에게만 전송합니다.
- 오디언스가 수신하는 메시지 수가 너무 많지 않도록 [빈도 제한 규칙을]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/#frequency-capping) 적용하세요. 이 경우, 사용자가 받을 수 있는 '경과/유지' 태그가 붙은 캠페인 또는 캔버스 단계의 수를 매주 2개로 제한하도록 빈도 상한을 설정합니다.
- 사용자 현지 시간으로 한가한 시간대(오전 12시~오전 8시)에는 메시지를 보내지 마세요.

변경할 유일한 설정은 조용한 시간대에 메시지가 트리거될 때 수행할 작업입니다. 메시지를 취소하는 대신 **다음에 가능한 시간에 보내기를** 선택하여 사용자가 프로모션을 놓치지 않도록 하세요.

!["시작 시간이 오전 12시, 종료 시간이 오전 8시로 설정된 '조용한 시간' 섹션입니다.][5]

### 6단계: 캔버스 사용자 지정

이제 템플릿 단계를 사용자 지정하여 캔버스를 만들겠습니다:

1. 30일 이상 앱을 방문하지 않은 모든 사용자에게 보낼 첫 번째 이메일을 사용자 지정하세요. 사용 사례에서는 사용자에게 오늘 앱을 방문하면 새로운 특전이 잠금 해제된다는 내용의 이메일을 맞춤 설정합니다. 

![캔버스 메시지 단계에서 사용자에게 오늘 방문하면 새로운 특전을 잠금 해제할 수 있도록 안내하는 이메일을 보냅니다.][6]

{: start="2"}
2\. **시작 세션** 경로에 대한 앱을 선택하여 "세션 시작?"이라는 작업 경로 구성 요소를 사용자 지정합니다. 

![특정 앱에서 시작된 세션의 작업 경로입니다.][7]

{: start="3"}
3\. ">1 세션" 그룹을 지난 달력 날짜에 앱을 한 번 이상 사용한 사용자로 정의하는 "세션?"이라는 의사 결정 분할 단계의 기본값을 그대로 유지합니다.
4\. ">1 세션" 그룹에 속하는 사용자에 대한 메시지 단계를 사용자 지정합니다. 사용 사례에서는 앱을 방문해 주신 사용자에게 감사의 인사를 전하고 잠금 해제된 특전을 강조합니다.
5\. 첫 번째 이메일을 받은 후 여러 세션을 진행한 사용자의 사용자 데이터를 업데이트하고 동기화할 수 있도록 광고 잠재고객 업데이트 단계에서 Google 잠재고객 동기화가 설정되어 있는지 확인합니다.
6\. [실험 경로]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/experiment_step#experiment-paths) 구성 요소의 기본값은 "A/B 테스트"로 유지합니다. 이렇게 하면 세션 수가 2개 미만인 사용자에게 두 가지 프로모션 중 하나(다음 단계에서 사용자 지정할 것)가 무작위로 전송됩니다.
7\. 실험 경로의 일부로 사용자에게 보낼 두 가지 프로모션을 사용자 지정합니다. 사용 사례에서는 3개월 구독 시 20% 프로모션을 적용하고 1개월 구독 시 10% 프로모션을 적용합니다.

![사용자의 세션 수에 따라 분기 경로가 있는 캔버스 단계.][8]{: style="max-width:70%;"}

### 7단계: 캔버스 테스트 및 실행

캔버스를 테스트하고 검토하여 예상대로 작동하는지 확인한 후 **캔버스 시작을** 선택하여 **캔버스를 실행**합니다. 이제 30일 이상 앱을 방문하지 않았고 메시징 채널을 구독한 사용자에게는 재방문을 유도하는 이메일이 발송됩니다!

{% alert tip %}
캔버스를 시작하기 전과 후에 고려해야 할 사항은 출시 전후 [체크리스트를]({{site.baseurl}}/user_guide/engagement_tools/canvas/ideas_and_strategies/pre_post_launch_checklist/#things-to-consider-before-launch) 확인하세요.
{% endalert %}
