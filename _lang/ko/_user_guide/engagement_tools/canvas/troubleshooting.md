# 캔버스 문제 해결

> 이 페이지에서는 캔버스 관련 문제를 해결하는 데 도움이 됩니다.

## 사용자가 트리거된 캔버스 단계를 받지 못한 이유는 무엇인가요?

먼저 사용자 지정 이벤트가 Braze에 전달되고 있는지 확인합니다. **애널리틱스** > **사용자 지정 이벤트 보고서로** 이동한 다음 해당 사용자 지정 이벤트와 날짜 범위를 선택합니다. 이벤트가 표시되지 않으면 이벤트가 올바르게 설정되었는지, 사용자가 올바른 작업을 수행했는지 확인하세요.

사용자 지정 이벤트가 표시되면 다음을 수행하여 추가 문제를 해결하세요:

- 사용자의 프로필 다운로드를 확인하여 이벤트를 트리거한 사용자와 그 시기를 확인합니다. 이벤트가 트리거된 경우, 이벤트가 트리거된 시점의 타임스탬프와 캔버스가 라이브된 시간을 비교합니다. 이벤트는 캔버스가 시작되기 전에 트리거되었을 수 있습니다.
- 캔버스와 타겟팅에 사용된 모든 세그먼트의 변경 로그를 검토하여 사용자 지정 이벤트가 트리거되었을 때 사용자가 세그먼트에 있었는지 확인합니다. 해당 세그먼트에 속하지 않았다면 캔버스 단계를 받지 못했을 것입니다.
- 사용자가 세분화를 통해 통제 그룹에 입력되어 결과적으로 캔버스 단계를 받지 못하게 되었는지 확인합니다.
- 예정된 지연이 있는 경우 지연 전에 사용자의 사용자 지정 이벤트가 트리거되었는지 확인합니다. 지연 전에 이벤트가 트리거되었다면 캔버스 단계를 받지 못했을 것입니다.

{% alert note %}
인앱 메시지는 REST API가 아닌 SDK를 통해 전송된 이벤트에 의해서만 트리거될 수 있습니다.
{% endalert %}

## 캔버스가 예상대로 전송되지 않는 이유는 무엇인가요?

캔버스는 견고하고 복잡하며, 캔버스를 만들 때 많은 시간과 정성을 쏟는다는 것을 잘 알고 있습니다. 따라서 캔버스가 원하는 방식으로 전송되지 않는다면 캔버스 일정, 참가 오디언스 및 참가 설정을 확인하고 [캔버스 생성]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas/) 단계를 검토하는 것이 좋습니다.

### 스케줄

- 캔버스가 [올바르게 예약되어]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas/#entry-schedule-types) 있나요?
- 정확한 날짜와 시간을 선택하셨나요?
- [실행 기반 전달]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas/?tab=action-based%20delivery#entry-schedule-types)의 경우, 캔버스를 시작한 이후 사용자가 지정된 액션을 수행했나요?

### 항목 설정

[항목 설정]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas/?tab=basics#selecting-entry-controls)은 캔버스가 전송되는 방식을 이해하는 데 중요합니다. 캔버스에 입장할 수 있는 사람의 수를 제한했는지 확인합니다.

사용자는 더 이상 메시지를 받을 자격이 없는 경우 캔버스를 종료할 수도 있습니다. 예를 들어 캔버스에 푸시 알림만 포함되어 있고 사용자가 첫 번째 단계를 수신한 후 푸시 수신을 거부하면 해당 사용자는 캔버스에서 탈퇴하게 됩니다. [다른 캔버스 단계를]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/about/) 사용하여 대체 사용자 여정을 추가하는 것을 고려하세요.

### 오디언스 세분화

타겟 고객을 위해 다음 질문을 고려하세요:

- 올바른 세그먼트를 선택하셨나요?
- 세그먼트는 어떻게 설정되나요?
- 세그먼트에 사용자가 포함되어 있는지 확인하셨나요?
- 캔버스에 입장하는 사용자 수를 제한하는 추가 필터를 추가했나요?
- 사용자가 이형 상품의 첫 번째 단계를 받을 수 있는 자격이 있나요? 예를 들어 캔버스의 첫 번째 단계가 푸시 알림이지만 입력 오디언스가 모두 푸시를 사용하도록 설정되어 있으면 메시지를 수신하는 사용자가 없습니다.

## 오디언스가 대조군과 배리언트 그룹으로 균등하게 나뉘지 않은 이유는 무엇인가요?

캔버스를 만들 때 다음 [사용 사례와](#use-case) 같이 오디언스가 대조군과 배리언트 그룹으로 균등하게 나뉠 것으로 예상했을 수 있습니다. 그 이유와 해결 방법에 대해 논의해 보겠습니다!

사용자가 가입하는 그룹은 설정에 따라 다릅니다. 이것은 대조군 또는 배리언트 그룹일 수 있습니다. 사용자가 [입력 단계에서]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas/?tab=entry%20schedule#step-12-determine-your-canvas-entry-schedule) 정의한 모든 기준에 부합하면 캔버스에 입장하게 됩니다. 캔버스를 설정할 때, 각 배리언트와 대조군에 들어갈 사용자 비율을 정의합니다.

만약 대조군이 배리언트 그룹에 비해 크다면 (그리고 이것이 본인의 의도가 아니라면) 다음을 권장합니다.
1. 항목 오디언스 필터를 **포그라운드 푸시 사용으로** 설정합니다.
2. **푸시 구독 상태**, **이메일 구독 상태** 또는 둘 다에 대한 항목 오디언스 필터를 **옵트인** 또는 **구독한** 상태로 설정합니다.

대조군으로 캔버스를 만들 때는 참가 오디언스의 모든 사용자가 캔버스 내에서 메시지를 받을 수 있는지 확인합니다(예: 캔버스에 푸시 및 이메일 메시지가 포함되어 있는 경우).

### Use case

다음 시나리오를 상상해 봅시다:
- 캔버스는 하나의 배리언트와 대조군을 가지고 있습니다.
- 배리언트의 첫 번째 단계는 푸시 알림입니다.
- 사용자의 90%는 배리언트에 들어가도록 선택되었고, 10%는 대조군에 들어가도록 선택되었습니다.

![90% 배리언트와 10% 대조군이 있는 캔버스 예시.]({% image_buster /assets/img_archive/trouble15.png %})

이 시나리오에서 캔버스에 들어가는 사용자의 90%가 배리언트에 들어갈 것입니다. 

활성 사용자를 다시 살펴보면 29.8만 명의 사용자가 포함되어 있지만 이 중 64%만이 푸시 인에이블먼트가 활성화되어 있음을 알 수 있습니다:

!["푸시 인에이블먼트" 필터가 "true"로 설정된 세그먼트, 예상 사용자 수는 29.8만 명입니다.]({% image_buster /assets/img_archive/trouble16.png %})

이는 사용자의 90%가 배리언트에 들어가도록 지정했더라도 실제로 모든 사용자가 푸시 알림을 받을 수 있는 것은 아니라는 것을 의미합니다. 푸시 알림을 받을 수 없는 이러한 사용자는 여전히 배리언트에 들어갑니다.
