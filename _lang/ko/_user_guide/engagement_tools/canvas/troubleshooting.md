# 캔버스 문제 해결

> 이 페이지는 캔버스 관련 문제 해결에 도움이 됩니다.

## 사용자가 트리거된 캔버스 단계를 받지 못한 이유는 무엇인가요?

먼저 커스텀 이벤트가 Braze에 전달되고 있는지 확인합니다. **분석** > **커스텀 이벤트 보고서로** 이동한 다음 해당 커스텀 이벤트와 날짜 범위를 선택합니다. 이벤트가 표시되지 않으면 이벤트가 올바르게 설정되었는지, 사용자가 올바른 작업을 수행했는지 확인하세요.

커스텀 이벤트가 표시되면 다음을 수행하여 추가 문제 해결을 시도합니다:

- 사용자의 프로필 다운로드를 확인하여 이벤트를 트리거한 사용자와 이벤트를 수행한 시기를 확인합니다. 이벤트가 트리거된 경우 이벤트가 트리거된 시점의 타임스탬프와 캔버스가 유지된 시간을 비교합니다. 이벤트는 캔버스가 활성화되기 전에 트리거되었을 수 있습니다.
- 캔버스와 타겟팅에 사용된 모든 세그먼트에 대한 체인지로그를 검토하여 사용자가 커스텀 이벤트가 트리거되었을 때 해당 세그먼트에 있었는지 확인합니다. 해당 세그먼트에 속하지 않았다면 캔버스 단계를 받지 못했을 것입니다.
- 사용자가 세분화를 통해 대조군에 입력되어 결과적으로 캔버스 단계를 받지 못했는지 확인합니다.
- 예정된 지연이 있는 경우 지연 전에 사용자의 커스텀 이벤트가 트리거되었는지 확인하세요. 이벤트가 지연 전에 트리거되었다면 캔버스 단계를 받지 못했을 것입니다.

{% alert note %}
인앱 메시지는 REST API가 아닌 소프트웨어 개발 키트를 통해 전송된 이벤트에 의해서만 트리거될 수 있습니다.
{% endalert %}
