# 에이전트 설계

> 이 문서에서는 오디언스 정의, 차원 선택, Go 관련 기능 및 제한 사항 이해 등 Decisioning Studio Go 에이전트를 디자인하는 방법에 대해 설명합니다.

성공 측정기준, 차원, 작업 은행 및 제약 조건 등 의사 결정 에이전트에 대한 기본 개념은 [의사 결정 에이전트 디자인하기를]({{site.baseurl}}/user_guide/brazeai/decisioning_studio/getting_started/designing_decisioning_agents/) 참조하세요.

## Go와 Pro 기능 비교

디시전킹 스튜디오 고는 디시전킹 스튜디오 프로에 비해 간소화된 기능을 갖춘 셀프 서비스 플랫폼입니다. 이러한 차이점을 이해하면 Go의 범위 내에서 효과적인 에이전트를 설계하는 데 도움이 됩니다.

| 기능 | 의사 결정 스튜디오 Go | 디시전 스튜디오 프로 |
|-----------|----------------------|------------------------|
| **성공 측정기준** | 클릭만 | 모든 비즈니스 측정기준(매출, 전환 또는 ARPU) |
| **치수** | 제한된 액션 뱅크 | 무제한 크기 |
| **지원되는 CEP** | Braze, SFMC, 클라비요 | 모든 CEP(기본 및 커스텀) |
| **고객 데이터** | 참여만 가능 | 모든 1P 데이터 |
| **설정** | 셀프 서비스 | AI 의사 결정 서비스 지원 |
| **실험 그룹** | 이동 + 무작위 제어 + 선택적 BAU | 완벽한 사용자 지정 가능 |

## Go 에이전트 설계

의사 결정 스튜디오 Go 상담원을 디자인할 때는 다음 영역에서 의사 결정을 내리게 됩니다:

### 1단계: 대상을 정의하십시오.

오디언스는 상담원이 참여하게 될 고객 집합입니다. Go에서 오디언스는 CEP에 정의되어 있습니다:

{% tabs %}
{% tab Braze %}

**Braze의 오디언스 정의:**

1. 상담원이 타겟팅할 고객을 정의하는 세그먼트를 Braze에서 만드세요.
2. Decisioning Studio Go 포털에서 실험자를 구성할 때 이 세그먼트를 타겟팅 오디언스로 선택합니다.

{% alert tip %}
테스트를 격리하고 측정 가능한 상태로 유지하려면 Decisioning Studio Go 실험자를 위한 전용 세그먼트를 만드는 것을 고려하세요.
{% endalert %}

{% endtab %}
{% tab Salesforce Marketing Cloud %}

**SFMC의 오디언스 정의:**

1. 타겟 오디언스가 포함된 데이터 확장을 구성합니다.
2. 이 데이터 확장이 최신 고객 데이터로 매일 새로고침되는지 확인하세요.
3. 실험자를 구성할 때 Decisioning Studio Go 포털에서 이 데이터 확장을 참조하세요.

{% endtab %}
{% tab Klaviyo %}

**클라비요에서 오디언스 정의하기:**

1. 클라비요에서 타겟 오디언스를 정의하는 세그먼트를 생성하세요.
2. Decisioning Studio Go 포털에서 실험자를 구성할 때 이 세그먼트를 선택합니다.

{% endtab %}
{% endtabs %}

### 2단계: 차원 선택

차원은 상담원이 고객 경험을 개인화하기 위해 사용할 수 있는 '지렛대'입니다. 여기에는 제목란, 영웅 이미지와 같은 크리에이티브 차원과 이메일 발송 빈도 또는 시간대와 같은 전송 유형 차원이 포함됩니다. 

{% alert note %}
사용 가능한 구체적인 크기는 CEP와 캠페인 구성 방식에 따라 다릅니다. CEP에서 설정한 템플릿과 콘텐츠로 작업하세요.
{% endalert %}

### 3단계: 액션 뱅크 구성

작업 은행은 상담원이 각 차원에 대해 선택할 수 있는 특정 옵션을 정의합니다. For example:

- **이메일 템플릿**: 상담원이 사용할 수 있는 템플릿을 선택합니다(먼저 CEP에서 구성해야 함).
- **제목란**: 상담원이 테스트할 수 있는 제목란 배리언트 정의하기
- **보내기 시간**: 상담원이 선택할 수 있는 시간 창을 지정합니다.

### 4단계: 실험 그룹 설정

Decisioning Studio Go는 성능/성과를 측정할 실험 그룹을 자동으로 생성합니다:

| 그룹 | 설명 |
|-------|-------------|
| **의사 결정 스튜디오 Go** | AI에 최적화된 커스텀 추천을 받는 고객 |
| **무작위 제어** | 무작위로 선택된 옵션을 받는 고객(기준 비교) |
| **평상시처럼 업무 처리(선택 사항)** | 기존 캠페인을 수신하는 고객(현재 성능/성과와 비교하는 경우) |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

{% alert important %}
정확한 비교를 위해 고객이 두 개 이상의 실험 그룹에 속할 수 없도록 하고 고객이 편견 없이 무작위로 그룹에 배정되도록 합니다.
{% endalert %}

## 고려해야 할 제한 사항

Go 에이전트를 설계할 때는 이러한 제한 사항을 염두에 두세요:

- **클릭만** 가능합니다: Go는 클릭률에 최적화되어 있습니다. 매출, 전환 또는 기타 비즈니스 측정기준에 맞게 최적화해야 하는 경우 Decisioning Studio Pro를 고려하세요.
- **제한된 치수**: Go는 미리 정의된 차원 집합을 지원합니다. 커스텀 차원 또는 복잡한 개인화가 필요한 경우 Decisioning Studio Pro를 고려하세요.
- **세 명의 CEP**: Go는 Braze, Salesforce 마케팅 클라우드 및 클라비요와만 통합됩니다. 다른 플랫폼의 경우 Decisioning Studio Pro를 고려해 보세요.

## Best practices

- **간단하게 시작하세요**: 2~3개의 템플릿 또는 제목란 배리언트로 시작하세요. 이렇게 하면 상담원이 실험을 관리하기 쉽게 유지하면서 학습할 수 있는 충분한 옵션이 제공됩니다.
- **시간을 주세요**: 상담원이 학습하려면 충분한 데이터가 필요합니다. 성능/성과에 대한 결론을 내리기 전에 최소 2~4주 정도 기다리세요.
- **콘텐츠를 다양하게 유지하세요**: 옵션이 의미 있게 다른지 확인하세요. 사소한 변형을 테스트해서는 유의미한 인사이트를 얻지 못할 수도 있습니다.
- **정기적으로 모니터링합니다**: 실험 진행 상황 및 참여 측정기준을 모니터링하려면 Decisioning Studio Go 포털을 확인하세요.
