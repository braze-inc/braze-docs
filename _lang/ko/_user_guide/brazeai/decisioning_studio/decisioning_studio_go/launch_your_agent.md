# 에이전트를 시작하세요

> 데이터 소스를 연결하고 오케스트레이션을 설정하며 에이전트를 설계한 후, 시작할 준비가 되었습니다. 이 문서에서는 에이전트를 활성화하고 선택적 BAU 보고서를 설정하는 방법을 다룹니다.

## 에이전트를 시작합니다

Decisioning Studio Go 포털에서 모든 구성 단계를 완료한 후:

1. 모든 설정이 올바른지 확인하기 위해 에이전트 구성을 검토하세요.
2. CEP 통합이 활성화되어 있고 오케스트레이션이 준비되었는지 확인하세요.
3. Decisioning Studio Go 포털에서 **시작** (또는 동등한 작업)을 선택하여 에이전트를 활성화하세요.

시작되면, 에이전트는:
- CEP로부터 오디언스 데이터를 수신하기 시작합니다.
- 각 고객에 대한 개인화된 추천을 시작합니다.
- 구성된 CEP를 통해 오케스트레이션을 전송합니다.
- 참여 데이터를 수집하여 시간이 지남에 따라 학습하고 개선합니다.

## BAU 보고서 설정하기

기본적으로, Decisioning Studio Go 포털 보고서는 Decisioning Studio Go 그룹을 무작위 대조군과 비교합니다. 비교하고 싶은 기존 비즈니스 일상(BAU) 캠페인이 있는 경우, BAU 보고서를 설정하여 세 그룹을 한 곳에서 볼 수 있습니다.

### BAU 보고서의 이점

BAU 보고서를 설정하는 주요 이점은 Decisioning Studio Go의 잘못된 클릭 필터링을 적용하는 것입니다. 세 실험 그룹 모두에 적용될 때, 이는 다음의 노이즈를 제거하여 가장 정확하고 공정한 ("사과와 사과") 클릭 성능 비교를 가능하게 합니다:
- 의심되는 기계 클릭
- 구독 취소 링크 클릭

### BAU 보고를 위한 요구 사항

BAU 보고를 설정하기 전에 BAU 처리 그룹, Decisioning Studio Go 그룹 및 무작위 대조군 간의 동등한 비교를 보장하십시오:

- **중복 없음**: 어떤 수신자도 실험 기간 동안 하나의 그룹에만 속할 수 있습니다
- **무작위 배정**: 수신자는 편향 없이 그룹에 무작위로 배정됩니다
- **동등한 옵션**: BAU 그룹에 제공되는 모든 옵션(크리에이티브, 빈도, 시간, 인센티브 또는 제안)은 Decisioning Studio Go 및 무작위 대조군에도 제공됩니다

{% alert warning %}
"동등한 실험 설계"가 없으면 BAU 보고가 혼란스럽거나 오해의 소지가 있을 수 있습니다.
{% endalert %}

### 필요한 정보

실험 설계를 검증한 후 BAU 보고를 설정하기 위해 다음 세부 정보를 수집하십시오:

**CEP에서 캠페인 ID:**

| CEP | 허용된 유형 |
|-----|---------------|
| **Braze** | 캠페인 및 캔버스 |
| **Salesforce Marketing Cloud** | 여정만 |
| **Klaviyo** | 흐름만 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

**CEP에서 오디언스 ID:**

| CEP | 허용된 유형 |
|-----|---------------|
| **Braze** | 세그먼트만 |
| **Salesforce Marketing Cloud** | 데이터 확장만 |
| **Klaviyo** | 세그먼트만 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

기존의 BAU 오디언스를 추적하는 오디언스가 없다면, 하나를 생성해야 합니다.

### 고려 사항

- **클릭 KPI만**: 결정 스튜디오와 유사하게, BAU 보고서는 클릭 KPI만 포함하고 전환 KPI는 포함하지 않습니다.
- **캔버스 제한**: 현재 특정 캔버스 단계 ID로 필터링하는 것을 지원하지 않습니다. 모든 캔버스 단계의 이벤트가 BAU 데이터에 포함됩니다. 특정 캔버스 단계만 포함해야 하는 경우 BAU와의 비교가 무효화될 수 있습니다.

### BAU 보고서 설정하기

결정 스튜디오 포털의 지침을 따르십시오. 다음이 필요합니다:
- 모든 커뮤니케이션이 BAU 커뮤니케이션인 하나 이상의 캠페인 ID
- 매일 BAU 오디언스의 수신자를 추적하는 하나의 오디언스 ID

## 에이전트 모니터링

출시 후, 결정 스튜디오 포털에서 에이전트의 성능을 모니터링하십시오:

- **참여 측정기준**: 실험 그룹 간 클릭 비율 추적
- **학습 진행 상황**: 에이전트의 추천이 시간이 지남에 따라 어떻게 발전하는지 관찰하십시오.
- **그룹 비교**: 결정 스튜디오 Go 성능을 무작위 대조군 및 BAU(구성된 경우)와 비교하십시오.

{% alert tip %}
성능에 대한 결론을 내리기 전에 최소 2-4주간의 데이터 수집을 허용하십시오. 에이전트는 학습하고 효과적으로 최적화하기 위해 충분한 상호작용이 필요합니다.
{% endalert %}

## 문제 해결

에이전트가 예상대로 작동하지 않는 경우:

1. **오케스트레이션 확인**: CEP 통합이 활성화되어 있고, 캠페인과 여정이 실행 중이며, 전역 한도나 유사한 규칙이 오케스트레이션에 간섭하지 않는지 확인하십시오.
2. **데이터 흐름 확인**: 오디언스 데이터와 참여 데이터가 올바르게 수집되고 있는지 확인하십시오.
3. **실험 그룹 검토**: 적절한 무작위 할당과 그룹 간의 중복이 없도록 하십시오.
4. **지원에 문의**: 추가 지원을 위해 Braze 지원팀에 문의하십시오.
