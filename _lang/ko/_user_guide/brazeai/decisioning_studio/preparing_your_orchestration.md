# 오케스트레이션 준비하기

> 이 참조 문서에서는 고객 참여 플랫폼(CEP)을 선택하고 필요한 자격 증명 및 자산을 수집하는 등 BrazeAI Decisioning Studio™의 오케스트레이션을 설정하기 전에 준비해야 할 사항에 대해 설명합니다.

## 오케스트레이션이란 무엇인가요?

오케스트레이션은 Decisioning Studio와 고객 참여 플랫폼(CEP)을 연결하는 역할을 합니다. 의사 결정 에이전트가 각 고객에 대한 최적의 조치를 결정하면 오케스트레이션은 CEP를 통해 개인화된 커뮤니케이션을 트리거하여 이러한 결정을 실행합니다.

이렇게 생각하세요:

- **디시전 스튜디오에서** 전송 *대상과* 전송 *시기를* 결정합니다.
- **CEP가** 전송 *방법을* 처리합니다.

## CEP 선택

첫 번째 단계는 Decisioning Studio와 함께 사용할 고객 참여 플랫폼을 결정하는 것입니다. 선택에 따라 설정 복잡성과 사용 가능한 기능이 달라집니다.

### 지원되는 CEP

| CEP | 의사 결정 스튜디오 Go | 디시전 스튜디오 프로 | 통합 유형 |
|-----|:---------------------:|:----------------------:|------------------|
| **Braze** | ✓ | ✓ | 네이티브 API 통합(권장) |
| **Salesforce 마케팅 클라우드** | ✓ | ✓ | API 이벤트 + 여정 빌더 |
| **클라비요** | ✓ | ✓ | API 이벤트 + 플로우 |
| **기타 CEP** | — | ✓ | 커스텀(추천 파일) |
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3 .reset-td-br-4 role="presentation"}

{% alert tip %}
이미 Braze를 CEP로 사용 중인 경우 가장 원활한 설정 환경을 위해 기본 Braze 통합 기능을 사용하는 것이 좋습니다.
{% endalert %}

## 준비해야 할 사항

오케스트레이션을 설정하기 전에 선택한 CEP에 따라 다음 항목을 수집하세요.

{% tabs %}
{% tab Braze %}

| 항목 | 설명 |
|------|-------------|
| **REST API key** | 사용자 데이터, 메시지, 캠페인, 캔버스, 세그먼트 및 템플릿에 대한 권한이 있는 새로운 API 키입니다. |
| **Braze 대시보드 URL** | Braze 인스턴스 URL(예: `https://dashboard-01.braze.com`). |
| **앱 ID** | 추적하려는 앱과 연결된 API 키( **설정** > **앱 설정에서** 찾을 수 있음). |
| **이메일 표시 이름 및 주소** | 캠페인에 사용할 발신자 정보( **설정** > **이메일 기본** 설정에서 찾을 수 있음). |
| **기본 템플릿** | 상담원이 오케스트레이션에 사용할 메시지 템플릿입니다. 각 템플릿에 대해 API 트리거 캠페인을 만들 수 있습니다. |
| **테스트 사용자 ID** | 출시 전 통합 테스트를 위한 사용자 ID입니다. |

## 캠페인 계획하기

오케스트레이션을 설정하기 전에 다음 세부 사항을 고려하세요:

### 기본 템플릿

기본 템플릿은 의사 결정 에이전트가 사용할 수 있는 모든 메시지 템플릿입니다. 생각해 보세요:

- **템플릿은 몇 개인가요?** 상담원은 하나의 템플릿 또는 여러 개의 템플릿으로 작업할 수 있습니다. 여러 명일 경우 상담원은 각 고객이 받을 템플릿을 개인화할 수 있습니다.
- **어떤 채널이 있나요?** 이메일, 푸시, SMS 또는 이들의 조합. 각 채널에는 별도의 템플릿과 캠페인이 필요할 수 있습니다.
- **어떤 동적 요소가 있나요?** 상담원이 개인화할 메시징의 어떤 부분(제목란, CTA, 오퍼, 타이밍 등)을 식별할지 정하세요. 이는 API 트리거 속성 또는 동적 입력 안내자가 됩니다.

### 재자격 설정

캠페인을 통해 사용자가 여러 번 메시지를 받을 수 있도록 해야 합니다:

- 테스트를 위해 동일한 사용자에게 동일한 캠페인을 반복해서 보낼 수 있습니다.
- 프로덕션에서 상담원은 동일한 캠페인이 연속된 날에 사용자에게 최적이라고 판단할 수 있습니다.

{% alert note %}
테스트 재참가 자격을 설정할 때, Decisioning Studio 에이전트는 최대 게재빈도 제한을 준수하도록 설계되었으며 프로덕션 환경에서 하루에 한 번 이상 동일한 캠페인을 사용자에게 보내지 않습니다.
{% endalert %}

### API 트리거 속성

Braze 통합의 경우 상담원이 어떤 차원을 최적화할지 계획하세요. 이는 캠페인에 동적 값을 전달하는 API 트리거 속성이 됩니다:

| 차원 예시 | API 트리거 속성 |
|-------------------|---------------------|
| Subject line | {% raw %}`{{api_trigger_properties.${subject_line}}}`{% endraw %} |
| 클릭 유도 문안 | {% raw %}`{{api_trigger_properties.${cta_message}}}`{% endraw %} |
| 제안 | {% raw %}`{{api_trigger_properties.${offer_id}}}`{% endraw %} |
| 할인 금액 | {% raw %}`{{api_trigger_properties.${discount}}}`{% endraw %} |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

## Best practices

오케스트레이션을 준비할 때 이러한 모범 사례를 염두에 두세요:

1. **간단하게 시작하세요.** 하나의 채널과 하나 또는 두 개의 템플릿으로 시작하세요. 나중에 어떤 것이 효과적인지 알게 되면 확장할 수 있습니다.
2. **철저히 테스트하세요.** 시작하기 전에 소수의 사용자를 대상으로 통합을 테스트하여 동적 콘텐츠가 올바르게 채워지는지 확인하세요.
3. **설정을 문서화하세요.** 캠페인 ID, 템플릿 ID, API 키 및 기타 식별자를 추적하세요. Decisioning Studio 포털에서 이를 참조해야 합니다.
4. **팀과 조율하세요.** 오케스트레이션 설정에는 마케팅, 엔지니어링, 데이터 팀이 참여할 수 있습니다. 모든 사람이 프로세스에서 자신의 역할을 이해하도록 하세요.
5. **피드백 데이터를 계획하세요.** 오케스트레이션은 단순히 메시지를 보내는 것뿐만 아니라 상담원의 학습에 도움이 되는 참여 및 전환 데이터를 수집하는 것이기도 합니다. 자세한 내용은 [데이터 소스 준비하기를]({{site.baseurl}}/user_guide/brazeai/decisioning_studio/preparing_your_data_sources/) 참조하세요.
