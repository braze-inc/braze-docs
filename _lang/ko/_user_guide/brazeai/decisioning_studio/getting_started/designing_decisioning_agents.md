# 결정 에이전트 설계

> 이 참조 문서는 결정 에이전트를 설계하고 구성하기 위한 주요 개념과 모범 사례를 다룹니다.

## 결정 에이전트에 대하여

결정 에이전트를 설계하는 것은 Decisioning Studio를 설정하는 첫 번째 단계입니다. 결정 에이전트가 결정을 내릴 수 있도록 하려면, 극대화하고자 하는 결과와 에이전트가 이를 위해 취할 수 있는 행동을 정의해야 합니다.

### 주요 개념

다음 용어는 Decisioning Studio 가이드 전반에 걸쳐 참조됩니다.

| 용어 | 정의 |
| --- | --- |
| **결정 에이전트** | 결정 에이전트는 특정 비즈니스 목표를 충족하기 위해 맞춤 구성된 BrazeAI Decisioning Studio™의 커스텀 구성입니다. 이는 성공 측정기준, 차원 및 선택한 옵션에 의해 정의됩니다. |
| **성공 측정기준** | 매출, 전환 또는 사용자당 평균 매출(ARPU)과 같이 최적화하려는 특정 비즈니스 측정기준입니다. 이는 결정 에이전트가 행동을 통해 극대화하려고 하는 측정기준입니다. |
| **차원** | 차원은 결정 에이전트가 성공 측정기준을 극대화하기 위해 조작할 수 있는 *레버의 유형*으로 생각할 수 있습니다. 일반적인 차원에는 오퍼, 제목란, 크리에이티브, 채널 또는 발송 시간이 포함됩니다. |
| **액션 뱅크** | 액션 뱅크는 각 차원 "레버"에 대해 결정 에이전트가 접근할 수 있는 *특정 옵션*을 정의합니다. 예를 들어, 채널 차원의 경우 결정 에이전트가 접근할 수 있는 특정 채널을 정의합니다. 오퍼 차원의 경우 결정 에이전트가 테스트할 수 있는 특정 오퍼를 정의합니다. |
| **제약 조건** | 일반적으로 결정 에이전트는 액션 뱅크에 넣은 모든 조합의 행동을 취할 수 있습니다. 그러나 결정 에이전트의 행동을 중요한 비즈니스 규칙에 맞게 제한하는 제약 조건을 정의할 수도 있습니다. 예를 들어, 특정 오퍼가 자격이 없는 지역의 고객에게 선택되지 않도록 하거나, 결정 에이전트가 지출할 최대 예산을 설정하는 것일 수 있습니다. |

## 결정 에이전트를 설계하는 방법

결정 에이전트를 설정할 때, 네 가지 주요 설계 요소를 고려해야 합니다:

### "목표": 성공 측정기준 정의하기

> 에이전트가 극대화하고자 하는 결과는 무엇인가요?

성공 측정기준은 에이전트가 최적화할 비즈니스 결과입니다. 이는 클릭이나 오픈과 같은 대리 측정기준이 아니라 매출, 전환, ARPU 또는 고객 생애주기 가치와 같은 실제 비즈니스 결과와 직접적으로 일치해야 합니다.

### "누구": 오디언스 선택하기

> 결정 에이전트가 참여시킬 대상은 누구인가요?

에이전트가 서비스를 제공할 오디언스를 정의하세요. 이는 모든 고객, 특정 세그먼트(예: 로열티 프로그램 회원) 또는 생애 주기의 특정 단계에 있는 고객(예: 최근 구매자 또는 이탈 위험이 있는 가입자)일 수 있습니다.

### "무엇": 액션 뱅크 구성하기

> 결과를 이끌어내기 위해 에이전트가 선택할 수 있는 옵션은 무엇인가요?

액션 뱅크는 에이전트가 조작할 수 있는 모든 레버를 정의합니다—차원(예: 채널, 오퍼, 타이밍 및 빈도)과 각 차원 내의 특정 옵션입니다. 에이전트는 이러한 옵션의 다양한 조합을 실험하여 각 고객에게 가장 적합한 것을 찾습니다.

### "방법": 제약 조건 구성하기

> 에이전트가 따라야 할 규칙은 무엇인가요?

제약 조건은 에이전트가 따라야 하는 규칙입니다. 이는 특정 오퍼가 자격이 없는 지역의 고객에게 선택되지 않도록 하거나, 결정 에이전트가 지출할 최대 예산을 설정하는 것일 수 있습니다.

## 모범 사례 및 예시

결정 에이전트의 효과를 극대화하려면 다음을 수행해야 합니다:

- 매출, 전환 또는 ARPU와 같이 비즈니스 목표 및 목적과 밀접하게 일치하는 성공 측정기준을 선택하세요.
- 오퍼, 제목란, 크리에이티브, 채널 또는 발송 시간과 같이 성공 측정기준에 상당한 영향을 미칠 가능성이 가장 높은 차원 또는 "레버"에 집중하세요.
- 이메일 대 SMS 또는 일일 대 주간 빈도와 같이 성공 측정기준에 상당한 영향을 미칠 가능성이 가장 높은 각 차원의 옵션을 선택하세요.

구축할 수 있는 결정 에이전트의 몇 가지 예는 다음과 같습니다:

{% tabs %}
{% tab Repeat purchase agent %}
초기 판매 후 후속 전환을 증가시키기 위해 반복 구매 에이전트를 구축할 수 있습니다:

- Braze에서 오디언스와 메시지를 정의하세요
- Decisioning Studio가 매일 자동으로 실험을 실행하여 각 고객에 대한 제품 오퍼, 메시지 타이밍 및 빈도의 다양한 조합을 테스트합니다
- 시간이 지남에 따라 BrazeAI<sup>TM</sup>가 각 고객에게 가장 잘 작동하는 것을 학습합니다
- 재구매율을 극대화하기 위해 Braze를 통해 개인화된 발송을 오케스트레이션합니다
{% endtab %}
{% tab Cross-sell or upsell agent %}
인터넷 구독에서 사용자당 평균 매출(ARPU)을 극대화하기 위해 교차 판매 또는 업셀 에이전트를 구축할 수 있습니다:

- Braze에서 오디언스와 메시지를 정의하세요
- Decisioning Studio가 매일 자동으로 실험을 실행하여 각 고객에 대한 메시지, 발송 시간, 할인 및 플랜 오퍼의 다양한 조합을 테스트합니다
- BrazeAI<sup>TM</sup>가 어떤 고객이 리프로그 오퍼에 반응하는지, 어떤 고객이 업그레이드를 위해 할인이나 기타 인센티브가 필요한지를 학습합니다
- ARPU를 극대화하기 위해 Braze를 통해 개인화된 발송을 오케스트레이션합니다
{% endtab %}
{% tab Renewal and retention agent %}
계약 갱신을 확보하고 계약 기간과 순현재가치(NPV)를 극대화하기 위해 갱신 및 유지 에이전트를 구축할 수 있습니다:

- Braze에서 오디언스와 메시지를 정의하세요
- Decisioning Studio가 매일 자동으로 실험을 실행하여 각 고객에 대한 다양한 갱신 오퍼를 테스트합니다
- BrazeAI<sup>TM</sup>가 가격에 덜 민감하고 갱신을 위해 큰 할인이 필요하지 않은 고객을 식별합니다
- Braze를 통해 개인화된 발송을 오케스트레이션하여 계약 갱신 및 NPV를 극대화합니다
{% endtab %}
{% tab Winback agent %}
과거 가입자가 재가입하도록 유도하여 재활성화를 증가시키는 윈백 에이전트를 구축할 수 있습니다:

- Braze에서 오디언스와 메시지를 정의하세요
- Decisioning Studio가 매일 자동으로 실험을 실행하여 크리에이티브, 메시지, 채널 및 주기를 포함한 수천 개의 변수를 한 번에 테스트합니다
- BrazeAI<sup>TM</sup>가 각 개별 고객에게 가장 적합한 조합을 발견합니다
- Braze를 통해 개인화된 발송을 오케스트레이션하여 재활성화율을 극대화합니다
{% endtab %}
{% tab Referral agent %}
기존 고객의 비즈니스 신용카드 추천을 통해 신규 계정 개설을 극대화하는 추천 에이전트를 구축할 수 있습니다:

- Braze에서 오디언스와 메시지를 정의하세요
- Decisioning Studio가 매일 자동으로 실험을 실행하여 각 고객에 대한 다양한 이메일, 크리에이티브, 발송 시간 및 신용카드 오퍼를 테스트합니다
- BrazeAI<sup>TM</sup>가 특정 고객을 위한 이상적인 조합을 결정합니다
- Braze를 통해 개인화된 발송을 오케스트레이션하여 추천 전환을 극대화합니다
{% endtab %}
{% tab Lead nurturing and conversion agent %}
증분 매출을 창출하고 각 고객에 대해 적절한 비용을 지불하기 위해 리드 육성 및 전환 에이전트를 구축할 수 있습니다:

- Braze에서 오디언스와 메시지를 정의하세요
- Decisioning Studio가 매일 자동으로 실험을 실행하여 다양한 고객 세그먼트, 입찰 방법론, 입찰 수준 및 크리에이티브를 테스트합니다
- BrazeAI<sup>TM</sup>가 개인정보 보호 정책이 변경됨에 따라 유료 광고 성과를 최적화하기 위해 강력한 퍼스트파티 데이터를 활용합니다
- Braze를 통해 개인화된 발송을 오케스트레이션하여 고객당 비용을 최적화하면서 매출을 극대화합니다
{% endtab %}
{% tab Loyalty and engagement agent %}
고객 로열티 프로그램에 새로 등록한 회원의 구매를 극대화하기 위해 로열티 및 참여 에이전트를 구축할 수 있습니다:

- Braze에서 오디언스와 메시지를 정의하세요
- Decisioning Studio가 매일 자동으로 실험을 실행하여 각 고객에 대한 다양한 이메일 오퍼, 발송 시간 및 빈도를 테스트합니다
- BrazeAI<sup>TM</sup>가 로열티 프로그램에 새로 등록한 각 회원에게 가장 효과적인 방법을 학습합니다
- Braze를 통해 개인화된 발송을 오케스트레이션하여 구매 및 재구매율을 극대화합니다
