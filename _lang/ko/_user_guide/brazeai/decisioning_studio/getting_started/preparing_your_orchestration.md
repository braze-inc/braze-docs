# 오케스트레이션 준비

> 이 참조 문서는 BrazeAI Decisioning Studio™ 오케스트레이션 설정 전에 준비해야 할 사항을 설명합니다. 여기에는 고객 참여 플랫폼(CEP) 선택과 필요한 자격 증명 및 자산 수집이 포함됩니다.

## 오케스트레이션이란 무엇인가?

오케스트레이션은 Decisioning Studio와 고객 참여 플랫폼(CEP) 간의 연결을 의미합니다. 결정 에이전트가 각 고객에게 대한 최적의 조치를 결정하면, 오케스트레이션은 CEP를 통해 개인화된 커뮤니케이션을 트리거함으로써 해당 결정을 실행합니다.

이렇게 생각해 보세요:

- **디시전닝 스튜디오는** *무엇을* *언제* 보낼지 결정합니다
- **귀하의 CEP가** 전송 *방식을* 처리합니다

## CEP 선택하기

첫 번째 단계는 Decisioning Studio와 함께 사용할 고객 참여 플랫폼을 결정하는 것입니다. 선택 사항은 설정 복잡성과 사용 가능한 기능에 영향을 미칩니다.

### 지원되는 CEP

| CEP | 결정 스튜디오 고 | 디시전닝 스튜디오 프로 | 통합 유형 |
|-----|:---------------------:|:----------------------:|------------------|
| **Braze** | ✓ | ✓ | 네이티브 API 통합 (권장) |
| **세일즈포스 마케팅 클라우드** | ✓ | ✓ | API 이벤트 + 여정 빌더 |
| **클라비요** | ✓ | ✓ | API 이벤트 + 플로우 |
| **기타 CEP** | — | ✓ | 커스텀 (추천 파일) |
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3 .reset-td-br-4 role="presentation"}

{% alert tip %}
이미 Braze를 CEP로 사용 중이라면, 가장 원활한 설정 경험을 위해 Braze의 네이티브 통합을 사용하는 것을 권장합니다.
{% endalert %}

오케스트레이션을 설정하기 전에 선택한 CEP에 따라 다음 항목들을 수집하십시오.

{% tabs %}
{% tab Braze %}

| 항목 | 설명 |
|------|-------------|
| **REST API key** | 사용자 데이터, 메시지, 캠페인, 캔버스, 세그먼트 및 템플릿에 대한 권한이 부여된 새 API 키. |
| **Braze 대시보드 URL** | 귀하의 Braze 인스턴스 URL (예: `https://dashboard-01.braze.com`). |
| **앱 ID** | 추적하려는 앱과 연결된 API 키(**설정** > **앱 설정**에서 확인 가능). |
| **이메일 표시 이름 및 주소** | 캠페인에 사용할 발신자 정보(**설정** > **이메일 환경설정**에서 확인 가능). |

## 캠페인 계획 수립

오케스트레이션을 설정하기 전에 다음 세부 사항을 고려하십시오:

### 기본 템플릿

기본 템플릿은 의사 결정 에이전트가 사용할 수 있는 모든 메시지 템플릿을 의미합니다. 고려해 보십시오:

- **템플릿은 몇 개입니까?** 귀하의 에이전트는 하나의 템플릿 또는 여러 템플릿을 사용할 수 있습니다. 여러 개일 경우, 에이전트는 각 고객에게 개인화된 템플릿을 제공할 수 있습니다.
- **어떤 채널들?** 이메일, 푸시 알림, SMS 또는 이들의 조합. 각 채널마다 별도의 템플릿과 캠페인이 필요할 수 있습니다.
- **어떤 동적 요소들?** 에이전트가 메시지의 어느 부분을 개인화할지 식별하십시오(제목란, CTA, 제안, 시기 등). 이들은 API 트리거 속성 또는 동적 입력 안내가 될 것입니다.

### 재자격 설정

캠페인은 사용자가 메시지를 여러 번 수신할 수 있도록 해야 합니다:

- 테스트를 위해 동일한 캠페인을 동일한 사용자에게 반복적으로 발송해야 합니다.
- 생산 환경에서 에이전트는 동일한 캠페인이 연속된 날짜에 사용자에게 최적이라고 판단할 수 있습니다.

{% alert note %}
테스트 재적격성 설정 시, Decisioning Studio 에이전트는 최대 게재빈도 설정을 준수하도록 설계되어 운영 환경에서 동일한 캠페인을 사용자에게 하루에 한 번 이상 전송하지 않습니다.
{% endalert %}

### API 트리거 속성

Braze 통합을 위해 에이전트가 최적화할 차원을 계획하십시오. 이러한 속성은 API 트리거 속성이 되어 캠페인에 동적 값을 전달합니다:

| 예시 차원 | API 트리거 속성 |
|-------------------|---------------------|
| Subject line | {% raw %}`{{api_trigger_properties.${subject_line}}}`{% endraw %} |
| 행동 촉구 | {% raw %}`{{api_trigger_properties.${cta_message}}}`{% endraw %} |
| 제안 | {% raw %}`{{api_trigger_properties.${offer_id}}}`{% endraw %} |
| 할인 금액 | {% raw %}`{{api_trigger_properties.${discount}}}`{% endraw %} |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

## Best practices

오케스트레이션 준비 시 다음 모범 사례를 염두에 두십시오:

1. **간단하게 시작하세요.** 하나의 채널과 하나 또는 두 개의 템플릿으로 시작하세요. 나중에 무엇이 효과가 있는지 알게 되면 확장할 수 있습니다.
2. **철저히 테스트하십시오.** 출시 전에 소규모 사용자 그룹을 대상으로 통합 기능을 테스트하여 동적 콘텐츠가 올바르게 채워지는지 확인하십시오.
3. **설정을 문서화하십시오.** 캠페인 ID, 템플릿 ID, API 키 및 기타 식별자를 추적하십시오. 이것들을 Decisioning Studio 포털에서 참조해야 합니다.
4. **팀과 협력하십시오.** 오케스트레이션 설정에는 마케팅, 엔지니어링 및 데이터 팀이 참여할 수 있습니다. 모든 사람이 해당 프로세스에서 자신의 역할을 이해하도록 하십시오.
5. **피드백 데이터 계획 수립** 오케스트레이션은 단순히 메시지를 전송하는 것뿐만 아니라, 에이전트의 학습에 도움이 되는 참여 및 전환 데이터를 수집하는 역할도 합니다. 자세한 내용은 [데이터 소스 준비를]({{site.baseurl}}/user_guide/brazeai/decisioning_studio/getting_started/preparing_your_data_sources/) 참조하십시오.
