# BrazeAI Operator 액션 검토

> BrazeAI Operator<sup>TM</sup>가 대시보드에서 변경 사항을 제안할 때 액션을 검토하고 승인하는 방법을 알아봅니다.

![Operator가 검토를 위해 제안된 액션 카드를 표시함.]({% image_buster /assets/img/operator/suggested_actions.png %}){: style="max-width:40%; border:none; float:right; margin-left:15px;"}

## 액션 카드 작동 방식

Operator가 대시보드에서 변경 사항을 제안할 때(예: 폼 필드 채우기, 설정 조정, 이미지 생성) 각 변경 사항은 검토를 위해 액션 카드로 표시됩니다.

1. **Operator가 계획 요약:** Operator는 액션 카드 앞에서 수행할 작업을 설명합니다.
2. **개별 액션 카드 표시:** 각 제안된 변경 사항은 별도 카드에 표시됩니다. 기존 값 변경 시 이전 값과 새 값이 나란히 표시됩니다.
3. **검토 및 승인:** 각 카드를 검토하고 승인하거나 거부합니다.
4. **액션 실행:** 승인된 액션은 Braze에서 실행됩니다. 거부된 액션은 적용되지 않습니다.

승인 후 액션이 실패하면 Operator가 오류 세부 정보와 함께 알려줍니다.

### 지원 범위

액션 카드는 다음 편집기에서 지원됩니다.

- 인앱 메시지(클래식 편집기만)
- 콘텐츠 카드
- 이메일(HTML 편집기만)
- 푸시 알림
- SMS/MMS/RCS
- 웹훅

다른 페이지에서는 Operator가 UI에서 직접 액션을 실행하는 대신 실행할 단계 목록을 제공합니다. Operator 기능은 지속적으로 확장되며, 더 많은 제작 도구 지원이 예정되어 있습니다.

## 계획 수정하기

Operator의 계획을 변경하려면 먼저 대기 중인 액션을 승인하거나 거부한 다음, 새 채팅 메시지에서 원하는 변경 사항을 설명합니다.

승인된 액션은 Operator를 통해 취소할 수 없습니다. Operator에 새 변경 사항을 설명하거나 대시보드에서 수동으로 변경합니다.

## 액션 자동 승인

**액션 자동 승인** 스위치는 Operator 채팅 패널에 있습니다.

- **켜짐:** Operator가 제안한 액션이 수동 승인 없이 즉시 실행됩니다. 보안상 일부 액션(예: 이미지 생성, 워크스페이스 수준 변경)은 계속 명시적 승인이 필요합니다.
- **꺼짐(기본값):** 모든 제안된 액션은 설명된 수동 검토 프로세스를 거칩니다.

![Operator 채팅 패널의 자동 승인 스위치 및 확인 대화상자.]({% image_buster /assets/img/operator/auto-approval_toggle.png %}){: style="max-width:50%;"}

페이지를 새로 고치거나 새 탭을 열거나 로그아웃 후 다시 로그인하면 자동 승인이 초기화됩니다. 대시보드에서 페이지를 전환해도 초기화되지 않습니다. 언제든지 자동 승인을 끌 수 있습니다.
