# BrazeAI Operator 작업 검토

> BrazeAI Operator™가 대시보드에서 변경 사항을 제안할 때 작업을 검토하고 승인하는 방법을 알아보세요.

![운영자가 검토를 위해 제안된 조치 카드를 제시합니다.]({% image_buster /assets/img/operator/suggested_actions.png %}){: style="max-width:40%; border:none; float:right; margin-left:15px;"}

## 액션 카드 작동 방식

운영자가 대시보드에서 변경 사항(예: 양식 필드 입력, 설정 업데이트 또는 이미지 생성)을 제안할 때, 각 변경 사항은 검토를 위한 작업 카드로 표시됩니다.

1. **운영자가 계획을 요약합니다:** 운영자는 행동 카드를 보여주기 전에 무엇을 할 계획인지 설명합니다.
2. **개별 액션 카드가 나타납니다:** 제안된 각 변경 사항은 별도의 카드로 표시되며, 해당 카드는 운영자가 대시보드에서 변경하거나 수행하고자 하는 작업을 보여줍니다. 기존 값 변경 시에는 이전 값과 제안된 값이 나란히 표시되어 비교할 수 있습니다.
3. **검토 및 승인:** 각 카드를 검토한 후 승인하거나 거절하십시오.
4. **액션 실행:** 승인된 작업은 Braze에서 실행됩니다. 거부된 작업은 적용되지 않습니다.

승인 후 작업이 실패할 경우, 운영자가 실패에 대한 세부 정보를 알립니다.

### 가용성

액션 카드는 다음 편집기에서 지원됩니다:

- 인앱 메시지 (기존 편집기 전용)
- 콘텐츠 카드
- 이메일 (HTML 편집기 전용)
- 푸시 알림
- SMS/MMS/RCS
- Webhooks

다른 페이지에서는 Operator가 직접 작업을 수행하지 않고 UI에서 따라야 할 단계 목록을 제공합니다. 운영자 기능은 지속적으로 개선되고 있으며, 생성 도구에 대한 적용 범위 확대가 예상됩니다.

## 계획 수정

운영자 계획을 수정하려면 먼저 보류 중인 작업을 승인하거나 거부하십시오. 그런 다음 원하는 변경 사항을 새 채팅 메시지로 설명하세요.

승인된 작업은 운영자를 통해 되돌릴 수 없습니다. 운영자에게 새로운 변경 사항을 설명하거나 대시보드에서 수동으로 변경하십시오.

## 자동 승인 작업

**자동 승인 작업** 토글은 운영자 채팅 패널에 위치합니다.

- **켜짐:** 운영자가 제안한 조치는 수동 승인 없이 즉시 실행됩니다. 일부 작업은 안전을 위해 여전히 명시적인 승인이 필요합니다. 예를 들어 이미지 생성이나 작업 공간 수준 설정 변경 등이 해당됩니다.
- **꺼짐 (기본값):** 제안된 모든 조치는 설명된 수동 검토 절차를 따릅니다.

![운영자 채팅 패널의 자동 승인 토글 및 확인 모달]({% image_buster /assets/img/operator/auto-approval_toggle.png %}){: style="max-width:50%;"}

자동 승인은 페이지를 새로고침하거나 새 탭을 열거나 로그아웃 후 다시 로그인할 때 재설정됩니다. 대시보드에서 페이지를 이동해도 초기화되지 않습니다. 자동 승인은 언제든지 해제할 수 있습니다.
