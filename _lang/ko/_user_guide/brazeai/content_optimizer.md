# 콘텐츠 최적화 프로그램

> 콘텐츠 최적화 도구는 AI를 사용하여 대규모로 메시지 콘텐츠를 테스트하고 최적화하는 데 도움을 주는 에이전트입니다. 이 도구는 콘텐츠 변형의 높은 볼륨을 자동으로 생성하고 평가합니다.

{% alert important %}
콘텐츠 최적화 도구는 현재 베타 상태이며 이메일 메시지에만 사용할 수 있습니다. 시작하는 데 도움이 필요하면 고객 성공 매니저에게 문의하십시오.
{% endalert %}

## 콘텐츠 최적화 도구에 대하여

콘텐츠 최적화 도구는 캔버스 단계에서 실행되는 에이전트입니다. 이 도구는 테스트할 메시지 구성 요소를 정의하고, 생성적 AI 또는 수동 입력을 사용하여 변형을 생성하며, 사용자에게 전송할 콘텐츠 조합을 자동으로 최적화하는 데 도움을 줍니다. 이 기능은 다음을 도와줍니다:

- 이메일의 제목란, 본문 헤더, 본문 내용 또는 주요 CTA를 최적화합니다.
- 수동 A/B 테스트 설정 없이 메시지 성능을 지속적으로 개선합니다.
- 아이디어 구상을 위해 AI를 활용하여 콘텐츠 변형을 신속하게 테스트합니다.
- 성능이 저조한 콘텐츠는 자동으로 단계적으로 제거하고, 성과가 좋은 콘텐츠는 확대합니다.

## 작동 방식

콘텐츠 최적화기는 비상황별 [다중 무장 강도](https://en.wikipedia.org/wiki/Multi-armed_bandit) 알고리즘을 사용하여 고성능 변형에 더 많은 전송을 할당하고 저성능 변형에 대한 할당을 줄입니다. 시간이 지남에 따라, 이는 최소한의 수동 개입으로 메시지 콘텐츠의 지속적인 개선을 초래합니다.

단계가 처음 시작될 때, 콘텐츠 최적화기는 초기 성능 데이터를 수집하기 위해 변형을 무작위로 전송합니다. 이 초기 탐색 기간 후, 알고리즘은 고성능 콘텐츠 조합으로 트래픽을 이동시키기 시작하고 저성능 옵션에 대한 할당을 점차 줄입니다. 탐색 기간 동안, 트래픽은 알고리즘이 상대 성능에서 학습할 수 있도록 사용 가능한 변형에 일반적으로 분배됩니다.

콘텐츠 최적화기는 캔버스의 메시지 단계와 유사하며, 방해금지 시간, [지능형 타이밍]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_timing), 및 이벤트 로깅과 같은 기능을 갖추고 있습니다. 기본 메시지를 생성하고 최적화할 콘텐츠 구성 요소(예: 제목란, 본문 텍스트 또는 클릭 유도 문구)를 정의하여 콘텐츠 최적화기 단계를 구성할 수 있습니다. 각 구성 요소의 변형은 AI로 생성하거나 수동으로 입력할 수 있으며, 구성 요소를 메시지 콘텐츠에 매핑하기 위해 기본 메시지에 Liquid 태그를 추가해야 합니다.

각 사용자는 콘텐츠 최적화기 단계에 들어갈 때마다 하나의 메시지를 받습니다. 재입장은 새로운 것으로 간주되며, 이전 변형에 대한 기억이 없습니다.

최상의 결과를 위해, 사용자가 시간이 지남에 따라 단계에 점진적으로 들어가는 캔버스에서 콘텐츠 최적화기를 사용하십시오. 예를 들어, 일관된 일일 볼륨이 있는 반복적이거나 항상 켜져 있는 캔버스에서 사용하십시오. 모든 사용자가 한 번에 단계에 들어가면, 에이전트는 초기 결과에서 학습할 시간이 없습니다. 이 단계는 실시간 최적화 엔진보다 정적 A/B 테스트처럼 작동합니다.

이는 사용자가 장기간에 걸쳐 단계에 들어오는 경우(예: 지연 단계, 예약된 진입 또는 API 트리거 흐름을 통해) 단일 전송 또는 단기 캔버스에서 콘텐츠 최적화기를 여전히 사용할 수 있음을 의미합니다. 대부분의 사용자에게 도달하기 전에 성능 차이를 관찰할 수 있도록 단계에 충분한 트래픽과 시간을 확보해야 합니다.


### 핵심 개념

| 기간                    | 설명 |
|-------------------------|-------------|
| 기본 메시지   | 모든 전송 설정을 포함하여 변형이 구축되는 기본 메시지 템플릿입니다. |
| 콘텐츠 구성 요소  | 테스트 및 최적화할 수 있는 메시지 내 요소(예: 제목란 또는 주요 CTA)입니다. 마케터는 구성 요소가 나타나야 하는 메시지에 관련된 Liquid 태그를 삽입해야 합니다. |
| 콘텐츠 배리언트    | 콘텐츠 구성 요소가 가질 수 있는 다양한 값입니다. |
| 콘텐츠 조합| 콘텐츠 변형을 혼합하고 조합하여 생성된 고유한 메시지입니다. |
| 최적화 이벤트       | 콘텐츠 최적화기가 성능을 평가하고 시간에 따라 콘텐츠 조합에 트래픽을 할당하는 방법을 결정합니다. 예를 들어 이메일의 클릭 수 또는 열림 수입니다. 단계의 모든 콘텐츠 구성 요소에 적용됩니다. 콘텐츠 최적화기는 이 이벤트에서 지속적으로 학습하고 자동으로 더 높은 성능의 콘텐츠 조합으로 전달을 전환합니다. |

## Limitations

- 콘텐츠 최적화 도구는 현재 베타 상태이며 이메일 메시지에만 사용할 수 있습니다.
- 에이전트는 단계당 최대 125개의 조합을 생성할 수 있습니다:
   - 단계당 최대 3개의 구성 요소
   - 각 구성 요소에 대해 최대 5개의 변형
- 사용자당 각 진입에 대해 하나의 메시지만 전송됩니다. 재진입에 대한 이전 전송의 기억이 없습니다.
- 마케터는 정의된 콘텐츠 구성 요소 변형이 렌더링되어야 하는 메시지 작성기에서 각 구성 요소에 대해 Liquid 태그를 수동으로 삽입해야 합니다.
