# 콘텐츠 최적화 도구

> 콘텐츠 최적화 도구는 AI를 사용하여 대규모로 메시지 콘텐츠를 테스트하고 최적화하는 데 도움을 주는 에이전트입니다. 이 도구는 콘텐츠 변형의 높은 볼륨을 자동으로 생성하고 평가합니다.

{% alert important %}
콘텐츠 최적화 도구는 현재 베타 버전이며 이메일 메시지에만 사용할 수 있습니다. 시작하는 데 도움이 필요하면 고객 성공 매니저에게 문의하십시오.
{% endalert %}

## 콘텐츠 최적화 도구에 대하여

콘텐츠 최적화 도구는 캔버스 단계에서 실행되는 에이전트입니다. 이 도구는 테스트할 메시지 구성 요소를 정의하고, 생성적 AI 또는 수동 입력을 사용하여 변형을 생성하며, 사용자에게 전송할 콘텐츠 조합을 자동으로 최적화하는 데 도움을 줍니다. 이 기능은 다음을 도와줍니다:

- 이메일의 제목란, 본문 헤더, 본문 내용 또는 주요 CTA를 최적화합니다.
- 수동 A/B 테스트 설정 없이 메시지 성능을 지속적으로 개선합니다.
- AI를 활용하여 콘텐츠 변형을 신속하게 테스트합니다.
- 성과가 저조한 콘텐츠는 자동으로 단계적으로 제거하고, 성과가 좋은 콘텐츠는 확대합니다.

[콘텐츠 최적화 도구 단계]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/content_optimizer_step/)를 만드는 방법을 알아보세요.

## 작동 방식

콘텐츠 최적화기는 비맥락적 [다중 무장 강도](https://en.wikipedia.org/wiki/Multi-armed_bandit) 알고리즘을 사용하여 고성능 변형에 더 많은 전송을 할당하고 저성능 변형에 대한 할당을 줄입니다. 시간이 지남에 따라, 이는 최소한의 수동 개입으로 메시지 콘텐츠의 지속적인 개선을 가져옵니다.

Braze의 독점적인 강도 최적화 알고리즘은 콘텐츠 최적화 단계의 조합적 특성을 위해 특별히 구축되었습니다. 각 메시지가 여러 구성 요소로 구성되어 있으므로, 강도는 메시지로 결합될 때 각 구성 요소(예: 제목, 본문, CTA)의 성능과 상호 작용에 대해 동시에 학습합니다. 더 구체적으로, 주어진 조합이 전송될 때, 동일한 구성 요소를 공유하는 모든 조합은 해당 전송의 데이터로부터 이익을 얻습니다. 이것은 강도가 표준 강도 알고리즘에 비해 동일한 양의 데이터에서 훨씬 더 빠르게 학습할 수 있게 합니다.

단계가 처음 시작될 때, 콘텐츠 최적화기는 초기 성능 데이터를 수집하기 위해 변형을 무작위로 전송합니다. 이 초기 탐색 기간 후, 알고리즘은 고성능 콘텐츠 조합으로 트래픽을 이동시키기 시작하고 저성능 옵션에 대한 할당을 점차 줄입니다. 탐색 기간 동안, 트래픽은 알고리즘이 상대 성능으로부터 학습할 수 있도록 사용 가능한 변형에 일반적으로 분배됩니다.

콘텐츠 최적화기는 캔버스의 메시지 단계와 유사하며, 방해 금지 시간, [지능형 타이밍]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_timing), 및 이벤트 로깅과 같은 기능을 갖추고 있습니다. 기본 메시지를 생성하고 최적화할 콘텐츠 구성 요소(예: 제목, 본문 텍스트 또는 CTA)를 정의하여 콘텐츠 최적화 단계의 구성을 설정할 수 있습니다. 각 구성 요소에 대한 변형은 AI로 생성하거나 수동으로 입력할 수 있으며, Liquid 태그는 구성 요소를 메시지 내용에 매핑하기 위해 기본 메시지에 추가해야 합니다.

각 사용자는 콘텐츠 최적화 단계에 진입할 때마다 하나의 메시지를 받습니다. 재진입은 이전 변형에 대한 기억 없이 새로운 것으로 간주됩니다.

최상의 결과를 얻으려면 사용자가 시간이 지남에 따라 단계에 점진적으로 진입하는 캔버스에서 콘텐츠 최적화기를 사용하세요. 예를 들어, 일관된 일일 볼륨이 있는 반복적이거나 항상 켜져 있는 캔버스에서 사용합니다. 모든 사용자가 한 번에 단계를 입력하면, 에이전트는 초기 결과로부터 학습할 시간이 없습니다. 이 단계는 실시간 최적화 엔진보다 정적 A/B 테스트처럼 작동합니다.

이는 단일 전송 또는 단기 캔버스에서 콘텐츠 최적화기를 여전히 사용할 수 있지만, 사용자가 장기간에 걸쳐 단계에 진입하는 경우에만 가능하다는 것을 의미합니다(예: 지연 단계, 예약된 진입 또는 API 트리거 흐름을 통해). 단계가 대부분의 사용자에게 도달하기 전에 성능 차이를 관찰할 수 있도록 충분한 트래픽과 시간이 있는지 확인하세요.

### 핵심 개념

| 기간                    | 설명 |
|-------------------------|-------------|
| 기본 메시지   | 변형이 구축되는 주요 메시지 템플릿으로, 모든 전송 설정을 포함합니다. |
| 콘텐츠 구성 요소  | 테스트 및 최적화할 수 있는 메시지 내의 요소(예: 제목란 또는 주요 CTA)입니다. 마케터는 구성 요소가 나타나야 하는 메시지에 관련 Liquid 태그를 삽입해야 합니다. |
| 콘텐츠 변형    | 콘텐츠 구성 요소가 가질 수 있는 다양한 값입니다. |
| 콘텐츠 조합| 콘텐츠 변형을 혼합하고 조합하여 생성된 고유한 메시지입니다. |
| 최적화 이벤트       | 콘텐츠 최적화기가 성능을 평가하고 시간에 따라 콘텐츠 조합에 트래픽을 할당하는 방법을 결정합니다. 예를 들어, 이메일의 클릭 수 또는 열림 수와 같은 것입니다. 단계의 모든 콘텐츠 구성 요소에 적용됩니다. 콘텐츠 최적화기는 이 이벤트로부터 지속적으로 학습하고 자동으로 더 높은 성과를 내는 콘텐츠 조합으로 전달을 전환합니다. |

## Limitations

- 콘텐츠 최적화 도구는 현재 베타 버전이며 이메일 메시지에만 사용할 수 있습니다.
- 에이전트는 단계당 최대 125개의 조합을 생성할 수 있습니다:
   - 단계당 최대 3개의 구성 요소
   - 각 구성 요소에 대해 최대 5개의 변형
- 사용자당 항목당 하나의 메시지만 전송됩니다. 재입력에 대한 이전 전송의 기억이 없습니다.
- 마케터는 메시지 작성기에서 정의된 콘텐츠 구성 요소 변형이 렌더링되어야 하는 각 구성 요소에 대해 Liquid 태그를 수동으로 삽입해야 합니다.
