# 프로모션 코드

> 이 페이지에서는 프로모션 코드 목록을 생성하고 이를 캠페인 및 캔버스에 추가하는 방법을 다룹니다.

## 프로모션 코드에 대하여

프로모션 코드는 프로모션 코드라고도 하며, 구매에 대한 강한 강조를 통해 사용자 참여를 유지하는 훌륭한 방법입니다. 프로모션 코드 목록에서 가져온 메시지를 생성할 수 있습니다. 

각 프로모션 코드의 유효 기간은 최대 6개월입니다. 목록당 최대 2천만 개의 코드를 저장하고 관리할 수 있습니다. 프로모션 코드의 성과를 관리하고 분석함으로써, 프로모션 전략 및 메시지에 대한 목표 지향적인 결정을 내릴 수 있습니다.

{% alert important %}
프로모션 코드는 캔버스의 인앱 메시지로 전송할 수 없습니다.
{% endalert %}

## 프로모션 코드 목록 생성 {#create}

### 1단계: 프로모션 코드 섹션으로 이동

\![프로모션 코드를 생성하는 버튼.]({% image_buster /assets/img/promocodes/promocode1.png %}){: style="float:right;max-width:30%;margin-left:15px;"}

1. 대시보드에서 **데이터 설정** > **프로모션 코드**로 이동합니다.
2. **프로모션 코드 목록 생성**을 선택합니다.

### 2단계: 프로모션 코드를 명명하세요

1. 프로모션 코드 목록의 이름을 지정하고 선택적 설명을 추가하세요.
2. 다음으로, 프로모션 코드에 대한 코드 스니펫을 만드세요. 

코드 스니펫을 만들 때 고려해야 할 몇 가지 세부 사항은 다음과 같습니다:

- 저장 후에는 코드 스니펫을 편집할 수 없습니다.
- 스니펫은 대소문자를 구분합니다. 예를 들어, "Birthday_promo"과 "birthday_promo"는 두 개의 다른 스니펫으로 인식됩니다.
- Liquid에서 스니펫 이름을 사용하여 이 프로모션 코드 세트를 참조하세요.
- 코드 스니펫이 다른 목록에서 이미 사용되고 있지 않은지 확인하세요.

\!["SpringSale2025"라는 이름의 프로모션 코드 목록과 코드 스니펫 "spring25".]({% image_buster /assets/img/promocodes/promocode3.png %}){: style="max-width:80%"}

### 3단계: 프로모션 코드 옵션 선택

각 프로모션 코드 목록에는 생성 시 설정되는 해당 만료 날짜와 시간이 있습니다. 최대 만료 기간은 목록을 생성하거나 편집하는 날로부터 6개월입니다. 

그 기간 내에 만료 날짜를 반복적으로 변경하고 업데이트할 수 있습니다. 이 만료 날짜는 이 목록에 추가된 모든 코드에 적용됩니다. 만료 시, 코드는 Braze 시스템에서 삭제되며, 해당 목록의 코드 스니펫을 호출하는 메시지는 전송되지 않습니다.

\![모든 남은 코드가 2025년 4월 30일 오전 12시에 만료되는 목록 만료 설정.]({% image_buster /assets/img/promocodes/promocode4.png %}){: style="max-width:80%"}

선택적이고 맞춤형 임계값 알림을 설정할 옵션도 있습니다. 설정된 경우, 이러한 알림은 목록의 사용 가능한 프로모션 코드가 부족하거나 프로모션 코드 목록이 만료에 가까워질 때 지정된 수신자에게 이메일을 보냅니다. 수신자는 하루에 한 번 통지를 받게 됩니다.

\![5일 후 프로모션 코드 목록이 만료될 때 "marketing@abc.com"에 통지하는 임계값 경고의 예.]({% image_buster /assets/img/promocodes/promocode5.png %}){: style="max-width:80%"}

### 4단계: 프로모션 코드를 업로드하세요.

Braze는 코드 생성이나 사용을 관리하지 않으므로, 프로모션 코드를 CSV 파일로 생성하고 Braze에 업로드해야 합니다. 

CSV 파일이 다음 지침을 따르는지 확인하세요:

- 프로모션 코드에 대한 열이 포함되어야 합니다.
- 행당 하나의 프로모션 코드가 있어야 합니다.

우리의 내장 통합 기능인 [Voucherify]({{site.baseurl}}/partners/ecommerce/loyalty/voucherify/) 또는 [Talon.One]({{site.baseurl}}/partners/ecommerce/loyalty/talonone/)를 사용하여 프로모션 코드를 생성하고 내보낼 수 있습니다.

{% alert important %}
최대 파일 크기는 100MB이며, 최대 목록 크기는 사용되지 않은 코드 20MM입니다. 잘못된 파일이 업로드된 경우, 새 파일을 업로드하면 이전 파일이 대체됩니다.
{% endalert %}

1. 업로드가 완료되면 **목록 저장**을 선택하여 방금 입력한 모든 세부정보와 코드를 저장하세요.

\![성공적으로 업로드된 "springsale"라는 이름의 CSV 파일.]({% image_buster /assets/img/promocodes/promocode7.png %})

{:start="2"}
2\. 저장 선택 후, **가져오기 기록**에 새 행이 나타납니다.
3\. 가져오기가 완료되었는지 확인하려면 테이블 상단에서 <span style="font-size: 14px;margin-bottom: .5rem;height: 16px;width: 16px;" class="fas fa-sync" ></span> **동기화**를 선택하여 테이블을 새로 고치세요.

\![업로드 중인 프로모션 코드.]({% image_buster /assets/img/promocodes/promocode8.png %})

{% alert note %}
더 큰 파일은 가져오는 데 몇 분이 걸릴 수 있습니다. 기다리는 동안 페이지를 떠나 다른 작업을 할 수 있습니다. 가져오기가 완료되면 상태가 테이블에서 **완료**로 변경됩니다.
{% endalert %}

## 프로모션 코드 목록 업데이트

목록을 업데이트하려면 기존 목록 중 하나를 선택하십시오. 이름, 설명, 목록 만료 및 임계값 알림을 변경할 수 있습니다. 새 파일을 업로드하고 **목록 업데이트**을 선택하여 목록에 더 많은 코드를 추가할 수 있습니다. 목록의 모든 코드는 가져오기 날짜와 관계없이 동일한 만료일을 가집니다.

{% alert important %}
프로모션 코드는 삭제할 수 없습니다.
{% endalert %}

### 잘못된 프로모션 코드 목록 수정 

잘못된 프로모션 코드가 포함된 CSV 파일을 업로드하고 **목록 저장**을 선택한 경우, 다음 방법 중 하나로 해결할 수 있습니다:

- 전체 목록 사용 중단: 현재 프로모션 코드 목록을 캠페인, 캔버스 또는 템플릿에서 사용하지 마십시오. 그런 다음 올바른 코드가 포함된 CSV 파일을 업로드하고 메시지에 사용하십시오.
- 잘못된 코드를 사용: 잘못된 프로모션 코드 목록에서 프로모션 코드를 플레이스홀더로 보내는 캠페인을 생성하여 모든 잘못된 코드가 사용될 때까지 기다립니다. 그런 다음 동일한 목록에 올바른 프로모션 코드를 업로드하십시오.

## 프로모션 코드 사용 {#update}

메시지에 프로모션 코드를 보내려면, 이전에 생성한 프로모션 코드 목록 [복사 스니펫](#create) 옆의 **복사 스니펫**을 선택하십시오.

\![메시지에 붙여넣을 스니펫을 복사하는 옵션입니다.]({% image_buster /assets/img/promocodes/promocode9.png %}){: style="max-width:50%"}

코드 스니펫을 Braze의 메시지 중 하나에 붙여넣고, [Liquid]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/)을 사용하여 목록에서 고유한 프로모션 코드 중 하나를 삽입하십시오. 해당 코드는 전송된 것으로 표시되어 다른 메시지가 동일한 코드를 전송하지 않도록 합니다.

\![예시 메시지 "이번 봄에 독점 제안으로 자신에게 좋은 것을 선물하세요" 다음에 코드 스니펫이 옵니다.]({% image_buster /assets/img/promocodes/promocode10.png %}){: style="max-width:50%"}

### 캔버스 단계 전반에 걸쳐

캠페인이나 멀티채널 메시지가 있는 캔버스에서 코드 스니펫이 사용될 때, 각 사용자는 고유한 코드를 받습니다. 프로모션 코드가 참조되는 여러 단계가 있는 캔버스에서 사용자는 들어가는 각 단계마다 새로운 코드를 받습니다.

캔버스에서 하나의 프로모션 코드를 할당하고 단계를 통해 재사용하려면:

1. 첫 번째 단계(사용자 업데이트)에서 프로모션 코드를 사용자 정의 속성으로 할당합니다.
2. 이후 단계에서 새로운 코드를 생성하는 대신 해당 사용자 정의 속성을 참조하기 위해 Liquid를 사용합니다.

사용자가 여러 채널에서 코드 자격을 갖추면 각 채널에서 동일한 코드를 받습니다. 예를 들어, 이메일과 푸시로 메시지를 받으면 동일한 코드가 두 곳 모두에 전송됩니다. 보고서에도 단일 코드가 반영됩니다.

{% alert note %}
프로모션 코드가 없으면 코드에 의존하는 테스트 또는 실시간 메시지는 전송되지 않습니다.
{% endalert %}

### 인앱 메시지 캠페인 {#promotion-codes-iam-campaigns}

인앱 메시지 캠페인 [인앱 메시지 캠페인]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages)을 생성한 후, 인앱 메시지 본문에 [프로모션 코드 목록 스니펫]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/promotion_codes#creating-a-promotion-code-list)을 삽입할 수 있습니다. 

인앱 메시지의 프로모션 코드는 사용자가 인앱 메시지를 표시하도록 트리거할 때만 차감되고 사용됩니다.

### 테스트 메시지에서

테스트 전송 및 시드 그룹 이메일 전송은 요청하지 않는 한 프로모션 코드를 소모합니다. 테스트 전송 및 시드 그룹 이메일 전송 중에 프로모션 코드가 사용되지 않도록 이 기능 동작을 업데이트하려면 Braze 계정 관리자에게 문의하세요.

## 사용자 프로필에 프로모션 코드를 저장하기 {#save-to-profile}

후속 메시지에서 동일한 프로모션 코드를 참조하려면 코드를 사용자 프로필에 사용자 정의 속성으로 저장해야 합니다. 이는 할인 코드를 사용자 정의 속성인 "프로모션 코드"에 할당하는 [사용자 업데이트 단계]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/user_update/)을 통해 수행할 수 있으며, 메시지 단계 바로 전에 진행됩니다.

먼저 사용자 업데이트 단계의 각 필드에 대해 다음을 선택합니다:

- **속성 이름:** 프로모션 코드
- **작업:** 업데이트
- **키 값:** 프로모션 코드의 Liquid 코드 조각, 예: {% raw %}`{% promotion('spring25') %}`{% endraw %}

둘째, 사용자 정의 속성(이 예에서는 {% raw %}`{{custom_attribute.${Promo Code}}`{% endraw %})을 메시지에 추가합니다. 할인 코드는 템플릿에 포함됩니다.

## 다채널 및 단일 채널 전송

다채널 및 단일 전송 캠페인 및 캔버스의 경우, 메시지의 Liquid에서 참조된 모든 프로모션 코드는 메시지가 전송되기 **전**에 사용되도록 차감됩니다. 이는 다음이 발생하도록 보장합니다:

- 다채널 메시지에서 동일한 프로모션 코드가 사용됩니다.
- 메시지가 실패하거나 중단되면 추가 프로모션 코드는 사용되지 않습니다.

사용자가 Liquid 조건 논리 태그로 분할된 하나의 메시지에서 두 개의 프로모션 코드 목록을 참조하는 경우, 사용자가 따르는 조건 흐름에 관계없이 모든 프로모션 코드가 여전히 차감됩니다.

사용자가 새로운 Canvas 단계를 입력하거나 Canvas에 다시 들어가고, 해당 사용자에게 메시지를 위해 프로모션 코드 Liquid 스니펫이 다시 적용되면 새로운 프로모션 코드가 사용됩니다.

### 예시

다음 예시에서는 두 개의 프로모션 코드 목록 `vip-deal`과 `regular-deal`가 차감됩니다. 여기 Liquid가 있습니다:

{% raw %}
```
{% if user.is_vip %}
  {% promotion('vip-deal') %}
{% else %}
  {% promotion('regular-deal') %}
{% endif %} 
```
{% endraw %}

Braze는 사용될 것으로 예상되는 것보다 더 많은 프로모션 코드를 업로드할 것을 권장합니다. 프로모션 코드 목록이 만료되거나 프로모션 코드가 소진되면 이후 메시지는 중단됩니다.

{% alert tip %}
**프로모션 코드가 Braze에서 어떻게 소진되는지에 대한 비유입니다.** <br><br>메시지를 보내는 것이 우체국에서 편지를 보내는 것과 같다고 상상해 보세요. 편지를 직원에게 주면, 직원은 귀하의 편지에 쿠폰이 포함되어야 한다고 확인합니다. 직원이 스택에서 첫 번째 쿠폰을 꺼내어 봉투에 추가합니다. 직원이 편지를 보냈지만, 어떤 이유로 편지가 우편에서 분실됩니다(그리고 쿠폰도 이제 분실됩니다). <br><br>이 시나리오에서 Braze는 우편 직원이며, 귀하의 프로모션 코드는 쿠폰입니다. 웹훅 결과와 관계없이 프로모션 코드 스택에서 꺼낸 후에는 이를 회수할 수 없습니다.
{% endalert %}

## 자주 묻는 질문

### 어떤 메시징 채널에서 프로모션 코드를 사용할 수 있나요?

프로모션 코드는 현재 이메일, 모바일 푸시, 웹 푸시, 콘텐츠 카드, 웹훅, SMS 및 WhatsApp에서 지원됩니다. Braze 트랜잭셔널 이메일 캠페인 및 인앱 메시지는 현재 프로모션 코드를 지원하지 않습니다.

### 테스트 및 시드 전송이 사용량에 포함되나요?

기본적으로 테스트 전송 및 시드 그룹 이메일 전송은 사용자당, 테스트 전송당 프로모션 코드를 사용합니다. 그러나 테스트 중에 프로모션 코드를 사용하지 않도록 이 동작을 업데이트하려면 Braze 계정 관리자에게 문의할 수 있습니다.

### 여러 메시징 채널이 동일한 프로모션 코드 스니펫을 사용할 때 어떤 일이 발생합니까?

특정 사용자가 여러 채널을 통해 코드를 받을 수 있는 경우, 각 채널을 통해 동일한 코드를 받게 됩니다. 수신된 채널에 관계없이 하나의 프로모션 코드만 사용됩니다.

### 하나의 메시지에서 동일한 프로모션 코드 목록을 참조하기 위해 여러 Liquid 스니펫을 사용할 수 있습니까?

네. Braze는 메시지 내에서 해당 스니펫의 모든 인스턴스에 동일한 프로모션 코드를 적용하여 사용자가 고유한 코드를 하나만 받도록 보장합니다.

### 프로모션 코드 목록이 만료되거나 비어 있으면 어떤 일이 발생합니까?

만료된 코드는 6개월 후에 삭제됩니다.

메시지가 비어 있거나 만료된 목록의 프로모션 코드를 포함해야 하는 경우, 메시지는 취소됩니다. 

메시지가 조건부로 프로모션 코드를 삽입하는 Liquid 로직을 포함하는 경우, 프로모션 코드를 포함해야 하는 경우에만 메시지가 취소됩니다. 메시지가 프로모션 코드를 포함하지 않아야 하는 경우, 메시지는 정상적으로 전송됩니다.

### 잘못된 프로모션 코드를 업로드한 경우, 업데이트할 수 있습니까?

네. 전체 목록을 사용 중단하거나 자리 표시자를 사용하여 목록을 삭제하여 이 문제를 해결할 수 있습니다. 자세한 내용은 [프로모션 코드 업데이트](#update)를 참조하십시오.

### 향후 메시지를 위해 프로모션 코드를 사용자의 프로필에 저장할 수 있습니까?

네. 사용자 업데이트 단계를 통해 사용자의 프로필에 프로모션 코드를 저장할 수 있습니다. 자세한 내용은 [사용자 프로필에 프로모션 코드 저장](#save-to-profile)을 참조하십시오.
