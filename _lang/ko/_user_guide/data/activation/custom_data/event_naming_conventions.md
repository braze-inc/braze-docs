# 이벤트 이름 지정 규칙

> 이 페이지에서는 적절한 이벤트 이름 지정 규칙과 모범 사례에 대해 설명합니다. 이벤트 및 속성 분류의 일관성을 유지하면 데이터를 깔끔하게 유지하여 신규 및 기존 Braze 플랫폼 사용자가 사용할 수 있습니다. 이렇게 하면 나중에 잘못된 오디언스에게 캠페인을 트리거하거나 잘못된 이벤트를 사용한 후 잘못된 결과를 생성하는 등의 문제를 방지할 수 있습니다.

## 모범 사례

- 이름 지정 규칙을 명확하게 유지하세요.
- 이벤트 이름의 대소문자와 서식을 일관되게 사용하세요.
- 이벤트에 비슷한 이름을 붙이지 마세요.
- 긴 이벤트 속성 문자열은 Braze 대시보드에서 잘리거나 끊어지므로 피하세요.

## 이름 지정 규칙

### 이벤트 그룹 사용

그룹을 사용하여 제품의 일부를 구분하여 이벤트 이름을 지정할 수 있습니다. 제품을 그룹으로 분류하면 모든 사용자가 이벤트가 무엇을 의미하고 어떤 기능을 하는지 명확하게 이해할 수 있습니다.

### 이벤트 이름 지정 구조

가장 일반적인 네이밍 구조는 `group_noun_action` 입니다. 계측 오류 및 식별자 오류를 방지하기 위해 이벤트는 모두 소문자로 입력해야 합니다.

### 속성

하나의 이벤트에 태그를 지정한 다음 속성을 사용하여 차이점을 식별합니다. 이는 캠페인의 채널과 같이 본질적으로 동일하지만 약간의 차이가 있는 이벤트에 유용합니다. 또한 사용자가 이벤트를 통해 어떻게 이동하는지 쉽게 확인할 수 있습니다. 예제 및 추가 컨텍스트는 [이벤트 속성정보 객체를]({{site.baseurl}}/api/objects_filters/event_object/#event-properties-object) 참조하세요.

## 예제

전자상거래 회사에 소속되어 있으며 고객이 언제 앱에 가입하고 언제 뉴스레터에 가입했는지 추적하고 싶다고 가정해 보겠습니다. 다음은 효과적인 이벤트 이름의 예입니다:

- `user_signup`
- `newsletter_subscribed`

이 두 이벤트 이름은 추적 중인 이벤트를 명확하게 나타냅니다. 커스텀 이벤트를 더 많이 만들수록 네이밍 규칙을 이해하기 쉽게 유지해야 합니다. 예를 들어 `signup_event_1` 과 같은 이벤트 이름은 `user_signup` 과 비교하여 명확성이 부족하고 이벤트가 추적하는 내용을 전달하지 못하므로 사용하지 않는 것이 좋습니다.
