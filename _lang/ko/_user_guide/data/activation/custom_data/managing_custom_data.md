# 고객 데이터 관리하기

> 이 페이지에서는 캠페인 및 세그먼트에 커스텀 데이터를 미리 채우고, 더 이상 유용하지 않은 데이터를 차단하고, 커스텀 이벤트 및 속성 및 속성을 관리하는 방법을 다룹니다.<br><br>특히 커스텀 속성을 관리하는 방법을 알아보려면 [커스텀 속성 관리하기를]({{site.baseurl}}/user_guide/data_and_analytics/custom_data/custom_attributes/#managing-custom-attributes) 참조하세요.

## 고객 데이터 미리 채우기

개발팀이 고객 데이터를 통합하기 전에 커스텀 데이터를 사용하여 캠페인 및 세그먼트를 설정하고 싶을 때가 있을 수 있습니다. Braze를 사용하면 이러한 데이터가 추적을 시작하기 전에 대시보드에 커스텀 이벤트 및 속성을 미리 입력하여 드롭다운과 캠페인 생성 프로세스의 일부로 사용할 수 있도록 할 수 있습니다.

커스텀 이벤트 및 속성을 미리 채우려면 다음과 같이 하세요:

1. **데이터 설정** > **커스텀 이벤트** 또는 **커스텀 속성** 또는 **제품으로** 이동합니다.

\![커스텀 속성 또는 커스텀 이벤트 또는 제품으로 이동합니다.]({% image_buster /assets/img_archive/prepopulate_page.png %}){: style="max-width:90%;" }

{: start="2"}
2\. 커스텀 속성, 이벤트 또는 제품을 추가하려면 해당 페이지로 이동하여 **커스텀 속성 추가** 또는 **커스텀 이벤트 추가** 또는 **제품 추가를** 선택합니다.<br><br>커스텀 속성의 경우 이 속성에 대한 [데이터 유형]({{site.baseurl}}/user_guide/data_and_analytics/custom_data/custom_attributes/#custom-attribute-data-types) (예: 부울 또는 문자열)을 선택합니다. 속성의 데이터 유형에 따라 해당 속성에 사용할 수 있는 세분화 필터가 결정됩니다. <br><br>\![새 속성 또는 이벤트 추가하기]({% image_buster /assets/img_archive/prepopulate_add.png %}){: style="max-width:80%;" }
3\. **저장을** 선택합니다.

### 커스텀 이벤트 및 커스텀 속성 이름 지정하기

고객 이벤트 및 커스텀 속성은 대소문자를 구분합니다. 나중에 개발팀이 이러한 커스텀 이벤트 및 속성을 통합할 때 이 점을 염두에 두세요. 커스텀 이벤트 및 속성의 이름을 여기에 지정한 대로 정확하게 지정해야 하며, 그렇지 않으면 Braze에서 다른 커스텀 이벤트 또는 속성을 생성합니다.

## 부동산 관리

커스텀 이벤트 또는 제품을 만든 후 해당 이벤트 또는 제품에 대한 **속성 관리를** 선택하여 새 속성을 추가하고, 기존 속성을 차단 목록에 추가하고, [트리거 이벤트에서]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/triggered_delivery/) 이 속성을 사용하는 캠페인 또는 캔버스를 확인합니다.

커스텀 이벤트의 커스텀 속성정보.]({% image_buster /assets/img_archive/manageproperties1.png %}){: style="max-width:80%"}

이렇게 추가된 커스텀 속성, 이벤트, 제품 또는 이벤트 속성정보를 추적할 수 있게 하려면 개발팀에 앞서 추가할 때 사용한 정확한 이름을 사용하여 SDK에서 생성하도록 요청해야 합니다. 또는 Braze [API를]({{site.baseurl}}/api/basics/) 사용하여 해당 속성에 대한 데이터를 가져올 수 있습니다. 그 후에는 커스텀 속성, 이벤트 등을 실행할 수 있고 사용자에게 적용할 수 있습니다.

{% alert note %}
모든 사용자 프로필 데이터(커스텀 이벤트, 커스텀 속성, 커스텀 데이터)는 해당 프로필이 활성화되어 있는 동안 저장됩니다.
{% endalert %}

## 커스텀 고객 데이터 차단하기

때때로 너무 많은 데이터 포인트를 기록하거나, 마케팅 전략에 더 이상 유용하지 않거나, 오류로 기록된 커스텀 속성, 커스텀 이벤트 또는 구매 이벤트를 식별할 수 있습니다. 

엔지니어링 팀이 앱이나 웹사이트의 백엔드에서 이 데이터를 제거하기 위해 작업하는 동안 이 데이터가 Braze로 전송되는 것을 막으려면 커스텀 데이터 오브젝트를 차단 목록에 추가하면 됩니다. 차단 목록은 특정 커스텀 데이터 오브젝트가 앞으로 Braze에 기록되는 것을 방지하여 특정 사용자를 검색할 때 표시되지 않도록 합니다.

{% alert important %}
커스텀 데이터를 차단하려면 캠페인, 캔버스 및 세그먼트에 액세스하고 편집할 수 있는 [고객 승인 권한이]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/#list-of-permissions) 필요합니다.
{% endalert %}

차단된 데이터는 소프트웨어 개발 키트에서 전송되지 않으며, Braze 대시보드는 다른 소스(예: API)에서 차단된 데이터를 처리하지 않습니다. 그러나 차단 목록은 사용자 프로필에서 데이터를 제거하거나 해당 커스텀 데이터 오브젝트에 대해 발생하는 데이터 포인트의 양을 소급하여 감소시키지 않습니다.

### 커스텀 속성, 커스텀 이벤트 및 제품 차단하기

{% alert important %}
이벤트 또는 속성이 차단되면 해당 이벤트 또는 속성을 사용하는 모든 세그먼트, 캠페인 또는 캔버스가 아카이브됩니다.
{% endalert %}

특정 커스텀 속성, 이벤트 또는 제품의 추적을 중지하려면 다음 단계를 따르세요:

1. **커스텀 속성**, **커스텀 이벤트** 또는 **제품** 페이지에서 검색하세요.
2. 커스텀 속성, 이벤트 또는 제품을 선택합니다. 커스텀 속성 및 이벤트의 경우 한 번에 최대 100개까지 차단 목록에 추가할 수 있습니다.
3. **차단 목록을** 선택합니다.

커스텀 속성 페이지에서 차단된 여러 개의 선택된 커스텀 속성이 있습니다.]({% image_buster /assets/img_archive/blocklist_custom_attr.png %})

최대 300개의 커스텀 속성과 300개의 커스텀 이벤트를 차단 목록에 추가할 수 있습니다. 특정 기기 속성의 수집을 방지하려면 [소프트웨어 개발 키트 가이드를]({{site.baseurl}}/developer_guide/platform_integration_guides/sdk_primer/#blocking-data-collection) 참조하세요. 

{% alert important %}
**휴지통** 상태가 된 커스텀 속성 또는 커스텀 이벤트는 삭제될 때까지 차단 목록 한도에 포함됩니다.
{% endalert %}

고객 이벤트 및 속성이 차단 목록에 추가되면 다음 사항이 적용됩니다:

- Braze로 전송된 데이터는 처리되지 않으며, 차단된 이벤트 및 속성은 더 이상 데이터 포인트로 간주되지 않습니다.
- 다시 활성화하지 않으면 기존 데이터를 사용할 수 없습니다.
- 차단된 이벤트 및 속성은 필터나 그래프에 표시되지 않습니다.
- 활성 캔버스의 초안 내에서 차단 목록에 있는 데이터에 대한 참조는 잘못된 값으로 로드되어 오류가 발생할 수 있습니다.
- 차단된 이벤트 또는 속성을 사용하는 모든 것이 보관됩니다.

이를 위해 Braze는 차단 목록 정보를 각 기기에 전송합니다. 엄청난 수의 이벤트와 속성(수십만 또는 수백만 개)을 차단하는 것은 데이터 집약적인 작업이 될 수 있으므로 이 점을 고려해야 합니다.

### 차단 목록 작성 시 고려 사항

많은 수의 이벤트와 속성을 차단 목록에 추가하는 것은 가능하지만 권장하지는 않습니다. 이벤트가 수행되거나 속성이 (잠재적으로) Braze로 전송될 때마다 해당 이벤트나 속성을 전체 차단 목록과 비교하여 확인해야 하기 때문입니다.

차단 목록을 위해 최대 300개의 항목이 소프트웨어 개발 키트로 전송됩니다. 300개 이상의 항목을 차단 목록에 추가하면 이 데이터는 소프트웨어 개발 키트에서 전송됩니다. 향후 이벤트 또는 속성을 사용할 필요가 없는 경우 다음 릴리스에서 앱 코드에서 제거하는 것을 고려하세요. 차단 목록의 변경 사항이 전파되는 데 몇 분 정도 걸릴 수 있습니다. 언제든지 차단 목록 이벤트 또는 속성을 다시 활성화할 수 있습니다.

## 고객 데이터 삭제하기

타겟팅 캠페인 및 세그먼트를 구축하다 보면 더 이상 커스텀 이벤트나 커스텀 속성이 필요하지 않을 수도 있습니다. 예를 들어, 일회성 캠페인의 일부로 특정 커스텀 속성을 사용한 경우 해당 데이터를 [차단 목록에](#blocklisting-custom-attributes-custom-events-and-products) 추가한 후 삭제하고 앱에서 해당 참조를 제거할 수 있습니다. 모든 데이터 유형(문자열, 숫자, 중첩된 고객 속성 등)을 삭제할 수 있습니다.

{% alert important %}
커스텀 데이터를 삭제하려면 [Braze 관리자]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/#admin) 권한이 있어야 합니다.
{% endalert %}

커스텀 이벤트 또는 커스텀 속성을 삭제하려면 다음과 같이 하세요:

1. 삭제하려는 데이터 유형에 따라 **데이터 설정** > **커스텀 속성** 또는 **커스텀 이벤트로** 이동합니다.
2. 고객 데이터로 이동하여 <i class="fa-solid fa-ellipsis-vertical"></i> **작업** > **차단 목록을** 선택합니다.
3. 고객 데이터가 7일 동안 차단 목록에 등록된 후 <i class="fa-solid fa-ellipsis-vertical"></i> **작업** > **삭제를** 선택합니다.

### 삭제 작동 방식

커스텀 데이터를 삭제하면 다음과 같은 일이 발생합니다: 

- **커스텀 속성의 경우:** 모든 사용자 프로필에서 속성 데이터를 영구적으로 제거합니다.
- **커스텀 이벤트용:** 모든 사용자의 프로필에서 이벤트 메타데이터를 영구적으로 제거합니다.

삭제할 속성이나 이벤트를 선택하면 해당 상태는 **휴지통으로** 변경됩니다. 향후 7일 동안 해당 속성 또는 이벤트를 복원할 수 있습니다. 7일이 지나도 복원하지 않으면 데이터가 영구 삭제됩니다. 속성 또는 이벤트를 복원하면 차단된 상태로 다시 설정됩니다.

삭제해도 고객 프로필에 커스텀 데이터 오브젝트가 추가로 기록되는 것을 막을 수는 없으므로 이벤트 또는 속성을 삭제하기 전에 커스텀 데이터가 더 이상 기록되지 않는지 확인해야 합니다.

### 알아두어야 할 사항

커스텀 데이터를 삭제할 때는 다음 사항에 유의하세요:

* **삭제는 영구적으로** 이루어집니다. 데이터를 복구할 수 없습니다.
* 데이터는 Braze 플랫폼과 고객 프로필에서 제거됩니다.
* 삭제 후 커스텀 속성 이름 또는 커스텀 이벤트 이름을 '재사용'할 수 있습니다. 즉, 삭제 후 사용자 정의 데이터가 Braze에 "다시 나타나는" 경우, 이는 중지되지 않은 통합이 동일한 사용자 정의 데이터 이름으로 데이터를 전송하고 있기 때문일 수 있습니다.
* 삭제 후 고객 데이터가 다시 표시되는 경우 항목을 다시 차단 목록에 추가해야 할 수 있습니다. 커스텀 데이터가 삭제되므로 차단 목록 상태는 유지되지 않습니다.
* 커스텀 데이터를 삭제해도 [데이터 포인트가]({{site.baseurl}}/user_guide/data/data_points) 기록되지 않으며 사용할 새 데이터 포인트도 생성되지 않습니다.

## 데이터 유형 비교 강제 적용

Braze는 당사로 전송되는 속성 데이터의 데이터 유형을 자동으로 인식합니다. 그러나 하나의 속성에 여러 데이터 유형이 적용되는 경우 속성의 데이터 유형을 강제로 알려주면 해당 속성이 무엇인지 알 수 있습니다. **데이터 유형** 열의 드롭다운에서 선택합니다.

{% alert note %}
데이터 유형 강제 적용은 이벤트 속성정보나 구매 속성정보에는 적용되지 않습니다.
{% endalert %}

\![커스텀 속성 데이터 유형 드롭다운]({% image_buster /assets/img_archive/custom_events_view_data_type_dropdown.png %})

{% alert warning %}
속성에 대해 데이터 유형을 강제하도록 선택하면 지정된 유형이 아닌 데이터가 들어오는 경우 해당 유형으로 강제로 변환됩니다. 이러한 강제가 불가능한 경우(예: 문자가 포함된 문자열을 숫자로 강제 변환하는 경우) 데이터는 무시됩니다. 유형 변경 전에 수집된 모든 데이터는 이전 유형으로 계속 저장되며(따라서 세그먼트화할 수 없음), 영향을 받는 사용자 프로필의 속성 옆에 경고가 표시됩니다.
{% endalert %}

### 데이터 유형 강제 지정

| 강제 데이터 유형 | 설명 |
|------------------|-------------|
| 부울 | `1`, `true`, `t` (대소문자 구분 없음)의 입력은 다음과 같이 저장됩니다. `true` |
| 부울 | `0`, `false`, `f` (대소문자 구분 없음)의 입력은 다음과 같이 저장됩니다. `false` |
| 번호 | 정수 또는 플로트(예: `1`, `1.5`)는 숫자로 저장됩니다. |
