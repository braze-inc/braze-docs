# 로그 동기화 및 통합 가시성

> 클라우드 데이터 수집(CDI) **동기화 로그** 대시보드를 사용하면 CDI에서 처리한 모든 데이터를 모니터링하고, 데이터가 성공적으로 동기화되었는지 확인하고, '부정확하거나 누락된' 데이터의 문제를 진단할 수 있습니다.

동기화 로그에 액세스하려면 **데이터 설정** > **클라우드 데이터 수집으로** 이동하여 **동기화 로그** 탭을 선택합니다.

## 동기화 로그 대시보드 이해하기

기본 동기화 **로그** 페이지에서는 최근 동기화의 현재 또는 최종 상태별 개요를 포함하여 모든 동기화 실행에 대한 개괄적인 개요를 볼 수 있습니다.

* **실행 중입니다:** 현재 진행 중인 작업을 동기화합니다.  
* **성공:** 동기화 작업이 완료되고 모든 행이 성공적으로 처리되었습니다.  
* **부분적인 성공:** 동기화 작업이 완료되었지만 하나 이상의 행에서 오류가 발생했습니다.  
* **오류:** 완료하지 못한 작업 동기화.  
* **한도 초과:** 데이터 제한을 초과하여 처리가 중지된 동기화 작업.

![총 6,576건의 동기화 성공이 있는 동기화 로그의 예입니다.]({% image_buster /assets/img/cloud_ingestion/sync_logs1.png %}){: style="max-width:80%"}

동기화 로그는 각 동기화에 대해 다음과 같은 세부 정보도 제공합니다:

* **동기화 이름:** 동기화 구성의 이름입니다.  
* **ID를 실행합니다:** 특정 동기화 실행을 위한 고유 식별자입니다. 자세한 내용을 보려면 이 ID를 선택하세요. 이는 [CDI API 엔드포인트에서]({{site.baseurl}}/api/endpoints/cdi) 사용하거나 Braze Support에서 동기화 실행을 참조하는 데에도 사용할 수 있습니다.   
* **상태:** 실행 상태(성공, 부분 성공, 오류, 실행 중)입니다.  
* **소스에서 읽은 새 행입니다:** 이 실행을 위해 데이터 웨어하우스에서 가져온 새 행의 수입니다.  
* **결과:** 실행 내에서 성공 또는 실패한 행 수에 대한 분석입니다.  
* **마지막 "UPDATED_AT":** 이 동기화 실행에서 처리된 가장 최근 레코드의 타임스탬프입니다.  
* **실행 시작 시간:** 동기화 작업이 시작되었습니다.  
* **실행 기간:** 동기화 작업을 완료하는 데 걸린 총 시간입니다.

![동기화 로그에 대한 세부 정보.]({% image_buster /assets/img/cloud_ingestion/sync_logs3.png %}){: style="max-width:80%"}

### 데이터 보존

모든 행 수준 페이로드 및 오류 세부 정보를 포함한 동기화 로그 데이터는 최대 **30일** 동안 보관됩니다. 30일이 지난 로그는 자동으로 삭제됩니다.

처리된 행 수와 같은 동기화 실행 메타데이터는 최소 12개월 동안 보관됩니다.

### 동기화 로그 필터링하기

동기화 로그 테이블을 필터링하여 특정 실행을 찾을 수 있습니다. 사용 가능한 필터는 다음과 같습니다:

* **작업 시작일:** 미리 정의된 범위(예: '지난 30일') 또는 커스텀 날짜 범위를 선택합니다.  
* **상태:** 하나 이상의 동기화 상태를 기준으로 필터링합니다(예: **오류** 및 **부분 성공** 상태만 표시).  
* **동기화 이름:** 이름으로 특정 동기화를 검색합니다.

특정 동기화를 조사하려면 동기화 로그 표에서 관련 **실행 ID를** 선택합니다. **실행 세부정보** 페이지에서 동기화에 대한 세분화된 행별 로그를 확인할 수 있습니다.

### 실행 개요

이 섹션에는 시작 시간, 종료 시간, 지속 시간 및 소스에서 읽은 총 행 수를 포함하여 선택한 실행이 요약되어 있습니다. 또한 성공한 행 수와 오류가 발생한 행 수에 대한 카운트도 제공합니다.

### 이 실행에서 처리된 행

이 표는 동기화 중에 처리된 데이터에 대한 행 수준의 가시성을 제공하여 개별 레코드의 유효성을 검사할 수 있도록 합니다.

* **검색:** **사용자 ID로 검색** 막대를 사용하여 실행 결과 내에서 특정 사용자를 검색할 수 있습니다.  
* **사용 가능한 세부 정보:**   
  * **UPDATED_AT:** 해당 특정 행의 `UPDATED_AT` 열에 있는 타임스탬프입니다.  
  * **ID:** 사용자 식별자(예: `external_id`, `email`, 또는 `alias_name`)는 기록을 Braze 사용자 프로필과 일치시키는 데 사용됩니다.  
  * **상태:** 해당 행에 대한 개별 처리 상태**(성공** 또는 **오류**)입니다.  
  * **소스 페이로드:** 데이터 페이로드를 볼 수 있는 링크입니다.  
  * **오류 이유:** 상태가 **오류인** 경우 이 열에는 행이 동기화되지 않은 이유를 설명하는 메시징이 표시됩니다.

#### 페이로드 보기

특정 행에 대해 Braze로 전송된 정확한 데이터를 확인하려면 **소스** 페이로드 열에서 **페이로드 보기를** 선택합니다. 해당 사용자에 대해 처리된 원시 JSON 페이로드가 표시됩니다.

![동기화 로그의 특정 행에 대한 페이로드 예제입니다.]({% image_buster /assets/img/cloud_ingestion/sync_logs2.png %}){: style="max-width:80%"}

#### 동기화 로그 내보내기

**행 내보내기를** 선택하면 동기화 실행을 위한 행 수준 로그를 내보낼 수 있습니다. 그런 다음 내보내기 대상을 선택합니다:

* **오류가 있는 행입니다:** **오류** 상태인 행만 포함된 파일을 다운로드합니다.
* **모든 행:** 실행 중에 처리된 모든 행이 포함된 파일을 다운로드합니다.

{% alert important %}
모든 행에 대한 동기화 로그 내보내기는 현재 얼리 액세스 중입니다. Contact your Braze account manager if you're interested in participating in the early access.
{% endalert %}

대시보드에서 로그를 직접 내보낼 수 없습니다. 내보내기가 생성되면 로그 내보내기 파일을 다운로드할 수 있는 링크가 포함된 이메일이 전송됩니다. 

## Notifications

이메일 알림을 구성하여 CDI 동기화 상태에 대한 정보를 계속 받을 수 있습니다. 이러한 설정은 동기화를 만들 때 구성되며 언제든지 업데이트할 수 있습니다.

### 오류 알림

동기화 수준 오류에 대한 알림을 받으려면 이메일 주소 연락처가 하나 이상 있어야 합니다. 이러한 알림은 전체 동기화 작업이 실행되지 않거나 완료되지 않은 경우 또는 동기화에서 자격 증명 만료 또는 소스 테이블 누락과 같이 변경을 위해 사용자 개입이 필요한 오류가 발생한 경우에 전송됩니다.

추가 알림에는 다음이 포함됩니다:

- **행 오류입니다:** 동기화 내에서 특정 비율의 행이 업데이트에 실패하면 알림을 받습니다.
- **실패 임계값(%):** 경고를 트리거할 행 실패의 비율을 지정합니다. 예를 들어 이 값을 **1로** 설정하면 동기화 실행에서 1% 이상의 행에서 오류가 발생하면 알림이 전송됩니다.
- **동기화 성공:** 동기화가 성공적으로 완료되면 알림을 받습니다.
- **행이 변경되지 않더라도 알림을 보냅니다:** 동기화 실행이 성공적으로 완료되어 새 행이나 업데이트된 행이 하나도 처리되지 않은 경우에도 알림을 받습니다.
