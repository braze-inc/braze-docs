# 수집 사용 사례

> 이 기사는 승차 공유 앱이 어떤 사용자 데이터를 수집할지 결정하는 방법에 대한 사용자 데이터 수집 사용 사례를 다룹니다.

택시 또는 승차 공유 앱인 StyleRyde가 수집할 사용자 데이터를 결정하고자 한다고 가정해 보겠습니다. 다음 질문과 브레인스토밍 과정은 그들의 마케팅 및 개발 팀이 따르기에 훌륭한 모델입니다. 이 연습이 끝날 때쯤이면 두 팀 모두 목표를 달성하는 데 도움이 되는 맞춤 이벤트와 속성을 수집하는 것이 무엇인지에 대한 확고한 이해를 가져야 합니다.

## 사례 질문 1: 목표는 무엇인가요?

StyleRyde의 목표는 사용자가 그들의 앱을 통해 택시를 호출하도록 하는 것입니다.

## 사례 질문 2: 앱 설치 후 그 목표에 도달하기 위한 단계는 무엇인가요?

1. StyleRyde는 사용자가 등록 프로세스를 시작하고 개인 정보를 입력하도록 해야 합니다.
2. StyleRyde는 사용자가 SMS를 통해 받은 코드를 앱에 입력하여 등록 프로세스를 완료하고 확인하도록 해야 합니다.
3. StyleRyde는 사용자가 택시를 호출하려고 시도하도록 해야 합니다.
4. StyleRyde는 사용자가 택시를 호출할 때 사용할 수 있어야 합니다.

이러한 행동은 다음과 같은 맞춤 이벤트로 태그될 수 있습니다:

- 등록 시작
- 등록 완료
- 택시 호출 성공
- 실패한 택시 호출

이벤트를 구현한 후, StyleRyde는 다음과 같은 캠페인을 실행할 수 있습니다:

1. 등록을 시작했지만 특정 시간 내에 등록을 완료하지 않은 사용자에게 메시지를 보냅니다.
2. 등록을 완료한 사용자에게 축하 메시지를 보냅니다.
3. 특정 시간 내에 성공적인 택시 호출이 뒤따르지 않은 실패한 택시 호출을 한 사용자에게 사과 메시지와 프로모션 크레딧을 보냅니다.
4. 성공적인 택시 호출이 많은 파워 사용자에게 그들의 충성도에 감사하는 프로모션을 보냅니다.

## 사례 질문 3: 우리가 수집하고 메시지에 활용할 수 있는 다른 사용자 정보는 무엇입니까?

- 사용자에게 프로모션 크레딧이 있는지 여부?
- 사용자가 드라이버에게 주는 평균 평점?
- 사용자를 위한 고유 프로모션 코드?

이러한 특성은 다음과 같은 사용자 정의 속성으로 태그될 수 있습니다:

- 프로모션 크레딧 잔액 (소수형)
- 평균 드라이버 평점 (정수형)
- 고유 프로모션 코드 (문자열형)

이러한 속성을 사용하면 사용자에게 다음과 같은 캠페인을 보낼 수 있습니다:

1. 앱을 7일 동안 사용하지 않았고 계정에 프로모션 크레딧이 있는 사용자에게 앱으로 돌아와 크레딧을 사용하라고 상기시킵니다.
2. 우리의 메시지 템플릿과 [개인화 기능]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/overview/#personalized-messaging)을 사용하여 사용자에게 전송되는 메시지에 고유 프로모션 코드 속성을 끌어옵니다.
