# 데이터 포인트

> Braze에서 데이터는 곧 액션을 의미합니다. Braze에 도착하는 각 데이터는 세그먼트 멤버십을 업데이트하고, 메시징을 트리거 및 취소할 수 있으며, 메시징 개인화 등에 즉시 사용할 수 있습니다. 데이터 포인트는 비즈니스에 가장 영향력 있는 정보를 정의하는 데 도움이 됩니다. 추적할 정보를 신중하게 고려하면 사용자 경험에 가장 큰 영향을 미치는 데이터를 타겟팅할 수 있습니다.

데이터 포인트는 고객 프로필에 대해 기록된 정보를 기반으로 합니다. 이 정의에 대한 자세한 내용은 Braze 계약에서 확인할 수 있습니다. 고객 성공 팀이 고객의 필요에 맞는 데이터 모범 사례를 추천해 드립니다. 

## 정의

"데이터 포인트"는 세션 시작, 세션 종료, 커스텀 이벤트 또는 구매 기록과 최종 사용자 프로필에 설정된 속성에 의해 측정되는 청구 가능한 Braze 서비스 사용 단위를 의미합니다. 명확성을 위해, 한 시점에 최종 사용자의 프로필에 설정된 위에서 언급한 각 데이터(세션 시작, 세션 종료, 커스텀 이벤트, 구매 기록 및 모든 속성 등)는 각각 하나의 데이터 포인트로 간주합니다.

예를 들어 푸시 토큰, 기기 정보, 이메일 열기 및 푸시 알림 클릭과 같은 모든 캠페인 참여 추적 이벤트 등 Braze 서비스에서 기본값으로 수집하는 데이터 및 이벤트는 데이터 포인트로 *계산되지 않습니다*.

데이터 포인트 할당에 어떤 데이터가 포함되는지 알아보려면 이 도움말의 [소비 횟수](#consumption-count) 섹션을 참조하세요.

## 데이터 포인트 사용량 보기

데이터 포인트 사용량을 확인하려면 **설정** > **청구로** 이동하여 **총 데이터 포인트 사용량** 탭을 선택합니다.

데이터 포인트 대시보드 구성 요소에 대한 자세한 내용은 [청구를]({{site.baseurl}}/user_guide/administrative/app_settings/subscription_and_usage/) 참조하세요.

{% alert tip %}
**데이터 포인트를 낭비하지 마세요. 변경되는 데이터만 업데이트하세요!**<br><br>
데이터 포인트 사용량을 최소화하려면 동일하게 변경되지 않는 데이터를 보내지 않도록 프로그램을 설정하고 새롭고 관련성 있는 데이터만 Braze에 전달하는 것이 좋습니다. Braze는 온보딩 과정에서 이러한 모범 사례를 확립하기 위해 여러분과 협력할 것입니다.
{% endalert %}

## 소비 횟수

요약하면, 데이터 포인트는 사용자의 프로필 데이터가 업데이트되거나 특정 작업을 수행할 때 누적됩니다. 기본적으로 데이터 포인트는 각 사용자의 `session starts`, `session ends`, `events`, `purchases` 의 개수입니다.

다음 섹션에서 Braze가 데이터 포인트를 축적하는 방법에 대해 자세히 알아볼 수 있습니다. Braze 데이터 포인트의 미묘한 차이에 대해 궁금한 점이 있으면 Braze 계정 매니저가 답변해 드립니다.

다음 작업은 데이터 포인트를 기록하지 않습니다:
- Braze에서 사용자 삭제하기
- 메시징에서 연결된 콘텐츠 사용하기
- 전 세계 및 구독 그룹을 중심으로 구독 상태가 변경됩니다.
- [API 호출을]({{site.baseurl}}/api/endpoints/user_data/external_id_migration/post_external_ids_rename/) 통해 사용자의 외부 ID 이름 바꾸기
- 이벤트, 속성 또는 이벤트 속성정보 차단하기

### 특별한 상황

#### 배열

배열은 커스텀 속성 내에 저장된 항목의 정렬된 컬렉션입니다. 소비 측면에서 배열을 업데이트하는 데는 API 호출당 하나의 데이터 포인트가 필요합니다. 배열에 값을 점진적으로 추가하면 값당 하나의 데이터 포인트로 계산됩니다. 

{% alert tip %}
전체 배열을 한 번에 설정하면 단일 데이터 포인트로 계산됩니다. 따라서 어레이는 고객 프로필을 관련 정보로 최신 상태로 유지하고 비용을 절감할 수 있는 훌륭한 도구입니다.
{% endalert %}

#### 중첩 고객 속성

중첩 고객 속성은 속성 집합을 다른 속성의 속성으로 정의하는 객체를 말합니다. 개체의 모든 키는 데이터 포인트로 계산됩니다.

{% alert note %}
커스텀 속성 오브젝트를 `null` 로 업데이트하면 데이터 포인트도 소모됩니다.
{% endalert %}

#### CSV

CSV 가져오기를 통해 업로드한 커스텀 속성은 데이터 포인트에 포함됩니다. 그러나 세분화 목적의 CSV 가져오기( `external_id`, `braze_id`, 또는 `user_alias_name` 를 유일한 필드로 사용하여 가져오기)는 데이터 포인트를 기록하지 않습니다.

또한, 구독 상태 변경은 데이터 포인트를 기록하지 않으므로 CSV 파일의 `email_subscribe`, `push_subscribe`, `subscription_group_id` 또는 `subscription_state` 필드를 업데이트해도 요금이 부과되지 않습니다.
