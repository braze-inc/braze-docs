# 내보내기 문제 해결

> 이 페이지에는 Braze에서 CSV 또는 API를 통해 데이터를 내보낼 때 발생할 수 있는 오류 메시지가 나열되어 있습니다.

## 일반적인 오류

### 'AccessDenied' 

#### 자체 S3 버킷을 사용하는 경우

**자체 S3 버킷을** 사용하는 경우 이런 일이 발생할 수 있습니다:

- 예상한 개체가 더 이상 S3 버킷에 없으니 엔지니어링 팀에 문의하세요.
- Braze 대시보드에 구성된 S3 인증정보에 올바른 권한이 없는 경우, 팀에 올바른 인증정보를 확인하세요.

#### Braze S3 버킷을 사용하는 경우

**Braze S3 버킷을** 사용하는 경우 이런 일이 발생할 수 있습니다:

- 개체가 더 이상 존재하지 않습니다. 다운로드 링크가 만료되면 파일이 S3에서 자동으로 삭제되므로 만료된 내보내기 링크를 클릭한 경우 이러한 문제가 발생할 수 있습니다. 별도의 언급이 없는 한 파일은 4시간 후에 삭제됩니다. 이 경우 내보내기를 다시 실행하세요.
- S3가 개체를 제공할 준비가 되기 전에 다운로드 링크를 바로 선택했습니다. 몇 분 기다렸다가 다시 시도하세요. 보고서 규모가 클수록 일반적으로 시간이 더 오래 걸립니다. 
- 내보내기가 너무 커서 이 zip 파일을 만드는 동안 서버의 메모리가 부족했습니다. 이 경우 내보내기를 시도하는 사용자에게 자동으로 이메일이 전송됩니다. 이 문제가 지속적으로 발생하는 경우 앞으로는 자체 S3 버킷을 사용하는 것이 좋습니다.

### '만료된 토큰'

이메일이 충분히 오래 전에 전송되어 S3 파일이 만료된 경우 이런 일이 발생합니다. 별도의 언급이 없는 한 파일은 4시간 후에 삭제됩니다. 파일이 만료되기 전에 내보내기를 다시 실행하고 다운로드하세요.

이 문제는 데이터를 다운로드하는 S3 버킷에 더 이상 액세스할 수 없는 Braze로 인해 발생할 수도 있습니다. 다음 단계에 따라 S3 인증정보를 업데이트했는지 확인하세요.

### "파일이 더 이상 존재하지 않는 것 같습니다. 버킷에서 개체를 삭제하는 것이 없는지 확인해 주세요."

내보내기가 포함된 Braze의 이메일이 전송되는 시점과 S3가 실제로 개체를 서비스할 준비가 되는 시점 사이에 약간의 지연이 있을 수 있습니다. 이 오류가 표시되면 몇 분 정도 기다렸다가 다시 시도하세요.

### 필드에 아포스트로피 추가

CSV 내보내기에서 필드가 다음 문자 중 하나로 시작하면 Braze는 자동으로 필드에 아포스트로피를 추가합니다:

- -
- =
- +
- @

예를 들어, "-1943" 필드는 "'-1943"으로 내보냅니다. 이는 [`/users/export/segment` 엔드포인트에서]({{site.baseurl}}/api/endpoints/export/user_data/post_users_segment/) 반환되는 JSON 내보내기에는 적용되지 않습니다.
