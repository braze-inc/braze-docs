# 수집 사용 사례

> 이 문서에서는 차량 공유 앱이 어떤 사용자 데이터를 수집할지 결정하는 방법에 대한 사용자 데이터 수집 사용 사례를 다룹니다.

StyleRyde라는 택시 또는 차량 공유 앱이 어떤 사용자 데이터를 수집할지 결정하려고 한다고 가정해 보겠습니다. 다음 질문과 브레인스토밍 프로세스는 마케팅 및 개발 팀이 따라야 할 훌륭한 모델입니다. 이 연습이 끝나면 두 팀 모두 목표를 달성하기 위해 어떤 사용자 지정 이벤트와 속성을 수집하는 것이 적합한지 확실히 이해하게 될 것입니다.

## 사례 질문 1: 목표는 무엇인가요?

사용자가 앱을 통해 택시를 호출하는 것을 목표로 하는 StyleRyde의 목표는 간단합니다.

## 사례 질문 2: 앱 설치 후 목표에 도달하기 위한 단계는 무엇인가요?

1. 스타일라이드는 사용자가 등록 절차를 시작하고 개인 정보를 입력해야 합니다.
2. 스타일라이드는 사용자가 SMS를 통해 받은 앱에 코드를 입력하여 등록 절차를 완료하고 인증해야 합니다.
3. 스타일라이드는 사용자가 택시 호출을 시도해야 합니다.
4. 사용자가 택시를 호출할 때 StyleRyde를 사용할 수 있어야 합니다.

이러한 작업은 다음과 같은 사용자 지정 이벤트로 태그를 지정할 수 있습니다:

- 등록 시작
- 등록 완료
- 성공적인 택시 호출
- 택시 호출 실패

이벤트를 구현한 후 스타일라이드는 다음과 같은 캠페인을 실행할 수 있습니다:

1. 등록을 시작했지만 특정 기간 내에 등록을 완료하지 않은 사용자에게 메시지를 보냅니다.
2. 등록을 완료한 사용자에게 축하 메시지를 보냅니다.
3. 택시 호출에 실패한 후 일정 시간 내에 택시 호출에 성공하지 못한 사용자에게 사과와 프로모션 크레딧을 보내세요.
4. 택시 호출에 성공한 횟수가 많은 파워 유저에게 프로모션을 보내 로열티에 대한 감사의 마음을 전하세요.

## 사례 질문 3: 메시지를 알리기 위해 수집하고 사용할 수 있는 다른 사용자 정보에는 어떤 것이 있나요?

- 사용자에게 프로모션 크레딧이 있는지 여부
- 사용자가 드라이버에게 부여하는 평균 평점은?
- 사용자를 위한 고유 프로모션 코드?

그런 다음 이러한 특성에 다음과 같은 사용자 지정 속성으로 태그를 지정할 수 있습니다:

- 프로모션 크레딧 잔액(소수점 유형)
- 평균 운전자 평점(정수형)
- 고유 프로모션 코드(문자열 유형)

이러한 속성을 통해 다음과 같은 사용자에게 캠페인을 보낼 수 있습니다:

1. 7일 동안 앱을 사용하지 않았고 계정에 프로모션 크레딧이 있는 사용자에게 앱으로 돌아와 크레딧을 사용하도록 알림을 보냅니다.
2. Using our message templates and [personalization features]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/overview/#personalized-messaging) to drag the unique promotion code attribute into messaging directed at users.
