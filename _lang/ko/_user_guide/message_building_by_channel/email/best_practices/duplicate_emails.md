# 중복 이메일

> 중복 이메일의 경우, 하나의 이메일이 구독 취소되면 해당 이메일 주소를 가진 다른 프로필(최대 100개 프로필)이 동일한 구독 상태를 반영하도록 업데이트됩니다. 이는 구독 취소 및 글로벌 이메일 구독 상태와 개별 구독 그룹 상태와 같은 구독 상태의 다른 변경 사항에 적용됩니다.

## 이메일 구독 업데이트

Braze는 이메일 캠페인이 전송될 때 중복 이메일 주소를 자동으로 확인하고 제거합니다. 이렇게 하면 이메일이 한 번만 전송되고 "중복 제거"가 이루어져, 여러 사용자 프로필이 공통 주소를 공유하더라도 동일한 이메일에 여러 번 도달하지 않도록 확인합니다.

{% alert tip %}
Braze가 제공하는 [사용자 이메일 구독 관리]({{site.baseurl}}/user_guide/message_building_by_channel/email/managing_user_subscriptions/#managing-user-subscriptions) 및 특정 구독 상태를 가진 사용자에게 캠페인을 타겟팅하는 도구에 익숙해지도록 하세요. 이 도구는 [스팸 방지 법]({{site.baseurl}}/help/best_practices/spam_regulations/#spam-regulations) 준수에 매우 중요합니다.
{% endalert %}

사용자가 이메일 주소를 공유하는 경우, 이러한 사용자 중 한 명을 업데이트하면 구독 변경 사항이 이 사용자들 간에 전파됩니다(최대 100명).

## 메시지 전송 동작

대상 사용자가 동일한 발송에 포함될 때 중복 제거가 발생하므로, 트리거된 캠페인(API 트리거 캠페인 제외) 및 캔버스는 서로 다른 사용자가 서로 다른 시간에 트리거 이벤트를 기록할 경우 동일한 이메일 주소로 여러 번 전송될 수 있습니다(재적격으로 인해 사용자가 제외될 수 있는 기간 내에서도).

## 예시

예를 들어, 사용자 A와 사용자 B가 이메일 `johndoe@example.com`을 공유하지만 그들의 프로필이 다른 시간대에 있을 경우, 캠페인 트리거 이벤트가 사용자의 시간대에서 전송을 포함하면 이메일 `johndoe@example.com`는 두 개의 이메일을 받게 됩니다.

사용자 A의 이메일 주소를 기존 사용자 B가 공유하는 다른 이메일 주소로 설정하거나 업데이트하면, **사용자가 이메일을 업데이트할 때 다시 구독** 설정이 켜져 있지 않는 한 사용자 A는 사용자 B에서 이미 존재하는 구독 상태를 상속받습니다.

{% alert important %}
API 호출을 통해 API 캠페인을 전송하면(API 트리거 캠페인 제외), 동일한 이메일 주소를 가진 여러 사용자가 세그먼트 청중에 지정된 경우, 우리는 호출에 나열된 만큼 그 주소로 전송합니다. 이는 API 호출이 의도적으로 구성되었다고 가정하기 때문입니다.
<br><br>
**API-트리거 캠페인**<br>
API-트리거 캠페인이 청중이 정의된 위치에 따라 중복 제거하거나 중복을 보낼 수 있습니다. <br>\- 대상 세그먼트에 중복 이메일이 있거나 API-트리거 호출의 [수신자 필드]({{site.baseurl}}/api/endpoints/messaging/send_messages/post_send_triggered_campaigns/) 내에서 중복 ID로 인해 중복 제거가 발생할 수 있습니다. <br>\- API-트리거 호출의 수신자 필드 내에서 별도의 사용자 ID를 직접 타겟팅하면 중복 이메일이 발생합니다.
{% endalert %}
