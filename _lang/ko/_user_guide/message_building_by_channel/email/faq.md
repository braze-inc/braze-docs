# 자주 묻는 질문

> 이 기사는 이메일에 대한 몇 가지 자주 묻는 질문에 대한 답변을 제공합니다.

### 이메일이 발송되고 여러 프로필이 동일한 이메일 주소를 가질 때 어떤 일이 발생합니까?

일치하는 이메일을 가진 여러 사용자가 캠페인을 받기 위해 세그먼트에 포함되어 있는 경우, 발송 시 해당 이메일 주소를 가진 무작위 사용자 프로필이 선택됩니다. 이렇게 하면 이메일이 한 번만 전송되고 중복이 제거되어 동일한 이메일 주소로 여러 번 전송되지 않도록 보장됩니다.

이 중복 제거는 타겟 사용자가 동일한 발송에 포함된 경우에 발생합니다. 트리거된 캠페인은 서로 다른 사용자가 서로 다른 시간에 트리거 이벤트를 기록하는 경우 동일한 이메일 주소로 여러 번 전송될 수 있습니다(재적격성으로 인해 사용자가 제외될 수 있는 기간 내에서도). 사용자는 캔버스 진입 시 이메일로 중복되지 않으므로, 사용자가 속도 제한된 진입으로 인해 약간 다른 시간에 진행되는 경우 캔버스의 첫 번째 단계 이상에서 중복되지 않을 수 있습니다. 주어진 이메일 주소에 연결된 사용자가 이메일을 열거나 클릭하면, 이메일 주소를 공유하는 모든 사용자 프로필이 캠페인을 열고 클릭한 것으로 표시됩니다.

#### 예외: API 트리거 캠페인

API 트리거 캠페인은 청중이 정의된 위치에 따라 중복을 제거하거나 중복을 전송합니다. 간단히 말해, 중복 이메일은 여러 세부 정보를 받기 위해 API 호출 내에서 별도의 `user_ids`로 직접 타겟팅되어야 합니다. 다음은 API 트리거 캠페인에 대한 세 가지 가능한 시나리오입니다:

- **시나리오 1: 타겟 세그먼트의 중복 이메일:** 대시보드의 청중 필터에서 그룹화된 여러 사용자 프로필에 동일한 이메일이 나타나면, 해당 이메일을 수신하는 프로필은 하나만 존재합니다.
- **시나리오 2: 수신자 객체 내의 서로 다른 `user_ids`에서 중복 이메일:** \`recipients\` 객체에서 참조된 여러 `External_user_IDs` 내에 동일한 이메일이 나타나면 이메일이 두 번 전송됩니다.
- **시나리오 3: 수신자 객체 내의 중복 user_ids로 인한 중복 이메일:** 동일한 사용자 프로필을 두 번 추가하려고 하면, 프로필 중 하나만 이메일을 받게 됩니다.

### 내 아웃바운드 이메일 설정에 대한 업데이트가 소급 적용되나요?

아니요. 아웃바운드 이메일 설정에 대한 업데이트는 기존 전송에 소급 적용되지 않습니다. 예를 들어, 이메일 설정에서 기본 표시 이름을 변경해도 활성 캠페인이나 캔버스의 기존 기본 표시 이름이 자동으로 교체되지 않습니다. 

### "좋은" 이메일 배달률이란 무엇인가요?

일반적으로 "마법의 숫자"는 배달된 메시지의 약 98%이며 반송률은 3%를 넘지 않아야 합니다. 배달률이 그 이하로 떨어지면 보통 우려할 만한 이유가 있습니다.

그러나 배달률이 98%를 초과하더라도 여전히 배달 문제를 겪을 수 있습니다. 예를 들어, 모든 반송이 특정 도메인에서 발생하는 경우, 이는 해당 제공업체에 대한 평판 문제가 있다는 명확한 신호입니다.

또한, 메시지가 배달되더라도 스팸으로 분류될 수 있으며, 이는 잠재적으로 심각한 평판 문제를 나타냅니다. 배달되는 메시지 수뿐만 아니라 사용자가 실제로 받은 메시지를 확인하기 위해 열람 및 클릭률도 모니터링하는 것이 중요합니다. 제공업체가 모든 스팸 사례를 보고하지 않기 때문에, 스팸 비율이 1%라도 우려와 추가 분석의 원인이 될 수 있습니다.

마지막으로, 귀하의 비즈니스와 발송하는 이메일 유형도 배달에 영향을 미칠 수 있습니다. 예를 들어, 주로 [거래 이메일]({{site.baseurl}}/api/api_campaigns/transactional_api_campaign)을 보내는 사람은 많은 마케팅 메시지를 보내는 사람보다 더 나은 비율을 기대해야 합니다.

### 내 이메일 배달 지표가 100%에 도달하지 않는 이유는 무엇인가요?

이메일 배달 메트릭스(배달, 반송 및 스팸 비율)는 소프트 반송된 이메일이 72시간의 재시도 기간 후에 배달되지 않는 경우 100%에 합산되지 않을 수 있습니다.

소프트 반송은 "메일박스 가득 차 있음", "서버 일시적으로 사용 불가"와 같은 일시적 또는 전환 문제로 인해 반송되는 이메일입니다. 소프트 반송된 이메일이 72시간 후에도 여전히 배달되지 않으면 이 이메일은 캠페인 배달 메트릭스에 포함되지 않습니다.

### 오픈 추적 픽셀이란 무엇입니까?

[오픈 추적 픽셀]({{site.baseurl}}/user_guide/administrative/app_settings/email_settings/#changing-location-of-tracking-pixel)은 발신자의 이메일 클릭 추적 도메인을 활용하여 이메일 오픈 이벤트를 추적합니다. 픽셀은 이메일의 HTML에 추가된 이미지 태그입니다. 가장 일반적으로 본문 태그 내의 마지막 HTML 요소입니다. 사용자가 이메일을 로드할 때, 브랜드 추적 도메인에서 이미지를 가져오기 위한 요청이 이루어지며, 이는 오픈 이벤트를 기록합니다.

### 이메일 캠페인이나 캔버스가 중지되면 어떻게 됩니까?

사용자는 캔버스에 들어가는 것이 금지되며 더 이상의 메시지가 전송되지 않습니다. 이메일 캠페인 및 캔버스의 경우, 중지 버튼이 즉시 전송이 중지된다는 의미는 아닙니다. 이는 전송 요청이 발송되면 사용자에게 배달되는 것을 중지할 수 없기 때문입니다.

### 왜 이메일 클릭 수가 오픈 수보다 더 많습니까?

다음과 같은 이유로 클릭 수가 오픈 수보다 더 많을 수 있습니다:
- 사용자가 단일 오픈 내에서 이메일 본문을 여러 번 클릭하고 있습니다.
- 사용자가 휴대폰의 미리보기 창 내에서 일부 이메일 링크를 클릭합니다. 이 경우, Braze는 이 이메일을 클릭되었지만 열리지 않은 것으로 기록합니다.
- 사용자가 이전에 미리 본 이메일을 다시 엽니다.

### 왜 이메일 오픈 및 클릭 수가 0입니까?

추적 도메인에 잘못된 구성으로 인해 이메일 오픈 및 클릭 수가 0일 수 있습니다. 이는 다음과 같은 이유로 발생할 수 있습니다:
- 추적 URL이 `http` 대신 `https`인 SSL 문제가 있습니다.
- 사용자 에이전트 문자열이 열기 이벤트, 클릭 이벤트 또는 둘 다에 대해 채워지지 않는 CDN에 문제가 있습니다.

### 서버 클릭을 유발할 때의 잠재적 위험은 무엇입니까?

과도하게 긴 메시지나 너무 많은 느낌표와 같은 이메일 메시지의 특정 요소는 이메일 보안 응답을 유발할 가능성이 있습니다. 이러한 응답은 보고서, IP 평판에 영향을 미치고 사용자가 구독 취소를 할 수 있습니다. 

이러한 응답을 처리하는 모범 사례에 대해서는 [클릭률 증가 처리]({{site.baseurl}}/help/help_articles/email/open_rates/)를 참조하십시오.

### Braze는 "구독 취소" 메트릭에 포함된 구독 취소 링크를 추적할 수 있습니까?

Braze는 이메일 내에서 다음 Liquid가 사용되는 경우 구독 취소 링크를 추적합니다: {%raw%}`${set_user_to_unsubscribed_url}`{%endraw%}

### 이메일에 "브라우저에서 이 이메일 보기" 링크를 추가할 수 있습니까?

아니요, Braze는 이 기능을 제공하지 않습니다. 이는 증가하는 대다수의 이메일이 모바일 장치와 현대 이메일 클라이언트에서 열리며, 이들은 이미지와 콘텐츠를 문제 없이 렌더링하기 때문입니다.

**우회:** 이와 동일한 결과를 얻으려면 이메일의 내용을 외부 랜딩 페이지(예: 웹사이트)에 호스팅할 수 있으며, 이는 이메일 본문을 편집할 때 **링크** 도구를 사용하여 구축 중인 이메일 캠페인에서 링크할 수 있습니다.
