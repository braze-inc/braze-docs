# Braze에 이메일 목록 가져오기 {#importing-email-lists}

> 성공적인 이메일 발신자로 자리 잡기 위한 중요한 단계는 고품질 이메일 목록을 확보하는 것입니다. 적절한 이메일 목록 관리는 배달 가능성을 향상시키고 보다 정확하고 깔끔한 캠페인 결과를 제공합니다.

## 가져오기 전에 고려 사항

### 이메일 목록 검증

Braze에 이메일 목록을 가져오기 전에 목록에 진짜 이메일 주소만 포함되어 있는지 확인하십시오. 높은 반송률은 이메일 발신자 평판에 해를 끼칠 수 있습니다. 

이메일 목록 정리 서비스는 이메일 주소가 올바른 구문을 따르고 이메일 주소의 물리적 속성을 가지고 있는지 확인하고, 이메일 도메인을 검증하며, 이메일 서버에 연결하여 이메일 주소가 존재하는지 인증함으로써 이를 수행할 수 있습니다.

### 이메일 주소가 이미 사용자와 연결되어 있는지 확인하십시오.

API 또는 SDK를 통해 사용자를 생성하기 전에 [`/users/export/ids`]({{site.baseurl}}/api/endpoints/export/user_data/post_users_identifier/) 엔드포인트를 호출하고 사용자의 `email_address`를 지정하십시오. 사용자 프로필이 반환되면 해당 Braze 사용자는 이미 해당 이메일 주소와 연결되어 있습니다.

새 사용자가 생성될 때 고유한 이메일 주소를 찾는 것을 강력히 권장하며, 동일한 이메일 주소를 가진 사용자를 전달하거나 가져오는 것을 피하십시오. 그렇지 않으면 메시지 전송, 타겟팅, 보고 및 기타 기능에 영향을 미치는 의도치 않은 결과가 발생할 수 있습니다.

예를 들어, 중복 프로필이 있지만 특정 사용자 정의 속성이나 이벤트가 하나의 프로필에만 존재한다고 가정해 보십시오. 여러 기준으로 캠페인이나 캔버스를 트리거하려고 할 때 Braze는 두 개의 사용자 프로필이 있기 때문에 사용자를 적격으로 식별할 수 없습니다. 또는 캠페인이 두 사용자가 공유하는 이메일 주소를 타겟팅하는 경우 **사용자 검색** 페이지는 두 사용자 프로필 모두가 캠페인을 수신한 것으로 표시됩니다.

### 참여한 사용자를 식별하십시오

가장 참여한 사용자를 식별하기 위해 먼저 깊이 이탈한 사용자를 제거하십시오. 6개월 이상 이메일에 참여하지 않은 사용자에게 이메일을 보내지 않는 것이 모범 사례입니다. 이는 이메일 발신자의 평판에 해를 끼칠 수 있습니다. 이메일 목록을 가져올 때, 지난 6개월 이내에 귀하의 이메일을 열어본 사용자만 포함해야 합니다.

장기적으로는 [선택 종료 정책]({{site.baseurl}}/user_guide/message_building_by_channel/email/best_practices/sunset_policies/)을 구현하는 것도 고려해야 합니다.

### 차단 목록 피하기

기존 이메일 제공업체에서 전환하는 경우, 차단 목록에서 사용자를 가져오지 않도록 하십시오. 차단 목록에는 구독 취소, 이메일을 스팸으로 표시하거나 하드 바운스된 이메일 주소가 포함됩니다.

## 가져오기 방법

이메일 목록이 준비되면 Braze REST API 또는 CSV 파일을 통해 사용자를 가져오는 여러 방법이 있습니다. 전용 [사용자 가져오기]({{site.baseurl}}/user_guide/data/user_data_collection/user_import/) 기사에서 자세히 알아보십시오.
