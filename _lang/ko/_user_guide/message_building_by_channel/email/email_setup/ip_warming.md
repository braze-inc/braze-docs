# IP 워밍

> IP 워밍은 이메일 받은편지함 제공자가 전용 IP 주소에서 메시징을 수신하는 데 익숙해지도록 하는 관행입니다. 이것은 모든 이메일 서비스 공급자(ESP)와 함께 이메일을 보내는 데 있어 매우 중요한 부분이며, Braze에서 메시지가 대상 받은편지함에 일관되게 높은 비율로 도달하는지 확인하는 표준 관행입니다.

IP 워밍은 인터넷 서비스 공급자(ISP)와 긍정적인 평판을 쌓는 데 도움을 주기 위해 설계되었습니다. 새로운 IP 주소가 이메일을 보내는 데 사용될 때마다 ISP는 해당 이메일이 사용자에게 스팸을 보내는 데 사용되지 않는지 프로그래밍 방식으로 모니터링합니다. IP와 도메인 평판을 신용 점수처럼 생각해 보세요. ISP는 이 평판을 사용하여 메일이 받은편지함에 도착할지 스팸 폴더에 도착할지를 결정합니다. 신용 점수와 마찬가지로 긍정적인 평판을 쌓는 데는 시간이 걸리며, 나빠진 평판을 다시 쌓는 데는 더 오랜 시간이 걸립니다.

## IP를 워밍할 시간이 없으면 어떻게 하나요?

**IP 워밍은 필수입니다.** IP를 적절하게 워밍업하지 않으면 이메일 패턴이 의심을 불러일으킬 경우 이메일 전달 속도가 크게 제한되거나 느려질 수 있습니다. 도메인 또는 IP가 ISP에 의해 차단될 수도 있으며, 이는 이메일이 사용자의 받은편지함 대신 스팸 폴더로 직접 이동하게 만들 수 있습니다. 따라서 IP를 적절하게 워밍하는 것이 중요합니다.

ISP는 스팸 의심이 발생할 때 사용자를 보호하기 위해 이메일 전달을 제한합니다. 예를 들어, 100,000명의 사용자에게 보낼 경우, ISP는 첫 시간 동안 그 사용자 중 5,000명에게만 이메일을 전달할 수 있습니다. 그런 다음 ISP는 열람률, 클릭률, 탈퇴, 스팸 신고와 같은 참여 측정기준을 모니터링합니다. 따라서 스팸 신고가 상당히 많이 발생하면 나머지 발송을 사용자의 받은편지함으로 전달하는 대신 스팸 폴더로 보내기로 선택할 수 있습니다. 

참여가 보통이면, 이메일이 스팸인지 여부를 더 확실하게 판단하기 위해 더 많은 참여 데이터를 수집하면서 이메일을 계속 제한할 수 있습니다. 이메일의 참여 측정기준이 매우 높다면, 이 이메일에 대한 제한을 완전히 해제할 수 있습니다. ISP는 이 데이터를 사용하여 이메일 평판을 생성하며, 이는 결국 이메일이 자동으로 스팸으로 필터링될지 여부를 결정합니다.

도메인 또는 IP가 ISP에 의해 차단된 경우, [메시지 활동 로그]({{site.baseurl}}/user_guide/administrative/app_settings/message_activity_log_tab/)의 메시지 로그에는 이러한 ISP에 이의를 제기하고 차단 목록에서 제외되기 위해 방문해야 하는 웹사이트에 대한 정보가 포함됩니다.

## IP 워밍 스케줄

전달 가능성을 지원하기 위해 IP 워밍 스케줄을 엄격히 준수할 것을 강력히 권장합니다. 또한, 일관된 확장은 전달 측정기준을 개선하므로 날을 건너뛰지 않는 것이 중요합니다. 기존 이메일 발송 이력과 전달 가능성 측정기준을 기반으로 스케줄을 선택하세요.

{% alert tip %}
계정 팀의 일원으로 전담 전달 가능성 리소스에 관심이 있으시면, Braze 계정 매니저에게 연락하여 추가 정보를 확인하세요.
{% endalert %}

대부분의 경우 최대 발송량이 아닌 평균 일일 발송량까지 워밍업하세요. ISP는 주로 최근 몇 주간의 발송 행동을 기반으로 평판을 평가하므로, 최대 발송량이 몇 달에 한 번만 발생하는 경우(예: 시즌 기간 중 700만 건), 발송일에 가까워질 때 해당 최대 발송량까지 점진적으로 늘릴 수 있습니다. 그러나 1~2주마다 최대 발송량에 도달하는 경우에는 처음부터 해당 최대 발송량까지 워밍업하세요.

IP 워밍이 완료되고 원하는 일일 볼륨에 도달하면, 매일 그 볼륨을 유지하는 것을 목표로 해야 합니다. 일부 변동은 예상되지만, 원하는 볼륨에 도달한 후 주 1회 대량 발송만 하는 것은 전달 측정기준과 발신자 평판에 부정적인 영향을 미칠 수 있습니다. 

{% alert important %}
대부분의 ISP는 평판 데이터를 30일 동안만 저장합니다. 메시지를 한 달 동안 보내지 않으면 IP 워밍 프로세스를 반복해야 합니다.
{% endalert %}

## 워밍 중 발송 제한 방법

Braze의 빌트인 사용자 제한 기능은 IP 주소를 워밍업하는 데 도움이 되는 유용한 도구입니다. 캠페인 생성 중 원하는 메시징 세그먼트를 선택한 후, [타겟 사용자]({{site.baseurl}}/user_guide/message_building_by_channel/email/html_editor/creating_an_email_campaign/#step-4-build-the-remainder-of-your-campaign-or-canvas) 단계에서 **고급 옵션** 드롭다운을 선택하여 사용자를 제한하세요. 워밍 스케줄이 진행됨에 따라 발송하는 이메일의 양을 늘리기 위해 이 한도를 점진적으로 높일 수 있습니다.

## 하위 도메인 세분화

많은 ISP와 이메일 접근 제공자는 더 이상 IP 주소 평판만으로 필터링하지 않습니다. 이러한 필터링 기술은 이제 도메인 기반 평판도 고려합니다. 이는 필터가 IP 주소만을 골라내는 것이 아니라 발신자의 도메인과 관련된 모든 데이터를 살펴본다는 것을 의미합니다. 이러한 이유로 이메일 IP를 워밍업하는 것 외에도 마케팅, 트랜잭션 및 회사 메일을 위해 별도의 도메인 또는 하위 도메인을 사용하는 것이 좋습니다. 

{% alert important %}
하위 도메인 세분화는 대량 발송자에게 특히 중요합니다. 이러한 발신자는 계정을 설정할 때 이 관행을 준수하는지 확인하기 위해 Braze 담당자와 협력해야 합니다.
{% endalert %}

도메인을 세분화하여 최상위 도메인을 통해 회사 메일을 보내고, 마케팅 및 트랜잭션 메일은 다른 도메인 또는 하위 도메인을 통해 보내는 것을 권장합니다.

## 모범 사례

이러한 모범 사례를 따르면 IP 워밍을 하지 않았을 때의 모든 부정적인 결과를 피할 수 있습니다:

### 이메일의 소량 발송부터 시작하세요

가능한 한 점진적으로 매일 보내는 양을 늘리세요. 갑작스럽고 대량의 이메일 캠페인은 ISP에 의해 가장 의심스럽게 여겨집니다. 따라서 이메일을 소량으로 보내기 시작하고 궁극적으로 보내려는 이메일의 양까지 점진적으로 늘려야 합니다. 각 ISP에서 개별적으로 IP를 워밍하고 있다는 점을 기억하세요. ISP는 서로 평판 데이터를 공유하지 않습니다. 워밍 볼륨을 구성할 때 단일 ISP에서 볼륨을 너무 빠르게 늘리지 않도록 주의하세요. 볼륨에 관계없이 안전을 위해 IP를 워밍하는 것을 권장합니다. [IP 워밍 스케줄](#ip-warming-schedules)을 참조하세요.

### 흥미로운 소개 콘텐츠를 제공하세요

첫 번째 콘텐츠가 매우 매력적이며 사용자가 클릭하고 이메일을 열람하고 참여할 가능성을 극대화하는지 확인하세요. IP를 워밍업할 때는 무차별적인 발송보다 잘 타겟팅된 이메일을 항상 선호하세요.

### 일관된 발송 주기를 설정하세요

IP 워밍이 완료되면 발송 주기를 생성하고 이메일을 하루 또는 며칠에 걸쳐 분산시키세요. 일관된 스케줄을 최대한 유지함으로써, 발송량이 며칠 이상 중단되거나 크게 감소할 경우 발생할 수 있는 IP 쿨다운을 방지할 수 있습니다. 

[IP 워밍 스케줄](#ip-warming-schedules)을 참조하여 특정 시간에 대량 발송하는 대신 더 긴 시간 프레임에 걸쳐 발송을 분산하세요.

### 이메일 목록을 정리하세요

이메일 목록이 깨끗하고 오래되거나 확인되지 않은 이메일이 없는지 확인하세요. [CASL 및 CAN-SPAM을 준수]({{site.baseurl}}/user_guide/administrative/privacy/spam_regulations/)하는 것이 이상적입니다.

### 발신자 평판을 모니터링하세요

IP 워밍 프로세스를 수행할 때 발신자 평판을 주의 깊게 모니터링해야 합니다. 다음 측정기준을 특히 주의 깊게 살펴보세요:
- **반송률:** 캠페인의 반송률이 3-5%를 초과하면, [Keep It Clean: The Importance of Email List Hygiene](https://www.braze.com/blog/email-list-hygiene/) 문서의 가이드라인에 따라 목록의 청결도를 평가해야 합니다. 또한, 참여하지 않거나 비활성 이메일 주소로의 발송을 중단하기 위해 [일몰 정책]({{site.baseurl}}/user_guide/message_building_by_channel/email/best_practices/sunset_policies/)을 구현하는 것을 고려해야 합니다.
- **스팸 신고:** 캠페인이 스팸으로 신고되는 비율이 0.08%를 초과하면, 발송하는 콘텐츠를 재평가하고, 관심 있는 오디언스를 대상으로 하고 있는지 확인하며, 이메일이 그들의 관심을 끌 수 있도록 적절하게 작성되었는지 확인해야 합니다.
- **열람률:** 열람률은 받은편지함 배치를 위한 유용한 대리 지표입니다. 고유 열람률이 25%를 넘으면 받은편지함 전달률이 높다는 것을 의미하며, 이는 발신자 평판이 긍정적임을 나타냅니다.

{% alert tip %}
Braze는 IP를 워밍하는 데 [Intelligent Timing]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_timing/)을 사용하지 않을 것을 권장합니다. IP 워밍 캠페인은 가장 먼저 보내는 캠페인 중 하나이기 때문에, Braze는 최적의 발송 시간을 계산할 수 있을 만큼 사용자에 대한 충분한 정보를 가지고 있지 않습니다. 이 경우 Intelligent Timing이 적용된 모든 메시지는 대체 시간을 기본값으로 사용하여 어차피 같은 시간에 전송됩니다.
{% endalert %}

{% alert tip %}
IP 워밍 동안 도메인과 IP가 아직 긍정적인 평판을 확립하지 못했기 때문에 메일이 스팸 폴더로 전송되는 것은 정상입니다. 메일이 스팸 폴더에 도착하면 메일 관리자가 Braze 발신 도메인 및 IP를 회사의 허용 목록에 추가해야 할 수 있습니다.
{% endalert %}
