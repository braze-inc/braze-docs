# 자동화된 IP 워밍

> 자동화된 IP 워밍업을 사용하여 새 IP 주소에서 이메일 볼륨을 점진적으로 늘려 받은편지함 제공업체와 발신자 평판을 구축하세요.

{% include early_access_beta_alert.md feature='Automated IP warming' %}

## 작동 방식

자동화된 IP 워밍업을 사용하여 일일 전송량을 점진적으로 늘려 수신편지함 제공업체가 사용자의 전송 패턴을 학습하고 신뢰할 수 있도록 할 수 있습니다. 작업 공간에 도메인을 추가하면 홈 대시보드의 **중단한** 부분부터 시작하기 섹션에서 **자동화 IP 워밍업** 타일을 선택할 수 있으며, 이 타일은 60일 동안 여기에 유지됩니다.

Braze는 참여도가 가장 높은 가입자에게 먼저 전송하므로 일일 볼륨이 모범 사례에 맞는 속도로 증가할 수 있습니다. 그런 다음 Braze는 참여 및 전달 가능성 신호를 추적합니다. Braze가 문제를 감지하면 시스템이 자동으로 일정을 조정합니다.

{% alert note %}
IP 워밍은 한 번만 수행할 수 있습니다.
{% endalert %}

## 필수 조건

자동화된 IP 워밍업을 수행하려면 다음이 필요합니다:

- 확인된 하위 도메인 및 활성 IP 주소
- IP 워밍업 보기 및 실행 권한
    - "사용 데이터 보기"를 클릭하여 IP 워밍업 섹션을 확인합니다.
    - "이메일 템플릿 보기"를 클릭하여 IP 워밍업을 위한 이메일 템플릿을 보고 선택합니다.
    - "이메일 설정 관리"를 클릭하여 IP 워밍업을 시작합니다.
- "액세스 캠페인" 
- "캠페인에 대한 승인 워크플로가 켜져 있는 경우 '캠페인 승인 및 거부' 
    - Braze는 자동화된 IP 워밍업을 통해 생성된 캠페인을 자동으로 승인합니다.

## 자동화 IP 워밍업 계획 설정하기

### 1단계: 일정 설정

1. **정보 보내기** 섹션에서 **발신 주소에서** IP 주소를 워밍업할 대상을 선택합니다.
2. 현재 일일 전송량과 타겟팅 전송량을 입력합니다.
3. IP 워밍업 자동화를 위한 시작 날짜를 선택합니다. 이 날짜는 요금제 출시 후 최소 하루가 지나야 합니다.
4. 전송 시간을 입력합니다. 이렇게 하면 회사 시간대로 메시징이 전송됩니다.
5. Select **Next: 세그먼트** 로 이동하여 설정을 계속합니다.

![일정 세부 정보 예시.]({% image_buster /assets/img/automated_ip_warming_schedule.png %})

### 2단계: 세그먼트 선택 및 순위 매기기

1. 다음으로 타겟팅할 세그먼트를 선택합니다. IP 워밍업 중에는 참여도가 가장 높은 사용자에게 먼저 전송을 시작하고 시간이 지남에 따라 점차적으로 전송량을 늘리면서 참여도가 낮은 세그먼트를 천천히 추가합니다. 
2. 그런 다음 세그먼트를 드래그 앤 드롭하여 참여도가 높은 순서부터 낮은 순서로 순위를 매깁니다. 참여도가 높다는 것은 이메일을 지속적으로 열어 클릭하는 수신자를 의미합니다. 참여도가 낮은 수신자에는 이메일 참여가 일정하지 않거나 오랫동안 이메일에 참여하지 않은 수신자가 포함됩니다.
3. Select **Next: 메시지를** 으로 보내 설정을 계속합니다.

![자동화된 IP 워밍업 타겟팅을 위해 두 개의 세그먼트가 선택되었습니다.]({% image_buster /assets/img/automated_ip_warming_segment.png %})

### 3단계: 보낼 메시지를 선택합니다.

1. **이메일 템플릿 선택을** 선택합니다.
2. 보낼 메시징에 대한 이메일 템플릿을 선택합니다. IP 워밍업 중에 전송하는 콘텐츠는 열기와 클릭을 유도해야 합니다. 과거에 반응이 좋았던 콘텐츠를 선택하는 것이 좋습니다. 예를 들어 프로모션 오퍼를 사용하여 즉각적인 참여와 구매를 유도할 수 있습니다.
3. **템플릿 선택을 선택합니다**. 시작하기 전에 필요한 템플릿의 수를 계산합니다. 시스템이 중단 없이 전달 가능성 문제를 조정할 수 있도록 필요한 최소한의 템플릿보다 더 많은 템플릿을 제공하는 것이 좋습니다.
4. 필요한 템플릿 개수를 추가한 후 **다음을 선택합니다: 요약**.

{% alert important %}
IP 워밍업 세분화 툴에서 생성한 캠페인의 변경 사항(예: 예정 날짜, 세그먼트, 볼륨 변경)은 IP 워밍업 **요약** 페이지에 반영되지 않습니다.
{% endalert %}

### 4단계: 검토 및 시작

IP 워밍업 요금제의 세부 정보를 검토하세요. 그런 다음 **실행을** 선택합니다.

## 활성 IP 워밍 중

IP 워밍업 캠페인은 다음 날 바로 시작하지 않는 한 1~2일 전에 생성됩니다. 이러한 캠페인의 이름은 자동으로 다음 형식으로 지정됩니다: `IP Warming Day [X] - [Date] - [Template Name]`.

타겟팅한 일일 전송 목표에 도달하면 시스템에서 평판을 보호하기 위해 해당 날짜의 전송을 중지합니다. 

이 시스템은 다음과 같은 업계 벤치마크를 기반으로 사용자의 상태를 모니터링합니다: 

- 전달률이 90% 이하로 떨어짐
- 열람율 10% 미만
- 5% 이상의 바운스
- 0.1%를 초과하는 스팸 불만 비율

통계가 기준치보다 낮으면 발신자 평판에 대한 위험을 완화하기 위해 다음 날 볼륨을 늘리는 대신 시스템이 볼륨을 보류합니다.

## IP 워밍업 요금제 중지

Braze를 사용하면 IP 워밍업과 향후 캠페인 생성을 중지할 수 있지만 캠페인이 이미 활성화되었거나 향후 24시간에서 48시간 동안 예정된 경우 특정 캠페인을 수동으로 중지해야 할 수 있습니다. IP 워밍업 요금제를 중지하면 관련된 모든 캠페인도 중지됩니다.

그러나 중지되면 IP 워밍업을 다시 시작할 수 없습니다. 대신 중단한 부분부터 다시 시작할 수 있도록 새 계획을 세워야 합니다:

- 새로운 IP 워밍업을 시작하면 이전 트래커가 제거되므로 중지된 요금제의 기존 데이터를 다운로드하여 기록을 위해 보관합니다.
- **현재 일일 전송 볼륨을** 가장 최근 볼륨으로 업데이트하기
- 이전 캠페인을 이미 수신한 사용자를 제외하여 마지막 IP 워밍업에서 동일한 세그먼트를 사용하려는 경우 세그먼트에 필터를 추가합니다.

## IP 워밍업이 완료되면

IP 워밍업 마지막 날이 회사 시간대의 자정에 종료되면 IP 워밍업이 완료된 것으로 표시됩니다. 예를 들어 IP 워밍업 계획에서 마지막으로 전송한 캠페인이 오후 8시에 전송된 경우 계획은 4시간 후에 완료된 것으로 표시됩니다.

트래커는 요금제 종료 후 90일 동안 홈페이지에 유지됩니다. 90일이 지나면 트래커가 제거됩니다. 데이터를 다운로드하면 이러한 표준 이메일 측정기준이 포함됩니다:

- _발송됨_	
- _전달됨_	
- _반송 수_	
- _스팸 신고_	
- _총 열람_	
- _고유 열람_	
- _클릭됨_	
- _탈퇴됨_

하루에 볼륨 요건을 충족하는 데 사용되는 캠페인이 여러 개 포함되어 있는 경우 일일 보기에 집계됩니다.
