# 자동화된 IP 워밍

> 자동화된 IP 워밍을 사용하여 새로운 IP 주소에서 이메일 볼륨을 점진적으로 증가시켜 발신자 평판을 구축하세요.

{% multi_lang_include early_access_beta_alert.md feature='Automated IP warming' %}

## 작동 방식

자동화된 IP 워밍을 사용하여 일일 발송량을 점진적으로 늘릴 수 있으며, 이를 통해 받은편지함 제공자가 발신 패턴을 학습하고 신뢰할 수 있도록 합니다. 작업 공간에 도메인을 추가하면 홈 대시보드의 **자동화된 IP 워밍** 섹션에서 **중단한 곳에서 계속하기** 타일을 선택할 수 있으며, 이 타일은 60일 동안 유지됩니다.

Braze는 가장 참여도가 높은 구독자에게 먼저 발송하여 일일 볼륨이 모범 사례에 맞는 속도로 성장할 수 있도록 합니다. 그런 다음, Braze는 참여 및 전달 가능성 신호를 추적합니다. Braze가 문제를 감지하면 시스템이 자동으로 일정을 조정합니다.

{% alert note %}
IP 워밍은 한 번만 수행할 수 있습니다.
{% endalert %}

## 필수 조건

자동화된 IP 워밍을 수행하려면 다음이 필요합니다:

- 검증된 하위 도메인 및 활성 IP 주소
- IP 워밍을 보기 및 시작할 수 있는 권한
    - IP 워밍 섹션을 보려면 "사용 데이터 보기"
    - IP 워밍을 위한 이메일 템플릿을 보려면 "이메일 템플릿 보기"
    - IP 워밍을 시작하려면 "이메일 설정 관리"
- "캠페인 접근" 
- 캠페인에 대한 승인 워크플로우가 활성화된 경우 "캠페인 승인 및 거부" 
    - Braze는 자동화된 IP 워밍에서 생성된 캠페인을 귀하를 대신하여 자동으로 승인합니다.

## 자동화된 IP 워밍 계획을 설정하세요.

### 1단계: 일정을 설정하세요

1. **정보 전송** 섹션에서 IP 주소를 워밍업할 **보낸 사람 주소**를 선택하세요.
2. 현재 일일 발송량과 목표 발송량을 입력하세요.
3. 자동화된 IP 워밍을 위한 시작 날짜를 선택하세요. 이 날짜는 계획이 시작된 후 최소 하루 이상이어야 합니다.
4. 발송 시간을 입력하세요. 이것은 회사의 시간대에 맞춰 메시지를 전송합니다.
5. Select **Next: 설정을 계속하기 위한 세그먼트**입니다.

![예시 일정 세부정보.]({% image_buster /assets/img/automated_ip_warming_schedule.png %})

### 2단계: 세그먼트를 선택하고 순위를 매기세요

1. 다음으로, 타겟할 세그먼트를 선택하세요. IP 워밍 중에 Braze는 가장 높은 참여를 보이는 사용자에게 메시지를 전송하기 시작하고 시간이 지남에 따라 발송량을 점차 증가시키며 참여가 적은 세그먼트를 천천히 추가합니다. 
2. 그런 다음, 세그먼트를 드래그 앤 드롭하여 높은 참여에서 낮은 참여로 순위를 매기세요. 높은 참여는 귀하의 이메일을 지속적으로 열고 클릭하는 수신자를 포함합니다. 낮은 참여는 귀하의 이메일에 대한 참여가 일관되지 않거나 오랫동안 참여하지 않은 수신자를 포함합니다.
3. Select **Next: 설정을 계속하기 위한 메시지**입니다.

![자동화된 IP 워밍을 위해 타겟으로 선택된 두 개의 세그먼트입니다.]({% image_buster /assets/img/automated_ip_warming_segment.png %})

### 3단계: 전송할 메시지를 선택하세요

1. **이메일 템플릿 선택**을 선택하세요.
2. 전송할 메시지에 대한 이메일 템플릿을 선택하세요. IP 워밍 중에 보내는 콘텐츠는 열람 및 클릭을 유도해야 합니다. 과거에 좋은 반응을 얻었던 콘텐츠를 선택하는 것을 추천합니다. 예를 들어, 즉각적인 참여와 구매를 유도하기 위해 프로모션 제안을 사용할 수 있습니다.
3. **템플릿 선택** Braze는 시작하기 전에 필요한 템플릿 수를 계산합니다. 시스템이 중단되지 않고 전달 가능성 문제를 조정할 수 있도록 최소 요구 사항보다 더 많은 템플릿을 제공하는 것을 추천합니다.
4. 필요한 수의 템플릿을 추가한 후, **다음:**을 선택합니다. 요약**

{% alert important %}
IP 워밍 도구에서 생성된 캠페인에 대한 변경 사항(예: 예정된 날짜, 세그먼트, 볼륨 변경)은 IP 워밍 **요약** 페이지에 반영되지 않습니다.
{% endalert %}

### 4단계: 검토 및 시작

IP 워밍 계획의 세부 사항을 검토합니다. 그런 다음, **시작**을 선택합니다.

## 활성 IP 워밍 중

IP 워밍 캠페인은 1~2일 전에 생성되며, 다음 날 IP 워밍을 시작하는 경우를 제외합니다. 이 캠페인은 다음 형식으로 자동으로 이름이 지정됩니다: `IP Warming Day [X] - [Date] - [Template Name]`.

목표한 일일 발송 목표에 도달하면, 시스템은 평판을 보호하기 위해 그 날의 발송을 중단합니다. 

시스템은 다음 산업 벤치마크를 기반으로 귀하의 건강을 모니터링합니다: 

- 전달률이 90% 이하로 떨어짐
- 열람율이 10% 이하
- 반송률이 5% 초과
- 스팸 불만율이 0.1% 초과

통계가 우리의 기준치 이하일 경우, 시스템은 발신자 평판에 대한 위험을 완화하기 위해 다음 날 볼륨을 증가시키기보다는 볼륨을 유지합니다.

## IP 워밍 계획 중지

Braze는 IP 워밍과 향후 캠페인 생성을 중지할 수 있도록 하지만, 캠페인이 이미 활성화되었거나 다음 24~48시간 내에 예정되어 있는 경우, 특정 캠페인을 수동으로 중지해야 할 수 있습니다. IP 워밍 계획을 중지하면 모든 관련 캠페인도 중지됩니다.

그러나 중지된 경우, IP 워밍을 재개할 수 없습니다. 대신, 중단한 지점에서 다시 시작하기 위해 새로운 계획을 설정해야 합니다:

- 새로운 IP 워밍을 시작하면 이전 추적기가 제거되므로 기록을 위해 중단된 계획의 기존 데이터를 다운로드합니다.
- 가장 최근 볼륨으로 **현재 일일 발송 볼륨** 업데이트
- 이전 캠페인을 이미 받은 사용자를 제외하여 마지막 IP 워밍에서 동일한 세그먼트를 사용할 계획이라면 세그먼트에 필터 추가

## IP 워밍이 완료되면

IP 워밍의 마지막 날이 귀사의 시간대 자정에 끝날 때 IP 워밍이 완료된 것으로 표시됩니다. 예를 들어, IP 워밍 계획에서 마지막 캠페인이 오후 8시에 발송되면, 계획은 4시간 후에 완료된 것으로 표시됩니다.

추적기는 계획이 끝난 후 90일 동안 홈페이지에 유지됩니다. 90일 후, 추적기가 제거됩니다. 데이터 다운로드에는 다음과 같은 표준 이메일 측정기준이 포함됩니다:

- _발송됨_	
- _전달됨_	
- _반송 수_	
- _스팸 보고서_	
- _총 열람 수_	
- _고유 열람_	
- _클릭됨_	
- _탈퇴됨_

하루에 여러 캠페인이 볼륨 요구 사항을 충족하기 위해 사용되는 경우, 이는 일일 보기에서 집계됩니다.
