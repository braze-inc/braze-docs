# 애플리케이션-투-퍼슨 10자리 긴 코드

> A2P 10DLC는 미국에서 기업이 표준 10자리 긴 코드(10DLC) 전화번호를 통해 애플리케이션-투-퍼슨(A2P) 유형 메시지를 전송할 수 있도록 허용하는 시스템을 의미합니다. 이 등록된 긴 코드는 표준 긴 코드보다 더 높은 처리량, 더 나은 전달성 및 개선된 준수를 제공합니다.

{% alert important %}
현재 미국 고객에게 메시지를 전송하기 위해 미국 긴 코드를 보유하고 있거나 사용하는 모든 고객은 10DLC를 위해 긴 코드를 등록해야 하며, 이를 이행하지 않을 경우 모든 메시지에 대해 강력한 필터링을 경험하게 됩니다. 이 신청 과정은 4-6주가 소요됩니다.
{% endalert %}

## 왜 필요한가

10DLC 서비스는 긴 코드를 사용하여 A2P 메시징을 용이하게 하기 위해 특별히 만들어졌습니다. 역사적으로 긴 코드는 사람 간(P2P) 메시징을 위해 설계되었지만, 마케팅 목적으로 사용될 때는 기업이 제한된 처리량과 높은 필터링에 의해 제약을 받게 되었습니다. 

10DLC는 다음과 같은 문제를 완화하는 데 도움을 줍니다: 
- **더 높은 처리량**: 10DLC 번호는 일반 긴 코드보다 더 많은 메시지를 지원합니다.
- **더 나은 전달성**: 10DLC 번호는 A2P 트래픽을 위해 지정되므로, 이러한 번호로 전송된 메시지는 수신자에게 도달할 가능성이 더 높고, 일반 지역 긴 코드로 전송된 메시지보다 통신사에 의해 필터링되거나 거부될 가능성이 적습니다. 
- **개선된 준수**: 상업적 문자 메시징을 위해 지역 긴 코드를 사용하는 것은 [CTIA](https://api.ctia.org/wp-content/uploads/2019/07/190719-CTIA-Messaging-Principles-and-Best-Practices-FINAL.pdf) 지침에 위배됩니다. 10DLC 번호는 대량 메시징을 위해 지정되었으며, 브랜드가 짧은 코드에 의존하지 않고 산업 규정을 준수할 수 있도록 허용합니다.
- **예산 친화적**: 10DLC는 SMS를 보내거나 소량의 SMS를 보내고자 하는 기업에게 훌륭한 옵션입니다. 하루에 100,000개 이상의 메시지를 보내는 브랜드의 경우, 짧은 코드를 사용하는 것을 추천합니다. 

2019년 이후로, 통신사들은 상업 메시징을 위해 10DLC를 채택하기 시작했으며, 현재 Verizon과 AT&T이 10DLC를 지원하고 있으며, 모든 주요 통신사들이 곧 따를 것으로 예상합니다. 단기적으로는 불편을 초래할 수 있지만, 장기적으로는 고객들이 원치 않는 메시지로부터 보호받으면서 더 나은 배달률을 누릴 수 있습니다. 

## 알아야 할 사항

### 접근

A2P 10DLC로 긴 코드를 등록하는 데는 4-6주가 소요됩니다.

### 비용 

A2P 10DLC에 등록하는 데는 여러 종류의 수수료가 포함될 수 있습니다:

| 수수료 유형 | 설명 |
| -------- | ---------- |
| 등록 수수료 | 모든 주요 미국 네트워크에서 브랜드와 사용 사례를 등록할 때 적용되는 명목상의 수수료입니다. |
| 2차 심사 수수료 | 브랜드는 [브랜드 신뢰 점수](#trust-score)에 이의를 제기하고 전체 처리량을 개선하기 위해 2차 심사 프로세스를 요청할 수 있으며, 이 프로세스에는 수수료가 부과됩니다. |
| 통신사 수수료 | 10DLC 등록 후 사용자에게 발송되는 아웃바운드 SMS 및 MMS 메시지에 대해 통신사가 부과하는 수수료입니다. 2021년 10월 1일부터, 등록되지 않은 트래픽(표준 긴 코드)에 대한 통신사 수수료가 등록된 트래픽(10DLC)보다 더 높아질 것입니다. |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

업데이트된 [수수료 추정치](https://support.twilio.com/hc/en-us/articles/1260803965530-What-pricing-and-fees-are-associated-with-the-A2P-10DLC-service-)를 확인하려면 Twilio 10DLC 기사를 방문하세요.

### 처리량

귀하의 10DLC에 대한 메시지 처리량은 브랜드 신뢰 점수, 일일 메시지 한도 및 귀하의 메시징 사용 사례를 포함한 여러 요인에 따라 달라집니다.

#### 브랜드 신뢰 점수 {#trust-score}

캠페인 레지스트리(TCR)는 귀하의 회사와 관련된 특정 기준을 검토하고 각 브랜드의 메시징 처리량을 결정하는 신뢰 점수를 할당하기 위해 평판 알고리즘을 사용하는 제3자 기관입니다. 이 신뢰 점수는 고객이 미국 10DLC 메시징에 등록할 때 할당됩니다. 신뢰 점수가 높을수록 초당 메시지 수(MPS)가 더 좋아집니다. 

|     | 신뢰 점수 | AT&T | T-Mobile | Verizon |
| --- | ----------- | ---- | -------- | ------- |
| 높음 | 75-100 | 75 MPS | 75 MPS | 75 MPS |
| 중간 | 50-74 | 40 MPS | 40 MPS | 40 MPS |
| 낮음 | 1-49 | 4 MPS | 4 MPS | 4 MPS | 
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3  .reset-td-br-4 role="presentation" }

{% alert tip %}
Russel 3000 지수에 상장된 회사는 10DLC 등록 및 검토 후 높은 처리량과 브랜드 신뢰 점수를 부여받습니다.
{% endalert %} 

#### 일일 메시지 한도

일일 한도는 브랜드 신뢰 점수에 따라 2,000에서 200,000 메시지까지 다양하며 모든 긴 코드에 적용됩니다. 높은 브랜드 신뢰 점수는 초당 60개의 메시지 처리량을 제공하지만, 통신사가 설정한 일일 메시지 한도는 여전히 적용됩니다. 이는 브랜드의 일일 최대 메시지가 부과된 일일 한도보다 높을 경우 짧은 코드가 더 나은 옵션이 될 것임을 의미합니다. 

#### 메시징 사용 사례

처리량은 선택한 메시징 사용 사례의 유형에 따라 영향을 받습니다. 대부분의 고객은 표준 마케팅 또는 혼합 마케팅 사용 사례에 해당합니다. 덜 일반적인 다른 사용 사례는 서로 다른 처리량 값에 영향을 받을 수 있습니다.

사용 사례에 따라 최대 처리량을 달성하는 데 필요한 신뢰 점수가 달라질 수 있습니다. 다음 표는 표준 사용 사례와 일반 사용 사례 신뢰 점수 범위를 나열합니다. 긴급 서비스나 자선과 같은 특별한 사용 사례의 경우 [Twilio 문서](https://support.twilio.com/hc/en-us/articles/1260803225669-Message-throughput-MPS-and-Trust-Scores-for-A2P-10DLC-in-the-US)를 참조하십시오.

| 표준 사용 사례 | 설명 |
| ------------------ | ----------- |
| 마케팅 | 세일 및 한정 시간 제공과 같은 프로모션 콘텐츠. |
| 혼합 | 고객 관리와 같은 여러 사용 사례를 포함하는 캠페인. |

| 공익 광고 | 주어진 주제에 대한 인식을 높이기 위한 PSA. |
| 고객 관리 | 지원, 계정 관리 및 기타 고객 상호작용. |
| 배송 알림 | 배송 메시지의 상태. |

| 2FA | OTP와 같은 계정 인증. | 
| 보안 경고 | 침해된 시스템에 대한 알림. |
| 사기 경고 | 잠재적인 사기 활동에 대한 메시지. |

## 다음 단계

아직 10DLC에 등록하지 않은 고객은 고객 성공 관리자와 협력하여 긴 코드를 등록해야 합니다. **고객이 긴 코드를 등록하지 않으면 2021년 10월 1일부터 긴 코드를 사용하는 모든 A2P 발신자는 모든 메시지에 대해 심한 필터링을 경험하게 됩니다.** 고객 성공 관리자에게 연락하여 10DLC 등록을 시작하세요.
