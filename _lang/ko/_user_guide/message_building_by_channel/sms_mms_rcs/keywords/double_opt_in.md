# 더블 옵트인

> 더블 옵트인 기능은 사용자가 SMS, MMS 또는 RCS 메시지를 받기 전에 명시적으로 옵트인 의사를 확인하도록 요구할 수 있게 해줍니다. 이는 사용자가 채널에 참여할 가능성이 높거나 이미 참여하고 있는 사용자에게 초점을 맞추고, 준수 모범 사례를 따르는 데 도움이 됩니다.

더블 옵트인이 활성화되면, 사용자는 캠페인이나 캔버스에서 메시지를 받기 전에 명시적인 동의를 요청하는 메시지를 받습니다. 

1991년 전화 소비자 보호법(TCPA)의 명시적인 요구 사항은 아니지만, Braze는 사용자가 SMS, MMS 또는 RCS 프로그램의 일원이 되는 것에 대해 인식하고 동의하고 있음을 확인하기 위해 더블 옵트인을 구성할 것을 권장합니다. 준수에 대한 자세한 정보는 [Laws, regulations, and abuse prevention for SMS, MMS, and RCS]({{site.baseurl}}/user_guide/message_building_by_channel/sms_mms_rcs/laws_and_regulations/)를 참조하십시오.

## 더블 옵트인 워크플로

더블 옵트인은 인바운드 및 아웃바운드 옵트인 캠페인을 통해 명시적인 동의를 얻을 수 있게 해줍니다.

### 아웃바운드

사용자가 전화번호를 제공하면, 동의를 요청하는 메시지를 받습니다.

\![브랜드가 "BRAND 텍스트 업데이트에 오신 것을 환영합니다!"라고 문자 메시지를 보내는 아웃바운드 SMS 메시지의 스크린샷 최신 제안을 위한 주 1회 메시지. 옵트인하려면 Y를 회신하십시오.", 사용자가 "Y"로 회신하고, 브랜드가 "감사합니다!"라고 응답합니다. 이제 BRAND 알림에 옵트인되었습니다. 여기 첫 구매 시 10% 할인 코드 SMS10이 있습니다!"]({% image_buster /assets/img/double_opt_in_outbound.png %}){:style="max-width:40%;"}

### 인바운드

사용자가 옵트인 키워드를 포함한 메시지를 보내면, 동의를 요청하는 메시지를 받습니다.

\![사용자가 "JOIN"을 보내고 "우리의 SMS 프로그램에 JOIN하고 싶다면 Y를 회신하십시오."라는 응답을 받는 인바운드 SMS 메시지의 스크린샷. 3msg/주, 언제든지 STOP을 입력하여 중지할 수 있으며, 그 후 "Y"를 다시 문자로 보내십시오.]({% image_buster /assets/img/double_opt_in_inbound.png %}){:style="max-width:40%;"}

## 더블 옵트인 활성화

더블 옵트인을 활성화하려면 해당 구독 그룹의 **글로벌 키워드** 테이블로 이동하여 **편집**를 클릭하십시오. **옵트인 키워드 카테고리**에서. 다음으로, 옵트인 방법을 선택하십시오 (**옵트인** 또는 **더블 옵트인**). **더블 옵트인**을 선택하면 추가 [구성 가능한 필드](#configurable-fields)을 표시하기 위해 페이지가 확장됩니다.

\![옵트인 방법 섹션에는 선택할 수 있는 두 가지 옵트인 방법이 있습니다: 옵트인 및 더블 옵트인.]({% image_buster /assets/img/double_opt_in_method.png %}){:style="max-width:50%;"}

### 구성 가능한 필드 {#configurable-fields}

| 카테고리   |    필드    | 설명   
| ----------- |----------- |---------------- 
| 옵트인 프롬프트 | 키워드 | 이것은 사용자가 옵트인 의사를 나타내기 위해 문자로 보낼 수 있는 키워드입니다. `START`은 필수 키워드입니다. 이 옵트인 프롬프트는 [구독 소스](#subscription-sources) 섹션에 나열된 소스에 의해 구독 상태가 업데이트될 때 사용자에게도 전송됩니다.
| | 답장 메시지 | 이것은 사용자가 옵트인 키워드를 문자로 보낸 후 받을 초기 응답입니다 (예: "이 번호로부터 메시지를 받으려면 Y를 답장하십시오." Msg&Data 요금이 적용될 수 있습니다.” )
| 더블 옵트인 확인 | 키워드 | 이것은 사용자가 옵트인 의사를 확인하기 위해 답장할 수 있는 키워드입니다. 최소한 하나의 키워드가 필요합니다. 이 키워드는 **옵트인 프롬프트 응답 메시지** 필드에 지정되어야 합니다.
| | 답장 메시지 | 이는 사용자가 옵트인에 명시적으로 동의한 후 수신하게 되는 확인 응답입니다. 사용자의 구독 그룹 상태는 `Subscribed`로 설정됩니다.
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3 role="presentation" }

사용자가 옵트인 프롬프트를 받으면, 옵트인 의사를 확인할 수 있는 30일이 주어집니다. 사용자가 30일 기간 후에 구독을 원할 경우, 옵트인 키워드를 문자로 보내어 더블 옵트인 워크플로우를 다시 시작해야 합니다.

\![구성 가능한 필드는 두 개의 섹션, 옵트인 프롬프트와 더블 옵트인 확인으로 나뉘며, 각 섹션에는 키워드와 응답 메시지 필드가 있습니다.]({% image_buster /assets/img/double_opt_in_fields.png %})

## 구독 그룹 상태

사용자가 더블 옵트인 워크플로우를 완료한 후에야 그들의 [구독 그룹 상태]({{site.baseurl}}/sms_rcs_subscription_groups/)가 `Subscribed`로 업데이트됩니다. 사용자가 워크플로우를 시작했지만 완료하지 않으면, 그들은 `Unsubscribed` 상태로 남아 있으며 해당 구독 그룹에서 메시지를 받을 수 없습니다.

사용자는 [다른 출처에서 구독된 경우]({{site.baseurl}}/user_guide/message_building_by_channel/sms/sms_subscription_group#how-users-sms-subscription-groups-get-set) (예: REST API, SDK) 더블 옵트인 워크플로우에 들어갈 수 있습니다.

## 구독 출처 {#subscription-sources}

사용자는 수신 메시지 외부에서 발생하는 구독 업데이트를 통해서도 더블 옵트인 워크플로우에 들어갈 수 있습니다. 이 출처에는 REST API, SDK 및 선호 센터의 업데이트가 포함됩니다. 사용자가 이러한 출처를 통해 더블 옵트인 워크플로우에 들어가면, **옵트인 프롬프트 응답 메시지**를 받게 됩니다.

각 구독 출처는 다음 표에 설명된 대로 서로 다른 등록 동작을 가지고 있습니다.

출처    | 더블 옵트인 등록 동작   
----------- | -----------
SDK | 사용자는 Braze SDK를 통해 구독할 때 자동으로 이중 옵트인 워크플로에 들어갑니다.
REST API | 사용자는 구독 상태가 `/subscription/status/set`, `/v2/subscription/status/set` 또는 `/users/track`을 통해 설정되고 선택적 매개변수 `use_double_opt_in_logic`가 `true`로 전달될 때 워크플로에 들어갈 수 있습니다 (예: [{"subscription_group_id" : "subscription_group_identifier", "subscription_state" : "구독됨", "use_double_opt_in_logic": true}]). 이 매개변수가 생략되면 사용자는 이중 옵트인 워크플로에 들어가지 않습니다.
Shopify | 사용자의 구독 상태가 Shopify 통합에 의해 설정될 때 사용자는 이중 옵트인 워크플로에 들어가지 않습니다.
사용자 가져오기 | 사용자의 구독 상태가 사용자 가져움에 의해 설정될 때 사용자는 이중 옵트인 워크플로에 들어가지 않습니다.
[환경 설정 센터]({{site.baseurl}}/user_guide/message_building_by_channel/email/preference_center) | 사용자는 환경 설정 센터를 통해 구독할 때 자동으로 이중 옵트인 워크플로에 들어갑니다.
사용자 업데이트 단계 | 사용자는 사용자 업데이트 단계를 통해 구독 상태가 설정되고 선택적 매개변수 `use_double_opt_in_logic`가 `true`로 전달될 때 이중 옵트인 워크플로에 들어갈 수 있습니다. 이 매개변수가 생략되면 사용자는 이중 옵트인 워크플로에 들어가지 않습니다.
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

## 다국어 지원
수신 메시지의 경우, 이중 옵트인은 구독 그룹에 정의된 모든 언어에 대해 지원됩니다. 이는 다양한 언어로 자동 응답을 정의할 수 있음을 의미하며, Braze는 일치하는 키워드가 수신될 때 특정 언어와 관련된 자동 응답을 보냅니다.

수신 메시지 외부에서 발생하는 구독 업데이트를 통해 이중 옵트인 워크플로에 들어가는 사용자는 영어 키워드만 전송됩니다.
