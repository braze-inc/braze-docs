# 퍼지 옵트아웃

![iOS 메시지 채팅은 수신된 모호한 탈퇴 "제발 멈춰"에 대한 응답으로 발신 탈퇴 메시지를 보여줍니다.]({% image_buster /assets/img/sms/fuzzy1.jpg %}){: style="float:right;max-width:30%;margin-left:15px;"}

> Braze로 SMS, MMS, RCS를 전송하는 사용자는 정의된 관련 법률, 규정 및 업계 표준을 준수해야 합니다. 법률에 따르면 사용자가 "STOP"이라고 문자를 보내면 해당 메시징 프로그램과 관련된 모든 후속 메시징이 중단됩니다. Braze는 이러한 메시지를 자동으로 처리하고 사용자의 구독을 취소합니다.<br><br>퍼지 옵트아웃은 수신 메시지가 [옵트아웃 키워드]({{site.baseurl}}/user_guide/message_building_by_channel/sms_mms_rcs/keywords/optin_optout/)와 일치하지 않지만, 옵트아웃 의도를 나타내는 경우를 인식하려고 시도합니다. 퍼지 옵트아웃이 활성화되어 있고 인바운드 키워드 응답이 '퍼지'로 간주되면 Braze는 사용자에게 옵트아웃을 안내하는 응답 메시지로 자동 응답합니다.

현재는 [로컬 언어]({{site.baseurl}}/user_guide/message_building_by_channel/sms/keywords/keyword_handling/#multi-language-support)로 영어를 사용하여 생성된 옵트아웃 키워드만 지원됩니다.

## 모호한 것으로 간주되는 것은 무엇입니까?

인바운드 응답이 '모호한' 것으로 간주되는 기준은 다음과 같습니다.
- QWERTY 키보드에서 문자 하나를 왼쪽 또는 오른쪽의 문자로 바꾸면 일치하는 옵트아웃 키워드가 생성됩니다.
- 메시지의 하위 문자열이 옵트아웃 키워드와 일치합니다.

예를 들어, "Stpo" 또는 "Please stopppp"는 모호한 것으로 간주되며, 모호한 옵트아웃 응답이 전송됩니다. 사용자가 옵트아웃 키워드로 응답하면 구독 취소 이벤트가 트리거됩니다.

## 퍼지 옵트아웃 구성

퍼지 옵트 아웃을 구성하려면 구독 그룹 키워드 관리 페이지로 이동하십시오.

1. **오디언스** > **구독 그룹 관리**로 이동하여 **SMS/MMS/RCS** 구독 그룹을 선택하세요.
2. **글로벌 키워드에서** **옵트아웃** 카테고리를 찾아 연필 아이콘을 선택합니다.
3. **퍼지 옵트 아웃**을 켜서 활성화합니다.
4. 희미한 옵트아웃 응답을 원하는 대로 수정하십시오.

## 모호한 탈퇴 메시지에 대한 모범 사례

구독자에게 명확하고 준수하며 긍정적인 경험을 보장하기 위해, 모호한 탈퇴 메시지를 신중하게 구성하는 것이 중요합니다. 모호한 탈퇴 메시지의 주요 목적은 **지정된 탈퇴 키워드와 유사하지만 정확히 일치하지 않는 메시지를 보내는 사용자를 안내하는 것**입니다. 메시지는 사용자가 성공적으로 탈퇴하는 방법을 안내합니다.

### 중요한 고려 사항

{% alert warning %}
**절대** 모호한 탈퇴 메시지를 탈퇴를 확인하도록 구성하지 마세요. 모호한 탈퇴 메시지에는 사용자가 이미 성공적으로 탈퇴했다는 의미의 언어가 포함되어서는 안 됩니다. 예를 들어, **사용하지 마세요** "당신은 탈퇴되었습니다," "이 번호로부터 더 이상 메시지를 받지 않을 것입니다," 또는 "당신은 이제 탈퇴되었습니다"와 같은 문구를 사용하지 마세요.
{% endalert %}

모호한 탈퇴 메시지는 사용자가 성공적으로 탈퇴하기 전에 전송됩니다. 확인 언어를 사용하면 구독자가 탈퇴하지 않았음에도 불구하고 탈퇴했다고 믿게 되어 원치 않는 메시지가 계속 오고, 구독자가 불만을 느끼며, 상당한 준수 위험이 발생합니다.

{% alert warning %}
**절대** 모호한 탈퇴 메시지를 정확한 탈퇴 키워드와 동일하거나 유사하게 구성하지 마세요.
{% endalert %}

모호한 메시지가 정확한 탈퇴 키워드와 동일하거나 너무 가까운 경우(예: "STOP"이 정확한 키워드이고 모호한 메시지가 "탈퇴하려면 STOP을 문자로 보내세요"인 경우), 사용자의 초기 메시지가 실제로 탈퇴를 초래했는지 아니면 다른 조치를 취해야 하는지에 대한 혼란을 초래할 수 있습니다. 모호한 메시지는 항상 사용자가 취해야 할 조치를 명확히 해야 합니다.
