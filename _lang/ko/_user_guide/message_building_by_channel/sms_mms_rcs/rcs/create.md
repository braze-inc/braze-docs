# RCS 메시지 생성하기

> RCS 캠페인은 고객에게 직접 다가가 프로그래밍 방식으로 대화하는 데 유용합니다. Liquid 및 기타 동적 콘텐츠를 사용하여 사용자와 개인화된 경험을 만들고 브랜드에 대한 사용자 경험을 촉진하고 향상시키는 환경을 조성할 수 있습니다.

## RCS 메시지 생성하기

### 1단계: 메시지 구축 위치 선택하기

메시지를 캠페인으로 보내야 할지 캔버스로 보내야 할지 잘 모르시겠어요? 캠페인은 단일의 간단한 메시징 캠페인에 적합하며, 캔버스는 여러 단계의 사용자 여정에 적합합니다.

{% tabs %}
{% tab Campaign %}
1. **메시징** > **캠페인으로** 이동하여 **캠페인 만들기를** 선택합니다.
2. **SMS/MMS/RCS를** 선택하거나 여러 채널을 타겟팅하는 캠페인의 경우 **멀티채널을** 선택합니다.
3. 캠페인 이름을 명확하고 의미 있는 것으로 정하세요.
4. 필요에 따라 [팀과]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/teams/) [태그를]({{site.baseurl}}/user_guide/administrative/app_settings/tags/) 추가합니다.
   * 태그를 사용하면 캠페인을 더 쉽게 찾고 보고서를 구축할 수 있습니다. 예를 들어 [보고서 빌더를]({{site.baseurl}}/user_guide/analytics/reporting/report_builder/) 사용할 때 특정 태그를 기준으로 필터링할 수 있습니다.

{: start="5"}
5\. 캠페인에 필요한 만큼 배리언트를 추가하고 이름을 지정하세요. 추가된 각 배리언트에 대해 서로 다른 플랫폼, 메시지 유형 및 레이아웃을 선택할 수 있습니다. 이 주제에 대한 자세한 내용은 [다변량 및 A/B 테스트를]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/) 참조하세요.
- **SMS 및 RCS 배리언트 테스트**: Braze를 사용하면 하나의 캠페인에 SMS와 RCS 배리언트를 모두 포함할 수 있으므로 각각의 성능/성과를 비교할 수 있습니다. 메시지 작성의 첫 번째 단계에서 SMS 및 RCS 배리언트를 추가할 수 있습니다.

{: start="6"}
6\. RCS가 활성화된 [구독 그룹을]({{site.baseurl}}/sms_rcs_subscription_groups/) 선택합니다. 구독 그룹을 선택할 때 Braze는 자동으로 세그먼트화 필터를 추가하여 가입한 사용자만 캠페인을 수신할 수 있도록 합니다. 해당 구독 그룹에 속한 긴 코드와 짧은 코드만 타겟팅 사용자에게 SMS를 보내는 데 사용됩니다.
- **SMS 대체**: Braze는 RCS 발신자가 포함된 모든 구독 그룹에 대체용 SMS 코드를 하나 이상 포함할 것을 강력히 권장합니다. 이는 RCS 메시징이 전달되지 않을 경우 전달 가능성을 위해 중요합니다. 여기에는 사용자 기기의 비호환성, 특정 국가 또는 지역의 불완전한 통신사 서비스 범위 등이 원인으로 작용할 수 있습니다. SMS 대체를 인에이블하면 메시지가 사용자에게 계속 전달되어 사용자와 소통할 수 있는 기회를 놓치지 않을 수 있습니다.   

{: start="7"}
7\. SMS와 RCS 중에서 선택합니다. RCS 메시지를 작성하기 전에 보낼 채널을 선택하세요. 일반적으로 SMS보다 사용자 참여도가 높으므로 가급적 RCS를 사용하는 것이 좋지만, 최대한의 유연성과 제어권을 가질 수 있도록 항상 SMS로 전송할 수 있는 옵션을 제공합니다.

### 2단계: RCS 메시지 유형 선택하기

RCS 메시지 유형은 **텍스트** 또는 **미디어** 중에서 선택합니다.

텍스트 또는 미디어 메시지 유형 중에서 선택할 수 있는 옵션입니다.]({% image_buster /assets/img/rcs/rcs_text_media.png %}){: style="max-width:65%;"}

{% tabs %}
{% tab Text %}
이름에서 알 수 있듯이 RCS 문자 메시지는 텍스트를 매체로 사용합니다. 최대 160자까지 입력하면 RCS 메시지는 텍스트 전용(또는 "기본") 메시지로 요금이 청구됩니다. 160자를 초과하거나 리치 요소를 사용하는 경우 메시징은 리치(또는 "단일") RCS 메시지로 청구됩니다(글자 수 제한은 3072자로 증가). 

#### 기능

- 문자 메시지 유형에는 모든 SMS 기능이 포함됩니다. URL 클릭 추적은 고급 추적만 가능하므로 사용자 수준의 세분화된 보고가 가능합니다. 
- 또한 이제 랜딩 페이지 방문이나 주문과 같이 참여도가 높은 사용자 행동을 유도하는 **추천 답글** 및 **추천 작업** 버튼을 포함할 수 있는 옵션이 제공됩니다. 
    - **추천 답글은** 사용자가 클릭하여 텍스트 입력 시 미리 입력할 수 있는 추천 답변이 포함된 버튼으로, 제한된 선택지를 제공함으로써 사용자가 답변을 생각해야 하는 번거로움을 덜어줍니다. 
    - **추천 작업은** 사용자의 기기에서 작업을 시작하는 버튼입니다. 버튼은 일반적으로 두 개의 설명 단어 중 하나와 사용자가 버튼의 기능을 이해하는 데 도움이 되는 시각적 아이콘으로 구성됩니다. Braze 커런츠는 현재 OpenURL 추천 액션을 지원합니다. 이는 버튼을 선택한 사용자가 웹페이지 또는 기타 URL 식별자 위치로 리디렉션되는 URL과 유사한 기능을 합니다. 

유행하는 패션 스타일을 홍보하는 RCS 메시징에 대한 세 가지 추천 액션을 담은 GIF입니다: "동화 속 왕족", "엣지 있는 학계", "당신의 다른 스타일을 보여주세요".]({% image_buster /assets/img/rcs/rcs_suggested_actions.gif %}){: style="max-width:70%;"}

#### 고려 사항

- 텍스트 글자 수 제한의 경우, 텍스트 전용(기본) RCS 메시지의 경우 최대 160자, 리치(단일) RCS 메시지의 경우 최대 3072자까지 작성할 수 있습니다. 
- 버튼 제한의 경우 메시지당 최대 5개까지 버튼을 추가할 수 있습니다. 이러한 버튼은 추천 작업 또는 추천 답글이 될 수 있습니다.
- 텍스트 블록이 길고 버튼이 너무 많으면 사용자에게 불편을 줄 수 있으므로 가급적 간결함을 추구하는 것이 좋습니다. 
- 경우에 따라서는 SMS보다 RCS를 통해 긴 텍스트 전용 메시지를 보내는 것이 더 비용 효율적일 수 있습니다. 이는 긴 SMS 메시지는 여러 세그먼트로 세분화되어 각각 요금이 청구되는 반면, RCS 메시지는 메시지당 요금이 청구되기 때문입니다. 자세한 내용과 안내는 Braze 계정 매니저에게 문의하세요.
{% endtab %}

{% tab Media %}
RCS 미디어 메시지를 사용하면 SMS로는 불가능한 참여형 미디어 형식을 사용할 수 있습니다. 여기에는 이미지, 동영상, 설명서 파일이 포함됩니다. 이러한 미디어 옵션은 오디언스를 더욱 깊이 참여시키고 완전히 새로운 사용 사례를 인에이블먼트할 수 있도록 도와줍니다. 현재는 [미디어 라이브러리를]({{site.baseurl}}/user_guide/engagement_tools/templates_and_media/media_library/) 통한 이미지 업로드만 지원됩니다. 

#### 기능

- 미디어 메시지 유형은 텍스트, 추천 답글, 추천 작업 등 문자 메시지 유형에서 사용할 수 있는 모든 것을 지원합니다.
- JPEG 및 PNG 파일 형식을 포함한 이미지 파일을 지원합니다. 이미지 파일은 미디어 라이브러리에서 업로드를 통해 사용할 수 있습니다. 
- MP4, MPEG, MV4 파일 형식을 포함한 동영상 파일을 지원합니다. 동영상 파일은 메시지 작성기에서 직접 URL로 추가할 수 있습니다. 
- PDF 형식의 설명서 파일을 지원합니다. 설명서 파일은 메시지 작성기에서 직접 URL을 통해 추가할 수 있습니다. 

미디어 파일을 업로드할 수 있는 옵션이 있는 RCS 작성기입니다.]({% image_buster /assets/img/rcs/rcs_media_type.png %})

| 파일 유형 | 사양 |
| --- | --- |

| 이미지 파일 | 지원되는 파일 형식은 JPG, JPEG, GIF입니다.
| 비디오 파일 | 지원되는 파일 형식은 H263, M4V, MP4, MPEG-4, MPEG, WEBM입니다. |
| 설명서 파일 | 지원되는 파일 형식: PDF |

#### 고려 사항

RCS 메시지를 수신하는 사용자 경험은 대상 국가의 이동통신사 서비스 범위, 모바일 기기 하드웨어, 모바일 기기 운영 체제 등 여러 요인에 따라 약간씩 달라질 수 있습니다. 

일반적으로 RCS는 Android 기기와 더 자연스럽게 통합됩니다(이 방법은 주로 Google에서 구현했으며, 피어 투 피어 RCS 메시징은 Android 커뮤니티에서 광범위하게 채택되고 있습니다). 기기마다 렌더링 속도와 품질이 다를 수 있습니다.

### 3단계: RCS 메시지 작성하기

필요에 따라 언어와 개인화[(Liquid]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/), [연결된 콘텐츠]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/connected_content/), 이모티콘)를 사용하여 메시지를 작성하세요. 초과 요금이 부과될 가능성을 줄이려면 메시지 카피 한도를 준수하세요.

{% alert important %}
계속 진행하기 전에 [RCS 메시지 한도에 대한 가이드라인을](#step-2-select-your-rcs-message-type) 읽어보세요. RCS 메시지는 [메시지당 요금이]({{site.baseurl}}/sms_rcs_billing_calculators/) 부과되므로 각 유형의 RCS 메시징에 포함될 수 있는 내용의 뉘앙스를 이해하는 것이 좋습니다.
{% endalert %}

### 4단계: 메시지 미리보기 및 테스트하기

Braze는 항상 메시지를 보내기 전에 미리 보고 테스트할 것을 권장합니다. **테스트** 탭으로 이동하여 콘텐츠 테스트 그룹 또는 개별 사용자에게 테스트 RCS를 보내거나 Braze에서 직접 사용자로 메시지를 미리 볼 수 있습니다.

### 5단계: 나머지 캠페인 또는 캔버스 구축하기

그런 다음 나머지 캠페인 또는 캔버스를 구축합니다. 다음 섹션에서 도구를 사용하여 RCS 메시지를 구축하는 방법에 대한 자세한 내용을 참조하세요.

#### 5.1단계: 전달 일정 또는 트리거 선택

RCS 메시지는 예약된 시간, 동작 또는 API 트리거에 따라 전달될 수 있습니다. 자세한 내용은 [캠페인 예약하기를]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/) 참조하세요.

실행 기반 전달의 경우 캠페인 기간과 조용한 시간을 설정할 수도 있습니다.

사용자가 캠페인을 다시 수신할 수 있도록 허용하거나 최대 게재빈도 설정 규칙을 인에이블하는 등 전달 제어를 지정합니다.

#### 5.2단계: 타겟팅할 사용자 선택하기

세그먼트 또는 필터를 선택하여 오디언스 범위를 좁혀 사용자를 타겟팅하세요. 이미 구독 그룹을 선택했어야 하는데, 이 그룹은 사용자가 원하는 커뮤니케이션 수준이나 카테고리에 따라 사용자를 좁혀줍니다.

그런 다음 세그먼트에서 더 큰 오디언스를 선택하고 선택적 [필터를]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters/) 사용하여 해당 세그먼트를 더 좁힐 수 있습니다. 현재 대략적인 세그먼트 인구가 어떻게 보이는지 자동으로 미리 볼 수 있습니다. 정확한 세그먼트 멤버십은 항상 메시지 전송 직전에 계산된다는 점에 유의하세요.

#### 5.3단계: 전환 이벤트 선택하기

Braze를 사용하면 사용자가 캠페인을 수신한 후 특정 액션, 즉 전환 이벤트를 얼마나 자주 수행하는지 추적할 수 있습니다. 사용자가 지정된 작업을 수행하면 전환이 카운트되는 기간을 최대 30일까지 허용할 수 있습니다.

전환 이벤트는 캠페인의 성공 여부를 측정하는 데 도움이 됩니다. 예를 들어
- 지역 타겟팅을 사용하여 사용자의 구매를 최종 목표로 하는 RCS 메시지를 트리거하는 경우 전환 이벤트를 **구매로** 설정하세요.
- 사용자를 앱으로 유도하려는 경우 전환 이벤트를 **세션 시작으로** 설정하세요.

특정 사용 사례에 따라 커스텀 전환 이벤트를 설정할 수도 있습니다. 캠페인의 성공을 진정으로 측정할 수 있는 창의적인 방법을 찾아보세요.
