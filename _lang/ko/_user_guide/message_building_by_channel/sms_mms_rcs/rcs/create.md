# RCS 메시지 만들기

> RCS 캠페인은 고객에게 직접 다가가 프로그래밍 방식으로 대화하는 데 유용합니다. Liquid 및 기타 동적 콘텐츠를 사용하여 사용자와 개인적인 경험을 만들고 브랜드에 대한 자연스러운 사용자 경험을 조성하고 향상시킬 수 있습니다.

## RCS 메시지 만들기

### 1단계: 메시지를 작성할 위치 선택

메시지를 캠페인으로 보내야 할지 캔버스로 보내야 할지 잘 모르시겠나요? 캠페인은 단일 타겟팅 메시징 캠페인에 더 적합하고, 캔버스는 다단계 사용자 여정에 더 적합합니다.

{% tabs %}
{% tab Campaign %}
1. **메시징** > **캠페인**으로 이동하여 **캠페인 만들기**를 선택합니다.
2. **SMS/MMS/RCS**를 선택하거나, 여러 채널을 타겟팅하는 캠페인의 경우 **멀티채널**을 선택하세요.
3. 캠페인의 이름을 명확하고 의미 있게 지정하세요.
4. 필요에 따라 [Teams]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/teams/)과 [태그]({{site.baseurl}}/user_guide/administrative/app_settings/tags/)를 추가하세요.
   * 태그를 사용하면 캠페인을 더 쉽게 찾고 보고서를 작성할 수 있습니다. 예를 들어, [보고서 빌더]({{site.baseurl}}/user_guide/analytics/reporting/report_builder/)를 사용할 때 특정 태그로 필터링할 수 있습니다.

{: start="5"} 
5. 캠페인에 필요한 만큼 배리언트를 추가하고 이름을 지정하세요. 추가된 각 배리언트에 대해 서로 다른 플랫폼, 메시지 유형 및 레이아웃을 선택할 수 있습니다. 이 주제에 대한 자세한 내용은 [다변량 및 A/B 테스트]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/)를 참조하세요.
- **SMS 및 RCS 배리언트 테스트**: Braze를 사용하면 단일 캠페인에 SMS와 RCS 배리언트를 모두 포함할 수 있으므로 각각의 성과를 비교할 수 있습니다. 메시지 작성의 첫 번째 단계에서 SMS 및 RCS 배리언트를 추가할 수 있습니다.

{: start="6"} 
6. RCS가 활성화된 [구독 그룹]({{site.baseurl}}/sms_rcs_subscription_groups/)을 선택합니다. 구독 그룹을 선택하면 Braze가 자동으로 세그먼트 필터를 추가하여 구독한 사용자만 캠페인을 수신할 수 있도록 합니다. 해당 구독 그룹에 속하는 긴 코드와 짧은 코드만 타겟 사용자에게 SMS를 보내는 데 사용됩니다.
- **SMS 대체**: Braze는 RCS 발신자가 포함된 모든 구독 그룹에 대체용 SMS 코드도 하나 이상 포함할 것을 강력히 권장합니다. 이는 RCS 메시지가 전달되지 않을 경우 전달 가능성을 위해 중요합니다. 사용자 기기의 비호환성, 특정 국가 또는 지역의 불완전한 통신사 서비스 범위 등이 원인이 될 수 있습니다. SMS 대체를 활성화하면 RCS가 전달되지 않을 때에도 SMS를 통해 RCS 메시지를 전달할 수 있으므로 사용자와 소통할 기회를 놓치지 않습니다.

{% alert note %}
MMS 대체는 지원되지 않습니다.
{% endalert %}

{: start="7"}
7. SMS와 RCS 중에서 선택합니다. RCS 메시지를 작성하기 전에 보낼 채널을 선택하세요. 일반적으로 SMS보다 사용자 참여도가 높기 때문에 가급적 RCS를 사용하는 것이 좋지만, 최대한의 유연성과 제어를 위해 항상 SMS로 전송할 수 있는 옵션도 제공합니다.

### 2단계: RCS 메시지 유형 선택

RCS 메시지 유형은 **텍스트** 또는 **미디어** 중에서 선택합니다.

![텍스트 또는 미디어 메시지 유형에서 선택할 수 있는 옵션입니다.]({% image_buster /assets/img/rcs/rcs_text_media.png %}){: style="max-width:65%;"}

{% tabs %}
{% tab Text %}
이름에서 알 수 있듯이 RCS 문자 메시지는 텍스트를 매체로 사용하는 데 중점을 둡니다. 최대 160자까지 입력하면 RCS 메시지는 텍스트 전용(또는 "기본") 메시지로 청구됩니다. 160자를 초과하거나 리치 요소를 사용하는 경우 메시지는 리치(또는 "단일") RCS 메시지로 청구됩니다(글자 수 제한은 3072자로 증가). 

#### 기능

- 문자 메시지 유형에는 모든 SMS 기능이 포함됩니다. URL 클릭 추적의 경우 고급 추적만 가능하므로 사용자 수준의 세분화된 보고가 가능합니다. 
- 또한 이제 랜딩 페이지 방문이나 주문과 같이 참여도가 높은 사용자 행동을 유도하는 매력적인 **추천 답글** 및 **추천 작업** 버튼을 포함할 수 있는 옵션이 제공됩니다. 
    - **추천 답글**은 사용자가 클릭하면 텍스트 입력란에 미리 채워지는 추천 응답이 포함된 버튼으로, 제한된 선택지를 제공하여 사용자가 답변을 직접 생각해야 하는 번거로움을 없애줍니다. 
    - **추천 작업**은 사용자 기기에서 작업을 시작하는 버튼입니다. 일반적으로 한두 개의 설명 단어와 버튼의 기능을 이해하는 데 도움이 되는 시각적 아이콘으로 구성됩니다. Braze는 현재 OpenURL 추천 작업을 지원합니다. 이는 URL과 유사하게 작동하며, 버튼을 선택한 사용자가 웹페이지 또는 기타 URL로 식별되는 위치로 리디렉션됩니다. 

![유행하는 패션 스타일을 홍보하는 RCS 메시지에 대한 세 가지 추천 작업의 GIF입니다: "동화의 왕족", "엣지 있는 학문", "다른 스타일을 보여줘".]({% image_buster /assets/img/rcs/rcs_suggested_actions.gif %}){: style="max-width:70%;"}

#### 고려 사항

- 텍스트 글자 수 제한의 경우, 텍스트 전용(기본) RCS 메시지는 최대 160자, 리치(단일) RCS 메시지는 최대 3072자까지 작성할 수 있습니다. 
- 버튼 제한의 경우 메시지당 최대 5개까지 버튼을 추가할 수 있습니다. 이러한 버튼은 추천 작업 또는 추천 답글이 될 수 있습니다.
- 텍스트 블록이 길고 버튼이 너무 많으면 사용자에게 불편을 줄 수 있으므로 가급적 단순하게 구성할 것을 권장합니다. 
- 경우에 따라 SMS보다 RCS를 통해 긴 텍스트 전용 메시지를 보내는 것이 더 비용 효율적일 수 있습니다. 긴 SMS 메시지는 여러 세그먼트로 나뉘어 각각 요금이 청구되는 반면, RCS 메시지는 메시지당 요금이 청구되기 때문입니다. 자세한 내용과 안내는 Braze 계정 매니저에게 문의하세요. 
{% endtab %}

#### 기능

- 미디어 메시지 유형은 텍스트, 추천 답글, 추천 작업 등 문자 메시지 유형에서 사용할 수 있는 모든 것을 지원합니다.
- JPEG 및 PNG 파일 형식을 포함한 이미지 파일을 지원합니다. 이미지 파일은 미디어 라이브러리에서 업로드하여 사용할 수 있습니다. 
- MP4, MPEG, MV4 파일 형식을 포함한 동영상 파일을 지원합니다. 동영상 파일은 메시지 작성기에서 URL을 직접 입력하여 추가할 수 있습니다. 
- PDF 형식의 문서 파일을 지원합니다. 문서 파일은 메시지 작성기에서 URL을 직접 입력하여 추가할 수 있습니다.

#### 고려 사항

RCS 메시지를 수신하는 사용자 경험은 대상 국가의 통신사 서비스 범위, 모바일 기기 하드웨어, 모바일 기기 운영체제 등 여러 요인에 따라 약간씩 다를 수 있습니다. 

일반적으로 RCS는 Android 기기와 더 자연스럽게 통합됩니다(이 방식은 주로 Google에서 구현했으며, 피어 투 피어 RCS 메시징은 Android 커뮤니티에서 광범위하게 채택되고 있습니다). 기기에 따라 렌더링 속도와 품질이 다를 수 있습니다.

### 3단계: RCS 메시지 작성

필요에 따라 언어 및 개인화([Liquid]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/), [연결된 콘텐츠]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/connected_content/), 이모지)를 사용하여 메시지를 작성하세요. 초과 요금이 부과될 가능성을 줄이려면 메시지 글자 수 제한을 준수해야 합니다.

{% alert important %}
계속 진행하기 전에 [RCS 메시지 제한에 대한 가이드라인](#step-2-select-your-rcs-message-type)을 읽어보세요. RCS 메시지는 [메시지당 요금이]({{site.baseurl}}/sms_rcs_billing_calculators/) 부과되므로 각 유형의 RCS 메시지에 포함할 수 있는 내용의 세부 사항을 이해하는 것이 좋습니다.
{% endalert %}

### 4단계: 메시지 미리보기 및 테스트

RCS 렌더링은 사용자의 운영체제, 기기 제조업체, 통신사 및 메시징 앱(예: Google Messages vs. Apple Messages)에 의해 제어되므로 메시지 모양이 다를 수 있습니다. 따라서 Braze에 표시되는 RCS 미리보기가 최종 사용자가 실제로 받는 것과 정확히 일치하지 않을 수 있습니다. 레이아웃, 미디어 크기, 버튼, 브랜딩 요소 또는 지원되는 기능에서 차이가 발생할 수 있습니다. Braze는 항상 메시지를 보내기 전에 미리 보고 테스트할 것을 권장합니다. **테스트** 탭을 사용하여 콘텐츠 테스트 그룹이나 개별 사용자에게 테스트 RCS를 보내고, Braze 내에서 사용자 관점으로 메시지를 미리 볼 수 있습니다. 그러나 최종 렌더링은 가능한 한 실제 기기에서 항상 검증해야 하며, Braze는 모든 OS, 기기 및 통신사 조합에서 완벽한 일치를 보장할 수 없습니다.


### 5단계: 나머지 캠페인 또는 캔버스 구축하기

그런 다음 나머지 캠페인 또는 캔버스를 구축합니다. RCS 메시지를 작성하기 위한 도구 활용 방법에 대한 자세한 내용은 다음 섹션을 참조하세요.

#### 5.1단계: 전달 스케줄 또는 트리거 선택

RCS 메시지는 예약된 시간, 동작 또는 API 트리거에 따라 전달할 수 있습니다. 자세한 내용은 [캠페인 스케줄 설정]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/)을 참조하세요.

실행 기반 전달의 경우 캠페인의 기간과 방해금지 시간을 설정할 수도 있습니다.

사용자가 캠페인을 다시 받을 수 있도록 허용하거나 최대 게재빈도 설정 규칙을 활성화하는 등 전달 제어를 지정합니다.

#### 5.2단계: 타겟팅할 사용자 선택

세그먼트 또는 필터를 선택하여 오디언스의 범위를 좁혀 사용자를 타겟팅할 수 있습니다. 사용자와 소통하려는 수준 또는 카테고리별로 사용자를 좁혀주는 구독 그룹을 이미 선택했어야 합니다.

{% multi_lang_include target_audiences.md %}

다음으로, 세그먼트에서 더 큰 오디언스를 선택하고 선택적 [필터]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters/)로 해당 세그먼트를 더 좁힙니다. 대략적인 세그먼트 인구의 미리보기가 자동으로 표시됩니다. 정확한 세그먼트 멤버십은 메시지가 전송되기 전에 항상 계산된다는 점을 유의하세요.

{% alert tip %}
RCS 리타겟팅을 사용하여 SMS 및 RCS 상호 작용을 기반으로 사용자를 타겟팅하는 데 관심이 있으신가요? [리타겟팅]({{site.baseurl}}/sms_mms_rcs_user_retargeting/)을 참조하세요.
{% endalert %}

#### 5.3단계: 전환 이벤트 선택

Braze를 사용하면 사용자가 캠페인을 수신한 후 특정 행동 또는 전환 이벤트를 얼마나 자주 수행하는지 추적할 수 있습니다. 사용자가 지정된 행동을 수행하면 최대 30일 동안 전환이 카운트되도록 허용할 수 있습니다.

전환 이벤트는 캠페인의 성공 여부를 측정하는 데 도움이 됩니다. 예를 들어:
- 지역 타겟팅을 사용하여 사용자의 구매를 최종 목표로 하는 RCS 메시지를 트리거하는 경우 전환 이벤트를 **구매**로 설정합니다.
- 사용자를 앱으로 유도하려는 경우 전환 이벤트를 **세션 시작**으로 설정합니다.

특정 사용 사례에 따라 커스텀 전환 이벤트를 설정할 수도 있습니다. 캠페인의 성공을 진정으로 측정하는 방법을 창의적으로 생각해 보세요.

## 팁

### 메시지 개인화를 위한 Liquid 사용

Liquid를 사용하려는 경우 선택한 개인화에 기본값을 포함해야 합니다. 그래야 수신자의 고객 프로필이 불완전한 경우 이름이나 일관된 문장 대신 빈 입력 안내 `Hi, !`가 표시되는 것을 방지할 수 있습니다.

### AI 카피 생성

매력적인 카피를 작성하는 데 도움이 필요하신가요? [AI 카피라이팅 어시스턴트]({{site.baseurl}}/user_guide/brazeai/generative_ai/copywriting/)를 사용해 보세요. 제품 이름이나 설명을 입력하면 AI가 메시징에 사용할 수 있는 사람이 쓴 것 같은 마케팅 문구를 생성합니다.

![AI 카피라이팅 어시스턴트를 열 수 있는 아이콘이 있는 메시지 작성기입니다.]({% image_buster /assets/img/rcs/rcs_ai_copywriter.png %}){: style="max-width:70%;"}

### 대화형 메시지 워크플로 만들기

대화형 메시지 워크플로를 사용하면 사용자에게 동적으로 응답하여 양방향 메시징 경험을 만들 수 있습니다. 워크플로를 구축하려면 캔버스를 만든 다음 추천 답글과 [행동 경로]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/action_paths/)를 결합하여 사용자가 선택한 답글에 따라 워크플로를 안내합니다.

1. 캔버스 빌더에서 여러 추천 답글이 포함된 RCS 메시지 단계를 만듭니다.

![추천 답글이 포함된 RCS 메시지 작성기입니다.]({% image_buster /assets/img/rcs/suggested_replies.png %})

{: start="2"}
2. 해당 메시지를 각 추천 답글에 대한 행동 그룹이 있는 행동 경로에 연결합니다.
3. 각 행동 그룹에 대해:
  - 트리거로 **인바운드 SMS 메시지 보내기**를 선택합니다.
  - 메시지 본문을 해당 추천 답글과 동일하게 설정합니다. 

![각 추천 답글에 대해 하나씩 세 개의 행동 그룹으로 구성된 행동 경로 단계입니다.]({% image_buster /assets/img/rcs/quick_reply.png %})

{: start="4"}
4. 각 행동 그룹을 RCS 메시지 단계에 연결한 다음 관련 추천 답글에 기반한 콘텐츠를 추가합니다.
5. 후속 메시지에 추천 답글을 추가하여 대화형 워크플로를 계속합니다.
6. 워크플로가 완료될 때까지 2~4단계를 반복합니다.

![두 개의 행동 경로가 있는 대화형 워크플로를 보여주는 캔버스입니다.]({% image_buster /assets/img/rcs/full_conversational_workflow.png %})
