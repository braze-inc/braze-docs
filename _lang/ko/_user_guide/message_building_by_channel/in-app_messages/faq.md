# 자주 묻는 질문

> 이 문서는 인앱 메시지에 대한 자주 묻는 질문에 대한 답변을 제공합니다.

### 인브라우저 메시지란 무엇이며 인앱 메시지와 어떻게 다른가요?

브라우저 내 메시지는 웹 브라우저로 전송되는 인-앱 메시지입니다. 인브라우저 메시지를 만들려면 인앱 메시지 캠페인 또는 캔버스를 만들 때 **Send To** 필드에서 **웹 브라우저**를 선택해야 합니다. 

### 기기가 오프라인 상태일 때 인앱 메시지가 표시되나요?

상황에 따라 다릅니다. 인앱 메시지는 세션 시작 시 전달되므로 기기가 오프라인 상태가 되기 전에 페이로드를 다운로드할 수 있으며, 오프라인 상태에서도 인앱 메시지를 표시할 수 있습니다. 페이로드가 다운로드되지 않으면 인앱 메시지가 표시되지 않습니다.

### 사용자가 이미 기기에 인앱 메시지 페이로드를 가지고 있고 메시지 만료가 변경된 경우, 기기에서 만료가 업데이트됩니까?

사용자가 세션을 시작하면 Braze는 사용자가 받을 수 있는 모든 인앱 메시지에 변경 사항이 있는지 확인하고 해당 메시지를 업데이트합니다. 따라서 만료가 변경되고 세션을 기록하면 인앱 메시지가 업데이트된 정보와 함께 기기로 전송됩니다.

### 인앱 메시지 캠페인에 대한 방해금지 시간을 어떻게 설정하나요?

방해금지 시간 기능은 인앱 메시지 캠페인에서 사용할 수 없습니다. 이 기능은 특정 시간 동안 메시지가 사용자에게 전송되지 않도록 하는 데 사용됩니다. 인앱 메시지 캠페인의 경우, 사용자가 앱 내에서 활동 중일 때만 인앱 메시지를 받게 됩니다. 

특정 시간 동안 인앱 메시지를 보내기 위한 해결 방법으로 다음 샘플 Liquid 코드를 사용하세요. 이렇게 하면 메시지가 지정된 시간대에서 오후 7시 59분 이후 또는 오전 8시 이전에 인앱 메시지가 표시되는 경우 중단될 수 있습니다.

{% raw %}
```liquid
{% assign time = 'now' | time_zone: ${time_zone} %}{% assign hour = time | date: '%H' | plus: 0 %}
{% if hour > 19 or hour < 8 %}
{% abort_message("Outside allowed time window") %}
{% endif %}
MESSAGE HERE
```
{% endraw %}

### 인앱 메시지에 대한 자격은 언제 계산되나요?

인앱 메시지의 자격은 전달 시 계산됩니다. 인앱 메시지가 오전 7시에 전송되도록 예약된 경우, 이 인앱 메시지에 대한 적격성은 오전 7시에 확인됩니다.

인앱 메시지가 표시되면 인앱 메시지가 다운로드되고 트리거되는 시점에 따라 자격이 결정됩니다.

### 템플릿 인-앱 메시지란 무엇입니까?

앱 내 메시지는 **캠페인 적격성을 표시하기 전에 다시 평가**가 선택되었거나 메시지에 다음 Liquid 태그 중 하나가 있는 경우 템플릿화된 앱 내 메시지로 전달됩니다:

- `canvas_entry_properties`
- `connected_content`
- {% raw %}`{sms.${*}}`{% endraw %}와 같은 SMS 변수
- `catalog_items`
- `catalog_selection_items`
- `event_properties`

이는 세션 시작 시 기기가 전체 메시지 대신 해당 인앱 메시지의 트리거를 받게 됨을 의미합니다. 사용자가 인앱 메시지를 트리거하면 사용자의 기기가 실제 메시지를 가져오기 위해 네트워크 요청을 보냅니다.

{% alert note %}
메시지는 기기가 인터넷에 접속할 수 없는 경우 전달되지 않습니다. 메시지가 전달되지 않을 수 있습니다 Liquid 논리가 해결되는 데 너무 오래 걸리면.
{% endalert %}

### 내 보관된 인앱 메시지 캠페인이 여전히 인앱 메시지 노출을 제공하는 이유는 무엇입니까?

이것은 인앱 메시지 캠페인이 활성화되었을 때 세그먼트 기준을 충족한 사용자에게 발생할 수 있습니다.

이를 방지하려면 캠페인 설정 중에 **캠페인을 표시하기 전에 캠페인 적격성을 재평가**를 선택하십시오. 

### Braze는 인앱 메시지 만료를 "1일 후"로 설정할 때 어떻게 계산하나요?

Braze는 사용자가 메시지를 받을 자격이 있는 후 24시간 후에 만료 시간을 하루로 계산합니다.
