# 퍼널 보고서

> 퍼널 리포팅은 고객이 캠페인이나 캔버스를 수신한 후의 여정을 분석할 수 있는 시각적 보고서를 제공합니다. ![퍼널 보고서 2]({% image_buster /assets/img/funnel_report/funnel_report2.png %}){: style="float:right;max-width:15%;margin-bottom:15px; border: 0"}

캠페인이나 캔버스에서 대조군 또는 여러 배리언트를 사용하는 경우, 다양한 배리언트가 전환 퍼널에 미치는 영향을 보다 세분화된 수준에서 파악하고 이 데이터를 기반으로 최적화할 수 있습니다.

![퍼널 보고서 1]({% image_buster /assets/img/funnel_report/funnel_report1.jpg %}){: style="max-width:80%;"}

## 퍼널 보고서 설정

![퍼널 보고서 5]({% image_buster /assets/img/funnel_report/canvas_campaign.png %}){: style="float:right;max-width:40%;border:0;margin-left:15px;"}

기존 활성 캠페인 및 캔버스에 대한 퍼널 보고서를 실행할 수 있습니다. 이 보고서에는 캠페인 수신자가 캔버스 또는 캠페인에 진입한 날로부터 1~30일 동안 진행한 일련의 이벤트가 표시됩니다. 사용자가 지정된 순서대로 이벤트를 수행하면 퍼널의 해당 단계를 통해 전환된 것으로 간주됩니다.

퍼널 리포팅은 대시보드의 다음 위치에서 사용할 수 있습니다:

- 특정 캠페인의 **캠페인 분석** 페이지
- 특정 캔버스의 **캔버스 세부 정보** 페이지에서 **배리언트 분석** 버튼 선택

{% alert important %}
퍼널 보고서는 [API 캠페인]({{site.baseurl}}/api/api_campaigns/)에는 사용할 수 없습니다.
{% endalert %}

### 1단계: 날짜 범위 선택

보고서의 기간을 선택하고(지난 6개월 이내), 데이터를 세분화하여 캠페인 또는 캔버스에 진입한 후 설정된 기간(최대 30일) 내에 퍼널 이벤트를 완료한 사용자를 확인할 수 있습니다. 다음 예시에서 퍼널은 지난 7일 동안 이 캠페인 또는 캔버스를 수신하고 3일 이내에 퍼널을 완료한 사용자를 찾습니다.

{% alert note %}
퍼널 완료 기간을 하루로 설정한 경우 퍼널 이벤트는 메시지 수신 후 24시간 이내에 발생해야 합니다. 단, 여러 날을 선택하면 타이밍 기간은 회사 시간대의 캘린더 일수로 계산됩니다.
{% endalert %}

![기간 드롭다운에서 "지난 7일"을 선택한 캔버스에 대한 퍼널 보고서입니다.]({% image_buster /assets/img/funnel_report/funnel_report5.png %}){: style="max-width:90%;"}

### 2단계: 퍼널 단계에 대한 이벤트 선택

모든 퍼널 보고서의 첫 번째 이벤트는 사용자가 메시지를 수신하는 것입니다. 이후 선택한 후속 이벤트는 해당 이벤트와 이전 이벤트를 수행한 사용자 수를 퍼널링합니다.

#### 사용 가능한 퍼널 보고서 이벤트

| 캠페인 | 세션 시작, 구매, 커스텀 이벤트 수행, 메시지 참여 이벤트 |
| 캔버스 | 세션 시작, 구매, 커스텀 이벤트 수행, 캔버스 단계 수신, 단계와 상호 작용 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

{% alert note %}
**단계와 상호 작용** 보고서 이벤트는 이메일 또는 푸시 메시징 채널을 사용하는 캔버스 단계에서만 사용할 수 있습니다.
{% endalert %}

![사용 가능한 보고서 이벤트의 드롭다운이 있는 캔버스에 대한 퍼널 보고서입니다.]({% image_buster /assets/img/funnel_report/funnel_report3.png %}){: style="max-width:80%;"}

퍼널 보고서를 사용하면 처음 설정한 전환 이벤트나 메시지 참여 이벤트 외에도 메시지의 성과를 비교할 수 있습니다. 따라서 처음에 추가하지 않은 전환 이벤트가 있더라도 퍼널을 사용하여 해당 이벤트의 전환을 추적할 수 있습니다.

예를 들어 14일 보고 기간을 선택한 다음 `Added to cart` 및 `Made purchase` 이벤트를 선택하면, 메시지를 받은 후 14일 이내에 장바구니에 추가한 사용자 수와 장바구니에 추가한 후 캠페인을 받은 후 14일 이내에 구매한 사용자 수를 모두 확인할 수 있습니다.

다른 예로, 이메일을 클릭한 후 전환한 사용자의 비율을 확인하고 싶을 수 있습니다. 이를 계산하려면 두 번째 이벤트를 이메일 클릭으로, 세 번째 이벤트를 전환 이벤트 수행으로 설정한 보고서를 만들면 됩니다.

**보고서 작성**을 선택한 후 퍼널 보고서를 생성하는 데 몇 분 정도 걸릴 수 있습니다. 이 시간 동안 보고서를 벗어나 대시보드의 다른 페이지로 이동할 수 있습니다. 보고서가 준비되면 대시보드 내 알림을 받게 됩니다.

## 퍼널 보고서 해석하기

퍼널 보고서에서 설정한 배리언트와 대조군을 직접 비교할 수 있습니다. 각 연속 이벤트는 이전 사용자 중 해당 작업을 완료하고 퍼널을 통해 전환한 비율을 표시합니다.

### 퍼널 보고서 구성요소

- **가로축**: 해당 작업을 수행한 메시지 수신자의 비율을 표시합니다.
- **차트**: 수신된 메시지 수, 이전 작업을 수행한 사용자 수, 선택한 작업, 전환율 및 대조군 대비 변화율을 표시합니다.
- **재생성 옵션**: 보고서를 다시 생성할 수 있으며 현재 보고서가 마지막으로 생성된 시점을 표시합니다.
- **배리언트**: 색상 열로 표시되며, 퍼널 보고서는 최대 8개의 배리언트와 대조군을 지원합니다. 기본적으로 **차트**에는 세 가지 배리언트만 표시됩니다. 더 보려면 나머지 배리언트를 수동으로 선택할 수 있습니다.

![퍼널 보고서 차트.]({% image_buster /assets/img/funnel_report/funnel_report4.jpg %})

**여러 배리언트가 있는 캠페인의 경우**: Braze는 각 이벤트 및 배리언트에 대한 측정기준과 대조군 대비 변화율이 포함된 표를 표시합니다. 전환율은 메시지 수신자당 해당 이벤트(및 후속 이벤트)를 수행한 사용자 수입니다.

**재자격이 있는 캠페인의 경우**: 사용자가 보고 기간 내에 캠페인을 두 번 이상 수신하는 경우, Braze는 해당 사용자가 기간 내에 캠페인을 처음 수신한 후 취한 행동을 기반으로 퍼널에 포함할지 여부를 결정합니다.
- 사용자가 재자격으로 두 번 이상 전환할 수 있지만, 퍼널 보고서는 사용자가 이벤트를 두 번 이상 수행하더라도 최대 한 번만 전환으로 집계하므로 퍼널과 표준 전환 값 사이에 차이가 있을 수 있습니다.

**재자격이 있는 다변량 캠페인의 경우**: 사용자가 보고 기간 동안 캠페인에서 여러 배리언트를 수신하는 경우, Braze는 이 사용자가 캠페인 배리언트를 처음 수신한 후 취한 행동에 따라 배리언트 퍼널에 포함할지 여부를 결정합니다. 즉, 동일한 사용자가 퍼널 기간 동안 여러 배리언트를 수신한 경우 여러 다른 배리언트에 포함될 수 있습니다.

{% alert important %}
고아 사용자는 퍼널 보고서에서 추적되지 않습니다. 익명 사용자가 캔버스 또는 캠페인에 진입한 후 나중에 `changeUser()` 메서드를 통해 식별되면 해당 사용자의 Braze ID가 변경됩니다. 퍼널 보고서는 진입 시점의 사용자 ID와 일치하는 후속 이벤트만 추적하며, ID가 변경된 후 사용자가 수행한 이벤트는 고려하지 않습니다. 즉, 식별된 후 사용자가 수행한 전환 이벤트는 퍼널 보고서에 포함되지 않습니다.
{% endalert %}

## 자주 묻는 질문

### 캔버스의 분석과 퍼널 보고서가 다른 이유는 무엇인가요?

캔버스 단계 수준 분석과 퍼널 보고서는 동일한 날짜 범위에 대해 서로 다른 범위 지정 규칙을 사용하므로 일치하지 않을 수 있습니다. 차이점은 각 보고서가 "어떤 이벤트를 포함할 것인가"를 정의하는 방식에 있습니다.

**캔버스 분석(배리언트 분석):** 날짜 범위는 이벤트가 **발생한 시점**을 기준으로 필터링합니다. 1월 1일~7일을 선택하면, 사용자가 캔버스에 언제 진입했는지와 관계없이 해당 기간 동안 발생한 모든 진입 및 전환 이벤트를 볼 수 있습니다. 1월 1일에 진입했지만 1월 8일에 전환한 사용자는 진입 1건과 전환 0건으로 표시되는데, 전환이 선택한 날짜 범위 밖에서 발생했기 때문입니다. 캔버스 단계에 설정된 전환 기간은 14일을 훨씬 넘을 수도 있으므로, 단계 수준 분석은 더 긴 기간에 걸쳐 전환을 캡처할 수 있습니다.

**퍼널 보고서:** 날짜 범위는 사용자가 캔버스에 **진입한 시점**을 기준으로 필터링합니다. 1월 1일~7일을 선택하면, 해당 기간 동안 진입한 모든 사용자를 포함한 후 진입 후 최대 14일(또는 퍼널에서 설정한 기간) 동안의 행동을 추적합니다. 1월 1일에 진입하고 1월 8일에 전환한 동일한 사용자는 진입 1건과 전환 1건으로 표시되는데, 전환이 진입 후 기간 내에 발생했기 때문입니다.

또한 퍼널 보고서는 이벤트가 지정된 순서대로 발생해야 하며 각 사용자를 최대 한 번만 집계하는 반면, 캔버스 분석은 순서 제약 없이 모든 전환과 참여를 집계합니다.
