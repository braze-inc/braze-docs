# API 사용 대시보드

> API 사용량 대시보드를 사용하면 Braze로 들어오는 REST API 트래픽을 모니터링하여 REST API 사용 내 추세를 파악하고 잠재적인 문제를 해결할 수 있습니다.

## API 사용 대시보드 정보

API 사용량 대시보드를 보려면 **설정** > **API 및 식별자로** 이동한 다음 **대시보드를** 선택합니다.

기본값 대시보드는 지난 하루(24시간) 동안 워크스페이스에 대해 수신된 모든 REST API 요청을 보여주는 보기입니다. 사용 사례에 따라 대시보드 컨트롤을 조정하여 트래픽을 필터링하거나 그룹화하고 대시보드의 시간 범위도 구성할 수 있습니다.

총 요청 130건, 성공률 70%, 실패율 30%의 API 사용 대시보드.]({% image_buster /assets/img/api_usage_dashboard/api_usage_dashboard.png %})

## 사용 가능한 측정기준

API 사용량 대시보드에는 다음과 같은 통계가 포함되어 있습니다:

| 측정기준         | 설명 |
|----------------|-------------|
| 총 요청 수 | 대시보드에 적용된 필터와 컨트롤을 감안하여 현재 작업 공간에 대해 Braze로 전송된 총 요청 수입니다. |
| 성공률   | 전체 요청 중 Braze가 `2XX` 성공 응답을 발행한 비율입니다. |
| 오류율     | Braze가 `4XX` 또는 `5XX` 오류 응답을 발행한 전체 요청의 비율입니다. |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

## 대시보드 사용하기

대시보드에 적용할 필터를 포함합니다: API 키, 엔드포인트, 응답 코드, 그룹 데이터 및 날짜.]({% image_buster /assets/img/api_usage_dashboard/filters.png %}){: style="float:right;max-width:35%;margin-left:15px;"}

### 필터

**필터를** 선택하여 필터를 적용하여 워크스페이스에 대한 REST API 트래픽의 보기 범위를 좁히세요:

- API 키
- 엔드포인트
- 응답 코드

### 그룹 데이터

데이터를 다양한 데이터 계열로 그룹화하여 다음과 같은 다양한 사용 패턴을 살펴볼 수 있습니다:

- 응답 코드(기본값)
- API 엔드포인트
- API 키
- 성공과 실패만

### 날짜

필요에 따라 날짜 필터를 조정하여 더 짧거나 더 긴 시간 범위를 표시합니다. 여기에는 다음이 포함됩니다:

- 오늘(기본값)
- 커스텀
- 지난 3시간
- 지난 6시간
- 지난 12시간
- 지난 24시간
- 어제
- 지난 7일
- 지난 14일
- 지난 30일
- 지난달부터 현재까지

{% alert note %}
**지난 3시간** 및 **지난 6시간** 옵션은 트래픽을 분 단위로 표시합니다. 기간이 길수록 5분, 1시간 또는 하루 단위로 트래픽이 표시됩니다.
{% endalert %}

## 고려 사항

API 사용 대시보드에는 Braze가 `2XX`, `4XX`, 또는 `5XX` 응답을 수신하고 반환한 모든 REST API 요청이 포함됩니다. 여기에는 데이터 변환 출력 및 클라우드 데이터 수집 동기화가 포함됩니다. 소프트웨어 개발 키트 트래픽 및 사용자 업데이트 단계는 이 대시보드에 포함되지 않습니다.

대시보드에 표시되는 데이터는 최근 트래픽을 표시하는 데 최대 잠시 지연될 수 있습니다. 사용량이 많은 시간대에는 대시보드를 분당 최대 4회까지 새로고침할 수 있습니다. 대시보드를 다시 새로고침하기 전에 몇 분 정도 기다려야 할 수도 있습니다.
