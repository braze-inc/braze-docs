# 메시징 진단 대시보드

> **메시징 진단** 대시보드는 메시지 전송 결과에 대한 높은 수준의 분석을 제공하여 트렌드를 파악하고 메시징 설정의 잠재적인 문제를 진단할 수 있도록 합니다. 이 대시보드는 캠페인 또는 캔버스에서 메시지가 예상대로 전송되지 않은 이유를 이해하는 데 도움이 될 수 있습니다.

{% alert important %}
**메시징 진단** 대시보드는 현재 초기 액세스 중입니다. 초기 액세스에 참여하고 싶다면 고객 성공 매니저에게 문의하세요.
{% endalert %}

## 핵심 개념

### 전송 및 전달됨

이 대시보드는 Braze가 메시지를 내부적으로 처리한 방법을 보고하며, 메시지의 최종 전달 상태는 보고하지 않는다는 점을 이해하는 것이 중요합니다.

이 대시보드에서 "전송됨"으로 표시된 메시지는 Braze가 메시지를 성공적으로 처리하고 발송했음을 의미합니다. 대부분의 채널에 대해 이는 Braze가 관련 제3자 전송 파트너에게 메시지를 전달했음을 의미합니다. 그러나 이는 사용자의 기기로의 최종 전달을 보장하지 않습니다. 

Braze가 메시지를 "전송"할 때 최종 전달은 외부 서비스에 따라 달라질 수 있습니다. 각 채널에 대한 다음 예를 고려하세요.

| Channel | 최종 전달의 예 |
| --- | --- |
| 콘텐츠 카드 | 카드가 전송되었으며 보기 가능성이 있습니다. |
| Email | Braze는 메시지를 이메일 서비스 공급자(ESP)에게 전달합니다. 그 후 ESP는 최종 전달에 대한 책임이 있습니다. 예를 들어, 해당 ESP는 이메일 주소가 유효하지 않거나 받은 편지함이 가득 차 있는 경우 "반송"을 보고할 수 있습니다. |
| 인앱 메시지 | 메시지가 사용자에게 표시되었습니다. |
| LINE | 메시지가 성공적으로 전송 파트너에게 전달되었습니다. |
| 푸시 | Braze는 메시지를 적절한 푸시 알림 서비스(예: iOS의 Apple Push Notification 서비스 또는 Android의 Firebase Cloud Messaging)로 전달합니다. 해당 서비스는 알림을 기기로 최종 전달하는 책임이 있습니다. |
| SMS/MMS/RCS | Braze는 메시지를 SMS 게이트웨이(예: Twilio)로 전달합니다. 해당 게이트웨이는 모바일 통신사에 최종 전달하는 책임이 있습니다. |
| Webhooks | 웹훅 요청이 성공적으로 이루어졌으며, `2xx` 응답을 반환했습니다. |
| WhatsApp | 메시지가 성공적으로 전송 파트너에게 전달되었습니다. |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" } 

### 데이터 신선도

이 대시보드의 데이터 업데이트 빈도는 시스템 로드에 따라 변동할 수 있습니다. 업데이트 빈도는 보장되지 않지만, 대부분의 경우 한 시간 이내일 가능성이 높습니다.

## 대시보드 구성

**분석** > **대시보드 빌더**로 이동하여 Braze에서 생성한 대시보드 목록에서 **메시징 진단**을 선택하여 진단 대시보드에 접근할 수 있습니다.

대시보드를 실행하고 데이터를 보려면:

1. 대시보드 보고서의 소스로 **캠페인** 또는 **캔버스** 중 하나를 선택하세요. 
2. 하나 이상의 캠페인 또는 캔버스를 선택하세요.
3. 선택한 필터의 데이터를 로드하려면 **대시보드 실행**을 선택하세요.

## 데이터 해석

{% alert note %}
대시보드는 최대 최근 7일의 데이터만 표시합니다.

### 요약 타일

페이지 상단에는 선택한 기간에 대한 주요 요약 타일이 있습니다.

- **총 중단 수:** 중단된 메시지의 총 수입니다. 여기에는 Canvas에 들어가지 않았거나 단계 실패를 경험하거나 종료 이벤트를 수행하는 동안 종료 기준을 충족하여 Canvas에서 종료된 Canvas 청중 구성원이 포함됩니다.
- **메시지 전송 수:** Braze가 성공적으로 처리하고 전송한 메시지의 총 수입니다. 
  - **이메일, SMS/MMS/RCS, WhatsApp, LINE 및 푸시:** 메시지가 성공적으로 전송 파트너에게 전달되었습니다.  
  - **웹훅:** 웹훅 요청이 성공적으로 이루어졌으며, `2xx` 응답이 반환되었습니다.  
  - **Content Cards:** 카드가 전송되었으며 보기 가능성이 있습니다.    
  - **In-app messages:** 메시지가 사용자에게 표시되었습니다.

### 시간에 따른 메시지 결과

이 시계열 차트는 메시지가 중단되거나 사용자가 Canvas에서 제거된 다양한 이유를 일별로 분류하여 보여줍니다. 이 차트는 전송 수를 표시하지 않습니다.  

{% alert note %}
차트를 정리하기 위해 선택한 시간 범위 내에서 발생하지 않은 중단 또는 제거 이유는 차트에 표시되지 않습니다.
{% endalert %}

### 메시지 결과 분류

이 차트는 선택한 시간 범위 내의 모든 메시지 결과의 분류를 보여줍니다. 전체 그림을 제공합니다:
- 모든 결과에 대한 전송의 총 수 비율입니다.  
- 각 중단 및 제거 이유의 비율 분류입니다. 이것은 메시지가 전송되지 않는 가장 일반적인 이유를 신속하게 식별하는 데 도움이 됩니다.

다음 정의는 대시보드에 표시된 중단 결과를 설명합니다. 결과는 조사 중인 항목을 쉽게 찾을 수 있도록 카테고리별로 그룹화됩니다.

#### 비율 제한 및 타이밍

| 중단 결과 | 설명 |
| ---- | ---- |
| Frequency capped | 사용자가 작업 공간의 [최대 게재빈도 설정]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/#about-frequency-capping) 규칙에 따라 허용된 최대 메시지 수를 이미 수신했으므로 전송이 취소되었습니다. |
| 방해금지 시간 중단 | 방해금지 시간이 캠페인 또는 캔버스 단계에 대해 활성화되었으며 대체 메시지가 **대체 메시지**로 설정되었습니다. 사용자가 방해금지 시간 동안 캠페인을 트리거하거나 캔버스 메시지 단계에 들어갔으므로 메시지가 중단되었습니다. 그러나 이는 사용자를 캔버스에서 종료하지 않습니다. |
| 72시간 이상 비율 제한됨 | 메시지가 [전달 속도 비율 제한]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/#delivery-speed-rate-limiting)으로 인해 72시간 이상 제한되어 전송이 중단되었습니다. |

#### 사용자 자격 및 프로필

| 중단 결과 | 설명 |
| ---- | ---- |
| 중복 사용자 식별자 | 일치하는 식별자(예: 외부 ID, 이메일 주소, 전화번호)를 가진 여러 사용자가 이 메시지를 수신할 자격이 있었습니다. 동일한 사용자에게 중복 전송을 방지하기 위해 이 메시지가 중단되었습니다. |
| 사용자가 메시지 단계에 대한 사전 검사를 통과하지 못했습니다 | 이 사전 검사는 전달 유효성 검사 전에 실행됩니다. 이 경우 사용자가 이 메시지 단계에 대한 기본 사전 검사를 통과하지 못했습니다(사용자를 찾을 수 없거나 메시지 단계의 채널에 자격이 없음). **Note:** 다중 채널 메시지 단계의 경우 사용자를 찾을 수 없음을 의미합니다; 채널 자격은 단일 채널 메시지 단계에 대해서만 여기에서 확인됩니다. |
| 사용자가 트리거된 메시지에 대한 사전 검사를 통과하지 못했습니다. | 트리거된 메시지의 경우, Braze는 메시지를 보내기 위해 이 트리거에서 생성하기 전에 오디언스 적격성, 재적격성 및 채널 적격성에 대한 기본 사전 검사를 수행합니다. |
| 사용자가 더 이상 적격하지 않음 | 사용자는 처음에 대상 오디언스에 있었지만, Braze가 메시지를 보내거나 사용자를 캔버스에 입력하기 전에 더 이상 오디언스 기준에 맞지 않았습니다. 사용자가 처음으로 오디언스 기준을 충족한 시점과 오디언스에서 제외된 시점 사이의 시간은 다음과 같은 지연으로 인해 발생할 수 있습니다: {::nomarkdown}<ul><li>지능형 타이밍</li><li>방해금지 시간</li><li>현지 시간</li><li>전달 속도 제한(캔버스 진입에는 적용되지 않음)</li><li>메시징 파이프라인 지연</li></ul>{:/} |
| 단계에 대한 사용자 적격성 없음 | 사용자는 메시지 단계에 대한 [전달 유효성 검사]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/message_step/#delivery-validations)를 충족하지 못했거나 [억제 목록]({{site.baseurl}}/user_guide/engagement_tools/segments/suppression_lists)의 일부였기 때문에 캔버스를 종료했습니다. |
| 사용자가 재적격하지 않음 | 사용자는 메시지를 수신하거나 캔버스에 들어갈 수 있는 적격성이 있었지만, 재적격성 또는 재입력 설정으로 인해 전송이 취소되었습니다. 사용자가 이미 캠페인을 수신했거나 너무 최근에 캔버스에 들어갔거나, 동일한 캠페인에 대한 다른 전송이 이미 진행 중이거나, 재적격성 또는 재입력이 꺼져 있는 경우 이러한 일이 발생할 수 있습니다. |
| 사용자 프로필을 찾을 수 없음 | 사용자는 존재하지 않거나 Braze에서 더 이상 존재하지 않습니다. 일부 일반적인 경우는 다음과 같습니다: {::nomarkdown}<ul><li> 사용자는 API 메시징을 사용하여 타겟팅되었지만 Braze에서 존재하지 않았습니다. </li><li>사용자는 메시지가 전송되기 전에 또는 캔버스 단계가 실행되기 전에 삭제되었습니다. </li><li>사용자는 메시지가 전송되기 전에 다른 프로필과 병합되었습니다.</li></ul>{:/} |

#### 채널 및 전달

## Frequently asked questions

### "사전 검사" 실패는 무엇을 의미합니까?

"사전 검사"는 파이프라인 단계의 시작 부분에서 실행되는 고속 번들 검증 검사를 의미합니다(예: 메시지가 트리거되거나 캔버스 메시지 단계가 전송되는 경우). 최대 속도를 위해 설계된 조기 종료로 생각하세요. 사용자의 프로필의 모든 세부 사항을 검증하는 것과 같은 많은 개별 리소스 집약적인 검사를 실행하는 대신, Braze는 여러 기본 검증을 하나의 "첫 번째 통과"로 묶습니다.

사용자가 이 단일 번들 검사를 통과하지 못하면 즉시 제외됩니다. 이 번들 접근 방식은 Braze가 대량의 메시지를 고속으로 처리할 수 있게 하며, 각 메시지의 처리 지연을 줄여 캠페인과 캔버스의 더 빠르고 안정적인 성능에 기여할 수 있습니다.

### 왜 _총 중단_과 _메시지 전송_의 합이 예상 청중 규모보다 낮습니까?

이는 여러 가지 이유로 발생할 수 있습니다:

- **청중 기준:** 예상보다 적은 사용자가 청중 기준을 충족했을 수 있습니다(예: 캠페인이나 캔버스가 시작될 때 세그먼트에 없거나 필요한 속성이 없었습니다).
- **처리 진행 중:** 메시지가 여전히 활성 처리 중일 수 있습니다. 사용자가 여전히 캔버스의 이전 단계에 있을 수 있으며 메시지 단계에 도달하지 않았습니다.
- **데이터 신선도:** 대시보드 데이터는 약 15분마다 업데이트되지만 이는 보장이 아닙니다. 이 캠페인이나 캔버스의 최신 데이터가 아직 대시보드에 도달하지 않았을 수 있습니다.
- **엣지 케이스:** 현재 이 대시보드에 포착되지 않은 엣지 케이스를 겪고 있을 가능성이 있습니다. 이 경우라고 의심되면 [Braze Support]({{site.baseurl}}/user_guide/administrative/access_braze/support)에 문의하십시오.

### _Total Aborts_와 _Message Sends_의 합이 캠페인과 캔버스의 오디언스보다 큰 이유는 무엇입니까?

다음과 같은 이유로 발생할 수 있습니다:

- **다채널 메시지:** 캠페인 또는 캔버스 단계가 여러 채널(SMS 및 이메일 등)에서 전송되도록 구성되었습니다. 단일 사용자는 하나의 채널(예: 이메일)에서 "전송" 결과를 받고 다른 채널(예: "사용자가 SMS/MMS/RCS에 대해 활성화되지 않음")에서 "중단" 결과를 받을 수 있습니다. 이 경우, 해당 사용자는 차트에서 두 번 계산됩니다: 한 번은 "전송"으로, 한 번은 "중단"으로.
  - **예시:** 100명의 사용자에게 푸시 캠페인을 보내고 iOS와 Android 모두를 타겟팅합니다. 사용자가 iOS 기기만 있는 경우, iOS 푸시("전송")를 받지만 Android 푸시에 대해 중단을 트리거합니다("사용자가 Android 푸시, 앱 또는 기기에 대해 활성화되지 않음").
- **다중 메시지 단계(캔버스 전용):** 귀하의 캔버스는 주어진 경로에 여러 메시지 단계를 가질 수 있습니다. 이 대시보드는 모든 결과를 집계하므로, 단일 사용자가 선택한 시간 범위 내에서 여러 메시지 단계를 통과하면 여러 번 계산될 수 있습니다.
- **테스트 메시지:** 테스트 전송(대시보드에 계산됨)은 총 수치를 오디언스 크기보다 높게 만듭니다.
