### 익명 사용자 데이터는 어떻게 처리하나요?

{% apitags %}
사용자
{% endapitags %}

처음에 소프트웨어 개발 키트를 통해 사용자 프로필이 인식되면 Braze가 설정한 고유 사용자 식별자( `braze_id`)가 연결된 익명 사용자 프로필을 생성합니다.

익명 사용자를 더욱 추적하려면 익명 사용자에게 식별자를 태그할 수 있는 [사용자 별칭을]({{site.baseurl}}/user_guide/data_and_analytics/user_data_collection/user_profile_lifecycle#user-aliases) 구현할 수 있습니다. 그런 다음 이러한 사용자는 사용자 별칭 지정을 사용하여 내보내거나 API에서 참조할 수 있습니다.

별칭이 있는 익명 사용자 프로필이 나중에 `external_id` 으로 인식되면 정상적으로 식별된 사용자 프로필로 취급되지만 기존 별칭은 그대로 유지되며 해당 별칭으로 계속 참조할 수 있습니다.

식별된 사용자와 병합하려는 사용자 별칭 지정의 경우 유지하려는 실제 프로필과 관련된 모든 필드를 병합할 수 있습니다. [식별자 엔드포인트로 사용자 프로필 내보내기]({{site.baseurl}}/api/endpoints/export/user_data/post_users_identifier/) 기능을 사용하여 별칭 프로필에서 데이터를 삭제하기 전에 해당 데이터를 내보내야 합니다. 그런 다음 [사용자 추적 엔드포인트를]({{site.baseurl}}/api/endpoints/user_data/post_user_track/) 사용하여 이러한 이벤트를 보관한 프로필에 게시할 수 있습니다. 이렇게 하면 이전에 한 프로필에 기록되었지만 다른 프로필에는 기록되지 않았던 속성 등 유지하려는 모든 데이터를 보존할 수 있습니다.

Braze에서 신규 및 기존 사용자 데이터를 수집하는 다양한 방법에 대한 자세한 내용은 [데이터 수집 모범 사례를]({{site.baseurl}}/user_guide/data/user_data_collection/best_practices/) 참조하세요.

### 이미 수집하고 식별한 사용자를 Braze 외부에서 가져오기 하려면 어떻게 해야 하나요?

{% apitags %}
사용자
{% endapitags %}

이전에 식별된 사용자를 가져오기 위해 CSV를 Braze에 업로드하거나 API를 통해 데이터를 전송할 수 있습니다.

#### CSV

**오디언스** > **사용자 가져오기에서** CSV 파일을 통해 고객 프로필을 업로드하고 업데이트할 수 있습니다. 고객 데이터를 가져올 때 각 고객의 고유 식별자( `external_id`)를 지정해야 합니다.

CSV 가져오기를 시작하기 전에 엔지니어링 팀으로부터 Braze에서 사용자를 식별하는 방법을 이해하는 것이 중요합니다. 일반적으로 내부에서 사용하는 데이터베이스 ID입니다. 이는 모바일과 웹에서 Braze 소프트웨어 개발 키트에 의해 사용자를 식별하는 방식과 일치해야 각 고객이 모든 기기에 걸쳐 Braze 내에서 단일 사용자 프로필을 갖게 됩니다. Braze [고객 프로필 수명주기에]({{site.baseurl}}/user_guide/data/user_data_collection/user_profile_lifecycle/) 대해 자세히 알아보세요.

사용자 가져오기에 `external_id` 을 입력하면 Braze는 기존 사용자를 동일한 `external_id` 으로 업데이트하거나, 사용자를 찾을 수 없는 경우 `external_id` 을 설정하여 새로 식별된 사용자를 생성합니다.

자세한 내용을 확인하고 CSV 가져오기 템플릿을 다운로드하려면 [사용자 가져오기를]({{site.baseurl}}/user_guide/data_and_analytics/user_data_collection/user_import/#csv) 참조하세요.

#### API

API를 통해 사용자를 업로드하려면 [사용자 추적 엔드포인트를]({{site.baseurl}}/api/endpoints/user_data/post_user_track/) 사용하여 사용자를 Braze로 가져올 수 있습니다.

사용자가 이미 Braze에 존재하는지 확실하지 않은 경우 [식별자별 사용자 프로필 내보내기 엔드포인트를]({{site.baseurl}}/api/endpoints/export/user_data/post_users_identifier/) 구현하여 확인할 수 있습니다. 사용자가 이미 Braze에 존재하는 것으로 식별되는 경우, `/users/track` 엔드포인트를 사용하여 추가하려는 새 데이터를 이미 존재하는 사용자 프로필에 게시할 수 있습니다.

{% alert note %}
`/users/track` 엔드포인트를 사용할 때는 다음과 같은 뉘앙스를 염두에 두세요:

- 이 엔드포인트를 통해 별칭 전용 사용자를 만들 때는 `_update_existing_only` 플래그를 명시적으로 false로 설정해야 합니다.
- 이 엔드포인트로 구독 상태를 업데이트하면 외부 ID로 지정된 사용자(예: User1)가 업데이트되고 해당 사용자(User1)와 동일한 이메일을 가진 모든 사용자의 구독 상태가 업데이트됩니다.
{% endalert %}

### 푸시 구독 상태의 차이점은 무엇인가요?

{% apitags %}
사용자
{% endapitags %}

푸시 구독 상태 옵션에는 가입함, 옵트인, 탈퇴함의 세 가지가 있습니다.

기본적으로 사용자가 푸시를 통해 메시지를 받으려면 푸시 가입 상태가 가입 또는 옵트인 상태여야 하며 푸시 인에이블먼트 상태여야 합니다. 메시지 작성 시 필요한 경우 이 설정을 재정의할 수 있습니다.

|옵트인 상태|설명|
|---|---|
|가입한 가입자| Braze에서 고객 프로필이 생성될 때의 기본값 푸시 구독 상태입니다. |
|옵트인| 사용자가 푸시 알림 수신을 명시적으로 선호한다는 의사를 표시한 경우. 사용자가 OS 수준의 푸시 안내를 수락하면 Braze는 사용자의 옵트인 상태를 `Opted-In` 로 자동 이동합니다.<br><br>Android 12 이하의 사용자에게는 적용되지 않습니다.|
|탈퇴됨| 사용자가 애플리케이션 또는 브랜드가 제공하는 기타 방법을 통해 푸시 수신을 명시적으로 탈퇴한 경우. 기본값으로 Braze 푸시 캠페인은 `Subscribed` 또는 `Opted-in` 인 사용자만 푸시 타겟팅합니다.|

### 중복된 사용자를 식별한 경우에는 어떻게 하나요?

{% apitags %}
사용자
{% endapitags %}

중복 사용자를 식별한 경우 해당 고객 프로필을 정리해야 합니다. 다음 단계를 통해 그렇게 할 수 있습니다:

1. `/users/export/ids` 엔드포인트를 사용하여 고객 프로필을 내보내세요.
2. 정확한 고객 프로필 식별자(궁극적으로는 팀에서 정확한 정보를 결정해야 함) 또는 둘 중 하나를 선택합니다:
    - `/user/track` 엔드포인트를 사용하여 유지하려는 실제 프로필과 관련된 모든 필드를 병합합니다.
    - 사용자/삭제 엔드포인트를 사용하여 데이터를 병합하지 않고 중복되고 사용되지 않는 프로필을 삭제하세요. 고객 프로필을 삭제한 후에는 **해당 정보를 검색할 수 있는 방법이 없습니다**.

{% alert important %}
먼저 올바른 `external_id` 및 해당 커스텀 속성 및 이벤트가 포함된 새 사용자 프로필을 가져오는 것이 좋습니다. 고객 프로필을 삭제한 후에는 다시 검색할 수 없으므로 삭제는 마지막 단계로 진행해야 합니다.
{% endalert %}

몇 가지 추가 참고 사항

- 중복된 고객 프로필의 모든 참여 데이터(예: 캠페인 또는 받은 캔버스)는 손실됩니다. 과거 고객 참여 컨텍스트를 유지하는 유일한 방법은 커스텀 속성으로 추가하는 것입니다(예: 수신된 모든 캠페인 또는 캔버스의 배열 커스텀 속성).
- 고객 프로필을 마이그레이션할 때 중복된 사용자 프로필 중 어떤 프로필을 유지할지 결정하는 것도 팀의 몫입니다. Braze는 삭제할 프로필 목록을 결정하거나 제공할 수 없습니다.  
- 궁극적으로 팀에서 사용자 경험을 통해 가입 프로세스를 평가하고 사용자가 식별될 때만 `changeUser()` 메서드를 호출하는지 확인하는 것이 중요합니다.

{% endapi %}

### CSV를 통해 사용자 그룹을 가져오기 할 때 세그먼트를 만들려면 어떻게 해야 하나요?

{% apitags %}
세그먼트
{% endapitags %}

CSV 파일을 가져오려면 사용자 섹션 아래의 **사용자 가져오기** 페이지로 이동합니다. **최근** 가져오기 표에는 가장 최근에 가져온 파일 20개, 파일 이름, 파일 내 줄 수, 성공적으로 가져온 줄 수, 각 파일의 총 줄 수, 각 가져오기 상태가 나열됩니다.

**CSV 가져오기** 패널에는 가져오기 방법과 가져오기를 시작하는 버튼이 있습니다. **CSV 파일 선택을** 클릭하고 관심 있는 파일을 선택합니다. 그런 다음 사용자 **가져오기 시작을** 클릭하기 전에 "이 CSV의 사용자에 대해 수행할 작업"에서 이 목록으로 수행할 작업을 Braze에 알려주는 옵션이 있습니다.

**이 CSV에서 사용자 가져오기를** 선택하고 **이 특정 사용자 배치를 그룹으로 리타겟팅할 수 있도록 설정한** 다음 **이 CSV에서 가져온 사용자로부터 세그먼트 자동 생성을** 선택합니다. **가져오기 시작을** 클릭하면 Braze가 파일을 업로드하고 열 헤더와 각 열의 데이터 유형을 확인한 후 세그먼트를 생성합니다.

CSV 템플릿을 다운로드하려면 [사용자 가져오기를]({{site.baseurl}}/user_guide/data_and_analytics/user_data_collection/user_import/#csv) 참조하세요.

### 세그먼트를 만들 때 어떤 유형의 필터를 사용할 수 있나요?

{% apitags %}
세그먼트
{% endapitags %}

Braze 소프트웨어 개발 키트에는 특정 기능과 속성을 기반으로 사용자를 세분화하고 타겟팅할 수 있는 강력한 필터가 포함되어 있습니다. [세분화 필터]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters) 용어집을 사용하여 필터 카테고리(커스텀 데이터, 사용자 활동, 리타겟팅, 마케팅 활동, 사용자 속성, 설치 기여도, 소셜 활동, 테스트, 기타)별로 필터를 검색하거나 필터의 범위를 좁힐 수 있습니다.

{% endapi %}

### 가장 최근 위치를 기준으로 사용자를 세분화하여 위치 기반 캠페인 및 전략에 사용할 수 있도록 위치 타겟팅을 설정하려면 어떻게 해야 하나요?

{% apitags %}
세그먼트
{% endapitags %}

참여 아래의 **세그먼트** 페이지로 이동하여 현재 모든 사용자 세그먼트를 확인합니다. 이 페이지에서 새 세그먼트를 만들고 이름을 지정할 수 있습니다. 시작하려면 **세그먼트 만들기를** 클릭하고 세그먼트에 이름을 지정합니다.

세그먼트를 생성한 후에는 `Most Recent Location` 필터를 추가하여 사용자가 마지막으로 앱을 사용한 위치를 기준으로 사용자를 타겟팅합니다. 표준 원형 영역에서 사용자를 강조 표시하거나 커스텀 다각형 영역을 만들 수 있습니다.

- 원형 영역의 경우 원점을 이동하고 세그먼트의 위치 반경을 조정할 수 있습니다.
- 다각형 영역의 경우 세그먼트에 포함할 영역을 보다 구체적으로 지정할 수 있습니다.

{% alert tip %}
Braze 파트너의 도움을 받아 위치 타겟팅을 활용하고 싶으신가요? 사용 가능한 Braze [상황별 위치 파트너를]({{site.baseurl}}/partners/message_personalization/) 확인하세요.
{% endalert %}

### 지난 365일 동안의 커스텀 이벤트 및 구매 행동을 기반으로 정확한 사용자 목록을 타겟팅하려면 어떻게 해야 하나요?

{% apitags %}
세그먼트
{% endapitags %}

[세그먼트 확장을]({{site.baseurl}}/user_guide/engagement_tools/segments/segment_extension/) 사용할 수 있습니다! 세그먼트 확장을 사용하면 일반 세그먼트를 사용할 때보다 더 정확한 사용자 목록을 타겟팅할 수 있습니다.

워크스페이스당 최대 10개의 세그먼트 확장을 만들 수 있습니다. 이러한 확장 목록이 생성된 후에는 세그먼트에 필터로 포함하거나 제외할 수 있습니다. 세그먼트 확장을 만들 때 24시간에 한 번씩 목록을 다시 생성하도록 지정할 수도 있습니다.

1. 참여 아래에서 **세그먼트를** 확장하고 **세그먼트 확장을** 클릭합니다.
2. 세그먼트 확장 테이블에서 **\+ 새 확장 프로그램 만들기를** 클릭합니다.
3. 필터링하려는 사용자 유형을 설명하여 세그먼트 확장에 이름을 지정합니다. 이렇게 하면 세그먼트에 필터로 적용할 때 이 확장을 쉽고 정확하게 검색할 수 있습니다.
4. 타겟팅할 구매 또는 커스텀 이벤트 기준 중에서 선택합니다.
5. 사용자 목록에서 타겟팅할 구매한 아이템이나 특정 커스텀 이벤트를 선택합니다. 
6. 사용자가 이벤트를 완료해야 하는 횟수(이상, 미만 또는 같음)와 최대 365일까지 되돌아볼 일수를 선택합니다.

타겟팅 정확도를 높이려면 **속성 필터 추가를** 선택하고 구매 또는 커스텀 이벤트의 특정 속성을 기준으로 세그먼트를 세분화할 수 있습니다. Braze는 문자열, 숫자, 부울, 시간 객체를 기반으로 이벤트 속성정보 세분화를 지원합니다.

[중첩된 이벤트 속성정보를]({{site.baseurl}}/user_guide/data/custom_data/custom_events/nested_objects/) 기반으로 한 세그먼트 세분화도 지원합니다.

세그먼트 확장은 이벤트 속성정보의 장기 저장에 의존하며 30일 커스텀 이벤트 속성정보 저장 제한이 없습니다. 즉, 지난 1년 동안 추적된 이벤트 속성정보를 다시 볼 수 있으며, 확장 프로그램이 먼저 설정될 때까지 추적을 기다리지 않습니다.

{% alert note %}
세그먼트 확장 내에서 이벤트 속성정보를 사용해도 데이터 포인트 사용량에는 영향을 미치지 않습니다.
{% endalert %}

#### 세그먼트 확장을 최신 상태로 유지하기

{% apitags %}
세그먼트
{% endapitags %}

이 확장을 특정 시점의 단일 스냅샷 담당자로 지정할지, 아니면 매일 재생성할지 여부를 지정할 수 있습니다. 확장 프로그램은 항상 최초 저장 후 처리가 시작됩니다. 연장을 매일 재생성하려면 **매일 연장 재생성을** 선택하면 회사 시간대의 매일 자정 무렵에 재생성 처리가 시작됩니다.

완료했으면 **저장을** 클릭합니다. 확장 프로그램 처리가 시작됩니다. 확장 프로그램을 생성하는 데 걸리는 시간은 사용자 수, 캡처하는 커스텀 이벤트 또는 구매 이벤트의 수, 기록에서 되돌아보는 일수에 따라 달라집니다.

마지막으로, 확장을 만든 후에는 세그먼트를 만들거나 캠페인 또는 캔버스의 오디언스를 정의할 때 필터로 사용할 수 있습니다. 먼저 **사용자 속성** 섹션의 필터 목록에서 `Braze Segment Extension` 을 선택합니다. Braze 세그먼트 확장 필터 목록에서 이 세그먼트에 포함하거나 제외할 확장자를 선택합니다. 확장자 기준을 보려면 **확장자 세부정보 보기를** 클릭합니다. 이제 평소처럼 세그먼트 생성을 계속 진행하면 됩니다.

### 현지 시간대 전달은 무엇을 제공하나요?

### 현지 시간대 캠페인은 어떻게 예약하나요?

{% apitags %}
캠페인
{% endapitags %}

캠페인을 예약할 때 지정된 시간에 캠페인을 보내도록 선택한 다음 **현지 시간대의 사용자에게 캠페인 보내기를** 선택해야 합니다.

Braze는 모든 현지 시간대 캠페인을 24시간 전에 예약할 것을 적극 권장합니다. 이러한 캠페인은 하루 종일 보내야 하므로 24시간 전에 미리 예약하면 전체 메시지 세그먼트에 메시지가 도달할 수 있습니다. 그러나 필요한 경우 24시간 전에 캠페인을 예약할 수 있습니다. Braze는 전송 시간을 1시간 이상 놓친 사용자에게는 메시지를 보내지 않습니다.

예를 들어 오후 1시에 현지 시간대 캠페인을 오후 3시로 예약하면 현지 시간이 오후 3~4시인 모든 사용자에게는 캠페인이 즉시 전송되지만 현지 시간이 오후 5시인 사용자에게는 전송되지 않습니다. 또한 캠페인에 대해 선택한 전송 시간은 회사 시간대에 아직 발생하지 않은 시간이어야 합니다.

24시간 전에 예약된 현지 시간대 캠페인을 편집해도 메시징 캠페인의 일정은 변경되지 않습니다. 현지 시간대 캠페인을 편집하여 이후 시간(예: 오후 6시 대신 오후 7시)에 전송하기로 결정한 경우, 원래 전송 시간을 선택할 때 타겟팅된 세그먼트에 속했던 사용자는 여전히 원래 시간(오후 6시)에 메시지를 받게 됩니다. 현지 시간대를 편집하여 더 이른 시간(예: 오후 5시 대신 오후 4시)에 전송하는 경우에도 캠페인은 원래 시간(오후 5시)에 모든 세그먼트 멤버에게 전송됩니다.

{% alert note %}
캔버스 단계의 경우, 사용자는 현지 시간대 전달을 위해 24시간 동안 해당 단계에 있지 않아도 다음 단계를 받을 수 있습니다.
{% endalert %}

사용자가 캠페인에 다시 참여할 수 있도록 허용한 경우 원래 시간(오후 5시)에 다시 캠페인을 받게 됩니다. 그러나 이후 발생하는 모든 캠페인의 경우 업데이트된 시간에만 메시징이 전송됩니다.

{% endapi %}
{% api %}

### 현지 시간대 캠페인 변경은 언제 적용되나요?

{% apitags %}
캠페인
{% endapitags %}

현지 시간대 캠페인의 타겟팅 세그먼트에는 시간 기반 필터의 경우 전체 세그먼트에 전달을 보장하기 위해 최소 48시간의 기간이 포함되어야 합니다. 예를 들어 다음 필터를 사용하여 둘째 날 사용자를 타겟팅하는 세그먼트를 생각해 보세요:

- 처음 사용한 지 1일 이상 지난 앱
- 앱을 처음 사용한지 2일 미만입니다.

현지 시간대 전달은 전달 시간과 사용자의 현지 시간대에 따라 이 세그먼트의 사용자를 놓칠 수 있습니다. 이는 사용자가 자신의 시간대가 전달을 트리거하는 시간까지 세그먼트를 떠날 수 있기 때문입니다.

### 예약된 캠페인의 메시징 캠페인이 대기줄에 오르기 전의 '안전 지대'란 무엇인가요?

### Braze는 현지 시간대 전달을 위해 언제 사용자를 평가하나요?

{% apitags %}
캠페인
{% endapitags %}

현지 시간대 전달의 경우, Braze는 이 두 인스턴스 동안 사용자의 참가 자격을 평가합니다:

- 예정일의 사모아 시간(UTC+13) 기준
- 예정일 현지 시간 기준

사용자가 참가 자격을 얻으려면 두 가지 검사를 모두 통과해야 합니다. 예를 들어 캔버스가 2021년 8월 7일 오후 2시(현지 시간 기준)에 출시될 예정인 경우 뉴욕에 위치한 사용자를 타겟팅하려면 다음과 같은 자격 확인을 거쳐야 합니다:

- 2021년 8월 6일 오후 9시에 뉴욕
- 2021년 8월 7일 오후 2시 뉴욕

사용자는 출시 전 24시간 동안 해당 세그먼트에 있어야 합니다. 사용자가 첫 번째 확인에서 자격이 없는 경우 Braze는 두 번째 확인을 시도하지 않습니다.

Braze에서는 트리거 전에 오디언스가 평가됩니다( [속성 변경 트리거를]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/triggered_delivery/attribute_triggers#change-custom-attribute-value) 사용하지 않는 한). 이렇게 하면 트리거 동작이 평가되기 전에 사용자가 처음에 선택한 오디언스에 속하지 않은 경우 캠페인에서 이탈하게 됩니다.

### 하나의 배리언트가 있지만 여러 개의 브랜치가 있는 캔버스에 대해 오디언스와 전송 시간이 동일하면 어떻게 될까요?

### 캔버스를 중지하면 어떻게 되나요?

{% apitags %}
캔버스
{% endapitags %}

캔버스를 중지하면 다음 사항이 적용됩니다:

- 사용자는 캔버스에 들어갈 수 없습니다.
- 사용자가 흐름의 어느 위치에 있더라도 더 이상 메시징이 전송되지 않습니다.
    - **예외:** 이메일 캔버스는 즉시 중지되지 않습니다. 전송 요청이 SendGrid로 전송된 후에는 사용자에게 전달되는 것을 막을 수 있는 방법이 없습니다.

{% alert note %}
캔버스를 중지해도 단계에서 대기 중인 사용자는 종료되지 않습니다. 캔버스를 다시 활성화했는데 사용자가 여전히 대기 중인 경우 단계를 완료하고 다음 컴포넌트로 이동합니다. 그러나 사용자가 다음 구성 요소로 진행했어야 하는 시간이 지나면 대신 캔버스를 종료합니다.
{% endalert %}

### 예외 이벤트는 언제 트리거되나요?

{% apitags %}
캔버스
{% endapitags %}

예외 이벤트는 사용자가 연결된 캔버스 컴포넌트를 수신하기 위해 대기하는 동안에만 트리거됩니다. 사용자가 미리 동작을 수행하면 예외 이벤트가 트리거되지 않습니다.

특정 이벤트를 미리 수행한 사용자를 제외하려면 [필터를]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters/) 사용하세요.

### 캔버스를 편집하면 이미 캔버스에 있는 사용자에게 어떤 영향을 주나요?

{% apitags %}
캔버스
{% endapitags %}

다단계 캔버스의 일부 단계를 편집하면 이미 오디언스에 속해 있지만 해당 단계를 받지 못한 사용자에게 업데이트된 버전의 메시지가 전송됩니다. 아직 해당 단계에 대한 평가가 이루어지지 않은 경우에만 이런 일이 발생한다는 점에 유의하세요.

실행 후 편집할 수 있는 항목과 편집할 수 없는 항목에 대한 자세한 내용은 [실행 후 캔버스 변경하기를]({{site.baseurl}}/post-launch_edits/) 참조하세요.

### 캔버스에서 사용자 전환은 어떻게 추적하나요?

{% apitags %}
캔버스
{% endapitags %}

사용자는 캔버스 항목당 한 번만 변환할 수 있습니다.

전환은 해당 항목에 대해 사용자가 가장 최근에 수신한 메시징에 할당됩니다. 캔버스 시작 부분의 요약 블록은 메시지 수신 여부와 관계없이 해당 경로 내에서 사용자가 수행한 모든 전환을 반영합니다. 이후의 각 단계는 사용자가 가장 최근에 받은 단계인 동안에 발생한 전환만 표시합니다.

{% details Use cases %}

#### 사용 사례 1

10개의 푸시 알림이 있는 캔버스 경로가 있으며 전환 이벤트는 "세션 시작"("앱 열기")입니다:

- 사용자 A는 입장 후 첫 번째 메시지를 받기 전에 앱을 엽니다.
- 사용자 B는 푸시 알림을 받을 때마다 앱을 엽니다.

**결과:**
요약에는 두 개의 전환이 표시되고 개별 단계에는 첫 번째 단계에서는 1, 이후 모든 단계에서는 0의 전환이 표시됩니다.

{% alert note %}
전환 이벤트가 발생할 때 방해금지 시간이 활성화되어 있는 경우에도 동일한 규칙이 적용됩니다.
{% endalert %}

#### 사용 사례 2

방해금지 시간이 있는 원스텝 캔버스가 있습니다:

1. 사용자가 캔버스에 들어갑니다.
2. 첫 번째 단계는 지연이 없지만 방해금지 시간 내에 있으므로 메시징이 억제됩니다.
3. 사용자가 전환 이벤트를 수행합니다.

**결과:**
사용자는 전체 캔버스 배리언트에서 변환된 것으로 계산되지만 단계를 받지 않았으므로 단계는 계산되지 않습니다.

{% enddetails %}

### 순 사용자 수를 살펴볼 때 캔버스 분석과 세그먼트 중 어느 것이 더 정확할까요?

{% apitags %}

### 반복 참여 보고서를 예약하려면 어떻게 해야 하나요?

{% apitags %}
분석
{% endapitags %}

반복 참여 보고서를 예약하려면 다음과 같이 하세요:

1. 대시보드 계정에서 **데이터** 아래의 **참여 보고서로** 이동합니다.
2. **새 보고서 생성을** 클릭합니다.
3. 보고서에 컴파일하려는 [캠페인과 캔버스 메시지를]({{site.baseurl}}/user_guide/data_and_analytics/reporting/engagement_reports/#manually-select-campaigns-or-canvases) 개별적으로 또는 [태그별로]({{site.baseurl}}/user_guide/data_and_analytics/reporting/engagement_reports/#automatically-select-campaigns-or-canvases) 추가합니다.
4. 보고서에 [통계를 추가합니다]({{site.baseurl}}/user_guide/data_and_analytics/reporting/engagement_reports/#add-statistics-to-your-report).
5. 보고서의 압축 및 구분 기호를 선택합니다.
6. 이 보고서를 받아야 하는 Braze 사용자의 이메일 주소를 입력합니다.
7. 리포트에서 데이터를 실행할 [기간을]({{site.baseurl}}/user_guide/data_and_analytics/reporting/engagement_reports/#time-frame) 선택합니다.
8. 데이터 분석을 보고 싶은 [간격(일별, 주별 등)]({{site.baseurl}}/user_guide/data_and_analytics/reporting/engagement_reports/#data-display) 을 선택합니다.
9. 보고서를 [즉시]({{site.baseurl}}/user_guide/data_and_analytics/reporting/engagement_reports/#send-immediately) 또는 [나중에 지정된 시간에]({{site.baseurl}}/user_guide/data_and_analytics/reporting/engagement_reports/#send-at-designated-time) [전송하도록]({{site.baseurl}}/user_guide/data_and_analytics/reporting/engagement_reports/#send-immediately) 예약하세요.
10. 리포트가 도착하면 이메일에서 리포트를 실행하고 열어보세요!

### 참여 보고서와 보고서 빌더의 차이점은 무엇인가요?

{% apitags %}
분석
{% endapitags %}

참여 보고서는 트리거된 이메일을 통해 캠페인 및 캔버스의 특정 메시지에 대한 참여 통계의 CSV를 제공합니다. 특정 데이터는 캠페인 또는 캔버스 수준에서 집계되는 것과 개별 배리언트 또는 단계 수준에서 집계되는 것이 있습니다. 리포트는 대시보드에 저장되지 않으며, 리포트를 다시 실행하면 통계가 업데이트될 수 있습니다.

보고서 빌더를 사용하면 여러 캠페인 또는 캔버스의 결과를 단일 보기에서 비교할 수 있으므로 어떤 참여 전략이 주요 측정기준에 가장 큰 영향을 미쳤는지 쉽게 확인할 수 있습니다. 캠페인과 캔버스 모두 데이터를 내보내고 보고서를 저장하여 나중에 볼 수 있습니다.

Braze에서 보고서 및 분석을 사용하는 방법에 대한 자세한 내용은 [보고서 개요를]({{site.baseurl}}/user_guide/analytics/reporting/reports_overview/) 참조하세요.
