# 워크스페이스 만들기 및 관리하기

> 이 도움말에서는 워크스페이스를 만들고, 설정하고, 관리하는 방법에 대해 설명합니다. 

## 워크스페이스란 무엇인가요?

Braze에서 하는 모든 작업은 작업 공간 내에서 이루어집니다. 워크스페이스는 관련 모바일 앱 또는 웹사이트의 참여를 추적하고 관리할 수 있는 공유 환경입니다. 작업 공간은 동일하거나 매우 유사한 앱(예: 모바일 앱의 Android 및 iOS 버전)을 함께 그룹화합니다. 

## 작업 공간 만들기

### 1단계: 계획 세우기

시작하기 전에 팀 및 Braze 온보딩 매니저와 협력하여 사용 사례에 가장 적합한 워크스페이스 구성을 결정하세요. Braze에서 업무공간을 계획하는 방법에 대해 자세히 알아보려면 [시작하기를 참조하세요: 워크스페이스]({{site.baseurl}}/user_guide/getting_started/workspaces/) 가이드.

### 2단계: 작업 공간 추가

전역 헤더의 워크스페이스 드롭다운에서 새 워크스페이스를 만들거나 기존 워크스페이스 간에 전환할 수 있습니다.

1. 작업 공간 드롭다운을 선택한 다음 <i class="fa-solid fa-square-plus" style="color: #0b8294;"></i> **작업 공간 만들기를** 선택합니다.

'워크스페이스 만들기' 버튼이 있는 워크스페이스 드롭다운.]({% image_buster /assets/img/workspaces/workspace_create.png %}){: style="max-width:60%;"}

{:start="2"}
2\. 작업 공간에 이름을 지정하세요.

{% alert tip %}
회사 내 다른 사람들이 내 워크스페이스를 쉽게 찾을 수 있도록 이름 지정 규칙을 채택하는 것이 좋습니다. 예를 들어 "미국 도착 - 생산" 및 "미국 도착 - 스테이징".
{% endalert %}

{:start="3"}
3\. **만들기를** 선택합니다. Braze가 작업 공간을 만드는 데 몇 초 정도 걸릴 수 있습니다.

"미국 항해 시 - 스테이징"이라는 이름의 "워크스페이스 만들기" 모달이 표시됩니다.]({% image_buster /assets/img/workspaces/workspace_name.png %}){: style="max-width:60%" }

앱 인스턴스 추가를 시작할 수 있는 **앱 설정** 페이지로 이동합니다. 이 페이지는 **설정** > **앱 설정에서** 언제든지 액세스할 수 있습니다.

\!['앱 설정' 페이지 - 앱을 추가할 수 있는 버튼이 있는 스테이징 작업 공간.]({% image_buster /assets/img/workspaces/workspace_empty_state.png %})

### 3단계: 앱 인스턴스 추가하기

워크스페이스 내에서 수집되는 다양한 사이트와 앱을 "앱 인스턴스"라고 합니다.

1. **앱 설정** 페이지에서 **\+ 앱 추가를** 선택합니다.
2. 앱 인스턴스의 이름을 지정하고 이 앱 인스턴스가 어떤 플랫폼에 있는지 선택하세요. 여러 플랫폼을 선택하면 각 플랫폼에 대해 하나의 앱 인스턴스가 생성됩니다.

앱 세부 정보를 선택할 수 있는 옵션이 있는 "미국 여행 시 새 앱 추가 - 스테이징" 모달을 클릭합니다.]({% image_buster /assets/img/workspaces/workspace_add_app.png %}){: style="max-width:60%" }

{:start="3"}
3\. **앱 추가를** 선택하여 확인합니다.

#### 앱 API 키

앱 인스턴스를 추가하면 해당 API 키에 액세스할 수 있습니다. API 키는 앱 인스턴스와 Braze API 간에 요청을 할 때 사용됩니다. API 키는 앱 또는 웹사이트와 Braze SDK를 통합하는 데에도 중요합니다.

API 키 및 SDK 엔드포인트 필드가 있는 Upon Voyage iOS 앱의 설정 페이지.]({% image_buster /assets/img/workspaces/app_api_key.png %})

{% alert note %}
각 플랫폼에서 앱의 각 버전에 대해 별도의 앱 인스턴스를 생성해야 합니다. 예를 들어, iOS와 Android 모두에 무료 및 프로 버전의 앱이 있는 경우 워크스페이스 내에 4개의 앱 인스턴스(무료 iOS 앱, 무료 Android 앱, 프로 iOS 앱, 프로 Android 앱)를 만들 수 있습니다. 이렇게 하면 각 앱 인스턴스에 대해 하나씩 4개의 API 키를 사용할 수 있습니다.
{% endalert %}

#### 라이브 소프트웨어 개발 키트 버전

특정 앱의 앱 설정 페이지에 표시되는 라이브 소프트웨어 개발 키트 버전은 일일 총 세션의 5% 이상이고 지난 하루 동안 500회 이상의 세션이 있는 가장 높은 앱 버전입니다.

이 필드는 앱 또는 웹사이트에 Braze SDK를 통합한 후에 표시됩니다. 플랫폼에 최신 버전의 Braze 소프트웨어 개발 키트를 사용할 수 있는 경우 여기에 "최신 버전 사용 가능" 태그가 표시됩니다.

"라이브 소프트웨어 개발 키트 버전" 섹션에 필드 값이 "5.4.0"이고 새 버전을 사용할 수 있다는 아이콘이 표시됩니다.]({% image_buster /assets/img/workspaces/app_live_sdk_version.png %})

### 4단계: 필요에 따라 반복

2단계와 3단계를 반복하여 계획에 필요한 만큼의 워크스페이스를 설정하세요. 통합 및 캠페인 테스트를 위한 테스트 워크스페이스를 만드는 것이 가장 좋습니다.

{% alert tip %}
**테스트 워크스페이스 추가하기**<br>프로덕션 인스턴스에서 특정 사용자를 완전히 샌드박싱하여 앱 테스트를 수행할 수 있습니다. 새 워크스페이스를 만들고 애플리케이션을 게시할 때 Braze가 사용하는 API 키를 테스트 워크스페이스가 아닌 프로덕션 워크스페이스와 일치하도록 변경해야 합니다.
{% endalert %}

## 작업 공간 관리하기

### 즐겨찾기 추가

즐겨찾는 작업 공간을 추가하여 가장 자주 사용하는 작업 공간에 더욱 빠르게 액세스할 수 있습니다.

'즐겨찾는 작업 공간' 탭이 있는 작업 공간 드롭다운을 클릭합니다.]({% image_buster /assets/img/workspaces/workspace_favorites.png %}){: style="max-width:50%;"}

즐겨찾는 작업 공간을 추가하려면:

1. 프로필 드롭다운을 선택한 다음 **계정 관리를** 선택합니다.
2. **계정 프로필** 섹션에서 **즐겨찾는 워크스페이스** 필드를 찾습니다.
3. 목록에서 작업 공간을 선택합니다.
4. **변경 사항 저장을** 선택합니다.

즐겨찾기에 추가할 수 있는 작업 공간 수에는 제한이 없지만, 편의를 위해 이 목록을 짧게 작성하는 것이 좋습니다.

### 작업 공간 이름 바꾸기

작업 공간의 이름을 변경하려면

1. **설정** > **앱 설정으로** 이동합니다.
2. 워크스페이스 이름 위로 마우스를 가져가 <i class="image: /assets/img/braze_icons/pencil-01.svg" style="color: #0b8294;"></i> 을 선택합니다.
3. 워크스페이스에 새 이름을 지정한 다음 <i class="fa-solid fa-square-check" style="color: #0b8294;"></i> **저장을** 선택합니다.

작업 공간 이름 옆에 연필 아이콘이 나타납니다.]({% image_buster /assets/img/workspaces/workspace_rename.gif %}){: style="max-width:50%;"}

### 작업 공간 및 앱 인스턴스 삭제하기

워크스페이스 또는 앱 인스턴스를 삭제하려면 다음과 같이 하세요:

1. **설정** > **앱 설정으로** 이동합니다.
2. **작업 공간 삭제를** 선택하여 해당 작업 공간을 삭제하거나 해당 앱 인스턴스 옆의 휴지통 아이콘을 선택합니다.

현재 사용자를 타겟팅하는 데 사용 중이거나 사용자가 1,000명이 넘는 앱 인스턴스 또는 워크스페이스는 삭제할 수 없습니다. 그렇게 하려고 하면 오류 메시지가 표시됩니다. 계속 진행하여 삭제하려면 대시보드 링크와 삭제할 앱 인스턴스 또는 워크스페이스의 이름이 포함된 [Support 케이스를 만드세요]({{site.baseurl}}/user_guide/administrative/access_braze/support/).

{% alert warning %}
작업 공간을 삭제할 때는 주의하세요! 워크스페이스가 삭제된 후에는 복원할 수 없습니다.
{% endalert %}

작업 공간을 삭제할 수 있는 버튼과 앱을 삭제할 수 있는 휴지통 아이콘이 있는 앱 설정 페이지.]({% image_buster /assets/img/workspaces/workspace_delete.png %})

## 자주 묻는 질문

### 업데이트된 앱을 출시할 때 새 워크스페이스를 만들어야 하나요?

이는 앱을 업데이트하는 것인지 완전히 새로운 앱을 만드는 것인지에 따라 다릅니다.

#### 앱 업데이트하기

앱을 업데이트하는 경우 동일한 작업 공간 내에 새 앱 인스턴스를 만들어 이전 버전과 새 버전을 분리해야 합니다. 이렇게 하면 세그먼트화 중에 해당 앱을 선택하면 새 버전에서 사용자를 효과적으로 타겟팅할 수 있습니다. 이전 버전을 사용하는 사용자에게 메시지를 보내려면 필터를 사용하여 [이전 앱 버전을 타겟팅할]({{site.baseurl}}/user_guide/engagement_tools/campaigns/ideas_and_strategies/new_features#filtering-by-most-recent-app-versions) 수 있습니다.

새 워크스페이스를 만들면 사용자는 이전 워크스페이스와 새 워크스페이스의 두 곳에 존재하게 됩니다. 또한 잠재적으로 동일한 푸시 토큰을 가질 수도 있습니다. 이로 인해 사용자는 이미 업그레이드한 경우에도 이전 워크스페이스 사용자만을 대상으로 하는 마케팅 메시지를 받게 될 수 있습니다.

#### 새 앱 출시하기

완전히 새로운 앱을 앱 스토어에 출시하는 경우 새 워크스페이스를 만들어야 합니다. 새 작업 공간을 만들면 이전 앱 버전의 모든 기록 데이터와 사용자 프로필이 이 새 작업 공간에 존재하지 않게 됩니다. 따라서 기존 사용자가 새 앱 버전으로 업그레이드하면 이전 앱의 행동 데이터 없이 새 프로필이 생성됩니다.

### 하나의 워크스페이스에 여러 개의 앱 인스턴스가 있는데 어떻게 하면 하나의 앱만 메시지를 타겟팅할 수 있나요? {#singular-app}

메시지가 특정 앱만 타겟팅하도록 하려면 선택한 앱 인스턴스의 사용자만 타겟팅하는 세그먼트를 추가하세요. 이는 사용자가 동일한 작업 공간에서 서로 다른 앱 인스턴스에 대한 푸시 토큰을 두 개 가지고 있는 경우 특히 중요합니다. 이 시나리오에서 사용자는 현재 사용 중인 앱이 아닌 다른 앱에 대한 알림을 받을 수 있습니다. 이상적인 경험은 아닙니다!

기본값으로 세그먼트는 워크스페이스의 모든 앱과 웹사이트를 타겟팅합니다. 하나의 앱 또는 웹사이트만 타겟팅하는 세그먼트를 설정하려면:

1. 의미 있는 이름으로 세그먼트를 만듭니다. Braze에서는 "모든 사용자({이름} {플랫폼})" 형식을 사용합니다. 예: "모든 사용자(항해 시 iOS)".
2. **타겟팅할 앱 및 웹사이트의** 경우 **특정 앱의 사용자를** 선택합니다.
3. **특정 앱** 드롭다운에서 앱 또는 사이트를 선택합니다.

특정 앱의 사용자를 타겟팅하는 세그먼트입니다.]({% image_buster /assets/img/workspaces/users_from_specific_apps_filter.png %})

그런 다음 이 세그먼트를 메시지에 추가하고 필요한 경우 추가 세그먼트와 필터를 사용하여 오디언스를 더욱 세분화할 수 있습니다.

#### 캠페인

캠페인의 경우 컴포저의 **타겟 오디언스** 단계에 세그먼트를 추가합니다.

#### 캔버스

캔버스에서 **전달 유효성 검사** 섹션의 메시지 단계에 세그먼트를 추가합니다. 전달 유효성 검사는 메시지 전송 시 오디언스가 전달 기준을 충족하는지 다시 한 번 확인합니다. 각 메시지 단계마다 전달 유효성 검사를 지정하여 올바른 앱으로 전달되는지 확인하세요. 엔트리 레벨에서는 세그먼트를 세분화할 필요가 없습니다.

{% details Expand for steps in the original Canvas workflow %}

기존 캔버스 워크플로우에서 **오디언스** 섹션의 캔버스 컴포넌트 레벨에 세그먼트를 추가합니다. 엔트리 레벨에서는 세그먼트를 세분화할 필요가 없습니다.

{% enddetails %}
