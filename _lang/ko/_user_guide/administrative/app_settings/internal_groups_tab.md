# 내부 그룹

> 내부 그룹은 내부 또는 타사 테스트 그룹을 만들고 구성할 수 있는 좋은 방법입니다. SDK 또는 API 로그에 대한 인사이트를 제공하며 SDK 통합을 테스트할 때 유용합니다. 최대 1,000명의 사용자로 구성된 사용자 지정 내부 그룹을 무제한으로 만들 수 있습니다.

{% alert tip %}
We also recommend checking out our [Testing and Troubleshooting](https://learning.braze.com/path/developer/testing-and-troubleshooting) Braze Learning course, which covers how to use internal groups to conduct your own troubleshooting and debugging.
{% endalert %}

## 필수 조건

내부 그룹을 만들고 관리하려면 워크스페이스에 대한 [액세스 개발자 콘솔 권한이]({{site.baseurl}}/user_guide/administrative/manage_your_braze_users/user_permissions/#limited-and-team-role-permissions) 필요합니다.

## 내부 그룹 만들기

내부 그룹을 만들려면 다음과 같이 하세요: 

1. **설정** > **내부 그룹으로** 이동합니다.
2. **내부 그룹 만들기를** 선택합니다.
3. 그룹 이름을 '이메일 테스트 그룹'과 같이 지정합니다.
4. 다음 표에 나열된 대로 하나 이상의 그룹 유형을 선택합니다.

| 그룹 유형         | 설명                                                                                 |
|--------------------|---------------------------------------------------------------------------------------------|
| **사용자 이벤트 그룹**   | 테스트 기기의 이벤트 또는 로그를 확인하려면 이 기능을 사용하세요.                                    |
| **콘텐츠 테스트 그룹** | 푸시, 이메일, 인앱 메시지에서 이를 사용하여 렌더링된 메시지 사본을 전송할 수 있습니다. |
| **시드 그룹**         | 이메일 전송 시 시드 그룹의 모든 사람에게 이메일 사본을 자동으로 보냅니다.               |

### 테스트 사용자 추가

내부 그룹을 만든 후 테스트 사용자를 해당 그룹의 멤버로 추가합니다. 

1. 내부 그룹의 관리 페이지에서 **테스트 사용자 추가를** 선택합니다.
2. 테스트 사용자를 검색하고 선택하려면 다음 방법 중에서 선택하세요.

| 방법                  | 설명                                                                                                                                                                                                                                          |
|-------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **신원 확인된 사용자 추가** | 외부 사용자 ID, 이메일 주소, 전화번호 또는 푸시 토큰으로 사용자를 검색합니다.                                                                                                                                                           |
| **익명 사용자 추가**  | IP 주소로 검색합니다. 그런 다음 추가하는 각 테스트 사용자의 이름을 입력합니다. [이벤트 사용자 로그]({{site.baseurl}}/user_guide/administrative/app_settings/event_user_log_tab/) 페이지에서 모든 이벤트 로그가 연관되는 이름입니다. |
| **사용자 일괄 추가**      | 이메일 주소 또는 외부 ID 목록을 복사하여 붙여넣습니다. 대시보드에 이미 알려진 사용자만 추가할 수 있습니다. 자세한 내용은 [사용자 가져오기를]({{site.baseurl}}/user_guide/data/user_data_collection/user_import/) 참조하세요.          |

### 콘텐츠 테스트 그룹

메시지의 미리 보기 테스트를 보내는 것과 유사하게 콘텐츠 테스트 그룹을 사용하면 시간을 저장하고 미리 정의된 Braze 사용자 목록에 동시에 테스트를 시작할 수 있습니다. 이는 Braze의 푸시, 인앱 메시지, SMS, 이메일 및 콘텐츠 카드에 사용할 수 있습니다. 콘텐츠 테스트 그룹으로 태그된 그룹만 메시징의 미리 보기 섹션에서 사용할 수 있습니다.

{% alert note %}
[SMS]({{site.baseurl}}/user_guide/message_building_by_channel/sms_mms_rcs/) test messages can only be sent to valid phone numbers in the database.
{% endalert %}

메시지를 보낼 개별 Braze 사용자 또는 원하는 수의 내부 그룹을 선택하세요. 메시징에 Liquid 또는 기타 동적 개인화가 포함된 경우 Braze는 각 사용자가 사용할 수 있는 속성을 사용하여 메시지 콘텐츠를 개인화합니다. 속성이 없는 사용자의 경우 Braze는 설정된 기본값을 사용합니다.

또한 메시지를 무작위 사용자, 커스텀 사용자 또는 기존 사용자로 미리 보는 경우 미리 본 버전을 대신 보낼 수 있습니다. 확인란을 선택 취소하면 미리 보기 버전이 아닌 각 사용자의 속성을 기준으로 전송할 수 있습니다.

IP 풀을 사용하여 이메일을 보내는 경우 사용 가능한 드롭다운에서 풀을 선택하여 이메일을 보낼 IP 풀을 선택합니다.

### 시드 그룹

시드 그룹은 이메일 채널에 대해서만 지원됩니다. 시드 그룹에 사용자를 추가하여 각 이메일 배리언트 메시지의 사본을 그룹의 모든 구성원에게 보내세요.

시드 그룹은 API 캠페인에 사용할 수 없지만, 캠페인에서 API 트리거 항목을 사용하여 시드 그룹을 포함할 수 있습니다. 이를 사용하여 전달 가능성 측정기준을 측정하고 기록 및 보관 목적으로 이메일 콘텐츠를 기록할 수 있습니다. 

내부 그룹을 만들고 시드 그룹으로 사용할 태그를 지정한 후 캠페인 에디터의 **타겟 오디언스** 단계 또는 캔버스의 **보내기 설정** 단계에서 해당 그룹을 선택합니다. 

시드 이메일은 이메일 제목 줄의 시작 부분에 `[SEED]` 가 추가됩니다. 시드 이메일은 **그렇지 않다는** 점에 유의하세요:

- 증분은 대시보드 분석에서 전송합니다.
- 이메일 분석 또는 리타겟팅에 영향을 미칩니다. 
- 사용자 프로필의 **캠페인 수신** 목록을 업데이트합니다.
- 충격 주파수 제한.
- 전달 속도 제한을 고려하거나 영향을 미칩니다.

#### 구독 동작

시드 전송은 내부 QA 및 검토용으로 설계되었으므로 시드된 대시보드 사용자에 대한 구독 확인을 의도적으로 우회합니다. 즉, 시드 그룹에 속해 있는 유효한 이메일 주소를 가진 사용자는 가입하지 않아도 메시지를 받게 됩니다. 그러나 해당 그룹에 시드 사본을 보내도록 메시지를 구성해야 합니다.

{% alert tip %}
시드 그룹 회원들이 받은 편지함에서 메시지를 볼 수 없다고 보고하는 경우, 해당 회원들이 내부 그룹에 등록되어 있는지 확인하고, 제목이 다른지, Gmail에서 이메일을 한데 묶어 보내지 않았는지 확인하거나 스팸 폴더를 확인하도록 하세요.
{% endalert %}

#### 캠페인의 경우

이메일 캠페인을 작성할 때 편집기의 **타겟 오디언스** 섹션에서 시드 그룹을 편집합니다.

{% alert important %}
모든 캠페인에 자동으로 연결되도록 시드 그룹을 구성하는 경우 새 캠페인에만 적용됩니다. 기존 캠페인을 복사하는 경우에는 적용되지 않습니다. **타겟 오디언스** 섹션에서 복사한 캠페인에 원하는 시드 그룹을 수동으로 적용해야 합니다.
{% endalert %}

시드 그룹은 각 이메일 배리언트에 한 번씩 전송되며 사용자가 해당 특정 배리언트를 처음 수신할 때 전달됩니다. 예약된 메시지의 경우 일반적으로 캠페인이 처음 시작될 때 이 메시지가 표시됩니다. 액션 기반 또는 API 트리거 캠페인의 경우 첫 번째 사용자에게 메시지가 전송되는 시점입니다.

캠페인이 다변량이고 배리언트의 전송 비율이 0%인 경우 시드 그룹에 전송되지 않습니다. 또한 배리언트가 이미 전송되었고 **대상** 단계의 **시드 그룹 편집에서** 다시 전송하도록 업데이트하지 않은 경우 기본값으로 다시 전송되지 않습니다.

{% alert note %}
반복 캠페인이 있고 이형 상품 중 하나가 업데이트된 경우 업데이트된 이형 상품만 또는 모든 이형 상품에 다시 보내도록 선택하거나 업데이트 시 시드 그룹 전송을 해제할 수 있습니다.
{% endalert %}

#### 캔버스용

캔버스의 시드 그룹은 트리거된 캠페인과 유사하게 작동합니다. Braze는 이메일 메시지가 포함된 모든 단계를 자동으로 감지하여 사용자가 특정 이메일 단계에 처음 도달할 때 해당 단계로 전송합니다.

시드 그룹이 발송된 후 이메일 단계가 업데이트된 경우 Braze는 업데이트된 단계, 모든 단계로만 전송하거나 시드를 해제하는 옵션을 제공합니다.
