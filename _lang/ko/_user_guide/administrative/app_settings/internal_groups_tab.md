# 내부 그룹

> 내부 그룹은 내부 또는 타사 테스트 그룹을 구축하고 구성할 수 있는 좋은 방법입니다. SDK 또는 API 로그에 대한 인사이트를 제공하며 SDK 통합을 테스트할 때 유용합니다. 최대 1,000명의 사용자로 구성된 커스텀 내부 그룹을 무제한으로 만들 수 있습니다.

{% alert tip %}
또한 내부 그룹을 사용하여 직접 문제 해결 및 디버깅을 수행하는 방법을 다루는 [테스트 및 문제 해결](https://learning.braze.com/path/developer/testing-and-troubleshooting) Braze 학습 과정도 확인해 보시기 바랍니다.
{% endalert %}

## 전제 조건

내부 그룹을 만들고 관리하려면 먼저 워크스페이스에 대한 [액세스 개발자 콘솔 권한이]({{site.baseurl}}/user_guide/administrative/manage_your_braze_users/user_permissions/#limited-and-team-role-permissions) 필요합니다.

## 내부 그룹 만들기

내부 그룹을 만들려면 다음과 같이 하세요: 

1. **설정** > **내부 그룹으로** 이동합니다.
2. **내부 그룹 만들기를** 선택합니다.
3. 그룹에 '이메일 테스트 그룹'과 같은 이름을 지정합니다.
4. 다음 표에 나열된 대로 하나 이상의 그룹 유형을 선택합니다.

| 그룹 유형         | 설명                                                                                 |
|--------------------|---------------------------------------------------------------------------------------------|
| **사용자 이벤트 그룹**   | 테스트 기기의 이벤트 또는 로그를 확인하는 데 사용됩니다.                                    |
| **콘텐츠 테스트 그룹** | 푸시, 이메일, 인앱 메시지에서 렌더링된 메시지 사본을 전송하는 데 사용할 수 있습니다. |
| **시드 그룹**         | 이메일 전송 시 시드 그룹의 모든 사람에게 이메일 사본을 자동으로 보냅니다.               |

### 테스트 사용자 추가하기

내부 그룹을 만든 후 테스트 사용자를 해당 그룹의 멤버로 추가할 수 있습니다. 

1. 내부 그룹의 관리 페이지에서 **테스트 사용자 추가를** 선택합니다.
2. 테스트 사용자를 검색하고 선택하려면 다음 방법 중에서 선택하세요.

| 방법                  | 설명                                                                                                                                                                                                                                          |
|-------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **식별자 사용자 추가하기** | 외부 사용자 ID, 이메일 주소, 전화번호 또는 푸시 토큰으로 사용자를 검색합니다.                                                                                                                                                           |
| **익명 사용자 추가하기**  | IP 주소로 검색합니다. 그런 다음 추가되는 각 테스트 사용자의 이름을 입력합니다. [이벤트 사용자 로그]({{site.baseurl}}/user_guide/administrative/app_settings/event_user_log_tab/) 페이지에서 모든 이벤트 로그가 연결되는 이름입니다. |
| **사용자 일괄 추가**      | 이메일 주소 또는 외부 ID 목록을 복사하여 붙여넣습니다. 대시보드에 이미 알려진 사용자만 추가할 수 있습니다. 자세한 내용은 [사용자 가져오기를]({{site.baseurl}}/user_guide/data/user_data_collection/user_import/) 참조하세요.          |

### 콘텐츠 테스트 그룹

메시징의 미리보기 테스트를 보내는 것과 유사하게 콘텐츠 테스트 그룹을 사용하면 시간을 저장하고 미리 정의된 Braze 사용자 목록에 동시에 테스트를 시작할 수 있습니다. 이는 Braze에서 푸시, 인앱 메시지, SMS, 이메일 및 콘텐츠 카드에 사용할 수 있습니다. 콘텐츠 테스트 그룹으로 태그된 그룹만 메시징의 미리보기 섹션에서 볼 수 있습니다.

{% alert note %}
[SMS]({{site.baseurl}}/user_guide/message_building_by_channel/sms_mms_rcs/) 테스트 메시지는 데이터베이스에 있는 유효한 전화번호로만 보낼 수 있습니다.
{% endalert %}

개별 Braze 사용자 또는 메시지를 보낼 내부 그룹을 선택할 수 있습니다. 메시징에 Liquid 또는 기타 동적 개인화가 포함된 경우, Braze는 각 사용자가 사용할 수 있는 속성을 사용하여 메시지 콘텐츠를 개인화합니다. 속성이 없는 사용자의 경우 Braze는 설정된 기본값을 사용합니다.

또한 메시지를 무작위 사용자, 커스텀 사용자 또는 기존 사용자로 미리 보는 경우 미리 본 버전을 대신 보낼 수 있습니다. 확인란을 선택 취소하면 미리 보기 버전이 아닌 각 사용자의 속성을 기준으로 전송할 수 있습니다.

IP 풀을 사용하여 이메일을 보내는 경우 사용 가능한 드롭다운에서 풀을 선택하여 이메일을 보낼 IP 풀을 선택할 수 있습니다.

인앱 메시지 에디터의 테스트 섹션에서 콘텐츠 테스트 그룹을 선택합니다.]({% image_buster /assets/img_archive/content_test_preview.png %}){: style="max-width:60%" }

### 시드 그룹

시드 그룹은 이메일 채널에만 지원됩니다. 시드 그룹에 사용자를 추가하여 각 이메일 배리언트 메시지의 사본을 그룹의 모든 구성원에게 보낼 수 있습니다.

시드 그룹은 API 캠페인에 사용할 수 없지만, 캠페인에서 API 트리거 항목을 사용하여 시드 그룹을 포함할 수 있습니다. 이를 사용하여 전달 가능성 측정기준을 측정하고 기록 및 보관 목적으로 이메일 콘텐츠를 기록할 수 있습니다. 

내부 그룹을 생성하고 시드 그룹으로 사용할 태그를 지정한 후 캠페인 에디터의 **타겟 오디언스** 단계 또는 캔버스의 **보내기 설정** 단계에서 해당 그룹을 선택할 수 있습니다. 

시드 이메일에는 이메일 제목란에 `[SEED]` 가 추가됩니다. 시드 이메일은 **그렇지 않습니다**:

- 증분은 대시보드 분석에서 전송합니다.
- 이메일 분석 또는 리타겟팅에 영향을 미칩니다. 
- 고객 프로필의 **캠페인 수신** 목록을 업데이트합니다.
- 임팩트 최대 게재빈도 설정.
- 전달 속도 제한을 고려하거나 영향을 미칩니다.

{% alert tip %}
시드 그룹 구성원이 받은편지함에 메시지가 보이지 않는다고 신고하는 경우 내부 그룹에 등록되어 있는지 확인하고, 제목란이 다른지, Gmail에서 이메일을 한데 묶어 보내지 않았는지 확인하거나 스팸 폴더를 확인하도록 하세요.
{% endalert %}

#### 캠페인용

이메일 캠페인을 작성할 때 편집기의 **타겟 오디언스** 섹션에서 시드 그룹을 편집할 수 있습니다.

시드 그룹은 각 이메일 배리언트에 한 번씩 전송되며 사용자가 특정 배리언트를 처음 수신할 때 전달됩니다. 예약된 메시징 캠페인의 경우 일반적으로 캠페인이 처음 시작될 때 이 메시지가 표시됩니다. 액션 기반 또는 API 트리거 캠페인의 경우 첫 번째 사용자에게 메시지가 전송되는 시점이 됩니다.

캠페인이 다변량이고 배리언트의 전송 비율이 0%인 경우 시드 그룹에 전송되지 않습니다. 또한 배리언트가 이미 전송되었고 **타겟** 단계의 **시드 그룹 편집에서** 다시 전송하도록 업데이트하지 않은 경우 기본값으로 다시 전송되지 않습니다.

{% alert note %}
반복 캠페인이 있고 배리언트 중 하나가 업데이트된 경우 업데이트된 배리언트만 또는 모든 배리언트로 다시 보내도록 선택하거나 업데이트 시 시드 그룹 전송을 해제할 수 있습니다.
{% endalert %}

배리언트 1 이메일 캠페인을 보내도록 선택한 "이메일 시드 테스트" 시드 그룹입니다.]({% image_buster /assets/img_archive/seed_group_campaign.png %})

#### 캔버스용

캔버스의 시드 그룹은 트리거된 캠페인과 유사하게 작동합니다. Braze는 이메일 메시지가 포함된 모든 단계를 자동으로 감지하여 사용자가 특정 이메일 단계에 처음 도달하면 해당 단계로 전송합니다.

시드 그룹이 발송된 후 이메일 단계가 업데이트된 경우에는 업데이트된 단계로만 보내기, 모든 단계로 보내기 또는 시드 끄기 옵션이 표시됩니다.
