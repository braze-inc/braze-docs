# 알림 기본 설정

> 회사 계정의 메시징 및 활동을 모니터링하려면 특정 알림을 설정하고 알림이 전송될 위치를 선택할 수 있습니다.

**알림 기본 설정** 페이지에서 누가(있다면) 회사에 대한 알림을 받을지 구성할 수 있습니다. 캠페인 전송 또는 기술 오류에 대한 알림을 받을 사람을 구성할 수 있습니다. 주간 분석 보고서의 수신자를 지정할 수도 있습니다. 대부분의 알림에 대해 Braze는 이메일 및 웹후크 채널을 지원합니다.

\![Braze 대시보드의 알림 기본 설정 페이지]({% image_buster /assets/img_archive/notification_preferences.png %})

이 페이지에 액세스하려면 **설정** > **관리자 설정** > **알림 기본 설정**으로 이동하십시오.

## 사용 가능한 알림

다음 표는 사용 가능한 알림과 이를 전달하는 데 사용되는 채널을 설명합니다.

| 알림 | 설명 | 사용 가능한 알림 채널 |
|--------------|-------------|-----------------|
| AWS 자격 증명 오류 | Braze가 데이터 내보내기를 위해 Amazon Web Services 자격 증명을 사용하려고 시도하는 동안 오류가 발생하면 수신자에게 알립니다. 여기에는 Google Cloud Services 및 Azure(마이크로소프트 클라우드 서비스)에 대한 자격 증명 오류 알림이 포함됩니다. | 이메일, 웹후크 |
| 캠페인이 자동으로 중지됨 | Braze가 캠페인을 중단했을 때 수신자에게 알림을 보냅니다. | 이메일 |
| 캠페인 상호작용 만료 | 캠페인 상호작용 데이터 만료가 예정된 캠페인에 대해 수신자에게 알림을 보냅니다. 이 알림에는 리타게팅 필터에서 참조된 세그먼트, 캠페인 또는 캔버스에 대한 정보와 지난 30일 동안 메시지를 보내는 데 사용된 정보가 포함됩니다. | 이메일 |
| 캠페인/캔버스 업데이트됨 | 활성 캠페인 또는 캔버스가 업데이트되거나 비활성화될 때, 비활성 캠페인 또는 캔버스가 재활성화되거나 초안이 시작될 때 수신자에게 알림을 보냅니다. | 이메일 |
| 캠페인/캔버스 볼륨 한도 도달 | 캠페인 또는 캔버스가 볼륨 한도에 도달했을 때 수신자에게 알림을 보냅니다. | 이메일 | 
| 캔버스 상호작용 만료 | 캔버스 상호작용 데이터 만료가 예정된 캔버스에 대해 수신자에게 알림을 보냅니다. 이 알림에는 리타게팅 필터에서 참조된 세그먼트, 캠페인 또는 캔버스에 대한 정보와 지난 30일 동안 메시지를 보내는 데 사용된 정보가 포함됩니다. | 이메일 |
| 푸시 자격 증명 오류 | 앱의 푸시 자격 증명이 유효하지 않거나 곧 만료될 때 수신자에게 알림을 보냅니다. | 이메일, 웹후크 |
| 예약된 캠페인 전송/미전송 | 예약된 캠페인이 전송을 시작할 때 또는 예약된 캠페인이 전송을 시도하지만 전송할 수 있는 적격 사용자가 없을 때 수신자에게 알림을 보냅니다. | 이메일, 웹후크 |
| 예약된 캠페인 한도 도달 | 정기적으로 예약된 캠페인의 한도에 도달했을 때 수신자에게 알림을 보냅니다. | 이메일, 웹후크 |
| 예약된 캠페인 전송 완료 | 예약된 캠페인이 전송을 완료했을 때 수신자에게 알림을 보냅니다. | 이메일, 웹후크 |
| 주간 분석 보고서 | 매주 월요일에 지난 주의 작업 공간 활동 요약을 수신자에게 보냅니다. 수신자는 자신이 속한 각 작업 공간에 대한 요약을 받습니다. | 이메일 |
| 일일 캔버스/캠페인 입력 볼륨 한도 | 전송 한도에 도달할 때마다 알림을 보냅니다. | 이메일 |

## 주간 분석 보고서

Braze는 매주 월요일 오전 5시 EST에 귀하의 회사 내에서 지정한 개인에게 이메일로 주간 보고서를 선택적으로 보냅니다. 주간 보고서에 포함할 사용자 정의 이벤트를 **데이터 설정** > **사용자 정의 이벤트**에서 선택할 수 있습니다.

주간 보고서에 포함할 최대 다섯 개의 이벤트를 선택할 수 있습니다:

\![분석 보고서에 포함할 이벤트 선택]({% image_buster /assets/img_archive/company_analytics_report_new.png %})

## 슬랙 수신 웹후크 통합

슬랙에는 외부 소스에서 슬랙으로 메시지를 게시할 수 있는 [수신 웹후크 앱](https://my.slack.com/services/new/incoming-webhook/)이 있습니다. 시작하려면 수신 웹후크 앱을 엽니다.

1. 알림이 전송될 슬랙 채널을 선택하고 **수신 웹후크 통합 추가**를 클릭합니다.<br><br>
    \![슬랙에서 수신 웹후크 통합 추가]({% image_buster /assets/img_archive/slack_f.png %})<br><br>
  슬랙은 수신하고자 하는 알림을 위해 Braze에 입력해야 할 URL을 생성합니다.<br><br>
2. **웹후크 URL**을 복사합니다.<br><br>
    \![웹후크 URL 복사]({% image_buster /assets/img_archive/copy_url.png %})<br><br>
3. **회사 설정**의 **알림 기본 설정** 탭으로 이동합니다.<br><br>
4. 슬랙에 대해 활성화하려는 알림을 선택합니다. 또는 이 슬랙 채널로 보내고자 하는 여러 알림이 있는 경우 **일괄 추가**를 사용하여 여러 알림에 웹후크를 추가합니다.<br><br>
    \![슬랙 알림을 활성화하려면 선택하세요]({% image_buster /assets/img_archive/click_edit_f.png %}){: style="max-width:60%;"}<br><br>
5. 슬랙에서 생성한 URL을 입력하세요.

그게 전부입니다! 이 슬랙 채널에서 귀사의 알림을 받기 시작해야 합니다. 이 주제에 대한 슬랙의 도움말 기사를 확인할 수도 있습니다: [인커밍 웹후크를 사용하여 메시지 전송](https://api.slack.com/incoming-webhooks).
