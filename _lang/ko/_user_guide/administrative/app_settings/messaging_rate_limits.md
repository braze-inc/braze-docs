# 워크스페이스 메시징 사용량 제한

> 워크스페이스 메시징 사용량 제한을 사용하여 플랫폼에서 발송되는 메시지의 전달 속도를 조절하고, 사용자가 필요한 메시지를 받을 수 있도록 합니다.

{% alert important %}
워크스페이스 메시징 사용량 제한은 점진적으로 출시되고 있습니다. 아직 대시보드에서 이 설정이 표시되지 않을 수 있습니다.
{% endalert %}

## 작동 방식

워크스페이스 메시징 사용량 제한은 워크스페이스에서 발송되는 메시지의 총합에 적용됩니다. 워크스페이스 수준에서 사용량 제한을 설정하고 최적화하면 Braze 메시지의 발신 트래픽을 더 잘 제어할 수 있으며, 서버 성능에 영향을 줄 수 있는 잠재적인 수요 급증을 방지할 수 있습니다.
{% alert note %}
`/messages/send` 및 `/messages/schedule/create`와 같은 API 메시징 엔드포인트를 사용하여 발송된 메시지도 워크스페이스 메시징 사용량 제한에 포함되며 영향을 받습니다.
{% endalert %}
분당 발송되는 총 메시지 수는 설정된 워크스페이스 사용량 제한을 초과하지 않습니다. 처음 몇 분과 이후 몇 분 사이에 어떤 캠페인이 먼저 발송되는지에 대한 특정 순서는 없습니다.

예를 들어, 워크스페이스 메시징 사용량 제한이 분당 100,000개의 메시지이고, 다음 메시지가 모두 오후 12시에 처리되고 있다고 가정해 보겠습니다:

| 캠페인   | 메시지 수 | 발송 시간 |
|------------|--------------------|-----------|
| 캠페인 1 | 100,000            | 오후 12시     |
| 캠페인 2 | 100,000            | 오후 12시     |
| 캠페인 3 | 100,000            | 오후 12시     |
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3 role="presentation" }

메시지는 3분 간격에 걸쳐 발송됩니다.

메시지는 병렬로 처리됩니다. 처리 시 메시지는 선착순으로 워크스페이스 메시징 사용량 제한을 준수하도록 스케줄됩니다. 즉, 위의 예에서 매분 발송되는 메시지는 캠페인 1, 2, 3의 다양한 조합으로 합계 100,000개가 됩니다.

![세 캠페인의 메시지가 어떻게 발송되는지 보여주는 예시.]({% image_buster /assets/img/workspace_messaging_rate_limits2.png %})

다음 예시에서는 워크스페이스 메시징 사용량 제한이 분당 100,000개의 메시지이고, 다음과 같은 메시지가 설정되어 있다고 가정합니다:

| 캠페인   | 메시지 수 | 발송 시간 |
|------------|--------------------|-----------|
| 캠페인 1 | 1,000,000          | 오전 9시      |
| 캠페인 2 | 1,000,000          | 오전 9시 5분   |
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3 role="presentation" }

다음은 예상 발송 스케줄과 분당 발송 메시지 수입니다:

- 캠페인 1은 오전 9시~9시 10분에 스케줄되며, 분당 100,000개의 메시지가 발송됩니다.
- 캠페인 2는 원래 발송 시간보다 5분 후인 오전 9시 10분~9시 20분에 발송되도록 스케줄되며, 분당 100,000개의 메시지가 발송됩니다.

![두 캠페인의 분당 메시지 발송 방식을 보여주는 예시.]({% image_buster /assets/img/workspace_messaging_rate_limits1.png %})

{% alert note %}
워크스페이스 메시징 사용량 제한을 설정한 후에도 늘릴 수 있습니다. 그러나 증가 이전에 이미 처리된 모든 메시지는 이전에 설정된 제한이 적용됩니다.
{% endalert %}

## 워크스페이스 메시징 사용량 제한 설정

1. Braze 대시보드에서 **설정** > **워크스페이스 설정** > **워크스페이스 메시징 사용량 제한**으로 이동합니다.
2. **+ 사용량 제한 추가**를 선택한 다음 메시징 채널을 선택합니다.
3. **분당 메시지 수**에 사용량 제한을 입력합니다.
4. **저장**을 선택합니다.

## 알아두어야 할 사항

사용량 제한은 발송 시점, 즉 메시지 발송 시도가 시작되는 시점에 적용됩니다. 발송 완료까지 걸리는 시간에 변동이 있을 경우, 일부 분에서 완료된 발송 수가 사용량 제한을 약간 초과할 수 있습니다. 시간이 지나면 분당 발송 수의 평균은 사용량 제한을 초과하지 않습니다.

캠페인이나 캔버스에 자체 사용량 제한이 설정되어 있고 워크스페이스 수준의 사용량 제한도 적용되는 경우, 두 가지 모두 적용됩니다. 예를 들어, 캠페인의 사용량 제한이 500,000이지만 워크스페이스 사용량 제한으로 인해 현재 분당 100,000개의 메시지만 발송할 수 있는 경우, 워크스페이스 사용량 제한이 적용됩니다.

Braze는 메시지 발송을 분 단위로 균등하게 분배하려고 하지만, 이를 보장할 수는 없습니다. 예를 들어, 캠페인의 사용량 제한이 분당 500,000개의 메시지인 경우, 500,000개의 메시지를 분 전체에 걸쳐 균등하게 분배하려고 합니다(초당 약 8,400개의 메시지). 하지만 초당 속도에 약간의 변동이 있을 수 있습니다.

캠페인과 캔버스에서 개별 사용량 제한을 설정할 수도 있습니다. 이러한 제한은 워크스페이스 메시징 사용량 제한과 독립적으로 적용됩니다.

### 워크스페이스 메시징 사용량 제한에 포함되지 않는 메시지

- [트랜잭션 이메일 캠페인]({{site.baseurl}}/user_guide/message_building_by_channel/email/transactional_message_api_campaign)을 사용하여 발송된 메시지는 워크스페이스 메시징 사용량 제한에 포함되지 않습니다. 즉, 자체적으로 사용량 제한이 적용되며 설정된 워크스페이스 메시징 사용량 제한에 포함되지 않습니다.
- [시드 그룹]({{site.baseurl}}/user_guide/administrative/app_settings/internal_groups_tab#seed-groups) 및 [테스트 발송]({{site.baseurl}}/user_guide/engagement_tools/campaigns/testing_and_more/sending_test_messages)으로 보내는 메시지는 워크스페이스 메시징 사용량 제한에 포함되지 않습니다. 즉, 사용량 제한이 적용되지 않으며 설정된 워크스페이스 메시징 사용량 제한에 포함되지 않습니다.
- SMS 자동 응답은 워크스페이스 메시징 사용량 제한에 포함되지 않습니다. 즉, 사용량 제한이 적용되지 않으며 설정된 워크스페이스 메시징 사용량 제한에 포함되지 않습니다.
