# 수집 사용 사례

> 이 기사에서는 라이드 셰어링 앱이 어떤 사용자 데이터를 수집할지 결정하는 사용자 데이터 수집 사용 사례를 다룹니다.

택시 또는 라이드 셰어링 앱인 StyleRyde가 어떤 사용자 데이터를 수집할지 결정하고자 한다고 가정해 보겠습니다. 다음 질문과 브레인스토밍 과정은 그들의 마케팅 및 개발 팀이 따르기에 훌륭한 모델입니다. 이 연습이 끝날 때쯤, 두 팀은 목표를 달성하는 데 도움이 되는 커스텀 이벤트와 수집할 속성이 무엇인지에 대한 확고한 이해를 가져야 합니다.

## 사례 질문 1: 목표는 무엇인가요?

StyleRyde의 목표는 사용자가 그들의 앱을 통해 택시를 호출하는 것입니다.

## 사례 질문 2: 앱 설치 후 그 목표에 도달하기 위한 단계는 무엇인가요?

1. StyleRyde는 사용자가 등록 프로세스를 시작하고 개인 정보를 입력해야 합니다.
2. StyleRyde는 사용자가 SMS를 통해 받은 앱에 코드를 입력하여 등록 프로세스를 완료하고 확인해야 합니다.
3. StyleRyde는 사용자가 택시를 호출하려고 시도해야 합니다.
4. StyleRyde는 사용자가 택시를 호출할 때 이용 가능해야 합니다.

이러한 작업은 다음과 같은 사용자 지정 이벤트로 태그를 지정할 수 있습니다:

- 등록 시작
- 등록 완료
- 성공적인 택시 호출
- 택시 호출 실패

이벤트를 구현한 후, StyleRyde는 다음을 포함한 캠페인을 실행할 수 있습니다:

1. 등록을 시작했지만 특정 기간 내에 등록을 완료하지 않은 사용자에게 메시지를 보냅니다.
2. 등록을 완료한 사용자에게 축하 메시지를 보냅니다.
3. 택시 호출에 실패한 후 일정 시간 내에 택시 호출에 성공하지 못한 사용자에게 사과와 프로모션 크레딧을 보내세요.
4. 택시 호출에 성공한 횟수가 많은 파워 유저에게 프로모션을 보내 로열티에 대한 감사의 마음을 전하세요.

## 사례 질문 3: 우리가 수집하고 메시징에 정보를 제공하는 데 사용할 수 있는 다른 사용자 정보는 무엇입니까?

- 사용자에게 프로모션 크레딧이 있습니까?
- 사용자들이 운전사에게 주는 평균 평점은?
- 사용자를 위한 고유 프로모션 코드?

그런 다음 이러한 특성에 다음과 같은 사용자 지정 속성으로 태그를 지정할 수 있습니다:

- 프로모션 크레딧 잔액(소수점 유형)
- 평균 운전자 평점(정수형)
- 고유 프로모션 코드(문자열 유형)

이러한 속성을 사용하면 사용자에게 캠페인을 보낼 수 있습니다:

1. 사용하지 않은 지 7일이 지난 사용자에게 계정에 프로모션 크레딧이 있는 경우 앱으로 돌아와서 크레딧을 사용하도록 알림.
2. 메시지 템플릿 및 [개인화 기능을][13] 사용하여 고유 프로모션 코드 속성을 사용자를 대상으로 하는 메시지에 끌어다 놓을 수 있습니다.
