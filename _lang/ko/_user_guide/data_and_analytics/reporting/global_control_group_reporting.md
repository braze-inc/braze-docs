# 전역 대조군 보고서

> 글로벌 컨트롤 그룹 보고서를 사용하면 트리트먼트 샘플과 그룹을 비교할 수 있습니다. 트리트먼트 샘플은 무작위 버킷 번호 방법을 사용하여 생성된 대조군과 거의 동일한 수의 비대조군 사용자를 무작위로 선택한 것입니다.

대시보드에서 [글로벌 컨트롤 그룹]({{site.baseurl}}/user_guide/engagement_tools/testing/global_control_group/)에 대한 보고서를 보려면 **분석** > **글로벌 컨트롤 그룹 보고서**로 이동합니다. 

{% alert note %}
[이전 탐색]({{site.baseurl}}/navigation)을 사용하는 경우 이 페이지는 **데이터**에서 찾을 수 있습니다.
{% endalert %}

그런 다음 보고서를 실행할 매개변수(세션 또는 특정 커스텀 이벤트)를 선택하고 **리포트 실행**을 클릭합니다.

![][6]

## 보고서 정보

보고서를 생성할 때 세션 또는 커스텀 이벤트 중 하나를 선택하여 트리트먼트 그룹과 대조군을 비교합니다. 그런 다음 데이터를 볼 기간을 선택합니다. 서로 다른 기간에 여러 개의 대조군 실험을 저장한 경우 보고서에 두 개 이상의 실험 데이터를 포함하지 않아야 합니다.

보고서의 백분율 메트릭은 반올림된다는 점에 유의하세요. 예를 들어, 전환 수가 전체 대조군 또는 트리트먼트 그룹 대비 매우 낮은 비율인 경우 전환율은 0%로 반올림될 수 있습니다.

마지막으로, 플랫폼의 다른 여러 보고서와 마찬가지로 이 보고서에는 제어 지표부터의 변화에 대한 [신뢰도]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/#understanding-confidence) 백분율이 표시됩니다. 대조군과 트리트먼트군 간의 전환율이 동일한 경우 신뢰도는 0%이며, 이는 두 그룹 간에 성과에 차이가 있을 확률이 0%임을 의미합니다.

### 그룹 규모

2024년 5월 이전에는 글로벌 컨트롤 그룹은 사용자 아카이브에서 제외되었지만, 트리트먼트 샘플 그룹은 제외되지 않았습니다. 2024년 5월부터 두 그룹 모두 사용자 아카이브에서 제외됩니다. 이로 인해 트리트먼트 샘플 그룹과 글로벌 컨트롤 그룹의 크기가 크게 달라질 수 있습니다. 다음에 글로벌 컨트롤 그룹을 재설정하면 이 불일치가 해결되고 비슷한 그룹 크기가 표시됩니다.

## 보고서 메트릭

| 측정기준 | 정의 | 계산 |
| -- | -- | -- |
| 대조군에서 변경 | 이렇게 하면 트리트먼트 그룹과 대조군 간의 전환율 상승을 계산할 수 있습니다. | ((트리트먼트 전환율 - 대조군 전환율) ÷ 대조군 전환율) * 100 |
| Incremental Uplift | 트리트먼트 그룹과 대조군 간의 총 이벤트 수 차이입니다. 이 지표는 "트리트먼트 그룹이 얼마나 더 많은 전환 이벤트를 달성했습니까?"라는 질문에 답하기 위한 것입니다. | 트리트먼트용 총 이벤트 - 제어용 총 이벤트 |
| Incremental Uplift 백분율 | 트리트먼트의 전체 이벤트 중 트리트먼트에 기인할 수 있는 이벤트의 비율(자연 사용자 행동 대비)입니다. 이는 증분 상승(수)을 트리트먼트 그룹의 총 이벤트 수로 나누어 계산합니다. | 증분 상승(수) ÷ 트리트먼트 그룹에 대한 총 이벤트 수 |
| 전환율 | 선택한 기간 동안 매일 선택한 이벤트를 완료한 대조군 또는 트리트먼트군 사용자의 평균 백분율입니다. 전환 수가 매우 적고 대조군 또는 트리트먼트군이 매우 많은 경우 전환율은 0%로 반올림될 수 있습니다. | 선택한 기간 동안 매일 선택한 이벤트를 수행하는 사용자 비율의 평균(평균)입니다. |
| 예상 그룹 크기 | 선택한 기간 동안 대조군 및 트리트먼트군에 속한 예상 사용자 수입니다. | 보고서를 위해 선택한 기간 동안 대조군 및 트리트먼트 그룹이 도달한 최대 멤버십 규모입니다. |
| 총 이벤트 수 | 선택한 기간 동안 선택한 이벤트가 발생한 총 횟수입니다. 예를 들어 사용자가 해당 기간 동안 이벤트를 두 번 수행하면 이벤트가 두 번 증가합니다(예: 사용자가 해당 기간 동안 이벤트를 두 번 수행하면 이벤트가 두 번 증가). | 선택한 기간 동안 매일 이벤트가 발생한 횟수의 합계입니다. |
| 사용자 당 이벤트 | 선택한 기간 동안 각 그룹에 속한 사용자가 전환 이벤트를 완료한 예상 평균 횟수입니다. | 총 이벤트 수 ÷ 예상 그룹 규모. |
