# 2026년 3월 5일 출시

## 데이터&보고

### 새로운 데이터 센터

{% multi_lang_include release_type.md release="General availability" %}

브레이즈가 새로운 [데이터 센터]({{site.baseurl}}/user_guide/data/data_centers/)를 출시했습니다: JP-01. Braze 계정을 설정할 때 지역별 데이터 센터에 가입할 수 있습니다.

### 컨텍스트 변수

{% multi_lang_include release_type.md release="General availability" %}

[컨텍스트 변수는]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/context_variables) 사용자가 특정 캔버스를 이동하는 과정에서 생성하고 사용할 수 있는 임시 데이터 조각입니다. 사용자가 캔버스를 입력할 때마다(이전에 입력한 적이 있더라도) 최신 입력 데이터와 캔버스 설정을 기반으로 컨텍스트 변수가 재정의됩니다. 이 접근 방식은 각 캔버스 항목이 자체 독립적인 컨텍스트를 유지할 수 있도록 하여, 사용자가 동일한 여정 내에서 여러 활성 상태를 가질 수 있으면서도 각 상태에 대한 특정 컨텍스트를 유지할 수 있게 합니다.

### 클라우드 데이터 수집 소스

{% multi_lang_include release_type.md release="Early access" %}

[클라우드 데이터 수집에는]({{site.baseurl}}/user_guide/data/unification/cloud_ingestion/file_storage_integrations/#setting-up-cloud-data-ingestion-in-braze) 소스와 동기화를 분리하는 새로운 UI가 도입되어, 단일 소스를 여러 동기화에 재사용할 수 있습니다. 여러 동기화 환경을 구성할 때 중복 설정을 줄이고 설정을 간소화합니다. 기존 동기화가 있는 경우, 중단 시간 없이 자동으로 새로운 소스 및 동기화 구조로 마이그레이션됩니다. 시작하려면 **Cloud Data Ingestion** > **Sources로** 이동하여 소스를 확인, 편집 또는 생성한 후 동기화를 생성할 때 드롭다운에서 소스를 선택하세요.

### 커런츠 및 데이터 공유 이벤트용 추가 필드

{% multi_lang_include release_type.md release="General availability" %}

[현재 및 데이터 공유 이벤트에는]({{site.baseurl}}/user_guide/data/distribution/braze_currents/event_glossary/currents_changelogs#changes-in-version-5-release-date-2026-02-04) 분석 및 하위 시스템에 제공되는 데이터를 심화시키기 위해 다음과 같은 새로운 필드가 포함됩니다:

- `agentconsole.AgentExecuted`: 추가됨`error`(문자열)—발생한 오류에 대한 설명.
- `agentconsole.ToolInvocation`: 추가됨`request_id`(문자열)—전체 LLM 요청 및 완전한 실행에 대한 고유 ID.
- `users.messages.rcs.InboundReceive`: 추가됨`canvas_variation_name`(문자열)—사용자가 받은 캔버스 변형의 이름.

### Snowflake 데이터 공유용 캠페인 및 캔버스 필드

{% multi_lang_include release_type.md release="General availability" %}

[Snowflake 데이터 셰어는]({{site.baseurl}}/user_guide/data/distribution/braze_currents/event_glossary/currents_changelogs/#changes-for-data-sharing-3) 이제 기존 66개 테이블 전반에 걸쳐 캠페인 및 캔버스 정보를 반영하는 추가 필드를 포함합니다. 포함 항목은 다음과 같습니다:

- `campaign_name`
- `canvas_name`
- `canvas_step_name`
- `canvas_variation_name`
- `message_variation_name`
- `conversion_behavior`
- `experiment_split_name`

### CSV 사전 임포트 검증 및 오류 보고

{% multi_lang_include release_type.md release="General availability" %}

[CSV 사용자 가져오기 기능이]({{site.baseurl}}/user_guide/data/user_data_collection/user_import) 이제 가져오기 전 유효성 검사 및 상세한 오류 보고를 지원합니다. **사용자** **가져오기** 페이지에서 가져오기 전에 **'가져오기 전 파일 유효성 검사'를** 선택하세요. Braze가 파일을 스캔하여 완전히 실패할 행(오류)과 일부 값이 생략된 상태로 성공할 행(경고)의 식별자를 포함하는 보고서를 생성합니다. 보고서를 다운로드하여 CSV 파일을 수정 후 다시 업로드하거나, 현재 상태로 진행할 수 있습니다. 가져오기가 완료된 후, 실패한 행에 대한 다운로드 가능한 보고서도 제공되며, 각 문제의 정확한 원인이 포함됩니다.

### 메시징 진단 대시보드

{% multi_lang_include release_type.md release="Early access" %}

[메시징 진단 대시보드는]({{site.baseurl}}/user_guide/analytics/dashboard/diagnostics_dashboard) 메시지 전송 결과를 상위 수준에서 분석하여 메시징 설정의 추세를 파악하고 잠재적 문제를 진단할 수 있도록 지원합니다. 이 대시보드는 캠페인 또는 캔버스에서 보낸 메시지가 예상대로 전송되지 않은 이유를 파악하는 데 도움이 될 수 있습니다.

## BrazeAI<sup>TM</sup>

### 에이전트 콘솔의 Braze 에이전트

{% multi_lang_include release_type.md release="General availability" %}

[Braze 에이전트는]({{site.baseurl}}/user_guide/brazeai/agents/) Braze 내에서 생성할 수 있는 AI 기반 도우미입니다. 에이전트는 콘텐츠를 생성하고, 지능적인 결정을 내리며, 데이터를 풍부하게 하여 더욱 개인화된 고객 경험을 제공할 수 있도록 지원합니다. 에이전트를 생성할 때, 그 목적을 정의하고 어떻게 동작해야 하는지에 대한 가이드라인을 설정합니다. 실행 후 에이전트는 Braze에 [배포되어]({{site.baseurl}}/user_guide/brazeai/agents/deploying_agents) 개인화된 콘텐츠를 생성하거나, 실시간 결정을 내리거나, 카탈로그 필드를 업데이트할 수 있습니다.

### 채널 기반 속도 제한

{% multi_lang_include release_type.md release="General availability" %}

멀티채널 캠페인 또는 캔버스에 대한 전달 속도 제한을 설정할 때, 공유 속도 제한 또는 [채널별 제한]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/#multichannel-campaigns-and-canvases) 중 하나를 선택하여 설정할 수 있습니다. 다중 채널 캠페인 또는 캔버스가 채널 기반 속도 제한을 사용하는 경우, 속도 제한은 선택된 각 채널에 적용됩니다. 예를 들어, 캠페인 또는 캔버스 전체에서 분당 최대 5,000개의 웹훅과 2,500개의 SMS 메시지를 전송하도록 설정할 수 있습니다.

### 캔버스 컨텍스트 단계

{% multi_lang_include release_type.md release="General availability" %}

[캔버스 컨텍스트 단계는]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/context) 사용자가 캔버스를 이동할 때 하나 이상의 사용자 변수를 생성하고 업데이트할 수 있게 합니다. 예를 들어, 시즌별 할인을 관리하는 캔버스가 있는 경우 컨텍스트 변수를 사용하여 사용자가 캔버스에 들어갈 때마다 다른 할인 코드를 저장할 수 있습니다.

### Algolia - 검색 추천

[Algolia는]({{site.baseurl}}/partners/ecommerce/product_search_recommendations/algolia) 개발자가 빠르고 관련성 높으며 확장 가능한 검색 경험을 구축할 수 있도록 지원하는 검색 및 발견 플랫폼입니다. 강력한 API 중심 접근 방식을 통해 Algolia는 고급 순위 알고리즘과 AI 기반 인사이트를 결합하여 원활한 사이트 검색, 탐색 및 개인화된 콘텐츠 발견을 제공합니다.

### Anthropic - AI 모델 제공업체

[앤트로픽은]({{site.baseurl}}/partners/ai_model_providers/anthropic) 다양한 언어 작업에 유용하고 정직하며 안전하게 구축된 차세대 AI 어시스턴트 클로드를 개발하는 AI 안전 및 연구 기업입니다.

### Canva - 메시지 개인화 - 크리에이티브 스튜디오

[Canva는]({{site.baseurl}}/partners/canva) Canva 내 이미지를 Braze 미디어 라이브러리와 직접 동기화하여 크리에이티브 워크플로우를 간소화하고 모든 메시징 채널에서 시각적 자산을 최신 상태로 유지합니다.

### DOTS.ECO \- 보상

[DOTS.ECO]({{site.baseurl}}/partners/additional_channels_and_extensions/extensions/rewards/dots_eco) 사용자에게 추적 가능한 디지털 인증서를 통해 실제 환경적 영향력을 보상할 수 있게 합니다. 각 인증서에는 공유 가능한 인증서 URL 및 이미지 URL과 같은 메타데이터를 포함할 수 있으므로, 사용자는 자신의 영향력 증명을 확인하고 재방문할 수 있습니다.

### 피그마 - 개인화된 메시지 설정 - 크리에이티브 스튜디오

[Figma는]({{site.baseurl}}/partners/figma) 제품을 구축, 설계 및 프로토타이핑할 수 있는 협업 디자인 플랫폼입니다. 이 통합 기능을 사용하여 Figma의 이미지와 시각적 자산을 Braze 미디어 라이브러리로 직접 전송하세요.

### 플라이바이 - 메시지 개인화 - 위치

Radius Networks의 [Flybuy는]({{site.baseurl}}/partners/message_personalization/location/flybuy) 픽업, 전달, 드라이브스루, 매장 내 식사 전반에 걸쳐 서비스 속도를 최적화하기 위해 AI 기반 기술을 활용하는 선도적인 옴니채널 위치 플랫폼입니다. Flybuy는 통합 마케팅 제품군을 통해 브랜드가 초정밀 타겟팅된 순간 기반 메시지를 전달할 수 있도록 지원함으로써 고객 참여를 유도하고 구매 금액을 증대시키며 포괄적인 로열티 프로그램을 뒷받침합니다.

### Google Gemini - AI 모델 제공업체

[Google Gemini는]({{site.baseurl}}/partners/ai_model_providers/google_gemini) 텍스트, 코드, 이미지를 아우르는 고급 추론 능력을 결합하여 브랜드가 더 스마트하고 개인화된 경험을 제공할 수 있도록 지원하는 Google의 AI 모델 제품군입니다.

### 림빅 - 메시지 개인화 - 개인화 엔진

[림빅은]({{site.baseurl}}/partners/message_personalization/dynamic_content/personalization_engines/limbik) 당신의 AI 공명 레이어입니다—실제 오디언스가 메시지, 개념, AI 출력물을 어떻게 해석하고 반응할지 시장 출시 전에 예측합니다. 60개 이상의 국가와 25개 이상의 언어로 진행되는 지속적인 1차 연구를 기반으로, Limbik은 인간 검증된 합성 오디언스를 제공합니다. 이는 실제 오디언스의 반응을 기계 속도로 시뮬레이션하며 연구 등급의 정확도(95% 신뢰도, 1.5%~3% 오차 범위)를 자랑하는 디지털 인구 집단입니다. 림빅은 타겟 오디언스가 믿고 느끼는 바와 메시지가 즉각적으로 공명하도록 보장하는 능력을 제공합니다.

### 링크러너 - 메시지 오케스트레이션 - 기여도

[링크러너는]({{site.baseurl}}/partners/message_orchestration/attribution/linkrunner) 사용자 획득 캠페인을 추적하고 분석하는 데 도움을 주는 모바일 기여도 및 분석 플랫폼입니다.

### Mailizio - 메시지 오케스트레이션 - 템플릿

[Mailizio는]({{site.baseurl}}/partners/message_orchestration/templates/Mailizio) 직관적인 시각적 편집기를 통해 재사용 가능하고 브랜드 안전성을 보장하는 콘텐츠를 쉽게 디자인할 수 있는 이메일 생성 및 관리 플랫폼입니다. Mailizio와 Braze의 통합을 통해 콘텐츠 블록과 이메일 템플릿을 내보낸 후 동일한 자산을 활용해 인앱 메시지를 자동 생성할 수 있어 빠르고 완벽하게 통제된 캠페인 배포가 가능합니다.

### 오픈 로열티 - 데이터 및 분석 - 로열티

[오픈 로열티는]({{site.baseurl}}/partners/data_and_analytics/loyalty/openloyalty) 클라우드 기반 로열티 프로그램 플랫폼으로, 고객 충성도 및 보상 프로그램을 구축하고 관리할 수 있게 해줍니다. Braze와 Open Loyalty의 통합은 포인트 잔액, 등급 변경, 만료 알림 등의 로열티 데이터를 실시간으로 Braze에 직접 동기화합니다. 사용자의 로열티 상태가 변경될 때 개인화된 메시지(이메일, 푸시 알림, SMS)를 발송할 수 있습니다.

### OpenAI - AI 모델 제공업체

[OpenAI는]({{site.baseurl}}/partners/ai_model_providers/openai) GPT와 같은 고급 AI 모델을 개발하여 자연어 이해 및 생성을 가능하게 함으로써, 브랜드가 의미 있는 고객 상호작용을 구축하고 확장할 수 있도록 지원합니다.

### Shopgate - 채널

[Shopgate는]({{site.baseurl}}/partners/additional_channels_and_extensions/additional_channels/shopgate) 모바일 커머스 및 옴니채널 플랫폼으로, 판매자가 쇼핑 앱을 제작하고 주문 처리 도구 및 클라이언텔링(고객 데이터를 기반으로 한 개인화된 매장 내 고객 지원)을 통해 오프라인 매장의 효율성을 개선할 수 있도록 지원합니다.

### 스플리오 - 데이터 및 분석 - 코호트 가져오기

## SDK

### 소프트웨어 개발 키트 호환성 깨지는 업데이트

다음 SDK 업데이트가 릴리스되었습니다. 주요 업데이트는 아래에 나열되어 있으며, 그 외의 모든 업데이트는 해당 SDK 체인지로그를 확인하면 확인할 수 있습니다.

- [Android 소프트웨어 개발 키트 41.1.1](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md)
- [Flutter 소프트웨어 개발 키트 17.1.0](https://pub.dev/packages/braze_plugin/changelog)
- [Swift 소프트웨어 개발 키트 14.0.2](https://github.com/braze-inc/braze-swift-sdk/blob/main/CHANGELOG.md)
- [Xamarin 소프트웨어 개발 키트 9.0.0](https://github.com/braze-inc/braze-xamarin-sdk/blob/master/CHANGELOG.md)
    - [Braze Android 소프트웨어 개발 키트 37.0.0](https://github.com/braze-inc/braze-android-sdk/compare/v37.0.0...v41.0.0#diff-06572a96a58dc510037d5efa622f9bec8519bc1beab13c9f251e97e657a9d4ed)에서 [41.0.0으로](https://github.com/braze-inc/braze-android-sdk/compare/v37.0.0...v41.0.0#diff-06572a96a58dc510037d5efa622f9bec8519bc1beab13c9f251e97e657a9d4ed) Android 바인딩을 업데이트했습니다.
    - iOS 바인딩을 [Braze Swift 소프트웨어 개발 키트 13.3.0](https://github.com/braze-inc/braze-swift-sdk/compare/13.3.0...14.0.1#diff-06572a96a58dc510037d5efa622f9bec8519bc1beab13c9f251e97e657a9d4ed)에서 [14.0.1로](https://github.com/braze-inc/braze-swift-sdk/compare/13.3.0...14.0.1#diff-06572a96a58dc510037d5efa622f9bec8519bc1beab13c9f251e97e657a9d4ed) 업데이트했습니다.
    - Braze Android 소프트웨어 개발 키트에 필요한 새로운 전이적 NuGet 종속성을 추가했습니다:
        - Xamarin.AndroidX.DataStore.Preferences (1.1.7.1)
        - Xamarin.KotlinX.Serialization.Json.Jvm (1.9.0.2)
        - Xamarin.Kotlin.StdLib 버전이 2.0.21.3에서 2.3.0.1로 업데이트되었습니다. 프로젝트에서 이 패키지를 명시적으로 이전 버전으로 고정해 놓은 경우, 복원 오류를 방지하려면 업데이트해야 합니다.
    - 뉴스피드 기능을 제거했습니다.
        - 이 기능은 네이티브 Android 소프트웨어 개발 키트 버전 [38.0.0](https://github.com/braze-inc/braze-android-sdk/releases/tag/v38.0.0)에서 제거되었습니다.
        - 이 기능은 네이티브 Swift 소프트웨어 개발 키트 버전 [14.0.0](https://github.com/braze-inc/braze-swift-sdk/releases/tag/14.0.0)에서 제거되었습니다.
    - 열거형 BRZInAppMessageDismissalReason.BRZInAppMessageDismissalReasonWipeData케이스의 이름이 로 변경되었습니다BRZInAppMessageDismissalReason.WipeData.
- [엑스포 플러그인 4.0.0](https://github.com/braze-inc/braze-expo-plugin/releases/tag/4.0.0)
    - 이 버전은 Braze React Native 소프트웨어 개발 키트 19.0.0이 필요합니다.
    - (Android) 데이터 지속성 계층의 메모리 누수 문제를 수정했습니다.
    - (Android) 종료된 상태에서 앱이 실행될 때 푸시 알림 딥링크를 처리하기 위해 Braze.getInitialPushPayload() 지원이 추가되었습니다. 앱이 콜드 스타트될 때 푸시 알림의 딥링크가 Android에서 처리되지 않던 문제를 해결합니다.
- [React Native 소프트웨어 개발 키트 19.0.0](https://github.com/braze-inc/braze-react-native-sdk/releases/tag/19.0.0)
    - Braze Swift 소프트웨어 개발 키트의 네이티브 Swift 소프트웨어 개발 키트 버전 바인딩을 13.3.0에서 14.0.1로 업데이트합니다.
    - Braze Android SDK 40.0.2에서 41.0.0으로 네이티브 Android 소프트웨어 개발 키트 버전 바인딩을 업데이트합니다.
