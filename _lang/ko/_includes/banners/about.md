# Banners

> 배너를 사용하면 사용자에게 개인화된 메시징을 생성할 수 있으며, 이메일이나 푸시 알림과 같은 다른 채널의 도달 범위를 확장할 수 있습니다. 배너를 앱이나 웹사이트에 직접 삽입할 수 있어 사용자가 자연스럽게 느끼는 경험을 통해 참여할 수 있습니다.

![기기에서 렌더링된 배너의 예입니다.]({% image_buster /assets/img/banners/sample_banner.png %})

## 필수 조건

배너의 가용성은 귀하의 Braze 패키지에 따라 다릅니다. 시작하려면 계정 매니저 또는 고객 성공 매니저에게 문의하십시오.

## 배너를 사용하는 이유는 무엇입니까?

배너는 마케팅 및 제품 팀이 앱 또는 웹사이트 콘텐츠를 동적으로 개인화하여 실시간 사용자 적격성 및 행동을 반영할 수 있도록 합니다. 배너는 지속적으로 메시지를 인라인으로 표시하여 비침해적이고 맥락에 맞는 관련 경험을 제공하며, 각 사용자 세션 시작 시 자동으로 업데이트됩니다.

배너가 앱 또는 웹사이트에 통합된 후, 마케터는 간단한 드래그 앤 드롭 편집기를 사용하여 배너를 디자인하고 시작할 수 있으며, 지속적인 개발자 지원이 필요 없고 복잡성을 줄이며 효율성을 향상시킵니다.

| Use case | 설명 |
| --- | --- |
| 공지 | 다가오는 이벤트나 정책 변경과 같은 공지를 앱 경험의 최전선에 유지하십시오. |
| 제안 개인화하기 | 각 사용자의 탐색 기록, 장바구니 내용, 구독 등급 및 로열티 상태에 따라 개인화된 프로모션 및 인센티브를 표시하십시오. |
| 신규 사용자 참여 타겟팅 | 신규 사용자를 온보딩 흐름 및 계정 설정을 통해 안내하십시오. |
| 영업 및 프로모션 | 특집 콘텐츠, 인기 제품 및 진행 중인 브랜드 캠페인을 사용자 경험을 방해하지 않고 홈페이지에 지속적이고 직접적으로 강조하십시오. |

## 피처

배너의 기능은 다음과 같습니다:

- **쉬운 콘텐츠 구축:** 이미지, 텍스트, 버튼, 이메일 캡처 양식, 커스텀 코드 등을 지원하는 시각적 드래그 앤 드롭 편집기를 사용하여 배너를 생성하고 미리 볼 수 있습니다.
- **유연한 배치:** 배너가 나타날 수 있는 애플리케이션 또는 웹사이트 내 여러 위치를 정의하여 특정 맥락이나 사용자 경험에 대한 정밀한 타겟팅을 가능하게 합니다.
- **동적 개인화:** 배너는 새로운 사용자 세션마다 동적으로 새로고침되어 콘텐츠가 최신 상태로 유지되고 Braze의 내장 개인화 도구와 Liquid 로직을 사용하여 개인화됩니다.
- **네이티브 우선순위:** 여러 배너가 동일한 배치 대상을 목표로 할 때 표시 우선순위를 설정하여 올바른 메시지가 적시에 사용자에게 전달되도록 합니다.
- **커스텀 HTML 지원:** 고급 사용자 정의 또는 기존 웹 스타일과의 원활한 통합을 위해 커스텀 HTML 블록을 통합합니다.

## 배너에 대하여 {#about-banners}

### 배치 ID {#placement-id}

배너 배치는 [Braze SDK]({{site.baseurl}}/developer_guide/banners/placements/)로 생성한 앱 또는 웹사이트의 특정 위치로, 배너가 나타날 수 있는 위치를 지정합니다.

일반적인 위치에는 홈페이지 상단, 제품 상세 페이지 및 체크아웃 흐름이 포함됩니다. 배치가 생성된 후, 배너는 [배너 캠페인에서 할당될 수 있습니다.]({{site.baseurl}}/user_guide/message_building_by_channel/banners/create/)

작업 공간당 생성할 수 있는 배치 수에 대한 고정 제한은 없으며, 경험에 따라 필요한 만큼의 배치 ID를 생성할 수 있습니다. 각 배치는 작업 공간 내에서 고유해야 합니다. 단일 배치 ID는 동시에 최대 10개의 활성 캠페인에서 참조될 수 있습니다.

{% alert important %}
배너 캠페인을 시작한 후 배치 ID를 수정하지 마십시오.
{% endalert %}

### 배너 우선순위 {#priority}

여러 배너 메시지가 동일한 배치 ID를 참조할 때, 배너는 우선순위에 따라 표시됩니다: 높음, 중간 또는 낮음. 기본적으로 배너는 중간으로 설정되지만, 배너 캠페인을 생성하거나 편집할 때 [우선순위를 수동으로 설정할 수 있습니다.]({{site.baseurl}}/user_guide/message_building_by_channel/banners/create/#set-priority) 

여러 배너가 동일한 우선순위로 설정된 경우, 사용자가 자격이 있는 최신 배너가 먼저 표시됩니다.

### 메시지 전달

배너 메시지는 HTML 콘텐츠로 앱 또는 웹사이트에 전달되며, 일반적으로 iframe 내에서 렌더링됩니다. 이것은 배너가 장치 간에 일관되게 렌더링되도록 보장하며, 스타일과 스크립트를 나머지 코드와 분리하는 데 도움이 됩니다.

Iframes는 코드베이스의 변경 없이 동적이고 개인화된 콘텐츠 업데이트를 허용합니다. 각 iframe은 캠페인 타겟팅 및 개인화 로직을 사용하여 각 사용자 세션의 HTML을 검색하고 표시합니다.

### 크기 및 크기 조정

배너 크기 및 크기 조정에 대해 알아야 할 사항은 다음과 같습니다:

- 컴포저를 사용하면 배너를 다양한 크기로 미리 볼 수 있지만, 해당 정보는 SDK에 저장되거나 전송되지 않습니다.
- HTML은 렌더링되는 컨테이너의 전체 너비를 차지합니다.
- 고정된 치수 요소를 만들고 컴포저에서 해당 치수를 테스트하는 것이 좋습니다.

## Limitations

각 작업 공간은 최대 200개의 활성 배너 캠페인을 지원할 수 있습니다. 이 한도에 도달하면 새 캠페인을 만들기 전에 기존 캠페인을 [아카이브하거나 비활성화]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/about_statuses/#changing-the-status)해야 합니다.

또한, 배너 메시지는 다음 기능을 지원하지 않습니다:

- API 트리거 및 액션 기반 캠페인
- 연결된 콘텐츠
- 프로모션 코드
- 사용자 제어 해제
- `catalog_items` [`:rerender` 태그]({{site.baseurl}}/user_guide/data/activation/catalogs/using_catalogs/#using-liquid) 사용하기
