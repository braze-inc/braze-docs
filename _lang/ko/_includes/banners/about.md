# Banners

> 배너를 사용하면 이메일이나 푸시 알림과 같은 다른 채널의 도달 범위를 확장하면서 사용자를 위한 개인화된 메시지를 만들 수 있습니다. 배너를 앱이나 웹사이트에 직접 삽입할 수 있어 사용자가 자연스럽게 느끼는 경험을 통해 참여할 수 있습니다.

![기기에서 렌더링된 배너 예시]({% image_buster /assets/img/banners/sample_banner.png %})

## 필수 조건

배너 사용 가능 여부는 Braze 패키지에 따라 다릅니다. 시작하려면 계정 관리자 또는 고객 성공 관리자에게 문의하세요.

## 배너를 사용하는 이유는 무엇인가요?

마케팅 및 제품 팀은 배너를 통해 실시간 사용자 자격 및 행동을 반영하여 앱 또는 웹사이트 콘텐츠를 동적으로 개인화할 수 있습니다. 인라인으로 메시지를 지속적으로 표시하여 각 사용자 세션이 시작될 때 자동으로 업데이트되는 방해가 되지 않는 상황별 관련성 높은 경험을 제공합니다.

배너가 앱이나 웹사이트에 통합된 후 마케터는 간단한 드래그 앤 드롭 편집기를 사용하여 배너를 디자인하고 실행할 수 있으므로 지속적인 개발자 지원이 필요하지 않고 복잡성이 줄어들며 효율성이 향상됩니다.

| Use case | 설명 |
| --- | --- |
| 공지 사항 | 예정된 이벤트나 정책 변경과 같은 공지를 앱 경험의 최전방에 배치하세요. |
| 개인화된 오퍼 제공 | 각 사용자의 검색 기록, 장바구니 콘텐츠, 구독 등급, 로열티 상태에 따라 개인화된 프로모션과 인센티브를 표시하세요. |
| 신규 사용자 참여 타겟팅하기 | 신규 사용자에게 온보딩 흐름과 계정 설정을 안내합니다. |
| 영업 및 프로모션 | 사용자 경험을 방해하지 않고 홈페이지에서 기능별 콘텐츠, 인기 제품, 진행 중인 브랜드 캠페인을 지속적으로 직접 강조할 수 있습니다. |

## 피처

배너의 기능은 다음과 같습니다:

- **간편한 콘텐츠 구축:** 이미지, 텍스트, 버튼, 이메일 캡처 양식, 커스텀 코드 등을 지원하는 시각적인 드래그 앤 드롭 편집기를 사용하여 배너를 만들고 미리 볼 수 있습니다.
- **유연한 배치:** 애플리케이션 또는 웹사이트 내에서 배너가 표시될 수 있는 여러 위치를 정의하여 특정 컨텍스트 또는 사용자 경험에 대한 정확한 타겟팅을 인에이블먼트할 수 있습니다.
- **동적 개인화:** 배너는 새로운 사용자 세션이 시작될 때마다 동적으로 새로고침되므로, Braze의 구축된 개인화 도구와 Liquid 로직을 사용하여 콘텐츠를 최신 상태로 유지하고 개인화할 수 있습니다.
- **기본 우선순위 지정:** 여러 배너가 동일한 위치를 타겟팅하는 경우 표시 우선순위를 설정하여 적절한 메시지가 적시에 사용자에게 전달되도록 할 수 있습니다.
- **커스텀 HTML 지원:** 커스텀 HTML 블록을 통합하여 고급 사용자 정의 또는 기존 웹 스타일과 원활하게 통합할 수 있습니다.

## 배너 정보 {#about-banners}

### 배치 ID {#placement-id}

배너 배치는 [Braze 소프트웨어 개발 키트로 만든]({{site.baseurl}}/developer_guide/banners/placements/) 앱이나 웹사이트의 특정 위치로, 배너가 표시될 수 있는 위치를 지정합니다.

일반적인 위치로는 홈페이지 상단, 제품 상세 페이지, 결제 흐름 등이 있습니다. 게재 위치가 생성되면 배너 [캠페인에서 배너를 할당할]({{site.baseurl}}/user_guide/message_building_by_channel/banners/creating_campaigns/) 수 있습니다.

워크스페이스당 만들 수 있는 배치 수에는 정해진 제한이 없으며, 경험에 필요한 만큼의 배치 ID를 만들 수 있습니다. 각 배치는 워크스페이스 내에서 고유해야 합니다. 하나의 게재 위치 ID는 최대 10개의 활성 캠페인에서 동시에 참조할 수 있습니다.

{% alert important %}
배너 캠페인을 시작한 후에는 게재 위치 ID를 수정하지 마세요.
{% endalert %}

### 배너 우선 순위 {#priority}

여러 캠페인이 동일한 게재 위치 ID를 참조하는 경우 배너는 우선순위에 따라 높음, 중간, 낮음 순으로 표시됩니다. 새로 만든 배너는 기본값이 중간으로 설정되어 있지만 배너 캠페인을 만들거나 편집할 때 [우선순위를 수동으로 설정할]({{site.baseurl}}/user_guide/message_building_by_channel/banners/creating_campaigns/#set-priority) 수 있습니다. 

여러 개의 배너가 동일한 우선순위로 설정되어 있는 경우 사용자가 받을 자격이 있는 최신 배너가 먼저 표시됩니다.

### 메시지 전달

배너 메시지는 HTML 콘텐츠로 앱이나 웹사이트에 전달되며, 일반적으로 iframe 내에서 렌더링됩니다. 이렇게 하면 배너가 여러 기기에서 일관되게 렌더링되고 배너의 스타일과 스크립트를 나머지 코드와 분리하여 유지할 수 있습니다.

아이프레임을 사용하면 코드베이스를 변경할 필요 없이 동적 콘텐츠 업데이트를 개인화할 수 있습니다. 각 iframe은 캠페인 타겟팅 및 개인화 로직을 사용하여 각 사용자 세션에 대한 HTML을 검색하고 표시합니다.

### 크기 및 크기 조정

배너 크기 및 크기 조정에 대해 알아야 할 사항은 다음과 같습니다:

- 컴포저를 사용하면 배너를 다양한 크기로 미리 볼 수 있지만, 해당 정보는 SDK에 저장되거나 전송되지 않습니다.
- HTML은 렌더링되는 컨테이너의 전체 너비를 차지합니다.
- 고정된 치수 요소를 만들고 컴포저에서 해당 치수를 테스트하는 것이 좋습니다.

## Limitations

각 워크스페이스는 최대 200개의 활성 배너 캠페인을 지원할 수 있습니다. 이 제한에 도달하면 새 캠페인을 만들기 전에 기존 캠페인을 [보관하거나 비활성화해야]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/about_statuses/#changing-the-status) 합니다.

또한 배너 메시지는 다음 기능을 지원하지 않습니다:

- 캔버스 통합
- API 트리거 및 액션 기반 캠페인
- 연결된 콘텐츠
- 프로모션 코드
- 사용자 제어 해고
- `catalog_items` [`:rerender` 태그]({{site.baseurl}}/user_guide/data/activation/catalogs/using_catalogs/#using-liquid) 사용
