### 팀에 사용자 권한을 추가할 때 고려 사항

Braze 대시보드에서 권한을 저장하려고 할 때, 특히 작업 공간에서 사용자를 추가하거나 제거하거나 팀에 추가할 때 어려움이 발생할 수 있습니다. 사용자의 권한이 작업 공간 수준에서 이미 가지고 있는 권한과 동일한 경우 **저장/업데이트 사용자** 버튼이 회색으로 변할 수 있습니다. 모든 사용자가 전체 작업 공간과 동일한 권한을 가지고 있다면 팀을 가질 필요가 없기 때문에 이 제한이 존재합니다.

동일한 권한을 유지하면서 사용자를 팀에 성공적으로 추가하려면 작업 공간 수준에서 권한을 할당하지 마십시오. 대신 팀 수준에서만 권한을 할당하십시오.

## 제한된 사용자

제한된 사용자는 회사 관리자 및 작업 공간 관리자에 비해 제한이 있는 상태에서 Braze 대시보드의 특정 측면을 관리할 수 있는 특정 권한을 가지고 있습니다.

| 범위 | 설명 |
| --- | --- |
| 권한 | 제한된 사용자는 "대시보드 사용자 보기" 및 "대시보드 사용자 편집" 권한이 있는 경우 다른 제한된 사용자의 권한을 편집할 수 있습니다. 그들은 또한 새로운 제한된 사용자를 생성하고 그들의 권한 세트를 수정할 수 있습니다. 그러나 그들은 회사 관리자 계정을 생성하거나 관리할 수 없습니다. |
| 역할 제한 | 제한된 사용자가 "작업 공간 관리자"를 제외한 모든 권한을 가지고 있다면, 그들은 여전히 작업 공간 관리자에게 일반적으로 부여되는 모든 다른 권한에 접근할 수 있습니다. |
| 권한의 가시성 | 제한된 사용자가 하나의 작업 공간(예: Dev)에 대해 "대시보드 사용자 보기" 및 "대시보드 사용자 편집" 권한을 가지고 있지만 다른 작업 공간(예: Prod)에 대해서는 그렇지 않은 경우, 그들은 대시보드 사용자 세부 정보 페이지에서 Prod 작업 공간 권한을 보지 못합니다. |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

### 제한된 사용자 비교

| 제한된 사용자 유형 | 설명 |
| --- | --- |
| 작업 공간 관리자 | 작업 공간 관리자는 작업 공간 관리에 특정한 권한을 가지고 있지만 회사 관리자와 동일한 권한은 없습니다. 제한된 사용자는 필요한 권한이 체크되어 있다면 작업 공간 관리자의 권한과 유사한 권한을 상속받을 수 있습니다. |
| 관리자 (회사 관리자) | 회사 관리자는 대시보드 사용자를 삭제할 수 있는 능력을 포함하여 더 넓은 권한을 가지고 있습니다. 그러나 그들은 자신의 계정을 삭제할 수 없으며, 그 작업을 위해 다른 회사 관리자에게 연락해야 합니다. |
| 보기 전용 접근 | 캠페인 페이지와 같은 대시보드의 일부에 접근하려면 사용자는 그들에게 할당된 보기 권한이 있어야 합니다.
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

### 제한된 접근 오류

사용자는 "이 페이지에 접근하려면 '랜딩 페이지 보기' 권한이 필요합니다"와 같은 메시지를 받을 수 있습니다. 이런 경우, 사용자와 계정 관리자는 필요한 권한이 부여되었는지 확인해야 합니다. 그렇다면 사용자의 권한을 비활성화한 다음 다시 활성화하여 문제를 해결해 보십시오. 

{% alert note %}
한 대시보드 사용자에서 다른 대시보드 사용자로 사용자 권한을 병합하거나 가져오는 것은 불가능합니다.
{% endalert %}
