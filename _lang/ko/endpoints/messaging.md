guide_top_header: "메시징 엔드포인트"
guide_top_text: "Braze 메시징 API는 사용자에게 메시지를 보낼 수 있는 두 가지 옵션을 제공합니다. API 요청에 메시지 내용 및 설정을 <code class='highlighter-rouge'>//messages/send</code> 및 `/messages/schedule` 엔드포인트로 제공할 수 있습니다. 또는 Braze 대시보드에서 API 트리거 캠페인으로 메시지의 세부 정보를 관리하고 `/campaigns/trigger/send` 및`/campaigns/trigger/schedule` 엔드포인트를 사용하여 언제, 누구에게 전송되는지 제어할 수 있습니다. 다음 섹션에서는 두 가지 방법의 요청 사양에 대해 자세히 설명합니다. <br> <br> 다른 캠페인과 마찬가지로, 특정 사용자가 메시징 API 캠페인을 수신할 수 있는 횟수를 제한하려면 Braze 대시보드에서 [재자격 설정]({{site.baseurl}}/user_guide/참여_도구/캠페인/빌딩_캠페인/delivery_types/api_triggered_delivery/#re-eligibility-with-API-triggered-campaigns)을 구성하여 특정 사용자가 메시징 캠페인을 받을 수 있는 횟수를 제한할 수 있습니다. Braze는 API 요청 횟수에 관계없이 캠페인에 다시 참여하지 않은 사용자에게는 API 메시지를 전달하지 않습니다. <br> <br> 메시지 보내기 엔드포인트를 사용하면 지정된 사용자에게 즉시 메시지를 보낼 수 있습니다. 세그먼트를 타겟팅하는 경우, 요청 기록이 **메시지 활동 로그**에 저장됩니다. 메시지 예약 엔드포인트를 사용하여 지정된 시간에 메시지를 보내고, 이미 예약한 메시지를 수정하거나 취소할 수 있습니다."
