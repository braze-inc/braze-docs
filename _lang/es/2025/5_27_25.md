# Lanzamiento el 27 de mayo de 2025.

## Flexibilidad de los datos

### Copiar lienzos entre espacios de trabajo

{% multi_lang_include release_type.md release="General availability" %}

Ahora puedes copiar lienzos entre espacios de trabajo. Esto te permite comenzar rápidamente a redactar tu mensaje utilizando una copia de un Canvas de otro espacio de trabajo. Para obtener más información sobre lo que se copia, consulta [Copiar campañas y lienzos entre espacios de trabajo]({{site.baseurl}}/copying_to_workspaces/).

### Reglas de mensajería para el flujo de trabajo de aprobación 

{% multi_lang_include release_type.md release="General availability" %}

Utiliza [reglas de mensajería]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/approvals/messaging_rules) en tu flujo de trabajo de aprobación para limitar el número de usuarios a los que puedes llegar antes de que se requiera una aprobación adicional. De esta forma, podrás revisar tus campañas y lienzos antes de dirigirte a una audiencia más amplia.

### Diagramas de relaciones entre entidades para Snowflake y Braze

A principios de este año, creamos tablas de relaciones entre entidades para los datos compartidos entre Snowflake y Braze. Este mes, hemos añadido [nuevos diagramas interactivos]({{site.baseurl}}/partners/data_and_analytics/data_warehouses/snowflake/entity_relationships/) en los que puedes desplazarte, seleccionar y ampliar los detalles de cada tabla, lo que te permite hacerte una idea más clara de cómo interactúan tus datos con Braze.

## Desbloquear la creatividad

### Eventos recomendados

{% multi_lang_include release_type.md release="Early access" %}

[Los eventos recomendados]({{site.baseurl}}/user_guide/data/custom_data/recommended_events) están mapeados con los casos de uso más comunes del comercio electrónico. Al utilizar los eventos recomendados, puedes desbloquear plantillas de Canvas prediseñadas, paneles de informes que están mapeados con el ciclo de vida del cliente y mucho más.

## Canales robustos

### Canal de banners

{% multi_lang_include release_type.md release="General availability" %}

Con [Banners]({{site.baseurl}}/user_guide/message_building_by_channel/banners), puedes crear mensajes de mensajería personalizados para tus usuarios, al tiempo que amplías el alcance de tus otros canales, como el correo electrónico o las notificaciones push. Puedes integrar banners directamente en tu aplicación o sitio web, lo que te permite interactuar con los usuarios a través de una experiencia que resulta natural.

### Canal de servicios de comunicación enriquecidos (RCS)

{% multi_lang_include release_type.md release="General availability" %}

[Los servicios de comunicación enriquecidos (RCS)]({{site.baseurl}}/about_rcs/) mejoran los SMS tradicionales al habilitar a las marcas para que entreguen mensajes que no solo son informativos, sino también mucho más interactivos. Ahora compatible tanto con Android como con iOS, RCS ofrece características como medios de alta calidad, botones interactivos y perfiles de remitentes con marca directamente en las aplicaciones de mensajería preinstaladas de los usuarios, lo que elimina la necesidad de descargar una aplicación independiente.

### Página de configuración de push

{% multi_lang_include release_type.md release="General availability" %}

Utiliza la [página **Configuración de notificaciones push**]({{site.baseurl}}/user_guide/administrative/app_settings/push_settings) para configurar la configuración clave de tus notificaciones push, incluyendo el TTL para la notificación push y la prioridad FCM predeterminada para las campañas de Android. Esta configuración ayuda a optimizar la entrega y la eficacia de tus notificaciones push, lo que garantiza una mejor experiencia para tus usuarios.

### Códigos promocionales para campañas de mensajes dentro de la aplicación

{% multi_lang_include release_type.md release="Early access" %}

Puedes utilizar [códigos promocionales]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/promotion_codes) en campañas de mensajes dentro de la aplicación insertando un [fragmento de código promocional]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/promotion_codes#creating-a-promotion-code-list) en el cuerpo del mensaje de tu campaña de mensajes dentro de la aplicación.

### Gestión de errores de webhook y límite de velocidad

[Acerca de los webhooks]({{site.baseurl}}/user_guide/message_building_by_channel/webhooks/understanding_webhooks/#webhook-error-handling-and-rate-limiting) tiene una nueva sección que describe cómo Braze gestiona los errores de webhook y el límite de velocidad.

### Configuración regional de los mensajes dentro de la aplicación

Después de [añadir configuraciones regionales]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/using_locales) a tu espacio de trabajo, puedes dirigirte a usuarios en diferentes idiomas, todo ello en un único mensaje dentro de la aplicación.

### Amazon SES como proveedor de envío de correo electrónico (ESP)

Ahora puedes utilizar Amazon SES como ESP, de forma similar a como utilizarías SendGrid y SparkPost. Consulta [SSL en Braze]({{site.baseurl}}/user_guide/message_building_by_channel/email/email_setup/ssl#what-is-a-cdn-and-why-do-i-need-it) y [Enlaces universales y enlaces a aplicaciones]({{site.baseurl}}/user_guide/message_building_by_channel/email/universal_links#turning-off-click-tracking-on-a-link-to-link-basis) para conocer los matices de la configuración de SSL y el seguimiento de clics en cada enlace.

## Nuevas asociaciones Braze

### Ojo de águila - Fidelización

La integración bidireccional entre Braze y [Eagle Eye]({{site.baseurl}}/partners/eagle_eye/) te permite activar datos promocionales y de fidelización directamente en Braze, lo que permite a los especialistas en marketing personalizar la interacción con los clientes utilizando datos en tiempo real, como saldos de puntos, promociones y actividades de recompensa.

### Eppo - Pruebas A/B

La integración de Braze y [Eppo]({{site.baseurl}}/partners/eppo/) te permite configurar pruebas A/B en Braze y analizar los resultados en Eppo para descubrir información valiosa y vincular el rendimiento de los mensajes con métricas empresariales a largo plazo, como los ingresos o la retención.

### Mencioname - Referidos

Juntos, [Mention Me](https://www.mention-me.com/) y Braze pueden ser tu puerta de entrada para atraer clientes premium y fomentar una fidelización inquebrantable hacia tu marca. Al integrar fácilmente los datos de referencia propios en Braze, puedes entregar experiencias omnicanal altamente personalizadas dirigidas a los seguidores de tu marca. Para empezar, consulta Socios[ tecnológicos: Mencioname]({{site.baseurl}}/partners/mention_me).

### Shopify - Comercio electrónico

[Conecta varios dominios de tiendas Shopify]({{site.baseurl}}/shopify_connecting_multiple_stores/) a un único espacio de trabajo para obtener una visión global de tus clientes en todos los mercados. Crea y lanza programas y recorridos de automatización en un único espacio de trabajo sin duplicar esfuerzos en las tiendas regionales.

## Actualizaciones del SDK

Se han publicado las siguientes actualizaciones del SDK. Las actualizaciones de última hora se enumeran a continuación; todas las demás actualizaciones se pueden encontrar consultando los correspondientes registros de cambios del SDK.

- [SDK de Android 36.0.0](https://pub.dev/packages/braze_plugin/changelog)
    - Esta versión revierte el aumento de la versión mínima del SDK de Android del SDK de Braze para Android de API 21 a API 25 introducido en la versión 34.0.0. Esto permite que el SDK se compile de nuevo en aplicaciones compatibles con API 21. Ten en cuenta que, aunque estamos reintroduciendo la capacidad de compilar, no estamos reintroduciendo el soporte formal para<  API 25, y no podemos garantizar que el SDK funcione según lo previsto en dispositivos que ejecuten esas versiones.
    - Si tu aplicación es compatible con esas versiones, debes:
        - Valida que la integración de SDK funciona según lo previsto en dispositivos físicos (no solo emuladores) para esas versiones de la API.
        - Si no puedes validar el comportamiento esperado, debes llamar a [disableSDK](https://braze-inc.github.io/braze-android-sdk/kdoc/braze-android-sdk/com.braze/-braze/-companion/disable-sdk.html) o no inicializar el SDK en esas versiones. De lo contrario, podrías provocar efectos secundarios no deseados o un rendimiento inferior en los dispositivos de tus usuarios finales.
    - Se ha corregido un problema por el que los mensajes dentro de la aplicación provocaban una lectura en el hilo principal.
    `BrazeInAppMessageManager.displayInAppMessage` ahora es una función suspendida de Kotlin.
        - Si no llamas a esta función directamente, no es necesario que realices ningún cambio.
    - AndroidX Compose BOM actualizado a 2025.04.01 para gestionar las actualizaciones en las API de Jetpack Compose.
- [React Native SDK 15.0.0](https://github.com/braze-inc/braze-react-native-sdk/blob/master/CHANGELOG.md)
    - Actualiza el puente nativo de Android de Braze Android SDK 35.0.0 a 36.0.0.
    - Actualiza los enlaces de la versión nativa de iOS de Braze SWIFT SDK 11.9.0 a 12.0.0.
    - Actualiza la representación de la unidad dePushNotificationEvent.timestamp  a milisegundos en iOS.
        - Anteriormente, este valor se representaba en segundos en iOS. Ahora coincidirá con la implementación existente de Android.
- [SDK web 5.9.0](https://github.com/braze-inc/braze-web-sdk/blob/master/CHANGELOG.md)
