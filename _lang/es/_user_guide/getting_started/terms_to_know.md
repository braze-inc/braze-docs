glossaries:
  - name: Usuario activo
    description: "En Braze, un <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/campaigns/ideas_and_strategies/active_user_campaigns/\">usuario activo</a> se define como cualquier usuario que tenga una sesión en un periodo determinado."
  - name: Pruebas multivariantes A/B
    description: "Se trata de un tipo de prueba en la que se modifican una o varias variables y se comprueba qué cambios rinden mejor. <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/testing/multivariant_testing\">Las pruebas A/B</a> se pueden realizar en las campañas del panel."
  - name: Instancias de la aplicación
    description: Las instancias de la aplicación se refieren a los diferentes sitios y aplicaciones que se reúnen en un espacio de trabajo.
  - name: Alloys
    description: "Alloys son nuestros <a href=\"https://www.braze.com/docs/partners/home/\">socios tecnológicos</a>."
  - name: Usuarios anónimos
    description: "Cuando se reconoce un perfil de usuario a través del SDK, se crea un perfil de usuario anónimo con el <a href=\"https://www.braze.com/docs/api/basics/#user-ids\">ID de usuario Braze</a> asociado." 
  - name: Interfaz para la programación de aplicaciones (API)
    description: "<a href=\"https://www.braze.com/docs/api/basics/#api-overview\">La API de Braze</a> proporciona un servicio Web en el que puedes registrar las acciones realizadas por tus usuarios directamente a través de HTTP, en lugar de a través de los SDK para móviles. Esto te permite, por ejemplo, pasar datos de usuario a Braze que no son objeto de seguimiento dentro de tu aplicación o sitio web."
  - name: Braze (El producto)
    description: "A veces denominado panel, este producto controla todos los datos e interacciones en el corazón de la plataforma Braze. Los clientes de Braze la utilizan para administrar notificaciones, configurar campañas de mensajería dirigidas y ver análisis. Los desarrolladores la utilizan para administrar configuraciones para integrar aplicaciones, como claves de API y credenciales de notificación push."
  - name: Herramientas de interacción Braze
    description: "Braze ofrece muchas <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/\">herramientas de interacción</a> para llegar a tus clientes y usuarios con nuestras campañas, herramientas Canvas, y puedes optimizar la coherencia (y subir imágenes y otros contenidos) utilizando nuestras herramientas de plantillas y medios. A partir de ahí, puedes crear segmentos y geovallas para dirigirte a tu audiencia por ubicación u otros atributos."
  - name: Equipos Braze
    description: "Los administradores de Braze pueden dividir un subconjunto de usuarios del panel en <a href=\"https://www.braze.com/docs/user_guide/administrative/app_settings/manage_your_braze_users/teams/\">Equipos</a> con distintos roles y permisos de usuario. Esto permite a los administradores de Braze limitar el acceso a determinadas características por pertenencia a un grupo."
  - name: Campañas
    description: "Las campañas son métodos de mensajería personalizables para entregar una respuesta personalizada a tus clientes. Puedes <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/campaigns/\">crear campañas</a> utilizando diferentes canales de mensajería para enviar tus mensajes únicos." 
  - name: Canvas
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/canvas/\">Canvas</a> es una única interfaz unificada en la que los especialistas en marketing pueden configurar campañas con múltiples mensajes y pasos para formar un viaje cohesivo. Canvas te permite comparar y optimizar esas experiencias utilizando análisis exhaustivos para la experiencia completa del usuario."
  - name: Contenido conectado
    description: "<a href=\"https://www.braze.com/docs/user_guide/personalization_and_dynamic_content/connected_content/\">El contenido conectado</a> amplía la personalización del marketing para impulsar la interacción con los clientes y las conversiones. Puedes insertar cualquier información accesible mediante la API directamente en los mensajes que envíes a los usuarios. El Contenido Conectado permite extraer contenido directamente de tu servidor Web o de API de acceso público."
  - name: Tarjetas de contenido
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/content_cards/about/\">Las tarjetas de contenido</a> te permiten enviar a tus clientes un flujo dinámico y muy específico de contenido enriquecido directamente desde las aplicaciones que les gustan, sin interrumpir su experiencia. Las tarjetas de contenido pueden enviarse a usuarios de iOS, Android y Web."
  - name: Currents
    description: "<a href = \"https://www.braze.com/docs/user_guide/data/braze_currents/\">Currents</a>, nuestra exportación de transmisión de datos, está incluida en algunos paquetes Braze. Braze Currents te permite integrarte a través del Almacenamiento de Datos utilizando archivos planos o a nuestros socios de Análisis del Comportamiento y Datos del Cliente utilizando una carga útil JSON por lotes a un punto final designado."
  - name: Atributos personalizados
    description: "<a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_attributes/\">Los atributos personalizados</a> son una colección de rasgos únicos de tus usuarios. Son ideales para almacenar atributos sobre tus usuarios, o información sobre acciones de poco valor dentro de tu aplicación. Puedes asignar atributos personalizados a los usuarios dentro del panel. Puedes filtrar y segmentar a tus usuarios en función de estos atributos, tanto para campañas <a href=\"https://www.braze.com/docs/developer_guide/platforms/swift/analytics/setting_custom_attributes/\">en iOS</a> como en <a href=\"https://www.braze.com/docs/developer_guide/platforms/android/analytics/setting_custom_attributes/\">Android</a>."
  - name: Eventos personalizados
    description: "<a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_events\">Los eventos personalizados</a> son acciones que realizan tus usuarios; son los más adecuados para hacer un seguimiento de las interacciones de alto valor de los usuarios con tu aplicación."
  - name: Punto de datos
    description: "Se contabiliza un punto de datos cuando se establece o actualiza un <a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_attributes/\">atributo personalizado</a> (aunque lo estés actualizando con el mismo valor), se registra un <a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_events\">evento personalizado</a> o un evento de compra, se registra cualquier dato estándar (por ejemplo, <code>correo electrónico</code>, <code>nombre</code>, <code>apellidos</code>, <code>país</code> o <code>ciudad_de_origen</code>), cuando se inicia una sesión y cuando finaliza una sesión."
  - name: Vinculación en profundidad
    description: "<a href=\"https://www.braze.com/docs/user_guide/personalization_and_dynamic_content/deep_linking_to_in-app_content/\">Los vínculos profundos</a> se utilizan para dirigir a los clientes a su próxima acción o interacción. Mediante los vínculos en profundidad, puedes conectar un mensaje con un contenido específico dentro de un sitio web o una aplicación móvil."
  - name: Usuarios inactivos
    description: "Se considera que un usuario <a href=\"https://www.braze.com/docs/user_archival/#dormant-users\">está inactivo</a> cuando no ha tenido actividad en los últimos doce meses."
  - name: Punto final
    description: "En la API de mensajería Braze se utiliza un canal de mensajería de extremo a <a href=\"https://www.braze.com/docs/api/endpoints/\">extremo</a>, también conocido como <a href=\"https://www.braze.com/docs/api/endpoints/\">punto final de</a> API, para enviar y programar mensajes."
  - name: Eventos de excepción
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/canvas/create_a_canvas/exit_criteria/#exception-events\">Los eventos de excepción</a> se producen cuando un usuario ha alcanzado el objetivo deseado y no recibe un mensaje desencadenado. De este modo, el mensaje desencadenado sigue siendo relevante para el usuario."
  - name: Limitación de frecuencia
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/#frequency-capping\">La limitación de frecuencia</a> te permite gestionar la comunicación sin abrumar a tu audiencia. Es un límite automatizado de mensajes para evitar que los usuarios reciban demasiadas comunicaciones en poco tiempo."
  - name: HIPAA
    description: "HIPAA es el acrónimo de Ley de Portabilidad y Responsabilidad de los Seguros Sanitarios (Health Insurance Portability and Accountability Act). Braze <a href=\"https://www.braze.com/docs/developer_guide/disclosures/security_qualifications/#hipaa\">cumple la HIPAA</a>. Los requisitos de la HIPAA implican seguridad administrativa, física y técnica."
  - name: Mensajes dentro de la aplicación
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/in-app_messages/\">Los mensajes dentro de la aplicación</a> son mensajes móviles que aparecen dentro de tu aplicación. Te ayudan a hacer llegar contenido a tu usuario sin interrumpir su día con una notificación push. Los mensajes dentro de la aplicación, personalizados y adaptados, mejoran la experiencia del usuario y ayudan a tu audiencia a obtener el máximo valor de tu aplicación."
  - name: Usuarios inactivos
    description: "Se considera que un usuario <a href=\"https://www.braze.com/docs/user_archival/#inactive-users\">está inactivo</a> cuando cumple determinados criterios, como dejar de recibir correo electrónico o mensajes push y no haber estado activo durante más de seis meses."
  - name: Calentamiento de IP
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/email/email_setup/ip_warming\">El calentamiento de IP</a> es la práctica de aumentar gradualmente la cantidad de correo enviado desde una IP dedicada. Esto ayuda a establecer una reputación con los proveedores de servicios de Internet, minimizando la probabilidad de que tus mensajes sean marcados."
  - name: Pares clave-valor
    description: "Los <a href=\"http://www.braze.com/docs/user_guide/personalization_and_dynamic_content/key_value_pairs/\">pares clave-valor</a> son elementos de datos enlazados en los que la clave es un identificador único y el valor es el contenido. Pueden utilizarse para enviar cargas útiles de datos adicionales a los dispositivos de los usuarios."
  - name: Liquid
    description: "Liquid es un lenguaje de plantillas para clientes de uso común creado por Shopify y escrito en Ruby. <a href=\"https://www.braze.com/docs/user_guide/personalization_and_dynamic_content/liquid/\">Liquid</a> se utiliza para cargar y extraer contenido dinámico. Liquid te permite utilizar objetos, etiquetas y filtros para <a href=\"http://www.braze.com/docs/user_guide/personalization_and_dynamic_content/liquid/supported_personalization_tags/\">añadir personalización</a>."
  - name: Canal de mensajería
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/\">Los canales de mensajería</a> son formas de comunicarte virtualmente con tus clientes: mediante notificaciones push en su teléfono o navegador Web, correo electrónico, mensajes dentro de la aplicación, ¡y mucho más!"
  - name: Usuarios activos al mes (MAU)
    description: Se trata de usuarios que han iniciado sesión en los últimos 30 días.
  - name: Mensajería multicanal
    description: "Envío de mensajes a un usuario a través de varios medios, como una combinación de correo electrónico, web push y notificaciones push móviles. <a href=\"https://www.braze.com/docs/developer_guide/getting_started/platform_overview/#multichannel-messaging\">Los canales de mensajería</a> se utilizan mejor en conjunto y con regularidad para reactivar la interacción de los usuarios perdidos, retener a los usuarios activos y dinamizar a los embajadores de tu marca."
  - name: Pruebas multivariantes
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/testing/multivariant_testing/#multivariate-test\">Las pruebas multivariantes</a> te permiten probar múltiples variables comparando dos o más versiones de un mensaje para evaluar qué variante rinde mejor."
  - name: Nuevo usuario
    description: "Braze considera nuevo usuario a todo aquel que acaba de instalar tu aplicación. Alternativamente, un nuevo usuario también puede definirse como un usuario con un ID de usuario que no ha sido identificado previamente en Braze."
  - name: Personalización
    description: "Utilizar la tecnología para tener en cuenta las preferencias y tendencias individuales de cada usuario a la hora de comunicarse con él. <a href=\"https://www.braze.com/docs/user_guide/personalization_and_dynamic_content/\">La mensajería personalizada</a> ayuda a crear valiosas experiencias del cliente adaptándose a sus preferencias." 
  - name: Mensajes push
    description: "Un <a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/push/\">mensaje push</a>, o notificación push, es una notificación que aparece desde una aplicación móvil. Las notificaciones push suelen aparecer como diálogos emergentes y banners tanto para iOS como para Android."
  - name: Token de notificaciones push
    description: "Un token de notificaciones push es una clave única, creada y asignada por Apple o Google para crear una conexión entre una aplicación y un dispositivo iOS, Android o Web. La <a href=\"https://www.braze.com/docs/help/help_articles/push/push_token_migration/#push-token-migration\">migración de token de notificaciones push</a> consiste en importar a Braze esas claves ya generadas."
  - name: TTL para notificación push tiempo de vida
    description: "También conocido como <a href=\"https://www.braze.com/docs/user_guide/administrative/app_settings/push_settings/\">TTL para push</a>, el tiempo de vida se refiere al periodo durante el cual las campañas seguirán intentando entregarse a un usuario sin conexión."
  - name: Condición de carrera
    description: "Una <a href=\"http://www.braze.com/docs/user_guide/engagement_tools/testing/race_conditions/\">condición de carrera</a> es un concepto de ingeniería de software que describe una situación no deseada que se produce cuando un sistema intenta realizar varias operaciones simultáneamente, pero debido a la naturaleza del sistema, las operaciones deben realizarse en la secuencia correcta para que se realicen correctamente. <br><br>En la plataforma Braze, la segmentación de una campaña desencadenada en función de los datos de usuario registrados en el momento del evento puede provocar una condición de carrera. Esto ocurre cuando un cambio en el atributo del usuario sobre el que se segmenta la campaña aún no se ha procesado para el usuario en el momento en que se determina la pertenencia al segmento y se envía la campaña, y puede hacer que el usuario no reciba la campaña."
  - name: Limitación de tasa
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/\">El límite de velocidad</a> es el proceso de restringir el número de mensajes enviados durante un periodo de tiempo para evitar sobrecargar a tu audiencia."  
  - name: Etiquetas
    description: "<a href=\"https://www.braze.com/docs/user_guide/administrative/app_settings/tags/\">Las etiquetas</a> son una herramienta que te ayuda a clasificar, organizar y ordenar tu interacción en una o varias campañas."
  - name: Segmentación
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/segments/\">La segmentación</a> del panel te permite crear grupos o extensiones de usuarios basándote en potentes filtros de su comportamiento dentro de la aplicación, datos demográficos y mucho más."
  - name: Kit de desarrollo de software (SDK)
    description: "<a href=\"https://www.braze.com/docs/developer_guide/getting_started/sdk_overview/\">Los SDK</a> se integran en aplicaciones iOS y Android y proporcionan potentes herramientas de marketing, atención al cliente y análisis. Braze ofrece integración de SDK tanto para <a href=\"https://www.braze.com/docs/developer_guide/platforms/swift/sdk_integration/#integrating-the-sdk/\">iOS</a> como para <a href=\"https://www.braze.com/docs/developer_guide/platforms/android/sdk_integration/#step-1-configure-brazexml\">Android</a>."
  - name: Puesta de sol
    description: "La extinción se refiere al proceso de identificar a los usuarios desvinculados y cesar la mensajería activa a estos usuarios sin que tengan que realizar ninguna acción. Crear políticas de caducidad para tu <a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/email/best_practices/sunset_policies/\">correo electrónico</a> y mensajes <a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/push/best_practices/sunset_policies/#sunset-policies-for-push/\">push</a> puede ayudar a frenar el impacto en tus tasas de apertura." 
  - name: Archivado de usuarios
    description: "<a href=\"https://www.braze.com/docs/user_archival/\">El archivado de usuarios</a> se refiere a los usuarios que han sido archivados. En Braze, esto incluye tanto a los usuarios inactivos como a los inactivos."
  - name: Webhooks
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/webhooks/\">Los webhooks</a> te permiten desencadenar acciones fuera de la aplicación, como la entrega de mensajes de texto SMS. Puedes utilizar webhooks para proporcionar a otros sistemas y aplicaciones información en tiempo real. La flexibilidad de esta característica te permite enviar información a cualquier punto final."
