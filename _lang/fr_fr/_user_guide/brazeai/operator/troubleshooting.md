# Résolution des problèmes pour BrazeAI Operator

> Problèmes courants et solutions pour BrazeAI Operator<sup>TM</sup>.

## Problèmes courants

| Problème | Résolution |
| --- | --- |
| Pas de réponse | Actualisez la page et rouvrez le panneau Operator. |
| Réponses hors sujet | Reformulez la question de manière plus précise. Mentionnez la fonctionnalité ou le workflow concerné. |
| Messages d'erreur | Si Operator ne peut pas diffuser le contenu, une invite « Réessayer » peut apparaître. Operator peut être temporairement indisponible ou la connexion a été interrompue. Réessayez après quelques minutes. |
| Limite d'utilisation quotidienne atteinte | La limite d'utilisation à l'échelle de l'entreprise a été atteinte. Attendez que la limite se réinitialise (toutes les 24 heures). |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

## Limites

- **Accès aux données :** Operator peut voir le contenu de la page active. Il peut également consulter certaines données spécifiques à l'entreprise, comme les campagnes passées, les catalogues et les attributs personnalisés. Cependant, il ne peut actuellement pas générer de nouveaux rapports ni exécuter de requêtes de rapport.
- **Limites d'utilisation :** Operator est soumis à une limite d'utilisation quotidienne à l'échelle de l'entreprise qui se réinitialise toutes les 24 heures. Lorsque la limite est atteinte, une bannière « Limite d'utilisation quotidienne atteinte » apparaît dans le chat et aucune nouvelle requête n'est possible jusqu'à la réinitialisation.
