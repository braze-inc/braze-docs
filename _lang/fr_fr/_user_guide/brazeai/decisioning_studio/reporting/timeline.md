# Rapport Chronologie

> Le rapport Chronologie fournit un historique visuel des événements clés aux côtés de vos indicateurs de performance. Ces événements incluent les exécutions d'agents, les modifications de configuration, les mises à jour des garde-fous, et bien plus encore.

Les annotations apparaissent directement sur les graphiques d'uplift et dans un onglet dédié à la chronologie, vous offrant un contexte immédiat pour comprendre les variations de résultats sans avoir à rechercher les modifications historiques.

![Rapport Chronologie affichant un graphique avec des indicateurs de performance au fil du temps. Les événements clés tels que les exécutions d'agents, les modifications de configuration et les mises à jour des garde-fous sont représentés par des icônes le long de la chronologie. Sous le graphique, un tableau répertorie les événements avec des colonnes pour la date, le type, le libellé, les détails et la visibilité dans les graphiques.]({% image_buster /assets/img/decisioning_studio/reporting_timeline.png %})

Pour comparer les performances entre deux groupes, utilisez les menus déroulants pour sélectionner les critères de comparaison souhaités. Consultez le tableau suivant pour plus de détails :

| Champ | Description |
|-------|-------------|
| Date | Date à laquelle l'événement s'est produit. |
| Type | Catégorie de l'événement, comme une mise à jour système, une exécution d'agent ou une modification de configuration. |
| Libellé | Nom ou identifiant attribué à l'événement. |
| Détails | Informations complémentaires décrivant l'événement. |
| Visible dans les graphiques | Indique si l'événement est affiché dans les graphiques associés. |
