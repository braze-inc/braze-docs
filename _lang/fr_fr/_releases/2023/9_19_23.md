# Sortie du 19 septembre 2023

## BigQuery pour l'ingestion de données cloud

Vous pouvez désormais créer des intégrations d'ingestion de données cloud avec [BigQuery](https://cloud.google.com/bigquery), un entrepôt de données d'entreprise sans serveur. Pour plus d'informations, consultez les [intégrations d'ingestion de données cloud]({{site.baseurl}}/user_guide/data_and_analytics/cloud_ingestion/integrations/?tab=bigquery).

## Braze Data Transformation

[Braze Data Transformation]({{site.baseurl}}/user_guide/data/data_transformation/overview/) vous permet de créer et de gérer des intégrations webhook pour automatiser le flux de données des plateformes externes vers les profils utilisateurs de Braze. Ces nouvelles données utilisateur intégrées peuvent ensuite alimenter des cas d'utilisation marketing encore plus sophistiqués.

## Commentaires dans Canvas

[Les commentaires dans les Canvas]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/canvas_comments/) constituent un excellent espace de collaboration pour votre équipe marketing afin d'identifier, de discuter et de revoir les moindres détails de vos Canvas. Au fur et à mesure que vous construisez un Canvas, vous pouvez créer et gérer des commentaires pour signaler les zones qui pourraient nécessiter des retours supplémentaires de vos collègues.

## Centre de livrabilité

Le [Centre de livrabilité]({{site.baseurl}}/user_guide/analytics/dashboard/deliverability_center/) fournit plus d'informations sur les performances de vos e-mails en prenant en charge l'utilisation des outils Gmail Postmaster pour suivre les données sur les e-mails envoyés et recueillir des données sur votre domaine d'envoi. 

La livrabilité des e-mails est au cœur du succès d'une campagne. En utilisant le Centre de livrabilité dans le tableau de bord de Braze, vous pouvez visualiser vos domaines par réputation IP ou erreurs de distribution pour découvrir et résoudre tout problème potentiel de livrabilité des e-mails.

## Éditeur par glisser-déposer pour les messages in-app

Ces fonctionnalités supplémentaires ont été ajoutées à l'[éditeur par glisser-déposer pour les messages in-app]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/drag_and_drop/create/) :

- Les liens textuels qui ne ferment pas le message
- Action du bouton pour demander l'amorce de notification push
- Bloc éditeur de code personnalisé

Pour profiter de toutes les fonctionnalités disponibles dans l'éditeur par glisser-déposer, mettez à jour vos SDK vers les versions recommandées.

### Enregistrer des modèles personnalisés (accès anticipé)

Dans l'éditeur par glisser-déposer pour les messages in-app, les participants à l'accès anticipé peuvent créer et enregistrer des modèles de messages in-app personnalisés en utilisant le bouton **Enregistrer comme modèle**, disponible après avoir quitté l'éditeur. Avant de pouvoir l'enregistrer comme modèle, vous devez d'abord lancer la campagne OU l'enregistrer comme brouillon. 

Vous pouvez également créer et enregistrer des modèles de messages in-app en accédant à **Modèles** > **Modèles de messages in-app**.

## Désactivation du mode sombre pour les messages in-app

Les développeurs peuvent empêcher les messages in-app d'adopter le style du mode sombre lorsque l'appareil de l'utilisateur est en mode sombre. Pour savoir comment mettre cela en œuvre, consultez la documentation suivante par plateforme :

- [Swift]({{site.baseurl}}/developer_guide/platform_integration_guides/swift/in-app_messaging/customization/setting_delegates/#disabling-dark-mode)
- [Objective-C]({{site.baseurl}}/developer_guide/platform_integration_guides/ios/in-app_messaging/customization/handling_in_app_display/#disabling-dark-mode)

## Nouveaux champs pour l'archivage des messages

[L'archivage des messages]({{site.baseurl}}/user_guide/data/export_braze_data/message_archiving/) vous permet de sauvegarder une copie des messages envoyés aux utilisateurs à des fins d'archivage ou de conformité dans votre compartiment S3. Les champs suivants ont été ajoutés au PAYLOAD JSON livré à votre compartiment S3 chaque fois qu'un message est envoyé :

- `user_id`
- `campaign_name`
- `canvas_name`
- `canvas_step_name`

## Nouvelles étiquettes de personnalisation Liquid

Pour les messages in-app, vous pouvez utiliser les attributs d'application suivants dans Liquid. Les valeurs sont basées sur la clé API du SDK que vos applications utilisent pour demander l'envoi de messages :

- {% raw %}`{{app.${api_id}}}`{% endraw %}
- {% raw %}`{{app.${name}}}`{% endraw %}

Pour en savoir plus, consultez [les étiquettes de personnalisation prises en charge]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/supported_personalization_tags#targeted-app-information).

## Nouveaux partenariats Braze

### Antavo Loyalty Cloud – Extensions de canal

L'intégration d'[Antavo]({{site.baseurl}}/partners/data_and_analytics/loyalty/antavo/) et Braze vous permet d'utiliser les données liées au programme de fidélité pour créer des campagnes personnalisées afin d'améliorer l'expérience client. Antavo prend en charge la synchronisation des données de fidélité entre les deux plateformes — il s'agit d'une synchronisation unidirectionnelle, uniquement d'Antavo vers Braze.

### Ketch – Plateforme de données client

L'intégration de Braze et [Ketch]({{site.baseurl}}/partners/data_and_analytics/data_privacy/ketch/) vous permet de contrôler les préférences de communication des clients dans le centre de préférences Ketch et de propager automatiquement ces changements à Braze.

### Redpoint – Plateforme de données client

Redpoint est une plateforme technologique qui offre aux marketeurs une plateforme d'orchestration de campagnes entièrement intégrée. L'intégration de Braze et [Redpoint]({{site.baseurl}}/partners/data_and_analytics/customer_data_platform/redpoint/) vous permet de créer des segments Braze basés sur vos données CDP Redpoint. 

### Simon Data – Plateforme de données client
 
Utilisez l'intégration de Braze et [Simon Data]({{site.baseurl}}/partners/data_and_analytics/customer_data_platform/simonai/) pour créer et synchroniser des audiences sophistiquées avec Braze pour une orchestration en temps réel et sans code. Grâce à cette intégration, vous pouvez tirer parti des capacités de priorisation des campagnes et de correspondance d'identité de Simon, du support complexe des agrégats et bien plus encore pour améliorer vos campagnes Braze en aval.

## Mises à jour SDK

Les mises à jour SDK suivantes ont été publiées. Les mises à jour majeures sont répertoriées ci-dessous ; vous pouvez trouver toutes les autres mises à jour en consultant les journaux des modifications SDK correspondants.

- [Swift SDK 6.6.1](https://github.com/braze-inc/braze-swift-sdk/blob/main/CHANGELOG.md#661)
- [SDK Web 4.9.0](https://github.com/braze-inc/braze-web-sdk/blob/master/CHANGELOG.md#490)
- [SDK Android 28.0.0](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2800)
    - Version minimale du SDK mise à jour à 21 (Lollipop).
    - Les fonctions des indicateurs de fonctionnalité ont été modifiées.
    - `Braze.getFeatureFlag(id)` renverra désormais null si l'indicateur de fonctionnalité n'existe pas.
    - `Braze.subscribeToFeatureFlagsUpdates()` ne déclenchera un rappel que lorsqu'une demande d'actualisation sera terminée, et initialement si des données mises en cache existent. Il sera également appelé avec des indicateurs de fonctionnalité mis en cache en cas d'échec de l'actualisation.
        - Si vous souhaitez obtenir la valeur mise en cache immédiatement au démarrage de l'application, utilisez `Braze.getFeatureFlag(id)`.
    - Refactorisation de `DefaultInAppMessageViewWrapper.createButtonClickListener()` en `DefaultInAppMessageViewWrapper.createButtonClickListeners()`.
- [React Native SDK 7.0.0](https://github.com/braze-inc/braze-react-native-sdk/blob/master/CHANGELOG.md#700)
    - Met à jour le pont Android natif de [Braze Android SDK 26.3.2 à 27.0.1](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2701).
- [SDK Cordova 7.0.0](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2800)
    - Met à jour le pont Android natif de [Braze Android SDK 26.3.2 à 27.0.1](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2701).
- [SDK Roku 2.0.0](https://github.com/braze-inc/braze-roku-sdk/blob/main/CHANGELOG.md#200)
    - `getFeatureFlag` renverra invalide lorsque l'indicateur n'existe pas.
    - `BrazeTask` observe maintenant `BrazeFeatureFlagsUpdated` pour savoir quand les actualisations des indicateurs de fonctionnalité réussissent ou échouent. Les valeurs de données peuvent ne pas toujours être différentes.
