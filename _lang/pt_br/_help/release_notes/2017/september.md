# Setembro de 2017

## Nova funcionalidade para relatórios de engajamento

Agora você pode usar [os relatórios de engajamento][72] para agregar métricas de uma campanha em períodos específicos. Por exemplo, você pode exportar o número total de aberturas de um trimestre ou o número total de cliques de toda a vida útil de uma campanha ou de uma tela. Tudo o que você precisa fazer é:
- Selecione um período a partir do qual os dados serão exportados,
- Agendar um relatório de engajamento que será enviado a um ou mais destinatários regularmente; e
- Adicione campanhas e Canvas ao seu relatório com base em suas tags.

## Atualizações na página de perfil do usuário

A [página de perfil do usuário][73] foi atualizada.

## Notificações por push da Web que exigem ação do usuário para serem descartadas

Agora você pode configurar o comportamento de fechamento de mensagens para os web pushes do Chrome que exigem que o destinatário interaja com a mensagem para que ela seja descartada. Esse recurso requer o Web SDK versão 1.6.13 ou superior.

## Pré-cabeçalhos de e-mail

Ao criar uma mensagem de e-mail no Braze, agora é possível inserir facilmente um pré-cabeçalho na seção **Informações de envio**.

## Novo ponto de extremidade da API para exportação de eventos brutos

Adicionamos um novo [endpoint da API][71], /raw_data/status, que permite que você consulte se um determinado dia foi carregado na exportação de eventos brutos. Você pode usá-lo para verificar se os dados brutos de um determinado dia estão disponíveis, para ajudar na depuração e na automação.
