# Lançamento em 3 de maio de 2022

## Aprovação da campanha

A aprovação da campanha adiciona um processo de revisão ao seu fluxo de trabalho antes de lançar uma campanha. Agora, você pode garantir que cada confirmação seja aprovada para lançar a campanha. Para saber mais sobre o processo de aprovação, consulte [Aprovação de campanha]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/approvals/).

## Etapa do envio de mensagens do canva

As etapas de mensagens permitem que você adicione uma mensagem independente onde quiser no Canvas Flow. Acesse nosso artigo sobre [a etapa de mensagens]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/message_step/) para saber mais.

## Assistente de Copywriting com IA

O [assistente de Copywriting da Braze com IA]({{site.baseurl}}/user_guide/intelligence/ai_copywriting#ai-copywriting-assistant) passa um breve nome ou descrição do produto para a ferramenta de geração de texto publicitário GPT3 da OpenIA para gerar um texto de marketing semelhante a um humano para as suas mensagens. Essa funcionalidade está disponível por padrão para a maioria dos criadores de mensagens no dashboard da Braze.

## Guia de teste de unidade de notificação por push do iOS

Um [guia de teste de unidade do iOS para notificações por push]({{site.baseurl}}/developer_guide/platform_integration_guides/ios/push_notifications/unit_tests#unit-tests) foi adicionado ao guia do desenvolvedor. Este guia fornecerá testes unitários que verificarão se o delegado do app está configurado corretamente. 

## Questionário de privacidade do Google

A partir de abril de 2022, os desenvolvedores do Android deverão preencher o [Formulário de segurança de dados][4] ] do Google Play para divulgar práticas de privacidade e segurança. [Este guia]({{site.baseurl}}/developer_guide/platform_integration_guides/android/google_play_privacy#google-play-privacy-questionnaire) fornece instruções sobre como preencher esse novo formulário com informações sobre como o Braze lida com os dados do seu app. 

## RelayState SAML SSO

As instruções iniciais de configuração de logon único foram atualizadas para recomendar o uso de uma chave de API `RelayState`. Para saber mais, consulte a [configuração do SAML SSO]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/set_up/) e os artigos correspondentes [do OneLogin]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/onelogin/), [Okta]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/okta/) e [Azure Active Directory]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/entra/). 

## Talon.One atualizações de integração

Nosso [Talon.One]({{site.baseurl}}/partners/message_orchestration/channel_extensions/loyalty/talonone#talonone) artigo de parceria agora tem etapas de integração, pontos de extremidade e exemplos atualizados. Se estiver aproveitando a parceria com Talon.One, recomendamos atualizar sua integração de acordo.

## Lançado o Braze Web SDK V4

A equipe de SDKs da Braze lançou o SDK para Web v4. Para obter uma lista das principais alterações, atualizações e adições, acesse nosso [changelog do](https://github.com/braze-inc/braze-web-sdk/blob/master/CHANGELOG.md) Web SDK. Foi criado um [guia de upgrade](https://github.com/braze-inc/braze-web-sdk/blob/master/UPGRADE_GUIDE.md) para quem está fazendo upgrade da V3 para a V4.

## Desvalorização do Grouparoo

O suporte ao Grouparoo foi descontinuado a partir de abril de 2022.
