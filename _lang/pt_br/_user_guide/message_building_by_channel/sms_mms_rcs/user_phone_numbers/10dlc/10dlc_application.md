# Informações necessárias para o app 10DLC

Se tiver adquirido ajuda do Braze para registrar seu 10DLC, precisará nos enviar todas as informações necessárias listadas no e-mail abaixo, conforme acordado em seu contrato conosco.

Inclua todas as capturas de tela e anexos relevantes e envie tudo para [smsapplications@braze.com](mailto:smsapplications@braze.com). Se tiver alguma dúvida sobre o 10DLC, consulte o [guia Braze SMS 10DLC]({{site.baseurl}}/assets/pdf/SMS_10DLC_Guide.pdf) ou entre em contato com o gerente de sucesso do cliente Braze.

## Informações sobre a campanha de SMS

Será necessário registrar cada grupo de inscrições de SMS que contenha um código longo dos EUA com sua própria campanha A2P 10DLC (às vezes chamada de caso de uso). Por favor, forneça as informações abaixo uma vez para cada um de seus grupos de inscrição de SMS que contenham um código longo dos EUA.	

| Campo | Opções (se aplicável) | Descrição |
| ------ | ------------------------ | --------------- |
| Caso de uso da campanha (escolha o que melhor se encaixa) | {::nomarkdown} <ul><li>Marketing</li><li>Notificações de conta</li><li>Notificações de entrega</li><li>Atendimento ao cliente</li><li>Alerta de segurança</li><li>Envio de mensagens de alerta de fraude</li><li>Ensino superior</li><li>Misturado</li><li>Pesquisa de opinião e votação</li><li>Anúncio de serviço público</li></ul>{:/} | Nota: Se você planeja usar um caso de uso "Misto" para um grupo de inscrições, consulte o gerente de sucesso do cliente ou o gerente de integração do cliente para obter detalhes importantes com antecedência |
| Nome da campanha | | {::nomarkdown}<p>O nome da campanha de SMS precisa ser exibido de forma consistente no envio de mensagens, nos Termos de Serviço e em qualquer lugar em que os usuários inscrevam-se no programa.</p><p>Exemplos:</p><ul><li>Alertas de marcação de Braze</li><li>Notificações da conta Braze</li><li>Lembretes do Braze Cart</li></ul>{:/} |
| Descrição da campanha | | Essa campanha envia mensagens de marketing e promocionais aos usuários que aceitam receber SMS, incluindo ofertas especiais/cupons e notificações de lembrete de carrinho. |
| Endereço de e-mail do suporte ao cliente | | Onde os usuários finais podem entrar em contato por e-mail para obter ajuda e suporte? |
| Número de telefone do suporte ao cliente | | Onde os usuários finais podem entrar em contato por telefone para obter ajuda e suporte? |
| URL dos termos e condições de SMS | | {::nomarkdown} <p>Para concluir o registro US A2P 10DLC, sua página de termos de SMS deve incluir o seguinte:</p><ol><li>Nome do programa (marca)</li><li>Divulgação consistente do produto no site e no fluxo de mensagens</li><li>Frequência de mensagens (consistente com o fluxo de mensagens)</li><li>"Isenção de responsabilidade "Taxas de mensagens e dados podem ser aplicadas</li><li>Instruções de HELP/STOP OU informações de contato do atendimento ao cliente</li></ol><p>Se essa página não estiver ativa ou se alguma das divulgações exigidas estiver faltando, será necessário um rascunho dos Termos de SMS juntamente com a data prevista de implementação na sua página ativa para concluir o processo de registro.</p>{:/} |
| Como os usuários finais fazem a aceitação da sua campanha de SMS? (Compartilhe como os usuários finais podem fazer a aceitação do seu programa de SMS, por exemplo, como os usuários finais podem dar consentimento para receber suas mensagens SMS. Se forem usados vários métodos de aceitação, todos eles deverão ser listados aqui). | | {::nomarkdown}<p>Exemplos:</p><ul><li>Os usuários finais fazem a aceitação enviando uma mensagem de texto START para XXXXXXXXXXXX.</li><li>Os usuários finais fazem a aceitação visitando o site e adicionando seu número de celular. Em seguida, eles marcam uma caixa concordando em receber SMS.</li><li>Se o método de aceitação escolhido ainda não estiver ativo. Forneça um modelo/uma captura de tela do método de aceitação.</li></ul>{:/} |
| Palavras-chave de aceitação | | {::nomarkdown} <p>Se os usuários finais puderem digitar uma palavra-chave para começar a receber SMS dessa campanha, essas palavras-chave deverão ser fornecidas. Por padrão, o Braze configura as seguintes palavras-chave:</p><ul><li>START</li><li>UNSTOP</li><li>SIM</li></ul><p>Informe-nos se desejar adicionar alguma palavra-chave.</p> {:/} |
| Mensagem de confirmação de aceitação | | A mensagem de resposta automática enviada aos usuários finais para confirmar a aceitação deve ser fornecida. A resposta de aceitação deve incluir o nome da marca, a confirmação do registro de aceitação em uma campanha de mensagens recorrentes, como obter ajuda e uma descrição clara de como recusar. (por exemplo, Obrigado por se inscrever no Braze Marketing Alerts! A frequência das mensagens é variável. Pode haver a cobrança de tarifas de mensagens e consumo de dados móveis. Responda HELP para obter ajuda ou STOP para fazer a aceitação). |
| Palavras-chave de aceitação | | {::nomarkdown} <p>Os usuários finais devem ser capazes de digitar uma palavra-chave para parar de receber mensagens dessa campanha. Recomenda-se também que essas mensagens de cancelamento de inscrição incluam o nome da marca (por exemplo, Braze Marketing Alerts): Você cancelou a inscrição de todas as mensagens. Responder HELP para obter ajuda). <br><br>Máximo de 320 caracteres|
| Palavras-chave de ajuda | | Os usuários finais devem ser capazes de digitar uma palavra-chave para receber ajuda. Por padrão, o Braze configura as seguintes palavras-chave: -HELP -INFO Entre em contato conosco se desejar adicionar alguma palavra-chave. |
| Mensagens de ajuda | | Ao receber as palavras-chave de aceitação dos usuários finais, as operadoras esperam uma resposta gerada automaticamente, que pode incluir o nome da marca e informações adicionais de contato de suporte (por exemplo, Braze Marketing Alerts): Para obter ajuda, ligue para 555-555-555. Taxas de mensagens e dados podem ser aplicadas. A frequência da mensagem pode variar, envie uma mensagem de texto STOP para cancelar)<br><br>Máximo de 320 caracteres|
