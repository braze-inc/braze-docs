glossaries:
  - name: Usuário Ativo
    description: "No Braze, um <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/campaigns/ideas_and_strategies/active_user_campaigns/\">usuário ativo</a> é definido como qualquer usuário que tenha uma sessão em um determinado período."
  - name: Teste A/B Multivariado
    description: "Este é um tipo de teste onde uma ou mais variáveis são alteradas e testadas para ver quais mudanças têm o melhor desempenho. <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/testing/multivariant_testing\">Testes A/B</a> podem ser feitos em campanhas de painel."
  - name: Instâncias de Aplicativo
    description: Instâncias de aplicativo referem-se aos diferentes sites e aplicativos que são coletados em um espaço de trabalho.
  - name: Ligas
    description: "Ligas são nossos <a href=\"https://www.braze.com/docs/partners/home/\">Parceiros de Tecnologia</a>."
  - name: Usuários Anônimos
    description: "Quando um perfil de usuário é reconhecido via SDK, um perfil de usuário anônimo é criado com o <a href=\"https://www.braze.com/docs/api/basics/#user-ids\">ID de usuário do Braze</a> associado." 
  - name: Interface de Programação de Aplicativos (API)
    description: "A <a href=\"https://www.braze.com/docs/api/basics/#api-overview\">API do Braze</a> fornece um serviço web onde você pode registrar ações realizadas por seus usuários diretamente via HTTP, em vez de através dos SDKs móveis. Isso permite que você, por exemplo, passe dados do usuário para o Braze que não são rastreados dentro do seu aplicativo ou site."
  - name: Braze (O Produto)
    description: "Às vezes referido como o painel, este produto controla todos os dados e interações no coração da plataforma Braze. Os clientes do Braze o utilizam para gerenciar notificações, configurar campanhas de mensagens direcionadas e visualizar análises. Os desenvolvedores usam isso para gerenciar configurações para integrar aplicativos, como chaves de API e credenciais de notificações push."
  - name: Ferramentas de Engajamento Braze
    description: "A Braze oferece muitas <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/\">ferramentas de engajamento</a> para alcançar seus clientes e usuários com nossas campanhas, ferramentas Canvas, e você pode otimizar para consistência (e enviar imagens e outros conteúdos) usando nossas ferramentas de Modelos e Mídia. A partir daí, você pode criar segmentos e geofences para segmentar seu público por localização ou outros atributos."
  - name: Equipes Braze
    description: "Os administradores da Braze podem dividir um subconjunto de usuários do painel em <a href=\"https://www.braze.com/docs/user_guide/administrative/app_settings/manage_your_braze_users/teams/\">Equipes</a> com diferentes funções e permissões de usuário. Isso permite que os administradores da Braze limitem o acesso a certos recursos por associação a grupos."
  - name: Campanhas
    description: "As campanhas são métodos de mensagens personalizáveis para entregar respostas personalizadas aos seus clientes. Você pode <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/campaigns/\">criar campanhas</a> usando diferentes canais de mensagens para enviar suas mensagens exclusivas." 
  - name: Canvas
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/canvas/\">Canvas</a> é uma interface única e unificada onde os profissionais de marketing podem configurar campanhas com várias mensagens e etapas para formar uma jornada coesa. O Canvas permite que você compare e otimize essas experiências usando análises abrangentes para toda a experiência do usuário."
  - name: Conteúdo Conectado
    description: "<a href=\"https://www.braze.com/docs/user_guide/personalization_and_dynamic_content/connected_content/\">Conteúdo Conectado</a> expande a personalização de marketing para aumentar o engajamento e as conversões dos clientes. Você pode inserir qualquer informação acessível usando a API diretamente nas mensagens que envia para os usuários. O Conteúdo Conectado permite puxar conteúdo diretamente do seu servidor web ou de APIs publicamente acessíveis."
  - name: Cartões de Conteúdo
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/content_cards/about/\">Cartões de Conteúdo</a> permitem que você envie um fluxo dinâmico de conteúdo rico altamente segmentado para seus clientes diretamente nos aplicativos que eles amam, sem interromper sua experiência. Os Cartões de Conteúdo podem ser enviados para usuários de iOS, Android e web."
  - name: Correntes
    description: "<a href = \"https://www.braze.com/docs/user_guide/data/braze_currents/\">Correntes</a>, nossa exportação de streaming de dados, está incluída em certos pacotes do Braze. O Braze Currents permite que você integre através do Armazenamento de Dados usando arquivos planos ou com nossos parceiros de Análise Comportamental e Dados do Cliente usando cargas JSON em lotes para um ponto final designado."
  - name: Atributos Personalizados
    description: "<a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_attributes/\">Atributos personalizados</a> são uma coleção das características únicas dos seus usuários. Eles são melhores para armazenar atributos sobre seus usuários ou informações sobre ações de baixo valor dentro do seu aplicativo. Você pode atribuir atributos personalizados aos usuários dentro do painel. Você pode filtrar e segmentar seus usuários de acordo com esses atributos para campanhas tanto de <a href=\"https://www.braze.com/docs/developer_guide/platforms/swift/analytics/setting_custom_attributes/\">iOS</a> quanto de <a href=\"https://www.braze.com/docs/developer_guide/platforms/android/analytics/setting_custom_attributes/\">Android</a>."
  - name: Eventos Personalizados
    description: "<a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_events\">Eventos personalizados</a> são ações realizadas pelos seus usuários; eles são mais adequados para rastrear interações de alto valor dos usuários com seu aplicativo."
  - name: Ponto de Dados
    description: "Um ponto de dados é contado quando um <a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_attributes/\">atributo personalizado</a> é definido ou atualizado (mesmo que você esteja atualizando com o mesmo valor), um <a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_events\">evento personalizado</a> ou evento de compra é registrado, qualquer dado padrão (por exemplo, <code>email</code>, <code>primeiro_nome</code>, <code>sobrenome</code>, <code>país</code>, ou <code>cidade_natal</code>) é registrado, quando uma sessão começa e quando uma sessão termina."
  - name: Deep Linking
    description: "<a href=\"https://www.braze.com/docs/user_guide/personalization_and_dynamic_content/deep_linking_to_in-app_content/\">Links profundos</a> são usados para direcionar os clientes para sua próxima ação ou engajamento. Usando links profundos, você pode conectar uma mensagem a um conteúdo direcionado dentro de um site ou aplicativo móvel."
  - name: Usuários Inativos
    description: "Um usuário é considerado <a href=\"https://www.braze.com/docs/user_archival/#dormant-users\">inativo</a> quando não houve atividade nos últimos doze meses."
  - name: Ponto Final
    description: "Um fim de um canal de comunicação, também conhecido como <a href=\"https://www.braze.com/docs/api/endpoints/\">ponto final</a> de API, é usado dentro da API de mensagens do Braze para enviar e agendar mensagens."
  - name: Eventos de Exceção
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/canvas/create_a_canvas/exit_criteria/#exception-events\">Eventos de exceção</a> ocorrem quando um usuário atinge o objetivo desejado e não recebe uma mensagem acionada. Dessa forma, a mensagem acionada ainda é relevante para o usuário."
  - name: Limitação de Frequência
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/#frequency-capping\">Limitação de frequência</a> permite que você gerencie a comunicação sem sobrecarregar seu público. É um limite automatizado nas mensagens para evitar que os usuários recebam muitas comunicações em um curto período de tempo."
  - name: HIPAA
    description: "HIPAA é um acrônimo para Lei de Portabilidade e Responsabilidade de Seguros de Saúde. Braze é <a href=\"https://www.braze.com/docs/developer_guide/disclosures/security_qualifications/#hipaa\">compatível com HIPAA</a>. Os requisitos da HIPAA envolvem segurança administrativa, física e técnica."
  - name: Mensagens no Aplicativo
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/in-app_messages/\">Mensagens no aplicativo</a> são mensagens móveis que aparecem dentro do seu aplicativo. Elas ajudam você a levar conteúdo ao seu usuário sem interromper seu dia com uma notificação push. Mensagens no aplicativo personalizadas e adaptadas melhoram a experiência do usuário e ajudam seu público a obter o máximo valor do seu aplicativo."
  - name: Usuários Inativos
    description: "Um usuário é considerado <a href=\"https://www.braze.com/docs/user_archival/#inactive-users\">inativo</a> quando atinge certos critérios, como não receber mais e-mails ou mensagens push e não estar ativo por mais de seis meses."
  - name: Aquecimento de IP
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/email/email_setup/ip_warming\">Aquecimento de IP</a> é a prática de aumentar gradualmente a quantidade de e-mails enviados de um IP dedicado. Isso ajuda a estabelecer uma reputação com Provedores de Serviços de Internet, minimizando a probabilidade de suas mensagens serem sinalizadas."
  - name: Pares Chave-Valor
    description: "<a href=\"http://www.braze.com/docs/user_guide/personalization_and_dynamic_content/key_value_pairs/\">Pares chave-valor</a> são itens de dados vinculados onde a chave é um identificador único e o valor é o conteúdo. Eles podem ser usados para enviar cargas de dados extras para dispositivos de usuários."
  - name: Líquido
    description: "Líquido é uma linguagem de template comumente usada, voltada para o cliente, criada pela Shopify e escrita em Ruby. <a href=\"https://www.braze.com/docs/user_guide/personalization_and_dynamic_content/liquid/\">Líquido</a> é usado para carregar e puxar conteúdo dinâmico. Líquido permite que você use objetos, tags e filtros para <a href=\"http://www.braze.com/docs/user_guide/personalization_and_dynamic_content/liquid/supported_personalization_tags/\">adicionar personalização personalizada</a>."
  - name: Canal de Mensagens
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/\">Canais de mensagens</a> são maneiras de você se comunicar virtualmente com seus clientes – através de notificações push em seus telefones ou navegadores da web, e-mail, mensagens no aplicativo, e muito mais!"
  - name: Usuário Ativo Mensal (UAM)
    description: Estes são usuários que tiveram uma sessão nos últimos 30 dias.
  - name: Mensagens Multicanal
    description: "Mensagens para um usuário através de vários meios, como uma combinação de e-mail, push na web e notificações push móveis. <a href=\"https://www.braze.com/docs/developer_guide/getting_started/platform_overview/#multichannel-messaging\">Canais de mensagens</a> são melhor utilizados em conjunto e com regularidade para reengajar usuários perdidos, reter usuários ativos e energizar seus embaixadores da marca."
  - name: Teste Multivariado
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/testing/multivariant_testing/#multivariate-test\">Teste multivariado</a> permite que você teste múltiplas variáveis comparando duas ou mais versões de uma mensagem para avaliar qual variante tem o melhor desempenho."
  - name: Novo Usuário
    description: "A Braze considera um novo usuário como qualquer pessoa que instalou seu aplicativo recentemente. Alternativamente, um novo usuário também pode ser definido como um usuário com um ID de usuário que não foi identificado anteriormente dentro da Braze."
  - name: Personalização
    description: "Usando tecnologia para levar em conta as preferências e tendências individuais de cada usuário ao se comunicar com eles. <a href=\"https://www.braze.com/docs/user_guide/personalization_and_dynamic_content/\">Mensagens personalizadas</a> ajudam a construir experiências valiosas para os clientes, adaptando-se às suas preferências." 
  - name: Mensagens Push
    description: "Uma <a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/push/\">mensagem push</a>, ou notificação push, é uma notificação que aparece de um aplicativo móvel. As notificações push geralmente aparecem como diálogos pop-up e banners tanto para iOS quanto para Android."
  - name: Token Push
    description: "Um token push é uma chave única, criada e atribuída pela Apple ou Google para criar uma conexão entre um aplicativo e um dispositivo iOS, Android ou web. <a href=\"https://www.braze.com/docs/help/help_articles/push/push_token_migration/#push-token-migration\">Migração de token push</a> é a importação dessas chaves já geradas para o Braze."
  - name: Tempo de Vida do Push
    description: "Também conhecido como <a href=\"https://www.braze.com/docs/user_guide/administrative/app_settings/push_settings/\">TTL do Push</a>, tempo de vida refere-se ao período em que as campanhas continuarão a tentar ser entregues a um usuário offline."
  - name: Condição de Corrida
    description: "Uma <a href=\"http://www.braze.com/docs/user_guide/engagement_tools/testing/race_conditions/\">condição de corrida</a> é um conceito de engenharia de software que descreve uma situação indesejável que ocorre quando um sistema tenta realizar várias operações simultaneamente, mas devido à natureza do sistema, as operações devem ser feitas na sequência correta para serem realizadas corretamente. <br><br>Na plataforma Braze, segmentar uma campanha acionada com base nos dados do usuário registrados no momento do evento pode causar uma condição de corrida. Isso acontece quando uma mudança no atributo do usuário no qual a campanha é segmentada ainda não foi processada para o usuário no momento em que a adesão ao segmento é determinada e a campanha é enviada, podendo levar o usuário a não receber a campanha."
  - name: Limitação de Taxa
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/\">Limitação de taxa</a> é o processo de restringir o número de mensagens enviadas durante um período de tempo para evitar sobrecarregar seu público."  
  - name: Tags
    description: "<a href=\"https://www.braze.com/docs/user_guide/administrative/app_settings/tags/\">Tags</a> são uma ferramenta que ajuda você a categorizar, organizar e classificar seu engajamento em uma ou várias campanhas."
  - name: Segmentação
    description: "O <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/segments/\">painel de segmentação</a> permite que você crie grupos ou extensões de usuários com base em filtros poderosos de seu comportamento no aplicativo, dados demográficos e mais."
  - name: Kit de Desenvolvimento de Software (SDK)
    description: "<a href=\"https://www.braze.com/docs/developer_guide/getting_started/sdk_overview/\">SDKs</a> são integrados em aplicativos iOS e Android e fornecem ferramentas poderosas de marketing, suporte ao cliente e análise. A Braze oferece integração de SDK para <a href=\"https://www.braze.com/docs/developer_guide/platforms/swift/sdk_integration/#integrating-the-sdk/\">iOS</a> e <a href=\"https://www.braze.com/docs/developer_guide/platforms/android/sdk_integration/#step-1-configure-brazexml\">Android</a>."
  - name: Desativação
    description: "Desativação refere-se ao processo de identificar usuários desengajados e interromper o envio de mensagens ativas para esses usuários sem que eles precisem tomar qualquer ação. Criar políticas de desativação para suas mensagens de <a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/email/best_practices/sunset_policies/\">email</a> e <a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/push/best_practices/sunset_policies/#sunset-policies-for-push/\">push</a> pode ajudar a reduzir os impactos nas suas taxas de abertura." 
  - name: Arquivamento de Usuários
    description: "<a href=\"https://www.braze.com/docs/user_archival/\">Arquivamento de usuários</a> refere-se a usuários que foram arquivados. Na Braze, isso inclui tanto usuários inativos quanto dormentes."
  - name: Webhooks
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/webhooks/\">Webhooks</a> permitem que você acione ações fora do aplicativo, como entrega de mensagens de texto SMS. Você pode usar webhooks para fornecer a outros sistemas e aplicativos informações em tempo real. A flexibilidade deste recurso permite que você envie informações para qualquer endpoint."
