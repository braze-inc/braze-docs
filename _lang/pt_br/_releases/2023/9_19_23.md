# Versão de 19 de setembro de 2023

## BigQuery para ingestão de dados na nuvem

Agora você pode criar integrações de ingestão de dados na nuvem com o [BigQuery](https://cloud.google.com/bigquery), um data warehouse corporativo sem servidor. Para saber mais, consulte as [integrações de ingestão de dados na nuvem]({{site.baseurl}}/user_guide/data_and_analytics/cloud_ingestion/integrations/?tab=bigquery).

## Braze Data Transformation

O [Braze Data Transformation]({{site.baseurl}}/user_guide/data/data_transformation/overview/) permite que você crie e gerencie integrações de webhook para automatizar o fluxo de dados de plataformas externas para os perfis de usuários da Braze. Esses dados de usuários recém-integrados podem, então, alimentar casos de uso de marketing ainda mais sofisticados.

## Comentários no Canvas

Os [comentários nos canvas]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/canvas_comments/) podem ser um ótimo espaço de colaboração para a sua equipe de marketing identificar, discutir e revisar os detalhes mais finos dos seus canvas. Ao criar um canva, você pode fazer e gerenciar comentários para identificar áreas que podem exigir feedback adicional de seus colegas.

## Centro de Entregabilidade

O [Centro de Entregabilidade]({{site.baseurl}}/user_guide/analytics/dashboard/deliverability_center/) fornece mais insights sobre sua performance de e-mail, apoiando o uso das Ferramentas do Postmaster do Gmail para rastrear dados sobre e-mails enviados e coletar dados sobre seu domínio de envio. 

A entregabilidade do e-mail é o núcleo do sucesso da campanha. Usando o Centro de Entregabilidade no dashboard da Braze, você pode visualizar seus domínios por reputação de IP ou erros de entrega para descobrir e solucionar quaisquer problemas potenciais com a entregabilidade de e-mail.

## Editor de arrastar e soltar para mensagens no app

Esses recursos adicionais foram incluídos no [editor de arrastar e soltar para mensagens no app]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/drag_and_drop/create/):

- Links de texto que não descartam a mensagem
- Ação de botão para solicitar um push primer
- Bloco do editor de código personalizado

Para aproveitar todos os recursos disponíveis no editor de arrastar e soltar, atualize seus SDKs para as versões recomendadas.

### Salvar modelos personalizados (acesso antecipado)

No editor de arrastar e soltar para mensagens no app, os participantes do acesso antecipado podem criar e salvar modelos personalizados de mensagens no app usando o botão **Salvar como modelo**, disponível depois que você sair do editor. Antes de poder salvá-lo como modelo, você deve primeiro lançar a campanha OU salvá-lo como rascunho. 

Você também pode criar e salvar modelos de mensagens no app navegando até **Modelos** > **Modelos de mensagens no app**.

Os desenvolvedores podem impedir que as mensagens no app adotem o estilo do modo escuro quando o dispositivo do usuário estiver com o modo escuro ativado. Para saber como implementar isso, consulte a seguinte documentação por plataforma:

## Novos campos para arquivamento de mensagens

O [arquivamento de mensagens]({{site.baseurl}}/user_guide/data/export_braze_data/message_archiving/) permite salvar uma cópia das mensagens enviadas aos usuários para fins de arquivamento ou conformidade no seu bucket S3. Os seguintes campos foram adicionados à carga útil JSON entregue ao seu bucket S3 sempre que uma mensagem é enviada:

- `user_id`
- `campaign_name`
- `canvas_name`
- `canvas_step_name`

## Novas tags de personalização Liquid

Para mensagens no app, você pode usar os seguintes atributos do app no Liquid. Os valores são baseados na Chave da API SDK que seus apps usam para solicitar o envio de mensagens:

- {% raw %}`{{app.${api_id}}}`{% endraw %}
- {% raw %}`{{app.${name}}}`{% endraw %}

Para saber mais, consulte [Tags de personalização compatíveis]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/supported_personalization_tags#targeted-app-information).

## Novas parcerias da Braze

### Antavo Loyalty Cloud – Extensões de canal

A integração entre a [Antavo]({{site.baseurl}}/partners/data_and_analytics/loyalty/antavo/) e a Braze permite que você use dados relacionados ao programa de fidelidade para criar campanhas personalizadas e aprimorar a experiência do cliente. A Antavo oferece suporte à sincronização de dados de fidelidade entre as duas plataformas — essa é uma sincronização de dados unidirecional, somente da Antavo para a Braze.

### Ketch – Plataforma de dados do cliente

A integração entre a Braze e o [Ketch]({{site.baseurl}}/partners/data_and_analytics/data_privacy/ketch/) permite que você controle as preferências de comunicação do cliente na Central de Preferências do Ketch e propague automaticamente essas alterações para a Braze.

### Redpoint – Plataforma de dados do cliente

A Redpoint é uma plataforma de tecnologia que fornece aos profissionais de marketing uma plataforma de orquestração de campanhas totalmente integrada. A integração da Braze com o [Redpoint]({{site.baseurl}}/partners/data_and_analytics/customer_data_platform/redpoint/) permite que você crie segmentos da Braze com base nos dados do Redpoint CDP. 

### Simon Data – Plataforma de dados do cliente
 
Use a integração entre a Braze e o [Simon Data]({{site.baseurl}}/partners/data_and_analytics/customer_data_platform/simonai/) para criar e sincronizar públicos sofisticados com a Braze para orquestração em tempo real e sem código. Com essa integração, você pode aproveitar o melhor dos recursos de priorização de campanhas e de correspondência de identidades do Simon, o suporte a agregações complexas e muito mais para elevar suas campanhas na Braze.

## Atualizações do SDK

As seguintes atualizações do SDK foram lançadas. As atualizações com breaking changes estão listadas abaixo; todas as outras atualizações podem ser encontradas nos changelogs correspondentes do SDK.

- [Swift SDK 6.6.1](https://github.com/braze-inc/braze-swift-sdk/blob/main/CHANGELOG.md#661)
- [Web SDK 4.9.0](https://github.com/braze-inc/braze-web-sdk/blob/master/CHANGELOG.md#490)
- [Android SDK 28.0.0](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2800)
    - Atualização da versão mínima do SDK para 21 (Lollipop).
    - As funções de Feature Flags foram modificadas.
    - `Braze.getFeatureFlag(id)` agora retornará nulo se a Feature Flag não existir.
    - `Braze.subscribeToFeatureFlagsUpdates()` somente fará um retorno de chamada quando uma solicitação de atualização for concluída e, inicialmente, se houver dados armazenados em cache anteriormente. Ele também será chamado com Feature Flags armazenadas em cache para qualquer falha de atualização.
        - Se quiser o valor em cache imediatamente na inicialização do app, use `Braze.getFeatureFlag(id)`.
    - Refatoração de `DefaultInAppMessageViewWrapper.createButtonClickListener()` para `DefaultInAppMessageViewWrapper.createButtonClickListeners()`.
- [React Native SDK 7.0.0](https://github.com/braze-inc/braze-react-native-sdk/blob/master/CHANGELOG.md#700)
    - Atualiza a ponte nativa do Android do [SDK da Braze para Android 26.3.2 para o 27.0.1](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2701).
- [Cordova SDK 7.0.0](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2800)
    - Atualiza a ponte nativa do Android do [SDK da Braze para Android 26.3.2 para o 27.0.1](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2701).
- [Roku SDK 2.0.0](https://github.com/braze-inc/braze-roku-sdk/blob/main/CHANGELOG.md#200)
    - `getFeatureFlag` retornará inválido quando a flag não existir.
    - `BrazeTask` agora observa `BrazeFeatureFlagsUpdated` para saber quando as atualizações das Feature Flags são bem-sucedidas ou falham. Os valores dos dados nem sempre serão diferentes.
