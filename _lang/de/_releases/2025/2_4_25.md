# Release vom 4. Februar 2025

## Verbesserungen an Braze Docs

### Überarbeitung von Daten und Analytics
Um Ihnen die Suche zu erleichtern, haben wir die Artikel, die früher unter „Daten & Analytics" zusammengefasst waren, in [Daten]({{site.baseurl}}/user_guide/data) und [Analytics]({{site.baseurl}}/user_guide/analytics) aufgeteilt. 

### Entwicklerhandbuch
Wir haben alle Dokumente im [Braze Entwicklerhandbuch]({{site.baseurl}}/developer_guide/home) umfassend aufgeräumt und dabei auch die auf mehrere Seiten verteilten Anleitungen auf einer einzigen Seite zusammengefasst.

Außerdem gibt es eine neue [SDK-Referenzseite]({{site.baseurl}}/developer_guide/references), auf der alle Referenzdokumentationen und Repositories für jedes Braze SDK aufgelistet sind.

#### Unreal Engine Braze SDK
Wir haben alle Inhalte aus der README-Datei des Unreal Engine Braze SDK GitHub-Repositorys in den [entsprechenden Abschnitt auf Braze Docs]({{site.baseurl}}/developer_guide/sdk_integration/?sdktab=unreal%20engine) migriert und neu geschrieben.

## Datenflexibilität

### Dashboard zur API-Nutzung

{% multi_lang_include release_type.md release="General availability" %}

Mit dem [Dashboard zur API-Nutzung]({{site.baseurl}}/user_guide/analytics/dashboard/api_usage_dashboard) können Sie den bei Braze eingehenden REST API-Traffic überwachen, um Nutzungstrends Ihrer REST APIs zu erkennen und mögliche Probleme zu beheben.

### Tags zu angepassten Attributen hinzufügen

{% multi_lang_include release_type.md release="General availability" %}

Sie können [Tags zu einem angepassten Attribut hinzufügen]({{site.baseurl}}/user_guide/data/custom_data/custom_attributes#adding-tags), nachdem es erstellt wurde, sofern Sie die Berechtigung „Events, Attribute und Käufe verwalten" haben. Die Tags können dann zum Filtern der Attributliste verwendet werden.

### Endpunkte für Katalogauswahlen und asynchrone Katalogfelder 

{% multi_lang_include release_type.md release="General availability" %}

Die folgenden Endpunkte sind jetzt allgemein verfügbar:
* [POST: Katalogfelder erstellen]({{site.baseurl}}/api/endpoints/catalogs/catalog_fields/asynchronous/post_create_catalog_fields)
* [DELETE: Katalogfeld löschen]({{site.baseurl}}/api/endpoints/catalogs/catalog_fields/asynchronous/delete_catalog_field)
* [DELETE: Katalogauswahl löschen]({{site.baseurl}}/api/endpoints/catalogs/catalog_selections/asynchronous/delete_catalog_selection)
* [POST: Katalogauswahl erstellen]({{site.baseurl}}/api/endpoints/catalogs/catalog_selections/asynchronous/post_create_catalog_selections)

### E-Mail-Adresse verwenden, um Kampagnen oder Canvase zu triggern

{% multi_lang_include release_type.md release="General availability" %}

Sie können jetzt eine Empfänger:in per E-Mail-Adresse angeben, um Ihre [Kampagnen]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/targeting_users/) und [Canvase]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas/?tab=target%20audience#step-2c-set-your-target-entry-audience) zu triggern.

### Telefonnummer zur Identifizierung einer Nutzer:in über die API verwenden

{% multi_lang_include release_type.md release="General availability" %}

Sie können jetzt zusätzlich zu einem Alias und einer E-Mail-Adresse eine Telefonnummer verwenden, um eine Nutzer:in über den [`/users/identify` API-Endpunkt]({{site.baseurl}}/api/endpoints/user_data/post_user_identify) zu identifizieren.

### SAML-Trace abrufen
Wir haben [Schritte zum Abrufen eines SAML-Trace]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/set_up#obtaining-a-saml-trace) hinzugefügt, mit denen Sie Probleme mit SAML SSO gemeinsam mit dem Support effizienter lösen können.
 
### Regionale Rechenzentren
Da Braze wächst und neue Regionen bedient, haben wir einen [Artikel über die Rechenzentren von Braze]({{site.baseurl}}/user_guide/data/data_centers) hinzugefügt, um unseren operativen Ansatz zu erläutern.

 
## Kreativität entfesseln
 
### Preissenkungsbenachrichtigungen und Wiederverfügbarkeitsbenachrichtigungen

{% multi_lang_include release_type.md release="General availability" %}

Sie können Kund:innen jetzt benachrichtigen, wenn ein Artikel wieder vorrätig ist, indem Sie über ein Canvas und einen Katalog [Wiederverfügbarkeitsbenachrichtigungen]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/catalogs/catalog_triggers/back_in_stock_notifications) einrichten.

Sie können auch [Preissenkungsbenachrichtigungen]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/catalogs/catalog_triggers/price_drop_notifications) erstellen, um Kund:innen zu informieren, wenn der Preis eines Artikels gesunken ist, indem Sie Preissenkungsbenachrichtigungen in einem Katalog und Canvas einrichten.

### Vorschau für Auswahl 

{% multi_lang_include release_type.md release="General availability" %}

Nachdem Sie eine Auswahl erstellt haben, können Sie sich ansehen, [was eine Auswahl für eine zufällige oder eine bestimmte Nutzer:in zurückgeben würde]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/catalogs/selections/#test-and-preview).

### Katalogartikel mit Liquid als Template verwenden 

{% multi_lang_include release_type.md release="General availability" %}

Sie können [Katalogartikel, die Liquid enthalten, als Template verwenden]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/catalogs/using_catalogs/#using-liquid).

### Canvas-Templates
Wir haben neue Canvas-Templates für das [Onboarding von Nutzer:innen mit einer Präferenzumfrage]({{site.baseurl}}/user_guide/engagement_tools/canvas/get_started/braze_templates/preference_survey) und für die [Erstellung einer E-Mail-Registrierung mit doppeltem Opt-in]({{site.baseurl}}/user_guide/engagement_tools/canvas/get_started/braze_templates/email_signup) hinzugefügt.

### Leads mit Salesforce Sales Cloud für B2B verwalten
Eine Möglichkeit, wie B2B-Marketer Braze nutzen können, ist eine Integration mit der Salesforce Sales Cloud. Erfahren Sie mehr darüber, wie Sie diesen [Anwendungsfall]({{site.baseurl}}/user_guide/getting_started/b2b_use_cases/b2b_salesforce_sales_cloud) implementieren können.
 
## Robuste Kanäle

### Unterdrückungslisten

{% multi_lang_include release_type.md release="Beta" %}
 
[Unterdrückungslisten]({{site.baseurl}}/user_guide/engagement_tools/segments/suppression_lists) legen Gruppen von Nutzer:innen fest, die niemals Nachrichten erhalten. Administratoren können Unterdrückungslisten mit Segmentfiltern erstellen, um eine Nutzer:innen-Gruppe auf die gleiche Weise einzugrenzen wie bei der Segmentierung.

## Neue Braze-Partnerschaften

### Constructor – Dynamischer Content
[Constructor]({{site.baseurl}}/partners/ecommerce/product_search_recommendations/constructor/) ist eine Such- und Produktentdeckungsplattform, die KI und maschinelles Lernen nutzt, um personalisierte Suchergebnisse, Empfehlungen und Browsing-Erlebnisse für E-Commerce- und Einzelhandels-Websites bereitzustellen.
 
### Trustpilot – Dynamischer Content
[Trustpilot]({{site.baseurl}}/partners/trustpilot/) ist eine Online-Bewertungsplattform, die es Ihren Kund:innen ermöglicht, Feedback zu teilen, und mit der Sie Bewertungen verwalten und darauf reagieren können.
