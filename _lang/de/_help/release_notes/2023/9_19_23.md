# September 19, 2023 Veröffentlichung

## BigQuery für Cloud-Daten-Ingestion

Sie können jetzt Cloud Data Ingestion-Integrationen mit [BigQuery](https://cloud.google.com/bigquery), einem serverlosen Enterprise Data Warehouse, erstellen. Weitere Informationen finden Sie unter [Cloud Data Integestion Integrationen]({{site.baseurl}}/user_guide/data_and_analytics/cloud_ingestion/integrations/?tab=bigquery).

## Braze Data Transformation

Mit [Braze Data Transformation]({{site.baseurl}}/user_guide/data_and_analytics/data_transformation/overview/) können Sie Webhook-Integrationen erstellen und verwalten, um den Datenfluss von externen Plattformen in Braze-Benutzerprofile zu automatisieren. Diese neu integrierten Benutzerdaten können dann für noch anspruchsvollere Marketinganwendungen genutzt werden.

## Kommentieren in Canvas

[Kommentare in Canvases]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/canvas_comments/) können ein großartiger Raum für die Zusammenarbeit Ihres Marketingteams sein, um die Feinheiten Ihrer Canvases zu identifizieren, zu diskutieren und zu überprüfen. Während Sie ein Canvas erstellen, können Sie Kommentare abgeben und verwalten, um diese Bereiche zu identifizieren, die möglicherweise zusätzliches Feedback von Ihren Kollegen benötigen.

## Zustellzentrum

Das [Zustellbarkeitscenter]({{site.baseurl}}/user_guide/data_and_analytics/analytics/deliverability_center) bietet mehr Einblick in Ihre E-Mail-Leistung, indem es die Verwendung der Gmail Postmaster Tools unterstützt, um Daten über gesendete E-Mails zu verfolgen und Daten über Ihre Versanddomäne zu sammeln. 

Die Zustellbarkeit von E-Mails ist der Schlüssel zum Erfolg einer Kampagne. Mit dem Zustellbarkeitscenter im Braze Dashboard können Sie Ihre Domains nach IP-Reputation oder Zustellungsfehlern anzeigen, um mögliche Probleme mit der E-Mail-Zustellbarkeit zu erkennen und zu beheben.

## Drag-and-Drop-Editor für In-App-Nachrichten

Diese zusätzlichen Funktionen wurden dem [Drag-and-Drop-Editor für In-App-Nachrichten]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/drag_and_drop/create/) hinzugefügt:

- Textlinks, die die Nachricht nicht beenden
- Tastenaktion zur Anforderung von Push Primer
- Benutzerdefinierter Code-Editor-Block

Um alle Funktionen des Drag-and-Drop-Editors nutzen zu können, aktualisieren Sie Ihre SDKs auf die empfohlenen SDK-Versionen.

### Benutzerdefinierte Vorlagen speichern (früher Zugang)

Im Drag-and-Drop-Editor für In-App-Nachrichten können Teilnehmer mit frühem Zugang benutzerdefinierte In-App-Nachrichtenvorlagen über die Schaltfläche **Als Vorlage speichern** erstellen und speichern, die nach Verlassen des Editors verfügbar ist. Bevor Sie sie als Vorlage speichern können, müssen Sie die Kampagne zunächst starten ODER als Entwurf speichern. 

Sie können auch In-App-Nachrichtenvorlagen erstellen und speichern, indem Sie zu **Vorlagen** > **In-App-Nachrichtenvorlagen** navigieren.

{% alert important %}
Die Möglichkeit, benutzerdefinierte Vorlagen zu speichern, befindet sich derzeit in der Early Access-Phase. Wenden Sie sich an Ihren Braze-Kundenbetreuer, wenn Sie sich für die Teilnahme am Early Access interessieren.
{% endalert %}

## Deaktivieren des Dunkelmodus für In-App-Nachrichten

Entwickler können verhindern, dass In-App-Nachrichten das Design des dunklen Modus übernehmen, wenn das Gerät des Benutzers den dunklen Modus aktiviert hat. Wie Sie dies umsetzen, erfahren Sie in der folgenden Dokumentation für jede Plattform:

- [Schnell]({{site.baseurl}}/developer_guide/platform_integration_guides/swift/in-app_messaging/customization/setting_delegates/#disabling-dark-mode)
- [Objective-C]({{site.baseurl}}/developer_guide/platform_integration_guides/ios/in-app_messaging/customization/handling_in_app_display/#disabling-dark-mode)

## Neue Felder für die Nachrichtenarchivierung

Mit der [Nachrichtenarchivierung]({{site.baseurl}}/user_guide/data_and_analytics/export_braze_data/message_archiving/) können Sie eine Kopie der an Benutzer gesendeten Nachrichten zu Archivierungs- oder Compliance-Zwecken in Ihrem S3-Bucket speichern. Die folgenden Felder wurden zu der JSON-Nutzlast hinzugefügt, die bei jedem Versand einer Nachricht an Ihren S3-Bucket geliefert wird:

- `user_id`
- `campaign_name`
- `canvas_name`
- `canvas_step_name`

## Neue Tags zur Personalisierung von Flüssigkeiten

Für In-App-Nachrichten können Sie die folgenden App-Attribute in Liquid verwenden. Die Werte hängen davon ab, welchen SDK-API-Schlüssel Ihre Anwendungen für die Anforderung von Nachrichten verwenden:

- {% raw %}`{{app.${api_id}}}`{% endraw %}
- {% raw %}`{{app.${name}}}`{% endraw %}

Weitere Informationen finden Sie unter [Unterstützte Personalisierungs-Tags]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/supported_personalization_tags#targeted-app-information).

## Neue Lötpartnerschaften

### Antavo Loyalty Cloud - Kanal-Erweiterungen

Die Integration von [Antavo]({{site.baseurl}}/partners/message_orchestration/channel_extensions/loyalty/antavo/) und Braze ermöglicht es Ihnen, Daten aus Treueprogrammen zu nutzen, um personalisierte Kampagnen zu erstellen, die das Kundenerlebnis verbessern. Antavo unterstützt die Synchronisierung von Loyalitätsdaten zwischen den beiden Plattformen - dies ist eine einseitige Datensynchronisierung von Antavo zu Braze.

### Ketch - Plattform für Kundendaten

Die Integration von Braze und [Ketch]({{site.baseurl}}/partners/data_and_infrastructure_agility/customer_data_platform/ketch/) ermöglicht es Ihnen, die Kommunikationspräferenzen Ihrer Kunden im Ketch-Einstellungscenter zu steuern und diese Änderungen automatisch an Braze weiterzugeben.

### Redpoint - Plattform für Kundendaten

Redpoint ist eine Technologieplattform, die Vermarktern eine vollständig integrierte Plattform für die Orchestrierung von Kampagnen bietet. Die Integration von Braze und [Redpoint]({{site.baseurl}}/partners/data_and_infrastructure_agility/customer_data_platform/redpoint/) ermöglicht es Ihnen, Braze-Segmente auf der Grundlage Ihrer Redpoint CDP-Daten zu erstellen. 

### Simon Data - Plattform für Kundendaten
 
Nutzen Sie die Integration von Braze und [Simon Data]({{site.baseurl}}/partners/data_and_infrastructure_agility/customer_data_platform/simondata/), um anspruchsvolle Zielgruppen zu erstellen und mit Braze zu synchronisieren, um sie in Echtzeit und ohne Code zu orchestrieren. Mit dieser Integration können Sie das Beste aus Simons Kampagnenpriorisierungs- und Identitätsabgleichsfunktionen, der Unterstützung komplexer Aggregate und vielem mehr nutzen, um Ihre Braze-Kampagnen weiterzuentwickeln.

### OfferFit - Dynamischer Inhalt

Die Integration von [OfferFit]({{site.baseurl}}/partners/message_personalization/dynamic_content/offerfit/) und Braze ermöglicht es Ihnen, auf der Grundlage Ihrer Kundendaten automatisch die richtige Botschaft, den richtigen Kanal und das richtige Timing für jeden Kunden zu finden. Sie können Ihre Kampagnen für bereits identifizierte Kunden mit Geschäftszielen wie Cross-Selling, Upselling, Wiederkauf, Bindung, Erneuerung, Empfehlung und Rückgewinnung optimieren.

## SDK Aktualisierungen

Die folgenden SDK-Updates wurden veröffentlicht. Die wichtigsten Updates sind unten aufgeführt. Alle anderen Updates finden Sie in den entsprechenden SDK Changelogs.

- [Swift SDK 6.6.1](https://github.com/braze-inc/braze-swift-sdk/blob/main/CHANGELOG.md#661)
- [Web SDK 4.9.0](https://github.com/braze-inc/braze-web-sdk/blob/master/CHANGELOG.md#490)
- [Android SDK 28.0.0](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2800)
    - Die minimale SDK-Version wurde auf 21 (Lollipop) aktualisiert.
    - Die Funktionen von Feature Flags wurden geändert.
    - `Braze.getFeatureFlag(id)` gibt jetzt null zurück, wenn das Feature-Flag nicht existiert.
    - `Braze.subscribeToFeatureFlagsUpdates()` ruft nur dann zurück, wenn eine Aktualisierungsanfrage abgeschlossen ist, und zunächst, wenn zuvor zwischengespeicherte Daten vorhanden sind. Sie wird auch mit zwischengespeicherten Funktionsmerkmalen aufgerufen, falls die Aktualisierung fehlschlägt.
        - Wenn Sie den zwischengespeicherten Wert sofort beim Start der Anwendung haben möchten, verwenden Sie `Braze.getFeatureFlag(id)`.
    - Refactored `DefaultInAppMessageViewWrapper.createButtonClickListener()` in `DefaultInAppMessageViewWrapper.createButtonClickListeners()`.
- [React Native SDK 7.0.0](https://github.com/braze-inc/braze-react-native-sdk/blob/master/CHANGELOG.md#700)
    - Aktualisiert die native Android-Bridge von [Braze Android SDK 26.3.2 auf 27.0.1](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2701).
- [Cordova SDK 7.0.0](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2800)
    - Aktualisiert die native Android-Bridge von [Braze Android SDK 26.3.2 auf 27.0.1](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2701).
- [Roku SDK 2.0.0](https://github.com/braze-inc/braze-roku-sdk/blob/main/CHANGELOG.md#200)
    - `getFeatureFlag` gibt ungültig zurück, wenn das Flag nicht existiert.
    - `BrazeTask` beobachtet jetzt `BrazeFeatureFlagsUpdated`, um zu wissen, wann die Aktualisierungen von Feature Flags erfolgreich sind oder nicht. Die Datenwerte müssen nicht immer unterschiedlich sein.
