# Januar 10, 2023 Veröffentlichung

## Benutzeraktualisierungskomponente für Canvas Flow

Mit der Komponente [Benutzeraktualisierung]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/user_update/) können Sie die Attribute, Ereignisse und Käufe eines Benutzers in einem JSON-Composer aktualisieren, so dass Sie keine sensiblen Informationen wie API-Schlüssel eingeben müssen. 

## Einstellen von Abonnementgruppen über API

Bei der Erstellung neuer Benutzer über den [Endpunkt`/users/track` ]({{site.baseurl}}/api/endpoints/user_data/post_user_track/) können Sie Abonnementgruppen innerhalb des Benutzerattribut-Objekts festlegen. So können Sie in einem einzigen API-Aufruf einen Benutzer erstellen und den Status der Abonnementgruppe festlegen.

## Conversions Dashboard Early Access

Mit dem [Conversions-Dashboard]({{site.baseurl}}/user_guide/data_and_analytics/analytics/conversions_dashboard/) können Sie die Conversions über Kampagnen, Canvases und Kanäle hinweg analysieren und dabei verschiedene Attributionsmethoden verwenden. Sie können diese Attributionsmethoden gezielt verfolgen:

- **Öffnen Sie Konvertierungen:** Konversionen, die erfolgten, nachdem ein Benutzer die Nachricht geöffnet hatte
- **Klick-Konversionen:** Conversions, die erfolgten, nachdem ein Benutzer auf die Nachricht geklickt hat
- **Erhaltene Konvertierungen:** Konversionen, die nach dem Empfang der Nachricht durch einen Benutzer erfolgt sind
- **Konversionen beim letzten Klick:** Konversionen, die nach dem Anklicken der Nachricht durch den Benutzer stattgefunden haben, wenn die Nachricht die letzte war, die der Benutzer angeklickt hat (Diese Funktion wird derzeit mit einer kleinen Gruppe von Early-Access-Kunden getestet)

Diese Funktion befindet sich derzeit im Early Access. Wenn Sie daran interessiert sind, am Early Access teilzunehmen, wenden Sie sich bitte an Ihren Customer Success Manager.

## Canvas-Ausgangsereignisse für Braze Currents

Sie können jetzt nachverfolgen, wann Ihre Benutzer ein Canvas verlassen, indem sie entweder ein Ereignis ausführen oder eine Zielgruppe auswählen. Weitere Informationen finden Sie im Abschnitt [Veranstaltungen zum Thema Message Engagement]({{site.baseurl}}/user_guide/data_and_analytics/braze_currents/event_glossary/message_engagement_events/) im Currents Veranstaltungsglossar.

## SDK Aktualisierungen

Die folgenden SDK-Updates wurden veröffentlicht. Die wichtigsten Updates sind unten aufgeführt. Alle anderen Updates finden Sie in den entsprechenden SDK Changelogs.

- [Web SDK 4.5.1](https://github.com/braze-inc/braze-web-sdk/blob/master/CHANGELOG.md)
- [AppboyKit iOS SDK 4.5.2](https://github.com/Appboy/appboy-ios-sdk/releases/tag/4.5.2)
- [Swift SDK 5.8.0-5.8.1](https://github.com/braze-inc/braze-swift-sdk/blob/main/CHANGELOG.md#580)
	- Benennt die Klasse `BrazeLocation` in `BrazeLocationProvider` um, um das gleichnamige Modul nicht zu überschatten.
- [Flutter SDK 3.0.1](https://pub.dev/packages/braze_plugin/changelog)
- [Android SDK 24.0.0](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md)
	- Die Standort- und Geofence-Funktionalität wurde in ein neues Modul namens `com.braze:android-sdk-location` verlagert.
	- Appboy-Klassen und -Dateien wurden in großem Umfang nach Braze verschoben.
	- Das Standardverhalten von `DefaultContentCardsUpdateHandler` wurde dahingehend geändert, dass bei der Sortierung von Inhaltskarten die Erstellungszeit statt der letzten Aktualisierungszeit verwendet wird.
	- Entfernt BrazeUser.setFacebookData() und BrazeUser.setTwitterData().
