# Oktober 17, 2023 Veröffentlichung

## Kopieren in Arbeitsbereiche

[Das Kopieren von Kampagnen in einem Arbeitsbereich]({{site.baseurl}}/user_guide/engagement_tools/campaigns/managing_campaigns/copying_to_workspace/) ermöglicht es Ihnen, mit einer Kopie einer Kampagne in einem anderen Arbeitsbereich zu beginnen und so Ihre Nachrichten schneller zu verfassen. Dieser Text bleibt als Entwurf erhalten, bis Sie ihn bearbeiten und veröffentlichen. So können Sie Ihre erfolgreichen Kommunikationsstrategien beibehalten und ausbauen.

## Test Ströme Stecker

[Test Currents-Konnektoren]({{site.baseurl}}/user_guide/data_and_analytics/braze_currents/setting_up_currents/#test-currents-connectors) sind kostenlose Versionen unserer bestehenden Konnektoren, die Sie zum Testen und Ausprobieren verschiedener Ziele verwenden können. Testströme haben:

- Es gibt keine Begrenzung für die Anzahl der Test Currents-Anschlüsse, die Sie bauen können.
- Insgesamt maximal 10.000 Ereignisse pro rollierendem 30-Tage-Zeitraum. Diese Summe der Ereignisse wird stündlich auf dem Dashboard aktualisiert.

## Merkmal Flaggen

Mit [Funktionsflags]({{site.baseurl}}/developer_guide/platform_wide/feature_flags/about/) können Sie Funktionen für eine bestimmte oder zufällige Auswahl von Benutzern aus der Ferne aktivieren oder deaktivieren. Wichtig ist, dass Sie damit eine Funktion in der Produktion ein- und ausschalten können, ohne zusätzlichen Code zu verteilen oder den App Store zu aktualisieren. Auf diese Weise können Sie neue Funktionen sicher und zuverlässig einführen.

## Feature Flag Experimente

Mithilfe [von Experimenten mit Funktionskennzeichen]({{site.baseurl}}/developer_guide/platform_wide/feature_flags/experiments/) können Sie Änderungen an Ihren Anwendungen A/B-testen, um die Konversionsraten zu optimieren. Vermarkter können Feature Flags verwenden, um festzustellen, ob sich eine neue Funktion positiv oder negativ auf die Konversionsraten auswirkt oder welcher Satz von Feature Flag-Eigenschaften am optimalsten ist.

## Zusammenführen von Benutzerprofilen

Wenn Ihre Suche auf der Seite **Benutzer suchen** mehrere Benutzerprofile ergibt, können Sie [die Benutzerprofile zusammenführen]({{site.baseurl}}/user_guide/engagement_tools/segments/user_profiles#merge-profiles), indem Sie auf die Schaltfläche **Duplikate zusammenführen** klicken. Sie können auswählen, welches Benutzerprofil beibehalten werden soll. Das bedeutet, dass dieses Profil beibehalten wird und die Attribute des zusammengeführten Profils erhält.

## Leistungsdaten nach Segmenten

Sie können jetzt Berichtsvorlagen des Query Builders verwenden, um [Leistungsdaten]({{site.baseurl}}/user_guide/data_and_analytics/reporting/viewing_and_understanding_segment_data/#performance-data-by-segment) nach Segmenten für Kampagnen, Canvas, Varianten und Schritte aufzuschlüsseln.

## Benutzerprofile aktualisieren

Sie können jetzt den [Endpunkt`/users/track` ]({{site.baseurl}}/api/endpoints/user_data/post_user_track/) verwenden, um ein Benutzerprofil per Telefonnummer oder E-Mail zu aktualisieren.

## SDK Aktualisierungen
 
Die folgenden SDK-Updates wurden veröffentlicht. Die wichtigsten Updates sind unten aufgeführt. Alle anderen Updates finden Sie in den entsprechenden SDK Changelogs.
 
- [Braze Segment Swift Plugin v2.1.0](https://github.com/braze-inc/braze-segment-swift/blob/main/CHANGELOG.md)
- [Web SDK v4.10.0](https://github.com/braze-inc/braze-web-sdk/blob/master/CHANGELOG.md)
- [Web SDK v5.0.0](https://github.com/braze-inc/braze-web-sdk/blob/master/CHANGELOG.md)
    - Der [`subscribeToFeatureFlagsUpdates()`](https://js.appboycdn.com/web-sdk/latest/doc/modules/braze.html#subscribetofeatureflagsupdates) Callback wird jetzt immer aufgerufen, unabhängig davon, ob die Aktualisierung erfolgreich war oder nicht. Wenn der Empfang von Aktualisierungen fehlschlägt, wird der Callback mit den aktuell zwischengespeicherten Funktionsmerkmalen aufgerufen.
    - Die Methode [`getFeatureFlag()`](https://js.appboycdn.com/web-sdk/latest/doc/modules/braze.html#getfeatureflag) Methode gibt jetzt null zurück, wenn das Feature-Flag nicht existiert oder wenn Feature-Flags deaktiviert sind.
    - Die Methode `logContentCardsDisplayed()`, die in Version 4.0.4 veraltet war, wurde entfernt.
    - Die veraltete Initialisierungsoption `enableHtmlInAppMessages` wurde entfernt. Dies sollte stattdessen durch die Option `allowUserSuppliedJavascript` ersetzt werden.
    - Die Klasse Banner, die in 4.9.0 bereits veraltet war, wurde zugunsten von [`ImageOnly`](https://js.appboycdn.com/web-sdk/latest/doc/classes/braze.imageonly.html).
    - Der CSS-Klassenname `ab-banner` wurde im Rahmen der Entfernung der Klasse `Banner` entfernt. CSS-Anpassungen sollten stattdessen auf die Klasse `ab-image-only` abzielen.
    - Das SDK gibt keine Laufzeitfehler mehr aus. Wenn Braze-Methoden vor der Initialisierung aufgerufen werden, wird stattdessen eine Warnung auf der Konsole ausgegeben.
    - Das SDK fügt keine Standard-Braze In-App-Nachrichtenstile mehr zu benutzerdefinierten HTML In-App-Nachrichten hinzu. Diese Stile wurden zuvor von älteren In-App-Nachrichtentypen verwendet.
- [Android SDK 29.0.0](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md)
    - Umbenennung von `BannerImageCard`, `BannerImageCardView`, und `BannerImageContentCardView` in `ImageOnlyCard`, `ImageOnlyCardView` und `ImageOnlyContentCardView`.
    - Alle für Bannerkarten verwendeten Stile wurden zu Nur-Bild-Karten aktualisiert. Alle Tasten mit dem Wort `banner` sollten durch `image_only` ersetzt werden.
    - Die Informationen zur Gerätemarke werden jetzt gesendet. Wenn Sie dies verhindern möchten, lesen Sie den Abschnitt Blockieren der Datenerfassung.
- [Flutter SDK 7.0.0](https://pub.dev/packages/braze_plugin/changelog)
    - Aktualisiert die native Android-Bridge [von Braze Android SDK 26.1.1 auf 27.0.1](https://github.com/braze-inc/braze-android-sdk/blob/master/CHANGELOG.md#2701).
    - Fügt Unterstützung für Gradle 8 hinzu.
- [Swift SDK 7.0.0](https://github.com/braze-inc/braze-swift-sdk/blob/main/CHANGELOG.md)
    - Die Konfiguration `useUUIDAsDeviceId` ist jetzt standardmäßig aktiviert.
        - Weitere Einzelheiten zu den Auswirkungen finden Sie in diesem Artikel [Sammeln von IDFV - Swift]({{site.baseurl}}/developer_guide/platform_integration_guides/swift/initial_sdk_setup/swift_idfv/).
    - Der Typ Banner Content Card und die entsprechenden UI-Elemente wurden in `ImageOnly` umbenannt. Alle Methoden und Eigenschaften der Mitglieder bleiben gleich.
        - `Braze.ContentCard.Banner` → `Braze.ContentCard.ImageOnly`
        - `BrazeContentCardUI.BannerCell` → `BrazeContentCardUI.ImageOnlyCell`
    - Refaktoriert einige Textlayout-Logik in BrazeUI in eine neue Braze.ModalTextView Klasse.
    - Aktualisiert das Verhalten für Feature Flags Methoden.
        - `FeatureFlags.featureFlag(id:)` gibt jetzt null für eine ID zurück, die nicht existiert.
        - `FeatureFlags.subscribeToUpdates(:)` löst den Callback aus, wenn eine Aktualisierungsanfrage mit Erfolg oder Misserfolg abgeschlossen wird.
            - Der Callback wird auch sofort bei der ersten Anmeldung ausgelöst, wenn zuvor zwischengespeicherte Daten vorhanden sind.
- [AppboyKit iOS SDK 4.6.0](https://github.com/Appboy/appboy-ios-sdk/releases/tag/4.6.0)
    - Diese Version erfordert Xcode `14.x`.
    - Verzichtet auf die Unterstützung für iOS 9 und iOS 10.
    - Entfernt die Unterstützung für die veralteten `.framework` Assets beim Importieren über Carthage zugunsten der modernen `.xcframework` Assets.
        - Verwenden Sie den Befehl `carthage update --use-xcframeworks`, um das entsprechende Braze-Asset zu importieren.
        - Entfernt die Unterstützung für `appboy_ios_sdk_full.json` zu Gunsten der Verwendung von `appboy_ios_sdk.json`
