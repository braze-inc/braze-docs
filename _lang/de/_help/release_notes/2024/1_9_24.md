## Flexibilität der Daten

### Back-in-Stock-Benachrichtigungen für Kataloge

{% multi_lang_include release_type.md release="Früher Zugang" %}

Mit einer Kombination aus [Bestandsbenachrichtigungen]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/catalogs/back_in_stock_notifications/) über Kataloge und einem Canvas können Sie Kunden benachrichtigen, wenn ein Artikel wieder vorrätig ist. Jedes Mal, wenn ein Kunde ein ausgewähltes benutzerdefiniertes Ereignis ausführt, kann er automatisch benachrichtigt werden, wenn der Artikel wieder aufgefüllt wird.

### Katalog-Segmente

{% multi_lang_include release_type.md release="Früher Zugang" %}

[Katalogsegmente]({{site.baseurl}}/user_guide/engagement_tools/segments/sql_segments/catalog_segments/) sind eine Zielgruppe von Benutzern, die auf Katalogdaten in SQL Segment Extensions basieren. Diese SQL-Segmenterweiterungen können in einem Segment referenziert werden und dann von Kampagnen und Canvases angesprochen werden. Katalogsegmente verwenden SQL, um Daten aus Katalogen und Daten aus benutzerdefinierten Ereignissen oder Einkäufen zu verbinden. Dazu benötigen Sie ein gemeinsames Bezeichnungsfeld für Ihre Kataloge und Ihre benutzerdefinierten Ereignisse oder Einkäufe.

### Umstellung auf die Firebase Cloud Messaging API

{% multi_lang_include release_type.md release="Früher Zugang" %}

Erfahren Sie [, wie Sie]({{site.baseurl}}/developer_guide/platform_integration_guides/android/push_notifications/android/migrating_to_firebase_cloud_messaging/) von der veralteten Cloud Messaging API von Google auf die vollständig unterstützte Firebase Cloud Messaging (FCM) API [migrieren]({{site.baseurl}}/developer_guide/platform_integration_guides/android/push_notifications/android/migrating_to_firebase_cloud_messaging/).

## SDK Aktualisierungen

Die folgenden SDK-Updates wurden veröffentlicht. Die wichtigsten Updates sind unten aufgeführt. Alle anderen Updates finden Sie in den entsprechenden SDK Changelogs.

- [Swift SDK 7.5.0](https://github.com/braze-inc/braze-swift-sdk/blob/main/CHANGELOG.md)
    - Fügt Datenschutzmanifeste für `BrazeKit` und `BrazeLocation` hinzu, um die Datenerfassungsrichtlinien von Braze zu beschreiben. Weitere Einzelheiten finden Sie in der [Apple-Dokumentation](https://developer.apple.com/documentation/bundleresources/privacy_manifest_files/describing_data_use_in_privacy_manifests) zu Datenschutzmanifesten. Weitere Konfigurationen zur Verwaltung Ihrer Datenerfassungspraktiken werden in einer zukünftigen Version zur Verfügung stehen.
    - Behebt ein Problem mit den Codesignaturen von XCFrameworks, das in 7.1.0 eingeführt wurde.
- [Web SDK v5.1.0](https://github.com/braze-inc/braze-web-sdk/blob/master/CHANGELOG.md)
- [Unity SDK 5.0.0](https://github.com/braze-inc/braze-unity-sdk/blob/master/CHANGELOG.md)
    - Die native iOS-Bridge wurde von Braze Swift SDK 6.1.0 auf 7.4.0 aktualisiert.
        - Der Link zum iOS-Repository verweist jetzt auf die vorgefertigten dynamischen XCFrameworks aus diesem [Repository](https://github.com/braze-inc/braze-swift-sdk-prebuilt-dynamic).
    - Die native Android-Bridge wurde von Braze Android SDK 27.0.1 auf 29.0.1 aktualisiert.
    - `AppboyBinding.GetFeatureFlag(string id)` gibt nun `null` zurück, wenn das Feature-Flag nicht existiert.
    - `FEATURE_FLAGS_UPDATED` wird nur dann ausgelöst, wenn eine Aktualisierungsanfrage erfolgreich abgeschlossen wurde oder fehlgeschlagen ist, und bei der ersten Anmeldung, wenn zuvor Daten aus der aktuellen Sitzung zwischengespeichert wurden.
