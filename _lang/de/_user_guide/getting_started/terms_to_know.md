  - name: Aktiver Benutzer

  - name: A/B Multivariate Tests
    description: "Dabei handelt es sich um eine Art von Test, bei dem eine oder mehrere Variablen geändert und getestet werden, um festzustellen, welche Änderungen am besten funktionieren. <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/testing/multivariant_testing\">A/B-Tests</a> können in Dashboard-Kampagnen durchgeführt werden."
  - name: App-Instanzen
    description: "App-Instanzen referenzieren auf die verschiedenen Websites und Apps, die in einem Workspace gesammelt werden."
  - name: Technologie-Partner
    description: "Alloys sind unsere <a href=\"https://www.braze.com/docs/partners/home/\">Technologiepartner</a>."
  - name: Anonyme Benutzer
    description: "Wenn ein Nutzerprofil über das SDK erkannt wird, wird ein anonymes Nutzerprofil mit der zugehörigen <a href=\"https://www.braze.com/docs/api/basics/#user-ids\">Braze Nutzer:in</a> erstellt." 
  - name: Anwendungsprogramm-Schnittstelle (API)
    description: "Die <a href=\"https://www.braze.com/docs/api/basics/#api-overview\">Braze API</a> bietet einen Webdienst, mit dem Sie die Aktionen Ihrer Nutzer:innen direkt über HTTP aufzeichnen können, anstatt über die mobilen SDKs. So können Sie zum Beispiel Benutzerdaten an Braze weitergeben, die in Ihrer App oder Website nicht erfasst werden."
  - name: Braze (Das Produkt)
    description: "Dieses Produkt, das manchmal auch als Dashboard bezeichnet wird, steuert alle Daten und Interaktionen, die das Herzstück der Braze-Plattform bilden. Braze-Kunden verwenden es, um Benachrichtigungen zu verwalten, gezielte Messaging-Kampagnen einzurichten und Analysen einzusehen. Entwickler:in verwalten damit Einstellungen für die Integration von Apps, wie API-Schlüssel und Zugangsdaten für Push-Benachrichtigungen."
  - name: Braze Engagement Tools
    description: "Braze bietet zahlreiche <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/\">Tools zur Einbindung</a>, um Ihre Kunden und Nutzer mit unseren Kampagnen und Canvas-Tools zu erreichen. Außerdem können Sie mit unseren Templates &amp; Media-Tools die Konsistenz optimieren (und Bilder und andere Inhalte hochladen). Von dort aus können Sie Segmente und Geofences erstellen, um Ihre Zielgruppe nach Standorten oder anderen Attributen zu adressieren."
  - name: Lötende Teams
    description: "Braze-Administratoren können eine Teilmenge der Nutzer:innen des Dashboards in <a href=\"https://www.braze.com/docs/user_guide/administrative/app_settings/manage_your_braze_users/teams/\">Teams</a> mit unterschiedlichen Nutzer:innen-Rollen und Berechtigungen einteilen. Dies erlaubt es Braze-Administratoren, den Zugriff auf bestimmte Features nach Gruppenmitgliedschaft einzuschränken."
  - name: Kampagnen
    description: "Kampagnen sind angepasste Messaging-Methoden, um Ihren Kund:innen eine personalisierte Antwort zu liefern. Sie können über verschiedene Messaging-Kanäle <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/campaigns/\">Kampagnen erstellen</a>, um Ihre eindeutigen Nachrichten zu versenden." 
  - name: Canvas
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/canvas/\">Canvas</a> ist eine einheitliche Schnittstelle, auf der Marketer Kampagnen mit mehreren Nachrichten und Schritten einrichten können, um eine zusammenhängende Reise zu gestalten. Canvas erlaubt es Ihnen, diese Erfahrungen zu vergleichen und zu optimieren, indem Sie umfassende Analytics für das gesamte Nutzer:innen-Erlebnis nutzen."
  - name: Connected-Content
    description: "<a href=\"https://www.braze.com/docs/user_guide/personalization_and_dynamic_content/connected_content/\">Connected Content</a> erweitert die Personalisierung des Marketings, um die Kundenbindung und die Konversionsrate zu erhöhen. Sie können alle über API zugänglichen Informationen direkt in Nachrichten einfügen, die Sie an Nutzer:innen senden. Mit Connected-Content können Sie Inhalte entweder direkt von Ihrem Webserver oder über öffentlich zugängliche APIs beziehen."
  - name: Content-Cards
    description: "Mit <a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/content_cards/about/\">Content Cards</a> können Sie Ihren Kunden direkt in den Apps, die sie lieben, einen gezielten, dynamischen Strom reichhaltiger Inhalte senden, ohne dass diese ihr Erlebnis unterbrechen. Content Cards können an iOS-, Android- und Web-Benutzer gesendet werden."
  - name: Currents
    description: "<a href = \"https://www.braze.com/docs/user_guide/data/braze_currents/\">Currents</a>, unser Daten-Streaming-Export, ist in bestimmten Braze-Paketen enthalten. Braze Currents ermöglicht Ihnen die Integration durch Datenspeicherung unter Verwendung von Flat Files oder durch unsere Partner für Verhaltensanalyse und Kundendaten unter Verwendung von gebündelten JSON-Payloads an einen bestimmten Endpunkt."
  - name: Angepasste Attribute
    description: "<a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_attributes/\">Benutzerdefinierte Attribute</a> sind eine Sammlung der einzigartigen Eigenschaften Ihrer Benutzer. Sie eignen sich am besten zum Speichern von Attributen über Ihre Benutzer oder von Informationen über geringwertige Aktionen innerhalb Ihrer Anwendung. Sie können Benutzern innerhalb des Dashboards benutzerdefinierte Attribute zuweisen. Sie können Ihre Nutzer nach diesen Attributen filtern und segmentieren, sowohl für <a href=\"https://www.braze.com/docs/developer_guide/platforms/swift/analytics/setting_custom_attributes/\">iOS-</a> als auch für <a href=\"https://www.braze.com/docs/developer_guide/platforms/android/analytics/setting_custom_attributes/\">Android-Kampagnen</a>."
  - name: Angepasste Events
    description: "<a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_events\">Angepasste Events</a> sind Aktionen, die von Ihren Nutzer:innen ausgeführt werden. Sie eignen sich am besten für das Tracking hochwertiger Nutzer:innen-Interaktionen mit Ihrer Anwendung."
  - name: Datenpunkt
    description: "Ein Datenpunkt wird gezählt, wenn ein <a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_attributes/\">angepasstes Attribut</a> gesetzt oder aktualisiert wird (auch wenn Sie es mit demselben Wert aktualisieren), ein <a href=\"https://www.braze.com/docs/user_guide/data/custom_data/custom_events\">angepasstes Event</a> oder Kauf-Event protokolliert wird, beliebige Standarddaten (z.B., <code>email</code>, <code>first_name</code>, <code>last_name</code>, <code>country</code>, oder <code>home_city</code>) protokolliert wird, wann eine Sitzung beginnt und wann eine Sitzung endet."
  - name: Deeplinking
    description: "<a href=\"https://www.braze.com/docs/user_guide/personalization_and_dynamic_content/deep_linking_to_in-app_content/\">Deeplinks</a> werden verwendet, um Kund:innen zu ihrer nächsten Aktion oder ihrem nächsten Engagement zu führen. Mit Deeplinks können Sie eine Nachricht mit einem gezielten Inhalt auf einer Website oder in einer mobilen App verbinden."
  - name: "Inaktive Nutzer:innen"
    description: "Ein Benutzer gilt als <a href=\"https://www.braze.com/docs/user_archival/#dormant-users\">inaktiv</a>, wenn in den letzten zwölf Monaten keine Aktivität stattgefunden hat."
  - name: Endpunkt
    description: "Ein Endpunkt eines Kommunikationskanals, der auch als <a href=\"https://www.braze.com/docs/api/endpoints/\">API-Endpunkt</a> bezeichnet wird, wird innerhalb der Messaging-API von Braze zum Senden und Planen von Nachrichten verwendet."
  - name: Ausnahme-Events
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/canvas/create_a_canvas/exit_criteria/#exception-events\">Ausnahmeereignisse</a> treten auf, wenn ein Benutzer das gewünschte Ziel erreicht hat und keine ausgelöste Nachricht erhält. Auf diese Weise ist die getriggerte Nachricht für den Nutzer:innen immer noch relevant."
  - name: Frequency-Capping
    description: "<a href=\"https://www.braze.com/docs/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/#frequency-capping\">Frequency-Capping</a> ermöglicht es Ihnen, die Kommunikation zu steuern, ohne Ihre Zielgruppe zu überfordern. Es handelt sich um eine automatische Begrenzung der Nachrichten, um zu verhindern, dass Nutzer:innen zu viele Nachrichten in kurzer Zeit erhalten."
  - name: HIPAA
    description: "HIPAA ist eine Abkürzung für Health Insurance Portability and Accountability Act (US-Gesetz zum Schutz medizinischer Daten). Braze ist <a href=\"https://www.braze.com/docs/developer_guide/disclosures/security_qualifications/#hipaa\">HIPAA-konform</a>. Die HIPAA-Anforderungen umfassen administrative, physische und technische Sicherheit."
  - name: In-App-Nachrichten
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/in-app_messages/\">In-App-Nachrichten</a> sind mobile Nachrichten, die innerhalb Ihrer Anwendung erscheinen. Sie helfen Ihnen, Inhalte an Ihre Nutzer zu übermitteln, ohne sie mit einer Push-Benachrichtigung zu unterbrechen. Angepasste und maßgeschneiderte In-App-Nachrichten verbessern das Nutzererlebnis und helfen Ihrer Zielgruppe, den größten Nutzen aus Ihrer App zu ziehen."
  - name: Inaktive Benutzer
    description: "Ein Nutzer:innen gilt als <a href=\"https://www.braze.com/docs/user_archival/#inactive-users\">inaktiv</a>, wenn er bestimmte Kriterien erfüllt, z.B. keine E-Mails oder Push-Nachrichten mehr erhält und seit mehr als sechs Monaten nicht mehr aktiv war."
  - name: IP Erwärmung

Dies trägt dazu bei, einen guten Ruf bei den Serviceleistungen; Diensten aufzubauen und die Wahrscheinlichkeit zu minimieren, dass Ihre Nachrichten markiert werden."
  - name: Schlüssel-Werte-Paare

Sie können verwendet werden, um zusätzliche Daten an Nutzer:innen zu senden."
  - name: Liquid
    description: "Liquid ist eine weit verbreitete, von Shopify entwickelte und in Ruby geschriebene Template-Sprache für Kund:in. <a href=\"https://www.braze.com/docs/user_guide/personalization_and_dynamic_content/liquid/\">Liquid</a> wird zum Laden und Abrufen dynamischer Inhalte verwendet. Liquid erlaubt Ihnen die Verwendung von Objekten, Tags und Filtern, um <a href=\"http://www.braze.com/docs/user_guide/personalization_and_dynamic_content/liquid/supported_personalization_tags/\">eine persönliche Anpassung vorzunehmen</a>."
  - name: Messaging-Kanal
    description: "<a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/\">Messaging-Kanäle</a> bieten Ihnen die Möglichkeit, virtuell mit Ihren Kund:innen zu kommunizieren - über Push-Benachrichtigungen auf dem Telefon oder im Webbrowser, E-Mails, In-App-Nachrichten und vieles mehr!"
  - name: "Monatlich aktive Nutzer:innen (MAU)"

  - name: Multichannel-Messaging
    description: "Messaging einer Nutzerin oder eines Nutzers  über verschiedene Medien, z. B. eine Kombination aus E-Mail, Web-Push und mobilen Push-Benachrichtigungen. <a href=\"https://www.braze.com/docs/developer_guide/getting_started/platform_overview/#multichannel-messaging\">Messaging-Kanäle</a> werden am besten gemeinsam und regelmäßig genutzt, um verlorene Nutzer:innen wieder zu engagieren, aktive Nutzer:innen zu halten und Ihre Markenbotschafter zu aktivieren."
  - name: Multivariate Tests
    description: "Mit <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/testing/multivariant_testing/#multivariate-test\">multivariaten Tests</a> können Sie mehrere Variablen testen, indem Sie zwei oder mehr Versionen einer Nachricht vergleichen, um festzustellen, welche Variante am besten abschneidet."
  - name: "Neue:r Nutzer:in"

  - name: Personalisierung

  - name: Push-Nachrichten

Push-Benachrichtigungen erscheinen oft als Popup-Dialoge und Banner für iOS und Android."
  - name: Push-Token

  - name: Push Time to Live
    description: "Die auch als <a href=\"https://www.braze.com/docs/user_guide/administrative/app_settings/push_ttl_settings/\">Push-TTL</a> bezeichnete Zeit bis zur Auslieferung bezieht sich auf den Zeitraum, in dem Kampagnen weiterhin versuchen, einem Offline-Nutzer zugestellt zu werden."
  - name: Race-Condition
    description: "Eine <a href=\"http://www.braze.com/docs/user_guide/engagement_tools/testing/race_conditions/\">Race Condition</a> ist ein Konzept aus der Softwaretechnik, das eine unerwünschte Situation beschreibt, die auftritt, wenn ein System versucht, mehrere Operationen gleichzeitig auszuführen. Aufgrund der Beschaffenheit des Systems müssen die Operationen jedoch in der richtigen Reihenfolge ausgeführt werden, um korrekt zu funktionieren. <br><br>In der Braze-Plattform kann die Segmentierung einer getriggerten Kampagne anhand von Nutzerdaten, die zum Zeitpunkt des Events aufgezeichnet wurden, eine Race-Condition verursachen. Dies geschieht, wenn eine Änderung des Attributs der Nutzerin oder des Nutzers, auf dessen Grundlage die Kampagne segmentiert wird, zum Zeitpunkt der Bestimmung der Segmentzugehörigkeit und des Versands der Kampagne noch nicht die:den Nutzer:in verarbeitet wurde und dazu führen kann, dass die:der Nutzer:in die Kampagne nicht erhält."
  - name: Rate-Limiting
    description: "Bei <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/campaigns/building_campaigns/rate-limiting/\">der Ratenbegrenzung</a> wird die Anzahl der innerhalb eines bestimmten Zeitraums gesendeten Nachrichten begrenzt, um eine Überlastung Ihres Publikums zu vermeiden."  
  - name: Tags
    description: "<a href=\"https://www.braze.com/docs/user_guide/administrative/app_settings/tags/\">Tags</a> sind ein Werkzeug, mit dem Sie Ihr Engagement über eine oder mehrere Kampagnen hinweg kategorisieren, organisieren und sortieren können."
  - name: Segmentierung
    description: "Mit <a href=\"https://www.braze.com/docs/user_guide/engagement_tools/segments/\">der Dashboard-Segmentierung</a> können Sie Gruppen oder Erweiterungen von Nutzern:innen auf der Grundlage leistungsstarker Filter für deren In-App-Verhalten, demografische Daten und mehr erstellen."
  - name: Software-Entwicklungskit (SDK)
    description: "<a href=\"https://www.braze.com/docs/developer_guide/getting_started/sdk_overview/\">SDKs</a> werden in iOS- und Android-Anwendungen integriert und bieten leistungsstarke Marketing-, Kundensupport- und Analysetools. Braze bietet SDK-Integration sowohl für <a href=\"https://www.braze.com/docs/developer_guide/platforms/swift/sdk_integration/#integrating-the-sdk/\">iOS</a> als auch für <a href=\"https://www.braze.com/docs/developer_guide/platforms/android/sdk_integration/#step-1-configure-brazexml\">Android</a>."
  - name: Sonnenuntergang
    description: "Sunsetting bezieht sich auf den Prozess der Identifizierung von Nutzern, die sich nicht mehr engagieren, und die Beendigung der aktiven Nachrichtenübermittlung an diese Nutzer, ohne dass diese etwas unternehmen müssen. Die Erstellung von Sunset-Richtlinien für Ihre <a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/email/best_practices/sunset_policies/\">E-Mails</a> und <a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/push/best_practices/sunset_policies/#sunset-policies-for-push/\">Push-Nachrichten</a> kann dazu beitragen, die Auswirkungen auf Ihre Öffnungsraten einzudämmen." 
  - name: Benutzer Archivierung
    description: "<a href=\"https://www.braze.com/docs/user_archival/\">Nutzerarchivierung</a> referenziert auf Nutzer:innen, die archiviert wurden. Bei Braze umfasst dies sowohl inaktive als auch ruhende Benutzer."
  - name: Webhooks
    description: "Mit <a href=\"https://www.braze.com/docs/user_guide/message_building_by_channel/webhooks/\">Webhooks</a> können Sie Nicht-App-Aktionen wie den Versand von SMS-Nachrichten auslösen. Sie können Webhooks verwenden, um anderen Systemen und Anwendungen Realtime-Informationen zur Verfügung zu stellen. Die Flexibilität dieses Features erlaubt es Ihnen, Informationen an jeden beliebigen Endpunkt zu senden."
