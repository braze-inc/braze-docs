# 2025年2月4日リリース

## BrazeAI<sup>TM</sup>

### コンテンツオプティマイザー

{% multi_lang_include release_type.md release="Beta" %}

[Content Optimizerは]({{site.baseurl}}/user_guide/brazeai/content_optimizer)、自動エンゲージメント最適化を実現する、継続的でバリアント率の高いコンテンツテストのキャンバスステップである。メッセージステップに似たドラッグ＆ドロップ可能なインターフェイスを使って、テストしたいコンポーネントを定義し、AIを使ってバリアントを生成し（または手動で入力）、Liquidタグを使ってこれらのコンポーネントをメッセージコンテンツにマッピングすることができる。

非文脈に応じたor 状況に即したマルチアームドバンディットオプティマイザーをベースに構築されたContent Optimizerは、ユーザーごとに単一のメッセージを送信し、予測レコメンデーションに基づいて配信するコンポーネントバリアントの組み合わせを決定する。時間の経過とともにステップがデータを収集するにつれて、パフォーマンスの高いバリアントは自然に送信割り当てが増え、パフォーマンスの低いバリアントは減っていく。Content Optimizerは、継続的な最適化を可能にするために、毎日のユーザー数が安定している（少なくとも1日あたり数千人）反復送信キャンバスで最も効果的に機能する。

## チャネル & タッチポイント

### バナーのローカライゼーションを翻訳する

{% multi_lang_include release_type.md release="Early access" %}

ワークスペースにローカライゼーションを追加すれば、1つのバナーで[さまざまな言語のユーザーをターゲットに]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/localization/locales#translating-locales)できる。

### ドラッグ＆ドロップのコンテンツブロックの幅を設定する

ナビゲーションメニューのボタンを選択し、[コンテンツブロックの幅を調整する]({{site.baseurl}}/user_guide/message_building_by_channel/email/drag_and_drop/dnd_content_blocks/#using-the-editor-to-add-a-content-block)。メールのグローバルスタイル設定で指定されていない場合、デフォルトの幅は100%である。

![幅を編集するオプションを持つ両面矢印。]({% image_buster /assets/img_archive/content_block_width_updated.png %}){: style="max-width:30%;" }

### 自動IPウォーミングアップを使う

{% multi_lang_include release_type.md release="Early access" %}

[自動IPウォーミングを使って]({{site.baseurl}}/user_guide/message_building_by_channel/email/email_setup/ip_warming/#automated-ip-warming)、1日の送信量を徐々に増やすことで、受信トレイプロバイダーはあなたの送信パターンを学習し、信頼することができる。Brazeは、最もエンゲージメントの高いサブスクライバーに最初に配信することで、日々の配信量をベストプラクティスに合ったペースで増やすことができる。

## パートナーシップ:

### LinkedIn - キャンバスオーディエンスシンク

[Braze Audience Sync to LinkedInを]({{site.baseurl}}/partners/canvas_audience_sync/linkedin_audience_sync/)使用すると、BrazeインテグレーションからLinkedIn顧客リストにユーザーデータを追加し、行動トリガーやセグメンテーションなどに基づいて広告を配信することができる。ユーザーデータに基づいてBrazeキャンバスでメッセージ（プッシュ、メール、SMS、Webhookなど）をトリガーするために通常使用するあらゆる基準が、LinkedIn顧客リストのそのユーザーへの広告をトリガーできるようになりました。

### Oracle Crowdtwist - データ＆分析& 。

[Oracle Crowdtwistは]({{site.baseurl}}/partners/crowdtwist)、ブランドがパーソナライズされたカスタマーエクスペリエンスを提供できるようにする、クラウドネイティブなカスタマー・ロイヤルティ・ソリューションのリーディング・カンパニーである。同社のソリューションは、100以上の既成概念にとらわれないエンゲージメントパスを提供し、マーケターがより完全な顧客ビューを開発するための迅速なTime-to-Valueを提供する。

### フルストーリー - ダイナミックなコンテンツ

[フルストーリーの]({{site.baseurl}}/partners/fullstory/)行動データプラットフォームは、テクノロジー・リーダーがより良い情報に基づいた意思決定を行えるよう支援する。デジタル行動データを分析スタックに注入することで、Fullstoryの特許取得済みテクノロジーは、質の高い行動データのパワーを大規模に引き出し、すべてのデジタル訪問をアクション可能なインサイトに変換する。 

### 開封ロイヤルティ - データ＆分析& 。

[開封ロイヤリティは]({{site.baseurl}}/partners/openloyalty)クラウドベースのロイヤルティプログラムプラットフォームで、顧客ロイヤルティプログラムや報酬プログラムを構築・管理することができる。BrazeとOpen Loyaltyの統合は、ポイント残高、ティアの変更、有効期限警告などのロイヤルティデータをリアルタイムでBrazeに直接同期する。これにより、ユーザーのロイヤルティステータスが変化したときに、パーソナライズされたメッセージ（メール、プッシュ、SMS）をトリガーすることができる。

### エクステンション

[DOTS.ECO]({{site.baseurl}}/partners/docs.eco) は、追跡可能なデジタル証明書を通じて、ユーザーに実際の環境影響を報酬として与えることができる。各証明書には、共有可能な証明書URLや画像写真URLなどのメタデータを含めることができるため、ユーザーは影響力の証明を閲覧（および再確認）することができる。

### Mailizio - メッセージオーケストレーション

[Mailizioは]({{site.baseurl}}/partners/mailizio/)、直感的なビジュアルエディタを使って、再利用可能でブランドセーフなコンテンツを簡単にデザインできるメール作成・管理プラットフォームである。MailizioとBrazeの統合により、コンテンツブロックとメールテンプレートをエクスポートし、同じアセットからアプリ内メッセージを自動的に生成することができ、迅速かつ完全にコントロールされたキャンペーン展開が可能になる。

## 百花繚乱とデータシェア

### ストレージ送信先とデータシェアのエージェントコンソールイベント

{% multi_lang_include release_type.md release="General availability" %}

Storage送信先（AWS S3、GCS、Azure Blob Storage）とSnowflake Datashareで、`agentconsole.AgentExecuted` と`agentconsole.ToolInvocation` の2つの新しい[イベントが](http://braze.com/docs/user_guide/data/distribution/braze_currents/event_glossary/customer_behavior_events)利用可能になった。これらのイベントにより、Agent Consoleの使用状況とダウンストリームシステムでの詳細を分析することができ、エージェントの使用状況を理解し、最大限に活用することができる。エージェントを使用すると、キャンバスやカタログにコンテンツを生成したり、インテリジェントな意思決定に基づいてユーザーを別のパスにルーティングしたりするなど、Braze全体で特定のタスクを実行できるインテリジェントエージェントを作成およびデプロイできる。詳細は、[Currentsの変更履歴を](https://www.braze.com/docs/user_guide/data/distribution/braze_currents/event_glossary/currents_changelogs#changes-in-version-5-release-date-2026-02-04)参照のこと。

### 各チャネルの新しい「再試行」イベント

{% multi_lang_include release_type.md release="General availability" %}

メール、LINE、プッシュ通知、SMS、Webhooks、WhatsAppチャネルで新しい[リトライイベントが](https://www.braze.com/docs/user_guide/data/distribution/braze_currents/event_glossary/message_engagement_events)利用可能になった。これらのイベントは、フリークエンシーキャップによって、スケジュールされたメッセージが中止されるのではなく、遅延される場合の可視性を提供する。メッセージが優先順位を下げられたり、フリークエンシーキャップされた場合、設定されたリトライウィンドウ内でリトライできるようになり、メッセージ配信パターンやフリークエンシーキャップが与える影響について、より良いインサイトが得られるようになった。詳細は、[Currentsの変更履歴を](https://www.braze.com/docs/user_guide/data/distribution/braze_currents/event_glossary/currents_changelogs#changes-in-version-5-release-date-2026-02-04)参照のこと。

### TokenStateChangeイベントに新しいフィールド（'time_ms' ）を追加する。

{% multi_lang_include release_type.md release="General availability" %}

イベントに新しいフィールド`time_ms` が追加された。 [`users.behaviors.pushnotification.TokenStateChange`](https://www.braze.com/docs/user_guide/data/distribution/braze_currents/event_glossary/customer_behavior_events)イベントに新しいフィールドが追加され、プッシュトークンの状態変化をトラッキングするためのミリ秒レベルの粒度を提供する。この精度の向上により、同じ秒数内に複数の変更が発生した場合でも、プッシュトークンの最新のステータスを理解することができ、ダウンストリームシステムで正しいサブスクリプションステータスを確信してもらうことができる。詳細は、[Currentsの変更履歴を](https://www.braze.com/docs/user_guide/data/distribution/braze_currents/event_glossary/currents_changelogs#changes-in-version-5-release-date-2026-02-04)参照のこと。

### 匿名ユーザーをTealium送信先に送る

{% multi_lang_include release_type.md release="General availability" %}

外部ユーザーIDが定義されていないイベントを[Tealium]({{site.baseurl}}/partners/data_and_analytics/customer_data_platform/tealium/tealium_for_currents?redirected=1#tealium-for-currents)送信先にストリーミングできるようになった。Currents統合で「匿名ユーザーからのイベントを含める」チェックボックスを選択すると、外部ユーザーIDを持たないイベントは抑制される代わりに送信先に送信される。この機能は、下流の分析や、識別子や匿名でないユーザーを含むユースケースには不可欠である。

#### 匿名ユーザーをCustomHTTP送信先に送る

{% multi_lang_include release_type.md release="Beta" %}

外部ユーザーIDが定義されていないイベントをCustomHTTP送信先にストリーミングできるようになった。Currents統合で「匿名ユーザーからのイベントを含める」チェックボックスを選択すると、外部ユーザーIDを持たないイベントは抑制される代わりに送信先に送信される。この機能は、下流の分析や、識別子や匿名でないユーザーを含むユースケースには不可欠である。

### メール開封イベント -"machine_open" フィールド

[メール開封イベントは]({{site.baseurl}}/user_guide/data/distribution/braze_currents/event_glossary/message_engagement_events#email-open-events)、"machine_open" フィールド値を生成するようになった。 [_開封_]({{site.baseurl}}/user_guide/analytics/reporting/report_metrics#machine-opens)メトリックをレポートできる。
