# アカウントにアクセスする

> この記事では、Brazeアカウントの取得方法、アクセス許可後のログイン方法、ダッシュボードへのアクセスとダッシュボードのパフォーマンスのトラブルシューティング方法について説明する。

会社の最初の Braze ユーザーが初めてログインする場合、契約初日に `@alerts.braze.com` からメールの確認とログインを求めるウェルカムメールが届きます。

アカウントを確認したら、ダッシュボードの [[会社ユーザー]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/)] ページから他のユーザーを追加できます。追加されたすべてのユーザーに、アカウントの確認を求めるメールが届きます。

御社のBrazeアカウントの最初のユーザーでない場合は、御社のBrazeアカウント管理者に連絡し、アカウントの作成を依頼する。その後、`@alerts.braze.com` からメールの確認とログインを求めるウェルカムメールが届きます。

## ログイン

初回であれ、100万回目であれ、ログインする方法について話そう！会社の最初のユーザーの場合は、前のセクションのガイダンスに従ってください。そうでない場合は、会社の Braze 管理者がアカウントを作成した後にログインできます。

[Braze.com](https://www.braze.com) のホームサイトからログインするか、特定の [Braze インスタンス]({{site.baseurl}}/user_guide/administrative/access_braze/sdk_endpoints/)に対応するダッシュボード URL を使用する方法もあります。便利なように、Braze には以下のようないくつかのシングルサインオン (SSO) オプションがあります。

* [SAML SSO]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/set_up/)
    * [SAMLジャストインタイムプロビジョニング]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/saml_jit/)
* [Microsoft Entra SSO]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/entra/)
* [Okta]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/okta/)
* [OneLogin]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/onelogin/)

{% alert note %}
SSOでBrazeにログインした後、パスワードを使ってダッシュボードにログインすることはできなくなる。
{% endalert %}

## サポートされているブラウザー

Braze ダッシュボードは次のブラウザーをサポートしています。
- Chrome（バージョン87以降）
- Firefox（バージョン85以降）
- Safari (バージョン 15.4 以降)
- Edge (バージョン 87 以降)

ダッシュボードに予期せぬエラーが発生したと表示され、ブラウザのコンソールツールにエラー`ReferenceError: structuredClone is not defined` が表示されている場合、ブラウザが古い。このエラーが繰り返される場合は、ブラウザをアンインストールして再インストールする。

## 複数のBrazeダッシュボードにアクセスする

Brazeでは、同一クラスター内（例えばUS-01に2つのダッシュボードがある場合）で、同じメールアドレスを複数のダッシュボードユーザーに登録することはできない。同じメールで異なるクラスターにアカウントを作成できる（例えば、US-01に1つのダッシュボードがあり、US-05にもう1つある場合など）。同じクラスター内の複数のBrazeダッシュボードにアクセスする必要がある場合、以下の方法がある：

### メールエイリアスを使う

メールプロバイダーがGmailの場合、メールアドレスにアスタリスク`+`（*）と任意の文字列を追加することでエイリアスを作成できる。以下に例を示します。
- **元のメール：** `rocky@gmail.com`
- **別名メール：** `rocky+1@gmail.com`

両方のメールアドレスは同じ受信トレイにメールを転送するが、Brazeにログインする際には別々のアカウントとして認識される。

### 他のプロバイダーとは別のエイリアスを作成する

メールプロバイダがエイリア`+`スをサポートしていない場合でも、別々のエイリアスを作成できる。例えば、をに自動転送するように`rocky@braze.com`設定する`rocky.lotito@braze.com`といった方法がある。これにより、複数のアドレスが同じ受信トレイに集約されつつ、Brazeからは別々のメールとして認識される。

### 複数企業の開発者を利用する

複数企業向け開発者機能は、単一のユーザーアカウントを複数の企業間で共有することを可能にする。ユーザーは自身のユーザープロファイルメニューから、異なる会社のダッシュボードを切り替えることができる。

シングルサインオン（SSO）を利用していて、複数会社の開発者を設定したい場合、カスタムSAML SSO統合を設定することでSAMLカスタムエンティティIDのイネーブルメントを行う必要がある。[サービスプロバイダー（SP）が開始するログイン]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/set_up/)のステップに従うが、以下の変更を適用する：
- 各ダッシュボード統合において`braze_dashboard_<companyID>`**エンティティID**を変更せよ。
- 各ダッシュボードの機能`saml_sso_custom_entity_id`切り替え機能をイネーブルメントするには、顧客サクセスマネージャーまたはアカウントマネージャーに連絡すること。

### シングルサインオン（SSO）に関する考慮事項

シングルサインオン（SSO）を使用する場合、複数の異なるメールアドレスを持っていると問題が生じる可能性があることに注意せよ。アクセス問題を防ぐため、SSO設定が正しく構成されていることを確認せよ。

## トラブルシューティング

### パスワードのリセット

パスワードをリセットするには、ダッシュボード・ログイン・ページで「**パスワードをお忘れですか？**パスワードをリセットするリンクを受け取るメールアドレスの入力が求められます。

![ダッシュボードのログイン画面に「パスワードを忘れた場合」の案内が表示される。]({% image_buster /assets/img_archive/enable_reset.png %}){: style="max-width:60%"}

### ブラウザのキャッシュと Cookie のクリア

ダッシュボードやセグメントのパフォーマンスリストが読み込まれないなど、ダッシュボードのパフォーマンスに問題がある場合は、各ブラウザのステップに従って、ブラウザのキャッシュとCookieをクリアしてみてください。

{% alert important %}
Cookieをクリアするとログアウトするので、未保存の作業は失われる。
{% endalert %}

- [Chromeでキャッシュ&とCookieを消去する](https://support.google.com/accounts/answer/32050?hl=en&co=GENIE.Platform%3DDesktop)
- [MacのSafariでCookieを消去する](https://support.apple.com/en-gb/guide/safari/sfri11471/16.1/mac/13.0)
- [FirefoxでCookieとサイトデータを消去する](https://support.mozilla.org/en-US/kb/clear-cookies-and-site-data-firefox)
- [Microsoft EdgeですべてのCookieを削除する](https://support.microsoft.com/en-us/windows/manage-cookies-in-microsoft-edge-view-allow-block-delete-and-use-168dab11-0753-043d-7c16-ede5947fc64d#bkmk_deleteallcookies)

ブラウザのキャッシュとCookieをクリアしても問題が解決しない場合は、[サポートに]({{site.baseurl}}/support_contact/)連絡してください。

### ドラッグ＆ドロップエディターへのアクセス

ほとんどの企業ユーザーには、ドラッグ＆ドロップエディタが読み込まれるはずだ。しかし、VPN を使用していたり、ファイアウォールで保護されている場合は、ドメインを許可リストに追加する必要があることがあります。IT管理者に連絡し、`*.bz-rndr.com` が許可リストに登録されていることを確認する。

エディターでは、以下のエラーが原因で読み込みの問題が発生することがあります。

- **一時的なエラー:**接続、通信、データ転送に影響する可能性がある一時的な障害です。このような障害は多くの場合、短期間の状況が原因で引き起こされ、体系的な問題を示すものではないため、大規模な介入を必要とせず、自然に解決することが一般的です。
- **大規模なエラー:**これはインフラや製品の問題に関係している可能性があります。 このような状況に気付いて積極的に解決に取り組む際に、[Braze のシステムステータスのページ](https://braze.statuspage.io/)を確認できます。

{% alert important %}
それでも問題が解決しない場合は、[サポートチケットを開封する]({{site.baseurl}}/user_guide/administrative/access_braze/support/)。その前に、`*.bz-rndr.com` がユーザー側で許可リストに追加されていることを IT 管理者が確認していることを確認してください。
{% endalert %}

### Braze学習へのアクセス

Braze学習へのログインで問題が発生し、ダッシュボードにリダイレクトされるループに陥った場合は、以下のステップを実行する：

1. 複数のBrazeアカウントを持っている場合、間違ったアカウントで2回ログインすると、Brazeのダッシュボードに送られる。正しいアカウントにログインしていることを確認せよ。 
2. 広告ブロック機能を使っているなら、それがオフになっていることを確認しろ。シングルサインオン機能に必要なCookieをブロックする可能性がある。
3. 会社設定＞セキュリティ設定に移動し、シングルサインオン（SSO）が有効になっていることを確認する。
4. ダッシュボードのユーザープロファイルに、名と苗字の両方が含まれていることを確認せよ。名字がないと、ログイン手続きが妨げられることがある。
5. ダッシュボードから**サポート**＞**Braze学習**の順に進むことで、Braze学習にアクセスできる。 
6. 問題が解決しない場合は、アカウントを再作成することを検討せよ。無料トライアル期間中にBraze Learningにアクセスしたユーザーは、現在アクセスに問題が生じている可能性がある。

### 2 要素認証（2FA）の問題

ユーザーが2 要素認証に問題を抱えており、Braze ダッシュボードにアクセスできない場合、いくつかの理由が考えられる。最もよくあるケースでは、登録済みの電話番号やAuthyアプリがインストールされた端末にアクセスできなくなることがある。

管理者は、影響を受けたユーザーの2段階認証をリセットする必要がある。その手順は以下の通りだ： 

1. **ユーザー管理**へ移動する。
2. 2FAの問題が発生しているユーザーに対して**「ユーザー編集」**を選択する。
3. 2FAをリセットするオプションを選択する。
4. プロンプトが表示されたら、2段階認証のリセットを確認せよ。
5. リセットしても問題がすぐに解決しない場合は、Cookieとキャッシュを消去せよ。

セキュリティ上の理由から、Brazeはユーザーに代わって2段階認証をリセットできない。したがって、管理者が2段階認証をリセットできない場合は、サポートチケットを作成すること。

#### 考慮事項

- 会社レベルで二段階認証が義務付けられている場合：リセット後、Brazeは次回ログイン時にユーザーに再度2段階認証の設定を促す。
- 会社レベルで二段階認証が義務付けられていない場合：ユーザーはダッシュボードにログインする際に、再度2段階認証を設定する必要はない。2段階認証のイネーブルメントを行う場合、アカウント設定で設定できる。

{% alert note %}
このリセット処理は、過去1時間以内にトークンを過剰に要求したためにアカウントがロックされたユーザーにも適用される。
{% endalert %}
