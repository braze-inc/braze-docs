# メッセージアクティビティログ

> [**メッセージアクティビティログ**] では、キャンペーンや送信に関連するメッセージ (特にエラーメッセージ) を確認できます。

API キャンペーンのトランザクションの確認、失敗したメッセージの詳細のトラブルシューティング、および通知配信を改善したり既存の技術的な問題を解決したりする方法に関するインサイトの収集ができます。

ログにアクセスするには、[**設定**] > [**メッセージアクティビティログ**] に移動します。

![メッセージアクティビティログ]({% image_buster /assets/img_archive/message_activity_log.png %})

{% alert tip %}
この記事に加えて、「[品質保証とデバッグツール](https://learning.braze.com/quality-assurance-and-debugging-tools-in-the-dashboard/)」の Braze ラーニングコースを確認することをお勧めします。このコースでは、メッセージアクティビティログを使用して独自のトラブルシューティングとデバッグを実行する方法を説明しています。
{% endalert %}

**メッセージアクティビティログ**に記録された次の内容を基準にしてフィルターを適用できます。

- プッシュ通知のエラー
- メッセージ中止のエラー
- Webhook エラー
- メールエラー
- API メッセージレコード
- コネクテッドコンテンツのエラー
- REST API 接続オーディエンスのエラー
- ユーザーエイリアシングのエラー
- AB テストのエラー
- SMS/MMS エラー
- WhatsApp エラー
- ライブ・アクティビティのエラー
- 不正なユーザートリガーエラー
- Brazeエージェント[の日次呼び出し制限]({{site.baseurl}}/user_guide/brazeai/agents/deploying_agents/#monitor-your-agent)エラー

これらのメッセージの送信元として、Braze システム、お客様のアプリやプラットフォーム、またはサードパーティのパートナーがあります。このため、このログに表示されるメッセージの数が膨大になる可能性があります。

## ログメッセージについて

メッセージの意味を判断する際に、各メッセージの語句とそれに対応する列に注意を払うと、コンテキストの手がかりを使用してトラブルシューティングを行う際に役立ちます。 

例えば、ログエントリのメッセージが「」"empty-cart_app"と表示されていて、その意味がわからない場合、左側の**「タイプ**」列を確認するんだ。「メッセージ中止のエラー」と表示された場合、メッセージは Liquid を使用して[メッセージ中止]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/aborting_messages/#aborting-messages)として書き込まれたものであり、メッセージが中止された理由は、メッセージの対象受信者のアプリのカートが空だったためだと確実に推測できます。

### 一般的なメッセージ

表示されるメッセージには一般的なタイプや、さらに問題の診断と修正に役立つトラブルシューティングのリンクを提供するものもあります。

以下のメッセージは例示用であり、ログの [**メッセージ**] 列に表示される内容と正確には一致しない場合があります。

| メッセージタイプ | 潜在的なメッセージ | 説明 |
|---|---|---|
| ソフトバウンス | メールアドレス same@example.com がソフトバウンスしました。 | メールアドレスは有効で、メールメッセージは受信者のメールサーバーに届きましたが、「一時的な」問題で拒否されました。<br><br>ソフトバウンスの一般的な理由として、次のものがあります。{::nomarkdown}<ul> <li> メールボックスがいっぱいです (ユーザーの割り当てを超えている) </li> <li> サーバーがダウンしていました </li> <li> メッセージが大きすぎて受信者の受信トレイに収まりません </li>  </ul> {:/} メールがソフトバウンスした場合、通常72時間以内に再送信を試みるが、再送信の試行回数は受信者によって異なる。 |

### メッセージがリストされない理由

[メッセージアクティビティログ] のメッセージは、Braze、お客様のアプリやプラットフォーム、サードパティパートナーなど、さまざまなソースから送信されます。つまり、このログには膨大な数のメッセージが表示される可能性があり、想像できるように、すべてを一覧表示することはできません。

例えば、上の表に示したメッセージに加えて、次のような「ブロック」メッセージが考えられます。

- 残念ながら、からの[_IP_ADDRESS_]メッセージは送信されなかった。インターネットサービスプロバイダーのネットワークの一部が Braze の禁止リストに含まれているため、インターネットサービスプロバイダーに問い合わせてください。
- ローカルポリシーにより、メッセージが拒否されました。
- メッセージは、受信者によってスパムとしてブロックされました。
- サービスは利用できない。クライアントホストはSpamhaus[_IP_ADDRESS_]によってブロックされた。

## ストレージリテンション期間

過去60時間のエラーは、メッセージアクティビティログで確認できます。60時間以上経過したログは消去され、アクセスできなくなる。

### エラーログの保存数

保存されるログの数は、いくつかの条件に影響される。例えば、スケジュールされたキャンペーンが何千人ものユーザーに送信された場合、メッセージアクティビティログにはすべてのエラーではなく、エラーのサンプルが表示される可能性がある。以下は、保存されるログの数に影響を与える条件の概要である：
- 以下のエラータイプについては、同一キャンペーンまたはキャンバスステップ内で、固定された1時間以内に発生した同一エラータイプのエラーログを最大20件保存する：
    - コネクテッドコンテンツのエラー
    - 中止メッセージのエラー
    - Webhook エラー
    - SMS拒否エラー
    - SMS配信失敗エラー
    - WhatsAppの障害エラー
    - AB テストのエラー
- 同一のキャンペーンまたはキャンバスステップとアプリの組み合わせにおいて、以下のエラータイプについては、同一エラータイプのプッシュ通知エラーログが最大20件保存される：
    - 無効なプッシュ認証情報
    - 無効なプッシュトークン
    - プッシュ認証情報なし
    - トークンエラー
    - クォータを超えた
    - 再試行がタイムアウトした
    - 無効なペイロード
    - 予期しないエラー
- 以下のエラータイプについては、同一アプリ内で同一エラータイプにつき最大100件のエラーログが、1固定時間単位で保存される：
    - ライブアクティビティエラー（プッシュ認証情報なし）
    - ライブアクティビティエラー（無効なプッシュ認証情報）
    - その他のライブアクティビティのエラー
    - APNフィードバック削除トークンエラー
- 以下のエラータイプについては、同一キャンペーンまたはキャンバスステップ内で、固定の1時間以内に発生した同一エラータイプのエラーログを最大100件保存する。
    - メールソフトバウンスエラー
    - メールのハードバウンスエラー
    - メールブロックエラー
- 同一ワークスペースにおいて、1固定時計時間内に最大100件のユーザーエイリアシングエラーログが保存される。

## テスト送信

**メッセージ活動ログ**は、以下のメッセージングチャネルのテストログを表示する：

- SMS
- WhatsApp
- LINE
- Webhook

以下のチャネルではテスト送信ログは利用できない：メール、コンテンツカード、アプリ内メッセージ、プッシュ通知。

テスト送信ログには「[TEST SEND]」という接頭辞が付く。ただし、全てのテスト送信ログに接頭辞が付くとは限らない（例えば、コネクテッドコンテンツのエラーには接頭辞が付かない）。
