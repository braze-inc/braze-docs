# 細分化された権限への移行

> アカウントへのアクセスや特定のアクションを実行できるユーザーを管理することは、セキュリティと業務効率の両面で極めて重要です。より細かいコントロールを可能にするため、Brazeは細分化された権限を導入します。これはアカウント全体でのユーザーアクセスをより柔軟かつ精密に管理する方法です。

移行には以下の利点があります。

- **より精密なコントロール：**細分化された権限は、より細かいコントロール、より優れたセキュリティ、そしてより明確な監視を可能にします。ユーザーは必要なアクセス権限のみを付与されます。
- **自動マッピング：**現在のすべての権限は、[細分化された対応する権限](#legacy-to-granular-permissions-mapping)に自動的にマッピングされます。変更しない限り、ユーザーは同じアクセスレベルを維持します。

## 確認すべき内容

自社の移行が計画された場合、Brazeの管理者はメールとダッシュボード上のバナーで細分化された権限の移行について通知を受け取ります。移行の準備として、Brazeの管理者が以下を行うことをお勧めします。

1. 新しい権限フレームワークに移行した後、より細かなアクセス制御のために更新が必要なユーザー、ロール、または権限セットを特定します。 
2. 貴社がSCIMや[権限文字列]({{site.baseurl}}/scim_api_appendix/)に依存するコンプライアンスツールを使用してユーザープロビジョニングを自動化している場合は、新しい細分化された構造に合わせてそれらを更新してください。 
3. 混乱を防ぐため、今後の変更についてBrazeユーザーに通知してください。
4. スケジュールされた移行日時になると、貴社は自動的に細分化された権限に移行します。会社の管理者によるそれ以上のアクションは不要です。

{% alert important %}
権限の更新機能は、スケジュールされた移行時刻の15分前にロックされます。つまり、移行が完了するまで権限を変更できません。移行の完了には最大15分かかる見込みです。
{% endalert %}

## よくある質問

### 移行を拒否したり元に戻したりできますか？

Brazeは移行の取り消しをサポートしていません。移行プロセスを通じてサポートし、移行を厳密に監視して問題が発生した場合は迅速に対処します。

### 移行中に既存ユーザーはBrazeへのアクセスを失いますか？

いいえ、移行中にBrazeのダウンタイムは発生しません。ただし、移行中は権限の更新がロックされます。移行の完了には最大15分かかる見込みです。
