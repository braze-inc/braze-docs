# 細分化された権限への移行

> アカウントへのアクセス権限や特定のアクションを実行できる者を管理することは、セキュリティと業務効率の両面で極めて重要だ。より細かいコントロールを可能にするため、Brazeは細分化された権限管理を導入する。これはアカウント全体でのユーザーアクセス管理をより柔軟かつ精密に行う方法だ。

移行には以下の利点がある：

- **より精密なコントロール：**細かな権限設定は、より細かいコントロール、より優れたセキュリティ、そしてより明確な監視を可能にする。ユーザーは必要なアクセス権限のみを得る。
- **自動マッピング：**現在のすべての権限は[、細分化された](#legacy-to-granular-permissions-mapping)対応する[権限](#legacy-to-granular-permissions-mapping)に自動的にマッピングされる。ユーザーは、変更しない限り同じアクセスレベルを維持する。

## レビュー内容

自社の移行が計画された場合、Brazeの管理者はメールとダッシュボード上のバナーで通知を受け取る。これらは詳細な権限の移行について知らせるものだ。移行の準備として、Brazeの管理者は以下のことを行うことを推奨する。

1. 新しい権限フレームワークに移行した後、より細かなアクセス制御のために更新が必要なユーザー、ロール、または権限セットを識別する。 
2. もし貴社がSCIMや[権限]({{site.baseurl}}/scim_api_appendix/)文字列に依存するコンプライアンスツールを用いてユーザープロビジョニングのオートメーションを行っているなら、新しい細分化された構造に合わせてそれらを更新せよ。 
3. 今後の変更について、混乱を避けるため、Brazeユーザーに通知せよ。
4. スケジュールされた移行日時において、御社は自動的に細分化された権限管理に移行する。会社管理者はこれ以上のアクションは不要だ。

{% alert important %}
権限の更新機能は、移行のスケジュールされた時刻の15分前までにロックされる。つまり、移行が完了するまで権限を変更できない。移行には最大15分かかる見込みだ。
{% endalert %}

## よくある質問

### 移行を拒否したり元に戻したりできるか？

Brazeは移行の取り消しをサポートしていない。移行作業を通じて支援する。移行を厳密に監視し、問題が発生した場合は迅速に対処する。

### 移行期間中、既存ユーザーはBrazeへのアクセスを失うのか？

いや、移行中にBrazeのダウンタイムは発生しない。ただし、移行中は権限の更新はロックされる。移行には最大15分かかる見込みだ。
