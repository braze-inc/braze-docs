# 内部グループ

> 内部グループは、内部またはサードパーティのテストグループを構築し、整理するための素晴らしい方法です。これらは、SDK ログまたは API ログに関するインサイトを提供するため、SDK 連携をテストするときに役立ちます。最大1,000人のユーザーからなるカスタム内部グループを無制限に作成できます。

{% alert tip %}
また、[ Testing and Troubleshooting](https://learning.braze.com/path/developer/testing-and-troubleshooting) Braze Learning コースをチェックすることをお勧めします。このコースでは、内部グループを使用して独自のトラブルシューティングとデバッグを実行する方法について説明します。
{% endalert %}

## 前提条件

内部グループを作成および管理するには、[Access Dev Console レガシー権限]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/?sdktab=legacy%20permissions)、または以下の[詳細権限]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/?sdktab=granular%20permissions)が必要だ：

- API キーを表示
- API キーを編集
- 内部グループを表示する
- 内部グループを編集する
- メッセージアクティビティログを表示
- イベントユーザーログを閲覧
- API識別子を表示する
- API 使用状況ダッシュボードを表示
- API 制限を表示
- API 使用アラートを表示
- API 使用アラートを編集
- SDK デバッガーを編集
- SDK デバッガーを表示

## 内部グループの作成

内部グループを作成するには： 

1. [**設定**] > [**内部グループ**] に移動します。
2. **内部グループの作成**を選択します。
3. 「Email test group」などの名前をグループに付けます。
4. 次の表に示すように、1 つ以上のグループタイプを選択します。

| Group type         | 説明                                                                                 |
|--------------------|---------------------------------------------------------------------------------------------|
| **ユーザー・イベント・グループ**   | これを使って、テスト端末からのイベントやログを確認するのだ。                                    |
| **コンテンツテスト・グループ** | プッシュ通知、メール、アプリ内メッセージでこの機能を使い、メッセージのレンダリング済みコピーを送信する。 |
| **シードグループ**         | 送信時に自動的に、シードグループの全員にメールのコピーを送信する。               |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

### テストユーザーの追加

内部グループを作成したら、テストユーザーをそのグループのメンバーとして追加する。 

1. 内部グループの管理ページから、**テストユーザーを追加** を選択します。
2. テストユーザーを検索して選択するには、次の方法から選択します。

| 方法                  | 説明                                                                                                                                                                                                                                          |
|-------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **識別されたユーザーを追加** | 外部ユーザーID、電子メールアドレス、電話番号、プッシュトークンでユーザーを検索します。                                                                                                                                                           |
| **匿名ユーザーを追加**  | IPアドレスで検索する。次に、追加するテストユーザーごとに名前を指定する。これは、[イベントユーザーログ]({{site.baseurl}}/user_guide/administrative/app_settings/event_user_log_tab/)ページで全てのイベントログに関連付けられている名前だ。 |
| **ユーザーを一括追加**      | メールアドレスまたは外部ID のリストをコピーして貼り付けます。ダッシュボードで既に認識されているユーザーのみ追加できる。詳細については、[ユーザーインポート]({{site.baseurl}}/user_guide/data/user_data_collection/user_import/)を参照してください。          |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

### コンテンツテストグループ

メッセージのプレビューテストを送信するのと同じように、コンテンツテストグループは時間を節約し、事前に定義されたBrazeユーザーリストに対して同時にテストを開始できる。これは、Braze でプッシュ、アプリ内メッセージ、SMS、電子メール、およびコンテンツカードに使用できます。メッセージのプレビューセクションでは、コンテンツテストグループとしてタグ付けされたグループのみが利用可能だ。

{% alert note %}
[SMS]({{site.baseurl}}/user_guide/message_building_by_channel/sms_mms_rcs/) テストメッセージは、データベース内の有効な電話番号にのみ送信できます。
{% endalert %}

メッセージを送信する相手として、個々のBrazeユーザーまたは任意の数の内部グループを選択する。メッセージにLiquidやその他のダイナミックなパーソナライゼーションが含まれている場合、Brazeは各ユーザーに対して利用可能な属性を使用してメッセージ内容をパーソナライズする。属性を持たないユーザーに対しては、Brazeはデフォルト値を使用する。

さらに、メッセージをランダムユーザー、カスタムユーザー、または既存ユーザーとしてプレビューした場合、そのプレビュー版を代わりに送信できる。チェックボックスを外すと、プレビュー版ではなく各ユーザーの属性に基づいて送信できるようになる。

IPプールを使ってメールを送信する場合、ドロップダウンから送信元のIPプールを選択する。

![アプリ内メッセージエディタのテストセクションで、コンテンツテストグループを選択する。]({% image_buster /assets/img_archive/content_test_preview.png %}){: style="max-width:60%" }

### シードグループ

シードグループは、メールチャネルでのみサポートされます。シードグループにユーザーを追加すると、各メールのバリアントメッセージのコピーがグループの全メンバーに送信される。

シードグループはAPI キャンペーンでは使用できませんが、API トリガエントリを使用してシードグループをキャンペーンに含めることができます。これを用いて配信率を測定し、メールコンテンツの記録を保存する。過去の記録や保管の目的のためだ。 

内部グループを作成し、シードグループとして使用するためにタグ付けした後、キャンペーンエディタの「**ターゲットオーディエンス**」ステップ、またはキャンバスの**「送信設定**」ステップでそれを選択する。 

シードメールでは、メールの件名行の先頭に`[SEED]` が追加されます。シードメールは次の動作は**行わない**ことに注意してください。

- インクリメントはダッシュボードの分析データを送信する。
- メール分析またはリターゲットに影響する。 
- ユーザプロファイルの**Campaign Received** リストを更新します。
- フリークエンシーキャップに影響する。
- 配信速度のレート制限を考慮する、または影響を与える。

#### 購読行動

シード送信は内部QAとレビュー用に設計されているため、意図的にシード対象企業のユーザーに対するサブスクリプションチェックをバイパスする。これは、シードグループに属する有効なメールアドレスを持つユーザーは、たとえサブスクライバーではない場合でもメッセージを受け取ることを意味する。ただし、そのグループにシードコピーを送信するようメッセージを設定する必要がある。

{% alert tip %}
Seed Group メンバーが受信トレイにメッセージが表示されていないと報告した場合は、内部グループに表示されていることを確認し、件名が異なること、Gmail がメールをまとめてバンドルしていないことを確認するか、スパムフォルダをチェックしてもらいます。
{% endalert %}

#### キャンペーンの場合

メールキャンペーンを作成する際は、エディターの**「ターゲットオーディエンス**」セクションでシードグループを編集する。

{% alert important %}
シードグループをすべてのキャンペーンに自動適用するように設定した場合、それは新規キャンペーンにのみ適用される。既存のキャンペーンをコピーする場合には適用されない。**ターゲットオーディエンス**セクションで、コピーしたキャンペーンに手動で希望のシードグループを適用しなければならない。
{% endalert %}

シードグループは、各メールバリアントに一度送信を行い、ユーザーがその特定のバリアントを初めて受信したときに配信されます。スケジュールされたメッセージの場合、これは通常、キャンペーンが初めて開始されるときです。アクションベースまたはAPIトリガー型のキャンペーンでは、これが最初のユーザーにメッセージが送信される時刻である。

キャンペーンが多変量であり、バリアントの配信率が0%の場合、シードグループには配信されない。さらに、バリアントが既に送信済みで、**ターゲット**ステップの**「シードグループ編集」**で再送信用に更新されていない場合、デフォルトでは再送信されない。

{% alert note %}
定期的なキャンペーンがあり、いずれかのバリアントが更新された場合は、更新されたバリアントのみまたはすべてのバリアントに再度送信するか、更新時のシードグループへの送信をオフにすることができます。
{% endalert %}

![「メールシードテスト」シードグループは、バリアント1のメールキャンペーンを送信されるように選ばれた。]({% image_buster /assets/img_archive/seed_group_campaign.png %})

#### キャンバスの場合

キャンバスのシードグループは、トリガーされたキャンペーンと同様に機能する。Brazeは自動的にメールメッセージを含む全ステップを検出し、ユーザーが特定のメールステップに初めて到達した際にそれらに送信する。

シードグループが配信された後にメールステップが更新された場合、Brazeでは更新されたステップのみに送信するか、全てのステップに送信するか、あるいはシード機能を無効にするかの選択肢を提供する。
