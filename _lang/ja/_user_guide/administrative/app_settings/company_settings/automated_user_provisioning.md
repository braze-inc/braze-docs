# 自動ユーザープロビジョニング

> SCIMプロビジョニングを使用して、APIを通じてBrazeユーザーを自動的に作成・管理する。この記事では、提供すべき情報、SCIMトークンの生成方法、SCIM APIエンドポイントの場所について説明する。

{% multi_lang_include early_access_beta_alert.md feature='SCIM provisioning' %}

## SCIMプロビジョニング設定へのアクセス

1. Brazeのダッシュボードで、**設定**＞**管理設定**＞**SCIMプロビジョニング**に移動し、**SCIM連携の設定**を選択する。
2. **Brazeの設定**ステップでは、プロビジョニング方法を選択し、アクセス設定を提供する。

![SCIM統合を設定するページで、プロビジョニング方法の選択とアクセス設定の提供を行うセクションがある。]({% image_buster /assets/img_archive/scim_braze_config.png %}){: style="max-width:70%;"}

{: start="3"}
3\.**IdP設定**ステップでは、選択したプロビジョニング方法に対応するプラットフォーム内のステップに従うこと。

## ステップ 1: SCIMプロビジョニングを設定する

### ステップ1.1：SCIM を有効にする

1. Oktaで、**アプリケーション**＞**アプリケーション**に移動し、**アプリ統合の作成**を選択する。サインイン方法として**SAML 2.0**を選択する。
2. カスタムアプリを作成するには、以下の詳細（Braze [**IdP設定**ステップ](#accessing-scim-provisioning-settings)にある）を入力する：
- アプリのロゴ
- シングルサインオン URL
- オーディエンス URL (サービスプロバイダーエンティティ ID)
3. [**Finish**] を選択します。
4. **一般**タブを選択する。 
5. **アプリ設定**のセクションで、**編集**を選択する。
6. **プロビジョニング**フィールドで、**SCIM**を選択する。 

### ステップ1.2：アプリケーションの表示を無効にする

1. **アプリケーションの表示**設定フィールドで、**「ユーザーにアプリケーションアイコンを表示しない」**チェックボックスを選択する。これにより、ユーザーがアプリ経由でSSOにアクセスするのを防ぐ。このアプリはSCIM専用に設計されている。 
2. [**保存**] を選択します。

### ステップ1.3：SCIM連携を設定する

1. **プロビジョニング**タブを選択する。
2. **設定**＞**連携**＞**SCIM接続**で**編集**を選択し、**SCIMプロビジョニング設定**ページ内のテーブルに表示されるフィールド値を入力する。

### ステップ1.4:API 認証情報をテストする

**テスト用API認証情報**を選択せよ。統合が成功し保存できる場合、確認メッセージが表示される。

### ステップ1.5:アプリへのプロビジョニングを有効にする

1. **プロビジョニング** > **設定** > **アプリへ** > **アプリへのプロビジョニング** で、**編集**を選択する。
2. 次を有効にする: 
    - ユーザーを作成する
    - ユーザー属性を更新する
    - ユーザーを非アクティブにする
3. **SCIMプロビジョニング設定**ページのテーブル内に表示されるマッピングを使用して、**属性マッピング**セクションを確認し設定する。

## ステップ 2:ユーザーをアプリに割り当てる

1. **「割り当て**」タブを選択する。
2. **割り当て**を選択し、オプションを選ぶ。
3. アプリを、Brazeへのアクセス権を持つべき人に割り当てる。
4. 課題を完了したら**「完了」**を選択せよ。

## ステップ 1: SCIMプロビジョニングアプリを設定する

### ステップ1.1：Microsoft Entra 管理センターにログインする

Microsoft Entra 管理センターにログインする。

### ステップ1.2：SCIMアプリを作成し設定する

1. ナビゲーションメニューで、**Entra ID** > **エンタープライズアプリ** を選択する。
2. **新規アプリケーション**を選択せよ。
3. **「独自のアプリケーションを作成する」**を選択せよ。
4. パネルに、アプリの名前を入力する。
5. 「**アプリケーションで何をしたいですか？」**のセクションで、**「ギャラリーにないアプリケーションを統合する（非ギャラリー）」**を選択せよ。
6. [**作成**] を選択します。

### ステップ1.3：SCIMの連携を設定する

1. SCIMアプリケーションの**「管理**」＞「**プロビジョニング**」セクションに移動する。
2. **「アプリケーションの接続」**または「**新規設定**」を選択し、**SCIMプロビジョニング**設定ページの表内に表示されるフィールド値を入力する。

### ステップ1.4:アプリへのプロビジョニングを有効にする

1. SCIMアプリケーションの**「管理**」＞「**属性マッピング（プレビュー）**」セクションに移動する。
2. **Microsoft Entra ID ユーザー**を選択する。
3. **SCIMプロビジョニング**設定ページのテーブル内に表示される属性と一致させるため、**属性マッピング**セクションを確認し設定する。
4. **属性マッピング**ページを閉じる。

## ステップ 2:ユーザーをアプリに割り当てる

1. **管理** > **ユーザーとグループ** へ移動する。
2. **ユーザーまたはグループを追加**を選択する。
3. **「選択なし」**を選択して、ユーザーをアプリに割り当てる。
4. 割り当てを確認するには、**選択**ボタンを選択せよ。

## ステップ 1: SCIMの設定を行う

- **デフォルトのワークスペース:**新規ユーザーをデフォルトで追加するワークスペースを選択せよ。[SCIM API リクエスト]({{site.baseurl}}/post_create_user_account/)でワークスペースを指定しなかった場合、Braze はこのワークスペースにユーザーを割り当てます。
- **サービス提供元:**SCIMリクエストのオリジンドメインを入力する。Braze はこれを `X-Request-Origin` ヘッダーで使用し、リクエストがどこからのものであるかを確認します。
- **IP 許可リスト (オプション):**SCIM リクエストを特定の IP アドレスに制限できます。許可する IP アドレスのコンマ区切りリストまたは範囲を入力します。各リクエストのヘッダー`X-Request-Origin`は、リクエストのIPアドレスを許可リストと照合するために使用される。

![SCIM プロビジョニング設定フォームに3つのフィールド:デフォルトワークスペース、サービスOrigin、およびオプションのIP許可リスト。「SCIMトークンを生成」ボタンは無効になっている。]({% image_buster /assets/img/scim_unfilled.png %})

## ステップ 2:SCIMトークンを生成する

必須フィールドを入力してから [**SCIM トークンを生成**] を押すと、SCIM トークンが生成され、SCIM API エンドポイントが表示されます。他へ移動する前に SCIM トークンをコピーしておいてください。**このトークンは一度だけ現れる。** 

![[SCIM API エンドポイント] フィールドと [SCIM トークン] フィールドに、マスクされた値とコピーボタンが表示されている。トークンフィールドの下には「トークンをリセット」ボタンがある。]({% image_buster /assets/img/scim.png %})

Braze では、すべての SCIM リクエストに HTTP `Authorization` ヘッダー経由で添付された SCIM API ベアラートークンが含まれている必要があります。
