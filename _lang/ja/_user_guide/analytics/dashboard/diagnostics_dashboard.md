# メッセージング診断ダッシュボード

> **メッセージング診断**ダッシュボードは、メッセージ送信結果の概要を提示する。これにより、傾向を把握し、メッセージング設定における潜在的な問題を診断できる。このダッシュボードは、キャンペーンやキャンバスからのメッセージが期待通りに送信されなかった理由を理解するのに役立つ。

{% alert important %}
**メッセージング診断**ダッシュボードは現在、早期アクセス段階にある。早期アクセスに参加したい場合は、顧客サクセスマネージャーに連絡するんだ。
{% endalert %}

## 主要な概念

### 送付済み、配達済み

このダッシュボードは、メッセージの最終的なステータスではなく、Brazeが内部でメッセージをどのように処理したかをレポートしていることを理解することが重要だ。

このダッシュボードで「送信済み」とマークされたメッセージは、Brazeがメッセージを正常に処理し配信したことを意味する。ほとんどのチャネルでは、これはBrazeがメッセージを関連するサードパーティ送信パートナーに引き渡したことを意味する。ただし、ユーザーの端末への最終的な配信を保証するものではない。 

Brazeがメッセージを「送信」する場合、最終的な配信は外部サービスに依存する可能性がある。各チャネルについて、以下の例を考えてみよ。

| チャネル | 最終納品の例 |
| --- | --- |
| コンテンツカードによって促進された | カードは送付済みであり、閲覧可能である。 |
| メール | Brazeはメッセージをメールサービスプロバイダー (ESP) に渡す。その後、メールサービスプロバイダー (ESP) が最終的な配送を担当する。例えば、メールサービスプロバイダー (ESP) はメールアドレスが無効だったり受信トレイが満杯だったりすると「バウンス」をレポートするかもしれない。 |
| アプリ内メッセージ | そのメッセージはユーザーに表示された。 |
| LINE | メッセージは送信パートナーに正常に引き渡された。 |
| プッシュ | Brazeはメッセージを適切なプッシュ通知サービス（例えばiOS向けのAppleプッシュ通知サービスやAndroid向けのFirebase Cloud Messagingなど）に渡す。そのサービスは、通知を端末に最終的に届ける責任を負っている。 |
| SMS/MMS/RCS | BrazeはメッセージをSMSゲートウェイ（Twilioのようなもの）に渡す。そのゲートウェイは、モバイルキャリアへの最終的な配信を担当している。 |
| Webhook | Webhookのリクエストは正常に送信され、応答が`2xx`返された。 |
| WhatsApp | メッセージは送信パートナーに正常に引き渡された。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" } 

### データの鮮度

このダッシュボードのデータ更新頻度は、システム負荷によって変動する可能性がある。更新頻度は保証されないが、大抵の場合1時間以内である可能性が高い。

## ダッシュボードの設定

診断ダッシュボードには、**分析**＞**ダッシュボードビルダー**に移動し、Brazeが作成したダッシュボードの一覧から**「メッセージング診断」**を選択することでアクセスできる。

ダッシュボードを実行してデータを表示するには：

1. ダッシュボードレポートのソースとして**、キャンペーン**か**キャンバスの**いずれかを選択せよ。 
2. 1つ以上のキャンペーンまたはキャンバスを選択せよ。
3. 選択したフィルターのデータを読み込むには、**[ダッシュボードを実行]**を選択せよ。

## データの解釈

{% alert note %}
ダッシュボードには過去7日間のデータしか表示されない。
{% endalert %}

### 要約タイル

ページの上部には、選択した期間の主要な要約タイルが表示される。そこには以下の情報が示される：

- **合計中止数：**中止されたメッセージの総数だ。これには、ステップの失敗を経験したか、退出イベントの実行中に退出条件を満たしたためにキャンバスに入らなかった、またはキャンバスから退出したキャンバスのオーディエンスも含まれる。
- **メッセージングが実行される。**Brazeが正常に処理し送信したメッセージの総数。 
  - **メール、SMS/MMS/RCS、WhatsApp、LINE、プッシュ通知：**メッセージは送信パートナーに正常に引き渡された。  
  - **Webhooks:**Webhookのリクエストは正常に送信され、応答が`2xx`返された。  
  - **コンテンツカード:**カードは送付済みであり、閲覧可能である。    
  - **アプリ内メッセージ:**メッセージはユーザーに表示された。

### 時間の経過に伴うメッセージの結果

この時系列チャートは、メッセージが中断されたりユーザーがキャンバスから切断されたりする様々な理由を、日ごとの内訳で示している。このグラフは送信数を表示していない。  

{% alert note %}
チャートを整理するため、選択した時間範囲内で発生回数がゼロの中止またはドロップ理由はチャートに表示されない。
{% endalert %}

### メッセージの結果の内訳

このグラフは、選択した時間範囲内の全メッセージ結果の内訳を示している。それは次の完全な概要を提供する：
- 送信総数が全結果に占める割合。  
- 各中止およびドロップ理由の割合内訳。これにより、メッセージが送信されない最も一般的な原因を素早く識別できる。

## よくある質問

### 「事前チェック」の失敗とはどういう意味か？

「事前チェック」とは、パイプライン段階の最初（例えばメッセージのトリガー時やCanvasメッセージステップの送信時）に実行される、高速で一括処理される検証チェックを指す。それを、最高速度を追求した早期退出と考えるといい。多くの別々のリソースを消費するチェック（例えばユーザープロファイルの詳細を一つ一つ検証する作業）を実行する代わりに、Brazeはいくつかの基本的な検証を一つにまとめて「最初のパス」として処理する。

ユーザーがこの単一のバンドルチェックに失敗した場合、即座に除外される。このバンドル方式により、Brazeは大量のメッセージを高速で処理できる。各メッセージの処理遅延を削減することで、キャンペーンやキャンバスのパフォーマンスを高速かつ安定させるのに寄与する。

### なぜ_総中止数_と_メッセージ送信_数の合計が、予想していたオーディエンス数より少ないのか？

これにはいくつかの理由が考えられます。

- **オーディエンス条件：**キャンペーンやキャンバスが開始された時点で、予想より少ないユーザーしかオーディエンス条件を満たしていなかった可能性がある（例えば、対象セグメントに含まれていなかった、あるいは必要な属性を持っていなかった）。
- **処理中だ。**メッセージはまだ処理中かもしれない。ユーザーはまだキャンバスの初期ステップに留まっており、メッセージステップには到達していない可能性がある。
- **データの鮮度：**ダッシュボードのデータは約15分ごとに更新されるが、これは保証ではない。このキャンペーンまたはキャンバスの最新データが、まだダッシュボードに反映されていない可能性がある。
- **エッジケース：**現時点では、このダッシュボードで捕捉されていないエッジケースに遭遇している可能性がわずかに存在する。この状況が疑われる場合は、[Brazeサポートに]({{site.baseurl}}/user_guide/administrative/access_braze/support)連絡する。

### なぜキャンペーンとキャンバスの_合計中止数_と_メッセージ送信_数の合計が、そのキャンペーンとキャンバスのオーディエンス数より大きくなるのか？

これは以下の理由により発生する可能性がある：

- **マルチチャネルメッセージング：**キャンペーンまたはキャンバスステップは、複数のチャネル（SMSやメールなど）で送信されるように設定された。一人のユーザーが、あるチャネル（例えばメール）では「送信済み」の結果を受け取り、別のチャネル（例えば「SMS/MMS/RCSのイネーブルメントが許可されていないユーザー」）では「中止」の結果を受け取る可能性がある。この場合、そのユーザーはチャートで二重にカウントされる。一度は「送信」として、もう一度は「中止」としてだ。
  - **例:**100人のユーザーにプッシュキャンペーンを送信する。iOSとAndroidの両方を対象とする。ユーザーがiOS端末しか持っていない場合、iOSプッシュ（「送信済み」）は受信するが、Androidプッシュ（「Androidプッシュ、アプリ、または端末のイネーブルメントが有効化されていないユーザー」）はトリガーされる。
- **複数のメッセージステップ（キャンバスのみ）：**特定のパスにおいて、キャンバスには複数のメッセージステップが含まれる場合がある。このダッシュボードは全ての結果を集計するため、選択した時間範囲内で複数のメッセージステップを通過した場合、同一ユーザーが複数回カウントされる可能性がある。
- **テストメッセージ：**テスト送信（ダッシュボードでカウントされる）が、総カウント数をオーディエンス規模よりも高くしている。
