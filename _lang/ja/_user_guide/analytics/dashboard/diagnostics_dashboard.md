# メッセージング診断ダッシュボード

> **メッセージング診断**ダッシュボードは、メッセージ送信結果の概要を提示し、傾向を把握してメッセージング設定における潜在的な問題を診断できるようにします。このダッシュボードは、キャンペーンやキャンバスからのメッセージが期待通りに送信されなかった理由を理解するのに役立ちます。

{% alert important %}
**メッセージング診断**ダッシュボードは現在、早期アクセス段階にあります。早期アクセスへの参加をご希望の場合は、カスタマーサクセスマネージャーにお問い合わせください。
{% endalert %}

## 主要な概念

### 送信済みと配信済み

このダッシュボードは、メッセージの最終的な配信ステータスではなく、Brazeが内部でメッセージをどのように処理したかをレポートしていることを理解することが重要です。

このダッシュボードで「送信済み」とマークされたメッセージは、Brazeがメッセージを正常に処理し送信したことを意味します。ほとんどのチャネルでは、これはBrazeがメッセージを関連するサードパーティ送信パートナーに引き渡したことを意味します。ただし、ユーザーのデバイスへの最終的な配信を保証するものではありません。 

Brazeがメッセージを「送信」する場合、最終的な配信は外部サービスに依存する可能性があります。各チャネルについて、以下の例をご確認ください。

| チャネル | 最終配信の例 |
| --- | --- |
| コンテンツカード | カードは送信済みであり、閲覧可能です。 |
| メール | Brazeはメッセージをメールサービスプロバイダー（ESP）に渡します。その後、ESPが最終的な配信を担当します。例えば、メールアドレスが無効だったり受信トレイが満杯だったりすると、ESPは「バウンス」をレポートする場合があります。 |
| アプリ内メッセージ | メッセージはユーザーに表示されました。 |
| LINE | メッセージは送信パートナーに正常に引き渡されました。 |
| プッシュ | Brazeはメッセージを適切なプッシュ通知サービス（iOS向けのApple Push Notification serviceやAndroid向けのFirebase Cloud Messagingなど）に渡します。そのサービスが、デバイスへの通知の最終配信を担当します。 |
| SMS/MMS/RCS | BrazeはメッセージをSMSゲートウェイ（Twilioなど）に渡します。そのゲートウェイが、モバイルキャリアへの最終的な配信を担当します。 |
| Webhook | Webhookリクエストは正常に送信され、`2xx`レスポンスが返されました。 |
| WhatsApp | メッセージは送信パートナーに正常に引き渡されました。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" } 

### データの鮮度

このダッシュボードのデータ更新頻度は、システム負荷によって変動する可能性があります。更新頻度は保証されませんが、ほとんどの場合1時間以内に更新されます。

## ダッシュボードの設定

診断ダッシュボードには、**分析** > **ダッシュボードビルダー**に移動し、Brazeが作成したダッシュボードの一覧から**メッセージング診断**を選択することでアクセスできます。

ダッシュボードを実行してデータを表示するには：

1. ダッシュボードレポートのソースとして、**キャンペーン**または**キャンバス**のいずれかを選択します。 
2. 1つ以上のキャンペーンまたはキャンバスを選択します。
3. **ダッシュボードを実行**を選択して、選択したフィルターのデータを読み込みます。

![2025年5月25日から5月31日までのウェルカムシリーズキャンペーンにおけるキャンペーンとキャンバスの診断例。]({% image_buster /assets/img/campaign_canvas_dashboard_example.png %}){: style="max-width:90%;"}

## データの解釈

{% alert note %}
ダッシュボードには過去7日間分のデータのみが表示されます。 
{% endalert %}

### サマリータイル

ページの上部には、選択した期間の主要なサマリータイルが表示され、以下の情報が示されます：

- **合計中止数：**中止されたメッセージの総数です。これには、ステップの失敗を経験したか、退出イベントの実行中に退出条件を満たしたためにキャンバスに入らなかった、またはキャンバスから退出したキャンバスオーディエンスメンバーも含まれます。
- **メッセージ送信数：**Brazeが正常に処理し送信したメッセージの総数です。 
  - **メール、SMS/MMS/RCS、WhatsApp、LINE、プッシュ：**メッセージは送信パートナーに正常に引き渡されました。  
  - **Webhook：**Webhookリクエストは正常に送信され、`2xx`レスポンスが返されました。  
  - **コンテンツカード：**カードは送信済みであり、閲覧可能です。    
  - **アプリ内メッセージ：**メッセージはユーザーに表示されました。

### 時間経過に伴うメッセージ結果

この時系列チャートは、メッセージが中止されたりユーザーがキャンバスからドロップされたりしたさまざまな理由を、日ごとの内訳で示します。このチャートには送信数は表示されません。  

{% alert note %}
チャートを整理するため、選択した時間範囲内で発生回数がゼロの中止またはドロップ理由はチャートに表示されません。
{% endalert %}

### メッセージ結果の内訳

このチャートは、選択した時間範囲内の全メッセージ結果の内訳を示します。以下の全体像を提供します：
- 全結果に占める送信総数の割合。  
- 各中止およびドロップ理由の割合内訳。これにより、メッセージが送信されない最も一般的な原因を素早く特定できます。

### 中止結果

以下の定義は、ダッシュボードに表示される中止結果を説明します。結果はカテゴリー別にグループ化されており、調査対象の項目を簡単に見つけられます。

## よくある質問

### 「事前チェック」の失敗とはどういう意味ですか？

「事前チェック」とは、パイプライン段階の最初（メッセージのトリガー時やキャンバスメッセージステップの送信時など）に実行される、高速で一括処理される検証チェックを指します。これは、最高速度を追求した早期退出の仕組みと考えてください。多くの個別のリソース集約型チェック（ユーザープロファイルの詳細を一つ一つ検証するなど）を実行する代わりに、Brazeはいくつかの基本的な検証を1つの「初回パス」にまとめて処理します。

ユーザーがこの一括チェックに失敗した場合、即座にドロップされます。この一括処理方式により、Brazeは大量のメッセージを高速で処理でき、各メッセージの処理レイテンシーを削減することで、キャンペーンやキャンバスのパフォーマンスをより高速かつ安定させることに貢献します。

### 「その他」の中止結果とはどういう意味ですか？

これらは、既存のBrazeカテゴリーのいずれにも該当しなかった中止です。この結果の中止が大きな割合を占めている場合は、[Brazeサポート]({{site.baseurl}}/user_guide/administrative/access_braze/support)にお問い合わせください。

### _合計中止数_と_メッセージ送信数_の合計が、予想していたオーディエンスサイズより少ないのはなぜですか？

これにはいくつかの理由が考えられます：

- **オーディエンス条件：**キャンペーンやキャンバスが開始された時点で、予想より少ないユーザーしかオーディエンス条件を満たしていなかった可能性があります（例えば、対象セグメントに含まれていなかった、または必要な属性を持っていなかった）。
- **処理中：**メッセージがまだ処理中の可能性があります。ユーザーがまだキャンバスの初期ステップに留まっており、メッセージステップに到達していない場合があります。
- **データの鮮度：**ダッシュボードのデータは約15分ごとに更新されますが、これは保証ではありません。このキャンペーンまたはキャンバスの最新データが、まだダッシュボードに反映されていない可能性があります。
- **エッジケース：**現時点でこのダッシュボードで捕捉されていないエッジケースに遭遇している可能性がわずかにあります。この状況が疑われる場合は、[Brazeサポート]({{site.baseurl}}/user_guide/administrative/access_braze/support)にお問い合わせください。

### キャンペーンとキャンバスの_合計中止数_と_メッセージ送信数_の合計が、オーディエンスサイズより大きくなるのはなぜですか？

これは以下の理由により発生する可能性があります：

- **マルチチャネルメッセージ：**キャンペーンまたはキャンバスステップが、複数のチャネル（SMSやメールなど）で送信されるように設定されていました。1人のユーザーが、あるチャネル（メールなど）では「送信済み」の結果を受け取り、別のチャネル（「ユーザーがSMS/MMS/RCSに対して有効でない」など）では「中止」の結果を受け取る可能性があります。この場合、そのユーザーはチャートで2回カウントされます：1回は「送信」として、もう1回は「中止」としてです。
  - **例：**100人のユーザーにプッシュキャンペーンを送信し、iOSとAndroidの両方をターゲットにします。ユーザーがiOSデバイスのみを持っている場合、iOSプッシュ（「送信済み」）は受信しますが、Androidプッシュ（「ユーザーがAndroidプッシュ、アプリ、またはデバイスに対して有効でない」）の中止もトリガーされます。
- **複数のメッセージステップ（キャンバスのみ）：**キャンバスの特定のパスに複数のメッセージステップが含まれている場合があります。このダッシュボードはすべての結果を集計するため、選択した時間範囲内で複数のメッセージステップを通過した場合、同一ユーザーが複数回カウントされる可能性があります。
- **テストメッセージ：**テスト送信（ダッシュボードでカウントされます）により、合計カウントがオーディエンスサイズより大きくなっています。
