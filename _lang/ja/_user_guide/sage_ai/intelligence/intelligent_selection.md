# インテリジェントセレクション {#intelligent-selection}

> インテリジェントセレクションは、定期的なキャンペーンまたはキャンバスのパフォーマンスを1日2回分析し、各メッセージバリアントを受信するユーザーの割合を自動的に調整する機能です。 

他のバリアントよりもパフォーマンスが良いと思われるバリアントはより多くのユーザーに送信され、パフォーマンスの低いバリアントはより少ないユーザーをターゲットにします。各調整は [統計アルゴリズム] [227] を使用して行われ、ランダムな偶然ではなく、実際のパフォーマンスの違いに合わせて調整していることを確認しています。

![インテリジェントセレクションが有効になっているキャンペーンまたはキャンバスのA/Bテストセクション] [3]

インテリジェントセレクションは次のことを行います。
-パフォーマンスデータを繰り返し見て、キャンペーントラフィックをウィニングバリアントに徐々にシフトさせてください。
-統計的な信頼性を損なうことなく、より多くのユーザーが最もパフォーマンスの高いバリアントを受け取れることを確認します。
-パフォーマンスの低いバリアントを除外し、[従来のA/Bテストよりも迅速にパフォーマンスの高いバリアントを特定します][1]。
-より頻繁にテストを行い、ユーザーに最高のメッセージが表示されるという確信度を高めましょう。 

インテリジェントセレクションは、複数回送信が予定されているキャンペーンに最適です。キャンペーンの調整を開始するには初期結果が必要です。そのため、一度だけ送信するキャンペーンは効果がありません。このようなキャンペーンでは、[A/B テストのほうが効果的です][1]。

## キャンペーンにインテリジェントセレクションを追加するにはどうすればいいですか？

### キャンペーンインテリジェントセレクション
インテリジェントセレクションは、Braze キャンペーンコンポーザーの **Target Audiences** ステップで任意のマルチセンドキャンペーンに追加できます。一度しか送信しないキャンペーンでは、この機能を利用できません。

### キャンバスインテリジェントセレクション
キャンバスにバリエーションを追加するときは、バリエーションの割合のいずれかをクリックします。これにより、バリアント分布を編集し、インテリジェントセレクションを有効にすることができます。

![2 つのバリアントを含むキャンバス。それぞれ 50% のバリアント分布に設定されているため、インテリジェントセレクションを有効にできます。] [2]

キャンバスにコンバージョンイベントをまだ追加していない場合や、キャンペーンがソロバリアントまたはソロコントロールグループで構成されている場合、インテリジェントセレクションは利用できません。

{% alert note %}
24時間以内に再適格性が有効になっているキャンペーンでは、コントロールバリアントの整合性に影響するため、インテリジェントセレクションの使用は許可されません。詳細については、[インテリジェンスに関するよくある質問をご覧ください]({{site.baseurl}}/user_guide/intelligence/faqs/#why-is-re-eligibility-not-available-when-combined-with-intelligent-selection)。
{% endalert %}

## どれくらいの期間稼働しますか？

キャンペーンとキャンバスの場合、バリエーションの「本当の」コンバージョン率に関する十分な証拠が収集されるまで、インテリジェントセレクションが実行されます。「十分」は「後悔」と呼ばれる特別な指標によって決定されます。」 自信に似ていると考えることができます:どのバリアントが最適かを判断するのに十分なデータがあると、インテリジェントセレクションは自動的にオフになります。 

ほとんどの場合、インテリジェント・セレクションはいずれかのバリアントをウィニング・バリアントとして選択します。このバリアントは、今後の送信でオーディエンスの 100% に割り当てられます。

{% alert note %}
インテリジェント・セレクションでは、明確な勝者を1つも選ばずに最適化を中止してしまうことがあります。インテリジェントセレクションは、実験を続けてもコンバージョン率が現在のレートの 1% 以上向上しないという確信度が 95% に達すると、最適化を停止します。
{% endalert %}
