# カスタムエージェントのデプロイ

> カスタムエージェントを作成した後にキャンバスステップs またはカタログ フィールドs で使用するためのカスタムエージェントを配置する方法について説明します。概要については、[Brazeエージェント]({{site.baseurl}}/user_guide/brazeai/agents/)を参照してください。 

{% alert important %}
Braze代理店は現在ベータ版である。始めるには、顧客のサクセスマネージャーにお問い合わせください。
{% endalert %}

## エージェントの使用

エージェントの**Agent Usage** セクションでは、カタログ およびキャンバスでエージェントがアクティブに使用されている場所を参照してナビゲートできます。

![「Agent Usage」セクションには、キャンバスの2 つのアクティブなエージェントと1 つの非アクティブなエージェントが表示されます。]({% image_buster /assets/img/ai_agent/agent_usage.png %})

## キャンバスのエージェント  

エージェントをジャーニーのステップs として使用して、メッセージをパーソナライズしたり、リアルタイムで決定を導くことができます。設定ステップの詳細については、[エージェントステップ]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/agent_step/)を参照してください。

### ユースケース

| ユースケース | 説明 |
| --- | --- |
| リードスコアリングと認定 | エージェントステップを使用して、受信リードをスケール(1-10 など) で評価します。スコアがしきい値を上回るユーザーs を、低フィットのリードを除外しながら、ナーチャパスs にルーティングします。 |
| ダイナミックメッセージパーソナライゼーション | ユーザー 属性や最近の行動に基づいて、主題、商品の推奨事項、メッセージコピーをエージェントに生成させる。レスポンスは、メッセージステップに直接挿入できます。 |
| お客様フィードバック対応 | 顧客のコメントをエージェントに渡して、センチメントを分析し、em パスのエティックフォローアップメッセージを生成します。高値ユーザーs の場合、エージェントはレスポンスをエスカレーションするか、特典を含めることがあります。 |
| インテリジェントルーティング | エージェントアウトプット(ブール値または数値)を使用して、ユーザーを別のキャンバスパスに分割します。たとえば、ユーザーsを「危険にさらされている」または「正常」に分類し、それに応じてメッセージングケイデンスを調整します。 |
| 調査または回答の解釈 | 下流のパスを駆動する構造化された値(たとえば、インテントやニーズを分類する)を返す開封エンドの調査レスポンスやフリーテキストフィールドを、エージェントが解析できるようにします。 |
| マルチステップ推論 | コンテキストフィールドs を組み合わせて、マルチユーザー 属性s に基づいて次善のアクション(メール、SMS、または人間のアウトリーチ) を推奨するなど、複雑な決定を行うようにエージェントを設定します。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

## カタログ中の薬剤  

エージェントをアプリしてs をカタログ フィールドすると、行ごとに自動的に生成または計算されます。エージェントは、将来カタログに追加される新しい行に対しても実行されます。 

### ユースケース

| ユースケース | 説明 |
| --- | --- |
| 製品説明の作成 | たとえば、名前、カテゴリ、機能などの構造化された製品データからキャッチー記述を生成することで、新しいカタログエントリーの短縮マーケティングを自動的に作成します。 |
| 商品属性の充実 | 製品名と詳細に基づいて、カラーファミリー、スタイル、季節などの欠損値を入力します。たとえば、製品名が「Laguna Polarized Sunglasses」の場合、エージェントはスタイルを「スポーツ」、カラーファミリーを「ブルー」として割り当てることができます。 |
| 導出フィールドsの計算 | 既存のフィールドs を使用して、属性s に基づく「フィットスコア」や、販売およびレビューカウントからの「人気タグ」などの新しいデータを生成します。 |
| 項目を分類またはタグする | レコメンドロジックにタグを割り当てることで、パーソナライゼーションモデルがより効果的にプロダクトをSegmentできるようになります。例えば、「アウトドア」「フェスティバルレディ」「プレミアム」などのタグ商品。 |
| コンテンツのローカライズ | カタログの文字をグローバルキャンペーンs用の別の言語に変換するか、リージョン固有のチャネルs用の音色と長さを調整します。たとえば、「Classic Clubmaster Sunglasses」を「Gafas de sol Clubmaster」としてスペイン語に変換するか、SMS キャンペーンの記述を短くします。 |
| 審査又はフィードバックの要約 | 感情やフィードバックを新しいフィールドにまとめます。たとえば、ポジティブ、ニュートラル、ネガティブなどの感情スコアを割り当てたり、「ほとんどの顧客は非常にフィットしていますが、発送が遅いことに注意してください」などの短い文章サマリーを作成したりします。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

### ステップ

![カタログ フィールド内のエージェントステップ。]({% image_buster /assets/img/ai_agent/agent_in_catalog.png %}){: style="float:right;max-width:30%;margin-left:15px;"}

カタログ フィールドにエージェントを追加するには:

1. カタログで、新しいフィールドを追加します。  
2. **Apply AI agent**を選択します。
3. このフィールドにエージェントを割り当てます。  
4. 入力として渡す列を選択します。いずれも選択されていない場合、エージェントはカタログ内のすべての列にアクセスできます。  
5. カタログの行が更新dの場合にフィールドsを再計算するかどうかを決定します。このオプションを選択しない場合、エージェントは行ごとに1 回だけ実行されます。
6. **Add フィールド s**を選択してエージェントをデプロイし、コスト見積もりを確認します。**コスト見積もり**モーダルは、エージェントがこのカタログで実行される回数を示します。これは、行の総数にほぼ等しくなります。続行するには、**Confirm**を選択します。

### カタログエージェントの実行方法  

起動後、エージェントは、選択した列をコンテキストに入れて各行を実行し、評価して出力を生成します。エージェントは、エージェントのデプロイ後に追加されたすべての新しい行で実行されます。**カタログ行更新**時に再計算を選択した場合、このフィールド 更新のすべての値は、存在する送信元フィールドが変更された場合に変更されます。

エージェントを使用するカタログのフィールドを更新および変更できます。列からエージェントを削除するには、**Apply AI agent** を選択解除します。これにより、列は非エージェント列に戻り、フィールドは、エージェントアプリが最後にカタログで実行されたときの最新の値を保持します。

カタログの循環参照はサポートされていません。つまり、次のような状況は発生しません。

- Agentic Column 1では、Agentic Column 2を入力として使用します
- Agentic Column 2では、Agentic Column 1を入力として使用します

![カタログ フィールドの場合、"Apply AI agent"を選択します。]({% image_buster /assets/img/ai_agent/edit_agent_column.png %}){: style="max-width:80%;"}

{% alert note %}
ベータ期間中、カタログエージェントは、1 行につき最大25KB の入力値の処理に制限されます。
{% endalert %}

#### レスポンスフィールドの定義

エージェントが出力形式として[フィールド s]({{site.baseurl}}/user_guide/brazeai/agents/creating_agents/#fields)を使用している場合は、**レスポンスフィールド**のエージェントから対応するフィールドを選択してカタログ フィールドで使用できます。 

たとえば、次のフィールドs を含むカタログに商品説明を追加して、出力形式を構成するエージェントがあるとします。

| フィールド名 | 値 |
| --- | --- |
| **記述** | テキスト |
| **confidence_score_out_of_ten** | 数値 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

**product_description** という名前のフィールドをカタログに追加し、**description** を**Response フィールド** として選択して、列にエージェントの説明を入力できます。

!["Descriptor"エージェントアプリが嘘のフィールド"product_description"。"description"出力がレスポンスフィールドとして選択されます。]({% image_buster /assets/img/ai_agent/response_field.png %}){: style="max-width:80%;"}

また、**Edit Item**を選択し、エージェント生成の説明を編集で更新することで、エージェント生成のセルを手動で上書きすることもできます。編集内容をエージェント生成の説明に戻すには、セルのリフレッシュ記号を選択します。

### カタログsの異常処理  

- 失敗したカタログ呼び出しは再試行されません。
- 基礎モデルプロバイダーへのAPI 呼び出しが不正なAPI キー エラーやレート制限 エラーなどのエラーを返す場合、フィールド は更新しません。
- 失敗した実行の詳細については、エージェントのログを確認できます。
