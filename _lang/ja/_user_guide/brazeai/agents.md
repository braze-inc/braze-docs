# エージェントコンソール内のBrazeエージェント

> Brazeエージェントは、Braze内で作成できるAI搭載のヘルパーだ。エージェントはコンテンツを生成し、知的な判断を下し、データを充実させることができる。これにより、よりパーソナライズされたカスタマーエクスペリエンスを提供できるのだ。

{% alert important %}
Brazeエージェントにアクセスし利用するには、メッセージクレジットが必要だ。現在メッセージクレジットがなく、Braze Agentsを利用したい場合は、次のステップについてアカウントマネージャーに連絡すること。
{% endalert %}

## なぜBrazeエージェントを使うのか？

Brazeエージェントは、追加作業なしで、よりスマートでパーソナライズされた体験をチームが提供することを支援する。それらは自律的なエージェントとして振る舞い、単なるプロンプトへの応答にとどまらず、文脈を理解し、意思決定を行い、目標に向けてアクションを起こす。

実際には、エージェントは件名や製品内のテキストといったメッセージ文面を自動生成できる。これにより、顧客一人ひとりに合わせたコミュニケーションが提供されるのだ。またリアルタイムで適応し、好みや行動、その他のデータに基づいて人々を異なるキャンバスのパスに誘導することもできる。

メッセージングを超えて、エージェントは商品やプロファイルのフィールド値を計算または生成することでカタログを充実させ、データを常に最新の状態に保ち、ダイナミックな状態で維持できる。反復のまたは複雑な作業を引き受けることで、チームは手動設定ではなく戦略と創造性に集中できるようになる。Brazeエージェントは、バックグラウンドプロセスというよりは協力者のように振る舞う。問題を解決し、大規模な影響をもたらす手助けをするのだ。

### Brazeエージェントと他のBrazeAI機能の使い分け

ユーザーの特定の状況に基づいて、コンテンツをその場でパーソナライズされた形で提供するためにエージェントを使用する。例えば、エージェントが特定のユーザーのお気に入りのアイスクリームの味がチョコレートで、お気に入りのトッピングがグミベアだと知っている場合、そのユーザーがキャンバスを通過する際に、その組み合わせに特化したプッシュ広告を作成できる。

しかし、そのエージェントは試行錯誤を通じて学習せず、測定し最大化しようとする究極のマーケティング目標について全く認識していない。たとえコンバージョンを促進する文章を概ね書くように指示しても、そのエージェント的な文章がコンバージョンに与える影響を「監視」し、そのデータを将来のエージェント的な呼び出しに統合する仕組みは備えていない。これは報酬ベースのAI意思決定ではなく、「雰囲気」による意思決定と考えることができる。

一方、他のBrazeAIツールは、測定対象の指標を最大化するように設計されている。例えば、エージェントは、ユーザーの特性が特定の行動を起こす可能性や特定の製品を好む傾向にどう影響するかを、質的に評価するのが非常に得意だ。しかし、エージェントは試行錯誤を通じて学習しないため、確率を予測する精度を測る方法も、時間の経過とともに信号を改善する方法も全く理解していない。したがって、予測精度の正確さと時間の経過に伴う改善度で評価した場合、Predictive Suiteの使用はAgentステップを上回る性能を発揮する。

## 機能

Brazeエージェント向けの機能には以下が含まれる：

- **柔軟な設定：**Brazeが提供するLLMを使用するか、独自の[AIモデルプロバイダー]({{site.baseurl}}/partners/ai_model_providers)（OpenAI、Anthropic、Google Geminiなど）を接続する。
- **シームレスな統合：**エージェントを直接、キャンバスのステップやカタログフィールドに配置する。
- **テストとログ記録ツール：**エージェントを起動する前に、サンプル入力でテストして出力をプレビューせよ。エージェントが実行されるたびに、その実行時の入力と出力を含むログを表示する。
- **使用コントロール：**日次制限はパフォーマンスとコストの管理に役立つ。

## Brazeエージェントについて

エージェントは、その動作を定義する指示（システムプロンプト）で構成される。エージェントが実行されると、それはあなたの指示と渡されたデータを使って応答を生成する。

### 主要な概念

| 用語 | 定義 |
| --- | --- |
| [モデル]({{site.baseurl}}/user_guide/brazeai/agents/reference/#models) | この場合のエージェントの「頭脳」とは、大規模言語モデル（LLM）である。入力の解釈、応答の生成、推論の実行を行う。より強力なモデル（より関連性の高いデータでトレーニングされたもの）は、エージェントの能力と汎用性を高める。 |
| [指示]({{site.baseurl}}/user_guide/brazeai/agents/reference/#writing-instructions) | エージェント（システムプロンプト）に与えるルールやガイドライン。それらはエージェントが実行されるたびにどのように振る舞うべきかを定義する。明確な指示はエージェントをより信頼性が高く予測可能にする。 |
| コンテキスト | 実行時にエージェントに渡されるデータは、それがどこにデプロイされていても、ユーザープロファイルフィールドやカタログ行などである。この入力は、エージェントが出力を生成するために使用する情報を提供する。 |
| [出力変数]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/agent_step/#define-the-output-variable) | キャンバスステップで使用された際にエージェントが生成する出力。出力変数は、パーソナライゼーションされたコンテンツを提供したりワークフローのパスを誘導したりするために、エージェントの結果を保存する。出力変数は文字列、数値、またはブール値のデータ型である。  |
| [実行](#limitations) | エージェントの単一の実行。これは1日の制限にカウントされる。 |
| [出力フォーマット]({{site.baseurl}}/user_guide/brazeai/agents/creating_agents/#select-output) | エージェントの応答の事前定義されたデータ構造。 |
| [温度]({{site.baseurl}}/user_guide/brazeai/agents/reference/#temperature) | エージェントの出力に対する偏差のレベル。これはエージェントがどれほど正確か、あるいは創造的かを決める。 |

## 制限事項

以下の制限が適用される：

- 各エージェントにはデフォルトで1日あたり25万回の実行制限が設定されている。これは最大で1日あたり100万回まで増やせる。この制限を増やしたい場合は、顧客サクセスマネージャーに連絡すること。
- デフォルトでは、各実行は15秒以内に完了しなければならない。15秒後、エージェントはレスポンス`null`を返す。それが使用される場所である。
    - エージェントが頻繁にタイムアウトする場合は、Brazeのアカウントマネージャーに連絡してこの制限値を引き上げてもらうこと。
- 入力データはリクエストごとに25KBまでだ。長い入力は切り詰められる。

## 私のデータはどのように利用され、Brazeが提供するLLMに送信されるのか？

Brazeが提供するLLMを活用しているとBrazeが識別したBraze AI機能を通じてAI出力を生成するため（以下「出力」）、Brazeはお客様のシステムプロンプトまたはその他の入力（該当する場合、以下「入力」）をBraze提供のLLMに送信する。該当するBraze提供のLLMに送信されたデータは、Braze提供のLLMのトレーニングや改善には使用されない。君とBrazeの間では、Outputは君の知的プロパティだ。Brazeは、当該出力物について著作権所有権を主張しない。Brazeは、AIによって生成されたコンテンツ全般（出力物を含む）に関して、いかなる種類の保証も行わない。

Brazeエージェント向けにBrazeが提供するLLM「Auto」の識別子は「Google Geminiモデル」である。GoogleはBrazeを通じて送信された入力データと出力データを55日間保持する。その後、データは削除される。
