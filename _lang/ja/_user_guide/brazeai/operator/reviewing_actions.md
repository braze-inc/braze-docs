# BrazeAI Operator のアクションの確認

> BrazeAI Operator<sup>TM</sup> がダッシュボードで変更を提案したときの、確認と承認の方法を説明します。

![確認のため提案されたアクションカードを表示しているオペレータ。]({% image_buster /assets/img/operator/suggested_actions.png %}){: style="max-width:40%; border:none; float:right; margin-left:15px;"}

## アクションカードの仕組み

オペレータがダッシュボードで変更を提案するとき（フォーム入力、設定の更新、画像の生成など）、各変更は確認用のアクションカードとして表示されます。

1. **オペレータが計画を要約:** アクションカードを表示する前に、オペレータが実行予定の内容を説明します。
2. **個別のアクションカードが表示:** 提案された変更はそれぞれ別のカードになり、ダッシュボードで変更したい内容が示されます。既存の値を変更する場合は、変更前と変更後の値が並べて表示されます。
3. **確認して承認:** 各カードを確認し、承認または却下します。
4. **アクションの実行:** 承認したアクションは Braze で実行されます。却下したアクションは適用されません。

承認後にアクションが失敗した場合は、オペレータが失敗の詳細を通知します。

### 対応範囲

アクションカードは次のエディタで利用できます。

- アプリ内メッセージ（従来型エディタのみ）
- コンテンツカード
- メール（HTML エディタのみ）
- プッシュ通知
- SMS/MMS/RCS
- ウェブフック

上記以外のページでは、オペレータは自らアクションを実行する代わりに、UI で従う手順の一覧を提供します。オペレータの機能は随時拡張されており、作成ツールの対応範囲の拡大が予定されています。

## 計画の変更

オペレータの計画を変えたい場合は、まず保留中のアクションを承認または却下してから、新しいチャットメッセージで希望する変更を説明します。

承認したアクションはオペレータから取り消せません。オペレータに新しい変更を伝えるか、ダッシュボードで手動で変更してください。

## アクションの自動承認

**アクションの自動承認** トグルは、オペレータのチャットパネルにあります。

- **オン:** オペレータが提案したアクションは、手動承認なしで即座に実行されます。画像の生成やワークスペースレベルの設定変更など、安全のため明示的な承認が必要なアクションもあります。
- **オフ（デフォルト）:** 提案されたアクションは、いずれも上記の手動確認フローに従います。

![オペレータのチャットパネルにある自動承認トグルと確認モーダル。]({% image_buster /assets/img/operator/auto-approval_toggle.png %}){: style="max-width:50%;"}

自動承認は、ページの再読み込み、新しいタブを開く、ログアウトして再ログインするとリセットされます。ダッシュボード内で別のページに移動してもリセットされません。自動承認はいつでもオフにできます。
