# BrazeAIオペレータのアクションの確認

> BrazeAI オペレータ<sup>TM</sup> がダッシュボードの変更を提案する場合に、ローブアクションs を確認およびアプリする方法について説明します。

![審査のためのアクション カード案を提示する運営者。]({% image_buster /assets/img/operator/suggested_actions.png %}){: style="max-width:40%; border:none; float:right; margin-left:15px;"}

## アクション カードの仕組み

オペレータがダッシュボードの変更(フォームフィールドsの入力、設定sの更新、"画像sの生成など)を提案すると、それぞれの変更が確認のアクション カードとして提示されます。

1. **オペレータは計画を要約します。**オペレータは、アクション カード s を表示するまでに行う予定を説明します。
2. **個々のアクション カードs アプリイヤー:**提案されたそれぞれの変更は、オペレータがダッシュボードで変更または実行したいことを示す別個のカードとして表示されます。既存の値への変更については、比較のために前の値と提案された値の両方が並べて表示されます。
3. **確認とアプリ手順:**それぞれのカードを確認し、アプリがローブするか、拒否します。
4. **アクションが実行されます。**承認されたアクションはBrazeで実行されます。衰えたアクションsはアプリ嘘ではない。

アプリが無効になった後にアクションが失敗した場合、オペレータは障害の内容を通知します。

### 可用性

動作カードs は、以下のエディタでサポートされています。

- アプリ中の伝言メモ(従来のエディタのみ)
- コンテンツカードによって促進された
- メール(HTMLエディタのみ)
- プッシュ通知
- SMS/MMS/RCS
- Webhook

他のページでは、オペレータはアクションそのものを使用するのではなく、ユーザーインターフェイスでフォローするステップの一覧を提供します。オペレータの機能は定期的に改善されており、工具作成のためのc超過料金の拡大が期待されています。

## 計画の修正

オペレータのプランを変更するには、最初にアプリ が保留中のアクションs を許可または拒否します。次に、新しいチャットメッセージで希望する変更を説明します。

承認されたアクションはオペレータ経由で元に戻すことはできません。オペレータに新しい変更を説明するか、ダッシュボードで手動で変更を行います。

## アクションを自動承認する

**Auto-アプリ rove アクション s** トグルはオペレータチャットパネルにあります。

- **点灯:**オペレータの推奨アクションは、手動によるアプリ承認を必要とせずにすぐに実行されます。一部のアクションでは、"画像の生成やワークスペースレベルの設定の変更など、安全のために明示的なアプリ許可が必要です。
- **消灯(デフォルト):**提案されたすべてのアクションは、記載されている手作業による見直しの過程に従う。

![オペレータチャットパネルの自動アプリローブ切り替えと確認モーダル。]({% image_buster /assets/img/operator/auto-approval_toggle.png %}){: style="max-width:50%;"}

自動アプリローブは、ページを更新したり、新しいタブを開封したり、ログアウトしてから再度ログインするとリセットされます。ダッシュボード内のページ間を移動してもリセットされません。自動アプリローブはいつでも無効にできます。
