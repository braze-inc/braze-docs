# コンテンツオプティマイザー

> コンテンツオプティマイザーは、AIを活用して大量のコンテンツバリアントを自動生成・評価し、メッセージコンテンツを大規模にテスト・最適化する支援を行うエージェントである。

{% alert important %}
コンテンツオプティマイザーは現在ベータ版であり、メールメッセージでのみ利用可能だ。利用開始に関するサポートが必要な場合は、顧客サクセスマネージャーに連絡する。
{% endalert %}

## コンテンツオプティマイザーについて

コンテンツオプティマイザーは、キャンバスステップ内で動作するエージェントである。メッセージのテスト対象となる構成要素を定義し、生成AIや手動入力でバリアントを生成し、ユーザーに送信するコンテンツの組み合わせを自動的に最適化するのに役立つ。この機能は次のことを可能にする：

- メールの件名、ヘッダー、本文、または主要なCTAを最適化する。
- 手動でのABテスト設定なしに、メッセージのパフォーマンスを継続的に改善する。
- AIを活用したアイデア創出により、大量のコンテンツバリアントを迅速にテストする。
- パフォーマンスの低いコンテンツは自動的に段階的に廃止し、成功しているコンテンツは拡大する。

[コンテンツ]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/content_optimizer_step/)オプティマイザーステップの作成方法を学習する。

## ユースケース

### メール

| 最適化のユースケース | 目標 | 説明 |
| --- | --- | --- |
| 件名のバリエーション | 開封率を上げる | テストトーン、緊急性、パーソナライゼーション、そして絵文字の使用。 |
| ヘッダーのメッセージングスタイル | エンゲージメントを高める | 本文ヘッダーにおける感情に訴えるメッセージング、価値を重視したメッセージング、明確なメッセージングを比較せよ。 | 
| 本文の形式 | 読みやすさとエンゲージメントを高める | ストーリーテリングと機能リスト、箇条書きと段落、そしてコンテンツの長さを比較してみよう。 |
| CTAコピーの&トーン | クリックスルーを上げる | アクションを促す表現、利益を強調する表現、一人称のCTA表現を比較せよ。 |
| テーマ別コンテンツの組み合わせ | パフォーマンスの高い組み合わせを発見する | テーマに沿った件名、本文、CTAの構成要素を組み合わせて、最適な組み合わせを見つけ出すのだ。 |

## 仕組み

コンテンツオプティマイザーは、文脈に応じた or 状況に即した[マルチアームドバンディット](https://en.wikipedia.org/wiki/Multi-armed_bandit)アルゴリズムを用いて、高パフォーマンスのバリアントへの配信を増加させ、低パフォーマンスのバリアントへの配信を減らす。時間の経過とともに、これによりメッセージングの内容が継続的に改善される。手動での介入は最小限で済む。

Brazeの独自開発のバンディット最適化アルゴリズムは、コンテンツオプティマイザステップの組み合わせ的性質に特化して構築されている。各メッセージは複数の構成要素から成り立っているため、バンディットは各要素（件名、本文、CTAなど）のパフォーマンスと、それらを組み合わせてメッセージとした際の相互作用を同時に学習する。より具体的に言えば、ある組み合わせが送信されると、同じ構成要素を共有する全ての組み合わせがその送信データから恩恵を受ける。これにより、標準的なバンディットアルゴリズムと比較して、同じ量のデータでバンディットがはるかに速く学習できるようになる。

ステップが最初に起動すると、コンテンツオプティマイザーは初期のパフォーマンスデータを収集するために、ランダムにバリアントを送信する。この初期の探索期間の後、アルゴリズムはパフォーマンスの高いコンテンツの組み合わせへトラフィックをシフトし始め、パフォーマンスの低い選択肢への割り当てを徐々に減らしていく。探索期間中は、トラフィックは一般的に利用可能なバリアント全体に分散される。これによりアルゴリズムはそれらの相対的なパフォーマンスから学習できる。

コンテンツオプティマイザーは、キャンバスのメッセージステップに似ている。静かな時間帯、[インテリジェントタイミング]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_timing)、イベントログ記録といった機能を備えている。コンテンツオプティマイザーのステップを設定するには、ベースメッセージを作成し、最適化するコンテンツコンポーネント（件名、本文テキスト、行動喚起など）を定義する。各コンポーネントのバリアントはAIで生成するか手動で入力できる。また、ベースメッセージにLiquidタグを追加し、コンポーネントをメッセージ内容にマッピングする必要がある。

各ユーザーは、コンテンツ最適化ステップのエントリごとに1通のメッセージを受け取る。再入は新規として扱われ、以前のバリアントに関する記憶は一切保持されない。

最良の結果を得るには、ユーザーが段階的にステップを入力するキャンバスでコンテンツオプティマイザーを使用すること。例えば、毎日一定のボリュームがある定期的なキャンバスや常時稼働のキャンバスが該当する。もし全てのユーザーが一斉にステップに入ると、エージェントは初期の結果から学習する時間がなくなる。このステップは、ライブ最適化エンジンというよりは、静的なABテストに近い動作をする。

これは、単発送信や短期キャンバスでもコンテンツオプティマイザーは使用できるが、ユーザーが長期間にわたりステップに入力する場合に限られることを意味する（例えば、遅延ステップ、スケジュールされたエントリ、APIトリガーによるフローなど）。ステップには十分なトラフィックと時間が確保されていることを確認せよ。パフォーマンスの違いを観察する前に、ほとんどのユーザーに到達する前に。

### 主要な概念

| 用語                    | 説明 |
|-------------------------|-------------|
| ベースメッセージ   | すべての送信設定を含む、バリアントが構築されるメインメッセージテンプレート。 |
| コンテンツコンポーネント  | メッセージ内の要素（例えば件名や主要なCTAなど）は、テストや最適化が可能だ。マーケターは、コンポーネントを表示すべき箇所に、関連するLiquidタグをメッセージに挿入しなければならない。 |
| コンテンツバリアント    | コンテンツコンポーネントが取り得る異なる値。 |
| コンテンツの組み合わせ| コンテンツのバリアントを組み合わせて作成されたユニークなメッセージ。 |
| 最適化イベント       | コンテンツオプティマイザーがパフォーマンスを評価し、時間の経過とともにコンテンツの組み合わせにトラフィックを割り当てる方法を決定する。例えば、メールのクリック数や開封数などである。ステップ内の全てのコンテンツコンポーネントに適用される。コンテンツオプティマイザーはこのイベントから継続的に学習し、パフォーマンスの高いコンテンツの組み合わせに向けて配信を自動的にシフトする。 |

## 制限事項

- コンテンツオプティマイザーは現在ベータ版であり、メールメッセージでのみ利用可能だ。
- エージェントは1ステップあたり最大125通りの組み合わせを生成できる：
   - 1ステップあたり最大3つのコンポーネント
   - 各部品につき最大5種類のバリアント
- ユーザーごとに、エントリごとに、メッセージは一つだけ送信される。再入力のための過去の送信記録は存在しない。
- マーケターは、定義されたコンテンツコンポーネントのバリアントを表示すべきメッセージ作成画面内の各コンポーネントに、手動でLiquidタグを挿入しなければならない。
