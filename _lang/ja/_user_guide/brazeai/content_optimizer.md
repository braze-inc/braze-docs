# コンテンツオプティマイザー

> コンテンツオプティマイザは、大量のコンテンツバリアントを自動的に生成および評価するためにAI を使用して、大規模なメッセージコンテンツのテストおよび最適化を行うエージェントです。

{% alert important %}
コンテンツオプティマイザは現在ベータ版になっており、メールメッセージでのみ使用できます。始めるには、顧客のサクセスマネージャーにお問い合わせください。
{% endalert %}

## コンテンツ最適化ルーチンについて

コンテンツオプティマイザは、キャンバスステップで実行されるエージェントです。これは、テストするメッセージコンポーネントを定義し、Generative AIまたは手動入力を使用してバリアントsを生成し、ユーザーsに送信されるコンテンツの組み合わせを自動的に最適化するのに役立ちます。この機能は、次の場合に役立ちます。

- サブジェクト行、ボディヘッダー、ボディコンテンツ、またはプライマリCTA をメールs 用に最適化します。
- A/Bテストを手動で設定せずに、メッセージパフォーマンスを継続的に改善します。
- 大量の内容物バリアントsを迅速に試験し、AIを利用してアイデアイオンを得る。
- パフォーマンスの低いコンテンツを自動的にフェーズアウトし、勝者をスケールアップします。

[Content Optimizer ステップ]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/content_optimizer_step/) を作成する方法について説明します。

## ユースケース

### メール

| 最適化ユースケース | 目標 | 説明 |
| --- | --- | --- |
| サブジェクトラインのバリエーション | 開封率の向上 | 試験音、緊急性、パーソナライゼーション、絵文字の使用。 |
| ヘッダーメッセージング形式 | ブーストエンゲージメント | 感情的、価値主導、明確なメッセージングを身体ヘッダーで比較する。 | 
| 本文の内容形式 | 読みやすさとエンゲージメントの向上 | ストーリーテリングと機能リスト、箇条書きと段落、およびコンテンツの長さをテストします。 |
| CTAコピー&トーン | クリックスルーを上げる | アクション主導、ベネフィット重視、一人称CTA表現を比較する。 |
| テーマ付きコンテンツの組み合わせ | 高性能な組み合わせを発見 | テーマのあるサブジェクト、ボディ、およびCTAコンポーネントをミックスしてマッチングし、最適な全体的な組み合わせを見つけます。 |

## 仕組み

コンテンツオプティマイザは、非文脈に応じた or 状況に即した[マルチアームバンドIT](https://en.wikipedia.org/wiki/Multi-armed_bandit)アルゴリズムを使用して、パフォーマンスの高いバリアントに送信を割り当て、パフォーマンスの低いものへの割り当てを減らします。これにより、メッセージの内容が継続的に改善され、手動による介入は最小限に抑えられます。

ステップが最初に起動すると、コンテンツ最適化ルーチンはバリアントをランダムに送信して、最初のパフォーマンスデータを収集します。この最初の探索期間の後、アルゴリズムはトラフィックをパフォーマンスの高いコンテンツの組み合わせにシフトし始め、パフォーマンスの低いオプションへの割り当てを徐々に減らします。探索期間中、トラヒックは一般に利用可能なバリアントsに分散され、アルゴリズムがそれらの関連パフォーマンスから学習できるようにします。

コンテンツ最適化ルーチンは、キャンバスのメッセージステップに似ています。静かな時間、[インテリジェントタイミング]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_timing)、イベントログなどの機能があります。基本メッセージを作成し、最適化するコンテンツコンポーネント(件名、本文、通話先アクションなど)を定義することで、コンテンツ最適化ステップを設定できます。コンポーネントごとのバリエーションは、AI を使用して生成するか、手動で入力することができます。また、コンポーネントをメッセージ内容にマップするには、ベースメッセージにリキッドタグs を追加する必要があります。

それぞれのユーザーは、エントリごとに1つのメッセージをコンテンツオプティマイザステップに受信します。再エントリーは新しいものとして扱われ、以前のバリアントsのメモリーはありません。

最適な結果を得るには、キャンバスでコンテンツ最適化ルーチンを使用します。この場合、ユーザーは、定期的なキャンバスや常時オンのキャンバスなど、毎日一定の量でステップに徐々に入力されます。すべてのユーザーsが一度にステップに入ると、エージェントは初期の成果から学ぶ時間がなくなる。ステップは、ライブ最適化エンジンよりもスタティックA/Bテストのように動作します。

つまり、コンテンツオプティマイザは、シングルセンドまたは短期キャンバスでも使用できますが、ユーザーが長期間(遅延ステップ、スケジュールされた エントリ、API トリガーの流れなど) にわたってステップに入っている場合に限ります。ほとんどのユーザーs に到達するまでに、ステップにパフォーマンスの違いを監視するのに十分なトラフィックとタイムがあることを確認します。

## 制限事項

- コンテンツオプティマイザは現在ベータ版になっており、メールメッセージでのみ使用できます。
- エージェントは、ステップごとに最大125 の組み合わせを生成できます。
   - 1 ステップにつき最大 3 コンポーネント
   - コンポーネントごとに最大5 バリアント s
- 1つのエントリにつき1つのユーザーのみが送信されます。再入力する前のセンドのメモリがありません。
- マーケターは、定義されたコンテンツコンポーネントバリアントがレンダリングされるメッセージ作成画面に、コンポーネントごとに液体タグs を手動で挿入する必要があります。
