# 意思決定エージェントの設計

> このリファレンス記事は、意思決定エージェントの設計と設定に関する主要な概念とベストプラクティスを扱う。

## 意思決定エージェントについて

意思決定エージェントの設計は、Decisioning Studioの設定における最初のステップだ。意思決定エージェントが判断を下せるようにするには、最大化したい結果を定義し、そのためにエージェントが取れるアクションを定義する必要がある。

### 主要な概念

以下の用語は、Decisioning Studioガイド全体で参照される。

| 用語 | 定義 |
| --- | --- |
| **意思決定エージェント** | 意思決定エージェントとは、特定のビジネス目標を達成するためにカスタマイズされた、BrazeAI Decisioning Studio™ のカスタム設定である。これは、選択した成功指標、次元、およびオプションによって定義される。 |
| **成功指標** | 最適化したい具体的なビジネス指標。例えば、収益、コンバージョン、またはユーザーあたりの平均収益（ARPU）など。これは意思決定エージェントがそのアクションを通じて最大化を目指す指標である。 |
| **ディメンション** | 次元とは、意思決定エージェントが成功指標を最大化するために操作できる*手段の種類*と考えることができる。典型的な要素には、オファー、件名、クリエイティブ、チャネル、送信時間などが含まれる。 |
| **アクション銀行** | アクションバンクは、意思決定エージェントが各次元「レバー」に対してアクセス可能な*具体的な選択肢*を定義する。例えば、チャネル次元については、意思決定エージェントがアクセスできる特定のチャネルを定義する。オファー次元では、意思決定エージェントがテストできる特定のオファーを定義する。 
| **制約** | 一般的に、意思決定エージェントは、アクションバンクに設定したアクションの組み合わせを自由に実行できる。ただし、重要なビジネスルールを遵守させるため、意思決定エージェントのアクションを制限する制約を定義することもできる。例えば、特定のオファーが対象外の地域にいる顧客に選択されないようにしたり、意思決定エージェントが支出できる最大予算を設定したりすることが挙げられる。

## 意思決定エージェントの設計方法

意思決定エージェントを設定する際には、主に四つの設計要素について検討する必要がある：

### 「目標」：成功の指標を定義せよ

> エージェントに最大化させたい結果は何か？

成功指標とは、エージェントが最適化を目指すビジネス成果である。これは直接的に事業目標と一致させるべきだ。クリック数や開封率のような代理指標ではなく、収益、コンバージョン、ARPU、顧客生涯価値といった実際の事業成果である。

### 「誰が」: オーディエンスを選ぶ

> 意思決定エージェントは誰をエンゲージメントさせるのか？

エージェントが対応するオーディエンスを定義せよ。これは全顧客、特定のセグメント（例えばロイヤルティプログラム会員）、あるいはライフサイクルの特定の段階にある顧客（例えば最近購入した顧客や解約リスクのあるサブスクライバー）を指す可能性がある。

### 「何を」：アクションバンクを設定する

> エージェントは結果を導くために、どのような選択肢から選ぶことができるのか？

アクションバンクは、エージェントが操作できる全ての手段を定義する。つまり、チャネル、オファー、タイミング、頻度といった次元と、各次元内の具体的な選択肢である。エージェントはこれらのオプションの様々な組み合わせを試行錯誤し、各顧客にとって最適な方法を見つける。

### 「どのように」: 制約を設定する

> エージェントはどんなルールに従うべきか？

制約とは、エージェントが従わなければならない規則である。これは、対象外の地域にいる顧客に対して特定のオファーが選択されないようにする、あるいは意思決定エージェントが支出できる最大予算を設定することを意味する。

## ベストプラクティスと事例

意思決定エージェントの効果を最大化するには、次のことを行うべきだ：

- ビジネス目標や目的に密接に関連する成功指標を選ぶこと。例えば、収益、コンバージョン、ARPUなどだ。
- 成功指標に最も大きな影響を与える可能性が高い要素、つまり「レバー」に焦点を当てる。例えば、オファー、件名、クリエイティブ、チャネル、送信時間などがそれにあたる。
- 各次元（例えばメールとSMS、あるいは毎日と毎週の頻度など）について、成功指標に最も大きな影響を与える可能性が高い選択肢を選べ。

- Brazeでオーディエンスとメッセージを定義する
- Decisioning Studioは毎日自動的に実験を実行し、顧客ごとにメッセージ、送信時間、割引、プランオファーの異なる組み合わせをテストする。
- BrazeAI™は、どの顧客が飛び級オファーに反応しやすいか、またどの顧客がアップグレードのために割引やその他のインセンティブを必要とするかを学習する。

- Brazeでオーディエンスとメッセージを定義する
- Decisioning Studioは毎日自動的に実験を実行し、顧客ごとに異なる更新オファーをテストする。
- BrazeAI™は、価格に敏感でない顧客を識別する。これらの顧客は、更新に際して大幅な割引を必要としない。

- BrazeAI™は、個々の顧客ごとに最適な組み合わせを発見する。
