# データソースの準備

> このリファレンス記事では、AI デシジョンループをクローズし、エージェントが継続的に学習および改善できるようにするために必要な重要なフィードバックデータアセットについて説明します。

## AI デシジョンループを閉じる

すべての顧客データはエージェントにとって重要ですが([Connect データソース s]({{site.baseurl}}/user_guide/brazeai/decisioning_studio/decisioning_studio_pro/connect_data_sources/) を参照)、最も重要なデータアセットは、カスタマーエンゲージメントの決定が送信された後のh アプリをエージェントに伝えるものです。

これらのアセットは、エージェントが学習できるフィードバックループを作成します。

{% alert note %}
エージェントがカスタマーエンゲージメント プラットフォーム(Braze、SFMC、Klaviyoなど)とネイティブに統合されている場合、フィードバックデーターに必要な追加の設定ステップがないことがあります。これらは顧客データとともに自動的に送信される可能性があるためです。
{% endalert %}

## 重要なフィードバックデータアセット

フィードバックループを作成するための重要なアセットは3 つあります。

1. 変換データ
2. エンゲージメントデータ
3. アクティベーションデータ

### 変換データ

コンバージョンアセットは、オーケストレーション後に顧客に渡されたhアプリを表します。例えば、あるエージェントが最適化されたキャンペーンsを受け取る顧客の純現在価値(NPV)を最適化していると仮定すると、コンバージョンアセットには、NPVへの変化の日次更新が含まれる可能性がある。

| 必要条件 | 理由 |
|-------------|------|
| 各レコードには、すべてのデータアセットと整合性のある一意の顧客 識別子が含まれます | スタジオを決定するには、レコメンドからアクティベーション、コンバージョンまで、個々のカスタマージャーニーを追跡する必要があります。 |
| 各レコードにはタイムスタンプが関連付けられています | 顧客 アクションsの通信と順序の間の時刻を理解することは、エージェントのトレーニング計算と計量計算にとって非常に大切です。 |
| 非バイナリ(変換されたメトリックと変換されていないメトリックなど)のターゲットメトリックを使用する場合、それぞれのコンバージョンイベントにターゲットメトリック値が提供されます | スタジオの決定では、ターゲットメトリック値を使用して、推奨されるアクションの結果に基づいてエージェントを適切に報酬/ペナルティをアプリするトレーニングな経験を生成します。 |
| コンバージョンs がd と通信(e.g.クーポン償還)を一意に属性できる場合、コンバージョンとアクティベーションを照合するために必要なフィールドs が提供されます | コンバージョンイベントを特定の通信に関連付けることができる場合、これにより、清潔で正確なアトリビューションが可能になります。直接アトリビューションは、エージェントに最も明瞭なシグナルを提供しますが、できない場合(多くの場合)、近接ベースのアトリビューションが使用されます。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

### エンゲージメントデータ

エンゲージメントアセットは、クリック、開封s、および他のインプレッションs を含む顧客のアクション間s を記述します。エンゲージメントデータは、コンバージョンデータに含まれてもよいし、別個のものであってもよい。これは、コンバージョンのデータと同様の役割を果たしており、カスタマーエンゲージメント後にアプリが何を低下させたかを代理人に伝えている。

| 必要条件 | 理由 |
|-------------|------|
| 各レコードには、すべてのデータアセットと整合性のある一意の顧客 識別子が含まれます | スタジオを決定するには、個々の顧客ごとにエンゲージメントの行動を追跡する必要があります。 |
| 各レコードにはタイムスタンプが関連付けられています | 顧客 アクションsの通信と順序の間の時刻を理解することは、エージェントのトレーニング計算と計量計算にとって非常に大切です。 |
| クリック、開封s、または他のエンゲージメント情報を通信に一意に属性できる場合、エンゲージメントとアクティベーションを照合するために必要なフィールドs が提供されます | コンバージョンデーターと同様に、エンゲージメントを特定の通信に関連付けることができる場合、これにより、清潔で正確なアトリビューションが可能になります。ダイレクト・アトリビューションは、エージェントに最もクリアなシグナルを提供します。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

### アクティベーションデータ

アクティベーションアセットは、送信された通信をエージェントに伝えます。これは、多くの場合、オーケストレーションの設定方法によって異なります。エージェントがBraze、SFMC、またはKlaviyoとの直接統合を介してオーケストレーションを行う場合、エージェントは直接的にアクティベーションデータを取得できます。

{% alert note %}
エンゲージメント・データとアクティベーション・データは、同じデータ・アセット内に非常に多く存在します。
{% endalert %}

| 必要条件 | 理由 |
|-------------|------|
| 各レコードには、すべてのデータアセットと整合性のある一意の顧客 識別子が含まれます | スタジオを決定するには、レコメンドからアクティベーション、コンバージョンまで、個々のカスタマージャーニーを追跡する必要があります。 |
| 各レコードにはタイムスタンプが関連付けられています | 顧客 アクションsの通信と順序の間の時刻を理解することは、エージェントのトレーニング計算と計量計算にとって非常に大切です。 |
| 通信内容とアクティベーションイベントを一致させるために必要なフィールドが提供されます(`event_id` など)。 | エージェントアトリビューションとトレーニング処理では、送信する通信の特徴を正しく一致させる必要があります。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}
