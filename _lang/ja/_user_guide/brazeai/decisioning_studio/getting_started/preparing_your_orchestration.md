# オーケストレーションの準備

> この参考記事では、BrazeAI Decisioning Studio™のオーケストレーション設定前に準備すべき事項を説明する。これには、カスタマーエンゲージメントプラットフォーム（CEP）の選択や、必要な認証情報とアセットの収集が含まれる。

## オーケストレーションとは何か？

オーケストレーションとは、Decisioning Studioとカスタマーエンゲージメントプラットフォーム（CEP）との連携を指す。意思決定エージェントが各顧客に対する最適なアクションを決定すると、オーケストレーションがそれらの決定を実行する。具体的には、CEPを通じてパーソナライズされたコミュニケーションをトリガーする。

次の状況を考えてみてください。

- **Decisioning Studioは**、*何をいつ送るか*を決める。
- **お前のCEPが**送信*方法を*処理する

## CEPを選ぶ

最初のステップは、Decisioning Studioと組み合わせて使用するカスタマーエンゲージメントプラットフォームを決定することだ。選択内容によって設定の複雑さと利用可能な機能が異なる。

### サポートされているCEP

| CEP | Decisioning Studio Go | Decisioning Studio Pro | 統合タイプ |
|-----|:---------------------:|:----------------------:|------------------|
| **Braze** | ✓ | ✓ | ネイティブAPI統合（推奨） |
| **セールスフォース マーケティングクラウド** | ✓ | ✓ | APIイベント＋ジャーニービルダー |
| **クラヴィオ** | ✓ | ✓ | APIイベント＋フロー |
| **その他のCEP** | — | ✓ | カスタム（推奨ファイル） |
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3 .reset-td-br-4 role="presentation"}

{% alert tip %}
既にBrazeをCEPとして使用している場合、最もスムーズな設定体験を得るために、ネイティブのBraze統合を利用することを推奨する。
{% endalert %}

## キャンペーンの計画を立てる

オーケストレーションを設定する前に、以下の詳細を考慮せよ：

### 基本テンプレート

基本テンプレートとは、意思決定エージェントが使用する可能性のあるあらゆるメッセージテンプレートを指す。考えてみろ：

- **テンプレートは何個ある？**エージェントは一つのテンプレートでも複数のテンプレートでも扱える。複数ある場合、エージェントは各顧客が受け取るテンプレートをパーソナライズできる。
- **どのチャネルだ？**メール、プッシュ通知、SMS、またはそれらの組み合わせ。各チャネルには、別々のテンプレートとキャンペーンが必要となる場合がある。
- **どんなダイナミックな要素だ？**メッセージのどの部分をエージェントがパーソナライズするか識別する（件名、CTA、オファー、タイミングなど）。これらはAPIトリガープロパティまたはダイナミックなプレースホルダーとなる。

### 再資格設定

キャンペーンでは、ユーザーがメッセージを複数回受け取れるようにすべきだ。

- テストのためには、同じキャンペーンを同じユーザーに繰り返し送信する必要がある。
- 運用中、エージェントは同じキャンペーンが連続する日々にユーザーにとって最適であると判断する可能性がある。

{% alert note %}
テストの再適格性を設定する際、Decisioning Studioエージェントはフリークエンシーキャップを遵守するよう設計されており、本番環境では同じキャンペーンを1日に1回以上ユーザーに送信しない。
{% endalert %}

### APIトリガーのプロパティ

Brazeの統合では、エージェントが最適化する次元を計画せよ。これらはAPIトリガープロパティとなり、ダイナミックな値をキャンペーンに渡す：

| 例示の次元 | APIトリガープロパティ |
|-------------------|---------------------|
| 件名 | {% raw %}`{{api_trigger_properties.${subject_line}}}`{% endraw %} |
| アクションを促す | {% raw %}`{{api_trigger_properties.${cta_message}}}`{% endraw %} |
| オファー | {% raw %}`{{api_trigger_properties.${offer_id}}}`{% endraw %} |
| 割引額 | {% raw %}`{{api_trigger_properties.${discount}}}`{% endraw %} |

## ベストプラクティス

オーケストレーションの準備をする際には、以下のベストプラクティスを心に留めておくことだ：

1. **シンプルに始めろ。**まず1つのチャネルと1つか2つのテンプレートから始める。後で、何がうまくいくか学習したら、拡張すればいい。
2. **徹底的にテストせよ。**公開前に、少数のユーザーで連携機能をテストし、ダイナミックなコンテンツが正しく表示されることを確認せよ。
3. **設定のドキュメントを作成せよ。**キャンペーンID、テンプレートID、API キー、その他の識別子を管理する。これらの参照は、Decisioning Studio ポータルで行う必要がある。
4. **チームと調整しろ。**オーケストレーションの設定には、マーケティング、開発、データチームが関与する場合がある。全員がプロセスにおける自分の役割を理解していることを確認せよ。
5. **フィードバックデータの計画を立てる。**オーケストレーションは単にメッセージを送ることだけではない。エージェントの学習に役立つエンゲージメントやコンバージョンデータを収集することでもある。詳細については[、データソースの準備を]({{site.baseurl}}/user_guide/brazeai/decisioning_studio/getting_started/preparing_your_data_sources/)参照せよ。
