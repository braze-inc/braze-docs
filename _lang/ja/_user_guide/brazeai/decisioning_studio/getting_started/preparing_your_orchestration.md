# オーケストレーションを準備する

> この参考記事では、カスタマーエンゲージメントプラットフォーム(CEP)の選択、必要な認証情報とアセットの収集など、BrazeAI Decisioning Studio™のオーケストレーションを設定する前に準備する必要があることを説明します。

## オーケストレーションとは何か？

オーケストレーションは、Decisioning Studioとカスタマーエンゲージメントプラットフォーム（CEP）をつなぐものである。意思決定エージェントが顧客ごとに最適なアクションを決定すると、オーケストレーションはCEPを通じてパーソナライズされたコミュニケーションをトリガーすることで、その意思決定を実行する。

次の状況を考えてみてください。

- **デシジョニング・スタジオは**、*何を* *いつ*送るかを決定する
- **CEPが**送信*方法を*処理する

## CEPの選択

最初のステップは、Decisioning Studioでどのカスタマーエンゲージメントプラットフォームを使うかを決めることだ。この選択は、セットアップの複雑さと利用可能な機能に影響する。

### サポートされるCEP

| CEP | Decisioning Studio Go | デシジョン・スタジオ・プロ | 統合タイプ |
|-----|:---------------------:|:----------------------:|------------------|
| **Braze** | ✓ | ✓ | ネイティブAPIの統合（推奨） |
| **セールスフォース・マーケティングクラウド** | ✓ | ✓ | APIイベント＋ジャーニービルダー |
| **クラビオ** | ✓ | ✓ | APIイベント＋フロー |
| **その他のCEP** | - | ✓ | 顧客（推奨ファイル） |
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3 .reset-td-br-4 role="presentation"}

{% alert tip %}
すでにCEPとしてBrazeを使用している場合は、最もスムーズなセットアップのためにネイティブBrazeインテグレーションを使用することをお勧めする。
{% endalert %}

## 準備するもの

オーケストレーションを設定する前に、選択したCEPに基づいて以下の項目を収集する。

{% tabs %}
{% tab Braze %}

| 項目 | 説明 |
|------|-------------|
| **REST APIキー** | ユーザーデータ、メッセージ、キャンペーン、キャンバス、セグメンテーション、テンプレートの権限を持つ新しいAPIキー。 |
| **ダッシュボードURL** | BrazeインスタンスのURL（例えば、`https://dashboard-01.braze.com` ）。 |
| **アプリID** | 追跡したいアプリに関連付けられたAPIキー（**設定**>**アプリ設定に**ある）。 |
| **メールの表示名とアドレス** | キャンペーンに使用する送信者情報（**設定**>**メール設定に**ある）。 |
| **ベーステンプレート** | エージェントがオーケストレーションに使用するメッセージテンプレート。テンプレートごとにAPIトリガーキャンペーンを作成する。 |
| **テストユーザーID** | ローンチ前に統合をテストするためのユーザーID。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

{% endtab %}
{% tab Salesforce Marketing Cloud %}

| 項目 | 説明 |
|------|-------------|
| **アプリパッケージ認証情報** | クライアントID、クライアントシークレット、認証ベースURI、RESTベースURI、SOAPベースURIを、サーバー間API統合でインストールされたパッケージから取得する。 |
| **API権限** | チャネル、アセット、オートメーション、ジャーニー、コンタクト、データエクステンション、トラッキングイベントのスコープ。 |
| **データ拡張** | サブスクライバーデータ、エンゲージメントデータ、レコメンデーション用のデータエクステンションが必要になる。 |
| **メールテンプレート** | Decisioning Studioに使用させたいテンプレートと、それぞれのテンプレートID。 |
| **ジャーニービルダーへのアクセス** | APIイベントエントリソースでマルチステップ・ジャーニーの作成と有効化にアクセスできる。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

{% endtab %}
{% tab Klaviyo %}

| 項目 | 説明 |
|------|-------------|
| **秘密APIキー** | イベント、フロー、リスト、メトリクス、プロファイル、テンプレートのフルアクセス権限を持つ新しいAPIキー。 |
| **メールテンプレート** | Decisioning Studioに使用させたいテンプレート。テンプレートはフローに関連付けられていなければならない（この目的のためにプレースホルダーフローを作成することができる）。 |
| **送信者情報** | キャンペーンに使用する送信者名とメール・アドレス。 |
| **フロー・アクセス** | メトリックトリガーを使用したフローの作成と有効化にアクセスできる。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

{% endtab %}
{% tab Other CEPs %}

{% alert note %}
カスタムCEP統合は、Decisioning Studio Proでのみ利用できる。
{% endalert %}

Braze、SFMC、Klaviyo以外のCEPを使用している場合、Decisioning Studio Proはレコメンデーションファイルアプローチで統合できる：

| 項目 | 説明 |
|------|-------------|
| **データ・インジェスト機能** | CEPは、顧客ごとにパーソナライズされた意思決定を含むレコメンデーションファイル（通常はCSVまたはJSON）をインジェストできなければならない。 |
| **ダイナミックなコンテンツのサポート** | キャンペーンは、レコメンデーションデータに基づいてダイナミックなフィールドへの入力をサポートしなければならない。 |
| **顧客開発リソース** | あなたのチームは、レコメンデーションファイルを読み、コミュニケーションをトリガーするための統合を構築する必要がある。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

{% endtab %}
{% endtabs %}

## キャンペーンを計画する

オーケストレーションを設定する前に、以下の詳細を検討してほしい：

### ベーステンプレート

ベーステンプレートとは、メッセージングエージェントが使用する可能性のある、あらゆるメッセージテンプレートのことである。考えてみよう：

- **テンプレートはいくつある？**エージェントは1つのテンプレートでも複数のテンプレートでも対応できる。複数の場合、エージェントは各顧客がどのテンプレートを受け取るかをパーソナライズすることができる。
- **どのチャネルか？**メール、プッシュ、SMS、またはその組み合わせ。チャネルごとに別々のテンプレートやキャンペーンが必要になる場合もある。
- **ダイナミックな要素とは何か？**エージェントがパーソナライズされたメッセージの部分（件名、CTA、オファー、タイミングなど）を識別子で特定する。これらはAPIトリガープロパティやダイナミックなプレースホルダとなる。

### 再資格設定

キャンペーンでは、ユーザーが複数回メッセージを受信できるようにすべきである：

- テスト用には、同じキャンペーンを同じユーザーに繰り返し送信したい。
- 本番では、エージェントは、同じキャンペーンが連続する日にユーザーにとって最適であると判断することができる。

{% alert note %}
テスト用に再適格を設定している間、Decisioning Studioエージェントはフリークエンシーキャップを尊重するように設計されており、本番では同じキャンペーンを1日に1回以上ユーザーに送信することはない。
{% endalert %}

### APIトリガーのプロパティ

Brazeとの統合では、エージェントがどの次元を最適化するかを計画する。これらは、キャンペーンにダイナミックな値を渡すAPIトリガープロパティとなる：

| 寸法例 | APIトリガー・プロパティ |
|-------------------|---------------------|
| 件名 | {% raw %}`{{api_trigger_properties.${subject_line}}}`{% endraw %} |
| アクションを呼びかける | {% raw %}`{{api_trigger_properties.${cta_message}}}`{% endraw %} |
| オファー | {% raw %}`{{api_trigger_properties.${offer_id}}}`{% endraw %} |
| 割引額 | {% raw %}`{{api_trigger_properties.${discount}}}`{% endraw %} |
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

## ベストプラクティス

オーケストレーションの準備として、これらのベストプラクティスを念頭に置いておこう：

1. **シンプルに始めよう。**まずは1つのチャネルと1つか2つのテンプレートから始める。何が効果的かを学びながら、後で拡大することができる。
2. **徹底的にテストする。**設定を開始する前に、少人数のユーザーで統合をテストし、ダイナミックなコンテンツが正しく入力されることを確認する。
3. **セットアップをドキュメント化する。**キャンペーンID、テンプレートID、APIキー、その他の識別子を追跡する。Decisioning Studioポータルでこれらを参照する必要がある。
4. **チームと調整する。**オーケストレーションのセットアップには、マーケティング、開発、データチームが関わることもある。全員が自分の役割を理解していることを確認する。
5. **フィードバックデータの計画を立てる。**オーケストレーションはメッセージを送信するだけではない。エージェントの学習に役立つエンゲージメントやコンバージョンのデータ収集も重要だ。詳細は[データソースを準備するを]({{site.baseurl}}/user_guide/brazeai/decisioning_studio/getting_started/preparing_your_data_sources/)参照のこと。
