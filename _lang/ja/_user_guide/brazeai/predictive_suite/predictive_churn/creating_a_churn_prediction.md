# 解約予測の作成

> Braze ダッシュボード内で解約予測を作成する方法を学びます。

## ステップ 1:新しい予測を作成する

Braze で、[**分析**] > [**解約予測**] に移動します。

予測とは、トレーニング済みの機械学習モデルと、それが使用するすべてのパラメーターおよびデータの1つのインスタンスです。このページでは、現在アクティブな予測の一覧と、それらに関する基本情報が表示されます。ここでは、予測の名前変更、アーカイブ、および新規作成が可能です。アーカイブされた予測は非アクティブとなり、ユーザースコアは更新されません。

新しい予測を作成するには、[**予測を作成**] を選択し、新しい [**解約予測**] を選択します。

{% alert note %}
同時にアクティブにできる解約予測は5つまでに制限されています。解約予測を購入する前は、アクティブなプレビュー解約予測は1つに制限されます。プレビュー解約予測では、スコアの定期的な更新や、予測の出力に基づいたユーザーのターゲティングはできません。詳細については、アカウントマネージャーにお問い合わせください。
{% endalert %}

[**基本設定**] ページで、新しい予測にユニークな名前を付けます。また、この特定の予測に関するメモとして、オプションの説明を入力することもできます。

[**次へ**] を選択して次のステップに進みます。オプションとして、[**今すぐ構築**] を選択すると、すべてのデフォルト設定を使用して作成の最後のステップまでスキップできます。構築プロセスを開始する前に、設定を確認する機会があります。進行状況トラッカーでステップを選択すると、後からいつでもそのステップに戻ることができます。

## ステップ 2:解約を定義する

[**解約の定義**] パネルで、提供されたフィルターを使用して、ビジネスにおけるユーザーの解約をどのように定義するかを指定します。つまり、ユーザーがどの時間枠内でどのような行動をとった場合に、解約したとみなすかということです。

解約に先行する可能性のある行動を説明する必要はありません。ユーザーを解約ユーザーとするものだけを定義してください。これは、ユーザーが一度行う (`do`) か、行わなくなる (`do not`) かという観点で考えてください。これが解約を構成します。例えば、7日間アプリを開いていないユーザーを解約とみなすことができます。アンインストール、または購読解除、アカウントの無効化などのカスタムイベントによってユーザーが解約したとみなすこともできます。

#### 解約時間枠

解約時間枠は、ユーザーのアクティビティが解約の基準を満たす期間です。利用可能なデータに応じて、最大60日まで設定できます。この時間枠は、予測をトレーニングするための過去のデータを取得するために使用されます。予測が構築されると、正確な結果を得るのに十分なデータがあったかどうかが表示されます。

予測が構築され、ユーザーがスコアを受け取ると、_解約リスクスコア_ は、解約時間枠で設定した時間枠内にユーザーが解約する可能性を示します。

以下は、過去7日間のセッションの失効に基づくシンプルな定義の例です。

![ユーザーが7日間セッションを開始しない場合に解約とみなす解約の定義]({% image_buster /assets/img/churn/churn1.png %})

この場合、`do not` と `start a session` を選択します。`AND` や `OR` を使用して他のフィルターを組み合わせ、必要な定義を作成できます。検討すべき解約の定義に興味がありますか？以下の[解約定義のサンプル](#sample-definitions)セクションでインスピレーションを得ることができます。

{% alert note %}
`do` の場合、アクティブユーザーは解約する前にこの行で指定したアクションを実行していなかったと仮定します。そのアクションを実行すると、解約したことになります。<br><br>`do not` の場合、アクティブユーザーとは、それ以前の日数でそのアクションを実行し、その後停止したユーザーとみなします。<br><br>**例:** 解約が「過去60日間に購入していない」と定義されている場合、アクティブユーザーとは過去60日間に購入したユーザーです。その結果、過去60日間に購入しなかったユーザーはアクティブユーザーとはみなされません。つまり、この解約定義から作成された解約オーディエンスには、過去60日間に購入したユーザーのみが含まれます。これにより、結果として得られる解約予測オーディエンスが元の母集団よりも大幅に小さく見える場合があります。ワークスペース内のほとんどのユーザーがすでに解約の定義を満たしており、解約予測においてアクティブではない可能性があるためです。
{% endalert %}

定義の下に、定義に従って解約した過去のユーザーと解約しなかった過去のユーザーの推定数が表示されます。また、必要な最小値も表示されます。Braze は、予測が学習するのに十分なデータを持つために、過去のデータにこの最小数のユーザーが利用可能である必要があります。

## ステップ 3:予測対象ユーザーをフィルタリングする

予測対象ユーザーは、解約リスクを予測したいユーザーのグループです。予測対象ユーザーは、機械学習モデルが過去から学習するために参照するユーザーグループを定義します。デフォルトでは、[**すべてのユーザー**] に設定されています。これは、この予測がすべてのアクティブユーザーに対して解約リスクスコアを作成することを意味します（解約モデルにおいてアクティブとみなされるユーザーについては、前述の注記を参照してください）。

ユースケースに応じて、フィルターを使用してモデルで評価するユーザーを指定したい場合があります。そのためには、[**独自の予測対象ユーザーを定義**] を選択し、オーディエンスフィルターを選択します。例えば、ユーザー群にドライバーとライダーがいるライドシェアアプリで、ライダー向けの解約モデルを構築する場合、予測対象ユーザーをライダーのみにフィルタリングする必要があります。多くのユースケースでは、特定の予測対象ユーザーを選択する必要がないことに留意してください。例えば、ユースケースが解約する可能性が最も高いEU地域のユーザーをターゲットにすることである場合、すべてのユーザーに対してモデルを実行し、キャンペーンの Segment にEU地域のフィルターを含めるだけで済みます。

Braze は、予測対象ユーザーの推定サイズを表示します。希望するオーディエンスを指定してもモデルの実行に必要な最小値を満たさない場合は、より広いフィルターを指定するか、[**すべてのユーザー**] オプションを使用してみてください。「すべてのユーザー」グループのサイズは静的ではなく、解約の定義を考慮するため、モデルごとに異なります。例えば、解約の定義が30日間購入**しない**ことである場合、Braze は過去30日間に購入**した**ユーザーに対してモデルを実行し（今後30日間に購入**しない**可能性を予測します）、それらのユーザーが「すべてのユーザー」指標に反映されます。

{% alert note %}
予測対象ユーザーは1億ユーザーを超えることはできません。
{% endalert %}

予測時間枠が14日以下の場合、「最後に使用したアプリ」や「最後に購入した日」のような「最後の...」で始まるフィルターの時間枠は、解約の定義で指定された**解約時間枠を超えることはできません**。例えば、解約の定義の時間枠が14日の場合、「最後の...」フィルターの時間枠は14日を超えることはできません。

解約時間枠は、モデルが最後に実行された日から日数を遡って評価されます。したがって、解約時間枠が15日でモデルが最後に12月1日に実行された場合、モデルは11月16日から11月30日を分析して、オーディエンスの適格性とトレーニングのためのユーザーアクティビティを把握します。

#### フルフィルターモード

新しい予測をすぐに構築するために、Braze セグメンテーションフィルターのサブセットのみがサポートされています。フルフィルターモードでは、すべての Braze フィルターを使用できますが、予測の構築に1つの解約時間枠分の期間が必要になります。例えば、解約時間枠が15日に設定されている場合、フルフィルターモードでのみサポートされるフィルターを使用すると、ユーザーデータの収集と予測の構築に15日かかります。また、フルフィルターモードでは、オーディエンスサイズに関する一部の推定値は利用できません。

予測対象ユーザー定義のサンプルリストについては、以下の[解約定義のサンプル](#sample-definitions)セクションのサンプル定義をご確認ください。

![]({% image_buster /assets/img/churn/churn5.png %})

前のページと同様に、下部のパネルには、解約の定義と予測対象ユーザーの定義から得られる過去のユーザーの推定数が表示されます。これらの推定値は、予測を作成するために表示されている最小要件を満たす必要があります。

## ステップ 4:解約予測の更新頻度を選択する

機械学習モデルはユーザーのイベント発生確率スコアを生成し、それらのスコアはここで選択したスケジュールに基づいて更新されます。イベント発生確率スコアに基づいてユーザーをターゲットにすることができます。

有用と思われる**最大更新頻度**を選択してください。例えば、ユーザーの解約を防ぐために毎週プロモーションを送信する場合は、更新頻度を希望の曜日と時刻で [**毎週**] に設定します。

![予測更新スケジュールが毎日午後5時に設定されている。]({% image_buster /assets/img/churn/churn2.png %})

{% alert note %}
プレビューおよびデモの予測では、ユーザーの解約リスクは更新されません。また、予測の毎日の更新には、解約予測の毎週または毎月の更新に加えて追加の購入が必要です。この機能を購入するには、アカウントマネージャーにお問い合わせください。
{% endalert %}

## ステップ 5:予測を構築する

入力した詳細が正しいことを確認し、[**予測を構築**] を選択します。[**下書きとして保存**] を選択して変更を下書き形式で保存し、後でこのページに戻ってモデルを構築することもできます。[**予測を構築**] を選択すると、モデルを生成するプロセスが開始されます。データ量に応じて、30分から数時間かかる場合があります。この予測では、モデル構築プロセスの間、トレーニングが進行中であることを説明するページが表示されます。Braze のモデルは、カスタムイベント、購入イベント、キャンペーンインタラクションイベント、およびセッションデータを考慮します。

完了すると、ページは自動的に分析ビューに切り替わり、予測と結果の準備ができたことを通知するメールも届きます。エラーが発生した場合、ページは編集モードに戻り、何が問題だったかの説明が表示されます。

予測は、利用可能な最新のデータで最新の状態を保つために、**2週間ごとに自動的に**再構築（「再トレーニング」）されます。これは、予測の出力であるユーザーの _解約リスクスコア_ が生成されるタイミングとは別のプロセスであることに注意してください。後者は、ステップ 4 で選択した更新頻度によって決まります。

## 解約および予測対象ユーザー定義のサンプル {#sample-definitions}

**解約定義のサンプル**<br>
- 「7日以内にカスタムイベント『サブスクリプションキャンセル』を実行」<br>
- 「30日以内にカスタムイベント『トライアル期限切れ』を実行」<br>
- 「1日以内にアンインストールを実行」<br>
- 「14日以内に購入しない」<br>

上記で概説した解約定義に対応する予測対象ユーザー定義の例：<br>
- **サブスクリプション開始が2週間以上前、またはサブスクリプション開始が2週間未満**<br>この場合、2つの予測を作成し、新規サブスクライバーと長期サブスクライバーに異なるメッセージを送信することをお勧めします。これを「最初の購入が30日以上前」と定義することもできます。<br>
- **アンインストールしたユーザー**<br>最近購入したことがある顧客や、最近アプリを使用した顧客に焦点を当てることができます。<br>
- **解約の定義として購入しないリスクがあるユーザー**<br>最近ブラウジング、検索、またはアプリへのエンゲージメントが多い顧客に焦点を当てたい場合があります。適切な割引介入により、このよりエンゲージメントの高いグループの解約を防ぐことができるかもしれません。

## アーカイブされた予測

アーカイブされた予測は、ユーザースコアの更新を停止します。アーカイブを解除された予測は、事前に設定されたスケジュールでユーザースコアの更新を続行します。アーカイブされた予測は削除されることはなく、リストに残ります。
