# ダブルオプトインによるメール登録

> ダブルオプトインによるメールサインアップテンプレートを使用して、検証済みのメールサインアップでリーチを拡大します。新規ユーザーをターゲットに、メールの受信、サブスクリプションの確認、プロモーションコードの受信をシームレスに行う。

この記事では、ユーザーライフサイクルの検討段階のためにデザインされた**、ダブルオプトインテンプレートによるメールサインアップの**ユースケースを説明する。完了すると、セッションを開始したとき、またはオンボーディングが完了していないときに、ユーザーにメールとアプリ内メッセージを送信するキャンバスが作成されます。

## 前提条件

このテンプレートをうまく使うには、以下のものが必要だ：

- [複数ページのアプリ内メッセージで]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/drag_and_drop/create#multi-page)、1ページでユーザーのメールを取得し、もう1ページでサクセスメッセージを伝える。 
- ユーザーがメールアドレスを確認するための確認メール。
- ダブルオプトインしたユーザー限定のプロモーションコードを記載したウェルカムメール。

## テンプレートをニーズに合わせる

仮に君がステッピントンという健康アプリで働いているとしよう。このアプリはカロリートラッキング、デジタルエクササイズクラス、フラッシュモブマラソンといった機能で知られている。キャンバスを作成する前に、[アプリ内メッセージおよびブラウザ内メッセージを設定]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/drag_and_drop/create#multi-page)する[。]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/drag_and_drop/create#multi-page)このメッセージには、ユーザーがアプリで初めて利用した際の体験やインプレッションを把握するための一連の質問が含まれている。

テンプレートにアクセスするには、新しいキャンバスを作成するときに、**キャンバステンプレートを使用**> **ブレーズテンプレート**を選択します。次に、「**ダブルオプトインによるメール登録**」の横にある「**テンプレートを適用**」を選択する。あとは、テンプレートを見ながら、自分たちのニーズに合わせていくだけだ。

### ステップ1:詳細を設定する

キャンバスの詳細を目標に合わせて調整する。

1. テンプレート名の横にある**編集**を選択します。

![キャンバスの現在のタイトルと説明。]({% image_buster /assets/img/canvas_templates/email_signup1.png %}){: style="max-width:50%;"}

{:start="2"}
2\.キャンバス名を更新し、アプリを初めて使うユーザーをターゲットにしたキャンバスであることを指定します。
3\.このキャンバスには、ユーザーが二重オプトインするためのパーソナライズされたメッセージングが含まれていることを説明するよう、説明文を更新する。
4. **メール**というタグを追加し、キャンバスのホームページでフィルターできるようにします。

![キャンバスの新しい名前、説明、およびタグ。]({% image_buster /assets/img/canvas_templates/email_signup2.png %}){: style="max-width:90%;"}

### ステップ 2:変換イベントの割り当て

次に、コンバージョンイベントを設定する。コンバージョンイベントは、キャンバスの成功を測るために使える指標の一種だ。[**コンバージョンイベントのタイプ**] には、「**カスタムイベントを実行**」を選択します。次に、**カスタムイベント**名として **email_opt_in**を選択する。

![メール配信のオプトインに関するコンバージョンイベントタイプ向けの「コンバージョンイベントの割り当て」セクション。]({% image_buster /assets/img/canvas_templates/email_signup3.png %}){: style="max-width:90%;"}

テンプレートのコンバージョン期限を3日間に設定する。なぜなら、最も最近のユーザーを対象にしたいからだ。

### ステップ 3:エントリスケジュールを調整する

エントリースケジュールは**アクションベース**のままにしておく。そうすれば、ユーザーがアプリでセッションを開始した時に、あなたのCanvasに入力するようになる。こうすることで、タイムリーなエンゲージメントを通じて関係を築き始めることができる。

また、**アクションベースのオプション**は現状のままにしておくことを検討すべきだ。ユーザーがセッションを開始する時だけキャンバスに入力できるようにするためである。

![アクションベースの登録スケジュール。あらゆるセッションを開始したユーザーをCanvasに登録する。]({% image_buster /assets/img/canvas_templates/email_signup4.png %}){: style="max-width:90%;"}

**エントリ**ウィンドウでは、**開始時刻（必須）**を希望の日時に更新する。

![2025年1月16日12時30分開始のエントリー枠。ユーザーはこのメッセージを自分のローカルタイムゾーンで入力する。]({% image_buster /assets/img/canvas_templates/email_signup5.png %}){: style="max-width:90%;"}

### ステップ 4: ターゲットオーディエンスを選択する

テンプレートのデフォルトの[セグメンテーションフィルター]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters)を維持することで、ユーザープロファイルに`Email Available is false`メールアドレスを持たないステッピングトンユーザーを対象とする。

![「メールが利用可能ではない」というフィルターを適用したエントリオーディエンス。]({% image_buster /assets/img/canvas_templates/email_signup6.png %}){: style="max-width:90%;"}

### ステップ 5: 送信設定を選択する

デフォルトの配信設定を維持する。つまり、メッセージや通知の受信を購読またはオプトインしたユーザーのみに送信し、その他の設定（フリークエンシーキャップ、静穏時間帯、シードグループ）は適用しない。

![デフォルトの送信オプションは、サブスクライバーまたはオプトインしたユーザーのみに送信する設定だ。]({% image_buster /assets/img/canvas_templates/email_signup7.png %}){: style="max-width:90%;"}

### ステップ 6: キャンバスをカスタマイズする

次に、ユーザーに送信したいチャネルとコンテンツをカスタマイズして、キャンバスを構築する。メール登録の確認に集中しているから、テンプレートのキャンバスステップやチャネルを追加したり削除したりする必要はない。

1. 最初のメッセージステップ「**メール登録**」を選択する。ここでテンプレートを更新し、当社の複数ページにわたるアプリ内（およびブラウザ内）メッセージを使用するようにする。

- 1ページ目はメールをキャプチャしている。
- ページ2には確認メッセージが表示される。

![アプリ内メッセージの2ページ。ユーザーのメールを取得し、成功メッセージを表示する。]({% image_buster /assets/img/canvas_templates/email_signup8.png %}){: style="max-width:90%;"}

{:start="2"}
2\.ここから先は、**サブスクライバーによってサブスクライブされた**アクションパスステップをそのままにしておく。このステップでは、ユーザーを1日のうちに2つのグループに分ける：

- メールで Steppington を購読登録したユーザー
- Steppington にメールで購読登録していないユーザー

{:start="3"}
3\.次に、「メッセージの**確認**」ステップのために、メール本文をブランドの確認メールに置き換える。これはサブスクライバーにメールを送信し、メールアドレスの確認とメッセージングの許可を求めるものだ。
4. **購読の確認**アクションパスステップはそのままにしておいてください。このステップにより、ユーザーはメールを確認済みと未確認のグループにさらに分けられる。確認期限は1週間だ。
5. 最後に、「**歓迎＋割引**メッセージ」のステップを更新し、限定プロモーションコードを含む確認メールを送信する。

{% alert note %}
**メール確認**メッセージステップは、ユーザーの2回目のセッションで起動される。これは、最初のセッション開始イベントがキャンバスを起動するが、ユーザーが最初の**メール登録**メッセージステップに到達した後に発生する2回目のセッション開始が必要となるためだ。これによりユーザーは2回目のアプリ内メッセージを起動する資格を得る。
{% endalert %}

### ステップ 7:キャンバスをテストして開始する

キャンバスが期待通りに動作することを確認するためにテストとレビューを行った後、**[Launch Canvas**]を選択して起動する。

{% alert tip %}
キャンバスの起動前後に考慮すべき点については、[起動前と起動後のチェックリスト]({{site.baseurl}}/user_guide/engagement_tools/canvas/ideas_and_strategies/pre_post_launch_checklist/#things-to-consider-before-launch)をご確認ください。
{% endalert %}
