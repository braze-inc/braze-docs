# ダブルオプトインによるメール登録

> ダブルオプトインによるメールサインアップテンプレートを使用して、検証済みのメールサインアップでリーチを拡大します。新規ユーザーをターゲットに、メールの受信、サブスクリプションの確認、プロモーションコードの受信をシームレスに行う。

この記事では、ユーザーライフサイクルの検討段階のためにデザインされた**、ダブルオプトインテンプレートによるメールサインアップの**ユースケースを説明する。完了すると、セッションを開始したとき、またはオンボーディングが完了していないときに、ユーザーにメールとアプリ内メッセージを送信するキャンバスが作成されます。

## 前提条件

このテンプレートを正常に使用するには、以下が必要です。

- [複数ページのアプリ内メッセージで]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/drag_and_drop/create#multi-page)、1ページでユーザーのメールを取得し、もう1ページでサクセスメッセージを伝える。 
- ユーザーがメールアドレスを確認するための確認メール。
- ダブルオプトインしたユーザー限定のプロモーションコードを記載したウェルカムメール。

## テンプレートをニーズに合わせる

たとえば、カロリー"トラッキング、デジタルエクササイズ教室、フラッシュ・モブ・マラソンなどの機能で知られるヘルス・アプリ、ステッピントンで働いているとしよう。キャンバスを作成するには、[マルチページインアプリとブラウザー内メッセージ]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/drag_and_drop/create#multi-page)を設定します。このメッセージには、アプリでのユーザーの最初の乗り心地とインプレッションを判断するための一連の魅力的な質問が含まれています。

テンプレートにアクセスするには、新しいキャンバスを作成する際に、[**キャンバステンプレートを使用する**] > [**Braze テンプレート**] を選択します。次に、「**ダブルオプトインによるメール登録**」の横にある「**テンプレートを適用**」を選択する。あとは、テンプレートを見ながら、自分たちのニーズに合わせていくだけだ。

### ステップ1:詳細を設定する

目的を反映するには、キャンバスの詳細をAdjustします。

1. テンプレート名の横にある**編集**を選択します。

![キャンバスの現在のタイトルと説明。]({% image_buster /assets/img/canvas_templates/email_signup1.png %}){: style="max-width:50%;"}

{:start="2"}
2\.キャンバス名を更新し、アプリを初めて使うユーザーをターゲットにしたキャンバスであることを指定します。
3\.このキャンバスに、ダブルオプトインするユーザーs のパーソナライズされた メッセージングが含まれていることを説明するように記述をアップデートします。
4\.**メール**というタグを追加し、キャンバスのホームページでフィルターできるようにします。

![キャンバスの新しい名前、説明、およびタグ。]({% image_buster /assets/img/canvas_templates/email_signup2.png %}){: style="max-width:90%;"}

### ステップ 2:コンバージョンイベントを割り当てる

次に、コンバージョンイベントs を割り当てます。変換イベントは、キャンバスの成功を測定するために使用できるメトリクスの一種です。[**コンバージョンイベントのタイプ**] には、「**カスタムイベントを実行**」を選択します。次に、**カスタムイベント名**に対して**email_opt_in**を選択します。

!["コンバージョンイベントのS&quotを割り当てます。メールのオプションのコンバージョンイベント型のセクションです。]({% image_buster /assets/img/canvas_templates/email_signup3.png %}){: style="max-width:90%;"}

最新のユーザーを対象にするため、テンプレートのコンバージョン期限は3 日間を維持してください。

### ステップ 3:エントリスケジュールを調整する

エントリ スケジュールを**Action-Based** のようにして、ユーザーがアプリでセッションを起動したときにキャンバスに入るようにします。これにより、タイムリーなエンゲージメントでリレーションシップを構築することができます。

また、**Action Based Options**をそのままにして、ユーザーがセッションを起動したときにのみキャンバスに入るようにすることを検討してください。

![アクションベースのエントリ スケジュールで、キャンバスにセッションを起動するユーザーs を入力します。]({% image_buster /assets/img/canvas_templates/email_signup4.png %}){: style="max-width:90%;"}

**入力画面**で、**開始時刻(必須)**を希望の日時に更新します。

![2025年1月16日12時30分開始のエントリー枠。ユーザは、ローカルタイムゾーンにこのメッセージを入力します。]({% image_buster /assets/img/canvas_templates/email_signup5.png %}){: style="max-width:90%;"}

### ステップ 4: ターゲットオーディエンスを選択する

テンプレートのデフォルト[セグメンテーション フィルター]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters)`Email Available is false` を維持することで、ユーザープロファイル内にメールの住所を持たないSteppington ユーザーとして対象オーディエンスを定義します。

!["Email Available がfalse" フィルターのエントリーオーディエンスです。]({% image_buster /assets/img/canvas_templates/email_signup6.png %}){: style="max-width:90%;"}

### ステップ 5: 送信設定を選択する

デフォルト サブスクリプション 設定s を保持すると、受信メッセージまたは通知s を購読または選択したユーザーにのみ送信され、他の設定s(フリークエンシーキャップ、静かな時間、シードグループ)はスキップされます。

![既定の送信オプションは、購読またはオプトインしているユーザーにのみ送信します。]({% image_buster /assets/img/canvas_templates/email_signup7.png %}){: style="max-width:90%;"}

### ステップ 6: キャンバスをカスタマイズする

次に、ユーザーsに送信するチャネルsと内容をカスタマイズして、キャンバスを構築します。メールのサインアップの検証に重点を置いているため、テンプレートのキャンバスステップとチャネルを追加または削除する必要はありません。

1. 最初のメッセージステップ「**メール登録**」を選択する。ここでは、マルチページのアプリ(およびブラウザー内)メッセージを使用するようにテンプレートを更新します。

- メールsをキャプチャします。
- 2ページ目に確認メッセージが表示されます。

![ユーザー メールs をキャプチャし、サクセスメッセージを表示するアプリ内メッセージの2 ページ。]({% image_buster /assets/img/canvas_templates/email_signup8.png %}){: style="max-width:90%;"}

{:start="2"}
2\.ここから、**Subscribed**アクションパスステップをそのままにしておきます。このステップでは、ユーザーを1日のうちに2つのグループに分ける：

- メールで Steppington を購読登録したユーザー
- Steppington にメールで購読登録していないユーザー

{:start="3"}
3\.次に、「メッセージの**確認**」ステップのために、メール本文をブランドの確認メールに置き換える。これにより、メールが購読済みのユーザーに送信され、メールの住所を確認してメッセージングを選択するように促されます。
4\.**購読の確認**アクションパスステップはそのままにしておいてください。このステップでは、さらにユーザーをメールを確認した人と確認していない人に1週間のウィンドウで分割します。
5\.最後に、「**歓迎＋割引**メッセージ」のステップを更新し、限定プロモーションコードを含む確認メールを送信する。

{% alert note %}
**Verify Email** Message ステップは、ユーザーの2 番目のセッションでトリガー通知されます。これは、最初のセッションスタートイベントがキャンバスをトリガーするためですが、ユーザーが最初の**メールサインアップ**メッセージステップに到達した後、2番目のアプリ内メッセージをトリガーするためにユーザーが適格であることが必要であるためです。
{% endalert %}

### ステップ 7:キャンバスをテストして起動する

キャンバスをテストして確認し、期待通りに動作することを確認したら、**Launch Canvas** を選択して起動します。

{% alert tip %}
キャンバスの起動前後に考慮すべき点については、[起動前と起動後のチェックリスト]({{site.baseurl}}/user_guide/engagement_tools/canvas/ideas_and_strategies/pre_post_launch_checklist/#things-to-consider-before-launch)をご確認ください。
{% endalert %}
