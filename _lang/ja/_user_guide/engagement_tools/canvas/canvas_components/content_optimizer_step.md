# コンテンツ最適化エージェントのステップ

> コンテンツオプティマイザーエージェントステップでは、単一のステップ内で複数のバージョンのコンテンツコンポーネントを設定しテストできる。コンテンツのバリエーションを試すのに役立つ。そして時間の経過とともに、最も高いパフォーマンスを示す組み合わせに向けて自動的に最適化する。導入については、[コンテンツオプティマイザーを]({{site.baseurl}}/user_guide/brazeai/content_optimizer/)参照せよ。

{% alert important %}
コンテンツオプティマイザーは現在ベータ版である。利用開始に関するサポートが必要な場合は、顧客サクセスマネージャーに連絡する。
{% endalert %}

## コンテンツ最適化ステップの作成

最良の結果を得るには、ユーザーが段階的にステップを入力するキャンバスでコンテンツオプティマイザーエージェントを使用すること。もし全てのユーザーが一斉にステップに入ると、エージェントは初期の結果から学習する時間がなくなる。 

### ステップ 1: ステップを追加する

サイドバーから**コンテンツオプティマイザー**コンポーネントをドラッグ＆ドロップするか、ステップの下部にあるプラス<i class="fas fa-plus-circle"></i>ボタンを選択し、**コンテンツオプティマイザー**を選ぶ。

### ステップ 2:基本メッセージを作成する

基本メッセージは、そのステップの出発点だ。各コンテンツコンポーネントのバリアントは**、コンテンツオプティマイザー設定**タブで定義された組み合わせに基づいてダイナミックに挿入される。 

{% alert note %}
ベータ期間中は、メールが唯一のサポート対象チャネルである。
{% endalert %}

**メッセージングチャネル**タブから、**メール**を選択し、基本メールメッセージを作成する。サポートが必要な場合は、専用の[メール]({{site.baseurl}}/user_guide/message_building_by_channel/email)セクションを参照せよ。 

コンテンツオプティマイザーエージェントは、このバリアントで指定された送信設定（メールドメインや返信先アドレスなど）を使用して、すべてのメッセージを送信する。このメッセージは、新しいデザインから始めるか、既存のテンプレートを選ぶかのどちらかだ。このステップで、メッセージのどの部分を最適化したいのかを考える。これらを[ステップ4](#step-4)で定義する。

最適化対象のサポートコンポーネントには以下が含まれる：

- 件名
- 本文ヘッダー
- 本文コンテンツ
- プライマリー CTA

### ステップ 3:配送設定を指定する

**配信設定**タブでは、ステップがインテリジェントタイミングを使用するか、配信検証を使用するかを指定できる。詳細については、メッセージステップの[配信設定編集]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/message_step/#step-2-edit-delivery-settings)を参照せよ。

### ステップ 4: コンテンツコンポーネントとバリアントを追加する {#step-4}

コンテンツコンポーネントとは、テスト対象となるメッセージの個々の要素である。例えば、異なる件名、見出し、本文テキスト、または主要なアクション喚起などが該当する。これらのコンポーネントにより、メッセージの複数バージョンを生成し、時間の経過に伴うパフォーマンスに基づいて自動的に最適化できる。

各ステップには最大3つのコンテンツコンポーネントを追加でき、各コンポーネントには最大5つのバリアントを追加できる。これにより、合計125通りのユニークなコンテンツ組み合わせが可能となる。

![コンテンツオプティマイザーインターフェイスにおけるコンテンツコンポーネントの追加と設定の選択肢。インターフェイスには、件名、ヘッダー本文、本文コンテンツ、主要なCTAといった選択可能なコンポーネントが表示される。各コンポーネントには、異なるバリアントを入力するためのフィールドが用意されている。]({% image_buster /assets/img/content_optimizer/add_content_components.png %})

#### ステップ4.1：コンテンツコンポーネントを設定する

コンポーネントを設定するには：

1. **コンテンツオプティマイザー設定**タブに移動する。
2. 最適化したいコンポーネントを選択する。サポートされているオプション：
  - 件名
  - 本文ヘッダー
  - 本文コンテンツ
  - プライマリー CTA
3. 選択された各コンポーネントについて、そのコンテンツの代替バージョン（バリアント）のセットを定義する。トーン、構造、または内容において異なる、明確で区別できるバリアントを使用せよ。これにより、コンテンツオプティマイザーは上位のパフォーマンスを発揮するコンテンツをより効果的に識別できる。お前はできる：
  - 手動で独自のバリアントを作成せよ。
  - AIが生成した提案を使って、新しい選択肢を素早く探るのだ。

![メール最適化のためのコンテンツコンポーネントを追加・設定するオプションを表示するコンテンツオプティマイザー設定インターフェイス。各コンポーネントには、異なるバリアントを入力するためのフィールドがある。表示されるテキストには、コンポーネント名とバリアントテキストを入力するフィールドが含まれる。]({% image_buster /assets/img/content_optimizer/content_optimizer_settings.png %})

#### ステップ4.2：メッセージにLiquidを加えろ

各コンポーネントに対して少なくとも2つのバリアントを定義した後、それぞれの関連するLiquidタグをコピーし、ベースメッセージ内の対応する位置に貼り付ける。

- 例えば、件名を最適化する場合、メール作成画面の件名フィールドに \`<A href="https://example.com">例文</A>\` タグ{% raw %}`{% message_component "Subject" %}`{% endraw %}を貼り付ける。
- 長いテキストの中にコンポーネントタグを含めることもできる。そうすればコンポーネントの一部だけをテストできる。例えば：{% raw %}`Hey there, {% message_component "Subject" %}`{% endraw %}

![件名、本文ヘッダー、本文コンテンツ、主要なCTAなどのコンテンツコンポーネントを追加・設定するためのオプション。各コンポーネントには、異なるバリアントを入力するためのフィールドがある。]({% image_buster /assets/img/content_optimizer/optimization_liquid_in_use.png %})

選択したコンテンツコンポーネントにLiquidタグを追加しないと、**コンテンツオプティマイザー設定**タブに警告が表示され、**メッセージングチャネル**タブにエラーが表示される。キャンバスは、選択したすべてのコンポーネントがベースメッセージに正しく追加されるまで起動できない。

キャンバスが実行されるにつれ、エージェントはコンポーネント間でバリアントを組み合わせて、異なるコンテンツの組み合わせを生成する。時間の経過とともに、より高いパフォーマンスを発揮する組み合わせが優先的に提供されるため、手動での介入なしにパフォーマンスを向上させることができる。

### ステップ 5: 最適化イベントを選択する

最適化イベントは、コンテンツオプティマイザーエージェントがパフォーマンスを評価し、時間の経過とともにコンテンツの組み合わせにトラフィックを割り当てる方法を決定する。 

メールについては、以下のいずれかのイベント向けに最適化できる。エージェントは、メッセージ送信から7日以内に記録された開封とクリックを利用して、パフォーマンスの高いコンテンツの組み合わせへ配信をシフトする。

| イベント | 説明 | ユースケース |
| --- | --- | --- |
| 開封数 | 受信者がメールを開封する組み合わせを最適化する。 | 件名のテストや、可視性の向上を目指すこと |
| クリック数 | リンクとのエンゲージメントを促進する組み合わせを最適化する。ボットによるクリックや、Brazeが認識した配信停止クリックは含まれない。 | リンクからのトラフィック、エンゲージメント、またはコンバージョンの促進 |
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3 role="presentation" }

選択した最適化イベントはこのステップ内の全てのコンテンツコンポーネントに適用される。 

## 分析

パフォーマンスを確認するには、ステップレベルの分析パネルを開き、コンテンツのバリアントごとの指標と全体的な組み合わせのパフォーマンスを確認する。コンテンツ最適化ステップは[、メッセージステップと同じ分析]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/message_step/#analytics)を使用する。

![コンテンツオプティマイザーの分析データは、3つのボタンと送信の割り当て割合を示しており、これらは上昇傾向にある。]({% image_buster /assets/img/content_optimizer/content_optimizer_analytics.png %})

## トラブルシューティング

| 問題 | 説明 | 修正する |
| --- | --- | --- |
| 欠落したLiquidタグ | コンテンツコンポーネント（件名やCTAなど）を追加しても、ベースメッセージに対応するLiquidタグを挿入しないと、次のように表示される：<br>- **コンテンツ最適化設定**タブ上の警告 <br>- **メッセージングチャネル**タブのエラー | **コンテンツオプティマイザー設定**タブの各コンポーネントの下に表示されているLiquidスニペットをコピーし、メッセージの適切な部分に貼り付けろ。 |
| 孤立したLiquidタグ | コンテンツコンポーネントを削除しても、そのLiquidタグをベースメッセージに残したままにすると、メッセージが送信された際に期待通りにレンダリングされない可能性がある。 | 起動前にベースメッセージから未使用の`message_component`タグをすべて削除せよ。 |
