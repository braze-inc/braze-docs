# コンテンツオプティマイザエージェントのステップ

> コンテンツオプティマイザエージェントステップでは、1つのステップ内でコンテンツコンポーネントの複数バージョンを構成およびテストできます。コンテンツのバリエーションを試すのに役立ち、時間の経過とともに最適なパフォーマンスの組み合わせに向けて自動的に最適化されます。概要については、[コンテンツ最適化ルーチン]({{site.baseurl}}/user_guide/brazeai/content_optimizer/)を参照してください。

{% alert important %}
コンテンツ最適化ルーチンは現在ベータ版に入っています。始めるには、顧客のサクセスマネージャーにお問い合わせください。
{% endalert %}

## コンテンツオプティマイザステップの作成

最良の効果を得るには、キャンバスでコンテンツ最適化エージェントを使用します。ここでは、ユーザーs が徐々にステップに入ります。すべてのユーザーsが一度にステップに入ると、エージェントは初期の成果から学ぶ時間がなくなる。 

### ステップ 1: ステップを追加する

サイドバーから**Content Optimizer**コンポーネントをドラッグアンドドロップするか、ステップの下部にある<i class="fas fa-plus-circle"></i>プラスボタンを選択し、**Content Optimizer**を選択します。

### ステップ 2:基本メッセージを作成する

基本メッセージは、ステップの開始点です。コンテンツコンポーネントごとのバリエーションは、**Content Optimizer Settings**タブで定義された組み合わせに基づいてダイナミックな的に挿入されます。 

{% alert note %}
β期には、メールのみがサポートされるチャネルです。
{% endalert %}

**Messaging Channels**タブから**メール**を選択し、基本メールを作成します。ヘルプについては、専用の[Eメール]({{site.baseurl}}/user_guide/message_building_by_channel/email)セクションを参照してください。 

コンテンツオプティマイザエージェントは、このバリアントで指定された送信設定(メールドメインや返信先アドレスなど)を使用して、すべてのメッセージを送信します。新しいデザインから始めるか、このメッセージのテンプレートを選択します。このステップでは、最適化するメッセージのコンポーネントを検討します。これらは[ ステップ 4](#step-4) で定義します。

最適化するためにサポートされるコンポーネントは次のとおりです。

- 件名
- 本文ヘッダー
- 本文コンテンツ
- プライマリー CTA

### ステップ 3:配信設定の指定s

**Delivery Settings**タブで、ステップがインテリジェントタイミングまたはデリバリ検証を使用するかどうかを指定できます。詳しくは、メッセージステップの[配信設定の編集s]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/message_step/#step-2-edit-delivery-settings)を参照してください。

### ステップ 4: コンテンツコンポーネントとバリアントの追加 {#step-4}

コンテンツコンポーネントは、さまざまな件名、見出し、本文、アクションへの主要な呼び出しなど、テストするメッセージの個々の要素です。これらのコンポーネントを使用すると、複数バージョンのメッセージを生成し、経時的なパフォーマンスに基づいて自動的に最適化できます。

ステップごとに最大 3 つのコンテンツコンポーネントと、コンポーネントごとに最大 5 つの バリアント s を追加でき、合計で 125 の一意のコンテンツの組合せになります。

![コンテンツ最適化インターフェイスでコンテンツコンポーネントを追加および構成するためのオプション。インターフェイスには、Subject、Body Header、Body Content、Primary CTA などの選択可能なコンポーネントが表示され、それぞれにフィールドs を使用してさまざまなバリアントs を入力します。]({% image_buster /assets/img/content_optimizer/add_content_components.png %})

#### ステップ4.1：コンテンツコンポーネントの構成

コンポーネントを設定するには

1. **Content Optimizer Settings**タブに移動します。
2. 最適化するコンポーネントを選択します。サポートされるオプション:
  - 件名
  - 本文ヘッダー
  - 本文コンテンツ
  - プライマリー CTA
3. 選択したコンポーネントごとに、そのコンテンツの一連の代替バージョン(バリアントs)を定義します。音色、構成、内容が異なる明確なバリアントを使用します。これにより、コンテンツ最適化ルーチンは、上位パフォーマンスをより効果的に識別できます。次の操作を実行できます。
  - 独自のバリアントを手動で作成します。
  - AI が生成した提案を使用して、新しいオプションをすばやく探索します。

![コンテンツ最適化ルーチンの設定インターフェイスには、メール最適化のコンテンツコンポーネントを追加および構成するオプションが表示されます。各コンポーネントには、異なるバリアントs を入力するための入力フィールドs があります。表示されるテキストには、バリアントテキストを入力するためのコンポーネントの名前とフィールドs が含まれます。]({% image_buster /assets/img/content_optimizer/content_optimizer_settings.png %})

#### ステップ4.2：メッセージに液体を追加

コンポーネントごとに少なくとも2 つのバリアントs を定義した後、各コンポーネントに関連付けられたリキッドタグをコピーし、ベースメッセージの対応する場所に貼り付けます。

- たとえば、サブジェクト行を最適化する場合は、{% raw %}`{% message_component "Subject" %}`{% endraw %} タグをメールコンポーザーのサブジェクトフィールドに貼り付けます。
- コンポーネントの一部だけをテストするために、コンポーネントタグs を長いテキスト内に含めることもできます。例: {% raw %}`Hey there, {% message_component "Subject" %}`{% endraw %}。

![Subject、Body Header、Body Content、Primary CTA などのコンテンツコンポーネントを追加および設定するためのオプション。各コンポーネントには、異なるバリアントs を入力するためのフィールドs があります。]({% image_buster /assets/img/content_optimizer/optimization_liquid_in_use.png %})

選択したコンテンツコンポーネントにリキッドタグを追加しないと、**Content Optimizer Settings** タブに警告が表示され、**メッセージングチャネル** タブにエラーが表示されます。選択したすべてのコンポーネントがベースメッセージに適切に追加されるまで、キャンバスは起動できません。

キャンバスが実行されると、エージェントはコンポーネント間でバリアントをミックスして照合し、さまざまなコンテンツの組合せを生成します。時がたつにつれて、パフォーマンスの高い組み合わせがデリバリーに優先され、手作業による介入なしでパフォーマンスを改善するのに役立ちます。

### ステップ 5: 最適化イベントの選択

最適化イベントは、コンテンツ最適化エージェントがパフォーマンスを評価し、トラフィックをコンテンツの組み合わせに割り当てる方法を決定します。 

メールの場合、次のいずれかの状況に最適化できます。送信後7日間以内に登録された開封sとクリックを使用して、よりパフォーマンスの高いコンテンツの組合せに配信を移します。

| イベント | 説明 | ユースケース |
| --- | --- | --- |
| 開封数 | メールを開封する受信者s を取得する組合せに最適化します。 | 件名をテストする、または視認性を高めることを目的とする |
| クリック数 | リンクでエンゲージメントを駆動する組み合わせに最適化します。ボットクリックやBraze認識された配信停止クリックは含まれません。 | リンクからのトラフィック、エンゲージメント、またはコンバージョンのドライブ |
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3 role="presentation" }

選択した最適化イベントアプリは、このステップのすべてのコンテンツコンポーネントにあります。 

## 分析

パフォーマンスを確認するには、ステップレベルの分析パネルを開封して、内容バリアント別および総合的なコンビネーションパフォーマンス別のメトリクスを表示します。コンテンツオプティマイザステップは、メッセージステップと同じ分析を使用します。詳しくは、メッセージステップの[Analytics]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/message_step/#analytics)を参照してください。  

## トラブルシューティング

| 問題 | 説明 | 修正する |
| --- | --- | --- |
| 液タグ抜け | コンテンツコンポーネント(件名やCTAなど)を追加しても、対応するリキッドタグを基本メッセージに挿入しない場合は、次のように表示されます。<br>- **Content Optimizer Settings**タブの警告 <br>- **メッセージングチャネル**タブのエラー | **Content Optimizer Settings**タブの各コンポーネントの下に表示されているリキッドスニペットをコピーし、メッセージのアプリの適切な部分に貼り付けます。 |
| 孤立した液体タグ | コンテンツコンポーネントを削除しても、基本メッセージにリキッドタグを残すと、送信時にメッセージが期待どおりに表示されない場合があります。 | 未使用の`message_component` タグs を基本メッセージから削除してから起動します。 |
