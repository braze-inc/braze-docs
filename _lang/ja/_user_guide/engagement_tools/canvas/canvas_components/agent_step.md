# エージェントステップ  

> エージェントステップでは、AIを活用した意思決定とコンテンツ生成をキャンバスワークフローに直接組み込むことができます。より一般的な情報については、[Braze エージェント]({{site.baseurl}}/user_guide/brazeai/agents/)を参照してください。 

![キャンバスユーザージャーニーにおけるエージェントステップ。]({% image_buster /assets/img/ai_agent/agent_step.png %}){: style="float:right;max-width:30%;margin-left:15px;"}

## 仕組み

ユーザーがキャンバス内のエージェントステップに到達すると、Braze は設定した入力データ（フルコンテキストまたは選択したフィールド）を指定したエージェントに送信します。エージェントはモデルと指示を用いて入力を処理し、出力を返します。その出力は、ステップで定義した出力変数に格納されます。

この変数は主に3つの方法で利用できます：

- **意思決定：**エージェントの応答に基づいて、ユーザーを異なるキャンバスのパスに誘導します。例えば、リードスコアリングエージェントは1から10までの数値を返すことがあります。このスコアを使って、ユーザーへのメッセージングを続けるか、そのユーザーをジャーニーから外すかを決めることができます。
- **パーソナライゼーション：**エージェントの応答をメッセージに直接挿入します。例えば、エージェントは顧客のフィードバックを分析し、顧客のコメントを参照した上で解決策を提案する共感的なフォローアップメールを生成できます。
- **ユーザーデータの処理：**ユーザーデータを分析し標準化した後、ユーザープロファイルに保存するか、Webhook を使って送信します。例えば、エージェントは感情スコアや製品アフィニティの割り当てを返すことができます。そのデータは将来の使用のためにユーザープロファイルに保存できます。

## 前提条件

エージェントステップは、関連するコンテキストを取り込むために[キャンバスコンテキスト変数]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/context_variables)を使用し、キャンバス内で活用できる変数を出力します。

## エージェントステップの作成

### ステップ 1: ステップを追加する

サイドバーから**エージェント**コンポーネントをドラッグ＆ドロップするか、ステップの下部にある<i class="fas fa-plus-circle"></i>プラスボタンを選択し、**エージェント**を選びます。  

### ステップ 2: エージェントを選択する  

このステップでデータを処理するエージェントを選択します。既存のエージェントを選んでください。設定手順については、[カスタムエージェントの作成]({{site.baseurl}}/user_guide/brazeai/agents/creating_agents/)を参照してください。

### ステップ 3: エージェントの出力を設定する {#define-the-output-variable}

エージェントの出力は「出力変数」と呼ばれ、容易にアクセスできるよう[コンテキスト変数]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/context/#context-variable-types)に格納されます。出力変数を定義するには、変数に名前を付けます。

出力変数のデータタイプは[エージェントコンソール]({{site.baseurl}}/user_guide/brazeai/agents)から設定されることに注意してください。エージェントの出力は、文字列、数値、ブール値、またはオブジェクトとして保存できます。これにより、キャンバス内のテキストのパーソナライゼーションと条件付きロジックの両方に対応できる柔軟性が生まれます。各タイプの一般的な用途を以下に示します：

| データタイプ | 一般的な用途 |
| --- | --- |
| 文字列 | メッセージのパーソナライゼーション（件名、本文、返信） |
| 数値 | [オーディエンスパス]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/audience_paths)でのスコアリング、閾値、ルーティング |
| ブール値 | [条件分岐]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/decision_split)におけるYes/No分岐 |
| オブジェクト | 上記のデータタイプの1つまたは複数を、単一のLLM呼び出しで予測可能なデータ構造として活用する |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

出力変数は、コンテキスト変数と同じテンプレート構文を使って、キャンバス全体で使用できます。**コンテキスト変数**の Segment フィルターを使うか、Liquid を使ってエージェントの応答を直接テンプレート化します：{% raw %}`{{context.${response_variable_name}}}` {% endraw %}

オブジェクト出力変数から特定のプロパティを使用するには、Liquid でドット表記を使ってそのプロパティにアクセスします：{% raw %}`{{context.${response_variable_name}.field_name}}`{% endraw %}

![変数「agent_output」のオブジェクトデータタイプ出力を持つBody HTML ライター用エージェントステップ。]({% image_buster /assets/img/ai_agent/test_agent_step.png %}){: style="max-width:80%;"}

### ステップ 4: 追加のコンテキストを追加する（オプション）

エージェントステップが実行される際に参照するための追加のコンテキスト値を含めるかどうかを決めることができます。通常キャンバスで使用する Liquid テンプレート値をここに入力できます。

{% alert note %}
エージェントは既に**指示**セクションで設定されたコンテキストを自動的に受け取っています。既にそこで設定済みの Liquid 変数は、ここで再入力する必要はありません。
{% endalert %}

![Liquid を使用してエージェントステップに追加のコンテキストを追加するオプション。]({% image_buster /assets/img/ai_agent/agent_step_context.png %}){: style="max-width:80%;"}

### ステップ 5: エージェントをテストする

エージェントステップを設定した後、このステップの出力をテストおよびプレビューできます。

![エージェントの出力をランダムなユーザーとしてプレビューする。]({% image_buster /assets/img/ai_agent/agent_step_preview.png %}){: style="max-width:80%;"}

## エラー処理  

- 接続されたモデルがレート制限エラーを返した場合、Braze は指数関数的バックオフを用いて最大5回まで再試行します。  
- エージェントがその他の理由で失敗した場合（タイムアウトエラーや無効な API キーなど）、出力変数は `null` に設定されます。
    - エージェントが1日の呼び出し制限に達した場合、出力変数は `null` に設定されます。 
- エラーに備えるために、[デフォルトの Liquid 値]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/setting_default_values)を使用してください。例えば、**パーソナライゼーションを追加**モーダルで、{% raw %}`{{context.${response_variable_name}.push_title | default: 'Hello friend!'}}`{% endraw %} や {% raw %}`{{context.${response_variable_name}.push_body | default: 'Open our app to get your prize!'}}`{% endraw %} のようなデフォルトの Liquid 値を入力できます。
- 同一の入力に対する応答はキャッシュされ、数分以内に繰り返される同一の呼び出しに対して再利用されることがあります。
    - キャッシュされた値を使用する応答も、総呼び出し回数と1日あたりの呼び出し回数にカウントされます。
- エージェントステップは、大量のユーザーバッチの処理に時間がかかることがあります。このステップで保留中のユーザーが見られる場合は、ログを確認して呼び出しが行われているか検証してください。

## 分析  

エージェントステップのパフォーマンスをトラッキングするには、以下の指標を参照してください：  

| 指標 | 説明 |
| --- | --- |
| _入力済み_ | ユーザーがエージェントステップに入った回数です。 |
| _次のステップに進んだ_ | エージェントステップを通過した後、フローの次のステップに進んだユーザーの数です。 |
| _キャンバスを終了した_ | エージェントステップを通過した後、キャンバスを終了したユーザーの数です。 |

## よくある質問

### エージェントステップはいつ使うべきですか？

一般的に、特定の文脈に応じたデータを LLM にフィードし、人間には不可能な規模でキャンバスコンテキスト変数をエージェント的にインテリジェントに割り当てさせたい場合に、エージェントステップの使用をおすすめします。

例えば、以前チョコレートとストロベリーを注文したユーザーに、新しいアイスクリームのフレーバーをおすすめするパーソナライズされたメッセージを送るとします。エージェントステップと AI アイテムおすすめの違いは以下のとおりです：

- **エージェントステップ：**LLM を用いて、エージェントに与えられた指示とコンテキストデータポイントに基づき、ユーザーが何を望んでいるかについて定性的な判断を行います。この例では、エージェントステップは、ユーザーが異なるフレーバーを試したい可能性に基づいて、新しいフレーバーを提案するかもしれません。
- **AI アイテムおすすめ：**機械学習モデルを用いて、過去のユーザーイベント（購入履歴など）に基づき、ユーザーが最も欲しがる可能性の高い製品を予測します。この例では、AI アイテムおすすめは、ユーザーの過去2回の注文（チョコレートとストロベリー）と、それらがワークスペース内の他のユーザーの動作とどう比較されるかに基づいて、フレーバー（バニラ）を提案します。

### エージェントステップは入力データをどのように使いますか？

エージェントステップは、エージェントが使用するように設定されたコンテキストデータと、[エージェントに提供](#step-4-add-any-additional-context-optional)される追加のコンテキストを分析します。
