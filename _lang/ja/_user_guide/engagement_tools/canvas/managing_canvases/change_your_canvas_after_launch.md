# 起動後にキャンバスを編集する

> この参考記事では、初回開始後にキャンバスで変更できることについて説明します。

次の方法で、開始後にキャンバスを編集できます。

* ユーザージャーニーに新しいキャンバスのステップを挿入する
* 新しいバリアントと接続を追加する
* バリアントの分布を調整する
* すべてのキャンバスステップを停止または再開する

{% alert note %}
コントロールバリアントの分布は、開始後のみに減らせます。
{% endalert %}

ユーザージャーニーでは、以下を削除できます。

- [キャンバスのステップ]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/about/)
- キャンバスのバリエーション 
- キャンバスのステップ間の接続

キャンバスのユーザーージャーニーを編集したり、ステップを追加したい場合、以下の詳細が適用される：

- まだキャンバスに入っていないユーザーは、新しく作成されたステップの対象となります。 
- キャンバスのエントリ設定でステップの再エントリを許可している場合は、新しく作成されたステップにすでに適合しているユーザーも再度エントリの対象となります。
- 現在開始されているキャンバスにいるが、新しいステップが追加されるユーザージャーニーのポイントに到達していないユーザーは、新しく追加されたステップを受け取る資格があります。 

[遅延]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/delay_step/)または[アクションパス]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/action_paths/)ステップを削除した場合、オプションで現在ステップで待機しているユーザーを別のキャンバスステップにリダイレクトすることができます。遅延が発生した場合、ユーザーは遅延期間が終了するまでそのステップに留まる。アクションパスでは、ユーザーは評価ウィンドウの終了までそのステップに留まる。

キャンバスの初回開始時に、Braze はキャンバス内の後続のすべてのメッセージではなく、そのユーザーがいるメッセージステップのユーザーをキューに入れます。起動後にキャンバスを編集した場合、一部のユーザーは既にキューに追加されている可能性があり、変更が反映されないことがある。キャンバスを停止し、複製してから変更を加え、この新しいバージョンを起動すると、キャンバスは既にキューに入れられていないユーザーだけでなく、全てのユーザーを再度評価する。

具体的な編集のユースケースについては、[ベストプラクティス](#best-practices)のセクションを参照してください。一般的に、予期しない動作が発生する可能性もあるので、ベストプラクティスとしては、ライブキャンバスの編集を避けることをおすすめします。

{% details Expand for original Canvas editor details %}

キャンバスがどのワークフローで作成されたかに応じて、開始後のキャンバスは以下の編集が可能です。もしあなたのキャンバスがオリジナルのキャンバスワークフローを使用しているなら、公開後の編集を行うにはまずキャンバスフローに複製する必要がある。

既存の接続は編集も削除もできず、既存の接続されたステップの間にステップを挿入することもできません。キャンバスのユーザーージャーニーを編集したり、ステップを追加したい場合、以下の詳細が適用される：

- まだキャンバスに入っていないユーザーは、新しく作成されたステップの対象となります。 
- キャンバスのエントリ設定でステップの再エントリを許可している場合は、新しく作成されたステップにすでに適合しているユーザーも再度エントリの対象となります。
- 現在開始されているキャンバスにいるが、ユーザージャーニーで新しく追加されたステップに到達していないユーザーは、新しく追加されたステップを受け取る資格があります。
- 延期期間ステップがキャンバスの最後のステップである場合、そのステップに到達したユーザーは自動的にキャンバスから昇格され、新しく作成されたステップは受け取りません。

{% alert important %}
キャンバスステップの**遅延**時間や**ウィンドウ**設定を更新した場合、更新時点でそのステップにいるユーザーは、当初そのステップに入った時に割り当てられた遅延時間を遵守する。キャンバスに新規で入るユーザーと、そのステップの待機列にまだ入っていないユーザーだけが、更新された時間にメッセージを受け取る。
{% endalert %}

キャンバスの停止は、メッセージの受信を待機しているユーザーを終了させない。キャンバスを再イネーブルメントしても、ユーザーがまだメッセージを待機している場合、そのメッセージを受け取る（ただし、メッセージが送信されるべき時間が過ぎている場合は受け取れない）。

{% enddetails %}

## キャンバスの詳細

キャンバスを起動した後、以下の設定と詳細を編集できる：

* キャンバスの名前と説明
* チームとタグ
* エントリタイプ、スケジュール、コントロール
* サブスクリプションステータス
* レート制限
* フリークエンシーキャップ
* サイレント時間
* ターゲットオーディエンス

キャンバスの開始後:

- コンバージョンイベントは編集できない。 
- 次の手順は追加または削除できず、順番を変更してランクを調整することもできません。[オーディエンスパス]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/audience_paths/)、[アクションパス]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/action_paths/)、[実験パス]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/experiment_step/)
  - **回避策1:**新しいオーディエンスパス、アクションパス、または実験パスを作成し、その新しいステップへのパスを再設定します。
  - **回避策2:**キャンバスを複製して編集します。

### 個々のステップ

キャンバスの個々のステップについては、開始後に以下の詳細を編集できます。

* 名前
* メッセージの内容
* トリガー
* オーディエンス
* 例外イベント
* 遅延（遅延ステップのみ）

ただし、ステップのスケジュールタイプとコントロールパーセンテージは開始後に編集できません。アクションパスとオーディエンスパスのステップについては、公開後はランキングと評価期間を変更できない。

### キャンバスバリアントのパーセント

キャンバスの開始後は、コントロールバリアントをパーセントで減らすことしかできません。キャンバスでバリアントのパーセントを変更した場合、ユーザーが他のバリアントに再配布される可能性があることがわかります。

初回は、これらのユーザーは特定のバリアントがランダムに割り当てられ、その後キャンペーンを受け取ります。それ以降、キャンペーンが受信されるたびに（あるいはユーザーがキャンバス・バリアントを再入力するたびに）、バリアントの割合が変更されない限り、同じバリアントが配信される。

バリアントのパーセンテージが変わると、ユーザーは他のバリアントに再配分される可能性があります。再びパーセントを変更するまで、ユーザーはこれらのバリアントに留まります。ランダムバケット番号を `NOT` フィルターで絞り込んだブランチを使用しているキャンバスでは、ユーザーがユーザージャーニーで再度キャンバスに入る際に、毎回同じブランチを受け取るとは限りません。

#### コントロールグループ

バリアントのパーセントが変わらない限り、コントロールグループも変わりません。コントロールグループのパーセンテージが増減すると、以前にメッセージを受け取ったユーザーは、後の送信でコントロールグループに入れなくなり、コントロールグループのユーザーがメッセージを受け取ることもなくなります。

### ローカルの送信時間

現地時間のスケジュールされた送信時間に設定されたキャンバスは、スケジュールされた送信時間の24時間前まで編集できる。この期間は「セーフゾーン」と呼ばれます。 

{% alert tip %}
完全に新しいキャンバスを作成するなどの大規模な編集を行う場合は、最初のキャンバスを受け取ったユーザーを除外し、タイムゾーンの送信を考慮して、キャンバスのスケジュール時間を再調整してください。
{% endalert %}

エントリスケジュールが起動時にユーザーを即時参加させる設定の場合、キャンバスは5分刻みで最も近い時間に起動する。例えば、キャンバスを更新してユーザーを太平洋標準時午前8時31分に即時参加させる場合、開始時刻は太平洋標準時午前8時30分と会社のタイムゾーンに設定される。

### バリアントを削除する

キャンバスからバリアントが削除されると、次のことが起こる：

- バリアント内のステップ（他のバリアントと共有されているものを含む）は削除される。 
- キャンバスのステップ分析と上位分析（_総エントリ数_、_総退出数_、_コンバージョン率_など）は削除される。
- 削除されたバリアントのユーザーはステップから退出し、それに続くメッセー ジは送信されません。

### キャンバスエントリのプロパティ

キャンバスエントリプロパティは、送信時にステップにテンプレート化されません。これは、キャンバス起動後にキャンバスのエントリプロパティを編集した場合、その変更は新たにキャンバスに入るユーザーにのみ適用されることを意味する。もしあなたのキャンバスがユーザーの再入場を許可している場合、再入場するユーザーは更新されたキャンバスのエントリプロパティによって決定される。

## ベストプラクティス

キャンバスを起動した後で編集または追加する場合は、次のベストプラクティスを確認してください。

{% alert important %}
一般的に、キャンバスがアクティブな状態で変更を加えたり、ユーザーをキューに追加したりするのは避けるべきだ。
{% endalert %}

### 切断されたステップ

切断されたステップでキャンバスを開始し、開始後にキャンバスを保存することもできます。ワークフローからステップを切断する前に、保留中のユーザーのステップの分析ビューを確認することをお勧めします。

たとえば、ユーザーがキャンバスのワークフローの切断されたステップにいるとします。このユーザーは、次のステップがある場合に進む。そのステップの設定が、ユーザーがどのように進むべきかを決める。 

切断されたステップを作成・編集することで、キャンバスの残りの部分に直接接続することなく、独立したステップに変更を加えることができます。これは、キャンバスを再起動する前にステップをテストするのに役立つ。 

### 実験パスステップ

キャンバスにアクティブまたは進行中の「勝者パス」または「パーソナライズされたパス」実験がある場合、アクティブなキャンバスを更新すると（実験ステップ自体を更新するか否かに関わらず）、進行中の実験は終了する。そして実験ステップは勝者パスやパーソナライズされたパスを決定しない。実験を再開するには、既存の実験パスをを解除して新しい実験パスを開始するか、キャンバスを複製して新しいキャンバスを起動します。そうでなければ、ユーザーは最適化手法が選択されていないかのように実験パスを流れる。

### 遅延

時間差のあるキャンバスの編集はやや複雑だ。キャンバスを編集する際には以下の点に注意せよ：

- 遅延ステップで遅延時間を更新した場合、キャンバスに新しく入ったユーザーと、そのステップの待機列にまだ入っていないユーザーだけが、更新された遅延時間でメッセージを受け取る。
- 時間遅延のあるステップ（遅延やアクションパスなど）を削除し、それらのユーザーを別のキャンバスステップにリダイレクトする場合、ユーザーはステップの時間遅延が完了した後にのみリダイレクトされる。例えば、1日遅延の遅延ステップを削除し、それらのユーザーをメッセージステップにリダイレクトするとしよう。この場合、ユーザーは1日の遅延期間が終了した後にのみリダイレクトされる。
- キャンバスに1つ以上の実験パスステップが含まれている場合は、ステップを削除すると、このステップの結果が無効になる場合があります。

### キャンバスを停止する

キャンバスの停止は、ステップで待機中のユーザーを終了させない。キャンバスを再イネーブルメントしてもユーザーが待機状態のままだと、そのステップを完了して次のステップに進む。しかし、ユーザーが次のステップに進むべき時間が過ぎた場合、代わりにキャンバスから退出する。 

例えば、キャンバスフローワークフローを使って作成したキャンバスがあるとしよう。そのキャンバスは午後2時に起動するように設定され、1つのバリアントと2つのステップを持つ。1つ目のステップは1時間の遅延を持つ遅延ステップで、そこからメッセージステップへ進む。 

ユーザーは午後 2 時 1 分にこのキャンバスに入り、同時に遅延ステップに入ります。これは、ユーザーが午後3時1分にユーザージャーニーの次のステップ（メッセージステップ）へスケジュールされていることを意味する。午後2時30分にキャンバスを停止し、午後3時30分にイネーブルメントを行った場合、午後3時1分を過ぎているためユーザーはキャンバスから退出する。ただし、午後2時40分にキャンバスを再イネーブルメントすると、ユーザーは午後3時01分に予想通りメッセージステップに進む。

## 知っておくべきこと

起動後にキャンバス内の他のコンポーネントを編集したり、コンポーネントを追加したりすると、以下の一般的な問題がトリガーされる可能性がある。 

{% alert important %}
以下の問題は避けられる。キャンバスを公開した後に編集が必要な場合、まず既にキャンバスに入った全ユーザーが自身のユーザー体験を完了していることを確認することを推奨する。加えて、少なくとも1人のユーザーによって処理済みのステップは削除しないことを推奨する。
{% endalert %}

- 欠落したレポートデータ（メッセージバリアントが削除され、再追加された場合）
- ユーザーは想定されたパスをたどっていない
- メッセージは予期せぬ時に届く
- 編集はCurrentsデータを上書きしないため、キャンバスステップとの間に不一致が生じる場合がある（例えば`canvas_step_ids`、削除によりキャンバスに存在しないステップなど）。
- ユーザーは同じメッセージを二度受け取ることがある
- ユーザーは既存のレート制限のため、メッセージを受け取れない。
  - アクティブなキャンバスでレート制限を更新すると、新しい制限は今後のすべてのメッセージ送信に適用される。これには既にキャンバスにいるユーザーも含まれる。ただし、内部キャッシュ（最大30秒）のため、新しいレート制限が完全に適用されるまでにわずかな遅延が生じる場合がある。Brazeはユーザーを現在のメッセージステップにキューイングするため、各ステップのメッセージが実際に送信される時点で適用されるレート制限が有効になることに注意せよ。
- キャンバスが[自動的に停止]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/about_statuses/#available-statuses)されると、起動後のキャンバスの下書きも削除される。
