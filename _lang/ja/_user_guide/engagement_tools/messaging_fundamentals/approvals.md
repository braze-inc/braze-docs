# キャンペーンおよびキャンバスの承認

> アプリ rovals を使用して、キャンペーンとキャンバスに最終チェックポイントを追加してから起動します。このワークフローを使用すると、メッセージの必要なすべてのセクションでコンテンツを検証およびアプリローブできます。

## 仕組み

キャンペーンまたはキャンバスの詳細は、編集の最終ステップで確認できます。 

キャンバスとキャンペーンのどちらの場合も、変更を保存してからアプリロービングを実行する必要があります。アプリの適切な権限を持つユーザーは、サマリーの各セクションをアプリローブしてからメッセージを起動する必要があります。各セクションのデフォルトステータスは**承認待ち**です。

{% tabs %}
{% tab campaign %}
キャンペーンを起動するには、次のコンポーネントをアプリローブする必要があります。

- **メッセージ:**これがキャンペーンメッセージです。
- **配信:**これは配信の種類であり、ユーザーがいつキャンペーンを受信するかを決定します。
- **ターゲットオーディエンス**これにより、誰がキャンペーンを受信するかが決まります。
- **コンバージョンイベント:**これは、エンゲージメントとレポートの目的で追跡しているメトリクスです。
{% endtab %}

{% tab canvas %}
キャンバスを起動するには、次の主要コンポーネントを承認する必要があります。

- **コンバージョンイベント:**これは、エンゲージメントとレポートの目的で追跡しているメトリクスです。
- **エントリスケジュール:**これには、エントリ スケジュールの種類とユーザー s がキャンバスに入るときが含まれます。
- **ターゲットオーディエンス**これにより、誰がこのキャンバスに入るかが決まります。
- **送信設定:**これらは、キャンバス内のすべてのステップの送信オプションです。 
- **キャンバスを作成:**これはキャンバスのユーザージャーニーです。
{% endtab %}
{% endtabs %}

## 承認ワークフローの有効化

デフォルトでは、キャンペーンとキャンバスのアプリ承認ワークフロー設定は無効になっています。この機能を有効にするには、[**設定** > [**承認ワークフロー**] に移動し、該当するトグルを選択します。

- **[ワークスペース] のすべてのキャンペーンにアプリの承認ワークフローを使用する**
- **[ワークスペース] 内のすべてのキャンバスにアプリの承認ワークフローを使用する**

{% alert important %}
[API キャンペーンs]({{site.baseurl}}/api/api_campaigns) および[Trans アクション al E メールキャンペーンs]({{site.baseurl}}/user_guide/message_building_by_channel/email/transactional_message_api_campaign) では、キャンペーン アプリ roval はサポートされていません。
{% endalert %}

## ユーザー権限の設定

アプリ 認証ワークフローを有効にした後、企業ユーザーがキャンペーンおよびキャンバスをアプリまたは拒否できるように、ユーザー権限を設定する必要があります。どちらの権限も、ワークスペースまたは[Teams]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/teams/) にアプリ存在するか、[権限セット]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/#permission-sets) に追加することもできます。

{% tabs %}
{% tab campaign %}
[「キャンペーンを承認および却下」権限]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/#managing-limited-and-team-role-permissions)が必要です。この権限は、誰がキャンペーンの承認ステータスを更新できるかを制御します。この権限を使用すると、次の操作を実行できます。

- セルフアプリ運動
- キャンペーンの承認と開始
- キャンペーンを承認しますが、キャンペーンを起動しません("キャンペーンの送信、キャンバス&クォート;キャンペーンを起動する権限を持つ別のユーザー)。
- アプリもキャンペーンを開始しない

アプリ roval ステータス es が**Summary** ステップに設定された後、キャンペーンに加えられた後続の変更は、保存時にすべてのアプリ roval ステータスes をリセットします。このアプリは、下書き キャンペーンまたは起動後キャンペーンのいずれかで行われた変更にあります。たとえば、ターゲットオーディエンスのみを変更する場合、**Summary** ステップは、すべてのセクションのアプリ roval ステータス es をデフォルト ステート**Pending アプリ roval** に戻します。

{% endtab %}

{% tab canvas %}
[「キャンバスを承認および却下」権限]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/user_permissions/#managing-limited-and-team-role-permissions)が必要です。この権限コントロールでは、キャンバスのアプリ権限ステータスに更新できます。この権限を使用すると、次の操作を実行できます。

- セルフアプリによるキャンバスのローブ
- キャンバスの承認と起動
- キャンバスを承認するが起動しない("キャンペーンの送信、キャンバスとクォート;権限でキャンバスを起動できる別のユーザー)
- アプリがキャンバスをローブしたりローンチしたりしない

アプリ roval ステータス es が**Summary** ステップに設定された後、キャンバスに加えられた後続の変更はすべて、保存時にすべてのアプリ roval ステータスes をリセットします。これは、下書きキャンバスと開始後のキャンバスのいずれで行われた変更にも適用されます。たとえば、対象のオーディエンスのみを変更する場合、**Summary** ステップは、すべてのセクションのアプリ roval ステータス es を、**Pending アプリ roval** というデフォルトステートに戻します。

{% alert note %}
**認定ステータスと保存**

- **アプリ rove** を押すと、**Summary** ステップのセクションに対して、そのアプリ rove がすぐに保存されます。
- **Save**ボタンを押すと、キャンバスの内容と設定sの変更が保存され、アプリの承認ステータスは保存されません。

アプリロバールの損失を防ぐには:

1. 必要なキャンバス編集を行い、**Save**をクリックします。
2. キャンバスの保存が終了したら、アプリ は**Summary** ステップの関連する部分を探します。
3. アプリ承認後にキャンバスを追加変更した場合にのみ、再度**Save**をクリックします。キャンバスを変更して保存すると、すべてのアプリ認証ステータスは**保留中のアプリ認証範囲**にリセットされます。
{% endalert %}
{% endtab %}
{% endtabs %}

{% alert important %}
ライブキャンペーンを編集するには、"Approve and Deny Campaigns" permission が必要です。下書き版のキャンペーンがまだ利用できないため、ユーザーはアプリ変更をローブする必要があります。キャンバスの場合はこの限りではなく、ユーザーが変更を加えて下書きとして保存し、別のユーザーがキャンバスを承認して開始できます。
{% endalert %}
