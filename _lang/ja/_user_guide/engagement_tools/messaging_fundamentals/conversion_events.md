# コンバージョンイベント

> コンバージョンイベントは、メッセージの受信者がエンゲージメントを受け取ってから一定時間以内に価値の高いアクションを実行したかどうかを追跡する成功指標の一種です。コンバージョンイベントを使用すると、関連性のある有用な情報を確実に収集し、後にキャンペーンまたはキャンバスのインサイトを得るためにこれらの情報を使用できます。

## CDI の仕組み

アクティブユーザーsをターゲットとするパーソナライズされたのホリデーキャンペーンの場合、2～3日以内に**セッションを開始**のコンバージョンイベントがアプリ適切かもしれません。なぜなら、それはあなたのメッセージを受け取ることからユーザー エンゲージメント感を集めることができるからです。また、**Places Order**、**Upgrade App**、または任意のカスタムイベントs をコンバージョンイベントとして選択することもできます。

{% alert tip %}
コンバージョンの詳細については、キャンペーン設定に関する [Braze ラーニングコース](https://learning.braze.com/campaign-setup-delivery-targeting-conversions)をご覧ください。
{% endalert %}

### コンバージョントラッキングルール

変換事象属性 ユーザー アクションはエンゲージメント点に戻る。Braze がマルチコンバージョンs を処理する方法については、次の点に注意してください。

- **シングル-チャネル キャンペーンs**:コンバージョンは、デバイス単位ではなく、ユーザー単位で発生する。1 つのチャネル内で、1 つのユーザーが変換されるのは、1 つのメールが複数の機器に送信された場合でも、コンバージョンイベントごとに1 回だけです。たとえば、キャンペーンに&quot に設定されているコンバージョンイベントが1 つだけで、購入&クォートを設定し、ユーザーがコンバージョンの期限内に2 つの個別購入を行った場合、Braze は1 つのコンバージョンのみをカウントします。
- **複数チャネル キャンペーンs**:マルチチャネル キャンペーンs の場合、それぞれのチャネルには独自のコンバージョン商談があります。ユーザーは、そのチャネルでメッセージを受信した後、チャネルごとに1 回変換できます。つまり、ユーザーが複数のチャネルs(たとえば、メールとプッシュの両方)でメッセージを受信し、コンバージョン アクションを実行した場合、Brazeはチャネルごとに1つのコンバージョンを数えます。これにより、コンバージョン率が100%を超えることがあります。
- ユーザーが1つのコンバージョンイベントを、受信した2つの別々のコンバージョンのキャンペーンまたはキャンバスの期限内に実行すると、コンバージョンは両方で登録されます。
- ユーザーは、メッセージを開封またはクリックしなかった場合でも、ウィンドウで具体的なコンバージョンイベントを実行した場合、変換されたものとして数えられます。
- キャンバスの場合、コンバージョン "トラッキングは、個々のメッセージタイミングではなく、ユーザーがキャンバスに入ったときに開始される最終コンバージョン期限に基づいて機能します。Brazeは、キャンバス内のメッセージ間の遅延時間中でもコンバージョンを数えます。

### 主なコンバージョンイベント

1次コンバージョンイベントは、キャンペーンまたはキャンバスの作成時に追加する最初のイベントです。このイベントは、エンゲージメントとレポーティングに最も影響します。Braze では、1次コンバージョンを使用して次の操作を行います。

- [多変量]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/#multivariate-and-ab-testing)キャンペーンまたはキャンバスで成功メッセージのバリエーションを計算します。
- キャンペーンまたはキャンバスの収益を計算する期間を指定します。
- [インテリジェントセレクション]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_selection/)を使用して、キャンペーンおよびキャンバスのメッセージ配信を調整します。

1次コンバージョンのイベントカウントは、発生したコンバージョンイベントs の数です。マルチチャネル キャンペーンs の場合、Braze はコンバージョン s をチャネルごとにカウントします([ コンバージョン "トラッキング ルール](#conversion-tracking-rules) で説明されているように)。これは、コンバージョン数が一意のユーザーs の数を超え、100% を超えるコンバージョン率になることを意味します。Braze は、この数を一意の受信者s の数で割って、1次コンバージョンイベントレートを計算します。Braze は、チャネルに応じて、メッセージの送信時または表示時にユーザーを受信者と見なします。たとえば、プッシュまたはメールでは、Brazeがメッセージを送信した後、ユーザーが受信者になります。アプリ内メッセージまたはコンテンツカードの場合、ユーザーは受信者と見なされるメッセージを表示する必要があります。

{% alert note %}
リキッド`abort` タグを使用してメッセージを中止すると、Braze はバリアントs を経由するユーザーs に対してのみメッセージを中止します。コントロールグループ内のユーザー s へのメッセージはアボートされず、その結果、コンバージョン percen es がコントロールグループ s とバリアント s 間でスキューするタグになります。回避策として、[セグメンテーションを使って]({{site.baseurl}}/user_guide/engagement_tools/segments/creating_a_segment)、キャンペーンとキャンバスのエントリー時にユーザーをターゲティングする。
{% endalert %}

## コンバージョントラッキングを使ってキャンペーンを作成

### ステップ1:キャンペーンの設定

目的のメッセージングチャネル用の[キャンペーンを作成]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/creating_campaign)します。キャンペーンのメールとスケジュールを設定した後、"トラッキングに最大4つのコンバージョンイベントを追加できます。

コンバージョンイベントは、必要な数だけ使用してください。2 番目または3 番目のコンバージョンイベントを追加すると、レポート処理が大幅に強化されます。たとえば、キャンペーン ターゲット離脱ユーザーs の場合、プライマリ**Starts Session** コンバージョンイベントとともにセカンダリコンバージョンイベントを追加すると、キャンペーンがユーザーをアプリのライセンスに戻す際の効果を理解するのに役立ちます。 

### ステップ2:変換イベントを追加する

まず、使用するイベントの一般的なタイプを選択します。

| コンバージョンイベントタイプ   | 説明                |
|-------------------------|----------------------------|
| **セッションを開始**      | ユーザーは、指定したアプリのいずれかを開くと、コンバージョンしたものとみなされます (これがワークスペース内のすべてのアプリでのデフォルト設定)。|
| **購入**      | ユーザーは、[購入イベント]({{site.baseurl}}/api/objects_filters/purchase_object/)を記録すると、変換されたものとしてカウントされます。これにより、デフォルトでの購入を追跡したり、特定のプロダクトを指定したりできます。|
| **注文する**        | ユーザーは、[発注済eコマース推奨イベント]({{site.baseurl}}/user_guide/data/activation/custom_data/recommended_events/ecommerce_events?tab=ecommerce.order_placed#ecommerce-recommended-events)をトリガーすると、変換済みとしてカウントされます。デフォルトで注文を追跡したり、商品ごとにフィルターしたりできます。|
| **カスタムイベントの実行**| ユーザーは、既存のカスタムイベントのいずれかを実行するとコンバージョンしたものとみなされます (デフォルト設定ではなく、イベントの指定が必要)。|
| **アプリのアップグレード**         | ユーザーは、指定したいずれかのアプリでアプリのバージョンをアップグレードすると、コンバージョンしたものとみなされます (これがワークスペース内のすべてのアプリでのデフォルト設定)。Braze はベストエフォートで数値比較を行い、バージョン変更がアップグレードによるものかどうかを判断します。非数値バージョンは、バージョンが変更された場合に変換としてカウントされます。|
| **メールを開く**         | ユーザーがメールを開封した時点でコンバージョンしたとカウントされる（メールキャンペーンの場合のみ）。|
| **メールをクリック**        | ユーザーがメール内のリンクをクリックすると、コンバージョンしたとカウントされる（メールキャンペーンのみ）。|
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

{% alert important %}
**ネストされたプロパティーは、コンバージョンイベント s** ではサポートされません。コンバージョンイベント s ではネストされたプロパティーを使用できません。たとえば、`product_code` または`product_name` が`products` 配列内のネストされたプロパティ(`products[].product_code` など) の場合、それらを使用してコンバージョンイベントで特定の商品が購入されたかどうかを確認することはできません。
{% endalert %}

コンバージョンの期限を設定します。これは、Braze がコンバージョンを認識するまでの最長時間です。ユーザーが指定されたアクションをとった場合にBrazeがコンバージョンを数える最大30日間のウィンドウを設定できます。

変換イベントを選択したら、キャンペーン作成プロセスを続行してキャンペーンの送信を開始します。

### ステップ 3:結果の表示

**Details**ページに移動して、作成したキャンペーンに関連付けられたコンバージョンイベントごとの詳細を表示します。選択したコンバージョンイベントs に関係なく、1次コンバージョンのウィンドウで、この固有のキャンペーンと固有のバリアントs の合計収益属性d を確認することもできます。

{% alert note %}
キャンペーンの作成中にコンバージョンイベントs を選択しなかった場合、タイムデフォルトはs から3 日になります。
{% endalert %}

さらに、多変量分析メッセージの場合、コントロールグループと各バリアントのコンバージョン数とコンバージョン率を確認できます。
