# コンバージョンイベント

> コンバージョンイベントは、メッセージの受信者がエンゲージメントを受け取ってから一定時間以内に価値の高いアクションを実行したかどうかを追跡する成功指標の一種です。コンバージョンイベントを使用すると、関連性のある有用な情報を確実に収集し、後にキャンペーンまたはキャンバスのインサイトを得るためにこれらの情報を使用できます。

## 仕組み

アクティブユーザーを対象としたパーソナライズされたホリデーキャンペーンでは、メッセージ受信後のユーザーエンゲージメントを把握できるため、2～3日以内に「**セッション開始**」というコンバージョンイベントを設定するのが適切だろう。**注文を確定する**、**アプリをアップグレードする**といった追加のイベントや、カスタムイベントもコンバージョンイベントとして選択できる。

{% alert tip %}
コンバージョンの詳細については、キャンペーン設定に関する [Braze ラーニングコース](https://learning.braze.com/campaign-setup-delivery-targeting-conversions)をご覧ください。
{% endalert %}

### コンバージョントラッキングルール

コンバージョンイベントは、ユーザーのアクションをエンゲージメントに紐づける。Brazeが複数のコンバージョンを処理する方法について、以下の点に注意せよ：

- **シングルチャネルキャンペーン**：コンバージョンは、デバイス単位ではなく、ユーザー単位で発生する。単一のチャネル内では、ユーザーはコンバージョンイベントごとに一度だけ変換される。たとえメッセージが複数のデバイスに送信された場合でも同様である。例えば、キャンペーンに「購入を行う」というコンバージョンイベントが1つだけ設定されており、ユーザーがコンバージョン期限内に2回の別々の購入を行った場合、Brazeは1回のコンバージョンのみとしてカウントする。
- **マルチチャネルキャンペーン**：マルチチャネルキャンペーンでは、各チャネルごとに独自のコンバージョン機会がある。ユーザーは、あるチャネルでメッセージを受信した後、そのチャネルごとに1回変換できる。これは、ユーザーが複数のチャネル（例えばメールとプッシュ通知の両方）でメッセージを受け取り、コンバージョンアクションを実行した場合、Brazeは各チャネルごとに1回のコンバージョンをカウントすることを意味する。その結果、コンバージョン率が100%を超えることがある。
- ユーザーが、受け取った2つの別々のキャンペーンまたはキャンバスのコンバージョン期限内に1つのコンバージョンイベントを実行した場合、そのコンバージョンは両方に記録される。
- ユーザーは、メッセージを開封したりクリックしたりしていなくても、その期間内に特定のコンバージョンイベントを実行した場合、コンバージョンしたと見なされる。
- キャンバスの場合、コンバージョントラッキングは個々のメッセージのタイミングではなく、ユーザーがキャンバスに入った時点から始まる最終コンバージョン期限に基づいて機能する。Brazeは、Canvas内のメッセージ間の遅延期間中もコンバージョンをカウントする。

### 主なコンバージョンイベント

1次コンバージョンイベントとは、キャンペーンやキャンバス作成時に最初に追加するイベントである。このイベントは、エンゲージメントとレポーティングに最も影響します。Brazeは、あなたの1次コンバージョンイベントを以下の目的で使用する：

- [多変量]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/#multivariate-and-ab-testing)キャンペーンまたはキャンバスで成功メッセージのバリエーションを計算します。
- キャンペーンまたはキャンバスの収益を計算する期間を指定します。
- [インテリジェントセレクション]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_selection/)を使用して、キャンペーンおよびキャンバスのメッセージ配信を調整します。

1次コンバージョンイベントのカウントとは、発生したコンバージョンイベントの数を指す。マルチチャネルキャンペーンでは、Brazeはチャネルごとのコンバージョンをカウントする（コンバージョントラッキングルールで説明されている通り）。これはコンバージョン数がユニークユーザー数を上回り、コンバージョン率が100%を超える結果をもたらすことを意味する。Brazeは、このカウントをユニークな受信者数で割ることで、1次コンバージョンイベント率を算出する。Brazeは、メッセージが送信された時、あるいは表示された時、チャネルに応じてユーザーを受信者とみなす。例えば、プッシュ通知やメールの場合、Brazeがメッセージを送信した後にユーザーが受信者となる。アプリ内メッセージやコンテンツカードの場合、ユーザーはメッセージを表示しなければ受信者とみなされない。

{% alert note %}
Liquid`abort`タグを使ってメッセージを中止する場合、Brazeはバリアントを経由するユーザーに対してのみメッセージを中止する。コントロールグループのユーザーへのメッセージは中止されないため、バリアント群とコントロールグループ間でコンバージョン率に偏りが生じる可能性がある。回避策として、[セグメンテーションを使って]({{site.baseurl}}/user_guide/engagement_tools/segments/creating_a_segment)、キャンペーンとキャンバスのエントリー時にユーザーをターゲティングする。
{% endalert %}

## コンバージョントラッキングを使ってキャンペーンを作成

### ステップ1:キャンペーンの設定

目的のメッセージングチャネル用の[キャンペーンを作成]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/creating_campaign)します。キャンペーンのメッセージとスケジュールを設定した後、最大4つのコンバージョンイベントを追加してトラッキングできる。

必要なだけ多くのコンバージョンイベントを使用せよ。2つ目や3つ目のコンバージョンイベントを追加すると、レポートの内容が大幅に充実する。例えば、離脱ユーザーを対象としたキャンペーンでは、主要な**「セッション開始」**コンバージョンイベントに加えて、二次的なコンバージョンイベントを追加することで、キャンペーンがユーザーをアプリに呼び戻す効果を把握できる。 

### ステップ 2:変換イベントを追加する

まず、使用するイベントの一般的なタイプを選択します。

| コンバージョンイベントタイプ   | 説明                |
|-------------------------|----------------------------|
| **セッションを開始**      | ユーザーは、指定したアプリのいずれかを開くと、コンバージョンしたものとみなされます (これがワークスペース内のすべてのアプリでのデフォルト設定)。|
| **購入**      | ユーザーが[購入イベント]({{site.baseurl}}/api/objects_filters/purchase_object/)を記録した時点で、コンバージョンが発生したとみなされる。デフォルトではすべての購入をトラッキングする。または特定の商品を指定することもできる。|
| **注文する**        | ユーザーが[「注文完了」のe コマース推奨イベント]({{site.baseurl}}/user_guide/data/activation/custom_data/recommended_events/ecommerce_events?tab=ecommerce.order_placed#ecommerce-recommended-events)をトリガーした時点で、コンバージョンが発生したとみなされる。デフォルトではすべての注文をトラッキングする。または特定の製品でフィルターをかけることもできる。<br><br>「注文を配置する」イベントは現在、早期アクセス中だ。早期アクセスへの参加をご希望の場合は、Braze のアカウントマネージャーにご連絡ください。 |
| **カスタムイベントの実行**| ユーザーは、既存のカスタムイベントのいずれかを実行するとコンバージョンしたものとみなされます (デフォルト設定ではなく、イベントの指定が必要)。|
| **アプリのアップグレード**         | ユーザーは、指定したいずれかのアプリでアプリのバージョンをアップグレードすると、コンバージョンしたものとみなされます (これがワークスペース内のすべてのアプリでのデフォルト設定)。Braze はベストエフォートで数値比較を行い、バージョン変更がアップグレードによるものかどうかを判断します。非数値バージョンは、バージョンが変更された場合に変換としてカウントされます。|
| **メールを開く**         | ユーザーがメールを開封した時点でコンバージョンしたとカウントされる（メールキャンペーンの場合のみ）。|
| **メールをクリック**        | ユーザーがメール内のリンクをクリックすると、コンバージョンしたとカウントされる（メールキャンペーンのみ）。|
{: .reset-td-br-1 .reset-td-br-2 role="presentation"}

{% alert important %}
**ネストされたプロパティはコンバージョンイベントではサポートされていない**。コンバージョンイベントではネストされたプロパティは使えない。例えば、または`product_name`が`products`配列（例えば`products[].product_code`）内のネストされたプロパティである`product_code`場合、特定の製品購入がコンバージョンイベントで行われたかどうかを確認するためにそれらを使用することはできない。
{% endalert %}

コンバージョンの期限を設定します。これは、Brazeがコンバージョンとみなすまでに経過できる最大時間である。ユーザーが指定されたアクションを実行した場合、Brazeがコンバージョンをカウントする期間を最大30日間まで設定できる。

![例えば、「購入する」というコンバージョンイベントタイプは、何らかの購入を行ったユーザーのコンバージョンを記録します。このコンバージョン期限は 12 時間です。]({% image_buster /assets/img_archive/conversion_event_selection.png %})

変換イベントを選択したら、キャンペーン作成プロセスを続行してキャンペーンの送信を開始します。

### ステップ 3:結果の表示

作成したキャンペーンに関連付けられた各コンバージョンイベントの詳細を確認するには、**詳細**ページに移動する。選択したコンバージョンイベントに関わらず、1次コンバージョンイベントの期間中に、この特定のキャンペーンおよび特定のバリアントに属する総収益を確認できる。

{% alert note %}
キャンペーン作成時にコンバージョンイベントを選択しない場合、期間はデフォルトで3日となる。
{% endalert %}

さらに、多変量分析メッセージの場合、コントロールグループと各バリアントのコンバージョン数とコンバージョン率を確認できます。

![4つのコンバージョンイベントは、購入が3時間以内に行われた場合、購入が2時間以内に行われた場合、セッションが30分以内に開始された場合、セッションが25分以内に開始された場合に基づいてコンバージョンをトラッキングする。]({% image_buster /assets/img_archive/conversion_event_details.png %})
