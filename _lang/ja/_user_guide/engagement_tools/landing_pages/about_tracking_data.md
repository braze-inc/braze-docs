# ランディングページトラッキングデータについて

> Braze でのランディングページのトラッキングと匿名データについて学習します。

## トラッキング、追跡方法

### ウェブSDK

ユーザーがランディングページでフォームを送信すると、Braze Web SDKが初期化される。フォーム送信の前に、個人データは収集されず、SDK はユーザーを積極的に追跡しません。初期化の完了後、SDK はブラウザーにデータ (Cookie、ローカルストレージなど) を保存しません。 

BrazeのウェブSDKは、ユーザーがBrazeメッセージ内のLiquid{% raw %}`{% landing_page_url %}`{% endraw %}タグによって生成されたリンクを通じてランディングページに移動した時点で、即座に初期化される。

フォームが送信されると、SDKは以下のデータを収集する：

- フォーム送信イベント (イベント名と送信時刻)
- チームがフォームで指定したデータ（名前、メール、電話番号など）
- セッション開始時間
- デバイスID（デバイス用に生成されるが、保存はされないユニークなID）。
- IPアドレスによって決定される国

### 匿名化されたデータ

ユーザーがフォームを送信する前に、ランディングページでトラッキング追跡されるデータは、匿名化された識別不能な情報のみで構成される。これは、ランディングページが受け取ったページビュー（インプレッション）やクリック数などの標準的なWebサイトの集計指標で構成されている。

このデータは識別可能なユーザーにリンクされていないため、リターゲティングしたり、個々のユーザー行動を追跡したりするために使用することはできない。

## 重複するユーザープロファイルのマージ

Braze は、ランディングページフォームが送信されたときに、メールや電話機などの属性に基づいてユーザーを自動的に結合しません。フォームが送信された際、そのメールや電話番号が既存のユーザープロファイルと一致する場合、Brazeは別のユーザープロファイルを作成する。

重複するユーザープロファイルを結合するには、次の操作を行います。

- ランディングページのフォームが送信された際にエンド[`/users/merge`]({{site.baseurl}}/api/endpoints/user_data/post_users_merge/)ポイントをトリガーし、新規プロファイルを既存のプロファイルとマージする。
- 一致する識別子に基づいて重複プロファイルを定期的にマージするには、[バルクマージ]({{site.baseurl}}/user_guide/engagement_tools/segments/user_profiles/duplicate_users/#bulk-merging) をスケジュールします。
