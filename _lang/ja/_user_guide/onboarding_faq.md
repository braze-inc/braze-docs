### 匿名ユーザーデータをどのように処理しますか?

{% apitags %}
ユーザー
{% endapitags %}

最初に、SDKを介してユーザープロファイルが認識されると、Brazeは関連付けられた匿名ユーザープロファイルを作成します。 `braze_id`: Braze によって設定される一意のユーザー識別子。

匿名ユーザーをさらに追跡するには、匿名ユーザーに識別子をタグ付けできる [ユーザー エイリアス]({{site.baseurl}}/user_guide/data_and_analytics/user_data_collection/user_profile_lifecycle#user-aliases) を実装できます。これらのユーザーは、エイリアスを使用してエクスポートしたり、API で参照したりできます。

エイリアスを持つ匿名ユーザープロファイルが後で `external_id`通常の識別されたユーザー プロファイルとして扱われますが、既存のエイリアスは保持され、そのエイリアスで引き続き参照できます。

識別されたユーザーと結合するエイリアス ユーザーの場合は、保持する実際のプロファイルに関連する任意のフィールドを結合できます。エイリアス プロファイルからデータを削除する前に、 [識別子エンドポイントによるユーザー プロファイルのエクスポート]({{site.baseurl}}/api/endpoints/export/user_data/post_users_identifier/)を使用してそのデータをエクスポートする必要があります。その後、 [ユーザー追跡エンドポイント]({{site.baseurl}}/api/endpoints/user_data/post_user_track/) を使用して、保存したプロファイルにこれらのイベントを投稿できます。これにより、以前は一方のプロファイルに記録されていたが、もう一方のプロファイルには記録されていなかった属性など、保持したいデータがすべて保持されます。

Braze で新規および既存のユーザー データを収集するためのさまざまな方法の詳細については、 [データ収集のベスト プラクティス]({{site.baseurl}}/user_guide/data_and_analytics/user_data_collection/best_practices/)をご覧ください。

### Braze の外部ですでに収集して識別したユーザーをインポートするにはどうすればよいですか?

{% apitags %}
ユーザー
{% endapitags %}

以前に識別されたユーザーをインポートするには、CSV を Braze にアップロードするか、API 経由でデータを送信します。

#### CSV

**「オーディエンス** 」>**「ユーザーのインポート」**から CSV ファイルを介してユーザー プロファイルをアップロードおよび更新できます。顧客データをインポートする際には、各顧客の一意の識別子（別名： `external_id`。

{% alert note %}
[古いナビゲーション]({{site.baseurl}}/navigation)を使用している場合、これは **[ユーザー]**>**[ユーザーのインポート**] にあります。
{% endalert %}

CSV インポートを開始する前に、エンジニアリング チームから Braze でユーザーがどのように識別されるかを理解することが重要です。通常、これは内部で使用されるデータベース ID になります。これは、モバイルと Web 上の Braze SDK によってユーザーが識別される方法と一致し、各顧客がデバイス間で Braze 内に単一のユーザー プロファイルを持つようになります。Braze [ユーザー プロファイル ライフサイクル]({{site.baseurl}}/user_guide/data_and_analytics/user_data_collection/user_profile_lifecycle/)の詳細をご覧ください。

あなたが `external_id` インポート時に、Brazeは既存のユーザーを同じ情報で更新します `external_id` または、そのユーザーを新たに作成して `external_id` 見つからない場合は設定します。

詳細情報および CSV インポート テンプレートのダウンロードについては、 [ユーザー インポート]({{site.baseurl}}/user_guide/data_and_analytics/user_data_collection/user_import/#csv)を参照してください。

#### API

API 経由でユーザーをアップロードするには、 [ユーザー追跡エンドポイント]({{site.baseurl}}/api/endpoints/user_data/post_user_track/) を使用して Braze にインポートできます。

ユーザーが Braze に既に存在するかどうか不明な場合は、 [識別子エンドポイントによるユーザー プロファイルのエクスポート]({{site.baseurl}}/api/endpoints/export/user_data/post_users_identifier/) を実装して確認できます。ユーザーがすでにBrazeに存在することが判明した場合は、 `/users/track` エンドポイントを使用して、Braze に既に存在するユーザー プロファイルに追加する新しいデータを投稿します。

{% alert note %}
使用する際は、以下のニュアンスに注意してください。 `/users/track` 終点：

- このエンドポイントを通じてエイリアスのみのユーザーを作成する場合は、 `_update_existing_only` フラグを false に設定します。
- このエンドポイントを使用してサブスクリプション ステータスを更新すると、外部 ID (User1 など) で指定されたユーザーが更新され、そのユーザー (User1) と同じメール アドレスを持つすべてのユーザーのサブスクリプション ステータスも更新されます。
{% endalert %}

### プッシュサブスクリプションステータスの違いは何ですか?

{% apitags %}
ユーザー
{% endapitags %}

プッシュ サブスクリプションの状態オプションには、サブスクライブ済み、オプトイン済み、サブスクライブ解除済みの 3 つがあります。

デフォルトでは、ユーザーがプッシュ経由でメッセージを受信するには、プッシュ サブスクリプションの状態がサブスクライブ済みまたはオプトイン済みであり、プッシュが有効になっている必要があります。メッセージを作成するときに、必要に応じてこの設定を上書きできます。

|オプトイン状態|説明|
|---|---|
|サブスクライブ済み| Braze でユーザー プロファイルが作成されたときのデフォルトのプッシュ サブスクリプション状態。 |
|オプトイン| ユーザーはプッシュ通知を受信することを明示的に希望しています。Brazeは自動的にユーザーのオプトイン状態を `Opted-In` ユーザーが OS レベルのプッシュ プロンプトを受け入れた場合。<br><br>これは Android 12 以下のユーザーには適用されません。|
|登録解除済み| ユーザーは、アプリケーションまたはブランドが提供するその他の方法を通じて、プッシュ通知を明示的に登録解除しました。デフォルトでは、Brazeプッシュキャンペーンは、 `Subscribed` または `Opted-in` プッシュ用。|

### 重複したユーザーを特定した場合はどうすればいいですか?

{% apitags %}
ユーザー
{% endapitags %}

重複するユーザーを特定した場合は、それらのユーザー プロファイルをクリーンアップする必要があります。以下の手順に従ってください。

1. ユーザープロファイルをエクスポートするには、 `/users/export/ids` 終点。
2. 正しいユーザー プロファイルを特定し (最終的にはチームが正しい情報を決定する必要があります)、次のいずれかを実行します。
    - 実際のプロファイルに関係するフィールドをマージし、 `/user/track` 終点。
    - users/delete エンドポイントを使用して、データをマージせずに重複した役に立たないプロファイルを削除します。ユーザー プロファイルを削除すると、 **その情報を取得する方法はありません**。

{% alert important %}
まず、正しいユーザープロファイルをインポートすることをお勧めします。 `external_id` および対応するカスタム属性とイベント。ユーザー プロファイルは削除されると取得できなくなるため、削除は最後の手順にする必要があります。
{% endalert %}

注目すべき追加の事項:

- 重複したユーザー プロファイル上のエンゲージメント データ (受信したキャンペーンやキャンバスなど) はすべて失われます。過去のエンゲージメント コンテキストを保持する唯一の方法は、それをカスタム属性 (受信したすべてのキャンペーンまたはキャンバスの配列カスタム属性など) として追加することです。
- ユーザー プロファイルを移行する場合、重複するユーザー プロファイルのうちどのプロファイルを保持するかを決定するのもチームの責任です。Braze は削除するプロファイルを決定したり、そのリストを提供したりすることはできません。  
- 最終的には、あなたのチームがユーザーの経験からサインアッププロセスを評価し、あなたが望むものだけを呼び出すようにすることが重要になります。 `changeUser()` ユーザーが識別されたときの方法。

### CSV 経由でユーザー グループをインポートするときにセグメントを作成するにはどうすればよいですか?

{% apitags %}
セグメント
{% endapitags %}

CSV ファイルをインポートするには、[ユーザー] セクションの **[ユーザー インポート]** ページに移動します。「 **最近のインポート」** テーブルには、最大 20 件の最新のインポート、そのファイル名、ファイル内の行数、正常にインポートされた行数、各ファイルの合計行数、および各インポートのステータスがリストされます。

**CSV のインポート** パネルには、インポートの手順とインポートを開始するためのボタンが含まれています。**「CSV ファイルの選択」** をクリックし、目的のファイルを選択します。次に、**[インポートの開始] を**クリックする前に、[この CSV 内のユーザーに対してどのような処理を行いますか] で、このリストをどのように処理するかを Braze に指示するオプションがあります。

**この CSV にユーザーをインポートするを選択し、この特定のユーザー バッチをグループとして再ターゲットできるようにしてから**、 **この CSV からインポートされたユーザーからセグメントを自動的に生成する**を選択します。**「インポートの開始」を**クリックすると、Braze はファイルをアップロードし、列ヘッダーと各列のデータ型をチェックして、セグメントを作成します。

CSV テンプレートをダウンロードするには、 [ユーザーのインポート]({{site.baseurl}}/user_guide/data_and_analytics/user_data_collection/user_import/#csv)を参照してください。

### ユーザーを最新の位置情報でセグメント化し、位置情報に基づくキャンペーンや戦略で使用できるように、位置情報ターゲティングを設定するにはどうすればよいですか?

{% apitags %}
セグメント
{% endapitags %}

現在のユーザー セグメントをすべて表示するには、「エンゲージメント」の **「セグメント」** ページに移動します。このページでは、新しいセグメントを作成して名前を付けることができます。開始するには、**「セグメントの作成」を** クリックし、セグメントに名前を付けます。

セグメントを作成したら、 `Most Recent Location` アプリを最後に使用した場所に基づいてユーザーをフィルタリングしてターゲットにします。標準の円形領域でユーザーを強調表示することも、カスタムの多角形領域を作成することもできます。

- 円形領域の場合は、原点を移動してセグメンテーションの位置半径を調整できます。
- 多角形領域の場合は、セグメントに含める領域をより具体的に指定できます。

{% alert tip %}
Braze パートナーの支援を受けて位置情報ターゲティングを活用することに興味がありますか?利用可能な Braze [コンテキスト ロケーション パートナー]({{site.baseurl}}/partners/message_personalization/location)を確認してください。
{% endalert %}

### 過去 365 日間のカスタム イベントと購入行動に基づいて、正確なユーザー リストをターゲットにするにはどうすればよいですか?

{% apitags %}
セグメント
{% endapitags %}

[セグメント拡張機能]({{site.baseurl}}/user_guide/engagement_tools/segments/segment_extension/)を使用できます。セグメント拡張機能を使用すると、通常のセグメントを使用する場合よりも正確なユーザー リストをターゲットにすることができます。

ワークスペースごとに最大 10 個のセグメント拡張機能を作成できます。これらの拡張子リストが生成されると、セグメントのフィルターとして含めたり除外したりできるようになります。セグメント拡張を作成するときに、リストが 24 時間ごとに再生成されるように指定することもできます。

1. 「エンゲージメント」の下で、**「セグメント」** を展開し、**「セグメント拡張」**をクリックします。
2. セグメント拡張機能テーブルから、**「+ 新しい拡張機能を作成」を**クリックします。
3. フィルタリングするユーザーのタイプを説明するセグメント拡張機能に名前を付けます。これにより、この拡張機能をセグメント内のフィルターとして適用するときに、簡単に正確に検出できるようになります。
4. ターゲティングには、購入またはカスタム イベントの基準を選択します。
5. ユーザー リストの対象とする購入済みアイテムまたは特定のカスタム イベントを選択します。 
6. ユーザーがイベントを完了する必要がある回数（これより多い、少ない、または等しい）と、最大 365 日まで遡る日数を選択します。

ターゲティングの精度を高めるには、**「プロパティ フィルターの追加」を** 選択し、購入またはカスタム イベントの特定のプロパティに基づいてセグメント化します。Braze は、文字列、数値、ブール値、および時間オブジェクトに基づくイベント プロパティのセグメンテーションをサポートします。

[ネストされたイベント プロパティ]({{site.baseurl}}/user_guide/data_and_analytics/custom_data/custom_events/nested_objects/)に基づくセグメンテーションもサポートしています。

セグメント拡張機能はイベント プロパティの長期保存に依存しており、30 日間のカスタム イベント プロパティ保存制限はありません。つまり、過去 1 年以内に追跡されたイベント プロパティを振り返ることができ、拡張機能が最初に設定されるまで追跡を待つ必要はありません。

{% alert note %}
セグメントエクステンション内でイベントプロパティを使用しても、データポイントの使用には影響しません。
{% endalert %}

#### セグメント拡張機能を最新の状態に保つ

{% apitags %}
セグメント
{% endapitags %}

このエクステンションで特定の時点の 1 つのスナップショットを表すかどうか、またはこのエクステンションを毎日再生成するかどうかを指定できます。エクステンションは常に最初の保存後に処理を開始します。拡張機能を毎日再生成する場合は、**「拡張機能を毎日再生成」** を選択すると、会社のタイムゾーンで毎日深夜頃に再生成処理が開始されます。

完了したら、**「保存」**をクリックします。拡張機能の処理が開始されます。拡張機能の生成にかかる時間は、ユーザーの数、キャプチャするカスタム イベントまたは購入イベントの数、履歴を何日遡るかによって異なります。

最後に、拡張機能を作成したら、セグメントを作成するときやキャンペーンやキャンバスのオーディエンスを定義するときに、拡張機能をフィルターとして使用できます。まずは選択してください `Braze Segment Extension`**[ユーザー属性]** セクションのフィルター リストから選択します。Braze セグメント拡張フィルター リストから、このセグメントに含めるまたは除外する拡張を選択します。拡張機能の条件を表示するには、**「拡張機能の詳細を表示」を**クリックします。これで、通常どおりセグメントの作成を続行できます。

### 特定のキャンペーンまたはキャンバスのユニーク受信者数と送信数に差があるのはなぜですか?

{% apitags %}
キャンペーン
{% endapitags %}

この違いの考えられる理由の 1 つは、キャンペーンまたは Canvas で再資格が有効になっていることによる可能性があります。これをオンにすると、セグメントと配信設定の要件を満たすユーザーはメッセージを複数回受信できるようになります。再適格性がオンになっていない場合、送信数と一意の受信者数の違いの原因として考えられるのは、ユーザーがプロファイルに関連付けられたプラットフォーム間で複数のデバイスを持っていることです。

たとえば、iOS と Web の両方のプッシュ通知を備えた Canvas がある場合、モバイル デバイスとデスクトップ デバイスの両方を持つ特定のユーザーは複数のメッセージを受信する可能性があります。

### 現地時間帯配送では何が提供されますか?

{% apitags %}
キャンペーン
{% endapitags %}

ローカルタイムゾーン配信を使用すると、ユーザーの個々のタイムゾーンに基づいてセグメントにメッセージングキャンペーンを配信できます。ローカルタイムゾーン配信がない場合、キャンペーンは Braze の会社のタイムゾーン設定に基づいてスケジュールされます。

たとえば、ロンドンに拠点を置く企業が午後 12 時にキャンペーンを送信すると、アメリカ西海岸のユーザーには午前 4 時に届きます。アプリが特定の国でのみ利用可能である場合、これはリスクにはならない可能性がありますが、それ以外の場合は、ユーザーベースに早朝のプッシュ通知を送信することは避けることを強くお勧めします。

### Braze はどのようにしてユーザーのタイムゾーンを認識するのでしょうか?

{% apitags %}
キャンペーン
{% endapitags %}

### ローカルタイムゾーンキャンペーンをスケジュールするにはどうすればよいですか?

{% apitags %}
キャンペーン
{% endapitags %}

キャンペーンをスケジュールするときは、指定した時間に送信するように選択し、 **ローカルタイムゾーンでユーザーにキャンペーンを送信する**を選択する必要があります。

Braze では、すべてのローカル タイム ゾーン キャンペーンを 24 時間前にスケジュールすることを強くお勧めします。このようなキャンペーンは 1 日中送信する必要があるため、24 時間前にスケジュールを設定すると、メッセージがセグメント全体に届きます。ただし、必要に応じて、これらのキャンペーンを 24 時間未満前にスケジュールすることもできます。Braze は、送信時間を 1 時間以上過ぎたユーザーにはメッセージを送信しないことに注意してください。

たとえば、午後 1 時に、ローカル タイム ゾーン キャンペーンを午後 3 時にスケジュールした場合、キャンペーンはローカル時間が午後 3 時から午後 4 時までのすべてのユーザーに直ちに送信されますが、ローカル時間が午後 5 時までのユーザーには送信されません。さらに、キャンペーンに選択した送信時間は、会社のタイムゾーンではまだ発生していない必要があります。

24 時間以内にスケジュールされたローカル タイム ゾーン キャンペーンを編集しても、メッセージのスケジュールは変更されません。ローカル タイム ゾーン キャンペーンを編集して、後で送信するように設定した場合 (たとえば、午後 6 時ではなく午後 7 時)、元の送信時間が選択されたときに対象セグメントにいたユーザーは、元の時間 (午後 6 時) にメッセージを受信します。ローカル タイム ゾーンを編集して、より早い時間 (たとえば、午後 5 時ではなく午後 4 時) に送信する場合でも、キャンペーンは元の時間 (午後 5 時) にすべてのセグメント メンバーに送信されます。

{% alert note %}
Canvas ステップの場合、ローカル タイム ゾーン配信の次のステップを受信するために、ユーザーは 24 時間そのステップにいる必要はありません。
{% endalert %}

ユーザーがキャンペーンに再度参加できるように設定している場合は、元の時間 (午後 5 時) に再度キャンペーンを受け取ります。ただし、その後のキャンペーンでは、メッセージは更新された時間にのみ送信されます。

### Canvas ではユーザーコンバージョンはどのように追跡されますか?

{% apitags %}
キャンバス
{% endapitags %}

ユーザーは Canvas エントリごとに 1 回のみ変換できます。

コンバージョンは、そのエントリに対してユーザーが受信した最新のメッセージに割り当てられます。キャンバスの先頭にある概要ブロックには、メッセージを受信したかどうかに関係なく、そのパス内でユーザーが実行したすべてのコンバージョンが反映されます。後続の各ステップでは、ユーザーが最後に受けたステップの間に発生したコンバージョンのみが表示されます。

{% details Examples %}

#### 例:

10 個のプッシュ通知を含むキャンバス パスがあり、コンバージョン イベントは「セッション開始」(「アプリを開く」) です。

- ユーザー A は、入力後、最初のメッセージを受信する前にアプリを開きます。
- ユーザー B は、プッシュ通知のたびにアプリを開きます。

**導入効果:**
概要には 2 つの変換が表示されますが、個々のステップでは最初のステップで 1 つの変換が表示され、後続のすべてのステップで 0 の変換が表示されます。

{% alert note %}
コンバージョン イベントが発生したときに Quiet Hours がアクティブな場合は、同じルールが適用されます。
{% endalert %}

#### 例:

静かな時間を備えたワンステップのキャンバスがあります:

1. ユーザーがキャンバスに入ります。
2. 最初のステップには遅延はありませんが、Quiet Hours 内なので、メッセージは抑制されます。
3. ユーザーがコンバージョン イベントを実行します。

**導入効果:**
ユーザーは、Canvas バリアント全体ではコンバージョン済みとしてカウントされますが、ステップを受信して​​いないため、ステップとしてはカウントされません。

{% enddetails %}

### エンゲージメント レポートとレポート ビルダーの違いは何ですか?

{% apitags %}
分析
{% endapitags %}

エンゲージメント レポートでは、トリガーされた電子メールを介して、キャンペーンやキャンバスからの特定のメッセージのエンゲージメント統計の CSV が提供されます。特定のデータは、個々のバリアントまたはステップ レベルではなく、キャンペーンまたはキャンバス レベルで集計されます。レポートはダッシュボードに保存されないため、レポートを再実行すると統計が更新される可能性があります。

レポート ビルダーを使用すると、複数のキャンペーンまたはキャンバスの結果を 1 つのビューで比較できるため、どのエンゲージメント戦略が主要な指標に最も影響を与えたかを簡単に判断できます。キャンペーンとキャンバスの両方で、データをエクスポートし、レポートを保存して後で表示することができます。

Braze でのレポートと分析の使用に関する詳細については、 [レポートの概要]({{site.baseurl}}/user_guide/data_and_analytics/reporting/reports_overview/)を参照してください。
