# プロモーションコードを使う

> プロモーションコードの使用方法を学習し、キャンペーンやキャンバスの使用状況を確認する。

## 前提条件

プロモーションコードを使用する前に、[プロモーションコードリストを作成]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/promotion_codes/create/)する必要がある。

## プロモーションコードを使う

プロモーションコードをメッセージで送信するには、[以前に作成した]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/promotion_codes/create/#create)プロモーションコードリストの横にある**「スニペットをコピー」**を選択する。

![メッセージングに貼り付けるためにスニペットをコピーするオプションだ。]({% image_buster /assets/img/promocodes/promocode9.png %}){: style="max-width:70%"}

コードスニペットをBrazeのメッセージの一つに貼り付け、リストにある固有のプロモーションコードを[Liquid]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/)を使って挿入する。そのコードは送信済みとしてマークされている。これにより、他のメッセージが同じコードを送信することは保証されない。

![例文「この春は特別なオファーで自分へのご褒美を」の後にコードスニペットが続く。]({% image_buster /assets/img/promocodes/promocode10.png %}){: style="max-width:70%"}

### キャンバスを横切ってステップ

コードスニペットがマルチチャネルメッセージングを含むキャンペーンやキャンバスで使用されると、各ユーザーは固有のコードを受け取る。複数のステップからなるキャンバスで、各ステップがプロモーションコードを参照する場合、ユーザーは各ステップに進むたびに新しいコードを受け取る。

キャンバス内で1つのプロモーションコードを割り当て、複数のステップで再利用するには：

1. プロモーションコードをカスタム属性として、最初のステップ（ユーザー更新）で割り当てる。
2. 後続のステップでは、新しいコードを生成する代わりに、そのカスタム属性を参照するためにLiquidを使用する。

ユーザーが複数のチャネルでコードの資格を得た場合、各チャネルで同じコードを受け取る。例えば、メールとプッシュ通知でメッセージを受け取る場合、同じコードが両方に送信される。レポートもまた単一のコードを反映している。

{% alert note %}
プロモーションコードが利用できない場合、コードに依存するテストメッセージや本番メッセージは送信されない。
{% endalert %}

### アプリ内メッセージキャンペーン {#promotion-codes-iam-campaigns}

[アプリ内メッセージキャンペーン]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages)を作成した後、[プロモーションコードリストスニペット]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/promotion_codes/manage/#using-promotion-codes-1)をアプリ内メッセージボディに挿入できます。アプリ内メッセージ内のプロモーションコードは、ユーザーがアプリ内メッセージの表示をトリガーした場合にのみ、差し引かれ使用される。

### テストメッセージ

テスト送信とシードグループメール送信は、特に指定がない限りプロモーションコードを消費する。Brazeのアカウントマネージャーに連絡して、この機能の動作を更新し、テスト送信やシードグループのメール送信時にプロモーションコードが使用されないようにする。

## プロモーションコードをユーザープロファイルに保存する {#save-to-profile}

後続のメッセージで同じプロモーションコードを参照するには、コードをカスタム属性としてユーザープロファイルに保存する必要があります。これは、メッセージステップの直前に配置した[ユーザー更新ステップ]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/user_update/)で、割引コードを「プロモーションコード」のようなカスタム属性に割り当てることで実現できる。

まず、ユーザー更新ステップの各フィールドで以下を選択する：

- **属性名**プロモーションコード
- **アクション:**更新
- **キーの値:**プロモーションコードのLiquidコードスニペット、例えば {% raw %}`{% promotion('spring25') %}`{% endraw %}

次に、メッセージにカスタム属性（この例では）を追加{% raw %}`{{custom_attribute.${Promo Code}}}`{% endraw %}する。割引コードはテンプレート化されている。

## プロモーションコードの使用状況を確認する

残りのコード数は、**プロモーションコード**ページのプロモーションコード一覧の**残り**列にあります。

![未使用のコードの例だ。]({% image_buster /assets/img/promocodes/promocode11.png %})

このコード数は、既存のプロモーションコード一覧ページを再訪した際にも確認できる。未使用のコードをCSV ファイルとしてエクスポートすることもできます。 

![「ブラックフライデーセール」というプロモーションコードが残り992個ある。]({% image_buster /assets/img/promocodes/promocode12.png %}){: style="max-width:70%"}

## マルチチャネルとシングルチャネルの送信

マルチチャネル、シングル送信キャンペーン、キャンバスでは、メッセージのリキッドで参照されるすべてのプロモーションコードは、メッセージが送信される**前に**使用されるように差し引かれ、次のことが起こることを確認する：

- マルチチャネル・メッセージでは、同じプロモーション・コードが複数のチャネルで使用される。
- 余分なプロモーションコードは、メッセージが失敗したり中断した場合には使用されない。

ユーザーが1つのメッセージ内でLiquid条件分岐タグによって分割された2つのプロモーションコードリストを参照している場合、ユーザーがどちらの条件分岐フローをたどっても、すべてのプロモーションコードは依然として適用される。

ユーザーが新しいキャンバスステップに入ったり、キャンバスに再入したりした場合、そのユーザーへのメッセージに対してプロモーションコードのLiquidスニペットが再度適用されると、新しいプロモーションコードが使用される。

### 例

次の例では、プロモーションコードリスト`vip-deal`と`regular-deal`プロモーションコードリストの両方が差し引かれる。Liquid は以下のとおりです。

{% raw %}
```
{% if user.is_vip %}
  {% promotion('vip-deal') %}
{% else %}
  {% promotion('regular-deal') %}
{% endif %} 
```
{% endraw %}

Brazeは、使用予定数よりも多くのプロモーションコードをアップロードすることを推奨している。プロモーションコードリストが期限切れになったり、プロモーションコードがなくなったりすると、その後のメッセージは中止される。

{% alert tip %}
**Brazeでプロモーションコードがどのように使われるかを例えてみよう。**<br><br>メッセージを送ることは、郵便局で手紙を送るようなものだと想像してほしい。その手紙を店員に渡すと、クーポン券が同封されていることがわかる。店員は1枚目のクーポンを束から取り出し、封筒に入れる。店員は手紙を送ったが、何らかの理由で手紙は郵便物の中に紛れ込んでしまった（クーポンも紛失してしまった）。<br><br>このシナリオでは、Brazeは郵便局員であり、プロモーションコードはクーポンである。プロモーションコードのスタックから引き出された後は、Webhookの結果に関わらず、それを取り戻すことはできない。
{% endalert %}
