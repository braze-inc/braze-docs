# エクスポートに関するトラブルシューティング

> このページでは、CSV ワークフローとAPI ワークフローの両方でエクスポートの一般的なトラブルシューティングシナリオについて説明します。  

タブを使用して、** デフォルト Braze S3 バケット** にエクスポートするか、** クラウドストレージパートナー** にエクスポートするかを選択します。

{% sdktab Default export %}

デフォルト エクスポート送信先としてマークされたストレージパートナがない場合、Braze は独自のAmazon S3 バケットを使用してエクスポートファイルを保持します。この設定のファイルは一時的なもので、4 時間後に期限切れになります。  

## CSV輸出  
ダッシュボードからCSV をエクスポートすると、Braze メール はログインユーザーへのダウン読み込むを表示します。このリンクは、BrazeのS3バケットでホストされているZIPファイルを指します。ZIP 内には、エクスポートを構成する複数の小さいファイルがあります。  

リンクを使用するには、Braze ダッシュボードにログインする必要があります。また、このファイルは4 時間しか使用できません。その後、リンクは機能しなくなり、データは削除されます。非常に大規模なエクスポート(500,000 ユーザー以上)で失敗を繰り返すと、エクスポートが失敗する可能性があります。このような場合は、エクスポートをより小さなグループまたはフィールドs に分割するか、ストレージパートナの設定を検討してください。  

### 一般的なエラー

- `AccessDenied` エラーが表示された場合、ファイルの有効期限がすでに切れているか、準備ができていない状態で開封しようとしている可能性があります。レポートが大きいほど、生成に時間がかかるため、数分待ってから再試行してください。  
- `ExpiredToken`エラーは、4時間のウィンドウが経過したことを意味します。エクスポートを再実行して、新しいリンクを生成します。  
- 通常、`Looks like the file doesn't exist anymore` は、メールが送信されてもS3 への送信アップロードが完了していない場合にアプリされます。数分待つと、通常は解決します。  
- 特定のフィールドs の先頭に追加されるアポストロフィー(`-`、`=`、`+`、または`@` など) が期待されます。たとえば、`-1943` はCSV で`'-1943` になります。Braze は、スプレッドシートプログラムが誤って解釈されないようにします。これは、JSON エクスポート （[`/users/export/segment`エンドポイント]({{site.baseurl}}/api/endpoints/export/user_data/post_users_segment/)によって返されるものなど) には適用されません。  

## API輸出  
クラウドストレージを使用せずにAPI のエクスポートを行うと、Braze はファイルをそのS3 バケットに書き込みます。メールは受信されません。代わりに、API レスポンスに一時的なダウン読み込むURL が含まれています。エクスポートは、1 行に1 つのユーザーを持つ複数のJSON ファイルを含むZIP として提供されます。  

CSV エクスポートと同様に、API からのリンクは4 時間後に失効します。リンクを早すぎると、ファイルの準備ができていないためにエラーが表示されることがあります。リクエストで`callback_endpoint` を指定すると、ファイルが利用可能になったときにBraze から通知されます。  

大規模なAPI エクスポートもタイムアウトすることがあります。このアプリが表示される場合は、小規模なリクエストを作成するか、ストレージパートナを接続してボリュームを処理してください。  

### 一般的なエラー  
- `AccessDenied` または`ExpiredToken` は、通常、リンクの期限が切れたか、まだ準備ができていないことを意味します。エクスポートを再度実行するか、もう少し待機します。

ストレージパートナ(Amazon S3、Google Cloud Storage、またはAzure Blob など) を接続し、ダッシュボードの**Technology Partners** ページからデフォルトエクスポート送信先としてマークすると、Braze はエクスポートをバケットに直接書き込みます。この設定は、通常、エクスポートが大きいほど信頼性が高くなります。  

## CSV輸出  
CSV エクスポートでは、Braze メール はダウン読み込むリンクを表示します。このリンクは、短い時間(通常は約4 時間)後に期限切れになります。ストレージパートナが接続され、デフォルトのエクスポート送信先としてマークされている場合、Braze は接続されているバケットにエクスポートのコピーも配信します。このコピーは、期限切れとリテンションがストレージポリシーに従う独自のインフラにあります。  

クラウドストレージでは、CSV エクスポートはZIP ファイルにバンドルされます。郵便番号の中には、より小さなCSVファイルが複数存在する。大規模なエクスポートは、多くの場合、チャンクに分割され(たとえば、それぞれ約5000 ユーザー s)、チャンクサイズは異なる場合があります。ファイルが小さい場合は、欠落しているデータは示されません。メールされたリンクが失敗したが、ストレージ内のコピーが成功した場合、バケットからデータを直接取得することができます。  

### 一般的なエラー

- `AccessDenied` Brazeがバケットに書き込めなかったことを意味します。認証情報と権限がまだ有効であることをダブルチェックします。  
- `ExpiredToken` Brazeがバケットにアクセスできなくなった場合は、イヤーをアプリします。Braze ダッシュボードで認証情報をアップデートします。  
- ファイルによっては、予想よりも小さく見える場合がありますが、これは通常の動作です。エクスポートプロセスでは、安定性のために意図的にファイルが分割されます。  
- 特定のフィールドs の先頭に追加されるアポストロフィー(`-`、`=`、`+`、または`@` など) が期待されます。たとえば、`-1943` はCSV で`'-1943` になります。Braze は、スプレッドシートプログラムが誤って解釈されないようにします。これは、JSON エクスポート （[`/users/export/segment`エンドポイント]({{site.baseurl}}/api/endpoints/export/user_data/post_users_segment/)によって返されるものなど) には適用されません。  

## API輸出  
ストレージパートナーが接続されているAPI を介してデータをエクスポートすると、エクスポートファイルがバケットに書き込まれます。メールは送信されません。下の読み込む URL Brazeのリターンはまだ時間制限されている可能性がありますが、基礎となるオブジェクトはストレージ内に存在し、リテンション 設定s に従います。各ZIP ファイルには、1 行に1 つのJSON オブジェクトが含まれています。大規模なエクスポートは、単一のZIP ではなく複数のZIP ファイルに分割される可能性があります。これにより、通常、この方法は大量のエクスポートに対して信頼性が高くなります。  

### 一般的なエラー

- `AccessDenied` h Brazeがバケットに書き込めない、またはオブジェクトが後で削除された場合にアプリします。権限を確認し、外部がファイルをクリアしていることを確認します。  
- `ExpiredToken` Brazeのバケットへのアクセス認証情報が古くなっていることを意味します。ダッシュボードで更新します。  
- ファイルが不足しているか、予想よりも小さい場合は、まずBraze以外のものがオブジェクトを削除していないことを確認します。ファイルサイズ自体が小さくなることが予想されます。
