# ログと監視データの同期

> クラウドデータ取り込み（CDI）**同期ログ**ダッシュボードでは、CDIによって処理された全データを監視し、データの同期が正常に行われたかどうかを確認し、「不正」または欠落したデータに関する問題を診断できる。

同期ログにアクセスするには、**データ設定**＞**クラウドデータ取り込み**に移動し、**同期ログ**タブを選択する。

## 同期ログダッシュボードの理解

メインの**同期ログ**ページでは、すべての同期実行の概要を俯瞰できる。これには、現在のステータスまたは最終ステータスに基づく最近の同期の概要も含まれる。

* **ランニング：**現在進行中の同期ジョブを実行する。  
* **成功:**完了した同期ジョブは、すべての行が正常に処理された。  
* **部分的な成功：**完了した同期ジョブのうち、1つ以上の行でエラーが発生した。  
* **エラー:**完了できなかった同期ジョブを実行する。  
* **制限を超えた：**データ制限を超えたために処理が停止した同期ジョブを再開する。

![合計6,576件の成功を記録した同期ログの例だ。]({% image_buster /assets/img/cloud_ingestion/sync_logs1.png %}){: style="max-width:80%"}

同期ログは、各同期について以下の詳細も提供する：

* **同期名：**同期設定の名前。  
* **実行ID:**同期の特定の実行に対する一意の識別子。このIDを選択すると、詳細が表示される。これは[CDIエンドポイント]({{site.baseurl}}/api/endpoints/cdi)でも使用できる。あるいはBrazeサポートによる同期実行を参照するためにも使える。   
* **ステータス:**実行のステータス（成功、部分的な成功、エラー、実行中）。  
* **ソースから読み込んだ新しい行：**この実行でデータウェアハウスから取得した新規行の数。  
* **結果:**実行中に成功した行数と失敗した行数の内訳だ。  
* **最終**この同期実行で処理された**"UPDATED_AT":**最新のレコードのタイムスタンプ。  
* **実行開始時刻：**同期ジョブが開始された時。  
* **実行時間：**同期ジョブが完了するまでに要した合計時間。

![同期ログの詳細。]({% image_buster /assets/img/cloud_ingestion/sync_logs3.png %}){: style="max-width:80%"}

### データリテンション

同期ログデータは、すべての行レベルのペイロードとエラー詳細を含め、最大**30日間**保持される。30日以上経過したログは自動的に削除される。

同期実行のメタデータ（処理された行数など）は、少なくとも12か月間保持される。

### 同期ログのフィルター

同期ログテーブルにフィルターをかけると、特定の実行を見つけることができる。利用可能なフィルターには以下が含まれる：

* **勤務開始日：**あらかじめ定義された範囲（「過去30日間」など）またはカスタムの日付範囲を選択する。  
* **ステータス:**1つ以上の同期ステータスでフィルターする（例えば、**エラー**と**部分的な成功**ステータスのみを表示する）。  
* **同期名：**特定の同期を名前で検索する。

特定の同期を調査するには、同期ログテーブルから該当する**実行ID**を選択する。**実行詳細**ページでは、同期の行単位の詳細なログを確認できる。

### 実行概要

このセクションでは、選択された実行の概要を示す。これには開始時刻、終了時刻、実行時間、およびソースから読み込まれた行の総数が含まれる。また、成功した行数とエラーが発生した行数のカウントを提供する。

### この実行で処理された行数

この表は同期処理中に処理されたデータについて行単位の可視性を提供し、個々のレコードを検証できるようにする。

* **検索:**実行結果の中から特定のユーザーを検索するには**、ユーザー ID で検索する**バーを使うことができる。  
* **入手可能な詳細：**   
  * **UPDATED_AT:** その特定の`UPDATED_AT`行のカラムからのタイムスタンプ。  
  * **ID:**レコードをBrazeユーザープロファイルに照合するために使用されるユーザー識別子（例`external_id`：`email`、または`alias_name`）。  
  * **ステータス:**その行の個別ステータス（**成功**または**エラー**）。  
  * **送信ペイロード：**データペイロードを表示するリンク。  
  * **エラー理由：**ステータスが**エラー**の場合、この列には行の同期が失敗した理由を説明するメッセージが表示される。

#### ペイロードの閲覧

特定の行についてBrazeに送信された正確なデータを確認するには、**ソース**ペイロード列で**「ペイロードを表示」**を選択する。これは、そのユーザーに対して処理された生のJSONペイロードを表示する。

![同期ログ内の特定の行に対するペイロードの例。]({% image_buster /assets/img/cloud_ingestion/sync_logs2.png %}){: style="max-width:80%"}

#### 同期ログのエクスポート

**行をエクスポートする**を選択すると、同期実行の行レベルログをエクスポートできる。次に、エクスポート方法を選択する：

* **エラーのある行：**エラーステータスの行のみを含むファイルをダウンロードする。
* **すべての行：**実行中に処理されたすべての行を含むファイルをダウンロードする。

ログはダッシュボードから直接エクスポートできない。エクスポートが生成された後、ログエクスポートファイルをダウンロードするためのリンクが記載されたメールが届く。 

## 通知

CDI同期のステータスについて最新情報を得るために、メール通知を設定できる。これらの設定は同期を作成する際に設定され、いつでも更新できる。

### エラー通知

同期レベルのエラー通知を受け取るには、少なくとも1つのメール連絡先が必要だ。これらのアラートは、同期ジョブ全体が実行または完了に失敗した場合、または同期中にユーザー介入が必要なエラーが発生した場合に送信される。例えば、認証情報の期限切れやソーステーブルの欠落などが該当する。

追加の通知には以下が含まれる：

- **行エラー：**同期処理において、一定割合の行の更新に失敗した場合に通知を受け取る。
- **失敗閾値（%）：**アラートをトリガーすべき行の失敗率を指定する。例えば、これを**1**に設定すると、同期実行でエラーが発生した行が1%以上ある場合に通知が送信される。
- **同期成功：**同期が正常に完了した際に通知を受け取る。
- **行が変更されなくても警告する：**同期処理が正常に完了し、新規または更新された行がゼロ件であっても通知を受け取る。
