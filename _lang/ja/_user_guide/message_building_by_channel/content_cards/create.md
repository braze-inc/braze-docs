# コンテンツカードを作成する

> この記事では、キャンペーンとキャンバスを作成するときに、Braze でコンテンツカードを作成する方法について説明します。ここでは、メッセージタイプの選択、カードの作成、メッセージ配信のスケジュールについて説明します。

## ステップ 1:メッセージを作成する場所を選択する

キャンペーンは、単一のシンプルなメッセージング（例えば、一つのメッセージで製品についてユーザーに知らせるなど）に使います。キャンバスは、複数のステップからなるユーザージャーニー（例えば、ユーザーの行動履歴に基づいてカスタマイズされた商品提案を送信するといった場合）に使います。

{% tabs %}
{% tab Campaign %}

1. [**メッセージング**] > [**キャンペーン**] の順に進み、[**キャンペーンを作成**] を選択します。
2. [**コンテンツカード**] を選択するか、複数のチャネルをターゲットとするキャンペーンの場合は [**マルチチャネル**] を選択します。
3. キャンペーンに、明確で意味のある名前を付けます。
4. 必要に応じて[チーム]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/teams/)や[タグ]({{site.baseurl}}/user_guide/administrative/app_settings/tags/)を追加します。
   * タグを使用すると、キャンペーンを検索してレポートを作成しやすくなります。たとえば、[Report Builder]({{site.baseurl}}/user_guide/analytics/reporting/report_builder/) を使用する場合、関連するタグでフィルタリングできます。
5. キャンペーンに必要な数のバリアントを追加し、名前を付けます。追加したバリアントごとに、さまざまなプラットフォーム、メッセージタイプ、レイアウトを選択できます。バリアントの詳細については、[多変量テストおよび AB テスト]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/)を参照してください。

{% alert tip %}
キャンペーン内のすべてのメッセージが類似しているか、同じ内容になる場合は、メッセージを作成してからバリアントを追加してください。その後、[**バリアントを追加**] ドロップダウンから [**バリアントからコピー**] を選択できます。
{% endalert %}

{% endtab %}
{% tab Canvas %}

1. キャンバス作成ツールを使用して[キャンバスを作成]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas/)します。
2. キャンバスを設定したら、キャンバスビルダーでメッセージステップを追加します。ステップに、明確で意味のある名前を付けます。
3. メッセージングチャネルとして [**コンテンツカード**] を選択します。
4. コンテンツカードについて、Braze でオーディエンスの適格性とパーソナライゼーションを計算するタイミングを選択します。これは、ステップへのエントリ時、または最初のインプレッション発生時（推奨）に設定できます。コンテンツカードを含むステップは、スケジュール型またはアクション型に設定できます。
5. ユーザーが購入を完了したとき、またはカスタムイベントを実行したときに、コンテンツカードを削除するかどうかを選択します。
6. コンテンツカードの有効期限（フィード内の表示期間）を設定します。これは一定期間後、または特定の時刻に設定できます。
7. 必要に応じて、[**配信設定**] でこのステップのオーディエンスまたは受信者にフィルターを適用します。セグメントを指定し、フィルターを追加して、オーディエンスをさらに絞り込むことができます。オーディエンスのオプションは、遅延後のメッセージ送信時にチェックされます。
8. このメッセージと組み合わせる他のメッセージングチャネルを選択します。

{% endtab %}
{% endtabs %}

## ステップ 3:コンテンツカードを作成する

メッセージエディターの [**作成**] タブで、メッセージの内容と動作のあらゆる面を編集できます。

![メッセージエディターの [作成] タブにあるサンプルコンテンツカードの詳細。]({% image_buster /assets/img/content_card_compose.png %})

ここでの内容は、前のステップで選択した**カードタイプ**によって異なりますが、以下のオプションのいずれかを含むことができます。

#### 言語

[**言語を追加**] を選択して、表示されたリストから目的の言語を追加します。これで、[Liquid]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/conditional_logic/#conditional-logic) がメッセージに挿入されます。コンテンツを記述する前に言語を選択することをお勧めします。これにより、Liquid 内の適切な場所にテキストを入力できます。使用可能な言語の完全なリストについては、[サポートされている言語]({{site.baseurl}}/developer_guide/platform_integration_guides/android/advanced_use_cases/localization/#languages-supported)を参照してください。

![言語として英語、スペイン語、フランス語が選択されており、多言語対応するフィールドとしてタイトル、説明、リンクテキストが選択されているウィンドウ。]({% image_buster /assets/img/add_languages.png %}){: style="max-width:70%;"}

##### 右から左へのメッセージを作成する

右から左へのメッセージの最終的な表示は、サービスプロバイダーがそれらをどのようにレンダリングするかに大きく依存します。右から左へのメッセージを可能な限り正確に表示するためのベストプラクティスについては、[右から左へのメッセージを作成する]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/localization/right_to_left_messages/)を参照してください。

#### タイトルとメッセージ

何でも好きなことを書けます。制限はありませんが、メッセージを素早く伝え、顧客にクリックしてもらえるほど効果的です。明確で簡潔なタイトルとメッセージ内容を推奨します。これらのフィールドは画像のみのカードには提供されていないことに注意してください。

#### 画像

[**画像を追加**] を選択するか、画像 URL を指定して、コンテンツカードに画像を追加します。[**画像を追加**] を選択すると、**メディアライブラリー**が開き、以前にアップロードした画像を選択したり、新しい画像を追加したりできます。各メッセージタイプとプラットフォームには、独自の推奨比率と要件がある場合があるため、それらを確認してから画像を準備してください。コンテンツカードメッセージフィールドの合計サイズは 2&nbsp;KB に制限されていることに注意してください。

{% multi_lang_include alerts/important_alerts.md alert='dynamic image URL' %}

#### 先頭にピン留め

Braze はユーザーのフィード上部にピン留めされたカードを表示し、ユーザーはそれを閉じることができません。ユーザーのフィードに複数のピン留めカードがある場合、Braze はそれらを時系列順に並べます。カードを送信した後は、ピン留めオプションを後から更新することはできません。キャンペーン送信後にこのオプションを変更しても、影響があるのは今後の送信のみです。

![Braze のモバイルと Web のコンテンツカードプレビューを、「このカードをフィードの先頭にピン留めする」オプションを選択した状態で並べた図。]({% image_buster /assets/img/cc_pin_to_top.png %}){:style="border:none"}

#### オンクリック動作

カードに表示されたリンクを顧客がクリックすると、そのリンクはアプリの奥深くに誘導したり、別のサイトに誘導したりできます。コンテンツカードのオンクリック動作を選択した場合は、それに応じて**リンクテキスト**を更新することを忘れないでください。

コンテンツカードのリンクには、次のアクションを使用できます。

| アクション | 説明 |
|---|---|
| Web URL にリダイレクトする | ネイティブでない Web ページを開きます。 |
| [アプリにディープリンクする]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/deep_linking_to_in-app_content/#deep-linking-to-in-app-content) | アプリの既存画面にディープリンクします。 |
| カスタムイベントを記録する | トリガーする[カスタムイベント]({{site.baseurl}}/user_guide/data/custom_data/custom_events/)を選択します。別のコンテンツカードを表示したり、追加メッセージのトリガーとして使用できます。 |
| カスタム属性を記録する | 現在のユーザーに設定する[カスタム属性]({{site.baseurl}}/user_guide/data/custom_data/custom_attributes/)を選択します。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

[**カスタムイベントを記録する**] オプションと [**カスタム属性を記録する**] オプションでは、以下の SDK バージョンとの互換性が必要です。

{% sdk_min_versions swift:5.4.0 android:21.0.0 web:4.0.3 %}

## ステップ 4:追加の設定を行う（オプション）

[キーと値のペア]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/key_value_pairs/)を使用して、カードのカテゴリを作成し、[複数のコンテンツカードフィード]({{site.baseurl}}/developer_guide/customization_guides/content_cards/customizing_feed/#multiple-feeds)を作成し、カードの並べ替え方法をカスタマイズできます。

キーと値のペアをメッセージに追加するには、[**設定**] タブに移動し、[**新しいペアを追加**] を選択します。

## ステップ 5:キャンペーンまたはキャンバスの残りの部分を作成する

{% tabs %}
{% tab Campaign %}

キャンペーンの残りの部分を作成します。コンテンツカードを構築するためにツールを最適に使用する方法の詳細については、次のセクションに進んでください。

#### 配信スケジュールまたはトリガーを選択する

コンテンツカードは、スケジュールされた時刻、アクション、または API トリガーに基づいて配信できます。詳細については、[キャンペーンのスケジューリング]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/)を参照してください。

キャンペーンの継続時間と[サイレント時間]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/time_based_campaign/#quiet-hours)を設定し、コンテンツカードの有効期限を決定することもできます。具体的な有効期限日、またはカードが期限切れになるまでの日数（最大 30 日）を設定します。すべてのバリアントの有効期限は同じです。

一定期間後にカードを期限切れにする場合（例えば 2 週間後）、有効期限はカードの送信時刻から計算されます。スケジュール配信のキャンペーンでは、スケジュールされた開始時刻が基準となります。アクションベースのキャンペーンでは、ユーザーがトリガーアクションを実行した時刻が基準となります。例えば、アクションベースのカードが本日午後 2 時に送信され、有効期限が 1 日の場合、翌日の午後 2 時に期限切れとなります。

{% multi_lang_include alerts/note_alerts.md alert='Content Cards frequency capping' %}

アクションベースの配信では、コンテンツカードが表示されるまでに短い遅延が予想されます。例えば、キャンペーンがセッション開始でトリガーされる場合、このトリガーイベントはまず Braze のサーバーにフラッシュされる必要があります。次に、ユーザーのキャンペーン適格性が記録されます。SDK が同期すると、カードが作成され、同じ同期レスポンスで返されます。SDK の同期がユーザーの適格性の記録前に行われた場合、ユーザーはカードを受け取りません。初回セッションのユーザーの場合、この遅延は避けられません。既存のユーザーで即時の利用可能性が必要な場合は、スケジュール配信の使用を検討してください。

##### スケジュール配信

スケジュール配信のコンテンツカードキャンペーンでは、カードの作成タイミングを指定することで、Braze が新しいコンテンツカードキャンペーンのオーディエンス適格性とパーソナライゼーションを評価するタイミングを選択できます。詳細については、[カードの作成]({{site.baseurl}}/card_creation)を参照してください。

#### ターゲットとするユーザーを選択する

次に、セグメントやフィルターを選択して[対象ユーザーを]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/targeting_users/)絞り込み、オーディエンスを限定します。そのおおよそのセグメント人口がどのようになるか、自動的にプレビューが表示されます。正確なセグメントの所属は、メッセージが送信される前に常に計算されることに留意してください。

{% multi_lang_include target_audiences.md %}

#### コンバージョンイベントを選択する

Braze では、キャンペーンを受信した後、ユーザーが指定のアクションや[コンバージョンイベント]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/conversion_events/)を実行する頻度を追跡できます。ユーザーが指定したアクションを実行した場合にコンバージョンがカウントされる期間は、最大 30 日間まで設定できます。

{% endtab %}

{% tab Canvas %}

キャンバスコンポーネントが完成していない場合は、残りのセクションを完成させてください。キャンバスの残りの部分の構築方法、[多変量テスト]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/)と[インテリジェントセレクション]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_selection/)の実装などの詳細については、キャンバスのドキュメントの[キャンバスの構築]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas/#step-3-build-your-canvas)ステップを参照してください。

{% endtab %}

## ステップ 6:レビューとデプロイ

キャンペーンまたはキャンバスの最後のビルドが完了したら、その詳細を確認し、[テスト]({{site.baseurl}}/user_guide/engagement_tools/campaigns/testing_and_more/sending_test_messages/)を実行し、準備ができたら送信します。

{% alert warning %}
コンテンツカードは、開始後に編集できません。新規ユーザーへの送信停止、およびユーザーのフィードからの削除のみが可能です。このシナリオへのアプローチ方法については、「[開始済みのカードの更新]({{site.baseurl}}/user_guide/message_building_by_channel/content_cards/create/#updating-launched-cards)」を参照してください。
{% endalert %}

次に、[コンテンツカードレポート]({{site.baseurl}}/user_guide/message_building_by_channel/content_cards/reporting/)を参照して、コンテンツカードキャンペーンの結果にアクセスする方法を確認してください。

## 知っておくべきこと

### ペイロードとフィードの制限

パフォーマンスを支えるため、コンテンツカードには2つの主要な制約があります。各カードのペイロードサイズの上限と、フィードに表示できるカードの最大数です。

#### コンテンツカードのサイズ制限

単一のコンテンツカードの全データペイロードは、Liquid パーソナライゼーションのレンダリング**後**に 2 KB を超えてはなりません。これには以下が含まれます。

* タイトル
* メッセージ
* 画像 URL（URL 文字列自体の長さであり、画像ファイルのサイズではありません）
* リンクテキスト
* 指定された全プラットフォームのリンク URL（iOS、Android、Web の URL はそれぞれ別々にカウントされ、合計に加算されます）
* キーと値のペア（キー名とその値の両方）

カスタム属性などから長い文字列を取得するために Liquid を使うと、制限を超える可能性があります。 

キャンペーン作成ツールは、静的コンテンツが制限を超えた場合に警告を表示します。（Liquid を使用したダイナミックなコンテンツのサイズについては予測しません。）**メッセージサイズが 2 KB を超える場合、送信時に中断されます。**これらの中断は、メッセージアクティビティログで理由 `Content card maximum size exceeded` とともに確認できます。

{% alert important %}
テスト送信時、2 KB を超えるコンテンツカードも正常に配信・表示されることがあります。
{% endalert %}

コンテンツカードのペイロードサイズを管理するためのベストプラクティスを以下に示します。

* 長いリンクには URL 短縮サービスを使用してください。URL、特にトラッキングパラメータが大量に含まれるものは、サイズ制限の問題に直面することがあります。URL 短縮サービスを利用すれば、文字数を大幅に削減でき、ペイロードの空き容量を確保できます。
* Liquid でダイナミックなコンテンツを切り詰めてください。ユーザー属性や API 呼び出しからのダイナミックなテキストでカードをパーソナライズする場合、コンテンツの長さは予測できないことがあります。ダイナミックなテキストの長さを制限するために、`truncate` などの Liquid フィルターを積極的に活用してください。
* マルチプラットフォーム URL を効率的に扱ってください。2 KB の制限には、定義した全プラットフォームの URL が含まれます。各プラットフォームごとに長くてユニークな URL を使うと、ペイロードのサイズが膨れ上がります。可能であれば、全てのプラットフォームで機能する単一のリンクを使用するか、必要に応じて URL 短縮サービスを利用してください。
* より豊かなコンテンツにはバナーを検討してください。大量のコンテンツを常に必要とするユースケースでは、コンテンツカードは適切なチャネルではないかもしれません。バナーには 2 KB のペイロード制限がなく、アプリや Web サイトの体験に直接埋め込むリッチコンテンツに適しています。

#### フィードのカード枚数

各ユーザーのフィードには、有効期限が切れていないコンテンツカードを常時最大 250 枚まで含めることができます。この制限を超えると、Braze は未読のカードであっても、最も古いカードの返却を停止します。閉じたカードもこの制限にカウントされるため、閉じたカードが多いと、古いカードのための空きスペースが減ります。

カードの制限に関する問題を防止するため、以下のベストプラクティスを推奨します。

- **より短い有効期限を使用する：**時間制限のあるキャンペーン（週末セールなど）には、具体的な終了日を設定します。こうすることで、カードは自動的にフィードから削除され、関連性がなくなった後は制限数にカウントされなくなります。
- **アクションベースの削除を活用する：**トランザクションベースまたは目標ベースのカードに対して、削除イベントを設定します。例えば、プロファイルを完成させるようユーザーに促すカードは、`profile_completed` イベントが記録されたらすぐに削除すべきです。
- **長期にわたるキャンペーンを監査する：**定期的または継続的なキャンペーンを見直し、時間の経過とともにフィードに過剰なカードが蓄積されてユーザー体験を損なうことがないようにしてください。

### コンテンツカードの再適格性について

再適格性は、ユーザーが同じキャンペーンからメッセージを複数回受信できるかどうか、またそのタイミングを決定します。コンテンツカードについては、この仕組みを理解することが、繰り返しキャンペーンの管理や、ユーザーが重複したメッセージや古いメッセージを受け取ることの防止に不可欠です。

{% alert tip %}
コンテンツを 30 日以上持続させたいですか？[バナー]({{site.baseurl}}/user_guide/message_building_by_channel/banners)を試してみてください。
{% endalert %}

#### 再適格性の計算方法

再適格性を有効にすると、ユーザーがキャンペーンに「再参加」できるまでのカウントダウンは、メッセージが送信された後に開始されます。このカウントダウンが始まる具体的なタイミングは、カード作成時の設定によって異なります。

* [最初のインプレッション時]({{site.baseurl}}/user_guide/message_building_by_channel/content_cards/create/card_creation/#differences-between-creating-cards-at-launch-or-entry-versus-at-first-impression)に作成されるコンテンツカードは、インプレッション時間を用いて再適格性を計算します。
* キャンペーン開始時またはキャンバスステップのエントリ時に作成されたコンテンツカードは、送信時間とインプレッション時間のいずれか遅い方を使用します。

#### 30 日間の有効期限と再適格性

よくある混乱の原因は、キャンペーンの再適格性と、すべてのコンテンツカードが自動的に 30 日で期限切れになることとの相互関係です。 

すべてのコンテンツカードは、送信または削除されてから 30 日後に Braze のシステムから自動的に削除されます。長期にわたり繰り返し実施されるキャンペーンで再適格性を**無効**にしている場合でも、ユーザーは 30 日後に同じカードを再度受け取る可能性があります。元のカードが削除されると、システムはそのユーザーがキャンペーンを受信した記録を認識しなくなるため、次のセッションで再び適格となります。 

ユーザーが特定のキャンペーンからメッセージを一度だけ受け取るようにするには、キャンペーンまたはキャンバスステップにオーディエンスフィルターを追加して、当該キャンペーンからメッセージを受け取っていないユーザーを対象にしてください。このフィルターは、長期にわたるキャンペーンからの重複送信を防ぐ最も確実な方法です。

### ライブコンテンツカードの管理

コンテンツカードが送信されると、「受信トレイ」で待機し、ユーザーに配信される準備ができています（メールの場合と同様です）。コンテンツがコンテンツカードに取り込まれた後（表示時）、その有効期間中にコンテンツを変更することはできません。これは、コネクテッドコンテンツ経由で API を呼び出し、エンドポイントからのデータが変化している場合にも適用されます。このデータは更新されません。新規ユーザーへの送信停止、およびユーザーのフィードからの削除のみが可能です。キャンペーンを修正した場合、今後送信されるカードにのみ更新が適用されます。

#### 開始済みのカードの更新

既にカードを受け取ったユーザーに対してカードを変更するには、以下のいずれかの方法を使用する必要があります。

##### オプション 1:キャンペーンを複製する（即時変更に推奨）

{% alert tip %}
このオプションは、カードに最新のコンテンツを表示する場合、変更を即時反映させる必要がある場合、または再適格性が無効になっている場合に推奨します。
{% endalert %}

最初の方法は、キャンペーンをアーカイブし、複製した新しいキャンペーンを開始することです。

1. 元のキャンペーンを停止し、指示が出たら `Remove card after the next sync` を選択します。
2. キャンペーンを複製し、編集を加え、新しいバージョンを公開します。

キャンペーンを複製する際には、新しいバージョンのオーディエンスを定義する必要があります。セグメンテーションフィルターを使って、更新されたカードを受け取るユーザーをコントロールします。
* ユーザーにコンテンツカードの再適格性を付与しない場合、フィルター `Received Message from Campaign` を条件 `Has Not` に設定することで、コンテンツカードの前のバージョンを受信していないユーザーを抽出できます。
* 以前のカードを受信したユーザーが X 日後に再び適格になるようにする場合は、フィルター `Last Received Message from specific campaign` を X 日以上前に設定するか、**または**フィルター `Received Message from Campaign` に `Has Not` 条件を設定します。

###### 影響

* **既存の受信者：**新規および既存の受信者は、適格であれば次回のフィード更新時に更新済みのカードを表示できます。
* **レポート：**各バージョンのカードはそれぞれ別の分析結果を持ちます。

例えば、キャンペーンがセッション開始によってトリガーされるように設定しており、再適格性が 30 日に設定されているとします。あるユーザーが 2 日前にキャンペーンを受け取ったので、コピーを変更したいとします。まず、キャンペーンをアーカイブし、フィードからカードを削除します。次に、キャンペーンを複製して、新しいコピーで再び開始します。ユーザーが別のセッションを行うと、すぐに新しいカードを受け取ります。

##### オプション 2:同じキャンペーンを停止して再起動する

{% alert tip %}
このオプションは、通知センターまたはメッセージ受信トレイのユニークなメッセージ（プロモーションなど）で、分析データを統一することが重要な場合、またはメッセージの適時性が問題にならない場合（既存の受信者が更新されたカードを見る前に適格性の時間枠を待つことができるなど）に推奨します。
{% endalert %}

この方法では、すべての分析データを単一のキャンペーンに統合します。新たに適格となったユーザーは新しいカードを受け取りますが、既存の受信者については再び適格となるまで更新が遅れます。

1. キャンペーンを停止し、指示に従って [**次の同期後にカードを削除**] を選択します。
2. 必要に応じてキャンペーンを編集します。
3. キャンペーンを再開します。

###### 影響

* **既存の受信者：**すでに同カードを受け取っているユーザーは、再適格となるまで更新されたカードを受け取れません。再適格性がオフになっている場合、新しいカードを受信することはありません。
* **レポート：**1 つのキャンペーンに、開始されたカードの複数バージョンのすべてのレポート分析が含まれます。Braze は開始したバージョンを区別しません。

例えば、セッション開始によってトリガーされ、再適格性が 30 日に設定されたキャンペーンがあるとします。あるユーザーが 2 日前にキャンペーンを受け取ったので、コピーを変更したいとします。まず、キャンペーンを停止し、フィードからカードを削除します。次に、新しいコピーでキャンペーンを再公開します。ユーザーが別のセッションを行った場合、28 日後に新しいカードを受け取ります。

#### カードの削除と有効期限切れ

##### 手動でのカード削除

キャンペーンを停止すれば、いつでも全ユーザーのフィードからカードを手動で削除できます。

1. コンテンツカードキャンペーンを開き、[キャンペーンを停止] を選択します。
2. プロンプトが表示されたら、[**次回の同期後にカードを削除する**] を選択します。カードは次のフィード更新時に削除されます。

##### 自動カード削除 {#action-based-card-removal}

ユーザーが特定のアクション（購入の完了や機能の有効化など）を実行した際に、カードを自動的に削除できます。

キャンペーンまたはキャンバスステップで、削除イベントを指定します。ユーザーがそのイベントを実行すると、Braze がイベントを処理した後の次の更新時に、そのカードはフィードから削除されます。 

{% alert note %}
この削除は瞬時に行われるものではありません。処理に遅延が生じるため、カードが消えるまでに数分かかる場合があり、フィードの更新を複数回行う必要があるかもしれません。
{% endalert %}

{% alert tip %}
ユーザーのフィードからカードを削除するカスタムイベントや購入を複数指定できます。ユーザーがこれらのアクションの**いずれか**を実行すると、キャンペーンのカードによって送信された既存のカードがすべて削除されます。今後対象となるカードは、メッセージのスケジュールに従って送信され続けます。
{% endalert %}

![コンテンツカード削除条件パネルとコンテンツカード削除イベントオプション。]({% image_buster /assets/img/content_cards/content_card_removal_event.png %})

##### カードの有効期限

コンテンツカードは送信後最大 30 日間利用可能です。30 日が経過すると、Braze はユーザーフィードからそれらを削除し、Braze のシステムから完全に消去します。

#### カードを 30 日以上長持ちさせる

{% alert tip %}
コンテンツカードの 30 日という制限を超えてメッセージを長く表示する必要があるユースケースでは、バナーの使用を検討してください。バナーは持続性を考慮して設計されており、必須の有効期限は設定されていないため、必要とされる限り表示され続けることができます。
{% endalert %}

カードを常に利用可能に見せたい場合（つまり、30 日という上限を超えて長く表示させたい場合）、30 日ごとにカードを実質的に更新する定期的なキャンペーンを作成できます。

1. コンテンツカードの有効期間を 30 日に設定します。
2. キャンペーンの再適格性を 30 日に設定します。
3. キャンペーンが「セッション開始」でトリガーされるように設定します。
