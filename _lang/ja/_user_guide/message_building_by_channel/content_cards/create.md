# コンテンツカードの作成

> この記事では、Brazeでコンテンツカードを作成する方法について説明します。ここでは、メッセージの種類の選択、カードの作成、メッセージ配信のスケジュールについて説明します。

コンテンツカードは、Brazeプラットフォームでキャンペーンとキャンバスを使用して作成できます。

## ステップ 1:メッセージを作成する場所を選択する

メッセージをキャンペーンとキャンバスのどちらで送信すべきか迷っていますか?キャンペーンは単一のシンプルなメッセージングキャンペーンに適しており、キャンバスは複数ステップのユーザージャーニーに適しています。

{% tabs %}
{% tab Campaign %}

**ステップ:**

1. **「メッセージング**>**キャンペーン**」に移動し、「**キャンペーンを作成**」をクリックします<i class="fas fa-plus"></i>。
{% alert note %}
[古いナビゲーション]({{site.baseurl}}/navigation)を使用している場合は、[**エンゲージメント]** の下に **[キャンペーン**] があります。
{% endalert %}

{:start="2"}
2\.[ **コンテンツ カード**] を選択するか、複数のチャネルをターゲットとするキャンペーンの場合は **[マルチチャネル キャンペーン**] を選択します。
3\.キャンペーンには、明確で意味のある名前を付けます。
4\.必要に応じて、[チームと]({{site.baseurl}}/user_guide/administrative/manage_your_braze_users/teams/)[タグ]({{site.baseurl}}/user_guide/administrative/app_settings/manage_app_group/tags/)を追加します。
   \*タグを使用すると、キャンペーンを簡単に見つけてレポートを作成できます。たとえば、 [レポートビルダー]({{site.baseurl}}/user_guide/data_and_analytics/reporting/report_builder/)を使用する場合、特定のタグでフィルタリングできます。
5\.キャンペーンに必要な数のバリエーションを追加して名前を付けます。追加したバリアントごとに、異なるプラットフォーム、メッセージタイプ、レイアウトを選択できます。このトピックについて詳しくは、 [多変量分析と A/B テスト]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/)を参照してください。

{% alert tip %}
キャンペーン内のすべてのメッセージが類似しているか、同じ内容になる場合は、パターンを追加する前にメッセージを作成してください。次に、[**バリアントの追加**] ドロップダウンから [**バリアントからコピー**] を選択します。
{% endalert %}

{% endtab %}
{% tab Canvas %}

**ステップ:**

1. Canvas コンポーザーを使用して [Canvas を作成します]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas/)。
2. キャンバスを設定したら、キャンバスビルダーにメッセージステップを追加します。ステップに明確で意味のある名前を付けます。
3. メッセージングチャネルとして **コンテンツカード** を選択します。
4. Brazeがコンテンツカードのオーディエンスの適格性とパーソナライゼーションを計算するタイミングを選択します。これは、ステップ入力時または第一印象時(推奨)にすることができます。コンテンツ カードを含むステップは、スケジュールまたはアクションベースにすることができます。
5. ユーザーが購入を完了したとき、またはカスタムイベントを実行したときにコンテンツカードを削除するかどうかを選択します。
6. コンテンツカードの有効期限(フィードの時間)を設定します。これは、一定期間の後または特定の時間に行うことができます。
7. このステップのオーディエンスを、 **必要に応じて配信設定**でフィルタリングします。セグメントを指定し、フィルターを追加して、このステップの受信者をさらに絞り込みます。後から、メッセージの送信時に、オーディエンスオプションがチェックされます。
8. メッセージとペアリングする他のメッセージングチャネルを選択します。

{% endtab %}
{% endtabs %}

## ステップ 3:コンテンツ カードを作成する

メッセージの内容と動作のすべての側面は、メッセージエディターの [ **作成** ] タブで編集できます。

![メッセージエディターの「作成」タブのコンテンツカードの詳細の例][24]

ここでの内容は、前の手順で選択した **カードの種類** によって異なりますが、次のオプションのいずれかが含まれる場合があります。

#### 言語

「 **言語を追加** 」をクリックし、表示されたリストから目的の言語を選択します。これにより、メッセージに [Liquid]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/conditional_logic/#conditional-logic) が挿入されます。コンテンツを書く前に言語を選択し、Liquidのどこにテキストを入力できるようにすることをお勧めします。[利用可能な言語の完全なリスト][18]をご覧ください。

#### タイトルとメッセージ

何でも書いてください。制限はありませんが、メッセージを伝え、顧客のクリックを得るのが早ければ早いほど良いのです。明確で簡潔なタイトルとメッセージの内容をお勧めします。これらのフィールドは、画像のみのカードには提供されません。

#### 画像

コンテンツカードに画像を追加するには、[ **画像を追加** ]をクリックするか、画像のURLを入力します。「 **画像を追加** 」をクリックすると、 **メディアライブラリ**が開き、以前にアップロードした画像を選択したり、新しい画像を追加したりできます。各メッセージタイプとプラットフォームには、独自の推奨比率と要件がある場合がありますので、依頼したり、ゼロから画像を作成したりする前に、それらが何であるかを必ず確認してください。コンテンツ カード メッセージ フィールドの合計サイズは 2 KB に制限されています。

#### 先頭にピン留め

ピン留めされたカードはユーザーのフィードの上部に表示され、ユーザーが閉じることはできません。ユーザーのフィードに複数のカードがピン留めされている場合、ピン留めされたカードは時系列で表示されます。カードが送信された後は、ピン留めされたオプションをさかのぼって更新することはできません。キャンペーンの送信後にこのオプションを変更すると、今後の送信にのみ影響します。

![Braze for Mobile and Webのコンテンツカードプレビューを並べて表示し、「このカードをフィードの上部に固定する」オプションを選択します。[1]{:style="border:none"}

#### クリック時動作

カードに提示されたリンクをユーザーがクリックすると、そのリンクによって、ユーザーをアプリの奥深くに誘導したり、別のサイトに誘導したりできます。コンテンツカードにクリック時の動作を選択した場合は、それに応じて **リンクテキスト** を更新してください。

コンテンツ カード リンクに対しては、次のアクションを実行できます。

|アクション |説明 |
|---|---|
|Web URL へのリダイレクト |非ネイティブの Web ページを開きます。|
| [アプリへのディープリンク]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/deep_linking_to_in-app_content/#deep-linking-to-in-app-content) |アプリ内の既存の画面へのディープリンク。 |
|カスタムイベントのログ記録 |トリガーする [カスタムイベント]({{site.baseurl}}/user_guide/data_and_analytics/custom_data/custom_events/) を選択します。別のコンテンツカードを表示したり、追加のメッセージをトリガーしたりするために使用できます。|
|カスタム属性のログ記録 |現在のユーザーに設定する [カスタム属性]({{site.baseurl}}/user_guide/data_and_analytics/custom_data/custom_attributes/) を選択します。|
{: .reset-td-br-1 .reset-td-br-2}

注:[ __Log Custom Event__ ] オプションと __[Log Custom Attribute__ ] オプションには、次の SDK バージョンの互換性が必要です。

{% sdk_min_versions swift:5.4.0 android:21.0.0 web:4.0.3 %}

{% alert warning %}
コンテンツ カード メッセージ フィールドの合計サイズは 2 KB に制限されており、次のフィールドのバイト サイズの長さを加算して計算されます。タイトル、メッセージ、画像 URL、リンク テキスト、リンク URL、キーと値のペア (名前 + 値)。このサイズを超えるメッセージは送信されません。これには、画像のサイズではなく、画像 URL の長さが含まれることに注意してください。
{% endalert %}

## ステップ 4: 追加設定の構成 (オプション)

[キーと値のペア][19]を使用して、カードのカテゴリを作成したり、[複数のコンテンツ カード フィード]({{site.baseurl}}/developer_guide/customization_guides/content_cards/customizing_feed/#multiple-feeds)を作成したり、カードの並べ替え方法をカスタマイズしたりできます。

メッセージにキーと値のペアを追加するには、[ **設定** ] タブに切り替えて **[新しいペアを追加**] をクリックします。

## ステップ 5: キャンペーンまたはキャンバスの残りの部分を作成する

{% tabs %}
{% tab Campaign %}

キャンペーンの残りの部分を構築します。ツールを最大限に活用してコンテンツカードを作成する方法の詳細については、次のセクションを参照してください。

#### 配信スケジュールまたはトリガーを選択する

コンテンツ カードは、スケジュールされた時間、アクション、または API トリガーに基づいて配信できます。詳しくは、 [キャンペーンのスケジュール設定]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/)を参照してください。

また、キャンペーンの期間と [待機時間]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/time_based_campaign/#quiet-hours) を設定し、コンテンツカードの有効期限を決定することもできます。特定の有効期限またはカードの有効期限までの日数(最大30日)を設定します。すべてのバリエーションの有効期限は同じです。

{% alert note %}
フリークエンシーキャップは、コンテンツカードには適用されません。
{% endalert %}

##### スケジュールされた配信

配信がスケジュールされているコンテンツカードキャンペーンの場合、カードを作成するタイミングを指定することで、Brazeが新しいコンテンツカードキャンペーンのオーディエンスの適格性とパーソナライゼーションを評価するタイミングを選択できます。詳しくは、 [カード作成]({{site.baseurl}}/card_creation)を参照してください。

#### ターゲットとするユーザーを選択する

次に、セグメントまたはフィルターを選択して [ユーザーをターゲティング]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/targeting_users/) し、オーディエンスを絞り込む必要があります。そのおおよそのセグメント母集団が現在どのようになっているかのスナップショットが自動的に提供されます。正確なセグメントメンバーシップは、常にメッセージが送信される直前に計算されることに注意してください。

#### コンバージョン イベントを選択する

Brazeを使用すると、ユーザーがキャンペーンを受け取った後、特定のアクション( [コンバージョンイベント]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/conversion_events/))を実行する頻度を追跡できます。最大 30 日間の期間を指定して、ユーザーが指定したアクションを実行した場合にコンバージョンがカウントされるように設定することもできます。

{% endtab %}

## 知っておきたいこと

### 送信動作

コンテンツカードが送信されると、メールと同様に、ユーザーに配信される準備が整った「受信トレイ」で待機します。コンテンツがコンテンツ カードに取り込まれた後 (表示時)、コンテンツは有効期間中に変更できません。これは、接続コンテンツを介して API を呼び出し、エンドポイントからのデータが変更された場合でも適用されます。このデータは更新されません。新しいユーザーへの送信を停止し、ユーザーのフィードから削除することしかできません。キャンペーンを変更した場合、今後送信されるカードのみが更新されます。

古いカードを削除する必要がある場合は、キャンペーンを停止する必要があります。キャンペーンを停止するには、コンテンツ カード キャンペーンを開き **、[キャンペーンを停止]** を選択します。キャンペーンを停止すると、すでにカードを受け取っているユーザーへの対応方法を決定するよう求められます。ユーザーのフィードからコンテンツカードを削除する場合は、[ **フィードからカードを削除**] を選択します。カードは、次回の同期時に SDK によって非表示になります。

![コンテンツカードの無効化を確認するダイアログ][25]{: style="max-width:75%" }

{% alert tip %}
最大30日よりも長持ちするように見えるコンテンツカードが必要ですか?これを実現する方法の 1 つは、次のとおりです。

1. コンテンツ カードの期間を 30 日間に設定します。
2. キャンペーンの再適格性を 30 日間に設定します。
3. 「セッション開始」時にトリガーされるキャンペーンを設定します。

出来上がり！
{% endalert %}

### カード削除イベント {#action-based-card-removal}

一部のコンテンツカードは、ユーザーが何らかのアクションを実行するまでしか関連しません。たとえば、ユーザーにアカウントを有効にするように促すカードは、ユーザーがそのオンボーディング タスクを完了した後に表示しないでください。

キャンペーンまたはキャンバスメッセージ内に、オプションで **削除イベント** を追加して、以前に送信したカードをそのユーザーのフィードから削除するカスタムイベントまたは購入を指定できます。SDK または REST API を介してトリガーされます。

{% alert tip %}
複数のカスタムイベントと購入を指定して、ユーザーのフィードからカードを削除できます。これらのアクション **のいずれか** がユーザーによって実行されると、キャンペーンのカードによって送信された既存のカードはすべて削除されます。今後対象となるカードは、メッセージのスケジュールに従って引き続き送信されます。
{% endalert %}

![Content Card Removal Conditions panel with Content Card Removal Event option]({% image_buster /assets/img/content_cards/content_card_removal_event.png %})

### 起動したカードの更新

コンテンツカードは、送信後に編集することはできません。既に送信されたカードに変更を加える必要がある場合は、次の方法を検討してください。

#### オプション 1:キャンペーンの複製

1 つの方法は、キャンペーンをアーカイブし、フィードからアクティブなカードを削除することです。その後、キャンペーンを複製し、更新して開始し、対象となるユーザーが更新されたカードを受け取ることができます。

* ユーザーがコンテンツ カードに再適用されないようにするには、フィルター `Received Message from Campaign` を条件として `Has Not`に設定して、以前のバージョンのコンテンツ カードを受け取っていないユーザーをフィルター処理します。
* 以前のカードを受け取ったユーザーが X 日後に再適格になる場合は、条件を使用して、X `Last Received Message from specific campaign` 日以上前のフィルターを [ **or** `Received Message from Campaign` ] `Has Not` に設定します。

**例:**キャンペーンはセッション開始時にトリガーされ、再適格性は 30 日に設定されます。ユーザーが 2 日前にキャンペーンを受信し、コピーを変更したいと考えています。まず、キャンペーンをアーカイブし、フィードからカードを削除します。次に、キャンペーンを複製し、新しいコピーで再起動します。ユーザーが再度セッションを行うと、すぐに新しいカードを受け取ります。

##### インパクト

* **報告：**カードの各バージョンには、個別の分析があります。
* **既存の受信者:**新規および既存の受信者は、資格がある場合、次回のフィード更新時に更新されたカードが表示されます。

{% alert tip %}
このオプションは、カードに最新のコンテンツ(ホームページのバナーなど)を表示する場合、変更をすぐに表示する必要がある場合、または再適格性がオフになっている場合に推奨されます。
{% endalert %}

#### オプション 2:停止して再起動する

カードの再適格性が有効になっている場合は、次のことを選択できます。

1. キャンペーンを停止します。
2. ユーザーのフィードからアクティブなコンテンツカードを削除します。
3. 必要に応じてキャンペーンを編集します。
4. キャンペーンを再開します。

このアプローチでは、新たに対象となったユーザーは新しいカードを取得し、以前の受信者は再資格を得たときに新しいカードを取得します。

**例:**キャンペーンはセッション開始時にトリガーされ、再適格性は 30 日に設定されます。ユーザーが 2 日前にキャンペーンを受信し、コピーを変更する場合。まず、キャンペーンを停止し、フィードからカードを削除します。次に、新しいコピーでキャンペーンを再公開します。ユーザーが再度セッションを行うと、28 日以内に新しいカードが届きます。

##### インパクト

* **報告：**1 つのキャンペーンには、起動されたカード バージョンのすべてのレポート分析が含まれます。Brazeは、発売されたバージョンを区別しません。
* **既存の受信者:**すでにカードを受け取っているユーザーは、再適格になるまで更新されたカードを受け取りません。再資格がオフになっている場合、新しいカードを受け取ることはできません。

{% alert tip %}
このオプションは、通知センターまたはメッセージ受信トレイ内の一意のメッセージ (プロモーションなど) に推奨される場合、分析を統合することが重要な場合、またはメッセージの適時性が問題にならない場合 (既存の受信者が更新されたカードを表示する前に適格性ウィンドウを待機できるなど) に推奨されます。
{% endalert %}

#### ユーザーのフィードにカードを保持する

必要に応じて、アクティブなコンテンツカード キャンペーンをユーザーのフィードに残し、削除しないようにすることもできます。公開中のキャンペーンに編集が加えられると、キャンペーン カードの以前の未編集バージョンは引き続き公開され、編集後に条件を満たしたユーザーのみに新しいバージョンが表示されます。ただし、すでにキャンペーンに接触しているユーザーには、2 つのバージョンのカードが表示される場合があります。
