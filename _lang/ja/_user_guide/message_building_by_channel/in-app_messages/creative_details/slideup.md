# スライドアップアプリ内メッセージs

> 通常、スライドアップはアプリスクリーンの上部または下部に耳元をアプリします(これは、メールを作成するときに設定できます)。これらは、新しい利用規約、Cookie、および他の情報断片についてユーザーに警告するのに適しています。これらは目立たないものであり、ユーザー s がアプリとやり取りを続けながら、メッセージが表示されます。

\![2つのアプリ内メッセージは、1つは画面上部から、もう1つは下部から表示される。詳細は以下のセクションを参照のこと。]({% image_buster /assets/img/slideup-spec.png %}){: style="max-width: 40%; border: none;"}

## イメージとコピーの動作

スライドアップメッセージには、省略記号で切り詰める前に最大 3 行の文章を含めることができます。スライドアップの"画像は、切り抜かれたりクリップされたりすることはありません。50 x 50 ピクセルの"画像コンテナーに収まるように縮小されます。

- 画像はすべて、5 MB 未満でなければなりません。
- PNG、JPEG、GIFのファイルタイプのみ受け付けます。
- 500 KB の画像をお勧めします。

{% alert tip %} 自信を持ってアセットを創造しましょう！アプリ内メッセージ画像テンプレートとセーフゾーンオーバーレイは、あらゆるサイズのデバイスに対応するようデザインされている。[デザインテンプレートZIPをダウンロードする]({% image_buster /assets/download_file/Braze-In-App-Message-Design-Templates.zip %}) {% endalert %}

| 画像+テキスト | 1:1のアスペクト比<br>高解像度150 x 150 px<br> 50 x 50 px 以上 | さまざまなアスペクト比の画像が、トリミングなしで正方形の画像コンテナに収まる。 |

常にさまざまなデバイスで[メッセージをプレビューしてテスト]({{site.baseurl}}/user_guide/message_building_by_channel/in-app_messages/testing/)し、画像とメッセージの最も重要な領域が期待どおりに表示されることを確認する必要があります。作成画面でのメッセージのプレビューと、デバイスでの実際のレンダリングが異なる場合があることに注意してください。

## モバイル機器

モバイルデバイスでは、アプリ画面の上部または下部にスライドアップが表示されます。これは、メッセージを作成するときに指定できます。ユーザーは、スライドアップをスワイプして画面から消したり、クリックアクションが含まれている場合はタップして開封したりできます。クリックアクションがスライドアップに追加されると、山形">" が表示されます。
