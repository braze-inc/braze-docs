# 重複するメールアドレス

> 複数のプロファイルが同じメールアドレスを共有している場合、いずれかのプロファイルが配信停止すると、Brazeはそのメールアドレスを持つ他のプロファイル（最大100件）を同じサブスクリプション状態に更新する。これは配信停止やその他の変更、例えばグローバルなサブスクリプション状態や個々のサブスクリプショングループのステータスなどに適用される。

## メールサブスクリプションの更新

Brazeはメールキャンペーンを送信する際に、自動的に重複するメールアドレスをチェックし、削除する。これにより、複数のユーザープロファイルが同一のアドレスを共有している場合でも、Brazeがメールを複数回送信することを防ぐ。

{% alert tip %}
必ず、[ユーザーのメールサブスクリプションの管理]({{site.baseurl}}/user_guide/message_building_by_channel/email/managing_user_subscriptions/#managing-user-subscriptions)や、特定のサブスクリプション状態のユーザーをターゲットにしたキャンペーンの設定を行うために Braze が提供しているツールを十分に理解してください。これらのツールは、[反スパム法を]({{site.baseurl}}/help/best_practices/spam_regulations/#spam-regulations)遵守するために不可欠である。
{% endalert %}

ユーザーがメールアドレスを共有している場合、1つのプロファイルを更新すると、Brazeはそのサブスクリプション設定の変更を該当するユーザー全員（最大100人まで）に反映する。

## メッセージ送信の動作

ターゲットにした複数のユーザーが同じディスパッチに含まれている場合に重複が発生するため、キャンペーンやキャンバスがトリガーされると (API トリガーキャンペーンを除く)、同じメールアドレスを持つ異なるユーザーが異なる時刻にトリガーイベントを記録した場合、(再適格性によりユーザーが除外される可能性がある期間内であっても) 同じメールアドレスに複数回送信される可能性があります。

## 例

例えば、ユーザー A とユーザー B がメールアドレス `johndoe@example.com` を共有しているが、プロファイルが異なるタイムゾーンにある場合、キャンペーントリガーイベントにユーザーのタイムゾーンでの送信が含まれていると、メールアドレス `johndoe@example.com` は 2 通のメールを受信します。

ユーザー A のメールアドレスを、既存のユーザー B と共有する別のメールアドレスに設定するか、更新した場合、[**メールアドレスを更新したユーザーへの配信を再登録する**] 設定がオンになっていない限り、ユーザー A はすでに存在するユーザー B のサブスクリプション状態を継承します。

{% alert important %}
API 呼び出しで API キャンペーンを送信し (API トリガーキャンペーンを除く)、セグメントオーディエンスに同じメールアドレスを持つ複数のユーザーが指定されている場合、呼び出しにリストされた回数だけそのアドレスに送信されます。これは、APIコールが意図的に作られたものだと仮定しているからだ。
<br><br>
**API トリガーキャンペーン**<br>
API トリガーキャンペーンは、オーディエンスが定義された場所に応じて、重複排除または重複の送信を行います。<br>\- 対象セグメント内に重複メールがある場合、またはAPIトリガー呼び出しの[受信者フィールド]({{site.baseurl}}/api/endpoints/messaging/send_messages/post_send_triggered_campaigns/)内に重複IDがあるために重複メールがある場合、デデューピングが発生する可能性がある。<br>\- APIトリガー呼び出しの受信者フィールド内で別々のユーザーIDを直接ターゲットにした場合、メールの重複が発生する。
{% endalert %}
