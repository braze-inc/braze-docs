# よくある質問

> この記事では、電子メールに関するよくある質問に対する回答を提供します。

### メールが送信され、複数のプロファイルのメールアドレスが同じ場合はどうなりますか?

一致するメールを持つ複数のユーザがキャンペーンを受信するセグメント内のすべてのユーザである場合、そのメールアドレスを持つランダムなユーザプロファイルが送信時に選択されます。この方法では、メールは一度だけ送信され、重複排除されます。これにより、メールが同じメールアドレスに複数回ヒットしないようになります。

この重複排除は、ターゲットとなるユーザーが同じディスパッチに含まれている場合に発生することに注意してください。トリガーされたキャンペーンでは、一致するメールアドレスを持つユーザが異なる時間にトリガーイベントを記録する場合、同じメールアドレスに複数の送信が行われる可能性があります(再適用のためにユーザが除外される可能性がある期間内であっても)。キャンバスのエントリでは、ユーザは電子メールで除外されません。そのため、レート制限のあるエントリのために、ユーザが少し異なる時間に進行している場合、ユーザがキャンバスの最初のステップを超えて除外されない可能性があります。特定のメールアドレスに関連付けられたユーザがメールを開くかクリックすると、メールアドレスを共有するすべてのユーザプロファイルがopening としてマークされ、キャンペーンをクリックします。

#### 例外:APIトリガキャンペーン

API トリガーキャンペーンは、オーディエンスが定義されている場所に応じて重複排除または重複排除を実行します。つまり、重複するメールは、複数の詳細を受信するために、API コール内で個別の`user_ids` として直接的にターゲットにする必要があります。以下に、API トリガーキャンペーンの3 つのシナリオを示します。

- **シナリオ1:ターゲットセグメントでの重複メール:**同じメールが、API トリガキャンペーンのダッシュボードのオーディエンスフィルタでグループ化された複数のユーザプロファイルに表示される場合、1 つのプロファイルのみがメールを受信します。
- **シナリオ2:受信者オブジェクト内の異なる`user_ids` のE メールを複製します。**同じメールが\`recipients\` object によって参照される複数の`External_user_IDs` 内にある場合、メールは2 回送信されます。
- **シナリオ3:受信者オブジェクト内のuser\_ids が重複しているため、メールが重複しています。**同じユーザープロファイルを2 回追加しようとすると、1 つのプロファイルのみがメールを受信します。

### 送信メール設定の更新は遡及的に適用されますか?

いいえ。送信メールの設定を更新しても、既存の送信には影響しません。たとえば、メール設定でデフォルトの表示名を変更しても、アクティブなキャンペーンまたはキャンバスの既存のデフォルトの表示名は自動的に置き換えられません。 

### "good"メール配信レートとは何ですか?

通常、"magic number"は、バウンス率が3%以下のメッセージの約98%です。あなたの配送がそれ以下に落ちた場合、通常、心配の原因があります。

しかし、98%を上回る可能性があり、配信可能性の問題が残っています。たとえば、バウンスのすべてが1 つの特定のドメインから送信される場合、そのプロバイダにレピュテーションの問題があることを示す明確なシグナルです。

また、メッセージが配信されてスパムになる可能性があり、潜在的に深刻なレピュテーションの問題を示しています。配信されるメッセージの数だけでなく、オープンレートやクリックレートを監視して、ユーザが実際に受信ボックスにメッセージを表示しているかどうかを判断することが重要です。プロバイダーは通常、すべてのスパムインスタンスを報告しないため、1% のスパム率でさえも、懸念とさらなる分析の原因となる可能性があります。

最後に、ビジネスや送信するメールの種類も配信に影響する可能性があります。たとえば、ほとんどが[トランザクションメールを送信する人は、多くのマーケティングメッセージを送信する人よりも高いレートを期待する必要があります。

### メール配信メトリクスが100%にならないのはなぜですか?

メール配信メトリクス(配信、バウンス、スパム率)の合計が100%にならない場合があります。これは、メールがソフトバウンスされ、最大72時間の再試行期間後に配信されないためです。

ソフトバウンスは、"mailbox full," "server temporary not available,"and more などの一時的または一時的な問題によってバウンスするメールです。ソフトバウンスされたメールが72時間経過しても配信されない場合、このメールはキャンペーン配信メトリクスでは考慮されません。

### オープントラッキングピクセルとは。

[オープントラッキングピクセル]({{site.baseurl}}/user_guide/administrative/app_settings/email_settings/#changing-location-of-tracking-pixel) 送信者のメールクリックトラッキングドメインを利用して、メールオープンイベントを追跡します。ピクセルは、メールのHTML に付加されたイメージタグです。これは、通常、body タグ内の最後のHTML 要素です。ユーザがメールをロードすると、オープンイベントをログに記録するブランド追跡ドメインからイメージを入力するように要求されます。

### メールキャンペーンまたはキャンバスが停止した場合はどうなりますか?

ユーザがキャンバスに入らないようにし、それ以上のメッセージは送信されません。メールキャンペーンとキャンバスの場合、停止ボタンは送信がすぐに停止することを意味しません。これは、送信要求が送信されると、ユーザへの配信を停止できないためです。

### なぜ開いたよりも多くのメールクリックを見ているのですか?

次のいずれかの理由で、開いたクリック数よりも多くのクリック数が表示される場合があります。
\- ユーザーは、1 回のオープンでメールの本文を複数回クリックしています。
\- ユーザは、電話機のプレビューペイン内のメールリンクをクリックします。この場合、Braze はこのメールをクリックしたが開かれていないものとしてログに記録します。
\- ユーザは、前にプレビューしたメールを再び開きます。

### サーバーのクリックをトリガーする潜在的なリスクは何ですか?

メッセージが長すぎたり、感嘆符が多すぎたりするなど、電子メールメッセージの特定の要素には、電子メールセキュリティ応答をトリガーする可能性があります。これらの応答は、レポート作成、IP レピュテーションに影響を与え、ユーザの登録解除を招く可能性があります。 

これらの応答の処理方法に関するベストプラクティスについては、[クリックレートの増加の処理]({{site.baseurl}}/help/help_articles/email/open_rates/)を参照してください。

### ユーザのメールサブスクリプショングループを確認するにはどうすればよいですか?

- **ユーザープロファイル:**個々のユーザープロファイルには、[Search Users]({{site.baseurl}}/user_guide/engagement_tools/segments/user_profiles/#access-profiles) ページからBraze ダッシュボードからアクセスできます。ここでは、電子メールアドレス、電話番号、または外部ユーザーID でユーザープロファイルを検索できます。ユーザプロファイル内のEngagement タブでは、ユーザのメールサブスクリプショングループを表示できます。
- **Rest API:**個々のユーザープロファイルのサブスクリプショングループは、[リストユーザーのサブスクリプションググループのエンドポイント][9] または[リストユーザーのサブスクリプションググループのステータスエンドポイント][8] によって、Braze REST API を使用して表示できます。 

### ユーザのメールサブスクリプショングループを更新するにはどうすればよいですか?

ユーザがキャンバスに入らないようにし、それ以上のメッセージは送信されません。メールキャンペーンとキャンバスの場合、停止ボタンは送信がすぐに停止することを意味しません。これは、送信要求が送信された後、ユーザに配信されるのを停止できないためです。

### Braze は、"Unsubscribe"metric にカウントされた登録解除リンクを追跡できますか?

ブレーズは、サブスクリプション解除リンクがカスタムであり、サブスクリプション解除リンクへのクリックはサブスクリプション解除が実際に発生したことを保証しないため、この機能を提供しません。 

**回避策:**これと同じ結果を得るには、Braze にAPI 呼び出しを送信して、カスタムリンクから登録解除を記録したユーザプロファイルを更新します。 

### Can I add a "view this email in a browser" link to my emails?

いいえ、ろう付けはこの機能を提供しません。なぜなら、モバイル機器や現代の電子メールクライアントでは、画像やコンテンツを問題なくレンダリングする電子メールがますます多く開かれているからです。

**回避策:**これと同じ結果を得るには、メールの内容を外部のランディングページ(ウェブサイトなど) でホストし、メール本文を編集するときに**Link** ツールを使用して、構築しているメールキャンペーンからリンクできます。
