# IP ウォームアップ

> IP ウォームアップは、専用 IP アドレスからのメッセージ受信にメール受信トレイプロバイダーを慣れさせるための手法です。これはメールサービスプロバイダー (ESP) を使用したメール送信において非常に重要な要素であり、Braze における標準的な手法として、メッセージが常に高い到達率で送信先の受信トレイに届くことを確認するために行われます。

IP ウォームアップは、インターネットサービスプロバイダー (ISP) との間で良好なレピュテーションを確立するために設計されています。新しい IP アドレスを使用してメールを送信するたびに、ISP はプログラムでこれらのメールを監視し、ユーザーへのスパム送信に使用されていないことを確認します。IP とドメインのレピュテーションはクレジットスコアのようなものだと考えてください。ISP はこのレピュテーションを使用して、メールが受信トレイに届くかスパムフォルダーに振り分けられるかを判断します。クレジットスコアと同様に、良好なレピュテーションを構築するには時間がかかり、低下したレピュテーションを回復するにはさらに長い時間がかかります。

## IP をウォームアップする時間がない場合はどうすればよいですか？

**IP ウォームアップは必須です。**IP ウォームアップを適切に行わず、メールのパターンが疑わしい場合、メールの配信速度が大幅にスロットリング（制限）されたり遅くなったりする可能性があります。また、ドメインまたは IP が ISP によってブロックされる場合もあり、その結果、メールがユーザーの受信トレイのスパムフォルダーに直接振り分けられる可能性があります。そのため、IP ウォームアップを適切に行うことが重要です。

ISP は、スパムの疑いが生じた場合、ユーザーを保護するためにメールの配信を制限します。たとえば、100,000 人のユーザーに送信する場合、ISP は最初の 1 時間で 5,000 人のユーザーにのみメールを配信する可能性があります。次に、ISP は開封率、クリック率、配信停止、スパムレポートなどのエンゲージメント指標を監視します。そのため、かなりの数のスパムレポートが発生した場合、残りの送信をユーザーの受信トレイに配信するのではなく、スパムフォルダーに振り分けることを選択する可能性があります。

エンゲージメントが中程度の場合、ISP はメールの配信を制限し続け、より多くのエンゲージメントデータを収集して、そのメールがスパムかどうかをより確実に判断しようとする可能性があります。メールのエンゲージメント指標が非常に高い場合、ISP はこのメールのスロットリングを完全に停止する可能性があります。ISP はそのデータを使用してメールレピュテーションを作成し、最終的にメールが自動的にスパムにフィルターされるかどうかを決定します。

ドメインまたは IP が ISP によってブロックされている場合、[メッセージアクティビティログ]({{site.baseurl}}/user_guide/administrative/app_settings/message_activity_log_tab/)のメッセージログには、これらの ISP に異議申し立てを行い、ブロックリストから除外してもらうために訪問すべき Web サイトに関する情報が含まれます。

## IP ウォームアップスケジュール

配信到達性をサポートするために、IP ウォームアップスケジュールに厳密に従うことを強くお勧めします。また、日を空けないことも重要です。継続的なスケーリングが配信指標を改善するためです。既存のメール送信履歴と配信到達性の指標に基づいてスケジュールを選択してください。

{% alert tip %}
アカウントチームの一員として専任の配信到達性リソースをご希望の場合は、Braze アカウントマネージャーにお問い合わせください。
{% endalert %}

ほとんどの場合、ピーク送信量ではなく、1 日の平均送信量までウォームアップします。ISP は主に過去数週間の送信動作を見てレピュテーションを評価するため、ピーク送信量が数か月に一度しか発生しない場合（たとえば、季節的な期間に 700 万通）、送信日が近づいてからそのピークに向けてランプアップできます。ただし、ピーク送信量が 1〜2 週間ごとに発生する場合は、最初からそのピークまでウォームアップしてください。

IP ウォームアップが完了し、目標とする 1 日の送信量に達したら、その量を毎日維持することを目指してください。多少の変動は予想されますが、目標の送信量に達した後、週に一度だけ大量送信を行うと、配信指標や送信者レピュテーションに悪影響を及ぼす可能性があります。

{% alert important %}
ほとんどの ISP は、レピュテーションデータを 30 日間のみ保存します。メッセージを送信しない状態が 1 か月続いた場合、IP ウォームアップのプロセスを繰り返す必要があります。
{% endalert %}

## ウォームアップ時の送信制限方法

組み込みのユーザー制限機能は、IP アドレスのウォームアップに役立つ便利なツールです。キャンペーンの作成中に必要なメッセージングセグメントを選択した後、[ターゲットユーザー]({{site.baseurl}}/user_guide/message_building_by_channel/email/html_editor/creating_an_email_campaign/#step-4-build-the-remainder-of-your-campaign-or-canvas)ステップで**詳細オプション**ドロップダウンを選択してユーザーを制限します。ウォームアップスケジュールの進行に合わせてこの制限を徐々に引き上げ、送信するメールの量を増やすことができます。

## サブドメインセグメンテーション

多くの ISP やメールアクセスプロバイダーは、もはや IP アドレスのレピュテーションだけではフィルタリングを行いません。これらのフィルタリング技術は、ドメインベースのレピュテーションにも対応するようになっています。つまり、フィルターは送信者のドメインに関連付けられたすべてのデータを調べ、IP アドレスだけを個別に評価するわけではありません。このため、メール IP のウォームアップに加えて、マーケティング、トランザクション、および企業メール用に別々のドメインまたはサブドメインを持つことをお勧めします。

{% alert important %}
サブドメインのセグメンテーションは、大量送信者にとって特に重要です。これらの送信者は、アカウントを設定する際に Braze 担当者と連携して、この手法を遵守していることを確認する必要があります。
{% endalert %}

企業メールがトップレベルドメイン経由で送信され、マーケティングメールとトランザクションメールが別のドメインまたはサブドメイン経由で送信されるように、ドメインをセグメント化することをお勧めします。

## ベストプラクティス

以下のベストプラクティスに従うことで、IP ウォームアップを行わないことによるあらゆる悪影響を回避できます。

### 少量のメール送信から始める

毎日の送信量をできるだけ段階的に増やしていきます。急激かつ大量のメールキャンペーンは、ISP にとって最も疑わしいものと見なされます。そのため、まず少量のメールを送信し、最終的に送信する予定のメール量に合わせて段階的にスケールする必要があります。IP は各 ISP で個別にウォームアップしていることに留意してください。ISP 間でレピュテーションデータは共有されません。ウォームアップの送信量を構築する際は、特定の ISP で送信量を急激に増やしすぎないように注意してください。送信量に関係なく、安全のために IP をウォームアップすることをお勧めします。「[IP ウォームアップスケジュール](#ip-warming-schedules)」を参照してください。

### 魅力的な導入コンテンツを配信する

最初のコンテンツが非常に魅力的であることを確認し、ユーザーがメールをクリック、開封、エンゲージする可能性を最大限に高めます。IP をウォームアップする際には、無差別な一斉送信よりも、十分にターゲットを絞ったメールを常に優先してください。

### 一貫した送信ケイデンスを設定する

IP ウォームアップが完了したら、送信ケイデンスを作成し、1 日または数日間にわたってメールを分散させるようにします。できるだけ一貫性のあるスケジュールを作成することで、IP クールダウンを防ぐことができます。IP クールダウンは、送信量が数日間以上停止したり大幅に低下したりした場合に発生する可能性があります。

特定の時間に一度に大量送信するのではなく、より長い時間枠にわたって送信を分散させるには、[IP ウォームアップスケジュール](#ip-warming-schedules)を参照してください。

### メールリストをクリーンアップする

メールリストがクリーンで、古いメールや未検証のメールがないことを確認します。[CASL 準拠と CAN-SPAM 準拠]({{site.baseurl}}/user_guide/administrative/privacy/spam_regulations/)の両方を確認するのが理想的です。

### 送信者のレピュテーションを監視する

IP ウォームアッププロセスを実施する際は、送信者のレピュテーションを注意深く監視してください。以下の特定の指標を監視することが重要です。
- **バウンス率:**キャンペーンのバウンス率が 3〜5% を超える場合は、[Keep It Clean: The Importance of Email List Hygiene](https://www.braze.com/blog/email-list-hygiene/) の記事のガイドラインに従って、リストのクリーン度を評価する必要があります。さらに、[サンセットポリシー]({{site.baseurl}}/user_guide/message_building_by_channel/email/best_practices/sunset_policies/)を導入して、エンゲージメントのないまたは休止状態のメールアドレスへのメール送信を停止することも検討してください。
- **スパムレポート:**いずれかのキャンペーンが 0.08% を超える割合でスパムとして報告されている場合は、送信しているコンテンツを再評価し、関心のあるオーディエンスをターゲットにしていることを確認し、メールが適切な言葉で関心を引くようにしてください。
- **開封率:**開封率は、受信トレイへの配置を示す有用な指標です。ユニーク開封率が 25% を超えている場合、受信トレイへの配置率が高い可能性が高く、これは送信者のレピュテーションが良好であることを示しています。

{% alert tip %}
Braze は、IP をウォームアップする際に[インテリジェントタイミング]({{site.baseurl}}/user_guide/brazeai/intelligence/intelligent_timing/)を使用しないことをお勧めします。IP ウォームアップキャンペーンは最初に送信するキャンペーンの一部であるため、Braze にはユーザーの最適な送信時刻を計算するのに十分な情報がありません。この場合、インテリジェントタイミングを使用したすべてのメッセージはフォールバック時刻にデフォルト設定され、同時に送信されます。
{% endalert %}

{% alert tip %}
ドメインと IP がまだ良好なレピュテーションを確立していないため、IP ウォームアップ中にメールがスパムフォルダーに送信されるのは正常です。メールがスパムフォルダーに振り分けられる場合、メール管理者が Braze の送信ドメインと IP を会社の許可リストに追加する必要がある場合があります。
{% endalert %}
