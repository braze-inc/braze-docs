# 自動 IP ウォームアップ

> 自動 IP ウォームアップを使用すると、新しい IP アドレスからのメール送信量を徐々に増やし、受信トレイプロバイダーにおける送信者の評価を構築できます。

{% multi_lang_include early_access_beta_alert.md feature='Automated IP warming' %}

## 仕組み

自動 IP ウォームアップを使用すると、1日あたりの送信量を徐々に増やすことができ、受信トレイプロバイダーが送信パターンを学習し信頼するようになります。ワークスペースにドメインを追加すると、ホームダッシュボードの**「作業を再開する」**セクションで**「自動 IP ウォームアップ」**タイルを選択できます。このタイルは60日間表示されます。

Braze は最もエンゲージメントの高いサブスクライバーから優先的に送信するため、日々の送信量がベストプラクティスに沿ったペースで増加します。その後、Braze はエンゲージメントと配信可能性のシグナルを追跡します。Braze が問題を検知した場合、システムは自動的にスケジュールを調整します。

{% alert note %}
IP ウォームアップは一度だけ実行できます。
{% endalert %}

## 前提条件

自動 IP ウォームアップを行うには、以下が必要です。

- 検証済みサブドメインと有効な IP アドレス
- IP ウォームアップの表示と起動の権限
    - 「使用状況データを見る」で IP ウォームアップセクションを確認できます
    - 「メールテンプレートを表示」で IP ウォームアップ用のメールテンプレートを表示・選択できます
    - 「メール設定の管理」で IP ウォームアップを起動できます
- 「キャンペーンへのアクセス」
- キャンペーンの承認ワークフローが有効になっている場合は「キャンペーンの承認と却下」
    - Braze は自動 IP ウォームアップから作成されたキャンペーンを自動的に承認します。

## 自動 IP ウォームアップ計画を設定する

### ステップ 1: スケジュールを設定する

1. **送信情報**セクションで、IP アドレスをウォームアップする**送信元アドレス**を選択します。
2. 現在の日次送信量と目標送信量を入力します。
3. 自動 IP ウォームアップの開始日を選択します。この日付は計画の起動日から少なくとも1日後である必要があります。
4. 送信時間を入力します。メッセージは会社のタイムゾーンで送信されます。
5. **次へ: セグメント**を選択して設定を続けます。

![スケジュールの詳細例。]({% image_buster /assets/img/automated_ip_warming_schedule.png %})

### ステップ 2: セグメントを選択し、順位付けする

1. 次に、ターゲットとするセグメントを選択します。IP ウォームアップ期間中、Braze は最もエンゲージメントの高いユーザーへの送信から開始し、時間の経過とともに送信量を徐々に増やしながら、エンゲージメントの低いセグメントを段階的に追加していきます。
2. 次に、セグメントをドラッグ＆ドロップして、エンゲージメントの高い順から低い順に並べ替えます。高エンゲージメントには、メールを常に開封しクリックする受信者が含まれます。低エンゲージメントには、メールへの関与が不安定な受信者や、長期間メールに関与していない受信者が含まれます。
3. **次へ: メッセージ**を選択して設定を続けます。

![自動 IP ウォームアップの対象として選択された2つのセグメント。]({% image_buster /assets/img/automated_ip_warming_segment.png %})

### ステップ 3: 送信するメッセージを選択する

1. **メールテンプレートを選択**をクリックします。
2. 送信するメッセージのメールテンプレートを選択します。IP ウォームアップ期間中に送信するコンテンツは、開封やクリックを促すものにしてください。過去に好評だったコンテンツを選ぶことをお勧めします。例えば、プロモーションオファーを使って、即時のエンゲージメントや購入を促すことができます。
3. **テンプレートを選択**をクリックします。Braze は起動前に必要なテンプレート数を計算します。最低限必要な数よりも多くのテンプレートを用意することを推奨します。これにより、配信の問題が発生してもシステムが停止せずに調整できるようになります。
4. 必要な数のテンプレートを追加した後、**次へ: サマリー**を選択します。

{% alert important %}
IP ウォームアップツールから作成されたキャンペーンへの変更（スケジュール日、セグメント、ボリュームの変更など）は、IP ウォームアップの**サマリー**ページには反映されません。
{% endalert %}

### ステップ 4: 確認して起動する

IP ウォームアップ計画の詳細を確認します。次に、**起動**を選択します。

## アクティブな IP ウォームアップ中

IP ウォームアップキャンペーンは、翌日に IP ウォームアップを開始する場合を除き、1〜2日前に作成されます。これらのキャンペーンは自動的に以下の形式で命名されます: `IP Warming Day [X] - [Date] - [Template Name]`

目標とする1日の送信量に達すると、システムはその日の送信を停止して送信者の評価を保護します。

システムは以下の業界基準に基づいて健全性を監視します:

- 配信率が90%以下に低下
- 開封率が10%未満
- バウンス率が5%超
- スパム苦情率が0.04%超

統計がこれらの基準値を下回った場合、システムは送信者の評価へのリスクを軽減するため、翌日の送信量を増加させる代わりに現状の送信量を維持します。

## IP ウォームアップ計画を停止する

Braze では、IP ウォームアップと今後のキャンペーン作成を停止できます。ただし、既にアクティブなキャンペーンや今後24〜48時間以内にスケジュールされたキャンペーンについては、手動で停止する必要がある場合があります。IP ウォームアップ計画を停止すると、関連するすべてのキャンペーンも停止します。

ただし、一旦停止すると、IP ウォームアップは再開できません。代わりに、中断した箇所から再開するための新しい計画を設定する必要があります。具体的には以下の手順です。

- 停止した計画の既存データをダウンロードして記録として保管してください。新しい IP ウォームアップを開始すると、以前のトラッカーは削除されます。
- **現在の日次送信量**を最新の送信量に更新します。
- 前回の IP ウォームアップで使用したセグメントを再利用する場合は、以前のキャンペーンを受信済みのユーザーを除外するフィルターをセグメントに追加します。

## IP ウォームアップが完了した時

IP ウォームアップは、会社のタイムゾーンにおける最終日の深夜0時に終了した時点で完了とマークされます。例えば、IP ウォームアップ計画で最後に送信されたキャンペーンが午後8時に送信された場合、計画は4時間後に完了とマークされます。

トラッカーは計画終了後90日間ホームページに表示されます。90日後、トラッカーは削除されます。データのダウンロードには、以下の標準的なメール指標が含まれます。

- _送信済み_
- _配信済み_
- _バウンス数_
- _スパムレポート_
- _総開封数_
- _ユニーク開封数_
- _クリック数_
- _配信停止済み_

1日にボリューム要件を満たすために複数のキャンペーンが使用された場合、それらは日次ビューで集計されます。
