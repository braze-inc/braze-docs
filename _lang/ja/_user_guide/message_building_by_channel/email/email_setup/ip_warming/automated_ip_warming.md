# 自動 IP ウォームアップ

> オートメーションによるIPウォームアップ機能を使って、新しいIPアドレスからのメール送信量を徐々に増やせ。そうすることで受信トレイにおける送信者の評価を高められる。

{% multi_lang_include early_access_beta_alert.md feature='Automated IP warming' %}

## 仕組み

オートメーションされたIPウォーミングを利用すれば、1日あたりの送信量を徐々に増やせる。これにより受信トレイプロバイダーは送信パターンを学習し、信頼するようになる。ワークスペースにドメインを追加すると、ホームダッシュボードの**「作業を再開する」**セクションで**「自動IPウォームアップ**」タイルを選択できる。このタイルは60日間ここに表示されたままになる。

Brazeは最もエンゲージメント度の高いサブスクライバーを優先的に送信する。これにより、日々の送信量がベストプラクティスに沿ったペースで増加する。次に、Brazeはエンゲージメントと配信可能性のシグナルをトラッキングする。Brazeが問題を検知した場合、システムは自動的にスケジュールを調整する。

{% alert note %}
IPウォーミングは一度だけ実行できる。
{% endalert %}

## 前提条件

自動IPウォーミングを行うには、以下のものが必要だ：

- 検証済みサブドメインと有効なIPアドレス
- IP ウォームアップの表示と起動の権限
    - 「使用状況データを見る」でIPウォームアップのセクションを確認する
    - 「メールテンプレートを表示」でIPウォーミング用のメールテンプレートを表示し選択する
    - 「メール設定の管理」でIP ウォームアップを開始する
- アクセスキャンペーン 
- キャンペーンの承認ワークフローが有効になっている場合、「キャンペーンの承認と却下」 
    - BrazeはオートメーションによるIPウォームアップから作成されたキャンペーンを、代わりに自動的に承認する。

## オートメーションによるIPウォームアップ計画を設定する

### ステップ 1: スケジュールを設定する

1. **送信情報**セクションで、ウォームIPアドレスの**送信元アドレス**を選択する。
2. 現在の日次送信量と目標送信量を入力せよ。
3. オートメーションによるIPウォーミングの開始日を選択せよ。この日付は計画開始日から少なくとも1日後でなければならない。
4. 送信時間を入力せよ。これは会社のタイムゾーンでメッセージングを行う。
5. **次へ] を選択します。設定を続ける**ためのセグメント。

![スケジュールの詳細例。]({% image_buster /assets/img/automated_ip_warming_schedule.png %})

### ステップ 2:セグメントを選択し、順位付けする

1. 次に、ターゲットとするセグメントを選択する。IPウォーミング期間中、Brazeは最もエンゲージメントの高いユーザーへの配信を開始し、時間の経過とともに配信量を徐々に増やしていく。同時に、エンゲージメントの低いセグメントを徐々に追加していく。 
2. 次に、セグメントをドラッグ＆ドロップして、エンゲージメントが高い順に並べ替えろ。高いエンゲージメントには、メールを常に開封しクリックする受信者が含まれる。エンゲージメントが低いとは、メールへの関与が不安定な受信者や、長期間メールに関与していない受信者を指す。
3. **次へ] を選択します。設定を続けるための**メッセージ。

![オートメーションによるIPウォームアップの対象として選択された2つのセグメント。]({% image_buster /assets/img/automated_ip_warming_segment.png %})

### ステップ 3:メッセージングで送信するメッセージを選択せよ

1. **メールテンプレート**を選択せよ。
2. 送信するメッセージのメールテンプレートを選択せよ。IPウォーミング期間中に送信する内容は、開封率とクリック率を高めるようにすべきだ。過去に好評だったコンテンツを選ぶことをお勧めする。例えば、プロモーションオファーを使って、即時のエンゲージメントや購入を促すことができる。
3. **テンプレートを選択せよ**。Brazeは、ローンチ前に必要なテンプレートの数を計算する。最低限必要な数よりも多くのテンプレートを用意することを推奨する。これにより、配信の問題が発生してもシステムが停止せずに調整できるようになる。
4. 必要な数のテンプレートを追加した後、次**へを選択する。要約**。

{% alert important %}
IPウォーミングツールから作成したキャンペーンへの変更（スケジュールされた日、セグメント、ボリュームの変更など）は、IPウォーミングの**サマリー**ページには反映されない。
{% endalert %}

### ステップ 4: レビューと開始

IPウォームアップ計画の詳細を確認せよ。次に、**起動**を選択する。

## IPのアクティブなウォームアップ中

IP ウォームアップキャンペーンは、翌日にIP ウォームアップを開始する場合を除き、1～2日前に作成される。これらのキャンペーンは自動的に以下の形式で命名される：`IP Warming Day [X] - [Date] - [Template Name]`

目標とする1日の送信数が達成されると、システムはその日の送信を停止する。これは送信者の評価を守るためだ。 

このシステムは、以下の業界基準に基づいて健康状態を監視する： 

- 配達率は90％以下に低下する。
- 開封率が10％未満だ
- 5％を超えるバウンス
- スパム苦情率が0.1％を超える

統計が当社の基準値を下回った場合、システムは送信者の評価リスクを軽減するため、翌日の送信量を増加させる代わりに現状の送信量を維持する。

## IP ウォームアップ計画を停止する

Brazeでは、IPウォームアップと今後のキャンペーン作成を停止できる。ただし、既にアクティブなキャンペーンや、今後24～48時間以内にスケジュールされたキャンペーンについては、手動で停止する必要がある場合がある。IPウォームアップ計画を停止すると、関連するすべてのキャンペーンも停止する。

ただし、一旦停止すると、IP ウォームアップは再開できない。代わりに、中断した箇所から再開するための新たな計画を設定する必要がある。具体的には：

- 停止したプランの既存データをダウンロードして記録として保管しておけ。新しいIP ウォームアップを開始すると、以前のトラッカーは削除されるからだ。
- **現在の日次送信**量を最新の量に更新する
- 前回のIP ウォームアップで使用したセグメントを再利用する場合、以前にキャンペーンを受信したユーザーを除外するフィルターをセグメントに追加する。

## IP ウォームアップが完了した時

IPウォーミングは、貴社のタイムゾーンにおける最終日が深夜0時に終了した時点で完了とみなされる。例えば、IPウォーミング計画で最後に送信されたキャンペーンが午後8時に送信された場合、計画は4時間後に完了としてマークされる。

プラン終了後、トラッカーは90日間ホームページ上に残る。90日後、追跡装置は取り外される。データのダウンロードには、以下の標準的なメール指標が含まれる：

- _送信済み_	
- _配信済み_	
- _バウンス数_	
- _スパムレポート_	
- _総開封数_	
- _ユニーク開封数_	
- _クリック済み_	
- _配信停止済み_

1日にボリューム要件を満たすために複数のキャンペーンが使用された場合、それらは日次ビューで集計される。
