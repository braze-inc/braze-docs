# 自動 IP ウォームアップ

> 自動IPウォーミングを利用して、新しいIPアドレスからのメール量を徐々に増やし、受信トレイプロバイダーからの送信者評価を高める。

{% include early_access_beta_alert.md feature='Automated IP warming' %}

## 仕組み

自動IPウォーミングを使って、1日の送信量を徐々に増やすことで、受信トレイプロバイダーはあなたの送信パターンを学習し、信頼することができる。ワークスペースにドメインを追加すると、ホームダッシュボードの**Pick up where you left off**セクションで**Automated IP Warming**タイルを選択でき、このタイルは60日間ここに残る。

Brazeは、最もエンゲージメントの高いサブスクライバーに最初に配信することで、日々の配信量をベストプラクティスに合ったペースで増やすことができる。そして、Brazeはエンゲージメントと配信可能性のシグナルを追跡する。Brazeが何らかの問題を検出すると、システムが自動的にスケジュールを調整する。

{% alert note %}
IPウォーミングアップは1回しかできない。
{% endalert %}

## 前提条件

自動IPウォーミングアップを行うには、以下のものが必要である：

- 検証済みのサブドメインとアクティブIPアドレス
- IPウォームアップを表示および起動する権限
    - "使用データを見る "でIPウォーミングセクションを見る
    - "View Email Templates "でIPウォーミングのメールテンプレートを表示・選択する。
    - "メール設定の管理 "でIP ウォームアップを起動する。
- "アクセスキャンペーン" 
- キャンペーンの承認ワークフローがオンになっている場合は、「キャンペーンの承認と拒否」を選択する。 
    - Brazeは、自動IPウォーミングから作成されたキャンペーンをお客様に代わって自動的に承認する。

## 自動IP温暖化プランを設定する

### ステップ 1: スケジュールを設定する

1. **送信情報]**セクションで、IPアドレスをウォームアップする[**送信元アドレス**]を選択する。
2. 現在の日次送信量と目標送信量を入力する。
3. 自動IPウォーミングアップの開始日を選択する。この日付は、少なくともプラン開始の1日後でなければならない。
4. 送信時間を入力する。これにより、その会社のタイムゾーンでメッセージが送信される。
5. **次へ] を選択します。セグメンテーション** 、セットアップを続ける。

![スケジュール詳細の例。]({% image_buster /assets/img/automated_ip_warming_schedule.png %})

### ステップ 2:セグメンテーションの選択とランク付け

1. 次に、対象とするセグメンテーションを選択する。IPウォーミングの間、Brazeはエンゲージメントが最も高いユーザーに送信を開始し、時間の経過とともに送信量を徐々に増加させ、エンゲージメントの低いセグメントを徐々に追加していく。 
2. 次に、セグメンテーションをドラッグ＆ドロップして、エンゲージメントの高いものから低いものへとランク付けする。エンゲージメントが高い受信者とは、あなたのメールを常に開封し、クリックしてくれる受信者を指す。エンゲージメントが低い受信者とは、メールへのエンゲージメントが一貫していない受信者や、長い間メールにアクセスしていない受信者を指す。
3. **次へ] を選択します。メッセージング** セットアップを続行する。

![自動IPウォーミングのターゲットとして2つのセグメンテーションを選択した。]({% image_buster /assets/img/automated_ip_warming_segment.png %})

### ステップ 3:送信するメッセージを選択する

1. **メールテンプレートの選択を**選択する。
2. 送信するメッセージのメールテンプレートを選択する。IPウォーミング中に送るコンテンツは開封とクリックを促すものでなければならない。過去に好評だったコンテンツを選ぶことをお勧めする。例えば、すぐにエンゲージメントや購入を促すために、プロモーションオファーを利用することができる。
3. **Select テンプレートを選択する**。Brazeは、起動前に必要なテンプレート数を計算する。システムが配信の問題を停止することなく調整できるように、必要最小限のテンプレートよりも多くのテンプレートを提供することを推奨する。
4. 必要な数のテンプレートを追加したら、**Nextを選択する：まとめ**.

{% alert important %}
IPウォーミングツールから作成されたキャンペーンに加えられた変更（スケジュールされた日付、セグメンテーション、ボリュームの変更など）は、IP**ウォーミングサマリーページには**反映されない。
{% endalert %}

### ステップ 4: レビューと開始

IPウォーミングアップ・プランの詳細を確認する。次に、**Launchを**選択する。

## アクティブIPウォーミング中

IPウォームアップキャンペーンは、翌日にIPウォームアップを開始する場合を除き、1～2日前に作成される。これらのキャンペーンは、自動的に以下のフォーマットで命名される：`IP Warming Day [X] - [Date] - [Template Name]` 。

1日の送信目標数に達すると、システムはその日の送信を停止し、あなたの評判を守る。 

このシステムは、以下の業界ベンチマークに基づき、あなたの健康状態をモニターする： 

- 配達率が90%以下に低下する
- 開封率10％未満
- 5%以上のバウンス
- 0.1%以上のスパム苦情率

統計がベンチマークを下回った場合、送信者の評判へのリスクを軽減するため、翌日には件数を増やすのではなく、保留する。

## IPウォームアップ計画を中止する

Brazeでは、IPウォーミングと今後のキャンペーン作成を停止することができるが、キャンペーンがすでにアクティブであるか、今後24～48時間のスケジュールされた場合は、特定のキャンペーンを手動で停止する必要があるかもしれない。IPウォーミングプランを停止すると、関連するすべてのキャンペーンも停止する。

しかし、停止するとIPウォームアップは再開できない。その代わり、新しいプランを設定して、中断したところから再開する必要がある：

- 新しいIPウォームアップを開始すると、以前のトラッカーは削除されるため、停止したプランの既存のデータをダウンロードして記録しておく。
- **Currents daily send volumeを**最新のボリュームに更新する。
- 前回のキャンペーンを受けたユーザーを除外して、前回のIP ウォームアップと同じセグメントを使用する場合、セグメントにフィルターを追加する。

## IPウォームアップが完了したら

IPウォーミングは、IPウォーミングの最終日があなたの会社のタイムゾーンで午前0時に終了した時点で完了とマークされる。例えば、IPウォーミングプランで送信された最後のキャンペーンが午後8時に送信された場合、プランは4時間後に完了とマークされる。

このトラッカーは、プラン終了後も90日間ホームページに表示される。90日後、トラッカーは削除される。データのダウンロードには、これらの標準的なメール指標が含まれる：

- _送信済み_	
- _配信済み_	
- _バウンス数_	
- _スパムレポート_	
- _総開封数_	
- _ユニーク開封数_	
- _クリック済み_	
- _配信停止済み_

1日に出来高要件を満たすために使用される複数のキャンペーンが含まれる場合、これらは日次ビューに集約される。
