# 受信トレイビジョン

> Inbox Vision を使用すると、さまざまなメール クライアントやモバイル デバイスの視点からメールを表示できます。 

## Inbox Visionでメールをテストする

Inbox Vision は、 **ユーザーとしてプレビューする** ために **ランダム ユーザー** が選択された場合に機能し、カスタム ユーザーやその他のプレビュー ユーザー設定は保持されません。つまり、 **カスタム ユーザー** を選択して Inbox Vision を実行すると、Inbox Vision に表示される内容が他の場所のメッセージ プレビューと異なる可能性があります。 

Inbox Vision で電子メール メッセージをテストするには、次の手順を実行します。

1. ドラッグ アンド ドロップ エディターまたは HTML メール エディターに移動します。 
2. **[プレビューとテスト]** タブを開きます。
3. **Inbox Vision** を選択し、**Inbox Vision の実行を**クリックします。<br><br> ![][3]{: style="max-width:80%;"}<br><br> その後、Braze は、世界中で使用されているさまざまな電子メール クライアントに電子メールの HTML バージョンを送信します。この処理が完了するまでに 2 ～ 10 分ほどかかる場合があります。<br><br> レンダリングされた HTML プレビューは、次の 3 つのセクションに分かれます。 
- **Web クライアント** 
- **アプリケーションクライアント** 
- **モバイルクライアント**<br><br>
4. タイルを選択すると、プレビューの詳細が表示されます。<br><br> これらのプレビューを表示するには、メールに件名と有効な送信ドメインが含まれている必要があります。電子メールがデスクトップとモバイル デバイスでどのように表示されるかに注意してください。これらのプレビューを表示しながら、コンテンツを確認し、メールが意図したとおりに表示されているかどうかを確認できます。

{% alert tip %}
Inbox Vision を使用して、ダーク モードとライト モードの違いをテストし、メールが適切に受信されているかどうかを確認します。
{% endalert %}

![HTML エディターの Inbox Vision の概要。][1]

{: start="5"}
5\.必要に応じてテンプレートを変更し、**「テストの再実行」** をクリックして更新されたプレビューを表示します。

{% alert important %}
一般に、電子メールのコンテンツがユーザー プロファイル情報などのテンプレート情報に依存している場合、その電子メールは Inbox Vision で機能しません。これは、この機能を使用して電子メールを送信するときに、Braze テンプレートが空のユーザーで実行されるためです。
{% endalert %}

## コード分​​析

コード分​​析は、Braze が HTML に存在する可能性のある問題を強調表示し、各問題の発生回数を表示し、サポートされていない HTML 要素に関する情報を提供する方法です。 

### コード分​​析情報の表示

この情報は、**Inbox Vision** タブで選択することで確認できます。 <i class="fas fa-list"></i>**リストビュー**。このリスト ビューは、HTML 電子メール テンプレートでのみ使用できます。ドラッグ アンド ドロップ メール テンプレートを使用している場合は、代わりにプレビューをチェックして、考えられる問題を解決してください。

![Inbox Vision プレビューでのコード分析の例。][2]

{% alert note %}
特定の電子メール クライアントでは、コード分析がプレビューよりも速く表示されることがあります。これは、Braze がスクリーンショットを撮る前に、メールが受信トレイに届くまで待機するためです。
{% endalert %}

## スパムテスト

スパム テストでは、電子メールがスパム フォルダーに届くか、顧客の受信トレイに届くかを予測します。スパム テストは、IronPort、SpamAssassin、Barracuda などの主要なスパム フィルター、および Gmail.com や Outlook.com などの主要なインターネット サービス プロバイダー (ISP) フィルターで実行されます。

### スパムテスト結果の表示

スパム テストの結果を確認するには、**Inbox Vision** セクションの **[スパム テスト]** タブをクリックします。**スパム テスト結果** テーブルには、スパム フィルターの名前、ステータス、およびタイプがリストされます。

![3 つの列を持つスパム テスト結果テーブル:名前、ステータス、タイプ。スパムテストに合格したスパムフィルターと ISP フィルターのリストがあり、電子メールキャンペーンがスパムフォルダーに届かないことを示しています。][4]

これらの結果を確認し、電子メール キャンペーンを調整した後、**「テストの再実行」** をクリックしてスパム テストの結果を再読み込みします。

## テスト精度

すべてのテストは実際の電子メール クライアントを通じて実行されます。Braze は、すべてのレンダリングが可能な限り正確であることを確認するために一生懸命取り組んでいます。電子メール クライアントで問題が継続的に発生する場合は、 [サポート チケットを]({{site.baseurl}}/braze_support/)開いてください。
