# メール購読

> ユーザーのサブスクリプション状態について学び、サブスクリプショングループを作成・管理する方法、そしてユーザーのサブスクリプションに基づいてユーザーをセグメント化する方法を学ぶ。

このドキュメントは情報提供のみを目的としています。本書は、いかなる立場においても法的助言を提供することを意図したものではなく、また、法的助言を提供するものとして信頼されるものでもない。マーケティングおよび取引に関する電子メールの送信は、特定の法的要件の対象となる場合がある。御社に特有の適用されるすべての法律、規則および規制を確実に遵守するために、御社の法律顧問および / または規制コンプライアンスチームに助言を求めるべきです。

## 購読状態 {#subscription-states}

Brazeにはメールユーザー向けの3つのグローバルサブスクリプション状態がある。これらの状態は、ユーザーからのメッセージを遮断する。例えば、州`unsubscribed`のユーザーは、や`subscribed`を対象とした`opted-in`メッセージを受け取らない。

| 状態 | 定義 |
| ----- | ---------- |
| オプトイン済み | ユーザーが電子メールの受信を希望していることを明示的に確認した。メール送信についてユーザーの同意を得るには、明示的なオプトインプロセスをお勧めします。 |
| サブスクリプション登録済み | ユーザーが、メールの受信について購読解除も、明示的なオプトインも行っていない。これは、ユーザープロファイルが作成されたときのデフォルトのサブスクリプション状態である。 |
| 配信停止済み | ユーザーが明示的にメールを購読解除した。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

{% alert note %}
Braze は、購読状態のグローバルな変化、および購読グループ全体での変化についてデータポイントをカウントしません。
{% endalert %}

### 購読解除メールアドレス

Brazeは[、カスタムフッター]({{site.baseurl}}/user_guide/message_building_by_channel/email/custom_email_footer)から手動で配信停止したユーザーを自動的に配信停止する。ユーザーがメールアドレスを更新し、かつ**送信**設定で**「メール更新時にユーザーを再登録する」**がイネーブルドになっている場合、通常の送信が再開される。

ユーザーがあなたのメールを1通以上スパムとしてマークした場合、Brazeはそのユーザーにはトランザクションメールのみを送信する。トランザクションメールとは**、ターゲットオーディエンス**内の**「配信停止ユーザーを含む全ユーザーに送信」**オプションを指す。

{% alert tip %}
ユーザーを効果的に再参加させる方法については、[IPウォーミングアップの]({{site.baseurl}}/user_guide/message_building_by_channel/email/email_setup/ip_warming/)ベストプラクティスを参照されたい。
{% endalert %}

### バウンスおよび無効なメール

{% multi_lang_include analytics/metrics.md metric='Hard Bounce' %} {% multi_lang_include analytics/metrics.md metric='Soft Bounce' %} 

メールアドレスがハードバウンスした場合、Brazeは自動的にユーザーのサブスクリプション状態を「配信停止」に設定しない。アドレスがハードバウンス（無効または存在しない）した場合、Brazeはそのアドレスを無効とマークし、それ以降の送信を試みない。ユーザーがメールアドレスの変更を行った場合、Brazeは送信を再開する。Brazeはソフトバウンスを72時間再試行する。

### メール購読状態の更新

ユーザーのメール購読状態を更新するには、4 つの方法があります。

#### SDKの統合

Braze SDK を使用して、ユーザーの購読状態を更新します。

#### REST API

[エンド`/users/track`ポイント]({{site.baseurl}}/api/endpoints/user_data/post_user_track/)を使用して、ユーザーの[`email_subscribe`]({{site.baseurl}}/api/objects_filters/user_attributes_object)属性を更新する。

#### ユーザープロフィール

1. [**ユーザーを検索**] でユーザーを検索します。 
2. **エンゲージメント**の下で、ユーザーのサブスクリプションステータスを変更するには、**配信停止**、**購読済み**、または**オプトイン**を選択する。 

利用可能な場合、ユーザープロファイルにはユーザーの購読が最後に変更されたときのタイムスタンプも表示されます。

#### ユーザー設定センター

メールの末尾に[「ユーザー設定センター](#email-preference-center) Liquid」を含め、ユーザーが登録の追加や解除を行えるようにする。Brazeは、ユーザー設定センターからのサブスクリプション状態の更新を管理する。

### メール購読状態の確認

![John Doeのユーザプロファイルで、メール購読の状態が「購読済み」に設定されている。]({% image_buster /assets/img/push_example.png %}){: style="float:right;max-width:35%;margin-left:15px;"}

ユーザーのメールサブスクリプション状態は、以下の方法で確認できる：

1. **REST API でのエクスポート:**[セグメント別ユーザーのエクスポート]({{site.baseurl}}/api/endpoints/export/user_data/post_users_segment/)または[識別子によるユーザーのエクスポート]({{site.baseurl}}/api/endpoints/export/user_data/post_users_identifier/)エンドポイントを使用して、個々のユーザープロファイルをJSON 形式でエクスポートします。
2. **ユーザープロフィール:**[[ユーザーを検索]({{site.baseurl}}/user_guide/engagement_tools/segments/user_profiles/)]ページでユーザープロファイルを検索し、[**エンゲージメント**] タブを選択して、ユーザーの購読状態を表示して手動で更新します。

ユーザーがメールアドレスを更新すると、ユーザーの購読状態は購読中に設定されます。ただし、更新後のメールアドレスが Braze ワークスペースの他の場所に既に存在する場合を除きます。

## サブスクリプショングループ

購読グループは、[グローバルな購読状態](#subscription-states)からさらにオーディエンスを絞り込むことができるセグメントフィルターです。ワークスペースごとに最大350 のサブスクリプショングループを追加できます。サブスクリプショングループを使用すると、より詳細なサブスクリプションオプションをエンドユーザーに提示できます。

例えば、複数のカテゴリのメールキャンペーン（プロモーション、ニュースレター、製品アップデート）を送信するとする。その場合、購読グループを使用すると、顧客は[メールユーザー設定センター](#email-preference-center)を使用して、どのメールカテゴリを購読するか、または購読解除するかを 1 つのページから一括で選択できます。また、購読グループを利用して、毎日、毎週、毎月の購読グループを作成し、顧客にメールの受信頻度を選択させることもできる。

Braze ダッシュボードの [** 購読グループ**] ページに保存した購読グループをプログラムで管理するには、[[購読グループのエンドポイント]({{site.baseurl}}/api/endpoints/subscription_groups)] を使用します。

### 購読グループの作成

1. **オーディエンス**＞**サブスクリプショングループ管理**へ移動する。
2. **メールサブスクリプショングループの作成**を選択します。 
3. サブスクリプショングループに名前と説明を付けます。
4. [**保存**] を選択します。 

すべての購読グループは自動的にユーザー設定センターに追加されます。

![購読グループを作成するためのフィールド。]({% image_buster /assets/img/sub_group_create.png %}){: style="max-width:75%"}

### 購読グループによるセグメンテーション

セグメントを作成するときに、購読グループ名をフィルターとして設定します。これにより、グループにオプトインしたユーザーがメールを受信することが確認されます。これは、月刊ニュースレター、クーポン、メンバーシップ層などに最適です。

![「離脱ユーザー」セグメントのユーザーを対象に、「週次メール」サブスクリプショングループに属するユーザーをフィルターで絞り込む例。]({% image_buster /assets/img/segment_sub_group.png %}){: style="max-width:90%"}

### 購読グループのアーカイブ

アーカイブされた購読グループは編集できず、セグメントフィルターやユーザー設定センターに表示されなくなります。メール、キャンペーン、キャンバスでセグメントフィルターとして使用されているグループをアーカイブしようとすると、エラーメッセージが表示され、そのグループの使用をすべて削除するまでアーカイブできない。

[**購読グループ**] ページからグループをアーカイブするには、以下を実行します。

1. サブスクリプショングループのリストでグループを検索します。 
2. <i class="fa-solid fa-ellipsis-vertical"></i>ドロップダウンメニューから**アーカイブ**を選択します。

Brazeはアーカイブされたグループ内のユーザーに対する状態変更を処理しない。例えば、アレックスがサブスクリプショングループ1を購読している状態でアーカイブすると、アレックスは配信停止リンクをクリックしても「購読中」の状態が維持される。これは問題ではない。サブスクリプショングループ1はアーカイブされており、それを使ってメッセージを送信することはできないからだ。

#### 購読グループのサイズの表示

[**購読グループ**] ページの [**購読グループの時系列**] グラフを参照して、一定期間のユーザー数に基づく購読グループのサイズを確認できます。これらの購読グループのサイズは、セグメントサイズの計算など、Braze の他の分野とも一致します。

![12月2日から11日までの「サブスクリプショングループの時系列」グラフの例。グラフは、6日から7日にかけてユーザー数が1000万人近く増加したことを示しています。]({% image_buster /assets/img_archive/subscription_group_graph.png %})

#### キャンペーン分析での購読グループの表示

特定のメールキャンペーンから、サブスクリプション状態（購読中または配信停止）を変更したユーザー数を、そのキャンペーンの分析ページで確認できる。

1. キャンペーンの**Campaign Analytics** ページから、**Email Message Performance** セクションまでスクロールダウンします。
2. **Subscription Groups** の下にある矢印を選択すると、顧客が送信した状態変更の集計カウントが表示されます。

![顧客が送信した状態変更の合計数を表示する「メールメッセージパフォーマンス」ページ。]({% image_buster /assets/img/campaign_analytics_sub_groups.png %})

### ユーザーのメールサブスクリプショングループを確認する

- **ユーザープロフィール:**個々のユーザープロフィールは、[ユーザー検索]({{site.baseurl}}/user_guide/engagement_tools/segments/user_profiles/#access-profiles)ページのBrazeダッシュボードからアクセスできます。ここでは、メールアドレス、電話番号、または外部ユーザーIDでユーザープロフィールを検索できます。**エンゲージメント**タブでは、ユーザーのメールサブスクリプショングループも確認できる。
- **Braze REST API:**[ユーザーのサブスクリプショングループ一覧エンドポイント]({{site.baseurl}}/api/endpoints/subscription_groups/get_list_user_subscription_groups/)または[ユーザーのサブスクリプショングループステータス一覧エンドポイント]({{site.baseurl}}/api/endpoints/subscription_groups/get_list_user_subscription_group_status/)を使用し、個々のユーザープロファイルのサブスクリプショングループを表示する。 

## メールユーザー設定センター

メールユーザー設定センターでは、どのユーザーがサブスクリプショングループのニュースレターを受け取るかを管理できる。ダッシュボードの**「サブスクリプショングループ」**の下にある。作成した各購読グループは、ユーザー設定センターのリストに追加されます。 

ユーザー設定センターの追加やカスタマイズの方法については、[ユーザー設定センター]({{site.baseurl}}/user_guide/message_building_by_channel/email/preference_center/overview/)を参照してください。

## メール購読の変更{#changing-email-subscriptions}

大抵の場合、ユーザーは受信したメールに含まれるリンクを通じてメールサブスクリプションを管理する。すべてのメールの末尾に、法的要件を満たしたフッターと配信停止リンクを挿入せよ。ユーザーが配信停止URLを選択すると、Brazeは配信停止を行い、変更を確認するランディングページを表示する。このLiquidタグを含める：{%raw%}`${set_user_to_unsubscribed_url}`{%endraw%}

ユーザーがユーザー設定センターで「上記のすべての種類のメール配信を停止する」を選択した場合、Brazeはそのユーザーのグローバルメールサブスクリプションステータスを「`unsubscribed`停止済み」に設定し、すべてのグループからの配信を停止する。

### カスタムフッターを作成する {#custom-footer}

デフォルトのフッターを使いたくないなら、ワークスペース全体で使えるカスタムメールフッターを作成し、それをすべてのメールにテンプレートとして適用するんだ{% raw %}`{{${email_footer}}}`{% endraw %}。

これにより、メールテンプレートやメールキャンペーンごとに新しいフッターを作成する必要がなくなる。ステップについては、[カスタムメールフッターを]({{site.baseurl}}/user_guide/message_building_by_channel/email/custom_email_footer/)参照せよ。

#### 中国の IP アドレスの購読状態の管理

中国のIPアドレスを想定しているなら、リスト`unsubscribed`管理を配信停止リンクだけに頼るな。サポートチケットや顧客担当者のメールなど、別の配信停止方法を提供する。 

### カスタム購読解除ページの作成

ユーザーが配信停止URLを選択すると、Brazeは変更を確認するデフォルトのランディングページを表示する。

サブスクライバー後に表示されるランディングページを、デフォルトではなくカスタムで作成するには：

1. [**メール設定**] > [**購読ページおよびフッター**] に移動します。
2. カスタムランディングページの HTML を指定します。 

誤って配信停止を行った場合にユーザーが再購読できるよう、再購読リンク（例：{% raw %}`{{${set_user_to_subscribed_url}}}`{% endraw %}）を含めること。

![カスタムの配信停止ページに「お別れするのは残念だ」というプレビュー文を表示する。]({% image_buster /assets/img/custom_unsubscribe.png %})

### カスタムオプトインページを作成する

ユーザーがサブスクリプション前に通知設定をコントロールできるように、カスタムのオプトインページを使用する。この追加の連絡は、メールキャンペーンがスパムフォルダに入らないようにするのに役立つ。

1. [**設定**] > [**メール設定**] に移動します。
2. [**サブスクリプションのページおよびフッター**] を選択します。
3. **Custom opt-in page** セクションのスタイルをカスタマイズして、ユーザが購読済みであることをユーザに示す方法を確認します。

ユーザーはこの{% raw %}`{{${set_user_to_opted_in_url}}}`{% endraw %}タグを通じてこのページに到達する。

{% alert tip %}
二重オプトイン方式を使って、連絡の精度を高めろ。Brazeは追加の確認メールを送信する。ユーザーはリンク経由で通知設定を確認する。確認後、ユーザーは登録される。
{% endalert %}

![カスタムのオプトインメールに「これからも当方からの連絡を受け取りたいと思ってくれて嬉しい」というメッセージを添える。]({% image_buster /assets/img/custom_optin.png %})

## 購読とキャンペーンのターゲティング{#subscriptions-and-campaign-targeting}

デフォルトでは、Brazeはプッシュ通知やメールメッセージによるキャンペーンを、サブスクライバーまたはオプトイン済みのユーザーを対象とする。「**これらのユーザーに送信」**の横にあるドロップダウンを選択して、オーディエンスを変更する**。**

Braze は3つのターゲット状態をサポートしています。

- 購読中またはオプトイン済みのユーザー (デフォルト)。
- オプトイン済みのユーザーのみ。
- 購読解除したユーザーを含むすべてのユーザー。

{% alert important %}
これらのターゲット設定を使用する場合は、該当する[spam 法]({{site.baseurl}}/help/best_practices/spam_regulations/#spam-regulations) を順守する必要があります。
{% endalert %}

## ユーザー購読によるセグメント化{#segmenting-by-user-subscriptions}

「メールサブスクリプションステータス」と「プッシュサブスクリプションステータス」のフィルターを使って、サブスクリプションステータスでユーザーをセグメントする。

これを使って、オプトインもオプトアウトもしていないユーザーを対象とし、明示的なオプトインを促す。「メール／プッシュ通知のサブスクリプションステータスが購読済み」というフィルターでセグメントを作成し、購読しているがオプトインしていないユーザーにキャンペーンを送信する。

![セグメントフィルターとして使用されるメール購読ステータス。]({% image_buster /assets/img_archive/not_optin.png %})
