# メール購読

> ユーザーのサブスクリプション状態、サブスクリプショングループの作成・管理方法、およびサブスクリプションに基づいてユーザーをセグメント化する方法について説明します。

## 購読状態 {#subscription-states}

Braze にはメールユーザー向けの3つのグローバルサブスクリプション状態があります。これらの状態は、ユーザーへのメッセージ配信を制御します。例えば、`unsubscribed` 状態のユーザーは、`subscribed` や `opted-in` を対象としたメッセージを受信しません。

| 状態 | 定義 |
| ----- | ---------- |
| オプトイン済み | ユーザーがメールの受信を希望していることを明示的に確認した状態です。メール送信についてユーザーの同意を得るには、明示的なオプトインプロセスをお勧めします。 |
| 購読済み | ユーザーが配信停止も明示的なオプトインも行っていない状態です。ユーザープロファイルが作成されたときのデフォルトのサブスクリプション状態です。 |
| 配信停止済み | ユーザーがメールの配信停止を明示的に行った状態です。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

{% alert note %}
Braze は、グローバルおよびサブスクリプショングループ全体での購読状態の変更をデータポイントとしてカウントしません。
{% endalert %}

### 配信停止メールアドレス

Braze は、[カスタムフッター]({{site.baseurl}}/user_guide/message_building_by_channel/email/custom_email_footer)から手動で配信停止したユーザーを自動的に配信停止します。ユーザーがメールアドレスを更新し、**送信設定**で**「メール更新時にユーザーを再登録する」**が有効になっている場合、通常の送信が再開されます。

ユーザーがメールを1通以上スパムとしてマークした場合、Braze はそのユーザーにはトランザクションメールのみを送信します。トランザクションメールとは、**ターゲットオーディエンス**内の**「配信停止ユーザーを含む全ユーザーに送信」**オプションを指します。

{% alert tip %}
ユーザーを効果的に再エンゲージする方法については、[IP ウォーミング]({{site.baseurl}}/user_guide/message_building_by_channel/email/email_setup/ip_warming/)のベストプラクティスを参照してください。
{% endalert %}

### バウンスおよび無効なメール

{% multi_lang_include analytics/metrics.md metric='Hard Bounce' %} {% multi_lang_include analytics/metrics.md metric='Soft Bounce' %} 

メールアドレスがハードバウンスした場合、Braze はユーザーのサブスクリプション状態を自動的に「配信停止」に設定しません。アドレスがハードバウンス（無効または存在しない）した場合、Braze はそのアドレスを無効とマークし、それ以降の送信を試みません。ユーザーがメールアドレスを変更した場合、Braze は送信を再開します。Braze はソフトバウンスを72時間再試行します。

### メール購読状態の更新

ユーザーのメール購読状態を更新するには、4つの方法があります。

#### SDK の統合

Braze SDK を使用して、ユーザーの購読状態を更新します。

#### REST API

[`/users/track` エンドポイント]({{site.baseurl}}/api/endpoints/user_data/post_user_track/)を使用して、ユーザーの [`email_subscribe` 属性]({{site.baseurl}}/api/objects_filters/user_attributes_object)を更新します。例えば、ユーザーがカスタム配信停止リンクを使用した際にメール購読状態を配信停止に設定するには、リクエストのユーザー属性に `email_subscribe: "unsubscribed"` を含めます。

#### ユーザープロファイル

1. **ユーザーを検索**でユーザーを検索します。 
2. **エンゲージメント**の下で、**配信停止**、**購読済み**、または**オプトイン**を選択して、ユーザーのサブスクリプションステータスを変更します。 

利用可能な場合、ユーザープロファイルにはユーザーの購読が最後に変更されたときのタイムスタンプも表示されます。

#### ユーザー設定センター

メールの末尾に[ユーザー設定センター](#email-preference-center)の Liquid を含め、ユーザーがオプトインまたはオプトアウトできるようにします。Braze は、ユーザー設定センターからのサブスクリプション状態の更新を管理します。

### メール購読状態の確認

![John Doe のユーザープロファイルで、メール購読の状態が「購読済み」に設定されている。]({% image_buster /assets/img/push_example.png %}){: style="float:right;max-width:35%;margin-left:15px;"}

ユーザーのメールサブスクリプション状態は、以下の方法で確認できます。

1. **REST API でのエクスポート：**[セグメント別ユーザーのエクスポート]({{site.baseurl}}/api/endpoints/export/user_data/post_users_segment/)または[識別子によるユーザーのエクスポート]({{site.baseurl}}/api/endpoints/export/user_data/post_users_identifier/)エンドポイントを使用して、個々のユーザープロファイルを JSON 形式でエクスポートします。
2. **ユーザープロファイル：**[ユーザーを検索]({{site.baseurl}}/user_guide/engagement_tools/segments/user_profiles/)ページでユーザープロファイルを検索し、**エンゲージメント**タブを選択して、ユーザーの購読状態を表示して手動で更新します。

ユーザーがメールアドレスを更新すると、購読状態は「購読済み」に設定されます。ただし、更新後のメールアドレスが Braze ワークスペースの他の場所に既に存在する場合を除きます。

## サブスクリプショングループ

サブスクリプショングループは、[グローバルな購読状態](#subscription-states)からさらにオーディエンスを絞り込むことができるセグメントフィルターです。ワークスペースごとに最大350のサブスクリプショングループを追加できます。サブスクリプショングループを使用すると、より詳細なサブスクリプションオプションをエンドユーザーに提示できます。

例えば、複数のカテゴリのメールキャンペーン（プロモーション、ニュースレター、製品アップデート）を送信するとします。その場合、サブスクリプショングループを使用すると、顧客は[メールユーザー設定センター](#email-preference-center)を使用して、どのメールカテゴリを購読するか、または配信停止するかを1つのページから一括で選択できます。また、サブスクリプショングループを利用して、毎日、毎週、毎月のサブスクリプショングループを作成し、顧客にメールの受信頻度を選択させることもできます。

Braze ダッシュボードの**サブスクリプショングループ**ページに保存したサブスクリプショングループをプログラムで管理するには、[サブスクリプショングループのエンドポイント]({{site.baseurl}}/api/endpoints/subscription_groups)を使用します。

### サブスクリプショングループの作成

1. **オーディエンス** > **サブスクリプショングループ管理**に移動します。
2. **メールサブスクリプショングループの作成**を選択します。 
3. サブスクリプショングループに名前と説明を付けます。
4. **保存**を選択します。 

すべてのサブスクリプショングループは自動的にユーザー設定センターに追加されます。

![サブスクリプショングループを作成するためのフィールド。]({% image_buster /assets/img/sub_group_create.png %}){: style="max-width:75%"}

### サブスクリプショングループによるセグメンテーション

セグメントを作成するときに、サブスクリプショングループ名をフィルターとして設定します。これにより、グループにオプトインしたユーザーがメールを受信することを確認できます。月刊ニュースレター、クーポン、メンバーシップ層などに最適です。

![「離脱ユーザー」セグメントのユーザーを対象に、「週次メール」サブスクリプショングループに属するユーザーをフィルターで絞り込む例。]({% image_buster /assets/img/segment_sub_group.png %}){: style="max-width:90%"}

### サブスクリプショングループのアーカイブ

アーカイブされたサブスクリプショングループは編集できず、セグメントフィルターやユーザー設定センターに表示されなくなります。メール、キャンペーン、Canvas でセグメントフィルターとして使用されているグループをアーカイブしようとすると、エラーメッセージが表示され、そのグループの使用をすべて削除するまでアーカイブできません。

**サブスクリプショングループ**ページからグループをアーカイブするには、以下を実行します。

1. サブスクリプショングループのリストでグループを検索します。 
2. <i class="fa-solid fa-ellipsis-vertical"></i>&nbsp;ドロップダウンメニューから**アーカイブ**を選択します。

Braze はアーカイブされたグループ内のユーザーに対する状態変更を処理しません。例えば、アレックスがサブスクリプショングループ1を購読している状態でアーカイブすると、アレックスは配信停止リンクをクリックしても「購読中」の状態が維持されます。これは問題ではありません。サブスクリプショングループ1はアーカイブされており、それを使ってメッセージを送信することはできないからです。

#### サブスクリプショングループのサイズの表示

**サブスクリプショングループ**ページの**サブスクリプショングループの時系列**グラフを参照して、一定期間のユーザー数に基づくサブスクリプショングループのサイズを確認できます。これらのサブスクリプショングループのサイズは、セグメントサイズの計算など、Braze の他の領域とも一致します。

![12月2日から11日までの「サブスクリプショングループの時系列」グラフの例。グラフは、6日から7日にかけてユーザー数が約1000万人増加したことを示しています。]({% image_buster /assets/img_archive/subscription_group_graph.png %})

#### キャンペーン分析でのサブスクリプショングループの表示

特定のメールキャンペーンからサブスクリプション状態（購読または配信停止）を変更したユーザー数を、そのキャンペーンの分析ページで確認できます。

1. キャンペーンの**キャンペーン分析**ページから、**メールメッセージパフォーマンス**セクションまでスクロールダウンします。
2. **サブスクリプショングループ**の下にある矢印を選択すると、顧客が送信した状態変更の集計カウントが表示されます。

![顧客が送信した状態変更の合計数を表示する「メールメッセージパフォーマンス」ページ。]({% image_buster /assets/img/campaign_analytics_sub_groups.png %})

### ユーザーのメールサブスクリプショングループを確認する

- **ユーザープロファイル：**個々のユーザープロファイルは、[ユーザーを検索]({{site.baseurl}}/user_guide/engagement_tools/segments/user_profiles/#access-profiles)ページの Braze ダッシュボードからアクセスできます。ここでは、メールアドレス、電話番号、または外部ユーザー ID でユーザープロファイルを検索できます。**エンゲージメント**タブでは、ユーザーのメールサブスクリプショングループも確認できます。
- **Braze REST API：**[ユーザーのサブスクリプショングループ一覧エンドポイント]({{site.baseurl}}/api/endpoints/subscription_groups/get_list_user_subscription_groups/)または[ユーザーのサブスクリプショングループステータス一覧エンドポイント]({{site.baseurl}}/api/endpoints/subscription_groups/get_list_user_subscription_group_status/)を使用して、個々のユーザープロファイルのサブスクリプショングループを表示します。 

## メールユーザー設定センター

メールユーザー設定センターでは、どのユーザーがサブスクリプショングループのニュースレターを受け取るかを管理できます。ダッシュボードの**サブスクリプショングループ**の下にあります。作成した各サブスクリプショングループは、ユーザー設定センターのリストに追加されます。 

ユーザー設定センターの追加やカスタマイズの方法については、[ユーザー設定センター]({{site.baseurl}}/user_guide/message_building_by_channel/email/preference_center/overview/)を参照してください。

## メール購読の変更 {#changing-email-subscriptions}

ほとんどの場合、ユーザーは受信したメールに含まれるリンクを通じてメールサブスクリプションを管理します。すべてのメールの末尾に、法的要件を満たしたフッターと配信停止リンクを挿入してください。ユーザーが配信停止 URL を選択すると、Braze は配信停止を行い、変更を確認するランディングページを表示します。この Liquid タグを含めてください：{%raw%}`${set_user_to_unsubscribed_url}`{%endraw%}

ユーザーがユーザー設定センターで「上記のすべての種類のメール配信を停止する」を選択した場合、Braze はそのユーザーのグローバルメールサブスクリプションステータスを `unsubscribed` に設定し、すべてのグループからの配信を停止します。

### カスタムフッターの作成 {#custom-footer}

デフォルトのフッターを使用しない場合は、ワークスペース全体で使えるカスタムメールフッターを作成し、{% raw %}`{{${email_footer}}}`{% endraw %} を使用してすべてのメールにテンプレートとして適用します。

これにより、メールテンプレートやメールキャンペーンごとに新しいフッターを作成する必要がなくなります。手順については、[カスタムメールフッター]({{site.baseurl}}/user_guide/message_building_by_channel/email/custom_email_footer/)を参照してください。

#### 中国の IP アドレスの購読状態の管理

中国の IP アドレスが想定される場合は、`unsubscribed` リストの管理を配信停止リンクだけに頼らないでください。サポートチケットや顧客担当者のメールなど、別の配信停止方法を提供してください。 

### カスタム配信停止ページの作成

ユーザーが配信停止 URL を選択すると、Braze は変更を確認するデフォルトのランディングページを表示します。

デフォルトの代わりにカスタムランディングページを作成するには：

1. **メール設定** > **購読ページおよびフッター**に移動します。
2. カスタムランディングページの HTML を指定します。 

誤って配信停止を行った場合にユーザーが再購読できるよう、再購読リンク（例：{% raw %}`{{${set_user_to_subscribed_url}}}`{% endraw %}）を含めてください。

![カスタムの配信停止ページに「お別れするのは残念です！」というプレビューが表示されている。]({% image_buster /assets/img/custom_unsubscribe.png %})

### カスタムオプトインページの作成

カスタムオプトインページを使用して、ユーザーがサブスクリプション前に通知設定を確認・管理できるようにします。この追加のコミュニケーションにより、メールキャンペーンがスパムフォルダに入るのを防ぐことができます。

1. **設定** > **メール設定**に移動します。
2. **購読ページおよびフッター**を選択します。
3. **カスタムオプトインページ**セクションのスタイルをカスタマイズして、ユーザーが購読済みであることをどのように表示するかを確認します。

ユーザーは {% raw %}`{{${set_user_to_opted_in_url}}}`{% endraw %} タグを通じてこのページに到達します。

{% alert tip %}
ダブルオプトインプロセスを使用して、アウトリーチの精度を高めましょう。Braze は追加の確認メールを送信し、ユーザーはリンク経由で通知設定を確認します。確認後、ユーザーはオプトインされます。
{% endalert %}

![カスタムのオプトインメールに「引き続き当社からの連絡を受け取りたいと思っていただけて嬉しいです」というメッセージが表示されている。]({% image_buster /assets/img/custom_optin.png %})

## 購読とキャンペーンのターゲティング {#subscriptions-and-campaign-targeting}

デフォルトでは、Braze はプッシュ通知やメールメッセージによるキャンペーンを、購読済みまたはオプトイン済みのユーザーを対象とします。**ターゲットオーディエンス**で**「これらのユーザーに送信：」**の横にあるドロップダウンを選択して変更できます。

Braze は3つのターゲット状態をサポートしています。

- 購読済みまたはオプトイン済みのユーザー（デフォルト）。
- オプトイン済みのユーザーのみ。
- 配信停止したユーザーを含むすべてのユーザー。

{% alert important %}
これらのターゲット設定を使用する場合は、該当する[スパム法]({{site.baseurl}}/help/best_practices/spam_regulations/#spam-regulations)を遵守する責任があります。
{% endalert %}

## ユーザー購読によるセグメント化 {#segmenting-by-user-subscriptions}

「メールサブスクリプションステータス」と「プッシュサブスクリプションステータス」のフィルターを使用して、サブスクリプションステータスでユーザーをセグメント化します。

これを使用して、オプトインもオプトアウトもしていないユーザーをターゲットにし、明示的なオプトインを促します。「メール/プッシュサブスクリプションステータスが購読済み」というフィルターでセグメントを作成し、購読しているがオプトインしていないユーザーにキャンペーンを送信します。

![セグメントフィルターとして使用されるメール購読ステータス。]({% image_buster /assets/img_archive/not_optin.png %})
