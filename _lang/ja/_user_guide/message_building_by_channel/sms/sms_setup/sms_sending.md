# SMSメッセージ送信

> メッセージングは複雑になりがちですが、必ずしも複雑である必要はありません。以下のセクションでは、サブスクリプショングループの重要性、SMSセグメントとメッセージ本文の要件、利用可能な高度なカスタマイズオプションなど、BrazeでのSMSメッセージ送信の基本をリストアップします。

## SMS送信の基本

### サブスクリプション グループを選択する

SMS メッセージは、 [サブスクリプション グループ]({{site.baseurl}}/user_guide/onboarding_with_braze/sms_setup/sms_subscription_groups/)から送信する必要があります。サブスクリプション グループは、特定の種類のメッセージング目的で使用される送信電話番号 (ショート コード、ロング コード、英数字の送信者 ID など) のコレクションです。サブスクリプション グループを指定して、サブスクライブしたユーザーのみが対象になるようにする必要があります。一部のクライアントは、トランザクションSMSメッセージングやプロモーションSMSメッセージングなど、さまざまなユースケースに対して複数のサブスクリプショングループを持っていることに気付く場合があります。<br><br>

### 入力メッセージ本文

SMSメッセージ本文は、絵文字、Liquid、コネクテッドコンテンツを含む最大1,600文字を受け入れます。1 回のキャンペーン送信で、多数のメッセージセグメントが送信される可能性があります。Braze SMSメッセージ本文は、 [GSM-7](https://en.wikipedia.org/wiki/GSM_03.38) または [UCS-2](https://en.wikipedia.org/wiki/Universal_Coded_Character_Set) のいずれかのエンコード規格で構成できます。UCS-2 文字 (絵文字など) が使用されている場合、メッセージ本文は自動的にそのエンコード規格に合わせてフォーマットされます。<br><br> 

### メッセージ・セグメントと文字数制限の理解

SMSメッセージセグメントは、SMS業界がメッセージをカウントする方法です。メッセージ・セグメントは、1 回の SMS ディスパッチで送信される、定義された文字数 (GSM-7 エンコードの場合は 160、UCS-2 エンコードの場合は 67) のグループです。GSM-7 エンコードを使用して 161 文字の SMS をディスパッチすると、送信されたメッセージ セグメントが 2 つあることがわかります。複数のメッセージセグメントを送信すると、追加料金が発生する場合があります。<br><br>

### キーワードのカスタマイズ(オプション)

規制により、すべてのオプトイン、オプトアウト、およびヘルプ/情報SMSキーワードの応答に対する応答が必要です。Brazeでは、オプトイン、オプトアウト、ヘルプの応答をトリガーする独自のキーワードを定義したり、ユーザーに送信される独自の応答を管理したり、さまざまな言語のキーワードセットを定義したりできます。詳細については、 [キーワード処理]({{site.baseurl}}/user_guide/message_building_by_channel/sms/keywords/)に関するコレクションを参照してください。

## 送信のベストプラクティス {#sending-best-practices}

### 多国間SMS送信

一部のブランドは、さまざまな国の電話番号を持つユーザーのグループに送信したい場合があります。特定の国の電話番号にSMSメッセージを送信するには、同じ国からのロングコードまたはショートコードを使用するのがベストプラクティスです。実際、ショートコードは、ショートコードが作成されたのと同じ国の電話番号にのみSMSを送信できます。 

この制限を克服するために、サブスクリプション グループ [セットアップ プロセス][5] 中に、複数の異なる国からのロング コードとショート コードを保持するようにグループを設定できます。完了すると、ターゲットユーザーの電話番号と同じ国番号の電話番号を送信すると、キャンペーンの開始時に自動的に使用されます。国コードが異なる電話番号を持つユーザーに対して個別のキャンペーンを作成する必要はなく、1つのキャンペーンを開始したり、1つのキャンバスコンポーネントを使用して関連するユーザーをターゲットにしたりできます。

![SMSペイロードは、ターゲットユーザーの電話番号と同じ国コードを使用して送信されます][2]

#### おすすめの方法

1. **許可を得ます**。SMSをビジネスとして使用するための最も重要なルールの1つは、最初に顧客から連絡の許可を得る必要があることです。これを怠ると、ブランドが損なわれ、多額の弁護士費用が発生する可能性があります。<br><br>
2. **ユースケースに適した番号を選択してください**。SMS メッセージを送受信できる電話番号には、主にロングコード、ショートコード、英数字の送信者 ID の 3 種類があり、その機能と可用性は地域によって異なります。あなたのビジネスがバニティコードでより良く提供されるかどうかを事前に考えてください。<br><br>
3. **タイミングに注意してください**。顧客は、自分に直接宛てられた資料に対してより敏感であることを念頭に置いてください。ちょっとしたパーソナライゼーションは、受信者のファーストネームを使用したり、顧客の興味を反映した会話のタッチを追加したりするなど、大いに役立ちます。<br><br>
4. **双方向の会話に参加します**。SMSは顧客と関わるための非常に効果的なチャネルであるため、メッセージに対する応答を予測し、効果的に処理することが重要です。消費者の85%は、情報を受け取るだけでなく、企業に返信したり、会話に参加したりできることを望んでいます。<br><br>
5. **何が効果的かを測定します**。適切なタイミングで、最適な頻度で、最も効果的な行動喚起を使用して顧客にリーチしていますか?適切な追跡ツールを使用することで、ROIを証明する直接的で測定可能な指標を提供できます。 

### 大量送信

大量送信を計画していますか?スムーズに実行するためのベストプラクティスがいくつかあります。

- 必要に応じて、ターゲットオーディエンスの規模に基づいて、キャンペーン/キャンバスの配信速度制限を調整します。これにより、必要な送信量に達し、BrazeはTwilioが期待し、処理できる速度でメッセージを送信します。
- 160 文字の制限を守り、特殊文字の二重カウント(スラッシュ `\`、キャレット `^`、チルダ `~`など)に注意してください。
