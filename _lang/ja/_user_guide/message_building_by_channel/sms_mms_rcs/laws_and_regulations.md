# SMS、MMS、RCSに関する法律、規制、不正使用防止について

> SMS、MMS、RCSメッセージは、顧客やユーザーにリーチする最も直接的な方法のひとつであり、ユーザーの携帯電話に直接届くため、ブランドがこの関係を乱用したり、使いすぎたりしないような規制が存在しなければならない。 

{% alert warning %}
本条は、法律上の助言を提供することを意図したものではなく、また、それに依拠することもできない。SMS、MMS、RCS の使用は、特定の法的要件の対象となります。SMS、MMS、およびRCSの各サービスを、適用されるすべての法律に従って確実に利用するため、法律顧問の助言を求めること。
{% endalert %}

## コンプライアンスを正すための6つのルール

一般に、SMS、MMS、RCS メッセージを送信する際には、にアプリ到達する場合は、各自で最善の判断をすることをお勧めします。Braze と送信パートナーは、SMS、MMS、RCS の不正使用を防止するためのチェックを実施しています。

以下の6 つのルールに従う必要があります。

1. **SMS、MMC、RCS を送信する前に、ユーザーから明示的な同意を得ます。**ユーザーが同意を提供する場合は、必ず、準拠したユーザーデータ基盤に記録し、更新し、維持する必要があります。基本的な法的ガイドラインによれば、同意に関して保持する必要がある最も重要な情報は、以下のとおりです。
  - ユーザーが同意した日時
  - 同意したSMS、MMS、RCSメッセージングの種類
  - ユーザーの電話番号
  - 彼らが選んだ言語<br><br>

2. **送信するSMS、MMS、RCSの種類を明確に伝える**。ユーザーは、このチャネルで御社ブランドからどのようなメッセージが期待されるか、どのような情報やオファーを受け取るかを理解する必要があります。将来のキャンペーンの目的、メッセージの頻度を明示的に提示し、メッセージ / データ料金が適用されるという注意をユーザーに与えます。<br><br>

3. **必要不可欠な情報を継続的に更新し、表示します。**ブランドの利用規約およびSMS/MMS/RCSマーケティングプライバシーポリシーの最新バージョンが明確に表示され、オプトインページから簡単にアクセスできることを確認します。<br><br>

4. **合法的に取得したオプトイン電話番号のみに SMS、MMS、RCS を送信します**。技術的な移行計画の一環として、オプトインステータスをカスタマーエンゲージメントプラットフォーム内の各ユーザープロファイルに関連付けるメカニズムを、チームに確実に理解させます。<br><br>

5. **米国およびその他の関連地域では、確実に SHAFT に関する規制を遵守します。**セックス、ヘイト、アルコール、銃器、タバコ（SHAFT）にまつわる言葉を含むSMS、MMS、RCSメッセージの送信は、米国や他の一部の地域では一般的に違法とされている。<br><br>

6. **すべての事項をダブルチェックします。**貴社のSMS、MMS、RCSプログラムが、貴社のブランドが事業を展開する地域に適用されるすべての規則や規制に完全に準拠していることを確認するために、貴社の法務チームと協力する。<br><br>

## RCSコンプライアンスのベストプラクティス

{% alert note %}
RCS は新しく現在進化している分野です。州、国、その他の規制機関が新たな法規制または異なる法規制を導入する可能性があります。以下の情報は、一般的なガイダンスおよび教育目的で提供されています。ブランドは RCS メッセージを送信する前に、必ず法務チームに相談してください。
{% endalert %}

- **規制の重複を理解する:**RCS と SMS には多くの共通点があるため、同じ法律、規制、通信事業者ポリシーが適用されます。これには、オプトインを収集するための TCPA 規則や CTIA ガイドライン (SHAFT など) が含まれます。
- **移行に同意する:**業界のガイダンスによると、**同じ種類のメッセージング（販促や取引など）について**、ユーザーがその種類のメッセージングを受信するオプトインをすでに提供している場合、同意はSMSとRCS間で移行する。逆に、ユーザーがプロモーションSMSの受信をオプトインしていない場合、そのタイプのメッセージングに対するオプトインを収集せずにプロモーションRCSの送信を開始すべきではない。
- **ポリシーと免責事項を更新する:**モバイル利用規約& プライバシーポリシーを更新してRCSリファレンスを含めることをお勧めします。また、オプトインキャンペーンには、SMS に加えて RCS に関する文言も含めることが推奨されます。

## コンプライアンスへの配慮

### データとプライバシー

顧客のプライバシーは、有意義でお互いを尊重する関係を築くうえで重要です。顧客のプライバシーと情報を尊重することは、顧客と自社ブランドとの絆を生み出す、単なる機会の 1 つです。マーケティングツールを使用すると、データやプライバシーが残ることがあります。

Braze は [GDPR]({{site.baseurl}}/dp-technical-assistance/) をはじめと多くの[セキュリティ規制]({{site.baseurl}}/developer_guide/disclosures/security_qualifications/#security-qualifications)のガイドラインに従っています。

[CTIA](https://www.ctia.org/)(米国の無線通信業界を代表する業界団体)は、明確でわかりやすいプライバシーポリシーを維持し、目立つように表示することを推奨しています。

### 同意

オプトイン、ヘルプ、オプトアウトオプションは、SMS、MMS、RCS キャンペーンを作成する際には必須です。

電話消費者保護法[（TCPA](https://en.wikipedia.org/wiki/Telephone_Consumer_Protection_Act_of_1991)）は、企業が顧客にメッセージを送る場合、「書面による明示的な同意」を得ることを義務付けている。顧客との通信に SMS をどのように使用するかについて、顧客の承認を受ける必要があります。

[National Do Not Call Registry](https://www.donotcall.gov/)に準拠していることを忘れないでください。

Braze は、[サブスクリプショングループ]({{site.baseurl}}/user_guide/message_building_by_channel/sms_mms_rcs/keywords/) を使用して、同意の度合いに基づいてユーザーのグループを管理します。

### スパムとケイデンス

メールと同様に、ユーザーや顧客で受信トレイのバーンアウトが発生することがあります。しかし、これは、顧客にメッセージを過度に送信しない唯一の理由です。具体的には、コンプライアンス確保のために [FTC 法第 5 条](https://www.federalreserve.gov/boarddocs/supmanual/cch/ftca.pdf)を確認する必要がする必要あります (U.S の場合)。

スパムの考慮事項の中には、Braze レート制限sだけでなく、SMS機能全般(長短コード送信制限)に組み込まれているものもあります。ただし、キャンペーンを計画する際には、遵守すべき法令を考慮する必要があります。

### コンテンツ

効果的なトピックに思えるかもしれませんが、疑わしいときには暴力、性、ドラッグ、タバコ、その他の用品に関するトピックを避けます。これらのトピックに関するメッセージを送信する場合は、さまざまな通信事業者によってブロック送信されるメッセージに対して課金されることがあります。

[CTIA](https://www.ctia.org/)では、以下のトピックを一般的に"illegal"を定義するSHAFT 準拠を確実にすることを推奨しています(米国でメッセージングする場合)。

- 性別
- ヘイト
- アルコール
- 銃器
- タバコ

このトピックの詳細については、CTIA の[Messaging Principles and Best Practices for 2019](https://api.ctia.org/wp-content/uploads/2019/07/190719-CTIA-Messaging-Principles-and-Best-Practices-FINAL.pdf) を参照してください。

### スケジュール

遅い時間帯にメッセージを送信しないように指示する[TCPA](https://en.wikipedia.org/wiki/telephone_consumer_protection_act_of_1991)に準拠していることを確認します。正確な時間については、規制の内容を参照してください。ただ、とにかく遅い時間にはメールを送るべきではありません-高エンゲージメントを望んでいませんか?

### 国際

これらのベストプラクティスのほとんどは、アメリカ合衆国で規定されている指針に対応しています。U.S. 地域以外で顧客にリーチする場合は、その地域のベストプラクティスと法令を調査してください。米国、カナダ、欧州連合の一部では、通常アプリに存在する最も厳しい規則を順守する方法で行動することは、いつもベストプラクティスです。
