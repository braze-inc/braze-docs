# RCSメッセージを作成する

> RCS キャンペーンは、顧客に直接リーチしてプログラムを使って会話を行うのに適しています。Liquid などのダイナミックなコンテンツを使用して、ユーザー一人ひとりに合わせた体験を作り出し、控えめなブランド体験を強化する環境を作ることができます。

## RCSメッセージを作成する

### ステップ 1:メッセージを作成する場所を選択する

メッセージは、キャンペーンとキャンバスのどちらを使用して配信すべきでしょうか。キャンペーンは単一のターゲットを絞ったメッセージングキャンペーンに適しています。一方、キャンバスは複数ステップのユーザージャーニーに適しています。

{% tabs %}
{% tab Campaign %}
1. [**メッセージング**] > [**キャンペーン**] の順に進み、[**キャンペーンを作成**] を選択します。
2. **SMS/MMS/RCS** を選択するか、複数のチャネルを対象とするキャンペーンの場合は**マルチチャネル**を選択します。
3. キャンペーンに、明確で意味のある名前を付けます。
4. 必要に応じて[チーム]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/teams/)や[タグ]({{site.baseurl}}/user_guide/administrative/app_settings/tags/)を追加します。
   * タグを使用すると、キャンペーンを検索してレポートを作成しやすくなります。例えば、[レポートビルダー]({{site.baseurl}}/user_guide/analytics/reporting/report_builder/)を使用する場合、特定のタグでフィルターできます。

{: start="5"} 
5. キャンペーンに必要な数だけバリアントを追加して名前を付けます。追加したバリアントごとに、さまざまなプラットフォーム、メッセージタイプ、レイアウトを選択できます。このトピックの詳細については、「[多変量テストと AB テスト]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/)」を参照してください。
- **SMS と RCS のバリアントテスト**：Braze では、1つのキャンペーンに SMS と RCS の両方のバリアントを含めることができ、それぞれのパフォーマンスを比較できます。メッセージ作成の最初のステップで SMS と RCS のバリアントを追加できます。

{: start="6"} 
6. RCS 対応の[サブスクリプショングループ]({{site.baseurl}}/sms_rcs_subscription_groups/)を選択します。サブスクリプショングループを選択すると、Braze によって自動的にセグメンテーションフィルターが追加され、配信登録済みのユーザーのみがキャンペーンを受信できるようになります。そのサブスクリプショングループに属するロングコードとショートコードのみを使用して、対象ユーザーに SMS が送信されます。
- **SMS フォールバック**：RCS 送信者を含む各サブスクリプショングループには、フォールバック用に少なくとも1つの SMS コードも含めることを Braze は強く推奨しています。これは、RCS メッセージの配信に失敗した場合の到達性を確保するうえで重要です。その理由としては、ユーザーデバイスの互換性がないことや、特定の国や地域におけるキャリアのカバー範囲が不完全であることなどが考えられます。SMS フォールバックを有効にすると、RCS で配信できない場合でも SMS を通じてメッセージを配信できるため、ユーザーとつながる機会を逃すことがなくなります。

{% alert note %}
MMS フォールバックはサポートされていません。
{% endalert %}

{: start="7"}
7. SMS と RCS のいずれかを選択します。RCS メッセージを作成する前に、送信に使用するチャネルを選択します。RCS にはユーザーエンゲージメントの面で SMS よりも大きな利点があるため、可能な限り RCS を使用することを一般に推奨しています。ただし、最大限の柔軟性とコントロールを得ることができるように、SMS で送信するオプションも常に提供しています。 

![RCS または SMS/MMS メッセージタイプから選択するオプション。]({% image_buster /assets/img/rcs/rcs_message_type.png %}){: style="max-width:65%;"}

{% alert tip %}
キャンペーン内のすべてのメッセージが類似しているか、同じ内容になる場合は、メッセージを作成してからバリアントを追加します。その後、[**バリアントを追加**] ドロップダウンから [**バリアントをコピー**] を選択できます。
{% endalert %}

{% endtab %}
{% tab Canvas %}
1. キャンバス作成ツールを使用して[キャンバスを作成]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas/)します。
2. キャンバスを設定したら、キャンバスビルダーに **SMS/MMS/RCS** メッセージステップを追加します。 
3. ステップに、明確で意味のある名前を付けます。
4. RCS 対応の[サブスクリプショングループ]({{site.baseurl}}/sms_rcs_subscription_groups/)を選択します。サブスクリプショングループを選択すると、Braze によって自動的にセグメンテーションフィルターが追加され、配信登録済みのユーザーのみがキャンペーンを受信できるようになります。ユーザーをターゲット設定するには、そのサブスクリプショングループに属するロングコードとショートコードのみが使用されます。
- **SMS フォールバック**：RCS 送信者を含む各サブスクリプショングループには、フォールバック用に少なくとも1つの SMS コードも含めることを Braze は強く推奨しています。これは、RCS メッセージの配信に失敗した場合の到達性を確保するうえで重要です。その理由としては、ユーザーデバイスの互換性がないことや、特定の国や地域におけるキャリアのカバー範囲が不完全であることなどが考えられます。SMS フォールバックを有効にすると、メッセージが引き続きユーザーに配信され、ユーザーとつながる機会を逃すことがなくなります。

{: start="5"}
5. SMS と RCS のいずれかを選択します。RCS メッセージを作成する前に、送信に使用するチャネルを選択します。RCS にはユーザーエンゲージメントの面で SMS よりも大きな利点があるため、可能な限り RCS を使用することを一般に推奨しています。ただし、最大限の柔軟性とコントロールを得ることができるように、SMS で送信するオプションも常に提供しています。 

![RCS または SMS/MMS メッセージタイプから選択するオプション。]({% image_buster /assets/img/rcs/rcs_message_type.png %}){: style="max-width:65%;"}

### ステップ 2:RCS メッセージタイプを選択する

RCS メッセージタイプでは、[**テキスト**] と [**メディア**] のいずれかを選択します。

![テキストメッセージかメディアメッセージのタイプから選択するオプション。]({% image_buster /assets/img/rcs/rcs_text_media.png %}){: style="max-width:65%;"}

{% tabs %}
{% tab Text %}
RCS テキストメッセージは、その名前の通り、メディアとしてのテキストに焦点を当てています。入力した文字数が160文字以下の場合、RCS メッセージはテキストのみ (または「基本」) メッセージとして請求されます。160文字を超えるか、リッチ要素を使用した場合、メッセージはリッチ (または「シングル」) RCS メッセージとして請求されます (また、文字数制限は3072文字に増加します)。 

#### 特徴

- テキストメッセージタイプには、すべての SMS 機能が含まれています。URL クリックトラッキングでは、ユーザーレベルのレポート粒度を提供するために、高度なトラッキングのみが可能です。 
- さらに、ランディングページへの訪問や注文など、エンゲージメントの高いユーザーアクションを促す魅力的な**返信候補**ボタンや**アクション候補**ボタンを含めるオプションもあります。 
    - **返信候補**とは、ユーザーがクリックしてテキスト入力にあらかじめ入力される提案されたレスポンスを含むボタンのことで、制約のある選択肢を提供することで、ユーザーがレスポンスを考えなければならないという摩擦を取り除きます。 
    - **アクション候補**は、ユーザーのデバイス上でアクションを開始するボタンです。これらは通常、1つか2つの説明的な単語と、ユーザーがそのボタンの機能を理解するのに役立つ視覚的なアイコンで構成されています。Braze は現在、OpenURL でのアクション候補をサポートしています。これは URL と同じような機能で、ユーザーがボタンを選択すると、Web ページやその他の URL で特定される場所にリダイレクトされます。 

![トレンドのファッションスタイルを宣伝する RCS メッセージの3つのアクション候補の GIF：「Fairytale royalty」「Edgy academia」「Show me your other styles」。]({% image_buster /assets/img/rcs/rcs_suggested_actions.gif %}){: style="max-width:70%;"}

#### 考慮事項

- テキストの文字数制限は、テキストのみ (基本) の RCS メッセージでは160文字、リッチ (シングル) の RCS メッセージでは3072文字です。 
- ボタンの制限として、1つのメッセージに追加できるボタンは5つまでです。これらのボタンは、アクション候補または返信候補のいずれかになります。
- テキストブロックが長い場合や、ボタンが多すぎる場合は、ユーザーが苛立つことがあるため、可能な限りシンプルにすることをお勧めします。 
- 長いテキストのみのメッセージを送信する場合には、SMS を使用するよりも RCS を使用する方がコスト効率が良いことがあります。これは、長い SMS メッセージは複数のセグメントに分割され、各セグメントが請求対象となるのに対し、RCS メッセージはメッセージ単位で請求されるためです。詳細とガイダンスについては、Braze アカウントマネージャーにお問い合わせください。 
{% endtab %}

{% tab Media %}
RCS メディアメッセージでは、SMS では不可能な魅力的なメディアフォーマットを使用できます。これには画像、動画、ドキュメントファイルなどがあります。これらのメディアオプションは、オーディエンスのエンゲージメントをさらに深め、まったく新しいユースケースを実現するために用意されています。現時点では、[メディアライブラリー]({{site.baseurl}}/user_guide/engagement_tools/templates_and_media/media_library/)からの画像のアップロードのみがサポートされています。 

{% multi_lang_include alerts/important_alerts.md alert='dynamic image URL' %}

#### 機能

- メディアメッセージタイプでは、テキストメッセージタイプで利用可能な機能がすべてサポートされています。これには、テキスト、返信候補、アクション候補などが含まれます。
- JPEG や PNG などの画像ファイルをサポートしています。画像ファイルは、メディアライブラリーからアップロードすることで利用できます。 
- 動画ファイル (MP4、MPEG、MV4 など) をサポートしています。動画ファイルはメッセージ作成画面で URL を直接追加できます。 
- PDF 形式のドキュメントファイルをサポートしています。ドキュメントファイルはメッセージ作成画面で URL を使用して直接追加できます。 

![メディアファイルをアップロードするオプション付きの RCS コンポーザー。]({% image_buster /assets/img/rcs/rcs_media_type.png %})

#### ファイル仕様

| ファイルタイプ | 仕様 |
| --- | --- |
| すべて | - ファイルサイズは100 MBに制限されています <br><br>- ファイル URL は2048文字まで |
| 画像ファイル | サポートされている形式には JPG、JPEG、および GIF があります。
| 動画ファイル | サポートされているファイル形式は、H263、M4V、MP4、MPEG-4、MPEG、WEBM です。 |
| ドキュメントファイル | サポートされているファイル形式：PDF |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

#### 考慮事項

RCS メッセージ受信のユーザーエクスペリエンスは、送信先の国でのキャリアのカバレッジ、モバイルデバイスのハードウェア、モバイルデバイスのオペレーティングシステムなど、多くの要因によって若干異なる場合があります。 

一般的に、RCS は Android デバイスとより自然に統合します (この手法は主に Google により実装され、ピアツーピア RCS メッセージングは Android コミュニティで広く採用されています)。エクスペリエンスのレンダリングにかかる時間やレンダリングの品質は、デバイスによって異なる可能性があります。  
{% endtab %}

### ステップ 3:RCS メッセージを作成する

必要に応じて言語やパーソナライゼーション（[Liquid]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/)、[コネクテッドコンテンツ]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/connected_content/)、絵文字）を使ってメッセージを書きます。超過料金が発生する可能性を減らすため、メッセージのコピー数制限を必ず守ってください。

{% alert important %}
先に進む前に、[RCS メッセージの制限事項に関するガイドライン](#step-2-select-your-rcs-message-type)をお読みください。RCS メッセージは[メッセージ単位で課金される]({{site.baseurl}}/sms_rcs_billing_calculators/)ため、各タイプの RCS メッセージに含めることができる内容の細かな違いを理解しておくと良いでしょう。
{% endalert %}

### ステップ 4:メッセージをプレビューしてテストする

RCS の表示はユーザーのオペレーティングシステム、デバイスメーカー、通信事業者、メッセージングアプリ（例：Google Messages と Apple Messages）によって制御されるため、メッセージの表示が異なる場合があります。その結果、Braze に表示される RCS プレビューは、エンドユーザーが最終的に受け取る内容と完全に一致しない可能性があります。違いにはレイアウト、メディアサイズ、ボタン、ブランディング要素、またはサポートされている機能などが含まれます。Braze では、送信前にメッセージをプレビューしてテストすることを常に推奨しています。**テスト**タブを使って、テスト用 RCS をコンテンツテストグループや個々のユーザーに送信し、Braze 内で直接ユーザーとしてメッセージをプレビューできます。ただし、Braze はすべての OS、デバイス、キャリアの組み合わせにおいて完全な互換性を保証できないため、最終的な表示は可能な限り実機で検証してください。


### ステップ 5:キャンペーンまたはキャンバスの残りの部分を作成する

次にキャンペーンまたはキャンバスの残りの部分を作成します。RCS メッセージの作成に最適なツールの使い方については、以下のセクションを参照してください。

#### ステップ 5.1:配信スケジュールまたはトリガーを選択する

RCS メッセージは、スケジュールされた時間、アクション、または API トリガーに基づいて配信できます。詳細については、[キャンペーンのスケジューリング]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/)を参照してください。

アクションベースの配信では、キャンペーンの期間とクワイエットアワーを設定することもできます。

配信に関するコントロールを指定します（ユーザーがキャンペーンを再度受信できるようにすることや、フリークエンシーキャップルールを有効にすることなど）。

#### ステップ 5.2:ターゲットとするユーザーを選択する

セグメントまたはフィルターを選択してオーディエンスを絞り込みます。サブスクリプショングループはすでに選択済みのはずです。サブスクリプショングループは、ユーザーとのコミュニケーションのレベルやカテゴリーによってユーザーを絞り込みます。

{% multi_lang_include target_audiences.md %}

次に、セグメントの中からより大きなオーディエンスを選択し、オプションの[フィルター]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters/)でそのセグメントをさらに絞り込みます。そのおおよそのセグメント人口がどのようになるか、自動的にプレビューが表示されます。正確なセグメントのメンバーシップは、メッセージが送信される前に常に計算されることに留意してください。

{% alert tip %}
RCS リターゲティングを使用して、SMS や RCS でのやり取りに基づいてユーザーをターゲット設定することに関心がある場合は、[リターゲティング]({{site.baseurl}}/sms_mms_rcs_user_retargeting/)を参照してください。
{% endalert %}

#### ステップ 5.3:コンバージョンイベントを選択する

Braze では、キャンペーンを受け取ったユーザーが特定のアクション（コンバージョンイベント）を実行する頻度を追跡できます。ユーザーが指定されたアクションを取った場合にコンバージョンがカウントされる最大30日間の時間枠を設定できます。

コンバージョンイベントにより、キャンペーンの成否を測定できます。以下に例を示します。
- ジオターゲティングを使用して、ユーザーによる購入という最終目標を持つ RCS メッセージをトリガーする場合は、コンバージョンイベントを [**購入**] に設定します。
- ユーザーをアプリに誘導しようとしている場合は、コンバージョンイベントを [**セッションを開始**] に設定します。

独自のユースケースに基づいて、カスタムコンバージョンイベントを設定することもできます。キャンペーンの成功度を測る方法を工夫しましょう。

### ステップ 6:確認とデプロイ

キャンペーンやキャンバスを作り終えたら、詳細を確認し、テストし、送信します。

次に、「[SMS、MMS、RCS のレポート]({{site.baseurl}}/sms_mms_rcs_reporting/)」を参照して、RCS キャンペーンの結果にアクセスする方法を確認してください。

## ヒント

### メッセージのパーソナライゼーションに Liquid を使用する

Liquid を使用する場合は、使用するパーソナライゼーションのデフォルト値を必ず含めてください。これにより、受信者のユーザープロファイルが不完全な場合に、名前やまとまった文章の代わりに空白のプレースホルダー `Hi, !` を受信することがなくなります。

### AI コピーの生成

魅力的なコピーの作成でお困りですか？[AI コピーライティングアシスタント]({{site.baseurl}}/user_guide/brazeai/generative_ai/copywriting/)を使用してみてください。商品名や説明を入力すると、メッセージングに使用できる人間のようなマーケティングコピーが AI により生成されます。

![メッセージ作成画面に AI コピーライティングアシスタントを開くアイコンが付いている。]({% image_buster /assets/img/rcs/rcs_ai_copywriter.png %}){: style="max-width:70%;"}

### 会話型メッセージワークフローを作成する

会話型メッセージワークフローを使用すると、ユーザーに動的に応答し、双方向のメッセージング体験を作成できます。ワークフローを構築するには、キャンバスを作成し、返信候補と[アクションパス]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/action_paths/)を組み合わせて、ユーザーが選択した返信に基づいてワークフローを誘導します。

1. キャンバスビルダーで、複数の返信候補を含む RCS メッセージステップを作成します。

![返信候補付きの RCS メッセージコンポーザー。]({% image_buster /assets/img/rcs/suggested_replies.png %})

{: start="2"}
2. そのメッセージを、各返信候補に対応するアクショングループを持つアクションパスに接続します。
3. 各アクショングループで以下を設定します。
  - トリガーとして **SMS 受信メッセージを送信**を選択します。
  - メッセージ本文を、対応する返信候補と同じ内容に設定します。 

![各返信候補に対応する3つのアクショングループが設定されたアクションパスステップ。]({% image_buster /assets/img/rcs/quick_reply.png %})

{: start="4"}
4. 各アクショングループを RCS メッセージステップに接続し、関連する返信候補に基づいたコンテンツを追加します。
5. フォローアップメッセージに返信候補を追加して、会話型ワークフローを続けます。
6. ワークフローが完成するまで、ステップ2〜4を繰り返します。
