# RCSメッセージを作成する

> RCS キャンペーンは、顧客に直接リーチしてプログラムを使って会話を行うのに適しています。Liquid などのダイナミックコンテンツを使用して、ユーザー一人ひとりに合わせた体験を作り出し、控えめなブランド体験を強化する環境を作ることができます。

## RCSメッセージを作成する

### ステップ1:メッセージを作成する場所を選択する

メッセージは、キャンペーンとキャンバスのどちらを使用して配信すべきでしょうか。キャンペーンは単一のシンプルなメッセージングキャンペーンに適していますが、キャンバスはマルチステップのユーザーのジャーニーに適しています。

{% tabs %}
{% tab キャンペーン %}
1. [**メッセージング**] > [**キャンペーン**] の順に進み、[**キャンペーンを作成**] を選択します。
2. [**SMS/MMS/RCS**] を選択するか、複数のチャネルをターゲットとするキャンペーンの場合は [**マルチチャネル**] を選択します。
3. キャンペーンに、明確で意味のある名前を付けます。
4. 必要に応じて[チームや]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/teams/) [タグを]({{site.baseurl}}/user_guide/administrative/app_settings/tags/)追加します。
   * タグを使用すると、キャンペーンを検索してレポートを作成しやすくなります。例えば、[[レポートビルダー]({{site.baseurl}}/user_guide/analytics/reporting/report_builder/)] を使用する場合、特定のタグでフィルターできます。

{: start="5"}
5\.キャンペーンに必要な数だけバリアントを追加して名前を付けます。追加したバリアントごとに、さまざまなプラットフォーム、メッセージタイプ、レイアウトを選択できます。このトピックの詳細については、「[多変量テストと AB テスト]({{site.baseurl}}/user_guide/engagement_tools/testing/multivariant_testing/)」を参照してください。
- **SMSとRCSのバリアントテスト**：Braze では、1つのキャンペーンに SMS と RCS の両方のバリアントを含めることができます。これにより、SMS と ROSS のパフォーマンスを比較できます。メッセージ作成の最初のステップで SMS と RCS のバリアントを追加できます。

{: start="6"}
6. RCS 対応の[サブスクリプショングループ]({{site.baseurl}}/sms_rcs_subscription_groups/)を選択します。サブスクリプショングループを選択すると、Braze によって自動的にセグメンテーションフィルターが追加され、配信登録済みのユーザーのみがキャンペーンを受信できるようになります。そのサブスクリプショングループに属する長いコードと短いコードのみを使用して、対象ユーザーに SMS が送信されます。
- **SMS フォールバック**:RCS 送信者を含む各サブスクリプショングループには、フォールバック用に少なくとも1つの SMS コードも含めることが強く推奨されます。これは、RCS メッセージの配信に失敗した場合にも配信できるようにするうえで重要です。その理由としては、ユーザー・デバイスの互換性がないことや、特定の国や地域におけるキャリアのカバー範囲が不完全であることなどが考えられる。SMS フォールバックを有効にすると、メッセージが引き続きユーザーに送信され、ユーザーとつながる機会を逃すことがなくなります。   

{: start="7"}
7. SMS と RCS のいずれかを選択します。RCS メッセージを作成する前に、送信に使用するチャネルを選択します。RCS にはユーザーエンゲージメントの面で SMS よりも大きな利点があるため、可能な限り RCS を使用することを一般に推奨しています。ただし最大限の柔軟性とコントロールを得ることができるように、SMS で送信するオプションを常に提供しています。 

![RCSまたはSMS/MMSメッセージタイプから選択するオプション]({% image_buster /assets/img/rcs/rcs_message_type.png %}){: style="max-width:65%;"}

{% alert tip %}
キャンペーン内のすべてのメッセージが類似しているか、同じ内容になる場合は、メッセージを作成してからバリアントを追加します。その後、[**バリアントを追加**] ドロップダウンから [**バリアントをコピー**] を選択できます。
{% endalert %}

{% endtab %}
{% tab キャンバス %}
1. キャンバス作成ツールを使用して [[キャンバスを作成]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas/)] します。
2. キャンバスを設定したら、キャンバスビルダーに **SMS/MMS/RCS**メッセージングステップを追加する。 
3. ステップに、明確で意味のある名前を付けます。
4. RCS 対応の[サブスクリプショングループ]({{site.baseurl}}/sms_rcs_subscription_groups/)を選択します。サブスクリプショングループを選択すると、Braze によって自動的にセグメンテーションフィルターが追加され、配信登録済みのユーザーのみがキャンペーンを受信できるようになります。ユーザーをターゲット設定するには、そのサブスクリプショングループに属する長いコードと短いコードのみが使用されます。
- **SMS フォールバック**:RCS 送信者を含む各サブスクリプショングループには、フォールバック用に少なくとも1つの SMS コードも含めることが強く推奨されます。これは、RCS メッセージの配信に失敗した場合にも配信できるようにするうえで重要です。その理由としては、ユーザー・デバイスの互換性がないことや、特定の国や地域におけるキャリアのカバー範囲が不完全であることなどが考えられる。SMS フォールバックを有効にすると、メッセージが引き続きユーザーに送信され、ユーザーとつながる機会を逃すことがなくなります。

{: start="5"}
5\.SMS と RCS のいずれかを選択します。RCS メッセージを作成する前に、送信に使用するチャネルを選択します。RCS にはユーザーエンゲージメントの面で SMS よりも大きな利点があるため、可能な限り RCS を使用することを一般に推奨しています。ただし最大限の柔軟性とコントロールを得ることができるように、SMS で送信するオプションを常に提供しています。

### ステップ2: RCSメッセージタイプを選択する

RCS メッセージタイプでは、[**テキスト**] と [**メディア**] のいずれかを選択します。

![メッセージタイプとして [テキスト] または [メディア] を選択するオプション。]({% image_buster /assets/img/rcs/rcs_text_media.png %}){: style="max-width:65%;"}

{% tabs %}
{% tab テキスト %}
RCS テキストメッセージは、その名前の通り、メディアとしてのテキストに焦点を当てています。入力した文字数が160文字以下の場合、RCS メッセージはテキストのみ (または「基本」) として請求されます。160文字を超えるか、リッチ要素を使用した場合、メッセージはリッチ (または「シングル」) RCS メッセージとして請求されます (また、文字数制限は3072文字に増加されます)。 

#### 特徴

- テキストメッセージタイプには、すべての SMS 機能が含まれています。URL クリック追跡では、ユーザーレベルのレポート粒度を提供するために、高度なトラッキングのみが可能です。 
- さらに、ランディングページへの訪問や注文など、エンゲージメントの高いユーザーアクションを促す魅力的な**「返信先提案**」ボタンや「**アクション提案」**ボタンを含めるオプションも追加された。 
    - **サジェストされた返信先とは**、ユーザーがクリックし、テキスト入力にあらかじめ入力されるよう提案されたレスポンスを含むボタンのことで、制約のある選択肢を提供することで、ユーザーがレスポンスを考えなければならないという摩擦を取り除く。 
    - **おすすめのアクション**は、ユーザーのデバイス上でアクションを開始するボタンです。これらは通常、1つか2つの説明的な単語と、ユーザーがそのボタンが何をするのかを理解するのに役立つ視覚的なアイコンで構成されている。Braze は現在、OpenURL でのおすすめのアクションをサポートしています。これは URL と同じような機能で、ユーザーがボタンを選択すると、Web ページやその他の URL で特定される場所にリダイレクトされます。 

![トレンドのファッションスタイルを宣伝するRCSメッセージのための3つのアクションの提案のGIF：「Fairytale royalty」、「Edgy academia」、「Show me your other styles」。]({% image_buster /assets/img/rcs/rcs_suggested_actions.gif %}){: style="max-width:70%;"}

#### 考慮事項

- テキストの文字数制限は、テキストのみ (基本) の RCS メッセージでは160文字、リッチ (シングル) の RCS メッセージでは3072文字です。 
- ボタンの制限として、1つのメッセージに追加できるボタンは5つまでです。これらのボタンは、提案されたアクションまたは提案された返信先のいずれかになる。
- テキストブロックが長い場合や、ボタンが多すぎる場合は、ユーザーが苛立つことがあるため、可能な限りシンプルにすることをお勧めします。 
- 長いテキストのみのメッセージを送信する場合には、SMS を使用するよりも RCS を使用する方がコスト効率が良いことがあります。これは、長い SMS メッセージは複数のセグメントに分割され、各セグメントが請求対象となるのに対し、RCS メッセージはメッセージ単位で請求されるためです。詳細とガイダンスについては、Braze アカウントマネージャーにお問い合わせください。
{% endtab %}

{% tab メディア %}
RCS メディアメッセージでは、SMS では不可能な魅力的なメディアフォーマットを使用できます。これには画像、動画、ドキュメントファイルなどがあります。これらのメディアオプションは、オーディエンスのエンゲージメントをさらに深め、まったく新しいユースケースを実現できるようにします。現時点では、[メディアライブラリ]({{site.baseurl}}/user_guide/engagement_tools/templates_and_media/media_library/)からの画像のアップロードのみがサポートされています。 

#### 特徴

- メディアメッセージタイプでは、テキストメッセージタイプで利用可能な機能がすべてサポートされています。。これには、テキスト、おすすめの返信、おすすめのアクションなどが含まれます。
- JPEG や PNG などの画像ファイルをサポートしています。画像ファイルは、メディアライブラリからアップロードすることで利用できます。 
- 動画ファイル (MP4、MPEG、MV4 など) をサポートしています。動画ファイルはメッセージ作成画面で URL で直接追加できます。 
- PDF 形式のドキュメントファイルをサポートしています。ドキュメントファイルはメッセージ作成画面で URL を使用して直接追加できます。 

![メディアファイルをアップロードするオプションがある RCS 作成画面。]({% image_buster /assets/img/rcs/rcs_media_type.png %})

#### ファイル仕様

| ファイルタイプ | 仕様 |
| --- | --- |
| すべて | \- ファイルサイズは100MBに制限されている <br><br>\- ファイルURLは2048文字まで |
| 画像ファイル | サポートされている形式には JPG、JPEG、および GIF があります。
| 動画ファイル | サポートされているファイル形式は、H263、M4V、MP4、MPEG-4、MPEG、WEBMである。 |
| ドキュメントファイル | 対応ファイル形式PDF |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

#### 考慮事項

RCSメッセージ受信のユーザーエクスペリエンスは、送信先の通信事業者のカバレッジ、モバイルデバイスのハードウェア、モバイルデバイスのオペレーティングシステムなど、多くの要因によって若干異なる場合がある。 

一般的に、RCS は Android デバイスとより自然に統合します (この手法は主に Google により実装され、ピアツーピア RCS メッセージングは Android コミュニティで広く採用されています)。エクスペリエンスのレンダリングにかかる時間やレンダリングの品質は、デバイスによって異なる可能性があります。  
{% endtab %}

### ステップ 3:RCS メッセージを作成する

必要に応じて言語やパーソナライゼーション[（リキッド]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/liquid/)、[コネクテッドコンテンツ]({{site.baseurl}}/user_guide/personalization_and_dynamic_content/connected_content/)、絵文字）を使ってメッセージを書く。超過料金を請求される可能性を減らすため、メッセージのコピー数制限を必ず守ってください。

{% alert important %}
先に進む前に、[RCS メッセージの制限事項に関するガイドライン](#step-2-select-your-rcs-message-type)をお読みください。RCS メッセージは[メッセージ単位で課金される]({{site.baseurl}}/sms_rcs_billing_calculators/)ため、各タイプの RCS メッセージに含めることができる内容の細かな違いを理解しておくと良いでしょう。
{% endalert %}

### ステップ4:メッセージをプレビューしてテストする

Brazeでは、送信前にメッセージをプレビューしてテストすることを常に推奨しています。**テスト**タブに移動して、コンテンツテストグループまたは個々のユーザーにテストRCSを送信するか、Brazeで直接ユーザーとしてメッセージをプレビューする。

### ステップ 5: キャンペーンまたはキャンバスの残りの部分を作成する

次にキャンペーンまたはキャンバスの残りの部分を作成します。RCSメッセージの作成に最適なツールの使い方については、以下のセクションを参照のこと。

#### ステップ 5.1:配信スケジュールまたはトリガーを選択する

RCSメッセージは、スケジュールされた時間、アクション、またはAPIトリガーに基づいて配信することができる。詳細については、[キャンペーンのスケジューリング]({{site.baseurl}}/user_guide/engagement_tools/campaigns/building_campaigns/delivery_types/)を参照してください。

アクションベースの配信では、キャンペーンの期間とクワイエットアワーを設定することもできる。

配信に関するコントロールを指定します (ユーザーがキャンペーンを再度受信できるようにすることや、フリークエンシーキャップルールを有効にすることなど)。

#### ステップ 5.2:ターゲットとするユーザーを選択する

セグメントまたはフィルタを選択してユーザーを絞り込みます。サブスクリプショングループをすでに選択したはずである。サブスクリプショングループは、ユーザーとのコミュニケーションのレベルやカテゴリーによってユーザーを絞り込む。 

次に、セグメントから大きなオーディエンスを選択し、[フィルター]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters/)でセグメントをさらに絞り込みます。セグメントのおおよその人数について現在の状態を示すスナップショットが自動的に表示されます。正確なセグメントメンバーシップは常にメッセージが送信される直前に計算されることに注意してください。

{% alert tip %}
RCS リターゲティングを使用して、SMS や RCS でのやり取りに基づいてユーザーをターゲト設定することに関心がある場合には、[リターゲティング]({{site.baseurl}}/sms_mms_rcs_user_retargeting/)を参照してください。
{% endalert %}

#### ステップ 5.3:コンバージョンイベントを選択する

Brazeでは、キャンペーンを受け取ったユーザーが特定のアクション（コンバージョンイベント）を実行する頻度を追跡することができる。ユーザーが指定されたアクションを取った場合、コンバージョンがカウントされる最大30日間のウィンドウを許可することができる。

コンバージョンイベントにより、キャンペーンの成否を測定できます。以下に例を示します。
- ジオターゲティングを使用して、ユーザーによる購入という最終目標を持つ SMS メッセージをトリガーする場合は、コンバージョンイベントを [**購入**] に設定します。
- ユーザーをアプリに誘導しようとしている場合は、コンバージョンイベントを[**セッションを開始**] に設定します。

独自のユースケースに基づいて、カスタムコンバージョンイベントを設定することもできます。キャンペーンの成功度を測る方法を工夫しよう。

### ステップ 6: レビューと展開

キャンペーンやキャンバスを作り終えたら、詳細を確認し、テストし、送信する！

次に、「[SMS、MMS、RCS のレポートの作成]({{site.baseurl}}/sms_mms_rcs_reporting/)」を参照して、RCS キャンペーンの結果にアクセスする方法を理解してください。

## ヒント

### メッセージのパーソナライゼーションに Liquid を使用する

Liquid を使用する場合は、使用するパーソナライゼーションのデフォルト値を必ず含めてください。これにより、受信者のユーザープロファイルが不完全な場合に、名前やまとまった文章の代わりに空白のプレースホルダー `Hi, !` を受信することがありません。

### AI コピーの生成

魅力的なコピーの作成でお困りですか?[AI コピーライティングアシスタント]({{site.baseurl}}/user_guide/brazeai/generative_ai/copywriting/)を使用してみてください。商品名や説明を入力すると、メッセージングに使用できる人間のようなマーケティングコピーが AI により生成されます。

![AIコピーライティングアシスタントを開封するアイコンが付いたメッセージ作成画面]({% image_buster /assets/img/rcs/rcs_ai_copywriter.png %}){: style="max-width:70%;"}
