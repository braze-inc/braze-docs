# アンインストール追跡

Braze のアンインストール追跡では、以下の情報が提供されます。details:

1. アプリレベルの日次アンインストール統計情報: [**ホーム**] ページの時系列グラフに表示されます。
2. キャンペーンレベルのアンインストール統計情報: 特定キャンペーンの [**キャンペーンの詳細**] ページの時系列グラフに表示されます。この統計情報は、アンインストールしたキャンペーン受信者の日次数を示します。

アンインストール数の集計を時系列で表示すると、傾向や異常を視覚化しやすくなり、アプリのアンインストール数を容易に監視できます。同様に、キャンペーンレベルでアンインストール数を追跡することで、特定のキャンペーンがアプリのアンインストールを促進しているのか、阻害しているのかがわかります。

Braze では、通常のプッシュキャンペーンから基本レベルのアンインストール情報を自動的に収集します。ただし、プッシュキャンペーンの受信頻度はユーザーによって異なる場合があるため、Braze ではユーザー群のアンインストールアクティビティのより正確なスナップショットを提供するためにアンインストール追跡を用意しています。

{% alert note %} ダッシュボードでアンインストール追跡にオプトインする必要があります。この機能は現在、iOS、Android、および Fire OS のアプリで使用できます。 {% endalert %}

## 実装

アンインストール追跡は、追跡するアプリごとに、[**アプリ設定**] ページの [**設定**] で有効にできます。

{% alert note %}
古いナビゲーションを使用している場合、[**アプリ設定**] は [**設定**] であり、[**設定の管理**] の下にあります。
{% endalert %}

アプリのアンインストール追跡が有効になっている場合、セッションを記録していないユーザーや 24 時間以内にプッシュを受信していないユーザーには、バックグラウンドプッシュメッセージが毎晩送信されます。 

iOS で Braze のバックグラウンドプッシュにフィルターを適用する場合は、[ユーティリティメソッド][iOS ドキュメント]を使用できます。Android では、 [\`BrazeNotificationUtils.isUninstallTrackingPush()\`][8] を使用してアンインストールプッシュを検出できます。Braze がアンインストール追跡または通常のプッシュキャンペーン配信によるアンインストールを検出すると、ユーザーがアンインストールしたと見なした最良の推定時間を記録します。この時間は、標準属性項目としてユーザープロファイルに保存されます。

この時間を使用して、奪還キャンペーンのターゲットにするユーザーセグメントを定義できます。[**セグメント**] ページの「アンインストール済み」フィルターを使用すると、一定期間内にアプリをアンインストールしたユーザーを選択できます。アンインストールの正確な時間を判定することは難しいため、アンインストールしたすべてのユーザーがある時点でそのセグメントに確実に含まれるように、アンインストールフィルターの期間を広げることをお勧めします。

## アプリレベルの分析

アンインストールに関する日次統計は [**ホーム**] ページで確認できます。Braze が提供する他の統計情報と同様に、セグメント別の内訳を可視化できます。[**次に関する統計を表示**] を [利用状況分析] に設定し、ドロップダウンから [アンインストール数] を選択してグラフを表示します。

その後、ドロップダウンを使用してセグメント別およびアプリ別の内訳のグラフを表示できます。

{% alert note %}
アンインストール追跡が有効になっていないアプリでは、一部のユーザー (プッシュ通知対象のユーザー) のアンインストール数のみが報告されるため、日次アンインストール総数は表示されている数より多くなる可能性があります。
{% endalert %}

## キャンペーンのアンインストール追跡

キャンペーンのアンインストール追跡では、特定のキャンペーンの受信後に、選択した期間内にアプリをアンインストールしたユーザーの数を確認できます。このツールにより、マーケターは、キャンペーンがユーザーの意図しないネガティブな行動をどのように助長しているかというインサイトを得ることができ、キャンペーン全体の効果を測定するうえでに役立ちます。

Braze は、ユーザーのデバイスに送信されたプッシュメッセージが Firebase Cloud Messaging (FCM) または Apple プッシュ通知サービス (APN) から、アプリがもうインストールされていないというシグナルを返すタイミングを観察することで、アンインストールを追跡します。特定のアプリでグローバルなアンインストール追跡が有効になっている場合、Braze はユーザーに毎日サイレントプッシュメッセージを送信して、ユーザーがアンインストールしたかどうかを検出します。この「サイレント」プッシュはすべてのユーザーに送信されますが (ユーザーがアプリ設定でサイレントプッシュを無効にしている場合を除く)、ユーザーには表示されません。Braze がユーザーのアンインストールを検出した場合、そのプラットフォームは次の処理を行います。

* アプリのアンインストール総数を 1 増加します。
* ユーザーが過去 24 時間に正常に受信したすべてのキャンペーンのアンインストール数を 1 ずつ増加します。
* ユーザーが 24 時間に 3 つのキャンペーンを受信した後にアンインストールした場合、3 つのキャンペーンすべての「アンインストール」の数を増加します。

アンインストール追跡には、FCM と APN によるこの情報に関する制限が適用されます。Braze は、FCM または APN からユーザーがアンインストールしたことが通知された場合にのみアンインストール数を増加しますが、これらのサードパーティシステムは、いつでもアンインストールを通知する権利を留保します。そのため、正確な統計情報ではなく、傾向 (増加 / 減少) を検出するには、アンインストール追跡を使用する必要があります。

アンインストール追跡の使用の詳細については、[このブログの投稿][7] を参照してください。

キャンペーンのアンインストール統計情報は、[キャンペーンの詳細] ページにあります。マルチチャネルキャンペーンと多変量分析キャンペーンの場合、アンインストール数はそれぞれチャネル別とバリアント別の内訳で表示できます。

## トラブルシューティング

### 表示されているアンインストール数が突然急増している原因は何ですか?

表示されているアプリのアンインストール数が急増している場合、Firebase Cloud Messaging (FCM) と Apple プッシュ通知サービス (APNS) が古いトークンを異なる頻度で取り消したことが原因である可能性があります。
