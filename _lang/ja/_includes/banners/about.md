# バナー

> バナーを使えば、ユーザー向けのパーソナライズされたメッセージングを作成できる。同時に、メールやプッシュ通知といった他のチャネルのリーチも拡大できる。アプリやWebサイトに直接バナーを埋め込むことができるので、自然な感覚でユーザーとエンゲージメントできる。

![デバイス上に表示されたバナーの例。]({% image_buster /assets/img/banners/sample_banner.png %})

## 前提条件

バナーの利用可否は、お使いのBrazeパッケージによって異なる。アカウントマネージャーか顧客サクセスマネージャーに連絡して始めよう。

## なぜバナーを使うのか？

バナーは、マーケティングチームや製品チームがアプリやWeb サイトのコンテンツをダイナミックにパーソナライズすることを可能にする。これにより、ユーザーの適格性や行動をリアルタイムに反映できる。それらはメッセージを常にインラインで表示し、ユーザーセッションの開始時に自動的に更新される、邪魔にならず文脈に応じた体験を提供する。

バナーがアプリやWeb サイトに組み込まれた後、マーケターはシンプルなドラッグ＆ドロップエディターを使ってバナーをデザインし公開できる。これにより、継続的な開発者の支援が不要になり、複雑さが減り、効率が向上する。

| ユースケース | 説明 |
| --- | --- |
| 発表 | 今後のイベントやポリシー変更などの告知を、アプリ体験の最前面に表示し続けろ。 |
| パーソナライズされたオファー | 各ユーザーの閲覧履歴、カートの内容、サブスクリプションの階層、ロイヤルティステータスに基づいて、パーソナライズされたプロモーションや特典を表示する。 |
| 新規ユーザーのエンゲージメントを促進する | 新規ユーザーをオンボーディングフローとアカウント設定を通じて案内する。 |
| 販売・プロモーション | ホームページ上で、ユーザーの体験を妨げることなく、注目のコンテンツ、人気商品、進行中のブランドキャンペーンを継続的かつ直接的に強調表示する。 |

## 機能

バナーの機能には以下が含まれる：

- **簡単なコンテンツ作成：**画像、テキスト、ボタン、メールキャプチャフォーム、カスタムコードなどをサポートする視覚的なドラッグ＆ドロップエディターを使ってバナーを作成し、プレビューできます。
- **柔軟な配置：**バナーを表示できるアプリケーションや Web サイト内の複数の場所を定義することで、特定のコンテキストやユーザーエクスペリエンスへの正確なターゲティングが可能になります。
- **ダイナミックなパーソナライゼーション：**バナーは新しいユーザーセッションごとにダイナミックに更新される。これにより、Brazeの組み込みパーソナライゼーションツールとLiquidロジックを用いて、コンテンツが常に最新かつパーソナライズされた状態を維持する。
- **ネイティブ優先順位付け**複数のバナーが同じ配置をターゲットにした場合、表示優先度を設定する。これにより、適切なメッセージが適切なタイミングでユーザーに届くようにする。
- **カスタムコードエディタブロック：**カスタムコードエディタブロックを使って、高度なカスタマイズや既存のWebスタイルとのシームレスな統合のためにカスタムHTMLを追加する。

## バナーについて {#about-banners}

### 配置 ID {#placement-id}

バナー配置とは、[Braze SDKを使って作成する]({{site.baseurl}}/developer_guide/banners/placements/)アプリやWeb サイト内の特定の場所であり、バナーが表示される位置を指定するものだ。

一般的な配置場所としては、ホームページの最上部、商品詳細ページ、そして購入手続きの流れが挙げられる。配置が作成された後、[バナーキャンペーン内で]({{site.baseurl}}/user_guide/message_building_by_channel/banners/create/)バナーを[割り当て]({{site.baseurl}}/user_guide/message_building_by_channel/banners/create/)ることができる。

ワークスペースごとに作成できる配置の数に固定の制限はない。経験上必要となる限り、配置IDをいくつでも作成できる。各配置はワークスペース内で一意でなければならない。一つの配置IDは、同時に最大25個のアクティブなメッセージから参照されることがある。

{% alert important %}
バナーキャンペーンを開始した後は、配置IDを変更しないようにする。
{% endalert %}

### バナーの優先順位 {#priority}

複数のバナーメッセージが同じ配置IDを参照する場合、バナーは優先度順に表示される。優先度は高、中、低の順である。デフォルトではバナーは中程度に設定されているが、バナーキャンペーンを作成または編集する際に[優先度を手動で設定]({{site.baseurl}}/user_guide/message_building_by_channel/banners/create/#set-priority)できる。 

複数のバナーが同じ優先度に設定されている場合、ユーザーが対象となる最新のバナーが最初に表示される。

### メッセージング

バナーメッセージは、HTMLコンテンツとしてアプリやWeb サイトに配信される。通常はiframe内で表示される。これにより、バナーがあらゆるデバイスで一貫して表示されることが保証される。また、バナーのスタイルやスクリプトを他のコードから分離しておくのに役立つ。

Iframeは、コードベースの変更を必要としないダイナミックでパーソナライズされたコンテンツの更新を可能にする。各iframeは、キャンペーンターゲティングとパーソナライゼーションロジックを用いて、各ユーザーセッションのHTMLを取得し表示する。

{% multi_lang_include alerts/important_alerts.md alert='network dependency' %}

### 寸法とサイズ

バナーの寸法とサイズについて知っておくべきことは以下の通りだ：

- コンポーザーでは、さまざまなディメンションのバナーをプレビューできますが、その情報は保存されず、SDK に送信されません。
- HTMLは、それが描画されるコンテナの全幅を占める。
- 固定ディメンションの要素を作成し、そのディメンションをコンポーザーでテストすることをお勧めします。

## 制限事項

各ワークスペースは最大200のアクティブなバナーキャンペーンをサポートできる。この制限に達した場合、新しいキャンペーンを作成する前に、既存のキャンペーンを[アーカイブするか無効化する]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals/about_statuses/#changing-the-status)必要がある。

さらに、バナーメッセージでは以下の機能はサポートされていない。

- API トリガーキャンペーンとアクションベースのキャンペーン
- コネクテッドコンテンツ
- プロモーションコード
- ユーザーによる強制終了
- `catalog_items` [タグ`:rerender`]({{site.baseurl}}/user_guide/data/activation/catalogs/using_catalogs/#using-liquid)を使う
