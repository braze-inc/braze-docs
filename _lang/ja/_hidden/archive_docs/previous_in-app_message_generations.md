{% details フルスクリーン %}
これらは最も魅力的ですが、ユーザーのスクリーン全体を覆っているため、最も侵入的です。大規模で豊富な"画像を表示するのに適しており、重要な新機能や期限切れのプロモーションなど、非常に重要な情報を伝えるのに役立ちます。ユーザーエクスペリエンスは破壊的であるため、最優先の内容には控えめに使用してください。

![フルスクリーンメッセージ]({% image_buster /assets/img_archive/braze_fullscreen.png %}){: style="max-width:80%;"}

- ヘッダーと本文の文字、ボタン、バックグラウンドの色分け
- キーと値のペア

{% enddetails %}
{% details  モーダル %}
これらのメッセージは、ユーザーがアプリのUI の一部を見ることができるため、フルスクリーンメッセージほど侵入的ではありません。ボタンと"画像sがまだ含まれているため、よりインタラクティブで視覚的なキャンペーンが必要な場合は、モーダルの方がスライドアップよりも適している場合があります。これらは、アプリ 更新や緊急でない案件・行事など、中程度の重点内容に適しています。

{% details 従来のスライドアップ %}
これらは、最も侵入性の低いメッセージタイプですが、色やバッジのアイコンの使用状況に応じて、多少のアテンションを獲得することができます。これは、新しいユーザーをオンボーディングし、特にアプリ機能に向けるときに使用する電文形式かもしれません。なぜなら、それらはアプリ体験を一時停止せず、継続的な探索を可能にするからです。

### アプリ内メッセージファイルサイズを小さくする

Braze では、いくつかの理由から、"画像 s およびHTML アセットのzip はできるだけ小さくすることをお勧めします。

- HTMLと"画像の支払い読み込むが小さくなるほど読み込むが速くなり、顧客に応じてより迅速かつ確実に表示されます。
- HTMLと"画像の支払い読み込むを小さくすると、顧客のデータコストも抑えられます。Braze アプリ内メッセージ s は、セッションスタート時にバックグラウンドで停止読み込むされるため、選択した条件に基づいてリアルタイムでトリガーできます。そのため、10 HTML アプリ内メッセージ の1MB がある場合、すべてのメッセージがトリガーされていなくても、顧客 s はすべて10MB のデータチャージを負担します。これは、アプリ内メッセージs がキャッシュされ、セッション に読み込むされていない場合でも、時間の経過とともに急速に増加する可能性があります。

ファイルサイズを小さくするには、次の方法が役立ちます。

- HTMLアセットのZIPフォルダーにフォントファイルを含めるのではなく、HTML アプリ内メッセージsをカスタマイズするために、アプリのライケーションまたはWeb サイトに埋め込まれたリファレンスフォント。
- 無関係または重複するCSSやJavaScript がHTMLアセットのZIP に含まれていないことを確認します。
- すべてのs "画像で["画像Optim][25] を使用して、s を最小限の大きさに圧縮し、画質を落としません。
