# [![Braze Learning course]({% image_buster /assets/img/bl_icon2.png %})](https://learning.braze.com/segmentation-course){: style="float:right;width:120px;border:0;" class="noimgborder"}セグメントの作成

> セグメンテーションにより、人口統計学的、行動的、または技術的な特徴や行動に基づいてユーザーをターゲットにすることができます。セグメンテーションとメッセージングの自動化を創造的かつインテリジェントに使用することで、ユーザーをファーストタッチから長期顧客へとシームレスに移行させることができます。セグメントはデータの変更に応じてリアルタイムで更新され、ターゲティングやメッセージングの目的で必要な数だけセグメントを作成できます。

## ステップ:

### ステップ 3:セグメントセクションに移動

[**オーディエンス**] > [**セグメント**] に移動します。

{% alert note %}
[古いナビゲーションを使用している場合は]({{site.baseurl}}/navigation)、[**エンゲージメント**] に [**セグメント**] が表示されます。
{% endalert %}

### ステップ 3:セグメントに名前を付ける

「<i class="fas fa-plus"></i>**セグメントを作成**」をクリックしてセグメントの作成を開始します。フィルタリングするユーザーのタイプを説明して、セグメントに名前を付けます。これにより、このセグメントが今後の複数のキャンペーンやキャンバスのターゲットになることが確実になります。セグメントタイトルが曖昧だと、後で混乱を招く可能性があります。

オプションで、次の操作を実行できます。
-セグメントに説明を追加して対象者の意図を詳しく説明し、他のチームメンバーが参照できるようにメモを残します。
-[セグメントにチームを追加します]({{site.baseurl}}/user_guide/administrative/app_settings/manage_your_braze_users/teams/)。
-[セグメントにタグを追加してさらに整理できます]({{site.baseurl}}/user_guide/administrative/app_settings/tags/)。

![セグメントの作成] モーダルでは、セグメントに「Lapsed Users」という名前を付けて、セグメントの説明に「これは、過去 14 日以内の非アクティブなユーザーをターゲットとする主な休眠ユーザーセグメントです。「」ボタンが 2 つ付いています。キャンセルしてセグメントを作成。] [2]{: style="max-width:70%;"}

### ステップ 3:アプリまたはプラットフォームを選択してください

[**すべてのアプリのユーザー] (デフォルト) または [特定のアプリのユーザー****] を選択して、ターゲットにするアプリまたはプラットフォームを選択します**。「**すべてのアプリのユーザー」を選択すると、セッションやアプリデータに関係なく**、セグメントにはすべてのユーザーが含まれます。「**特定のアプリのユーザー**」を選択した場合は、セグメントに含めるアプリまたはプラットフォームを選択できます。

たとえば、アプリ内メッセージをiOSデバイスのみに送信したい場合は、iOSアプリを選択します。これにより、iOS デバイスと Android デバイスの両方を使用するユーザーは、自分の iOS デバイスでのみメッセージを受信するようになります。特定のアプリのリストで、「**Users from no apps**」オプションを使用すると、セッションもアプリデータもないユーザーを含めることができます（通常はユーザーインポートまたは REST API で作成）。

![使用したアプリセクションで「すべてのアプリのユーザー」オプションが選択されているセグメント詳細パネル] [5]

### ステップ 4: セグメントにフィルターを追加

セグメントに少なくとも 1 つのフィルターを追加して、オーディエンスを絞り込んでください。

#### フィルターグループ

フィルターはフィルターグループにまとめられています。すべてのフィルターは、少なくとも 1 つのフィルターを含むフィルターグループの一部である必要があります。セグメントには複数のフィルターグループを含めることができます。フィルターグループを追加するには、「**フィルターグループを追加**」をクリックします。

各フィルターの横にあるアイコンをクリックして、フィルターを複製または削除します。フィルターを複製した後、各ドロップダウンでその値を調整できます。

各フィルターグループ内のアイコンを使用して、そのフィルターグループとその中のフィルターを複製したり、そのフィルターグループをセグメントから削除したりすることもできます。

#### AND と OR を使用したセグメンテーションロジック

フィルターグループ内では、フィルターを「AND」または「OR」で結合できます。フィルターグループ間では、グループを「AND」または「OR」で結合できます。フィルターグループを使用する場合、次のようなセグメンテーションロジックを作成できます。
-(A と B と C) または (C と E と F)
-(A または B または C) および (C または D または F)

フィルターに「OR」を選択すると、そのフィルターの一つ、一部、またはすべての組み合わせを満たすユーザーがセグメントに含まれます。「AND」を選択すると、そのフィルターに合格しなかったユーザーはセグメントに含まれません。

#### フィルター演算子

選択した特定のフィルターに応じて、フィルター値を識別するための演算子が異なります。さまざまなタイプのカスタム属性に使用できる演算子について詳しくは、「[カスタム属性ストレージ]({{site.baseurl}}/user_guide/data_and_analytics/custom_data/custom_attributes/#setting-custom-attributes)」を参照してください。「is any of」演算子を使用する場合、そのフィールドに含めることができる項目の最大数は256であることに注意してください。

{% alert note %}
Brazeは、ユーザーが初めてアプリを使用するまでユーザーのプロファイルを生成しないため、アプリをまだ開いていないユーザーをターゲットにすることはできません。
{% endalert %}

![AND 演算子を使用してグループをセグメントフィルタする。] [9]

{% alert important %}
すでにセグメントメンバーシップフィルターを使用しているセグメントは、他のセグメントにさらに含めたり、ネストしたりすることはできません。
{% endalert %}

#### 除外グループ (オプション)

セグメントを構築する場合、1 つまたは複数の除外グループを適用できます。除外グループには、セグメントから除外するユーザーを識別する条件が含まれており、常に「AND NOT」演算子を使用してフィルターグループに接続されます。

ユーザーが除外グループの条件に該当する場合、フィルターグループ内の条件を満たしていても、そのユーザーはセグメントに含まれません。

フィルターグループの場合と同様にフィルターを追加して、除外グループを作成します。_除外グループの推定リーチャブルユーザー統計には_、除外基準が適用された後にセグメントに残っている推定ユーザー数が表示されます。

除外されたユーザーは、_セグメントのリーチ可能なユーザーの総数統計には含まれません_。

![2 つのフィルタを持つ除外グループ。] [12]

#### テストセグメント

セグメントにアプリとフィルターを追加したら、ユーザーを検索してセグメント条件と一致するかどうかを確認することで、セグメントが期待どおりに設定されているかどうかをテストできます。そのためには、「**ユーザーを検索**」をクリックして、`external_id``braze_id`ユーザーのまたはを検索します。

![「ユーザ検索」ボタンのあるユーザ検索セクション] [6]{: style="max-width:50%;"}

ユーザー検索は次の場合に利用できます。
-セグメントの作成
-キャンペーンまたはCanvasオーディエンスの設定
-オーディエンスパスの設定ステップ

ユーザーがセグメント、フィルター、アプリの条件に一致すると、以下が表示されます。

![「user007」のユーザ検索により、「user007 はすべてのセグメント、フィルタ、およびアプリケーションに一致しました」というアラートがトリガーされます。] [7]{: style=" max-width:60%;"}

ユーザーがセグメント、フィルター、またはアプリの条件の一部またはすべてに一致しない場合、トラブルシューティングのために欠落している条件が表示されます。

![「user1234」というユーザを検索すると、「user1234 は次のターゲット基準に一致しません。」というアラートが表示され、2 つの条件が表示される。1 つは在職期間が 1 年を超えることと、今日は記念日であることである。] [8]{: style=" max-width:60%;"}

#### シングルユーザーセグメント

ユーザー名やユーザーIDなど、ユーザーを識別する固有の属性を使用して、単一のユーザーセグメント（または少数のユーザーのセグメント）を作成できます。

ただし、セグメント統計は、結果が+/-1％以内という95％の信頼区間のランダムサンプルに基づいて計算されるため、セグメンテーション統計またはプレビューにはこの個々のユーザーが表示されない場合があります。ユーザーベースが大きいほど、セグメントのサイズが概算である可能性が高くなります。ターゲットとするユーザーが 1 人だけセグメントに含まれるようにするには、**セグメント詳細ページの**「**正確な統計情報を計算**」をクリックします。これにより、セグメント内の正確なユーザー数が 99.999% を超える精度で計算されます。

Brazeには、ユーザーIDまたはメールアドレスで特定のユーザーをターゲットにするテストフィルターがあります。

### ステップ 5: セグメントを保存

[**保存**] をクリックすると、ユーザーにメッセージを送信する準備が整います。

## セグメントメンバーシップの計算 {#segment-membership-calculation}

Brazeは、データが当社のサーバーに送り返されて処理されると、通常は瞬時にユーザーのセグメントメンバーシップを更新します。ユーザーのセグメントメンバーシップは、そのセッションが処理されるまで変更されません。たとえば、セッションが最初に開始されたときに失効したユーザーセグメントに入ったユーザーは、セッションが処理されるとすぐに離脱ユーザーセグメントから移動されます。

### リーチ可能なユーザーの総数の計算

各セグメントには、そのセグメントのメンバーであるユーザーの総数が表示されます。**すべてのアプリのユーザーをフィルタリングすると**、ウェブプッシュやメールなど、それらのユーザーと通信できるさまざまなチャネルもすべて表示されます。合計ユーザー数が、各チャネルで到達可能なユーザー数と異なる場合があります。

![合計 2,414 人のリーチ可能なユーザーを、電子メール、iOS プッシュ、Android プッシュ、ウェブプッシュ、Kindle プッシュ、Android チャイナプッシュでリーチ可能なユーザー別に表示した表] [10]

ユーザーが特定のチャネルを通じてリーチ可能として表示されるには、そのユーザーが次の両方を持っている必要があります。
\* プロフィールに関連付けられている有効なメールアドレス/プッシュトークン、および
\* アプリをオプトインまたはサブスクライブしました。

1 人のユーザーが異なるリーチ可能なユーザーグループに属している場合があります。たとえば、ユーザーが有効なメールアドレスと有効な Android プッシュトークンの両方を持っていて、両方にオプトインしても iOS プッシュトークンが関連付けられていない場合があります。リーチ可能なユーザーの総数と、異なるチャネルの合計との差は、セグメントの対象となったが、それらの通信チャネルではリーチできないユーザーの数です。

### セグメントサイズの統計

Brazeはセグメントサイズに関する以下の統計を提供します。推定統計はすべて（+/-1%）で、正確なセグメントメンバーシップは常に、キャンペーンまたはキャンバスで送信されたメッセージによってセグメントが影響を受ける前に計算されます。

#### フィルター統計

フィルターグループごとに、推定リーチ可能なユーザー数を表示できます。「**その他のファネル統計を拡大**」をクリックすると、チャネル全体の内訳が表示されます。

![First Name (ファーストネーム)] のフィルタが空白でなく、アプリケーションを複数回使用しているフィルタグループ。] [4]

#### セグメント統計

セグメント全体について、リーチ可能な推定ユーザー数と、各チャネルの推定ユーザー数をページの下部に表示できます。また、「**正確な統計を計算**」をクリックすると、リーチ可能なユーザーの正確な数（セグメント全体とチャンネルごとの両方）を表示できます。

以下の点に注意してください。
-正確な統計情報の計算には数分かかる場合があります。この関数はセグメントレベルでのみ正確な統計量を計算し、フィルターまたはフィルターグループレベルでは計算しません。
-大きなセグメントでは、正確な統計を計算してもわずかな変動が見られるのは正常です。この機能の精度は 99.999% 以上になると予想されます。

## セグメントのアーカイブ

特定のセグメントが不要になった場合、または廃止を希望する場合は、**セグメントページに移動してギアをクリックし**、ドロップダウンから [アーカイブ] **を選択してアーカイブできます**。

{% alert warning %}
セグメントをアーカイブすると、そのセグメントを使用しているキャンペーンやキャンバスもアーカイブされます（セグメントが単一のキャンバスコンポーネントでのみ使用されている場合でも）。これには、両方のセグメントと、それらを使用するキャンペーンまたはキャンバスもアーカイブされるネストされたセグメントも含まれます。
<br><br>
関連するセグメントをアーカイブすると、どのキャンペーンとキャンバスがアーカイブされようとしているかが警告リストに表示されます。
{% endalert %}

**セグメントのアーカイブを解除するには、「セグメント」**ページ内でそのセグメントに移動し**、「アーカイブ解除」を選択します。**
