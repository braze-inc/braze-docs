# API使用状況ダッシュボード

> API使用状況ダッシュボードを使用すると、Brazeに入力されるREST APIトラフィックを監視して、REST APIの使用状況の傾向を把握し、潜在的な問題をトラブルシューティングすることができる。

デフォルトのダッシュボードは、過去1日（24時間）にワークスペースに届いたすべてのREST APIリクエストのビューである。ユースケースに応じて、ダッシュボードのコントロールを調整し、トラフィックをフィルタリングまたはグループ化したり、ダッシュボードの時間範囲を設定することができる。

![]({% image_buster /assets/img/api_usage_dashboard/api_usage_dashboard.png %})

## サマリー統計

- **総リクエスト数である：**ダッシュボードに適用されたフィルタとコントロールに基づき、現在のワークスペースでBrazeに送信されたリクエストの総数。
- **成功率だ：**Brazeが`2XX` 成功応答を発行した全リクエストの割合。
- **エラー率だ：**Brazeが`4XX` または`5XX` エラー応答を発行した全リクエストの割合。

## ダッシュボード・コントロール

![]({% image_buster /assets/img/api_usage_dashboard/filters.png %}){: style="float:right;max-width:35%;margin-left:15px;"}

### フィルター

フィルタを適用して、ワークスペースのREST APIトラフィックを絞り込む。使用可能なフィルターは以下の通り：

- APIエンドポイント
- APIキー
- 応答コード

### グループデータ

データをさまざまなデータ系列にグループ化し、使用状況のさまざまなパターンを探る。利用可能なグループ分けのオプションは以下の通りである：

- レスポンスコード（デフォルト）
- APIエンドポイント
- APIキー
- 成功と失敗のみ

### 日付

必要に応じて、日付フィルタを調整して、表示する時間範囲を小さくしたり大きくしたりする。利用可能な日付は以下の通りである：

- 今日（デフォルト）
- カスタム
- 最後の3時間
- 直近6時間
- 直近12時間
- 直近24時間
- 昨日
- 直近7日間
- 過去14日間
- 過去30日間
- 先月から今日まで

{% alert note %}
**直近3時間**」と「**直近6時間」の**オプションは、トラフィックを分単位で表示する。より大きな時間帯は、時間単位または日単位でトラフィックを表示する。
{% endalert %}

## 考慮事項

API使用状況ダッシュボードには、Brazeが受信し、`2XX` 、`4XX` 、または`5XX` レスポンスを返したすべてのREST APIリクエストが含まれる。これには、データ変換の出力とクラウド・データ・インジェストの同期が含まれる。SDKトラフィックとユーザーアップデート手順は、このダッシュボードには含まれていない。

ダッシュボードに表示されるデータは、最近のトラフィックを表示するのに最大15分遅れることがある。使用量が多い時間帯は、1分間に最大4回までダッシュボードを更新できる。再度ダッシュボードを更新する前に、数分間待つ必要があるかもしれない。
