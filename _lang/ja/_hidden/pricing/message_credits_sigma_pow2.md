# メッセージクレジット-Sigma (機密)

> メッセージクレジットは、BrazeのネイティブSMS、MMS、WhatsAppサービスのクロスチャネルパッケージ構造です。メッセージクレジットを使用して、Brazeメッセージングチャネルを利用する際に柔軟で透明性の高いエクスペリエンスを提供します。購入したクレジットの割り当ては、このページの表に示されているどのチャネルでも使用できます。

{% alert note %}
チャネルが異なれば、消費されるクレジットを定義する測定単位も異なります。<br><br>
<b>WhatsApp:</b>会話<br>
<b>SMS:</b>セグメント<br>
<b>MMS:</b>セグメント<br><br>
つまり、WhatsAppメッセージに使用されるクレジットは会話の開始時に計算され、SMSとMMSメッセージの両方に使用されるクレジットは送信されたセグメントで計算されます。
<br><br>
最後に、配送業者の手数料は個別に（延滞して）請求され、このメッセージクレジットSKUには含まれません。
{% endalert %}

## 定義

カラムの定義は次のとおりです。

|---------|-------------------------------------------------|
| **チャンネルクレジット比率** | 各チャンネルのベースラインクレジット額 |
| **送信先** | Braze プラットフォームを通じて送信される特定の最終地域、国、またはメッセージのタイプ |
| **マルチプライヤー** | スケーラーとチャネルクレジットの比率（特定の目的地の価格による）|
| **1 つのメッセージで消費されるクレジット** | 1 つのメッセージの送信に使用されたメッセージクレジットの正確な数<br> (1 つのメッセージで消費されるクレジット = チャネルクレジット比率 x 送信先乗数) |

## SMS/MMS チャネルの詳細

### SMS セグメント

SMS メッセージセグメントは、SMS 業界がメッセージをカウントする方法です。メッセージセグメントは、1 回の SMS ディスパッチで送信される、定義済みの文字数 (GSM-7 エンコーディングでは 160 文字、UCS-2 エンコーディングでは 67 文字) までのグループです。GSM-7 エンコーディングを使用して 161 文字の SMS を送信すると、送信されたメッセージセグメントが 2 つあることがわかります。複数のメッセージセグメントを送信すると、追加料金が発生します。

### MMS セグメント

MMS の場合、メッセージの上限は 5 MB です (これにはマルチメディアアセットとメッセージ本文のサイズが含まれます)。安全のため、Brazeではメッセージ本文を含めながら、マルチメディアアセットの容量は600 KBを超えないようにすることを推奨しています。

## WhatsApp チャンネルの詳細

WhatsAppは双方向のメッセージングに焦点を当てたチャネルであるため、（個々のメッセージの数ではなく）会話に重点を置いています。会話とは、企業とエンドユーザーの間の 24 時間体制のやり取りです。

### 会話タイプの定義

**マーケティング会話:**認知度の向上から売り上げの促進、顧客のリターゲティングまで、幅広い目標を達成できるビジネス主導の会話。例としては、新製品、サービス、新機能の発表、ターゲットを絞ったプロモーション/オファー、カート放棄のリマインダーなどがあります。

**ユーティリティカンバセーション:**ビジネスで開始された会話により、ユーザーのアクションやリクエストをフォローアップできます。例としては、オプトインの確認、注文/配送管理 (配送の更新など)、アカウントの更新またはアラート (支払い通知など)、フィードバック調査などがあります。

**認証に関する会話:**ログインプロセスの複数のステップ（アカウントの検証、アカウントの回復、整合性の課題など）で、ワンタイムパスコードを使用してユーザーを認証できます。

{% alert note %}
認証に関する会話はケースバイケースでのみサポートされ、Brazeは特定のSLAを保証することはできません。さらに、Braze は暗証番号の生成をサポートしていません。
{% endalert %}

**サービス会話:**ユーザーが開始した会話で、テンプレート化されていないメッセージで応答されます。

{% alert note %}
マーケティングテンプレートまたはユーティリティテンプレートを使用して返信されたユーザーからの会話には、その旨が課金されます。
{% endalert %}
