# ユーザーアーカイブ

> 毎週日曜日の午前5時30分（東部標準時）に、Brazeは非アクティブなユーザーと休眠ユーザーをBrazeサービスから削除するプロセスを実行します。Braze はワークスペースのユーザー数が 250,000 人のしきい値に達しない限り、ユーザーをアーカイブしないことに注意してください。 

このプロセスは、Braze がキャンペーンのリーチ可能なオーディエンスに関する正確な統計情報を提供できるようにすることを目的としています。また、[GDPRの2つの主要概念にも準拠しています][1]。

1. 保管制限の原則—処理および保存された個人データは、必要以上に保管しないでください
2. 個人データを処理する正当な事業目的を持っていること。

つまり、処理および保存された個人データは、必要以上に保管しないでください。また、個人データは合法的な事業目的でのみ処理されるべきです。アーカイブされたユーザーの登録解除ステータスも、GDPRに従って削除されます。

{% alert note %} ユーザーは、ユーザーが非アクティブか休眠状態かを完全に制御できます。Braze Canvasにはこれを自動的に行う機能があり、非アクティブなユーザーや休眠中のユーザーの一部または全員に対してこの機能を効果的に無効にすることができます。 {% endalert %}

## ユーザーアーカイブ定義

### アクティブユーザ

Brazeでは、一定期間の「アクティブユーザー」とは、モバイルアプリまたはWebサイトでセッションを記録したユーザー、更新されたユーザー、メッセージを送信したユーザー、またはメッセージを操作したユーザーとして定義しています。

新しいユーザーがログインしたときにユーザーを識別するようにユーザー ID を設定すると、そのユーザーは別のアクティブユーザーとしてカウントされます。API 経由で更新したユーザーも、更新期間中はアクティブユーザーとしてカウントされます。

### 非アクティブユーザー

「非アクティブユーザー」とは、連絡が取れず、解約した可能性が高いユーザーです。非アクティブユーザーとは、次の条件をすべて満たすユーザーです。

- メールを受信できません。たとえば、メールアドレスを持っていない場合や、すべてのメールリストの登録が解除されている場合などです。
- SMS を受信できません。たとえば、有効な電話番号がない場合や、すべての SMS 購読グループから登録解除されている場合などです。
- プッシュを受信できません。たとえば、アプリをアンインストールしたか、プッシュ権限を無効にした場合などです。
- 6 か月以上モバイルアプリを使用していない、またはワークスペース内の Web サイトにアクセスしていない。
- 6 か月以上ワークスペースからメッセージを受信していない。
- 6か月以上更新されていません。

この場合、これらのユーザーにはメッセージを送ることができず、ブランドとの関わりもありません。これらのユーザーは事実上解約しています。

### 休眠ユーザー

「休眠ユーザー」とは、過去 12 か月間にアクティビティがなく、かつ以下のユーザーです。

- 12 か月以上モバイルアプリを使用していない、またはワークスペース内の Web サイトにアクセスしていない。
- 12 か月以上ワークスペースからメッセージを受信していない。
- 12 か月以上更新されていません。

## 迷惑メールブロック

Braze は 500 万セッションを超える個々のユーザー (「ダミーユーザー」) をブロックし、SDK イベントの取り込みは行いません。これらのイベントは通常、誤った統合の結果であるためです。これが正当なユーザーに対して起こったことがわかった場合は、[Brazeサポートにチケットを提出してください]({{site.baseurl}}/braze_support/)。

ダッシュボードのダミーユーザーを見つけるには、次の手順を実行します。

1. [セグメントを作成します]({{site.baseurl}}/user_guide/engagement_tools/segments/creating_a_segment/)。
2. `Session Count`フィルターを選択し、に設定します`more than 5,000,000`。
3. セグメントを CSV でエクスポートします。

必要に応じて、[`/users/delete`エンドポイントからユーザーを削除できます]({{site.baseurl}}/api/endpoints/user_data/post_user_delete/)。

## ユーザーアーカイブポリシーのカスタマイズ

Braze は、ユーザーアーカイブポリシーをカスタマイズできるデータオーケストレーション機能を提供します。Canvas User [Update コンポーネントを使用して、両方の長所を活用できるユーザーアーカイブポリシーを作成します]({{site.baseurl}}/user_update/)。

これにより、次のことが可能になります。

- 不要になったユーザープロファイルを削除して、GDPRとプライバシーのベストプラクティスを順守してください。
- ビジネス上正当な必要性があるユーザープロフィールはすべて保持してください。

### ステップ:

1. アーカイブ基準を満たし、維持したいユーザーをターゲットにします。<br><br>
      ![23週間以上前にメッセージを最後に受信し、キャンペーンまたはキャンバスステップからメッセージを受け取ったことがなく、これらのアプリを最後に使用したのは23週間以上前で、これらのアプリをまったく使用したことがないユーザーをターゲットとします。] [2]<br><br>
2. 再資格を6か月弱に設定してください。<br><br>
      ![再資格を有効にし、再資格期間を23週間に設定した入国管理] [3]<br><br>
3. User Update ステップを設定して、各プロファイルにイベントを追加します。<br><br>
      ![「do\_not\_archive」イベントをユーザーのプロファイルに追加するユーザー更新ステップ] [4]
