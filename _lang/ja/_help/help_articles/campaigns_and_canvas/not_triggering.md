# キャンペーンまたはキャンバスをトリガーしない

期待したトリガー動作が得られなかった理由はいくつかある。最も一般的なエラーのソリューションは、トリガーしているキャンペーンがセグメント内で同じトリガーイベントを使用していないことを確認することである。

## キャンペーントリガー

セグメントメンバーシップは、トリガーアクションの前に評価される。つまり、ユーザーが最初にセグメンテーションに該当しなければ、トリガーを実行してもキャンペーンは届かない。

キャンペーンがカスタムイベントからトリガーされる場合、このイベントがキャンペーンで使用したいセグメンテーションによって事前にフィルターされていないことを確認したい。 

例えば、セグメンテーションがイベント`SessionStart` "Has Used App more than once "を含み、キャンペーンのトリガーとなるイベントが`SessionStart` の場合、ユーザーはメッセージを受け取るが、それは必ずしも最初のセッションのためではない。なぜなら、ユーザーがキャンペーンを受け取るべきかどうかをチェックする最初のステップで、キャンペーンはセグメンテーションのターゲットオーディエンスを確認しているからである。 

つまり、アクションベースのキャンペーンまたはキャンバスをオーディエンスフィルターと同じトリガー（変更された属性やカスタムイベントの実行など）で構成することは避ける。[競合条件が][2]発生すると、トリガーイベントを実行したときにユーザーがオーディエンスにおらず、キャンペーンを受け取れなかったり、キャンバスに入れなかったりする。

{% alert tip %}
キャンペーンのトラブルシューティングについては、過去30日分の診断ログしか残っていないため、問題発生から30日以内に必ずBrazeサポートに連絡すること。
{% endalert %}
