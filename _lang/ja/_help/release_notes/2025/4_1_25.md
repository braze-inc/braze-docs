# 2025年4月1日リリース

## ろう付けナビゲーションの更新

Braze で更新されたナビゲーションは、デバイス間で機能やコンテンツに効率的にアクセスできるように設計されています。ナビゲーションバージョンを切り替えるオプションは使用できなくなりました。詳しくは、専用の[Navigating Braze]({{site.baseurl}}/user_guide/administrative/access_braze/navigation)記事を参照してください。

## 開発者ガイドデタングル

以前は、Swift SDK を6 ページに分割するなど、多くのプラットフォームレベルのタスクが複数のページに分割されていました。さらに、共有機能は各プラットフォームごとに個別に文書化されています。つまり、「プッシュ通知のセットアップ」などのトピックを検索すると、10 の異なるページが返されます。

**前:**

![以前のSwift ドキュメントは、Platform Integration Guides セクションにあります。]({% image_buster /assets/img/before_swift.png %})

これで、プラットフォームレベルのタスクが単一ページにマージされ、共有SDK 機能が同じページに(新しいSDK タブ機能を使用して) 存在するようになりました。たとえば、Braze SDK を統合するためのページが1 つしかないようになりました。ここでは、ページの上部にあるタブを選択して、プラットフォームを切り替えることができます。この場合、ページ内の目次でも、現在選択されているタブが反映されるように更新されます。

**以降:**

![更新されたSwift ドキュメントは、SDK アーティクルの統合のSwift タブにあります。]({% image_buster /assets/img/after_swift.png %})

![更新されたAndroid ドキュメントは、Integrating the SDK article.]({% image_buster /assets/img/after_android.png %}) のAndroid タブにあります。

## データの柔軟性

### Canvas エントリプロパティへの更新

キャンバスエントリプロパティは、[キャンバスコンテキスト変数]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/canvas_entry_properties_event_properties) の一部になりました。各コンテキスト変数には、名前、データ型、およびLiquid を含めることができる値が含まれます。詳細については、[コンテキストコンポーネント]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/context)を参照してください。

### 電話番号フィルタのセグメンテーションフィルタの更新

セグメンテーションフィルタは、2 つの電話番号フィルタへの変更を反映するように更新されています。

- [フォーマットされていない電話番号]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters#unformatted-phone-number) (旧**電話番号**):フォーマットされていない電話番号でユーザーをセグメント化します。
- [電話番号]({{site.baseurl}}/user_guide/engagement_tools/segments/segmentation_filters#phone-number)(旧**送信電話番号**):E.164 形式の電話番号フィールドでユーザーをセグメント化します。

### カスタムデータの削除

ターゲットを絞ったキャンペーンやセグメントを構築していくうちに、カスタムイベントやカスタム属性が不要になるかもしれない。[このカスタムデータ]({{site.baseurl}}/user_guide/data/custom_data/managing_custom_data#deleting-custom-data)を削除し、アプリからその参照を削除できるようになりました。

### メールアドレスと電話番号を持つユーザーのインポート

これで、電子メールアドレスまたは電話番号を使用して[import users]({{site.baseurl}}/user_guide/data/user_data_collection/user_import/#importing-with-email-addresses-and-phone-numbers) をインポートし、外部ID またはユーザーエイリアスを省略できます。

### サービスプロバイダーによるログイントラブルシューティング

サービスプロバイダ(SP) が開始したログインには、SAML およびシングルサインオンの問題を解決するための[トラブルシューティングセクション]({{site.baseurl}}/user_guide/administrative/access_braze/single_sign_on/set_up/#troubleshooting) があります。

### ユーザーインポートのトラブルシューティング

[User Import トラブルシューティングセクション]({{site.baseurl}}/user_guide/data/user_data_collection/user_import#troubleshooting) には、インポートされたCSV ファイルの欠落行のトラブルシューティング方法など、新しいエントリと更新されたエントリがあります。

### セグメント拡張に関するよくある質問

複数のカスタムイベントを使用するSegment Extension を作成する方法など、Segment Extensions の[よくある質問]({{site.baseurl}}/user_guide/engagement_tools/segments/segment_extension/#frequently-asked-questions) を参照してください。

### パーソナライズされた遅延と拡張遅延

{% multi_lang_include release_type.md release="早期アクセス" %}

ユーザに[パーソナライズされた遅延]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/delay_step#personalized-delays) を設定し、コンテキストステップでこれを使用して遅延するコンテキスト変数を選択できます。

また、遅延ステップを2年まで延長できるようになりました。たとえば、アプリの新規ユーザーをオンボーディングしている場合、セッションを開始していないユーザーをナッジするメッセージステップを送信する前に、2 か月間の遅延を追加できます。

### Snowflakeのデフォルトのユーザープロファイル属性

{% multi_lang_include release_type.md release="ベータ" %}

現在、Snowflakeには3つの[デフォルトのユーザープロファイル属性]({{site.baseurl}}/partners/data_and_analytics/data_warehouses/snowflake/user_attributes/)があります。各ビューは、独自のパフォーマンスを考慮した特定のユースケース用に設計されています。たとえば、ユーザープロファイルのデフォルト属性の定期的なスナップチャットを提供できます。

## 強力なチャネル

### メッセージングの基本

[Messaging Fundamentals]({{site.baseurl}}/user_guide/engagement_tools/messaging_fundamentals) は、Engagement Tools の新しいセクションで、アーカイブやローカライズメッセージなど、キャンペーンとキャンバスの共有された概念と用語を収容します。

### WhatsAppカスタムドメイン

これで、[カスタムドメイン]({{site.baseurl}}/user_guide/message_building_by_channel/whatsapp/whatsapp_campaign/custom_domains/) を1 つまたは複数のWhatsApp サブスクリプションググループに割り当てることができます。

### Canvas のアプリ内メッセージのトリガー

アプリ内メッセージ の[ トリガをセッション開始時、またはカスタムイベントと購入時にトリガーするように選択できるようになりました。遅延が経過し、オーディエンスオプションがチェックされると、ユーザがメッセージステップに到達したときにアプリ内メッセージがライブに設定されます。ユーザがセッションを開始し、アプリ内メッセージのトリガイベントを実行すると、アプリ内メッセージが表示されます。 

### キャンバスの入口ボリュームの制限

選択したケイデンス(毎日、キャンバスの寿命、またはキャンバスがスケジュールされるたびに)によってこのキャンバスに入る可能性のある人の数を制限できます。たとえば、[ エントリコントロール]({{site.baseurl}}/user_guide/engagement_tools/canvas/create_a_canvas/create_a_canvas?tab=action-based%20delivery#step-2c-set-your-target-entry-audience) を設定して、キャンバスが1 日に5000 人のユーザにのみ送信できるようにすることができます。

### 新しいユースケース:予約通知メールシステム

Braze 機能を使用して[ 予約リマインダーメールメッセージサービス]({{site.baseurl}}/user_guide/engagement_tools/canvas/ideas_and_strategies/booking_use_case) を構築する方法について説明します。このサービスにより、ユーザーは予定を予約することができ、次回の予定のリマインダーとともにユーザーにメッセージを送信します。このユースケースでは電子メールメッセージが使用されますが、ユーザープロファイルへの単一の更新に基づいて、任意のチャネルまたは複数のチャネルでメッセージを送信できます。

### 特定のリンクのトラッキングをクリックする

HTML エディタまたはドラッグアンドドロップエディタのコンポーネントにHTML コードを追加することで、特定のリンクのクリックトラッキング をオフにすることができます。

### Dynamic Appleプッシュ通知サービスゲートウェイの管理

[動的APNsゲートウェイ管理]({{site.baseurl}}/developer_guide/push_notifications/?sdktab=swift#swift_dynamic-apns-gateway-management)は、正しいAPNs環境を自動的に検出することでiOSプッシュ通知の信頼性と効率を高めます。これまでは、プッシュ通知用にAPNs環境(開発またはプロダクション)を手動で選択していましたが、ゲートウェイ設定の誤り、配信の失敗、BadDeviceTokenエラーが発生することがありました。

### バナーのフルッターサポート

{% multi_lang_include release_type.md release="早期アクセス" %}

バナーはFlutterをサポートするようになりました。さらに、すべてのバナーカードのドキュメントを使いやすくするために、オーバーホールしました。開始するには、次の記事を確認してください。

- [バナーについて]({{site.baseurl}}/developer_guide/banners/)
- [バナーキャンペーンの作成]({{site.baseurl}}/user_guide/message_building_by_channel/banners/creating_campaigns/)
- [アプリへのバナーの埋め込み]({{site.baseurl}}/developer_guide/banners/creating_placements/)

### WhatsAppのクリック追跡

{% multi_lang_include release_type.md release="早期アクセス" %}

[トラッキング]({{site.baseurl}}/user_guide/message_building_by_channel/whatsapp/whatsapp_campaign/click_tracking/)をクリックすると、誰かがWhatsAppメッセージのリンクをタップしたときに、どのコンテンツがエンゲージメントを促進しているかを明確に確認できます。ブレーズは、URL を短縮し、シーンの背後にトラッキングを追加し、クリックイベントが発生したときにログに記録します。

### プッシュについてよくある質問

新しい[Push FAQ]({{site.baseurl}}/user_guide/message_building_by_channel/push/faq)の記事をチェックしてください。この記事は、プッシュキャンペーンの設定時に発生する最もよくある質問のいくつかを扱っています。

### プッシュトラブルシューティング

[プッシュトラブルシューティング]({{site.baseurl}}/user_guide/message_building_by_channel/push/troubleshooting)では、プッシュ通知を使用して配信チャレンジをナビゲートするのに役立ついくつかのステップが用意されています。たとえば、プッシュ通知で配信の問題が発生した場合は、問題のトラブルシューティングのために実行できる手順をコンパイルしました。

## 新しいBrazeのパートナーシップ

### 可動インクダビンチ-ダイナミックコンテンツ

Braze and Movable Ink [Da Vinci]({{site.baseurl}}/partners/movable_ink_da_vinci)の統合により、ブランドはDa VinciのAI主導のコンテンツ決定エンジンを活用することで、高度にパーソナライズされたメッセージングを提供できるようになります。Da Vinciは、各ユーザーにとって最も関連性の高いコンテンツをキュレートし、Brazeを介してメッセージをシームレスに展開します。
