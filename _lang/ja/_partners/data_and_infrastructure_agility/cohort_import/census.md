# Census コホートインポート

> この記事では、[Census][1] から Braze にユーザーコホートをインポートする方法について説明します。[Census の統合について詳しくは、Census のメイン記事を参照してください。]({{site.baseurl}}/partners/data_and_infrastructure_agility/workflow_automation/census/)

## コホートインポート統合

### ステップ1:Braze サービス接続を作成

Census をCensus プラットフォームに統合するには、\[**接続**] タブに移動し、\[**新しい目的地] を選択して新しい** Braze サービス接続を作成します。

表示されるプロンプトで、この接続に名前を付け、Braze エンドポイント URL、Braze REST API キー、およびデータインポートキーを指定します。**データインポートキーはコホートを同期するために必要で、Braze の \[**パートナー統合] > \[**テクノロジーパートナー****] > \[Census] で確認できます。**

![][8]{: style="max-width:60%;"}

### ステップ2:Census 同期の作成

お客様を Braze と同期するには、同期を作成する必要があります。ここでは、データを同期する場所と、2 つのプラットフォーム間でフィールドをどのようにマッピングするかを定義します。

1. 「**同期**」タブに移動し、「**新規同期**」を選択します。<br><br> 
2. Composer で、データウェアハウスからソースデータモデルを選択します。<br><br>
3. モデルの同期先を設定します。同期送信先**としてBrazeを選択し**、**同期するオブジェクトとしてユーザーとコホートを選択します**。<br>![「目的地の選択」プロンプトで、接続として「Braze」が選択され、さまざまなオブジェクトが表示されます。][10]{: style="max-width:80%;"}<br><br>
4. **コホートに追加するユーザーを識別するソース列を選択し**、****識別子の種類として外部ユーザーIDを選択します****。<br><br>
5. **コホート名ドロップダウンで**、コホートを選択するか、コホートを作成するか、ソース列を選択してコホート名を入力します。<br><br>
6. \[**ソースデータからレコードが削除された場合] ドロップダウンを使用して、ユーザーがソースデータセットから削除されたときの処理**（\[**何もしない**] や \[**一致するレコードをコホートから削除**] など）を選択します。<br><br>
7. 最後に、Census データフィールドを同等の Braze フィールドにマッピングします。<br>![Census マッピング][11]{: style="max-width:80%;"}<br><br>
8. 詳細を確認して同期を作成します。 

これで、同期を実行できるようになりました。

同期中、マップしたフィールドはすべて最初にユーザーオブジェクトと同期され、Brazeにすでに存在するフィールドが更新。その後、更新されたユーザー指定されたコホートに追加されます。

同期後、Census コホートフィルター付きのBrazeSegment を作成して今後のBrazeキャンペーンやキャンバスに追加し、それらのユーザーをターゲットにすることができます。 

{% alert note %}
CensusとBrazeの統合を使用する場合、Censusは同期のたびにデルタ（変化するデータ）のみをBrazeに送信します。
{% endalert %}
