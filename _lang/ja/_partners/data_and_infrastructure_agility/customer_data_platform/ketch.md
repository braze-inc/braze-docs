# Ketch

> [Ketchは](https://www.ketch.com)、企業がデータの責任ある管理者となることをイネーブルメントする。Ketchは、簡素化されたプライバシー・オペレーションと、完全でダイナミックなデータ・コントロールとインテリジェンスを提供する。 

BrazeとKetchの統合により、Ketchのユーザー設定センター内で顧客のコミュニケーション設定をコントロールし、その変更を自動的にBrazeに反映させることができる。

## 統合

### ステップ1:Braze接続の設定

1. [Ketchインスタンスで](https://app.ketch.com)、**Data Systemsに**移動し、**Brazeを**選択する。次に、**New Connectionを**クリックする。
2. Braze接続に識別可能な名前を付け、APIベースの処理でこの接続を参照するために使用する。その接続のコードも作成されることに注意。このコードは、すべての接続で一意でなければならない。
3. ユーザーのIDマッピングを確認する。デフォルトでは、Ketchはユーザーのメールアドレス、またはBrazeの`external_id` 、ユーザーIDをマッピングする。
4. Braze APIキーを追加し、APIエンドポイントを指定する。この[APIエンドポイントは](https://www.braze.com/docs/api/basics/#endpoints)、あなたの組織が使用しているBrazeインスタンスに基づいていることに注意。

### ステップ2:サブスクリプションの環境設定を行う

1. **ポリシーセンター＞サブスクリプションと**進む。**ポリシーセンターの**下にサブスクリプションタブが表示されない場合は、マーケティング設定センターにアクセスできることを確認し、製品のこの部分にアクセスするための正しいアカウント権限を持っていることを確認する。
2. **新しいサブスクリプションを作成**] をクリックして、新しいトピックを作成する。各サブスクリプションには名前とコードがある。
3. サブスクリプショントピックを送信するチャネルを追加する。各チャネルはユーザーのマーケティング設定センターに表示される。また、Ketchユーザー設定センターに特定のオプトインやオプトアウトのシグナルをどのようにオーケストレーションさせたいかの詳細を追加することもできる。
4. オプトインおよびオプトアウト信号のオーケストレーションに使用するBraze接続を選択する。
5. Ketchユーザー設定を送信したいサブスクリプショングループのBraze`subscription_group_id` を入力する。



{% alert note %}
ユーザーのオプトインとオプトアウトのシグナルを収集し、オーケストレーションするためには、アイデンティティが適切に設定されていなければならない。Ketchでは、この統合のために、ユーザー・プレファレンス・シグナルをオーケストレーションする識別子としてメールを設定することを推奨している。
{% endalert %}


### ステップ3:アイデンティティを設定する

ユーザーがマーケティング設定センターを見ることができるのは、Ketchがユーザー's marketing preference identity. If Ketch cannot capture the user'の身元を正しく確認できた場合のみである。その場合、Ketchはユーザー設定を管理できないため、マーケティング設定ページはそのユーザーには表示されない。

1. マーケティング・プリファレンス・アイデンティティを設定するには、Ketchの**「設定」**ページに行き、「**アイデンティティ・スペース**」をクリックする。新しい ID スペースを作成するか、既存の ID スペースを編集して、その ID スペースをマーケティング優先 ID として割り当てる必要がある。プロパティに配備されたケッチ・タグが、そのアイデンティティ・スペースを適切に捉えていることを確認する。
2. **エクスペリエンス・サーバー**]、\[**プロパティ**] の順に選択し、目的のプロパティを編集する。そのプロパティのデータレイヤーで、カスタム・アイデンティティ・スペースを有効にする。次に、マーケティング・プリファレンスのIDをこのサイトでどのように取得するかを設定する。
3. IDスペースを設定したら、Ketchタグが配置されたWebサイトでユーザー設定センターを開封し、ユーザー設定センターが表示されるかどうかをテストする。
