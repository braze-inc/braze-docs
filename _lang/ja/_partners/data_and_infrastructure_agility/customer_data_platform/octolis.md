# オクトリス

> [Octolisは強力なデータアクティベーションプラットフォーム][0] (またはヘッドレスCDP) です。Octolisは、所有しているデータベースを基盤としているため、ビジネスツール内のデータを簡単に統合、準備、スコア付け、同期できます。

BrazeとOctolisの統合は、未加工のデータソースとBrazeの間のミドルウェアとして機能し、オンラインでもオフラインでも、さまざまなソースからデータを取得して統合することができます。
1. Eshop、CRM、POSシステムなどのソースからのデータを統合して組み合わせます。
2. ノーマライズとスコアリング
3. 計算されたフィールドとイベントを Braze にリアルタイムで同期

![][7]

## 前提条件

| 必要条件 | 説明 |
| ----------- | ----------- |
| オクトリスアカウント | このパートナーシップを利用するには、Octolis アカウントが必要です。 |
| Braze REST API キー | [**ユーザー.track 権限が付与された**][1] Braze REST API キー。<br><br> これは Braze ダッシュボードの **\[設定] > \[**API キー**]** から作成できます。 |
| Braze REST エンドポイント | [あなたの REST エンドポイント URL][2]。エンドポイントは、インスタンスの Braze URL によって異なります。 |
| Braze アプリキー | アプリ識別子キー。これは **Braze ダッシュボード > 設定の管理 > API キーにあります**。 |

## 統合

統合を開始する前に、接続、ソース、オーディエンス、同期に関する以下のセクションを参照してください。

詳細については、「Octolis [入門][4]」セクションを参照してください。

### ステップ1:Octolis をデータソースに接続する

Braze にデータを送信するには、少なくとも 1 [人のオーディエンス][5]作成しておく必要があります。オーディエンス複数のデータソースを組み合わせて準備ステップに適用し、計算フィールドを追加します。

これらのオーディエンスは、複数のデータソースに基づいて構築する必要があります。ソースは次のいずれかになります。
- Salesforce オブジェクト (取引先責任者、取引先など)
- Zendeskオブジェクト (チケット)
- SFTP 内のファイル (一部の連絡先を含む CSVファイル、イベントを含む JSON ファイル...)
- データベースのテーブル/ビュー。
- お客様のシステムの1つが、WebhookまたはAPI呼び出しを通じてレコードを送信します。

### ステップ2:Braze 送信先として追加

次に、Brazeを新しい送信先として設定するには、メイン画面内の現在の送信先の上部にある \[**\+ さらに追加**] を選択し、利用可能なビジネスツールから \[**Braze**] を選択します。

![][9]

選択したら、次の情報を入力します。

- お使いの Braze API キー:これは Braze ダッシュボードの **\[設定] > \[**API キー**]** から作成できます。
- タイムウィンドウ:オクトリスは、指定された期間にレート制限を適用します。
- リクエスト量:この期間内に実行できるリクエストの数。
- カスタム属性:ここで、Braze に送信する新しいフィールドとその形式（文字列、整数、浮動小数点）を指定し、そのうちの 1 つを同期に必須にする場合は \[**同期に必須**] にチェックマークを付けます。

![][10]

設定が完了すると、Brazeがホーム画面に新しい送信先として表示されます。

### ステップ3:新しい同期を作成

メニューから \[**同期**] をクリックし、右上の \[**同期を追加**] を選択します。以前に作成したオーディエンスから、選択したいオーディエンス選択します。
次に、**送信先としてBrazeを選択し**、データを送信するエンティティを選択します。

![][11]

### ステップ 4:出力設定を設定する

デフォルト、送信するすべての属性が Braze によって作成されますが、同期するフィールドのリストを文書化する必要があります。

![][12]{: style="max-width:75%;"}

設定フィールドの具体的な定義は次のとおりです。

| フィールド | 説明 |
| --- | --- |
| オーディエンスどこに同期させたいですか？ | レコードを作成または更新する Braze のエンティティ。 |
| レコードの識別にはどのフィールドが使用されますか? | レコードが Braze にすでに存在する場合、フィールドは Octolis を使用してレコードを識別します。 |
| 各レコードをどのくらいの頻度で送信しますか? | デフォルト、同期はすべてのインテグレーション (API、データベース、FTP) で段階的に行われます。つまり、前回の更新以降の新しい値のみが更新されます。必要に応じて、テーブル全体を定期的に送信することもできます。開始時に、Octolis はテーブル全体を送信します。 |
| どのフィールドを同期すべきか | オクトリスからBraze フィールドへのマッピング使用可能なすべてのフィールドのリストがドロップダウンメニューに表示されます。計算フィールドを Braze に送信するには、まず Braze エンティティ内に対応する列が作成されていることを確認する必要があります。 |
| オーディエンス同期させたいタイミングはいつですか？ | データが Braze に送信される方法：手動、リアルタイム、またはプログラミング。  |
| 録音中に同期... | Create: オプトインでは、Braze テーブルがマスターのままであることが重要です。フィールドが更新されたときに Octolis が同期をトリガーすることは望ましくありません。<br><br>Update: 一方、たとえば名フィールドの場合、顧客新しいエントリを入力するたびに Braze テーブルのフィールド更新できるようにする必要があります。 |
{: .reset-td-br-1 .reset-td-br-2}

## マルチキー重複排除

重複排除は、複数のソース、特にオンラインとオフラインのデータを調整する場合の大きな課題です。[Octolisの高度なノーコードモジュールを使用すると、複数のキーを使用して重複排除を行うことができます。][3]このモジュールは各マスターテーブルで使用できます。つまり、ロジックを各エンティティに適合させることができます。
