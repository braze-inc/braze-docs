# ゾータップ交響曲

Braze と Zoetap Symphony の統合により、リアルタイムのオーケストレーションを作成したり、メールやプッシュ通知キャンペーンを実行したりできます。

- Zeotapで苗字と名前を送信します。これに基づいて、ユーザーはBrazeを通じてパーソナライズされたメールを送信できます。
- カスタムイベントや購入イベントをZeotapを介してリアルタイムで送信し、これに基づいてユーザーはBraze内でキャンペーンリガーを作成して顧客をターゲットにすることができます。

{% alert note %}
メールマーケティングキャンペーンを作成するには、`Email Raw`未加工のメールをZeotapカタログにマッピングしてZeotapに登録します。
{% endalert %}

## 統合

このセクションでは、Braze と統合できる 2 つの方法について説明します。

### メソッド 1
この方法では、次のタスクを実行する必要があります。
1. Braze SDK をWeb サイトまたはアプリに統合します。
2. シンフォニーを通じてBraze ゼオタップを統合します。

- `User traits` **送信データタブの対応するBrazeフィールドにマッピングする必要があります**。`Event``Purchase`と属性をマップすると、Braze 内のイベントが重複することになります。
- Braze SDK `External ID` `User ID` の設定時に設定するマップ。

統合が正常に設定されると、Symphonyを通じてBrazeに送信されたカスタム属性に基づいてメールとプッシュ通知キャンペーンを作成できます。

### メソッド 2
この方法では、シンフォニーを通じてBraze ゼオタップを統合することができます。

- このメソッドは、アプリ内メッセージング、ニュースフィード、コンテンツカード、プッシュ通知などの Braze UI 機能をサポートしていません。
- Zeotapは、`hashed email` Zeotapカタログで利用可能なものをにマッピングすることを推奨しています. `External ID`

統合が正常に設定されると、Symphonyを通じてBrazeに送信されたカスタム属性に基づいてのみメールキャンペーンを作成できます。

## Braze へのデータフローと対応識別子

[データはユーザー](https://www.braze.com/docs/api/endpoints/user_data/post_user_track/)ートラックAPIを使用してZeotapからBrazeに流れます。次のポイントは、データフローをまとめたものです。

1. Zeotapは、ユーザープロファイル属性、カスタム属性、カスタムイベント、購入フィールドを送信します。
2. 関連するすべての Zeotap カタログフィールドを \[**送信するデータ**] タブの \[Braze] フィールドにマップします。
3. その後、データは Braze にアップロードされます。

さまざまな属性の詳細は、「[送信するデータ](#data-to-send-tab)」セクションにあります。

## デスティネーションセットアップ

Symphony でユーザーにフィルターを適用したり、条件を追加したりしたら、Braze の \[**宛先に送信**] でフィルターを有効にできます。新しいウィンドウが開き、送信先を設定できます。**使用可能な宛先のリストにある既存の送信先を使用するか、新しい宛先を作成できます**。

#### 新しい送信先を追加
次の手順を実行して、新しい送信先を追加します。
1. \[**新しい送信先を追加**] をクリックします。
2. **Braze** を検索してください。
3. **クライアント名**、**API キー**、**インスタンスを追加し**、送信先を保存します。

宛先が作成され、\[**使用可能な送信先**] で使用できるようになります。

#### ワークフローレベルの入力を追加
送信先を作成したら、次に説明するように、ワークフローレベルの入力を追加する必要があります。
1. 検索機能を使用して、利用可能な目的地のリストから目的地を選択します。
2. **クライアント名**、**API キー**、**インスタンスフィールドは**、送信先の作成時に入力した値に基づいて自動的に入力されます。
3. **このワークフローノード用に作成するオーディエンス名を入力します**。**これはカスタム属性として** Braze に送信されます。
4. \[**送信するデータ] タブでカタログから宛先へのマッピングを完了します**。マッピングの実行方法の詳細については、以下を参照してください。

#### \[送信データ] タブ
**送信データタブでは、ZeotapカタログのフィールドをBrazeに送信できるBrazeフィールドにマップできます**。マッピングは次のいずれかの方法で実行できます。
- **静的マッピング**-Zeotapが電子メール、電話、名、姓などの関連するBrazeフィールドに自動的にマッピングする特定のフィールドがあります。<br>
- **ドロップダウン選択**-Zeotapに取り込まれた関連フィールドをドロップダウンメニューのBrazeフィールドにマップします。<br>![Zeotapには、言語、都市、誕生日など、さまざまなユーザー特性が設定されています。][3]{: style="max-width:70%;"}<br>
- **カスタムデータ入力**-関連するZeotapフィールドマップされたカスタムデータを追加し、Brazeに送信します。<br>![Zeotapのユーザー特性として「ロイヤルティポイント」を選択します。][4]{: style="max-width:70%;"}

## Braze コンソールでデータを表示する

ワークフローで送信および公開する関連属性をマッピングすると、定義された基準に基づいてイベントが Braze に流れ始めます。Braze コンソールでは、電子メール ID または外部 ID で検索できます。

![][2]

Braze のユーザーダッシュボードのさまざまなセクションには、さまざまな属性があります。
- 「**プロファイル**」タブにはユーザー属性が含まれます。
- \[**カスタム属性**] タブには、ユーザー定義したカスタム属性が含まれています。
- **カスタムイベントタブには**、ユーザー定義したカスタムイベントが含まれています。
- **購入タブには**、ユーザーが一定期間に行った購入が表示されます。

## キャンペーン作成

ユーザーはBraze内でキャンペーンを作成し、リアルタイムで、またはスケジュールされた時間に基づいてユーザーをアクティベートできます。キャンペーンは、ユーザーが実行したアクション（カスタムイベント、購入）またはユーザー属性に基づいてトリガーできます。
