# Wyng

> [Wyng][0] を使用すると、適切なタイミングで消費者を引き付け、好みやその他のゼロパーティデータを収集し、リアルタイムでパーソナライズできるインタラクティブなデジタルエクスペリエンス（つまり、クイズ、プリファレンスセンター、プロモーション）を簡単に構築できます。

Braze と Wyng の統合により、Wyng エクスペリエンスから得たゼロパーティデータを活用して、Braze CampaignsとBraze Canvasでのインタラクションをパーソナライズできます。Wyng はユーザー設定センターも強化できるため、消費者はブランドと共有するデータや好み（コミュニケーションの好みを含む）コントロールできます。

## 前提条件

| 必要条件 | 説明 |
| ----------- | ----------- |
| Wyng アカウント | このパートナーシップを利用するには、Wyng アカウントが必要です。 |
| Braze REST API キー | `users.track`権限のあるBraze REST APIキー。<br><br> これは Braze ダッシュボードの **\[設定] > \[**API キー**]** から作成できます。 |
{: .reset-td-br-1 .reset-td-br-2}

## 統合

### ステップ1:Braze インテグレーションを接続する

Wyng で \[[**インテグレーション**][1]] に移動し、\[**追加**] タブを選択します。次に、**Braze** にカーソルを合わせ、\[**接続**] をクリックして統合を行います。

![Wyng プラットフォームの Braze パートナータイル][2]{: style="max-width:80%;"}

### ステップ2:Braze コネクターの設定

1. 開いた設定ウィンドウで、Braze REST API キーを入力します。
![認証情報のプロンプトがどのように表示されるかの画像, 写真。][4]{: style="max-width:80%;"}<br><br>
2. 次に、ドロップダウンを使用して、Brazeと共有したいWyng キャンペーンを選択します。![Braze コネクターの画像, 写真で、Braze と共有したい既存の Wyng キャンペーンを選択するよう求められます。][5]{: style="max-width:80%;"}<br><br>
3. 次に、サブスクリプション、属性オブジェクトとイベントオブジェクト、およびカスタムイベントを設定する必要があります。<br><br>
- **購読設定 (必須)**<br>
ユーザーをサブスクリプショングループにサブスクライブするには、「**サブスクリプションを追加**」をクリックし、サブスクリプショングループ名と ID を追加します。複数のグループ名と ID を追加するには、「**サブスクリプションを追加**」ボタンをもう一度クリックします。<br>![サブスクリプショングループ名と ID の入力を求める画像, 写真。][8]{: style="max-width:80%;"}<br><br>
- **ユーザートラック設定**<br>
\[**カスタムプロパティの追加**] をクリックして、`/users/track`エンドポイントに送信する属性イベントオブジェクトのペアを追加します。これを使用して、統合用に送信される各データトランザクションにハードコードされた属性値を追加します。複数のプロパティを追加するには、\[**カスタムプロパティの追加**] ボタンをもう一度クリックします。<br>![属性カスタムプロパティの追加を促す画像, 写真。][9]{: style="max-width:80%;"}<br><br>
- **カスタムイベントを送信**<br>
オプションで、「**カスタムイベント送信**」を有効にできます。有効になっている場合は、イベント名と対応するアプリ ID を含める必要があります。<br>![必要に応じてカスタムイベントを送信するように求める画像, 写真。][10]{: style="max-width:80%;"}<br><br>
4. 最後に、ユースケースに基づいて Wyng フィールドを Braze API フィールドにマッピングする必要があります。\[**フィールドを選択**] をクリックしてマップするフィールドを選択し、その後、**インテグレーションを保存します**。保存すると、これらのマップされたフィールドは \[**インテグレーション] > \[管理**] に表示されます。
![特定のBrazeフィールドにマップできるさまざまなWyng フィールドの例。][11]{: style="max-width:80%;"}
![利用可能な同期フィールドのリスト。][12]{: style="max-width:80%;margin-top:2px"}

### ステップ3:連携のテスト

Wyng で、Wyng キャンペーンでフォームの送信をテストします。メインの制作キャンペーンにレコードを追加したくない場合は、プレビューキャンペーンで送信することもできます。**インテグレーションダッシュボードに成功したトランザクションが表示されます**。

## このインテグレーションを使用する

データコネクターを設定すると、Wyng で作成してBrazeに追加したフィールドを他のデータフィールドと同じように使用して、キャンペーンのトリガーやオーディエンスのSegment 化、パーソナライズされたコンテンツのフィードを行うことができます。

用途は多岐にわたり、具体的な質問は \[contact@wyng.com] ][13] または特定のアカウントマネージャーにお問い合わせください。

## トラブルシューティング

### 送信失敗

送信に失敗した場合は、Braze にデータを送信するときに \[**ログを表示**] リンクをクリックして、失敗した送信と関連するエラーを確認してください。

![アクションヘッダーの下にある「ログを表示」リンク。][14]{: style="max-width:80%;"}

ログページには、送信の失敗、再試行回数、送信からのデータ、エラー、および送信を再プッシュするためのリンクが表示されます。

![送信に失敗した場合に表示される内容の例。][15]{: style="max-width:80%;"}

「**エラーを表示**」セクションには、エラーコードエラーの原因に関する追加情報が表示されます。その後、エラーコードを Braze と相互参照して原因を特定できます。

![Wyng プラットフォームに表示されるエラーログの例。][16]{: style="max-width:80%;"}

他にご不明な点がございましたら、Wyng サポート (\[support@wyng.com]][13]) にお問い合わせください。
