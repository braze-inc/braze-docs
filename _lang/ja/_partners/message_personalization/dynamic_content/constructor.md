# コンストラクタ

> [コンストラクタ](https://constructor.com/)は、AIや機械学習を利用して、eコマースや小売サイトのパーソナライズされた検索、おすすめ、閲覧体験を提供する検索・商品発見プラットフォームです。

ブレーズとコンストラクタの統合により、コンストラクタのオフサイト製品ディスカバリを使用して、カスタマイズされた製品レコメンドをBrazeメッセージで動的に生成し、配信することができます。

## ユースケース

- **放棄されたカートとオーダー後のフォローアップ**:ユーザーの行動とカートの内容に基づいて、動的な製品推奨事項を生成し、パーソナライズされた放棄されたカートのリマインダーまたは注文後の提案を送信します。
- **放棄されたカートアイテムの同様の製品推奨事項**:ユーザーのカートに残されているアイテムに類似した製品を提案し、それらを使用し続け、代替品を提供します。
- **最近見たアイテムリマインダー**:最近閲覧したがまだ購入していないアイテムについてユーザーに通知し、購入を完了するよう促す。
- **プロモーションキャンペーン**:季節的なセールや特別なオファーに対するユーザーの好みに合わせた、縮小された商品のお勧めで、パーソナライズされたプロモーションメッセージを配信します。
- **視覚的に類似した製品提案**:ユーザが最近表示したものと視覚的に類似した項目を推奨し、ユーザが好む関連オプションを見つけるのに役立ちます。

## 前提条件

| 必要条件 | 説明 |
|-------------|-------------|
| コンストラクタアカウント | このパートナーシップを活用するには、オフサイト検出サービスが有効になっているコンストラクタアカウントが必要です。 |

## 統合

コンストラクタオンボーディングチームと協力して、統合プロセスを完了します。ウェブサイトまたはその他の関連データソースからの行動データが、カスタマイズされた製品推奨を有効にするために利用可能であることを確認します。コンストラクタオンボーディングチームは、ブレーズメッセージで使用するために必要なHTML スニペットの設定にも役立ちます。

## コンストラクタのオフサイト検出API URL

コンストラクタのオフサイトディスカバリAPI URL を使用して、製品イメージをレンダリングし、ユーザーを適切な製品詳細ページに誘導できます。以下は、エンドポイント構造の内訳と、その使用方法の例です。

### 例

```html
<a href="https://offsite-discovery.cnstrc.com/v1/product/url?position=[position]&ui=[ui]&pod_id=[pod_id]&key=[key]&style_id=[style_id]&campaign_id=[campaign_id]" target="_blank">
  <img 
    src="https://offsite-discovery.cnstrc.com/v1/product/image?position=[position]&ui=[ui]&pod_id=[pod_id]&key=[key]&style_id=[style_id]&campaign_id=[campaign_id]"
    width="200" 
    border="0" 
    alt="Shop Now" 
  />
</a>
```

### パラメータ

| パラメータ | 説明 |
|-------------|-------------|
| `position` | 提案されたリスト内の特定の推奨項目のランク付けを参照します(`position = 2` など)。<br>![item.]({% image_buster /assets/img/constructor/constructor_position.png %})の位置ランキング |
| `ui` | 推奨結果をパーソナライズするために重要な、ユーザの識別子を表します。`ui` パラメータを顧客の`external_id` ブレーズに設定します。省略した場合、コンストラクタはユーザ固有の推奨事項ではなく一般的な推奨事項を返します。 |
| `pod_id` | 戦略を含むポッドのID、および推奨の検索ルール(たとえば、ベストセラー戦略を持つポッドは、パーソナライズされたベストセラーを生成します)。 |
| `key` | このカスタマーのコンストラクタインデックスキー。 |
| `style_id` | 製品カードに表示される画像を決定します。たとえば、異なる`style_ids` は、固有の製品カードイメージを表示します。 |
| `campaign_id` | メールキャンペーンの一意のID。 |
{: .reset-td-br-1 .reset-td-br-2 role="presentation" }

### オプション入力

| インプット | 説明 |
|-------------|-------------|
| `item_id` | シード項目を表します。代替、補完的、バンドルなど、品目ベースの戦略に必要です。たとえば、メールの最初の項目はシード項目で、後続の項目は代替項目として使用されます。 |
| `num_results` | メールに追加する製品の数。デフォルトは10、最大100 です。たとえば、`num_results = 3` は、3 つの推奨が追加されることを意味します。 |
