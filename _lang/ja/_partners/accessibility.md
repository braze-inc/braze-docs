# Braze でアクセシブルメッセージを作成する

> 意図せずとも障害者を排除したマーケティングコンテンツは、たとえ意図せずとも、何百万人もの人々があなたのブランドと交流することを妨げる可能性があります。マーケティングにおけるアクセシビリティとは、誰もが簡単にマーケティングを体験し、コミュニケーションを受け取って理解し、製品、サービス、またはブランドに投資したり、ファンになったりする機会を得られるようにすることです。メッセージをデザインするときは、時間をかけてすべての顧客がデザインにアクセスできるようにする方法を検討してください。

## アクセシビリティが重要な理由

- **使いやすさの向上:**アクセシビリティは、ユーザーがコンテンツをどのように利用するかを考えることになるため、アプリやサイトの使いやすさについて考えるよう促します。つまり、アクセシビリティは、障害を持つユーザーだけでなく、すべてのユーザーのオンライン体験を向上させることが多いということです。
- **市場リーチの拡大:**障害者の世界市場は10億人を超え、消費力は約7兆ドルです。
   > 「障害者の市場は大きく、世界の人口が高齢化するにつれて拡大しています。大規模な障害者市場がパープルポンドと呼ばれる英国では、障害者とその家族が毎年少なくとも2,490億ポンドを費やしています。米国では、障害者の年間自由裁量支出は2,000億ドルを超えています。障害者市場の世界的な推定額は約7兆ドルです。」<br>*ソース[W3C](https://www.w3.org/WAI/business-case/)*
- **法的リスクの最小化:**多くの国には、デジタルアクセシビリティを義務付ける法律があります。

## 考慮すべき障害領域

*このセクションは [ W3C から一部改作されています。多様な能力と障壁。](https://www.w3.org/WAI/people-use-web/abilities-barriers/)*

### ビジュアル

視覚障害は、片目または両眼の軽度または中程度の視力喪失から、両眼の重度または完全な視力喪失までさまざまです。特定の色に対する感度が低下したり、不足したり、明るい色に対する感度が上がったりする人もいます。

これらのユーザーがコンテンツを操作するには、次の機能が必要です。

- テキストサイズと画像を拡大または縮小する
- フォント、色、間隔の設定をカスタマイズ
- コンテンツの音声合成を聞く（つまり、スクリーンリーダーを使用する）
- 動画の音声解説を聞く
- 更新可能な点字を使用してテキストを読む

{% alert note %}
- [世界中で、少なくとも22億人が近距離または遠距離視力障害を患っています（WHOを参照）](https://www.who.int/news-room/fact-sheets/detail/blindness-and-visual-impairment)
- 男性の約12人に1人、女性200人に1人がある程度の色覚障害を患っており、世界では推定3億人がいます（[NHSを参照](https://www.nhs.uk/conditions/colour-vision-deficiency/)）
{% endalert %}

### ヒアリング

聴覚障害または聴覚障害には、片耳または両耳の軽度から中等度の聴覚障害が含まれます。オーディオコンテンツに関しては、部分的な聴覚障害でも問題になる可能性があります。

これらのユーザーは、お客様のコンテンツを理解するために、以下を頼りにしています。

- オーディオコンテンツのトランスクリプトとキャプション
- キャプションを表示し、テキストサイズやキャプションの色を調整するオプションを提供するメディアプレーヤー
- オーディオコンテンツの停止、一時停止、音量調整のオプション (システム音量とは無関係)
- バックグラウンドノイズとはっきり区別できる高品質のフォアグラウンドオーディオ

{% alert note %}
- 標準的な聴力検査によると、米国では12歳以上の8人に1人（13％、3,000万人）が両耳の難聴を患っています。
- 18歳以上のアメリカ人成人の約15％（3,750万人）が聴覚障害を報告しています（[NIHを参照](https://www.nidcd.nih.gov/health/statistics/quick-statistics-hearing)）
{% endalert %}

### 物理的

身体障害には、筋肉の制御や感覚の衰弱や制限、関節障害、動きを妨げる痛み、手足の欠損などがあります。

これらのユーザーは、(標準キーボードを使用していなくても) キーボードサポートを利用して機能を有効にします。これらのユーザーがコンテンツを操作するには、次のものが必要です。

- クリック可能な広い領域
- タスクを完了するのに十分な時間
- 現在のフォーカスを示す目に見えるインジケーター
- ページヘッダーやナビゲーションバーなどのコンテンツブロックをスキップするメカニズム

{% alert note %}
米国では約200万人が四肢を失った状態で生活しています（[Amputee Coalitionを参照](https://www.amputee-coalition.org/limb-loss-resource-center/resources-filtered/resources-by-topic/limb-loss-statistics/limb-loss-statistics/#1)）
{% endalert %}

### 認知

認知障害、学習障害、神経障害には、神経多様性障害や神経障害、および必ずしも神経学的ではない行動障害や精神障害が含まれます。神経系のあらゆる部分に影響を及ぼし、人々が情報をどれだけよく聞いたり、動かしたり、見たり、話したり、理解したりするかに影響を与える可能性があります。

個々のニーズに応じて、これらのユーザーは次のものを頼りにしています。

- 明確に構造化されたコンテンツ
- フォーム、ボタン、その他のコンテンツの一貫したラベル付け
- 予測可能なリンクターゲットと全体的な相互作用
- メニューや検索バーなど、さまざまなナビゲーション方法
- 点滅、点滅、その他の邪魔になるコンテンツをオフにする設定
- 画像がサポートするよりシンプルなテキスト


{% alert note %}
- 米国では5人に1人が学習と注意の問題を抱えています（[LDAを参照](https://ldaamerica.org/lda_today/the-state-of-learning-disabilities-today/#:~:text=LD%20Today,have%20learning%20and%20attention%20issues.)）
- [世界人口の約10〜20％がニューロダイバージェントであると考えられています（デロイトを参照）](https://www2.deloitte.com/us/en/insights/topics/talent/neurodiversity-in-the-workplace.html)
- 世界中で約100人に1人の子供が自閉症を患っています（[WHOを参照](https://www.who.int/news-room/fact-sheets/detail/autism-spectrum-disorders)）
{% endalert %}

## ベストプラクティス

### コンテンツ

- コンテンツはブランド名に訴えず、わかりやすい言葉で表現しましょう。米国の7年生の読解レベルまで書いてください。[Hemingway Appなどのリソースを使用して](https://hemingwayapp.com/)、テキストの読解レベルを確認できます。
- コンテンツを論理的に構成し、見出しが正しい階層に従っていることを確認してください。見出しレベルをスキップしないでください。
- コンテンツが長い場合は、中央揃えのテキストを避けてください。これは、認知障害または学習障害のあるユーザーにとっては読みにくい場合があります。2 行以上折り返すコンテンツは、左に揃える必要があります。
- デジタルデバイスで読みやすいサンセリフフォントを使用してください。
- テキストが切り捨てられないように、[必ず端末にテストメッセージを送信してコピーをテストしてください]({{site.baseurl}}/user_guide/engagement_tools/campaigns/testing_and_more/sending_test_push_notifications/)。メッセージが途切れると、コンテンツがユーザーに届かなくなるため、あなたとユーザーの両方が傷つきます。

### リンク

ユーザーを外部ページに誘導するなど、ナビゲーションにはリンクを使用してください。

{% alert tip %}
ボタンのような見た目と機能を持つものが必要な場合は、リンクをボタンのようにスタイリングするのではなく、常に実際のボタンを使用するようにしてください。<kbd>リンクとボタンは、普通のユーザーにも同じように感じられるかもしれません。リンクやボタンにマウスでカーソルを合わせ、マウスでクリックできます。ただし、ボタンとリンクのコントロールは異なります（たとえば、<kbd>ボタンはSpaceキーまたはEnterキーを押してアクティブにできますが</kbd>、<kbd>リンクはEnterキーでのみアクティブ化できます</kbd>）。リンクをボタンのようなスタイルにすると混乱を招く可能性があります。</kbd>
{% endalert %}

リンク先のユーザーをわかりやすく説明するリンクテキストを書いてください。スクリーンリーダーのユーザーは、コンテンツにざっと目を通す方法として、リンクからリンクへとスキップすることが多いので、リンクテキストがそのまま残るようにしてください。「ここをクリック」、「詳細」、「詳細はこちら」などのフレーズは、文脈から外れて読むと曖昧になるため、避けてください。

たとえば、天気予報を見るためのリンクを書く方法を考えてみましょう。

| 悪い  | より良い | 最高 |
| --- | --- | --- | 
| ここをクリック | 今日の天気にアクセスするにはここをクリック | 今日の天気 |
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3}

すべてのコンテンツと同様に、余分な単語をできるだけ少なくしてわかりやすくしてください。

### ボタン

フォームの送信やカルーセルの再生など、クリック可能なアクションにはボタンを使用します。

リンクテキストと同様に、ユーザーがボタンを押したときに発生するアクションを明確に説明するボタンテキストを記述します（たとえば、「続きを読む」ではなく「全文を読む」など）。ボタンのテキストが長すぎないかテストしてください。ボタンにすべてのテキストを表示できない場合は、テキストを新しい行に折り返すのではなく、省略記号で切り捨てられます。

### 画像

一部のユーザーは、マーケティングコンテンツ内の画像を見ることができません。アクセシビリティを考慮しないと、画像はすべてのユーザーが同じコンテンツを受け取るのを妨げる可能性があります。

#### 代替テキスト

代替テキストは、スクリーンリーダーやその他の支援技術がユーザーに提供する画像の内容の簡単な説明です。

- すべての画像について、画像の情報または機能を提供する代替テキストを記述します。
- [画像が装飾的である](https://www.w3.org/WAI/tutorials/images/decorative/)（コンテンツに追加されない）場合は、空のalt属性（`alt=""`）を使用してください。
- 代替テキストに「画像」や「画像」という単語を使用しないでください。

#### テキストの画像

スクリーンリーダーは画像に含まれるテキストを読み上げることができないため、テキストの画像は使用しないでください。テキストの画像も適切にサイズ変更されず、ユーザーのニーズや好みに合わせてカスタマイズすることもできません。実際のテキストでは、ユーザーは色やコントラストなどをカスタマイズしたり、品質を損なうことなくテキストのサイズを変更したりできます。テキストの画像を拡大すると、ピクセル化されて品質が低下し、読みにくくなります。

### ビデオ

動画にクローズドキャプションを付ける。視力障害のある人、騒がしい場所で視聴している人、ビデオ内の言語とは異なる言語を話す人に役立ちます。

### カラーコントラスト

十分な色のコントラストがあると、アクセシビリティがすぐに向上します。前景（テキスト）と背景色のコントラスト比は、[WCAG 2.1 AAレベルの要件を満たす必要があります](https://www.w3.org/TR/WCAG/#contrast-minimum)。

- 通常のテキスト (本文、ボタン、リンクなど) の 4.5:1 のコントラスト比
- 大きなテキスト（ヘッダーなど）のコントラスト比は 3:1

[WebAIM コントラストチェッカーツールを使用して](https://webaim.org/resources/contrastchecker/)、テキストが背景色に対して読みやすいかどうかを確認できます。

### フォーム

**長いフォームを小さなセクションにまとめる**<br>認知的負荷を軽減するには、長いフォームを小さなセクションに分割してください。これはチャンキングと呼ばれ、情報を利用しやすくするために使用される漸進的な開示パターンです。これはすべてのユーザーにメリットがありますが、特に認知障害のある人に役立ちます。

**ツールチップやその他のカーソルを合わせた状態で重要なコンテンツを非表示にしないでください**<br>カーソルを合わせた状態に含まれるコンテンツは見つけにくく、モバイルフレンドリーでもありません。また、拡大鏡を使うユーザーは、カーソルを合わせたときにしか表示されないコンテンツを見るのに苦労します。

**フィールド内の無効な文字をブロックしないでください**<br>特定の文字タイプがフォームフィールドに入力されないようにしないでください。ユーザーが好きなように入力できるようにして、何が問題なのかを示すエラーメッセージを表示する方がよいでしょう。キーボード入力をブロックすることは、支援技術のユーザーにとって特に問題になります。なぜなら、フォームに正しく入力されたかどうかを判断するためにインライン検証に大きく依存しているからです。

**明確なエラーメッセージを書く**<br>適切なエラーメッセージは、何が起こったのか、何が悪かったのか、どのように修正できるのかという3つの部分で構成されています。エラーメッセージは明確で理解しやすいものでなければなりません。簡単な言葉で話してみてください。派手な専門用語は必要ありません。
<br>

### カスタム HTML

メッセージにカスタム HTML を使用する場合:

- [セマンティック HTML](https://developer.mozilla.org/en-US/docs/Learn/Accessibility/HTML) を使用してください。つまり、ある要素を別の要素のようにスタイリングするのではなく、本来の目的に合った適切な HTML 要素を使用することになります。ほとんどの HTML 要素には、独自のアクセシビリティサポートが組み込まれています。
- HTML 内の言語属性を設定して、コンテンツの言語を識別します。スクリーンリーダーは、言語の発音と特性に基づいて、言語ごとに異なるサウンドライブラリを使用します。言語が指定されている場合、スクリーンリーダーは必要に応じて言語ライブラリを自動的に切り替えることができます。以下に例を示します。

{% raw %}
```html
<html lang="en-us">
```
{% endraw %}
