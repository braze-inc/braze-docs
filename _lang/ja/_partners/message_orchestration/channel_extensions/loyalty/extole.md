# エクストール

> SaaS企業である[Extole](https://www.extole.com/)社は、友達紹介マーケティングの業界リーダーであり、顧客獲得を増やすための効果的な紹介マーケティング・プログラムの作成と最適化を支援している。

BrazeとExtoleの統合により、Extoleの友達紹介プログラムや成長プログラムから顧客イベントや属性をBrazeに取り込むことができ、顧客獲得、エンゲージメント、ロイヤルティを高める、よりパーソナライズされたマーケティングキャンペーンを作成できるようになる。パーソナライズされた共有コードやリンクなど、Extoleのコンテンツ属性をBrazeのコミュニケーションに動的に取り込むこともできる。

## 前提条件

| 必要条件 | 説明 |
| ----------- | ----------- |
| エクストール口座 | このパートナーシップを利用するには、Extoleアカウントが必要である。 |
| Braze REST API キー | `users.track` 権限を持つ Braze REST API キー。これは、Brazeダッシュボードの**「設定**」>「**APIキー**」から作成できる。 |
| Braze API URL | あなたのBraze API URLは、あなたの[Brazeインスタンスに]({{site.baseurl}}/developer_guide/rest_api/basics/#endpoints)固有である。 |
{: .reset-td-br-1 .reset-td-br-2}

## ユースケース

以下の使用例では、ExtoleとBrazeの統合を活用する方法をいくつか紹介している。Extoleのインプリメンテーション・マネージャーやカスタマー・サクセス・マネージャーと協力して、貴社固有のニーズに合ったオプションを開発する。

- 紹介プログラムやエンゲージメントプログラムのカスタムイベントを活用して、BrazeのキャンペーンやCanvasをトリガーする。
- Extoleを使用したプログラムのデータを使用して、カスタムセグメント、ダッシュボード、レポートを作成する。
- Brazeでユーザーを自動的にマーケティングリストに登録または解除する

## 統合

以下のステップを完了し、統合を素早く立ち上げ、実行する。Extoleのインプリメンテーション・マネージャーとカスタマー・サクセス・マネージャーがこのプロセスをサポートし、あらゆる質問にお答えする。

### Brazeアカウントに接続する

1. My Extoleアカウントの[パートナー](https://my.extole.com/partners)ページでBrazeインテグレーションを選択する。
2. Brazeインテグレーションで、**Installを**選択してExtoleとBrazeの接続を開始する。
3. Braze REST APIキーから必要事項を入力する。 
4. Braze API URLを入力する。このURLは、Brazeアカウントがどのインスタンスにプロビジョニングされているかによって異なる。
5. Brazeに送信したいExtoleイベントを追加する。デフォルト・イベント、イベント・プロパティ、ユーザー属性は、[Extole Eventsの](https://dev.extole.com/docs/braze#extole-program-events)表に記載されている。
6. `FULFILLED` の状態以外に、Brazeに送りたい報酬の状態を追加する。利用可能な報酬状態の説明については、[Extole Rewardsの表を](https://dev.extole.com/docs/braze#extole-rewards)参照のこと。
7. Braze外部IDキーマッピングを選択する。これが、ExtoleがBrazeでユーザー・プロフィールを更新する方法だ。Brazeの外部IDキーをExtoleの`email_address` 、または`partner_user_id` 。より安全なので、`email_address` の代わりに`external_id` を使用することをお勧めする。
8. 設定を保存して接続を完了する。これで、ExtoleのイベントがBrazeのアカウントに流れるようになった。

| `extole_shared` | 参加者が自分の紹介リンクを友人にシェアする。 | イベント名  <br>開催時間  <br>パートナー（エクストール）  <br>external ID  <br>ファネル（支持者または友人）  <br>プログラム  <br>シェアチャンネル | メール <br>名 <br>姓 |

### エクストレイルの報酬

デフォルトでは、Extoleは`FULFILLED` 状態の報酬イベントをBrazeに送信するので、BrazeキャンペーンまたはCanvasを介して報酬通知をトリガーできる。その他の報酬状態については、以下の表を参照のこと。

| 報酬国 | 説明 | イベントプロパティ | ユーザー属性 |
| ----------- | ----------- | ----------- | ----------- |
| `FULFILLED` | デフォルトの状態である。報酬は、Extole報酬サプライヤーによって価値（クーポンやギフトカードなど）が割り当てられている。 | メール <br>額面  <br>クーポンコード  <br>額面タイプ  | メール <br>名  <br>姓 |
| `EARNED` | 報酬が作成され、その人物に関連付けられた。 | メール <br>額面  <br>クーポンコード  <br>額面タイプ  | メール <br>名  <br>姓 |
| `SENT` | 報酬は達成され、電子メールまたはデバイスで受取人に送信された。 | メール <br>額面  <br>クーポンコード  <br>額面タイプ  | メール <br>名  <br>姓 |
| `REDEEMED` | 報酬は、Extoleに送信されたコンバージョンまたは償還イベントで証明されるように、受領者によって使用された。| メール <br>額面  <br>クーポンコード  <br>額面タイプ  | メール <br>名  <br>姓 |
| `FAILED` | 問題が発生したため、報酬の発行または送付ができず、注意が必要である。 | メール <br>額面  <br>クーポンコード  <br>額面タイプ  | メール <br>名  <br>姓 |
| `CANCELED` | 報酬は無効化され、インベントリに戻る。 | メール <br>額面  <br>額面タイプ  | メール <br>名  <br>姓 |
| `REVOKED` | 達成された報酬は無効となった。例えば、Extole社はサプライヤーにギフトカードを要求したが、そのカードが誤って送られてきたと判断した。サプライヤーが報酬の取り消しを支持した場合、Extoleは資金の払い戻しを要求し、報酬は無効となる。 | メール <br>額面   <br>額面タイプ  | メール <br>名  <br>姓 |

## カスタマイズ

### Brazeでユーザーを検索し、作成する

Extoleが外部ID（ユーザーID）を持っていない新規メールやSMS購読などの特定のユースケースでは、ExtoleはBraze[Export user profile by identifierエンドポイントを]({{site.baseurl}}/api/endpoints/export/user_data/post_users_identifier/)使用してユーザーの識別子をチェックできる。ユーザーがBrazeに存在する場合、Extoleはプロフィール属性を追加・更新する。リクエストがユーザープロファイルを返さない場合、Extoleは`/users/track` エンドポイントを使用して、ユーザーのメールアドレスをエイリアス名とするユーザーエイリアスを作成する。

## この統合を使う

アカウントを接続すると、何もしなくても自動的にExtoleからBrazeにイベントが流れ始める。Brazeに送信されるイベントのライブビューは、Extoleのトラブルシューティング用Outbound Webhook Centerで見ることができる。
