# 2024年8月20日リリース

## キャンバス・コンテクストへのタイムゾーンの一貫性の標準化

{% multi_lang_include release_type.md release="Early access" %}

[キャンバスコンテキストステップの早期アクセスに]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/context)参加している場合、アクションベースのキャンバスのトリガーイベントプロパティからdatetimeタイプを持つすべてのタイムスタンプは、常に[UTCに](https://en.wikipedia.org/wiki/Coordinated_Universal_Time)正規化される。これについては、「[タイムゾーンの一貫性の標準化]({{site.baseurl}}/user_guide/engagement_tools/canvas/canvas_components/context#time-zone-consistency-standardization)」を参照のこと。

## データの柔軟性

### セルフサービスカスタムドメイン

{% multi_lang_include release_type.md release="General access" %}

[セルフサービスカスタムドメインにより]({{site.baseurl}}/user_guide/message_building_by_channel/sms_mms_rcs/link_shortening/custom_domains/)、Brazeダッシュボードから直接、SMS、RCS、WhatsApp用のカスタムドメインを設定・管理することができる。最大10個のカスタムドメインを簡単に追加、監視、管理できる。

### セグメントファネル統計

[View funnel statisticsを]({{site.baseurl}}/user_guide/engagement_tools/segments/creating_a_segment/#viewing-funnel-statistics)選択すると、フィルターグループの統計が表示され、追加されたフィルターがセグメンテーション統計にどのように影響するかを見ることができる。その時点までのすべてのフィルターによってターゲットにされたユーザーの推定カウントとパーセンテージが表示される。一度フィルターグループの統計が表示されると、フィルターを変更するたびに自動的に更新される。 

### プッシュ通知のための`/campaigns/details` エンドポイントの新しいレスポンスフィールド

`messages` 、プッシュ通知のレスポンスに2つの新しいフィールドが追加された：

- `image_url`:Android通知画像、iOS通知画像、またはWebプッシュアイコン画像の画像URL。
- `large_image_url`:Android ChromeとWindowsのWebプッシュアクション用のWeb通知画像URL。

### PIIフィールドを定義する

[特定のフィールドをPIIフィールドとして]({{site.baseurl}}/user_guide/administrative/app_settings/company_settings/security_settings#view-pii)選択および[定義する]({{site.baseurl}}/user_guide/administrative/app_settings/company_settings/security_settings#view-pii)ことは、ユーザーがダッシュボードで表示できる内容に影響するだけであり、当該PIIフィールド内のユーザーデータの取り扱いには影響しない。

ダッシュボードの設定を、[データのリテンションに関する]({{site.baseurl}}/api/data_retention/)ものを含め、御社に適用されるプライバシー規制やポリシーに合わせるために、法務チームに相談する。

### レポートビルダーのダウンロードリンクを共有する

**共有]**を選択し、[**リンクを共有**]または[**メールの送信またはスケジュール]を**選択することで、レポートへの[ダッシュボードリンクを共有]({{site.baseurl}}/user_guide/analytics/reporting/report_builder/#sharing-a-report)することができる。

## 創造性を引き出す

### ドラッグ＆ドロップメール用のカスタムヘッドタグ

`<head>` タグを使って、メールメッセージにCSSやメタデータを追加しよう。例えば、これらのタグを使ってスタイルシートやファビコンを追加することができる。リキッドは`<head>` タグでサポートされている。

## 強力なチャネル

### ファジー・アウト・アウトのベストプラクティス

[ベストプラクティスのセクションを]({{site.baseurl}})追加し、ファジーなオプトアウトメッセージを慎重に設定し、明確でコンプライアンスに準拠した、サブスクライバーにとって好ましい体験を提供できるようにした。

### WhatsApp Flows

{% multi_lang_include release_type.md release="Early access" %}

[WhatsApp Flowsは]({{site.baseurl}}/whatsapp_flows/)既存のWhatsAppチャネルを強化し、インタラクティブでダイナミックなメッセージング体験を実現する。 

### WhatsApp インバウンド製品に関する質問

ユーザーは、製品やカタログのメッセージに対して、[製品に関する質問を]({{site.baseurl}}/user_guide/message_building_by_channel/whatsapp/whatsapp_campaign/product_messages/#receiving-inbound-product-questions)することができる。これらはインバウンドメッセージとして届き、アクションパスでソートすることができる。

さらに、Brazeはこれらの質問から製品IDとカタログIDを抽出するので、回答をオートメーション化したり、別のチーム（顧客サポートなど）に質問を送信したい場合は、これらの詳細を含めることができる。

## AI と ML のオートメーション

### BrazeAI™の新しいユースケース記事

BrazeAI™を最大限に活用するための新しいユースケースを追加した。これらのガイドでは、AIをエンゲージメント戦略に応用する実践的な方法を紹介している：

- 解約予測解約リスクのある顧客を識別子し、早期にアクションを起こす。
- 予測イベントユーザーの主要なアクションを予測し、リアルタイムでエクスペリエンスを形成する。
- [レコメンデーション]({{site.baseurl}}/user_guide/brazeai/recommendations/use_case ):顧客行動に基づいて、より関連性の高いコンテンツや商品を提供する。

### MCPサーバー

{% multi_lang_include release_type.md release="Beta" %}

[BrazeのMCPサーバーは]({{site.baseurl}}/user_guide/brazeai/mcp_server/)、セキュアで読み取り専用の接続で、ClaudeやCursorのようなAIツールがPIIではないBrazeのデータにアクセスし、データを変更することなく質問に答えたり、傾向を分析したり、インサイトを提供したりできる。

## SDKのアップデート

以下のSDKアップデートがリリースされた。破壊的な更新は下記のとおりです。その他すべての更新は、対応する SDK の変更履歴をご確認ください。

- [Swift SDK 11.0.1](https://github.com/braze-inc/braze-swift-sdk/blob/main/CHANGELOG.md)
    - `BrazeSDKAuthDelegate.braze(_:sdkAuthenticationFailedWithError:)` の機能を拡張し、"Optional "認証エラーのトリガーとする。
        - デリゲートメソッド`BrazeSDKAuthDelegate.braze(_:sdkAuthenticationFailedWithError:)` は、"Required "と "Optional "の両方の認証エラーに対してトリガーされるようになった。
        - Required」SDK認証エラーのみを処理したい場合は、このデリゲートメソッドの実装内部に、`BrazeSDKAuthError.optional` がfalseであることを保証するチェックを追加する。
    - イネーブルメントが有効な場合に`Braze.Configuration.sdkAuthentication` の使い方を修正した。
        - 以前は、このコンフィギュレーションの値はSDKによって消費されず、トークンが存在する場合は常にリクエストに添付されていた。
        - このコンフィギュレーションがイネーブルメントの場合、SDKは発信ネットワークリクエストにのみSDK認証トークンをアタッチする。
    - `Braze.FeatureFlag` の全プロパティと`Braze.Banner` の全プロパティのセッターは`private` になった。これらのクラスのプロパティは読み取り専用になった。
    - バージョン`11.4.0` で非推奨となった`Braze.Banner.id` プロパティを削除した。
        - 代わりに、`Braze.Banner.trackingId` 、バナーのキャンペーントラッキングIDを読み取る。
- [React Native SDK 15.0.0](https://github.com/braze-inc/braze-react-native-sdk/blob/master/CHANGELOG.md)
    - ネイティブAndroid SDKのバージョンバインディングを[Braze Android SDK 36.0.0から37.0.0に](https://github.com/braze-inc/braze-android-sdk/compare/v36.0.0...v37.0.0#diff-06572a96a58dc510037d5efa622f9bec8519bc1beab13c9f251e97e657a9d4ed)更新。
    - ネイティブSwift SDKのバージョンバインディングを[Braze Swift SDK 12.0.0から13.0.0に](https://github.com/braze-inc/braze-swift-sdk/compare/12.0.0...13.0.0#diff-06572a96a58dc510037d5efa622f9bec8519bc1beab13c9f251e97e657a9d4ed)更新。
        - `sdkAuthenticationError` イベントが "Required "と "Optional "の両方の認証エラーに対してトリガーされるようになった。
- [Xamarin SDK 4.0.1](https://github.com/braze-inc/braze-xamarin-sdk/blob/7.0.0/CHANGELOG.md)
    - iOSとAndroidのバインディングに.NET 9.0のサポートを追加した。
        - これにより、。NET 7.0 のサポートが削除されます。
        - これには[iOS 12.2以上が](https://learn.microsoft.com/en-us/dotnet/maui/whats-new/dotnet-9?view=net-maui-9.0)必要だ。
    - Android バインドを[Braze Android 30.4.0 から32.0.0](https://github.com/braze-inc/braze-android-sdk/compare/v32.0.0...v37.0.0#diff-06572a96a58dc510037d5efa622f9bec8519bc1beab13c9f251e97e657a9d4ed) に更新しました。
